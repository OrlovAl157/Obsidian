## Основные понятия

### Collection (Коллекция)
```
Collection — это объект, который хранит в себе набор значений 
одного или нескольких типов и позволяющий обращаться к своим значениям
```

### Sequence (Последовательность)
```
Sequence — это коллекции, элементы которого находятся в строгом 
порядке и имеют индексы
```

### Iterable (Итерируемый объект)
```
Iterable — это интегрируемый объект. Итерируемый объект это тот, 
на основе которого можно создать iterator
```

### Iterator (Итератор)
```
Iterator — это объект, который выдает следующее свое значение 
в результате применения к нему функции next() и возбуждающий 
исключение StopIteration если применена функция next() к пустому iterator
```

### Range object
```
Range object — это особый неизменяемый объект( можно использовать как ключ в словаре), который является чем-то средним 
между sequence и iterator. К его элементам можно обращаться по индексам. 
Также доступны срезы. Эти два свойства роднит его с sequence.

Range obj не хранит в себе все значения, а хранит только следующее 
на очереди, то есть каждый элемент его вычисляется лениво, что роднит 
его с iterator
```

## Схема связей

```
                              range object
                                   |
                                   | (особый объект между
                                   |  sequence и iterator)
                                   ↓
       set ──┐
             │
      dict ──┼──→ Collections ─────→ Iterable ←──────→ iterator
             │         ↑                ↑              (функция next())
       list ─┤         │                │
             │         │                │
      tuple ─┤    Sequence              │
             │         ↑                │
     string ─┘         │                │
                       │                │
                  (упорядоченные    (на основе любого
                   с индексами)      итерируемого объекта
                                     можно создать итератор)
```

---

## Иерархия типов

```
Collections (коллекции)
    │
    ├── Sequence (последовательности)
    │      ├── list   (список)
    │      ├── tuple  (кортеж)
    │      └── string (строка)
    │
    ├── set  (множество)
    └── dict (словарь)


Iterable (итерируемые объекты)
    │
    ├── Collections (все коллекции итерируемые)
    ├── range object (специальный итерируемый объект)
    └── iterator (сам итератор тоже итерируемый)


Iterator (итераторы)
    │
    └── создаются из любого итерируемого объекта
        с помощью функции iter()
```

---

## Поток работы с итераторами

```
Создание итератора:
    
    iterable_obj (list, tuple, string, etc.)
          │
          │ iter()
          ↓
       iterator
          │
          │ next()
          ↓
    следующий элемент
          │
          │ next()
          ↓
    следующий элемент
          │
          │ next() (когда элементы закончились)
          ↓
    StopIteration (исключение)
```

---

## Пример использования

```python
# Iterable объект
my_list = [1, 2, 3]

# Создание iterator
my_iter = iter(my_list)

# Получение элементов
print(next(my_iter))  # → 1
print(next(my_iter))  # → 2
print(next(my_iter))  # → 3
print(next(my_iter))  # → StopIteration
```

---

## Ключевые отличия

```
Collection vs Sequence:
    Collection          → любой набор значений
    Sequence            → упорядоченный набор с индексами

Iterable vs Iterator:
    Iterable            → можно перебирать (for loop)
    Iterator            → выдает элементы по одному (next())

Range object:
    ├── Как Sequence    → доступ по индексам, срезы
    └── Как Iterator    → ленивое вычисление значений
```
