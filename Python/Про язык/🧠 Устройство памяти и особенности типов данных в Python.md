## 🖥️ Компьютерная память

### Типы памяти

|Тип|Назначение|Особенности|
|---|---|---|
|**Внутренняя**|Быстрый доступ, текущие задачи|Высокая скорость, ограниченный объем|
|**Внешняя**|Долговременное хранение|Большой объем, медленный доступ|

### Внутренняя память

|Тип|Название|Размер|Скорость|Энергозависимость|

|**ROM**|Постоянная память|Малый|Средняя|Нет ❌|

|Тип|Скорость|Цена|Надежность|Объем|

## 🐍 Механизм памяти в Python

### Управление памятью

|Тип управления|Языки|Особенности|
|---|---|---|
|**Ручное**|C, C++, Pascal|Программист сам выделяет и освобождает память|
|**Автоматическое**|Python, Java, C#, JavaScript|Память освобождается автоматически|

### Аллокатор памяти в Python

Python использует **трёхуровневую иерархию** для управления памятью:

```
┌─────────────────────────────────────────┐
│           АРЕНА (256 КБ)                │
│  ┌────────────────────────────────┐     │
│  │     ПУЛ (4 КБ)                 │     │
│  │  ┌──────┬──────┬──────┬─────┐  │     │
│  │  │ БЛОК │ БЛОК │ БЛОК │ ... │  │ ... │
│  │  │ 8 Б  │ 8 Б  │ 8 Б  │     │  │     │
│  │  └──────┴──────┴──────┴─────┘  │     │
│  └────────────────────────────────┘     │
└─────────────────────────────────────────┘
```

### Размеры объектов и распределение

|Размер объекта|Способ выделения памяти|
|---|---|
|**≤ 512 байт**|Через аллокатор (Блок → Пул → Арена)|
|**> 512 байт**|Напрямую от операционной системы|

### Таблица размеров блоков

|Запрос памяти (байт)|Размер блока (байт)|Индекс класса|
|---|---|---|
|1-8|8|0|
|9-16|16|1|
|17-24|24|2|
|25-32|32|3|
|33-40|40|4|
|41-48|48|5|
|...|...|...|
|505-512|512|63|

**Пример:**

- Запрос 6 байт → блок 8 байт
- Запрос 42 байта → блок 48 байт
- Запрос 505 байт → блок 512 байт

### Состояния пулов

|Состояние|Описание|Использование|
|---|---|---|
|**used**|Частично заполнен|Некоторые блоки заняты, некоторые свободны|
|**full**|Полностью заполнен|Все блоки заняты данными|
|**empty**|Пустой|Все блоки свободны, доступны для записи|

## 📦 Переменные и объекты

### Переменные в C/C++ vs Python

#### В C/C++: Переменная = Ящик с данными

```
┌──────────────────────┐
│  int num = 101;      │
└──────────────────────┘

Память:
┌─────────┬─────────┐
│ Адрес   │ 0x7f1   │
│ num     │ 101     │
└─────────┴─────────┘

num = 102;  // Перезаписывает значение в том же адресе

┌─────────┬─────────┐
│ Адрес   │ 0x7f1   │  ← Адрес НЕ изменился
│ num     │ 102     │  ← Значение изменилось
└─────────┴─────────┘
```

#### В Python: Имя = Ярлык на объект

```python
num = 101
```

```
┌────────────────────────────────────┐
│         PyObject                   │
│  ┌──────────────────────────┐      │
│  │ Type: int                │      │
│  │ Value: 101               │      │
│  │ Reference Count: 1       │      │
│  └──────────────────────────┘      │
└────────────────────────────────────┘
         ↑
         │
      [num] ← Имя указывает на объект
```

```python
num = 102  # Создаётся НОВЫЙ объект!
```

```
┌─────────────────┐         ┌─────────────────┐
│   PyObject      │         │   PyObject      │
│  Type: int      │         │  Type: int      │
│  Value: 101     │         │  Value: 102     │
│  RefCount: 0    │         │  RefCount: 1    │
└─────────────────┘         └─────────────────┘
                                     ↑
                                     │
                                  [num] ← Указывает на НОВЫЙ объект
```

### Ключевое различие

|Аспект|C/C++|Python|
|---|---|---|
|**Что такое переменная**|Ящик в памяти|Ярлык (имя) на объект|
|**Присваивание**|Изменяет значение в ящике|Перевешивает ярлык на другой объект|
|**Адрес**|Постоянный|Меняется при переприсваивании|
|**Копирование**|Создаёт новый ящик с копией|Создаёт новый ярлык на тот же объект|

### Структура PyObject

Каждый объект в Python имеет структуру:

|Поле|Назначение|
|---|---|
|**Type**|Тип объекта (int, str, list, ...)|
|**Value**|Значение объекта|
|**Reference Count**|Количество ссылок на объект|

**Пример:**

```python
num = 101
num1 = num
```

```
┌────────────────────────────┐
│       PyObject             │
│  Type: int                 │
│  Value: 101                │
│  Reference Count: 2        │  ← Два имени указывают на объект
└────────────────────────────┘
         ↑         ↑
         │         │
      [num]     [num1]
```

## 🔑 Функция id() и оператор is

### Функция id()

**Назначение:** Возвращает уникальный идентификатор объекта (адрес в памяти в CPython)

```python
name = 'Timur'
print(id(name))      # 2017717924208 (адрес в памяти)
print(hex(id(name))) # 0x1d5a8b6f670 (в 16-ричной системе)
```

### Оператор == vs is

|Оператор|Что проверяет|Пример|
|---|---|---|
|`==`|Равенство **значений**|`[1, 2] == [1, 2]` → `True`|
|`is`|Идентичность **объектов** (одинаковый `id`)|`[1, 2] is [1, 2]` → `False`|

**Примеры:**

```python
# Разные объекты с одинаковым содержимым
nums1 = [1, 2, 3]
nums2 = [1, 2, 3]

print(nums1 == nums2)  # True  (значения равны)
print(nums1 is nums2)  # False (разные объекты)
print(id(nums1))       # 2303843075456
print(id(nums2))       # 2303886617792 (разные адреса!)
```

```python
# Один объект, два имени
nums1 = [1, 2, 3]
nums2 = nums1  # Присваивание НЕ создаёт копию!

print(nums1 == nums2)  # True  (значения равны)
print(nums1 is nums2)  # True  (один и тот же объект)
print(id(nums1))       # 2002273207680
print(id(nums2))       # 2002273207680 (одинаковые адреса!)
```

### Проверка на None

```python
value = None

# ✅ Правильно
if value is None:
    print('Пусто')

# ❌ Не рекомендуется
if value == None:
    print('Пусто')
```

## ⚡ Интернирование объектов

**Интернирование** — оптимизация, при которой Python хранит только одну копию одинаковых неизменяемых объектов.

### Интернирование целых чисел

|Диапазон|Интернируется?|Причина|
|---|---|---|
|**-5 до 256**|✅ Да|Часто используемые числа|
|**< -5 или > 256**|❌ Нет|Редко используются|

**Примеры:**

```python
# Числа в диапазоне -5 до 256
num1 = 100
num2 = 100

print(num1 is num2)  # True  (один объект!)
print(id(num1))      # 1979656244560
print(id(num2))      # 1979656244560 (одинаковые!)
```

```python
# Числа вне диапазона
num3 = 1000
num4 = 1000

print(num3 is num4)  # False (разные объекты)
print(num3 == num4)  # True  (но значения равны)
```

### Интернирование строк

|Условие|Python 3.7|Python 3.8+|
|---|---|---|
|**Длина**|≤ 20 символов|≤ 4096 символов|
|**Символы**|ASCII буквы, цифры, `_`|ASCII буквы, цифры, `_`|
|**Специальные символы**|❌ Не интернируются|❌ Не интернируются|

**Примеры:**

```python
# Строки интернируются
s1 = 'beegeek'
s2 = 'beegeek'
s3 = 'bee' + 'geek'

print(s1 is s2)     # True
print(s1 is s3)     # True
print(id(s1))       # 2846528331184
print(id(s2))       # 2846528331184
print(id(s3))       # 2846528331184
```

```python
# Строки НЕ интернируются (есть спецсимвол !)
s1 = 'beegeek!'
s2 = 'beegeek!'

print(s1 is s2)     # False
print(id(s1))       # 2846528331312
print(id(s2))       # 2846528331440
```

### Функция sys.intern()

Принудительно интернирует любую строку:

```python
import sys

s1 = sys.intern('степик!')
s2 = sys.intern('степик!')

print(s1 is s2)  # True (интернированы!)
```

**Преимущества интернирования:**

- 💾 Экономия памяти
- ⚡ Быстрое сравнение (проверка адресов вместо посимвольного сравнения)

## 🔄 Изменяемые и неизменяемые типы

### Таблица типов данных

|Категория|Типы|Можно изменить содержимое?|
|---|---|---|
|**Неизменяемые**|`int`, `float`, `bool`, `str`, `tuple`, `frozenset`|❌ Нет|
|**Изменяемые**|`list`, `dict`, `set`, `bytearray`|✅ Да|

### Изменение объекта vs Присваивание

#### Присваивание (создаёт новый объект)

```python
nums = [1, 2, 3]
print(id(nums))  # 2130653012352

nums = [1, 2, 3, 4]  # Новый объект!
print(id(nums))      # 2130653007680 (другой адрес!)
```

```
До:  [nums] → [1, 2, 3]
После: [nums] → [1, 2, 3, 4]  (НОВЫЙ объект)
```

#### Изменение объекта (тот же объект)

```python
nums = [1, 2, 3]
print(id(nums))  # 2675065348480

nums.append(4)   # Изменяем существующий объект
print(id(nums))  # 2675065348480 (тот же адрес!)
```

```
До:    [nums] → [1, 2, 3]
После: [nums] → [1, 2, 3, 4]  (тот же объект, изменённый)
```

### Важное следствие

```python
nums1 = [1, 2, 3]
nums2 = nums1  # nums2 указывает на тот же объект!

nums1.append(4)

print(nums1)  # [1, 2, 3, 4]
print(nums2)  # [1, 2, 3, 4]  ← Тоже изменился!
```

```
[nums1] ──┐
          ├──→ [1, 2, 3, 4]
[nums2] ──┘
```

### Операторы + и += для списков

|Оператор|Что делает|Создаёт новый объект?|
|---|---|---|
|`+`|Создаёт новый список|✅ Да|
|`+=`|Изменяет текущий список (как `extend()`)|❌ Нет|

```python
# Оператор +
nums1 = [1, 2, 3]
nums2 = nums1
nums1 = nums1 + [4, 5]  # Новый объект!

print(nums1)  # [1, 2, 3, 4, 5]
print(nums2)  # [1, 2, 3]  ← Остался неизменным
```

```python
# Оператор +=
nums1 = [1, 2, 3]
nums2 = nums1
nums1 += [4, 5]  # Изменяет текущий список!

print(nums1)  # [1, 2, 3, 4, 5]
print(nums2)  # [1, 2, 3, 4, 5]  ← Тоже изменился!
```

## 📋 Копирование объектов

### Типы копирования

|Способ|Что копирует|Вложенные объекты|Модуль|
|---|---|---|---|
|**Присваивание** `=`|Только ссылку|—|Встроенное|
|**Поверхностное** (shallow)|Сам объект|Только ссылки|`copy.copy()`|
|**Глубокое** (deep)|Сам объект|Полные копии|`copy.deepcopy()`|

### Присваивание (НЕ копирование!)

```python
data1 = [1, 2, 3]
data2 = data1  # Не создаёт копию!

data1.append(4)
print(data2)  # [1, 2, 3, 4]  ← Изменился!
```

```
[data1] ──┐
          ├──→ [1, 2, 3, 4]
[data2] ──┘
```

### Поверхностное копирование (Shallow Copy)

**Создаёт новый объект, но вложенные элементы остаются ссылками.**

```python
import copy

data1 = [[1, 2], [3, 4]]
data2 = copy.copy(data1)

data1[0].append(99)

print(data1)  # [[1, 2, 99], [3, 4]]
print(data2)  # [[1, 2, 99], [3, 4]]  ← Тоже изменился!
```

```
[data1] → [[1,2,99], [3,4]]
              ↑        ↑
[data2] → [   •,       •   ]  ← Ссылки на те же вложенные списки
```

### Глубокое копирование (Deep Copy)

**Создаёт полностью независимую копию всей структуры.**

```python
import copy

data1 = [[1, 2], [3, 4]]
data2 = copy.deepcopy(data1)

data1[0].append(99)

print(data1)  # [[1, 2, 99], [3, 4]]
print(data2)  # [[1, 2], [3, 4]]  ← Не изменился!
```

```
[data1] → [[1,2,99], [3,4]]

[data2] → [[1,2], [3,4]]  ← Полностью независимая копия
```

### Способы создания копий

#### Поверхностное копирование

|Способ|Синтаксис|Типы данных|
|---|---|---|
|Метод `.copy()`|`new = old.copy()`|`list`, `dict`, `set`|
|Функция `copy.copy()`|`new = copy.copy(old)`|Любые|
|Срез `[:]`|`new = old[:]`|`list`|
|Конструктор|`new = list(old)`|Любые|

```python
import copy

data = [1, 2, 3]

# Все создают поверхностную копию
copy1 = data.copy()
copy2 = copy.copy(data)
copy3 = data[:]
copy4 = list(data)
```

#### Глубокое копирование

```python
import copy

data = [[1, 2], [3, 4]]
deep_copy = copy.deepcopy(data)
```

## 🗑️ Очистка памяти

Python использует **два механизма** очистки памяти:

### 1. Подсчёт ссылок (Reference Counting)

**Принцип:** Каждый объект хранит счётчик ссылок. Когда счётчик = 0 → объект удаляется.

```python
nums = [1, 2, 3]  # RefCount = 1
```

```
┌─────────────────────┐
│    PyObject         │
│  Type: list         │
│  Value: [1, 2, 3]   │
│  RefCount: 1    ←── │
└─────────────────────┘
         ↑
         │
      [nums]
```

```python
nums1 = nums  # RefCount = 2
```

```
┌─────────────────────┐
│    PyObject         │
│  Type: list         │
│  Value: [1, 2, 3]   │
│  RefCount: 2    ←── │
└─────────────────────┘
         ↑     ↑
         │     │
      [nums] [nums1]
```

#### Увеличение счётчика ссылок

|Действие|Пример|
|---|---|
|Создание объекта|`x = [1, 2, 3]`|
|Присваивание|`y = x`|
|Передача в функцию|`func(x)`|
|Добавление в коллекцию|`lst.append(x)`|

#### Уменьшение счётчика ссылок

|Действие|Пример|
|---|---|
|Удаление переменной|`del x`|
|Выход из функции|`return`|
|Переприсваивание|`x = [4, 5, 6]`|
|Удаление из коллекции|`lst.remove(x)`|

```python
import sys

nums = [1, 2, 3]
print(sys.getrefcount(nums))  # 2 (nums + аргумент функции)

nums1 = nums
print(sys.getrefcount(nums))  # 3

del nums1
print(sys.getrefcount(nums))  # 2
```

#### Проблема: Циклические ссылки

```python
nums1 = [1, 2, 3]
nums2 = [4, 5]

nums1.append(nums2)  # nums1 ссылается на nums2
nums2.append(nums1)  # nums2 ссылается на nums1

del nums1
del nums2
# Объекты не удалятся! RefCount обоих = 1
```

```
┌──────────┐         ┌──────────┐
│ [1,2,3,•]│────────→│ [4,5,•]  │
│          │←────────│          │
└──────────┘         └──────────┘
   RefCount=1          RefCount=1
   
Имена удалены, но объекты живы!
```

### 2. Сборщик мусора (Garbage Collector)

**Решает проблему циклических ссылок.**

#### Поколения объектов

|Поколение|Объекты|Частота проверки|
|---|---|---|
|**0**|Новые объекты|Очень часто|
|**1**|Пережили 1 сборку|Реже|
|**2**|Пережили 2+ сборки|Редко|

```
Новый объект → Gen 0 → [GC] → Gen 1 → [GC] → Gen 2
                ↓               ↓               ↓
             Удалён         Удалён        Долгожитель
```

### Модуль gc

|Функция|Назначение|
|---|---|
|`gc.enable()`|Включить сборщик мусора|
|`gc.disable()`|Отключить сборщик мусора|
|`gc.isenabled()`|Проверить, включён ли GC|
|`gc.collect()`|Запустить сборку мусора вручную|
|`gc.collect(n)`|Запустить GC для поколения n (0, 1, 2)|
|`gc.get_count()`|Количество объектов в каждом поколении|
|`gc.get_threshold()`|Пороги срабатывания GC|

```python
import gc

# Проверить состояние
print(gc.isenabled())  # True (по умолчанию включён)

# Запустить сборку мусора вручную
gc.collect()

# Отключить (для оптимизации производительности)
gc.disable()

# Включить обратно
gc.enable()
```

### Сравнение механизмов

|Механизм|Когда работает|Находит циклические ссылки|Можно отключить|
|---|---|---|---|
|**Подсчёт ссылок**|Постоянно (реал-тайм)|❌ Нет|❌ Нет|
|**Сборщик мусора**|Периодически|✅ Да|✅ Да|

## ⚠️ Типичные ошибки

### 1. Изменяемые аргументы по умолчанию

**❌ Неправильно:**

```python
def append(element, seq=[]):  # Плохо!
    seq.append(element)
    return seq

print(append(10))  # [10]
print(append(5))   # [10, 5]  ← Не ожидали!
print(append(1))   # [10, 5, 1]
```

**Проблема:** Значение по умолчанию создаётся один раз при определении функции.

**✅ Правильно:**

```python
def append(element, seq=None):
    if seq is None:
        seq = []
    seq.append(element)
    return seq

print(append(10))  # [10]
print(append(5))   # [5]
print(append(1))   # [1]
```

### 2. Путаница между == и is

**❌ Неправильно:**

```python
value = None
if value == None:  # Работает, но не рекомендуется
    print('Пусто')
```

**✅ Правильно:**

```python
value = None
if value is None:  # Идиоматично
    print('Пусто')
```

### 3. Копирование вложенных структур

**❌ Неправильно:**

```python
matrix = [[1, 2], [3, 4]]
copy = matrix[:]  # Поверхностная копия!

matrix[0][0] = 99
print(copy)  # [[99, 2], [3, 4]]  ← Изменилась!
```

**✅ Правильно:**

```python
import copy

matrix = [[1, 2], [3, 4]]
deep = copy.deepcopy(matrix)  # Глубокая копия

matrix[0][0] = 99
print(deep)  # [[1, 2], [3, 4]]  ← Не изменилась
```

### 4. Изменение списка во время итерации

**❌ Неправильно:**

```python
nums = [1, 2, 3, 4, 5]
for num in nums:
    if num % 2 == 0:
        nums.remove(num)  # Плохо!
```

**✅ Правильно:**

```python
nums = [1, 2, 3, 4, 5]
nums = [num for num in nums if num % 2 != 0]
```
