## Три типа методов в Python

В Python существует три типа методов:

|Тип|Декоратор|Первый параметр|Доступ|
|---|---|---|---|
|Метод экземпляра|нет|`self`|атрибуты объекта и класса|
|Метод класса|`@classmethod`|`cls`|только атрибуты класса|
|Статический метод|`@staticmethod`|нет|ничего (вспомогательная функция)|

---

## Методы экземпляра

Методы, с которыми мы работали до этого — это методы экземпляра. Через параметр `self` они имеют доступ к атрибутам конкретного объекта и могут полноценно управлять его состоянием.

```python
class Cat:
    def __init__(self, name, age):
        self.name = name
        self.age = age
        self.favorite_things = []

    def about(self):
        return f'Имя: {self.name}, возраст: {self.age}'   # читаем атрибуты объекта

    def loves(self, thing):
        self.favorite_things.append(thing)                  # изменяем атрибут объекта


cat = Cat('Кемаль', 1)
cat.loves('Тимур')
cat.loves('Дом')

print(cat.about())              # → Имя: Кемаль, возраст: 1
print(cat.favorite_things)      # → ['Тимур', 'Дом']
```

Метод экземпляра можно вызвать и через класс — тогда нужно явно передать экземпляр:

```python
cat.loves('Тимур')          # обычный вызов через экземпляр
Cat.loves(cat, 'Тимур')     # эквивалентный вызов через класс
print(Cat.about(cat))       # → Имя: Кемаль, возраст: 1
```

> 💡 Метод экземпляра через `self.__class__` может получить доступ и к самому классу — то есть технически может менять и состояние класса тоже.

```python
class MyClass:
    def method(self):
        print(self)             # → объект
        print(self.__class__)   # → <class '__main__.MyClass'>
```

## Методы класса — @classmethod

Метод класса связан с самим классом, а не с конкретным экземпляром. Вместо `self` принимает `cls` — ссылку на класс. Может менять состояние класса (атрибуты класса), что затронет все экземпляры. Но **не имеет доступа к атрибутам конкретного экземпляра**.

```python
class MyClass:
    @classmethod
    def my_classmethod(cls):
        print('Это метод класса')
        print(cls)              # → <class '__main__.MyClass'>


MyClass.my_classmethod()        # вызов через класс
```

> 💡 Параметр `cls` обязателен. Называть его иначе технически можно, но строго не рекомендуется.

### Главное применение — альтернативные конструкторы

Методы класса чаще всего используются как **фабричные методы** — альтернативные способы создания экземпляров. Такие методы принято называть с префиксом `from_`:

```python
class Cat:
    def __init__(self, breed, name):
        self.breed = breed
        self.name = name

    @classmethod
    def british(cls, name):
        return cls('Британский', name)   # cls('Британский', name) == Cat('Британский', name)

    @classmethod
    def from_string(cls, data):          # альтернативный конструктор из строки
        breed, name = data.split(',')
        return cls(breed.strip(), name.strip())


cat1 = Cat('Манчкин', 'Роджер')           # обычный способ
cat2 = Cat.british('Кемаль')              # через метод класса
cat3 = Cat.from_string('Шотландский, Барсик')

print(cat1.breed, cat1.name)   # → Манчкин Роджер
print(cat2.breed, cat2.name)   # → Британский Кемаль
print(cat3.breed, cat3.name)   # → Шотландский Барсик
```

> 💡 Использование `cls(...)` вместо `Cat(...)` важно при наследовании — если класс унаследован, `cls` будет указывать на дочерний класс, а не на родительский.

### Знакомые примеры из стандартной библиотеки

```python
from datetime import datetime

cats = dict.fromkeys(['Кемаль', 'Роджер'])          # dict.fromkeys — метод класса
dt = datetime.strptime('12.10.2022', '%d.%m.%Y')   # datetime.strptime — метод класса

print(cats)   # → {'Кемаль': None, 'Роджер': None}
print(dt)     # → 2022-10-12 00:00:00
```

## Статические методы — @staticmethod

Статические методы не имеют ни `self`, ни `cls` — они не связаны ни с объектом, ни с классом. По сути это обычные функции, которые помещены в класс для удобства и логической группировки. Чаще всего используются как вспомогательный (сервисный) код.

```python
class MyClass:
    @staticmethod
    def my_staticmethod():
        print('Это статический метод')


MyClass.my_staticmethod()   # вызов через класс
```

### Пример: вспомогательный метод внутри класса

```python
class Cat:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def get_age(self, in_human_years=False):
        if in_human_years:
            return Cat.age_in_human_years(self.age)   # вызываем статический метод
        return self.age

    @staticmethod
    def age_in_human_years(age):                      # не нужен ни self, ни cls
        return (age + (age < 5) - (age > 8)) * 7


cat = Cat('Кемаль', 2)
print(cat.get_age())          # → 2   (кошачьи годы)
print(cat.get_age(True))      # → 21  (человеческие годы)
```

> ⚠️ Статический метод **формально может** обратиться к атрибутам класса через его имя (`Cat.something`), но делать так **не рекомендуется** — это нарушает назначение статического метода как независимой вспомогательной функции.

## Вызов через экземпляр

Методы класса и статические методы можно вызывать не только через класс, но и через экземпляр — результат будет одинаковым:

```python
class MyClass:
    @classmethod
    def my_classmethod(cls):
        print('Метод класса:', cls)

    @staticmethod
    def my_staticmethod():
        print('Статический метод')


obj = MyClass()

MyClass.my_classmethod()    # через класс
obj.my_classmethod()        # через экземпляр — работает так же

MyClass.my_staticmethod()   # через класс
obj.my_staticmethod()       # через экземпляр — работает так же
```

## Сравнение всех трёх типов

```python
class MyClass:
    class_attr = 'атрибут класса'

    def __init__(self):
        self.inst_attr = 'атрибут экземпляра'

    def instance_method(self):
        print(self.inst_attr)        # ✅ доступ к атрибуту экземпляра
        print(self.class_attr)       # ✅ доступ к атрибуту класса
        print(self.__class__)        # ✅ доступ к классу

    @classmethod
    def class_method(cls):
        print(cls.class_attr)        # ✅ доступ к атрибуту класса
        # print(cls.inst_attr)       # ❌ нет доступа к атрибуту экземпляра

    @staticmethod
    def static_method():
        print(MyClass.class_attr)    # ⚠️ технически можно, но не рекомендуется
        # print(inst_attr)           # ❌ нет доступа ни к чему
```

## Когда что использовать?

|Ситуация|Какой метод|
|---|---|
|Нужен доступ к атрибутам объекта|метод экземпляра|
|Нужен альтернативный конструктор|метод класса (`@classmethod`)|
|Нужно изменить атрибут класса|метод класса (`@classmethod`)|
|Вспомогательная функция, не связанная с объектом|статический метод (`@staticmethod`)|
|Логически относится к классу, но не нужен ни `self`, ни `cls`|статический метод (`@staticmethod`)|

## Быстрая шпаргалка

```python
class Cat:
    species = 'Felis catus'   # атрибут класса

    def __init__(self, name, age):
        self.name = name       # атрибут экземпляра
        self.age = age

    # Метод экземпляра — работает с объектом
    def describe(self):
        return f'{self.name}, {self.age} лет'

    # Метод класса — альтернативный конструктор
    @classmethod
    def from_string(cls, data):
        name, age = data.split(',')
        return cls(name.strip(), int(age.strip()))

    # Статический метод — вспомогательная функция
    @staticmethod
    def is_valid_age(age):
        return isinstance(age, int) and 0 <= age <= 30


# Использование
cat1 = Cat('Кемаль', 3)
cat2 = Cat.from_string('Роджер, 5')

print(cat1.describe())         # → Кемаль, 3 лет
print(cat2.describe())         # → Роджер, 5 лет
print(Cat.is_valid_age(3))     # → True
print(Cat.is_valid_age(-1))    # → False
print(Cat.species)             # → Felis catus
```
