
1️⃣ Базовый шаблон (Полная форма)

Это каркас, который нужно держать в голове. Всегда начинается с подчеркнутого хранилища.

```python
class ClassName:
    def __init__(self, value):
        self._value = value      # Реальное хранилище (protected)

    @property
    def value(self):             # Getter
        """Документация для свойства"""
        return self._value

    @value.setter
    def value(self, new_value):  # Setter
        self._value = new_value

    @value.deleter
    def value(self):             # Deleter
        del self._value
```

---

2️⃣ Механика: «Что вместо чего»

@property заменяет явные геттеры/сеттеры (Java-style) на естественный доступ к атрибутам.

Было (методы) Стало (property)
obj.get_value() obj.value
obj.set_value(10) obj.value = 10
del obj.value (если есть метод) del obj.value

Суть: Property превращает методы в дескриптор, перехватывающий чтение, запись и удаление.

---

3️⃣ Минимальные варианты (Частые кейсы)

🔹 Только чтение (Read-only)

```python
@property
def total(self):
    return self._price + self._tax
# obj.total = 10 -> AttributeError: can't set attribute
```

🔹 Getter + Setter (Валидация)

```python
@property
def age(self):
    return self._age

@age.setter
def age(self, value):
    if not 0 <= value <= 150:
        raise ValueError("Некорректный возраст")
    self._age = value
```

---

4️⃣ Тонкости и Важные правила

⚠️ Имена должны совпадать!

```python
@property
def age(self): ...

@age.setter    # ❌ Ошибка: если написать @year.setter — не привяжется
def age(self, value): ...
```

⚠️ Подчеркивание — это не защита, а «знак вежливости»

```python
obj._value = 999  # ❌ Технически это сломает логику, но Python не запретит
```

Правило: Никогда не обращайся к _value напрямую за пределами класса. Это соглашение "мы все взрослые люди".

⚠️ Read-only = Отсутствие setter

Если есть только @property, свойство становится неизменяемым. Удобно для констант или вычисляемых значений, которые не должны меняться вручную.

---

5️⃣ Зачем это нужно? (Сценарии)

1. Инкапсуляция и контроль
   Скрываем реальное хранилище (_value), предоставляя чистый интерфейс (value).
2. Валидация данных
   Ловим некорректные значения в сеттере до того, как они испортят состояние объекта.
3. Вычисляемые свойства
   Позволяют обращаться к методу как к атрибуту (без скобок).
   ```python
   @property
   def full_name(self):
       return f"{self.first} {self.last}"
   # Использование: obj.full_name, а не obj.full_name()
   ```
4. Совместимость (Рефакторинг)
   Можно начать с простого атрибута obj.age, а потом добавить логику через property, не меняя код, который использует obj.age.

6️⃣ Взгляд под капот (Дескрипторы)

Для тех, кто хочет понять магию.

Что такое @property на самом деле?
Это синтаксический сахар для создания объекта класса property.

```python
# Эта запись:
value = property(getter, setter, deleter)

# Эквивалентна (через декораторы):
value = property(value)          # getter
value = value.setter(setter)     # setter
value = value.deleter(deleter)   # deleter
```

Что происходит при обращении obj.value?
Python вызывает механизм дескрипторов из протокола:

```python
ClassName.__dict__['value'].__get__(obj, ClassName)
```

Объект property перехватывает доступ и делегирует его твоим методам.

7️⃣ Итоговая формула (Золотой стандарт)

```python
class MyClass:
    def __init__(self, x):
        self._x = x

    @property
    def x(self):
        """Getter"""
        return self._x

    @x.setter
    def x(self, value):
        """Setter с логикой"""
        # Валидация, логирование и т.д.
        self._x = value

    @x.deleter
    def x(self):
        """Deleter (редко нужен)"""
        del self._x
```

🧠 Главные мысли для выноса в голову:

1. Имена: Геттер и сеттер должны называться одинаково.
2. Данные: Хранятся в _атрибут, доступ через атрибут.
3. Механика: Это дескриптор, перехватывающий obj.attr, obj.attr = val, del obj.attr.
4. Плюсы: Валидация, read-only свойства, вычисляемые поля, обратная совместимость.
