## Зачем ограничивать доступ к атрибутам?

Доступ к атрибутам объекта в идеале должны иметь только методы этого объекта. Такой подход защищает данные от случайного повреждения — внешний код не может просто взять и поменять значение атрибута в обход логики класса.

В классических языках (C++, Java, C#) для этого есть специальные ключевые слова:

|Модификатор|Где доступен|
|---|---|
|`public`|Везде|
|`protected`|Внутри класса и его наследников|
|`private`|Только внутри класса|

**Python устроен иначе** — в нём нет механизма, который физически запрещает доступ к атрибутам. Вместо этого используются **соглашения об именовании**.

---

## Три уровня доступа в Python

### 🟢 Публичный атрибут — `name`

Доступен откуда угодно. Используется по умолчанию.

```python
class Cat:
    def __init__(self, name):
        self.name = name   # публичный


cat = Cat('Кемаль')
print(cat.name)      # → Кемаль
cat.name = 'Роджер'  # ✅ легко изменить снаружи
print(cat.name)      # → Роджер
```

---

### 🟡 Защищённый атрибут — `_name`

Одно нижнее подчёркивание — это **соглашение**: "этот атрибут внутренний, не трогай его снаружи". Технически всё равно доступен извне, но сигнализирует другим программистам об осторожности.

```python
class Cat:
    def __init__(self, name):
        self._name = name   # защищённый


cat = Cat('Кемаль')
print(cat._name)      # → Кемаль  (технически работает)
cat._name = 'Роджер'  # ⚠️ работает, но нарушает соглашение
print(cat._name)      # → Роджер
```

> 💡 `_name` — это сигнал: "я предупредил, дальше на твой страх и риск". Никаких реальных ограничений нет.

---

### 🔴 Приватный атрибут — `__name`

Два нижних подчёркивания — более жёсткая защита. Прямой доступ снаружи невозможен.

```python
class Cat:
    def __init__(self, name):
        self.__name = name   # приватный


cat = Cat('Кемаль')
print(cat.__name)    # ❌ AttributeError: 'Cat' object has no attribute '__name'
```

---

## Искажение имени (name mangling)

Почему возникает `AttributeError`? Потому что Python **автоматически переименовывает** приватный атрибут по правилу:

```
__name  →  _ИмяКласса__name
```

```python
class Cat:
    def __init__(self, name):
        self.__name = name


cat = Cat('Кемаль')

print(cat.__dict__)   # → {'_Cat__name': 'Кемаль'}
#                              ↑ атрибут переименован!
```

Атрибут никуда не исчез — он просто получил другое имя. Поэтому обратиться к нему всё ещё можно, если знать настоящее имя:

```python
cat._Cat__name = 'Роджер'   # ⚠️ технически возможно, но делать так не стоит
print(cat.__dict__)          # → {'_Cat__name': 'Роджер'}
```

> ⚠️ **Важный нюанс:** искажение имени происходит только когда атрибут задаётся **внутри класса**. Если задать `__name` снаружи — он сохранится как есть, без переименования:

```python
cat.__age = 1               # задаём снаружи
print(cat.__dict__)         # → {'_Cat__name': 'Кемаль', '__age': 1}
#                                  ↑ переименован          ↑ не переименован
```

---

## Итог по модификаторам

|Синтаксис|Тип|Реальное ограничение|Смысл|
|---|---|---|---|
|`self.name`|публичный|нет|доступен всем|
|`self._name`|защищённый|нет (только соглашение)|"не трогай снаружи"|
|`self.__name`|приватный|частичное (name mangling)|"это внутренние данные"|

> 💡 В Python **ответственность за соблюдение соглашений лежит на программистах**, а не на языке. Приватность — это предупреждение, а не запрет.

## Аксессоры (геттеры, сеттеры, делитеры)

На практике принято делать атрибуты защищёнными или приватными, а для работы с ними предоставлять специальные публичные методы — **аксессоры**. Это гарантирует, что любые изменения атрибутов проходят через контролируемый интерфейс.

---

### Геттер (getter) — получение значения

Метод, который **возвращает** значение атрибута, не изменяя его. Даёт внешнему коду безопасный доступ на чтение.

```python
class Cat:
    def __init__(self, name):
        self._name = name

    def get_name(self):       # геттер
        return self._name


cat = Cat('Кемаль')
print(cat.get_name())   # → Кемаль
```

---

### Сеттер (setter) — изменение значения

Метод, который **устанавливает или изменяет** значение атрибута. Главное преимущество — можно добавить валидацию перед изменением.

```python
class Cat:
    def __init__(self, name):
        self._name = name

    def get_name(self):
        return self._name

    def set_name(self, name):   # сеттер
        if isinstance(name, str) and name.isalpha():
            self._name = name
        else:
            raise ValueError('Некорректное имя')


cat = Cat('Кемаль')
print(cat.get_name())    # → Кемаль

cat.set_name('Роджер')
print(cat.get_name())    # → Роджер

cat.set_name(-1)         # ❌ ValueError: Некорректное имя
cat.set_name('Roger2')   # ❌ ValueError: Некорректное имя (не только буквы)
```

> 💡 Возбуждение исключения — стандартная практика при некорректных данных. Так устроены и встроенные классы Python: `date(2022, 20, 3)` выбросит `ValueError: month must be in 1..12`.

---

### Делитер (deleter) — удаление атрибута

Метод, который **удаляет** атрибут из объекта. После удаления обращение к атрибуту вызовет `AttributeError`.

```python
class Cat:
    def __init__(self, name):
        self._name = name

    def get_name(self):
        return self._name

    def set_name(self, name):
        if isinstance(name, str) and name.isalpha():
            self._name = name
        else:
            raise ValueError('Некорректное имя')

    def del_name(self):       # делитер
        del self._name


cat = Cat('Кемаль')
print(cat.get_name())   # → Кемаль

cat.del_name()
print(cat.get_name())   # ❌ AttributeError: 'Cat' object has no attribute '_name'
```

## Полный пример — класс с тремя аксессорами

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self._owner = owner
        self.__balance = balance       # приватный — важные данные

    # Геттеры
    def get_owner(self):
        return self._owner

    def get_balance(self):
        return self.__balance

    # Сеттеры
    def set_owner(self, owner):
        if isinstance(owner, str) and owner.strip():
            self._owner = owner
        else:
            raise ValueError('Некорректное имя владельца')

    def deposit(self, amount):
        if amount <= 0:
            raise ValueError('Сумма должна быть положительной')
        self.__balance += amount

    def withdraw(self, amount):
        if amount <= 0:
            raise ValueError('Сумма должна быть положительной')
        if amount > self.__balance:
            raise ValueError('Недостаточно средств')
        self.__balance -= amount


account = BankAccount('Иван', 1000)

print(account.get_owner())     # → Иван
print(account.get_balance())   # → 1000

account.deposit(500)
print(account.get_balance())   # → 1500

account.withdraw(200)
print(account.get_balance())   # → 1300

account.withdraw(9999)         # ❌ ValueError: Недостаточно средств
```

## Быстрая шпаргалка

```python
class Cat:
    def __init__(self, name):
        self.__name = name       # приватный атрибут

    def get_name(self):          # геттер — только читаем
        return self.__name

    def set_name(self, name):    # сеттер — изменяем с проверкой
        if isinstance(name, str) and name.isalpha():
            self.__name = name
        else:
            raise ValueError('Некорректное имя')

    def del_name(self):          # делитер — удаляем
        del self.__name


cat = Cat('Кемаль')
cat.get_name()            # → 'Кемаль'
cat.set_name('Роджер')    # меняем
cat.get_name()            # → 'Роджер'
cat.set_name(123)         # ❌ ValueError
cat.del_name()            # удаляем атрибут
```

|Аксессор|Назначение|Типичное имя|
|---|---|---|
|Геттер|Вернуть значение атрибута|`get_name()`|
|Сеттер|Изменить значение с валидацией|`set_name(value)`|
|Делитер|Удалить атрибут|`del_name()`|
