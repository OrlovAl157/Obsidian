
## 📊 Иерархия управления памятью

```
┌─────────────────────────────────────────────────────┐
│                    СТЕК (Stack)                      │
│  Локальные переменные, ссылки на объекты           │
│  x → [ссылка на объект 42]                         │
└──────────────────┬──────────────────────────────────┘
                   │ ссылается на
                   ↓
┌─────────────────────────────────────────────────────┐
│                   КУЧА (Heap)                        │
│          Вся динамическая память программы          │
│                                                      │
│  ┌───────────────────────────────────────────────┐ │
│  │            АРЕНА (Arena)                       │ │
│  │  Большой блок памяти (~256 KB)               │ │
│  │  Делится на маленькие блоки                   │ │
│  │                                                │ │
│  │  ┌─────────────────────────────────────────┐ │ │
│  │  │      ПУЛ ОБЪЕКТОВ (Object Pool)         │ │ │
│  │  │  Кеш для маленьких объектов             │ │ │
│  │  │                                          │ │ │
│  │  │  ┌────────┐ ┌────────┐ ┌────────┐      │ │ │
│  │  │  │ Блок 1 │ │ Блок 2 │ │ Блок 3 │ ... │ │ │
│  │  │  │  int   │ │ float  │ │  str   │      │ │ │
│  │  │  └────────┘ └────────┘ └────────┘      │ │ │
│  │  └─────────────────────────────────────────┘ │ │
│  └───────────────────────────────────────────────┘ │
│                                                      │
│  ┌───────────────────────────────────────────────┐ │
│  │  Большие объекты (списки, словари и т.д.)    │ │
│  │  выделяются напрямую в куче                  │ │
│  └───────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

## 🔹 1. Блок памяти (Memory Block)

**Что это:** Минимальная единица хранения одного объекта.

**Структура блока:**

```
┌──────────────────────────┐
│  Тип объекта (type)      │  ← указатель на тип
├──────────────────────────┤
│  Счётчик ссылок (refcount) │  ← для сборки мусора
├──────────────────────────┤
│  Данные объекта          │  ← само значение
└──────────────────────────┘
```

**Пример:**

```python
x = 42  # Создаётся один блок памяти для числа 42
```

-----

## 🔹 2. Пул объектов (Object Pool)

**Что это:** Кеш для часто используемых маленьких объектов.

**Зачем:** Избежать многократного выделения/освобождения памяти для одинаковых объектов.

**Что кешируется:**

- **Целые числа:** от `-5` до `256`
- **Строки:** короткие строки, использованные как идентификаторы
- **Кортежи:** пустые кортежи `()`
- **Небольшие объекты:** до определённого размера

**Пример:**

```python
a = 100
b = 100
print(a is b)  # True — один и тот же объект из пула!

c = 1000
d = 1000
print(c is d)  # False — за пределами пула, разные объекты
```

-----

## 🔹 3. Арена (Arena)

**Что это:** Большой блок памяти (~256 KB), разбитый на маленькие блоки.

**Зачем:** Эффективное управление множеством мелких объектов.

**Как работает:**

1. CPython запрашивает у ОС большой блок (арену)
2. Делит его на маленькие блоки (например, 8, 16, 32 байта)
3. Выделяет блоки под объекты по запросу
4. При освобождении объекта блок возвращается в арену для повторного использования

**Преимущества:**

- Меньше обращений к ОС
- Быстрое выделение памяти
- Уменьшение фрагментации

-----

## 🔹 4. Куча (Heap)

**Что это:** Общая область динамической памяти для всех объектов Python.

**Что хранится:**

- Арены с маленькими объектами
- Большие объекты (списки, словари, классы) напрямую
- Все данные, созданные во время выполнения программы

**Управление:**

- **Выделение:** через менеджер памяти Python
- **Освобождение:** автоматически через сборщик мусора (garbage collector)

-----

## 🗑️ Сборка мусора (Garbage Collection)

### Механизмы очистки памяти:

#### 1️⃣ **Подсчёт ссылок (Reference Counting)**

```python
x = [1, 2, 3]  # refcount = 1
y = x          # refcount = 2
del x          # refcount = 1
del y          # refcount = 0 → объект удаляется
```

#### 2️⃣ **Циклический сборщик мусора**

Обнаруживает и удаляет циклические ссылки:

```python
a = []
b = []
a.append(b)  # a ссылается на b
b.append(a)  # b ссылается на a
# Без циклического GC это утечка памяти
```

#### 3️⃣ **Поколенческая сборка (Generational GC)**

Объекты делятся на поколения:

- **Поколение 0:** новые объекты (проверяются часто)
- **Поколение 1:** прожившие несколько сборок
- **Поколение 2:** долгоживущие объекты (проверяются редко)

-----

## 🔧 Полезные команды

### Проверить количество ссылок на объект:

```python
import sys
x = [1, 2, 3]
print(sys.getrefcount(x))  # Покажет количество ссылок
```

### Узнать размер объекта в памяти:

```python
import sys
x = [1, 2, 3]
print(sys.getsizeof(x))  # Размер в байтах
```

### Проверить, один ли это объект:

```python
a = 256
b = 256
print(a is b)  # True — из пула

c = 257
d = 257
print(c is d)  # False — вне пула
```

### Управление сборщиком мусора:

```python
import gc

gc.collect()              # Запустить сборку мусора вручную
gc.get_count()           # Счётчики объектов по поколениям
gc.disable()             # Отключить автоматическую сборку
gc.enable()              # Включить обратно
```

## 📝 Краткая сводка

|Уровень         |Описание                               |Размер        |
|----------------|---------------------------------------|--------------|
|**Блок памяти** |Один объект с метаданными              |16-64 байта   |
|**Пул объектов**|Кеш маленьких объектов                 |Десятки блоков|
|**Арена**       |Набор блоков для эффективного выделения|~256 KB       |
|**Куча**        |Вся динамическая память программы      |Гигабайты     |

-----

## 🎯 Ключевые принципы

1. **Блок → Пул → Арена → Куча** — это иерархия управления памятью
2. **Пул** оптимизирует работу с маленькими, часто используемыми объектами
3. **Арена** ускоряет выделение памяти, минимизируя обращения к ОС
4. **Сборщик мусора** автоматически освобождает неиспользуемую память
5. **Всё хранится в куче**, стек содержит только ссылки

## 💡 Оптимизация памяти

### Советы:

- Переиспользуйте объекты вместо создания новых
- Удаляйте большие объекты через `del`, когда они не нужны
- Используйте генераторы вместо списков для больших данных
- Избегайте циклических ссылок или используйте слабые ссылки (`weakref`)

### Пример оптимизации:

```python
# Плохо (создаёт большой список в памяти)
squares = [x**2 for x in range(1000000)]

# Хорошо (генератор, объекты создаются по требованию)
squares = (x**2 for x in range(1000000))
```
