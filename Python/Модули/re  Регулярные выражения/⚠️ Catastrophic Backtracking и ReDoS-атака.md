

## 🎯 Что такое ReDoS?

**ReDoS** (Regular Expression Denial of Service) — атака на сервисы, использующие регулярные выражения, позволяющая замедлить или вывести их из строя.

**Основа атаки:** Проблема регулярных выражений под названием **Catastrophic Backtracking** (катастрофический откат).

---

## 💥 Catastrophic Backtracking

### Суть проблемы

Когда регулярное выражение не находит совпадение, движок **перебирает строку огромным количеством способов**, пытаясь найти совпадение. Время выполнения растёт **экспоненциально**.

### Пример проблемного выражения

```python
import re

# Простое выражение с вложенными квантификаторами
pattern = r"(a+)+b"

# Строка из 20 символов 'a' (без 'b' в конце)
re.findall(pattern, "a" * 20)
# Время: 0.072 секунды

# Строка из 30 символов 'a'
re.findall(pattern, "a" * 30)
# Время: 75.5 секунд

# Прирост времени более чем в 1000 раз! 🔥
```

### Почему так происходит?

Движок пытается найти совпадение всеми возможными способами:

```
Строка: "aaaa" (ищем "(a+)+b")

Попытки движка:
1. (aaaa) + b ❌
2. (aaa)(a) + b ❌
3. (aa)(aa) + b ❌
4. (aa)(a)(a) + b ❌
5. (a)(aaa) + b ❌
6. (a)(aa)(a) + b ❌
7. (a)(a)(aa) + b ❌
8. (a)(a)(a)(a) + b ❌
... и так далее

Количество попыток растёт экспоненциально!
```

---

## 🚨 Опасные паттерны

Регулярное выражение подвержено Catastrophic Backtracking, если:

**К группе применён квантификатор И внутри группы используется:**

- Ещё один квантификатор
- Альтернация `|`

### Таблица опасных паттернов

|Паттерн|Проблема|Почему опасно|
|---|---|---|
|`(a+)+`|Вложенные квантификаторы|Экспоненциальный рост попыток|
|`([a-zA-Z_]+)*`|Квантификатор на группе с квантификатором|Множество комбинаций|
|`(a\|aa)+`|Квантификатор на группе с альтернацией|Перекрывающиеся варианты|
|`(a\|a?)+`|Альтернация + опциональность|Дублирование вариантов|
|`(.*)*`|Вложенные `*`|Катастрофическое количество путей|
|`(.+)+`|Вложенные `+`|Аналогично `(.*)*`|

### Примеры реальных уязвимых паттернов

```python
# ❌ ОПАСНО
r"(a+)+$"                    # Вложенные квантификаторы
r"([a-z]+)*@example\.com"    # Квантификатор на группе
r"(hello|hello world)+"      # Перекрывающаяся альтернация
r"(x+x+)+y"                  # Множественные вложенные квантификаторы
r"(a*)*b"                    # Двойная жадность
```

---

## 🛡️ Решения проблемы

|Метод|Описание|Когда использовать|
|---|---|---|
|**1. Переписать regex**|Упростить, убрать вложенность|Если возможно упрощение|
|**2. Проверить входные данные**|Ограничить длину, формат|Всегда рекомендуется|
|**3. Притяжательные квантификаторы**|Запретить откат|Для оптимизации производительности|
|**4. Атомарная группировка**|Запретить откат в группе|Для сложных паттернов|
|**5. Контроль времени**|Остановка при долгом выполнении|Критичные сервисы|

### 1. Переписать регулярное выражение

```python
# ❌ Плохо (catastrophic backtracking)
r"(a+)+b"

# ✅ Хорошо (упрощено)
r"a+b"
```

```python
# ❌ Плохо
r"([a-zA-Z_]+)*@example\.com"

# ✅ Хорошо
r"[a-zA-Z_]+@example\.com"
```

### 2. Проверка входных данных

```python
import re

def safe_match(pattern, text, max_length=1000):
    # Ограничиваем длину входных данных
    if len(text) > max_length:
        raise ValueError(f"Текст слишком длинный (>{max_length})")
    
    return re.findall(pattern, text)

# Использование
try:
    result = safe_match(r"(a+)+b", user_input, max_length=500)
except ValueError as e:
    print(f"Ошибка: {e}")
```

## 🔒 Притяжательные квантификаторы

**Синтаксис:** Добавить `+` после жадного квантификатора

### Таблица притяжательных квантификаторов

|Жадный|Притяжательный|Описание|
|---|---|---|
|`*`|`*+`|Ноль или более (без отката)|
|`+`|`++`|Один или более (без отката)|
|`?`|`?+`|Ноль или один (без отката)|
|`{m,n}`|`{m,n}+`|От m до n (без отката)|
|`{m,}`|`{m,}+`|От m и более (без отката)|
|`{,n}`|`{,n}+`|До n включительно (без отката)|

### Как работают

```
Жадный квантификатор:
1. Захватывает максимум
2. Если не сработало → откатывается
3. Пробует снова
4. Повторяет до нахождения или исчерпания вариантов

Притяжательный квантификатор:
1. Захватывает максимум
2. Если не сработало → сразу отказ
3. НЕ откатывается
```

### Примеры

```python
import re

# Жадный (с откатом)
pattern1 = r"a+b"
re.findall(pattern1, "aaaa")  # [] (пытается откатиться)

# Притяжательный (без отката)
pattern2 = r"a++b"
re.findall(pattern2, "aaaa")  # [] (быстрый отказ, без отката)
```

```python
# ❌ Опасно (catastrophic backtracking)
r"(a+)+b"

# ✅ Безопасно (притяжательный квантификатор)
r"(a++)b"  # или r"a++b"
```

### Важно

```python
# Никогда не найдёт совпадение!
pattern = r".*+."

# Почему?
# .*+ захватит ВСЕ символы и не откатится
# Для финальной точки . ничего не останется
```

## 🔐 Атомарная группировка

**Синтаксис:** `(?>regex)`

Для атомарной группировки вместо `:` используется символ `>`.

### Принцип работы

**Атомарная группа** — это группа, внутри которой **запрещён откат**.

```
Обычная группа (a+):
- Захватывает 'aaaa'
- Если не сработало → откатывается до 'aaa'
- Потом до 'aa', потом до 'a'

Атомарная группа (?>a+):
- Захватывает 'aaaa'
- Если не сработало → сразу отказ
- Откат невозможен
```

### Детальный пример с альтернацией

#### Без атомарной группировки (с откатом)

```python
import re

pattern = r'a(bc|b|x)cc'
text = 'abcc axcc abcc'

re.findall(pattern, text)
# Результат: ['bc', 'x', 'bc']
# Найдено: abcc, axcc, abcc
```

**Как работает:**

```
Текст: "abcc"
Паттерн: a(bc|b|x)cc

Шаг 1: Найден 'a'
Шаг 2: Пробуем 'bc' → совпало
Шаг 3: Ищем 'cc' → НЕ совпало (остался только один 'c')
Шаг 4: ОТКАТ! Возвращаемся к альтернативе
Шаг 5: Пробуем 'b' → совпало
Шаг 6: Ищем 'cc' → совпало!
Результат: "abcc" ✅
```

#### С атомарной группировкой (без отката)

```python
import re

pattern = r'a(?>bc|b|x)cc'
text = 'abcc axcc abcc'

re.findall(pattern, text)
# Результат: ['x']
# Найдено только: axcc
```

**Как работает:**

```
Текст: "abcc"
Паттерн: a(?>bc|b|x)cc

Шаг 1: Найден 'a'
Шаг 2: Пробуем 'bc' → совпало
Шаг 3: Атомарная группа ЗАФИКСИРОВАНА (откат ЗАПРЕЩЁН)
Шаг 4: Ищем 'cc' → НЕ совпало (остался только один 'c')
Шаг 5: ОТКАТ НЕВОЗМОЖЕН!
Шаг 6: Продолжаем поиск со следующего символа
Результат: совпадение НЕ найдено ❌
```

### Когда атомарная группировка найдёт совпадение

Если добавить дополнительный символ 'c':

```python
import re

pattern = r'a(?>bc|b|x)cc'
text = 'abccc axcc abcc'

re.findall(pattern, text)
# Результат: ['bc', 'x']
# Найдено: abccc, axcc
```

**Почему теперь находит 'abccc':**

```
Текст: "abccc"
Паттерн: a(?>bc|b|x)cc

Шаг 1: Найден 'a'
Шаг 2: Пробуем 'bc' → совпало
Шаг 3: Атомарная группа зафиксирована
Шаг 4: Ищем 'cc' → совпало! (осталось 'cc')
Результат: "abccc" ✅
```

### Ключевое отличие

|Аспект|Обычная группа `(bc\|b\|x)`|Атомарная группа `(?>bc\|b\|x)`|
|---|---|---|
|**Откат внутри группы**|✅ Разрешён|❌ Запрещён|
|**Поведение при несовпадении**|Пробует другие альтернативы|Сразу отказ|
|**После выбора альтернативы**|Можно вернуться|Нельзя вернуться|
|**Найдёт `abcc`**|✅ Да (откатится от `bc` к `b`)|❌ Нет (не откатится)|

### Схема работы атомарной группировки

```
Текст: "abcc axcc abcc"
Паттерн: a(?>bc|b|x)cc

Анализ "abcc":
┌─────────────────────────────────────┐
│ a(?>bc|b|x)cc                       │
│ ↓ ↓                                 │
│ a bc  ← Выбрали 'bc'                │
│   ↓                                 │
│   ЗАФИКСИРОВАНО (откат запрещён!)   │
│   ↓                                 │
│   Ищем 'cc' → есть только 'c'       │
│   ↓                                 │
│   НЕТ СОВПАДЕНИЯ ❌                 │
└─────────────────────────────────────┘

Анализ "axcc":
┌─────────────────────────────────────┐
│ a(?>bc|b|x)cc                       │
│ ↓    ↓                              │
│ a    x  ← Выбрали 'x'               │
│      ↓                              │
│      ЗАФИКСИРОВАНО                  │
│      ↓                              │
│      Ищем 'cc' → совпало!           │
│      ↓                              │
│      НАЙДЕНО: "axcc" ✅             │
└─────────────────────────────────────┘
```

### Важное правило

**После того как атомарная группа выбрала совпадение:**

1. Другие альтернативы НЕ рассматриваются
2. Откат к началу группы НЕВОЗМОЖЕН
3. Поиск либо продолжается дальше, либо завершается неудачей

### Примеры

```python
import re

# ❌ Catastrophic backtracking
pattern1 = r"(a+)+b"
re.findall(pattern1, "a" * 30)  # Очень долго!

# ✅ С атомарной группировкой
pattern2 = r"(?>a+)+b"
re.findall(pattern2, "a" * 30)  # Быстро!
```

```python
# Пример с захватом
text = "aaaab"

# Обычная группа (с откатом)
pattern1 = r"(a+)b"
re.findall(pattern1, text)  # ['aaaa']

# Атомарная группа (без отката)
pattern2 = r"(?>a+)b"
re.findall(pattern2, text)  # ['aaaab'] (вся строка)
```

### Эквивалентность с притяжательными квантификаторами

|Притяжательный|Атомарная группа|
|---|---|
|`x*+`|`(?>x*)`|
|`x++`|`(?>x+)`|
|`x?+`|`(?>x?)`|
|`x{m,n}+`|`(?>x{m,n})`|

```python
import re

# Эти паттерны идентичны
pattern1 = r"a++b"
pattern2 = r"(?>a+)b"

text = "aaaa"
re.findall(pattern1, text)  # []
re.findall(pattern2, text)  # []
```

## 📊 Сравнение квантификаторов

|Тип|Синтаксис|Поведение|Откат|Использование|
|---|---|---|---|---|
|**Жадный**|`*`, `+`, `?`, `{m,n}`|Максимум символов|✅ Да|По умолчанию|
|**Ленивый**|`*?`, `+?`, `??`, `{m,n}?`|Минимум символов|✅ Да|Когда нужен минимум|
|**Притяжательный**|`*+`, `++`, `?+`, `{m,n}+`|Максимум символов|❌ Нет|Оптимизация, защита от ReDoS|

## 🎯 Практические примеры

### Пример 1: Валидация email (уязвимый)

```python
# ❌ УЯЗВИМО к ReDoS
pattern = r"([a-zA-Z0-9_]+)+@example\.com"

# Атака
malicious_input = "a" * 50 + "!"
# Выполнение займёт очень долго!
```

**Исправление:**

```python
# ✅ Безопасно
pattern = r"[a-zA-Z0-9_]++@example\.com"
# или
pattern = r"(?>-zA-Z0-9_]+)@example\.com"
```

### Пример 2: Поиск повторяющихся слов (уязвимый)

```python
# ❌ УЯЗВИМО
pattern = r"(\w+\s+)*\w+"

# ✅ Безопасно
pattern = r"(\w++\s++)*\w+"
# или
pattern = r"(?>(\w+\s+)*)\w+"
```

### Пример 3: Парсинг HTML (уязвимый)

```python
# ❌ УЯЗВИМО (никогда не парсите HTML регулярками!)
pattern = r"<([a-z]+)([^>]*)>(.*)</\1>"

# Атака
malicious = "<div " + "a=" * 1000 + ">"
```

**Решение:** Используйте специализированные HTML-парсеры (BeautifulSoup, lxml), а не регулярные выражения!

## 🧪 Тестирование на уязвимости

### Простой тест производительности

```python
import re
import time

def test_regex_performance(pattern, test_strings):
    """Тестирует regex на производительность"""
    for length, string in test_strings:
        start = time.time()
        try:
            re.findall(pattern, string)
            elapsed = time.time() - start
            print(f"Длина {length}: {elapsed:.4f} сек")
        except KeyboardInterrupt:
            print(f"Длина {length}: ПРЕРВАНО (слишком долго!)")
            break

# Тест уязвимого паттерна
pattern = r"(a+)+b"
test_strings = [
    (10, "a" * 10),
    (15, "a" * 15),
    (20, "a" * 20),
    (25, "a" * 25),
]

test_regex_performance(pattern, test_strings)
```

### Признаки уязвимости

**⚠️ Regex уязвим, если:**

- Время выполнения растёт экспоненциально с увеличением длины входной строки
- Есть вложенные квантификаторы: `(a+)+`, `(.*)*`
- Есть альтернация с перекрытием: `(a|ab)+`
- Группа с квантификатором содержит квантификатор: `([a-z]+)*`

## 📋 Шпаргалка

### Быстрая проверка паттерна

```python
# ❌ ОПАСНО - Избегайте этого!
(x+)+
(x*)*
(x+)*
([a-z]+)+
(a|ab)+
(a|a?)+

# ✅ БЕЗОПАСНО - Используйте это!
(x++)         # Притяжательный
(?>x+)        # Атомарная группа
x+            # Без вложенности
[a-z]+        # Простой квантификатор
```

### Таблица замен

|Опасный паттерн|Безопасная замена|
|---|---|
|`(a+)+`|`a+` или `(a++)`|
|`([a-z]+)*`|`[a-z]*` или `([a-z]++)`|
|`(a\|aa)+`|`a+`|
|`(.*)*`|`.*` или `(.*+)`|
|`(\w+\s+)+`|`(\w++\s++)+`|

### Основные правила защиты

1. **Избегайте вложенных квантификаторов**
2. **Проверяйте входные данные** (ограничение длины)
3. **Используйте притяжательные квантификаторы** `*+`, `++`
4. **Используйте атомарные группы** `(?>...)`
5. **Упрощайте regex** где возможно
6. **Тестируйте на больших данных** перед продакшеном
7. **Устанавливайте таймауты** для выполнения regex

## 💡 Важно запомнить

✅ **Притяжательные квантификаторы:**

- Захватывают максимум
- НЕ откатываются
- Синтаксис: `*+`, `++`, `?+`, `{m,n}+`

✅ **Атомарная группировка:**

- Запрещает откат внутри группы
- Синтаксис: `(?>regex)`
- Эквивалентна притяжательным квантификаторам

✅ **ReDoS:**

- Атака через catastrophic backtracking
- Основана на вложенных квантификаторах
- Решается оптимизацией regex или проверкой данных

⚠️ **Никогда:**

- Не парсите HTML/XML регулярками (используйте парсеры!)
- Не используйте `(.*)*` или `(.+)+`
- Не забывайте проверять входные данные
