## 🎯 Главный вопрос: Почему id() не меняется?

**Краткий ответ:** `id()` возвращает адрес **объекта-списка**, а не адрес **массива элементов**. При переполнении перемещается только внутренний массив, а сам объект остаётся на месте.

---

## 📦 Двухуровневая структура списка


```python
┌───────────────────────────────────────────────────────────┐
│ УРОВЕНЬ 1: Объект списка (PyListObject)                   │
│ ═══════════════════════════════════════════════════════   │
│                                                           │
│  Адрес объекта: 0x7f1234567890  ← id(lst) ВСЕГДА ЭТОТ!    │
│                                                           │
│  ┌─────────────────────────────────────┐                  │
│  │ Заголовок объекта:                  │                  │
│  ├─────────────────────────────────────┤                  │
│  │ ob_refcnt: 1    (счётчик ссылок)    │                  │
│  │ ob_type: list   (тип объекта)       │                  │
│  │ ob_size: 4      (текущий размер)    │                  │
│  │ allocated: 6    (выделено мест)     │                  │
│  │ ob_item: 0xAAAA (указатель) ────┐   │                  │
│  └─────────────────────────────────┼───┘                  │
│                                    │                      │
└────────────────────────────────────┼──────────────────────┘
                                    │
                                    │ УКАЗАТЕЛЬ
                                    ↓
┌────────────────────────────────────────────────────────────┐
│ УРОВЕНЬ 2: Массив указателей на элементы                   │
│ ══════════════════════════════════════════════════════     │
│                                                            │
│  Адрес массива: 0xAAAA  ← ЭТОТ адрес МЕНЯЕТСЯ при росте!   │
│                                                            │
│  ┌──────┬──────┬──────┬──────┬──────┬──────┐               │
│  │ ptr0 │ ptr1 │ ptr2 │ ptr3 │ NULL │ NULL │               │
│  └───┬──┴───┬──┴───┬──┴───┬──┴──────┴──────┘               │
│      │      │      │      │                                │
│      ↓      ↓      ↓      ↓                                │
│    [1]    [2]    [3]    [4]                                │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 🔄 Что происходит при append()

### Шаг 1: Создание списка

python

```python
lst = [1, 2, 3, 4]

┌─────────────────────────────┐
│ Объект списка               │  id = 0x1000 ✓
│ size: 4                     │
│ allocated: 6 (4 * 1.5)      │
│ ob_item → 0x2000            │
└─────────────────────────────┘
         ↓
    ┌───────────────────┐
    │ Массив (0x2000)   │
    │ [1][2][3][4][_][_]│
    └───────────────────┘
```

---

### Шаг 2: Добавление в пределах резерва

python

```python
lst.append(5)  # Влезает!

┌─────────────────────────────┐
│ Объект списка               │  id = 0x1000 ✓ (НЕ изменился)
│ size: 5  ← изменилось       │
│ allocated: 6                │
│ ob_item → 0x2000 ✓          │  ← указатель НЕ изменился
└─────────────────────────────┘
         ↓
    ┌───────────────────┐
    │ Массив (0x2000)   │  ← ТОТ ЖЕ массив
    │ [1][2][3][4][5][_]│
    └───────────────────┘
```

---

### Шаг 3: Переполнение резерва

python

```python
lst.append(6)
lst.append(7)  # ⚠️ Переполнение!

┌─────────────────────────────┐
│ Объект списка               │  id = 0x1000 ✓ (ВСЁЁ ЕЩЁ тот же!)
│ size: 7  ← новый размер     │
│ allocated: 10 (7 * 1.5)     │
│ ob_item → 0x3000 ✗          │  ← ТОЛЬКО указатель изменился!
└─────────────────────────────┘
         ↓
    ┌──────────────────────────────────┐
    │ НОВЫЙ массив (0x3000)            │  ← Новое место в памяти
    │ [1][2][3][4][5][6][7][_][_][_]   │
    └──────────────────────────────────┘

    ┌───────────────────┐
    │ Старый (0x2000)   │  ← Удалён сборщиком мусора
    │ [освобождено]     │
    └───────────────────┘
```
