Отличный вопрос — и самый важный для понимания, потому что тут легко запутаться.

  

Отвечаю максимально чётко:

  

  

⭐ КРАТКИЙ ОТВЕТ

  

  

Хэш-таблица (dk_indices) НЕ содержит хэшей ключей.

  

Вообще НЕ содержит.

  

Она содержит ТОЛЬКО индексы, которые указывают на записи в массиве entries.

  

  

  

  

⭐ ПОЛНОЕ ОБЪЯСНЕНИЕ

  

  

  

✔ Что такое dk_indices?

  

  

Это массив, где каждый элемент — маленькое число, чаще всего int16 или int32.

  

Это число — индекс записи в entries[].

  

Пример:

dk_indices = [ -1, 2, -1, 0, -1, -1, 1, -1 ]

Это говорит:

  

- слот 1 → запись entries[2]
- слот 3 → запись entries[0]
- слот 6 → запись entries[1]

  

  

Но хэшей здесь нет.

  

  

  

  

⭐ А где же хэши?

  

  

Хэши хранятся в самих записях entries.

  

Пример:

entries[0] = { hash("A"), "A", 10 }

entries[1] = { hash("B"), "B", 20 }

entries[2] = { hash("C"), "C", 30 }

То есть каждая запись имеет:

  

- me_hash — хэш ключа
- me_key — сам ключ
- me_value — значение

  

  

  

  

  

⭐ Почему хэши НЕ в хэш-таблице?

  

  

Потому что Python использует индирекцию (косвенность):

  

  

dk_indices → (index) → entries[index] → {hash, key, value}

  

  

Такое разделение нужно для:

  

  

1) экономии памяти

  

  

Хэши могут быть 64-битные (Py_hash_t), а индексы — маленькие (обычно 16 или 32 бита).

Это экономит большие объемы памяти при больших словарях.

  

  

2) стабильный порядок вставки

  

  

Порядок задаётся массивом entries, а не хэш-таблицей.

  

  

3) уменьшения количества перехеширований

  

  

При росте словаря переносить нужно только индексы, а не ключи.

  

  

4) уменьшения коллизий и пробросов

  

  

Хэш-таблица остаётся компактной и быстрой.

  

  

  

  

⭐ То есть:

  

  

  

✔ dk_indices — это ТОЛЬКО “карта”

  

  

Она говорит: «в какой слот положить какую запись».

  

  

✔ entries — это ХРАНИЛИЩЕ

  

  

В нём лежат:

  

- хэш
- ключ
- значение

  

  

И там же сохраняется порядок вставки.

  

  

  

  

⭐ Итог в одной фразе

  

  

  

**Хэш-таблица в Python НЕ хранит хэши.

  

  

Она хранит индексы, указывающие на записи, где хэши действительно находятся.**

  

  

  

Если хочешь — могу нарисовать очень наглядную схему, где видно, что где лежит.
