🔑 Ключевая идея split()

Python работает не с разделителями, а с токенами (словами)

🔄 Алгоритм (упрощённо):

```
1. Идёт по строке посимвольно
2. Пропускает ВСЕ пробельные символы (isspace()):
   • ' ' (пробел)
   • \t (табуляция)
   • \n (новая строка)
   • \r (возврат каретки)
   • \v (вертикальная табуляция)
   • \f (прогон страницы)
3. Как только встречает НЕ-пробельный символ:
   • начинает копить слово
4. Копит символы, пока снова не встретит пробел
5. Сохраняет слово
6. Повторяет с шага 2
```

---

📝 Псевдокод для `split()`:

```python
def split_whitespace(s):
    result = []           # список для слов
    i = 0                 # текущая позиция
    n = len(s)            # длина строки
    
    while i < n:
        # Шаг 1: Пропустить ВСЕ пробельные символы
        while i < n and s[i].isspace():
            i += 1
        
        # Если дошли до конца строки — выходим
        if i >= n:
            break
        
        # Шаг 2: Найти начало слова
        start = i
        
        # Шаг 3: Читать слово до следующего пробела
        while i < n and not s[i].isspace():
            i += 1
        
        # Шаг 4: Сохранить слово
        word = s[start:i]
        result.append(word)
    
    return result
```

---

⚠️ Важный момент

Пустые строки физически не могут появиться, потому что:

```
"   Hello   World   "
    ↑       ↑       ↑
    │       │       └── пробелы игнорируются
    │       └── слово начинается здесь
    └── слово начинается здесь
```

В алгоритме:

· Слово начинается только после пробелов
· Слово заканчивается до пробелов
· Пробелы игнорируются, а не используются как разделители

🎯 Ключевая идея split(sep)

Python работает с разделителями как с указателями мест разреза

🔄 Алгоритм (упрощённо):

```
1. Идёт по строке
2. Ищет конкретную подстроку-разделитель (sep):
   • Может быть любой строкой: ',', '::', '###'
3. Как только находит разделитель:
   • Берёт всё от предыдущей позиции до разделителя
4. Сохраняет эту часть (даже если она пустая!)
5. Продолжает с позиции после разделителя
6. Повторяет до конца строки
```

---

📝 Псевдокод для `split(sep)`:

```python
def split_with_separator(s, sep):
    result = []           # список для частей
    i = 0                 # текущая позиция
    n = len(s)            # длина строки
    sep_len = len(sep)    # длина разделителя
    
    if sep_len == 0:      # пустой разделитель запрещён
        raise ValueError("empty separator")
    
    while True:
        # Шаг 1: Найти следующий разделитель
        j = s.find(sep, i)  # ищем sep начиная с позиции i
        
        # Если разделитель не найден
        if j < 0:
            j = n  # взять до конца строки
        
        # Шаг 2: Взять часть от i до j
        part = s[i:j]
        result.append(part)  # даже если part == ""
        
        # Если дошли до конца строки — выходим
        if j == n:
            break
        
        # Шаг 3: Перепрыгнуть через разделитель
        i = j + sep_len
    
    return result
```

---

⚠️ Важный момент

Пустые строки ВОЗМОЖНЫ и значимы, потому что:

```
"a,,b,c"
 ↑ ↑ ↑ ↑
 │ │ │ └── часть после последнего разделителя
 │ │ └── пустая часть между разделителями
 │ └── часть между разделителями
 └── часть до первого разделителя
```

В алгоритме:

· Каждый разделитель создаёт границу
· Части между разделителями сохраняются всегда
· Пустые части = значимые данные (например, пустые поля в CSV)

---

🆚 Сравнение псевдокодов

Аспект split() split(sep)
Цикл while i < n: while True:
Поиск Проверка s[i].isspace() s.find(sep, i)
Пустые Не создаются Всегда добавляются
Выход Когда i >= n Когда j == n
Сдвиг На 1 символ На len(sep) символов

---

💎 Вывод

`split()` → "Собираем слова, игнорируя пробелы"
`split(sep)` → "Режем строку везде, где есть sep"

```python
import re

result = re.split(r'\D+', '1 + 2 - 3 =')
print(result)  # ['1', '2', '3', '']
```

🤔 Почему появляется пустая строка в конце?

📍 Причина:

re.split() с аргументом работает аналогично `str.split(sep)`:

· Разделитель r'\D+' = один или более НЕ-цифровых символов
· Всякий раз, когда разделитель находится в конце строки → создаётся пустая строка

🔍 Разбор шаг за шагом:

```
Строка:  '1 + 2 - 3 ='
Шаблон: r'\D+'  (ищем НЕ цифры: пробелы, '+', '-', '=', и т.д.)

1. Начинаем: '1 + 2 - 3 ='
   ↑
   Найдено: '1' (цифра, не разделитель)

2. Находим разделитель: ' + ' (пробел,+,пробел)
   Результат пока: ['1']

3. Находим: '2' (цифра)
   Результат: ['1', '2']

4. Находим разделитель: ' - ' (пробел,-,пробел)
   Результат: ['1', '2', '3']

5. Находим разделитель: ' =' (пробел,=)
   ⚠️ РАЗДЕЛИТЕЛЬ В КОНЦЕ СТРОКИ!
   
   Алгоритм: "взять всё от последнего токена до разделителя"
   Но после '3' до '=' ничего нет → пустая строка!
```

📊 Аналогия с `str.split()`:

```python
# Точно такой же принцип:
'1,2,3,'.split(',')    # ['1', '2', '3', '']
'1 2 3 '.split(' ')    # ['1', '2', '3', '']
```

💡 Как убрать пустые строки?

Вариант 1: Убрать вручную

```python
result = [x for x in re.split(r'\D+', '1 + 2 - 3 =') if x != '']
# ['1', '2', '3']
```

Вариант 2: Использовать filter()

```python
result = list(filter(None, re.split(r'\D+', '1 + 2 - 3 =')))
# ['1', '2', '3']
```

Вариант 3: Другой подход (findall)

```python
result = re.findall(r'\d+', '1 + 2 - 3 =')
# ['1', '2', '3']  ← только цифры, без пустых строк
```

🎯 Когда это полезно?

Пустые строки не всегда являются ошибкой:

```python
# CSV с пустыми полями в конце
data = 'name,age,city,'
re.split(',', data)  # ['name', 'age', 'city', ''] ← важно сохранить!

# Разбор команд с необязательными аргументами
cmd = 'copy file1 file2 '
re.split(r'\s+', cmd)  # ['copy', 'file1', 'file2', '']
```

📝 Правило:

Если разделитель находится в конце строки → создаётся пустая строка
Это поведение по дизайну, а не баг. Используйте findall() или фильтрацию, если пустые строки не нужны.
