# Памятка: Оператор assert в Python

## 📝 Назначение

assert — инструмент для написания утверждений (проверок) в коде:

- Отладка и тестирование
- Документирование ожиданий
- Выявление ошибок на ранних этапах

## 🎯 Синтаксис

python
# Базовая форма
assert условие

# С сообщением об ошибке
assert условие, "Сообщение при ошибке"


## ⚡ Как работает

- *Условие ИСТИНА* → код выполняется дальше
- *Условие ЛОЖЬ* → вызывается AssertionError

## 📚 Примеры использования

### Простая проверка

python
def calculate_square_root(x):
    assert x >= 0, "Число должно быть неотрицательным"
    return x ** 0.5


### Проверка входных данных

python
def divide(a, b):
    assert b != 0, "Делитель не должен быть нулем"
    return a / b


### Проверка типов (в разработке)

python
def process_data(data):
    assert isinstance(data, list), "Данные должны быть списком"
    assert len(data) > 0, "Список не должен быть пустым"


## ❌ Частые ошибки

### Неправильно (скобки!)

python
assert(2 + 2 == 5, "Сообщение")  # ОШИБКА!


*Почему ошибка:* Python видит кортеж (False, "Сообщение"), который всегда True

### Правильно

python
assert 2 + 2 == 5, "Сообщение"  # ВЕРНО


## 🔧 Эквивалентная запись

python
assert x > 0, "x должен быть положительным"

# Эквивалентно:
if not x > 0:
    raise AssertionError("x должен быть положительным")


## ⚠ Важные правила

### 1. Только для разработки

python
# НЕ для обработки пользовательского ввода
assert user_input.isdigit()  # ❌ Плохо

# Лучше использовать обычные проверки
if not user_input.isdigit():
    print("Введите число")    # ✅ Хорошо


### 2. Не обрабатывать через try-except

python
# ❌ Неправильно
try:
    assert condition
except AssertionError:
    # обработка...

# ✅ AssertionError должен указывать на баг в коде


### 3. Можно отключить

bash
python -O script.py  # Отключает все assert


## 🎯 Когда использовать

|✅ Подходит                    |❌ Не подходит                    |
|------------------------------|---------------------------------|
|Проверка инвариантов          |Валидация пользовательского ввода|
|Тестирование условий          |Обработка ожидаемых ошибок       |
|Документирование предположений|Проверки в production            |

## 💡 Лучшие практики

```
# Хорошие утверждения
assert len(results) == expected_count
assert hasattr(obj, 'required_method')
assert value is not None

# С информативными сообщениями
assert temperature > -273.15, "Температура ниже абсолютного нуля"
assert file_path.endswith('.txt'), f"Неверный формат: {file_path}"

```

-----

*Запомните:* assert — ваш помощник в разработке, а не механизм обработки ошибок!
