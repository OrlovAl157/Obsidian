# 🎨 Декораторы в Python

## 💡 Что такое декоратор?

**Декоратор** — это функция, которая принимает другую функцию, создаёт **новую функцию** с расширенным поведением и возвращает её.

```python
# Простой пример
@my_decorator
def hello():
    return "Hello"

# Это то же самое, что:
def hello():
    return "Hello"
hello = my_decorator(hello)
```

**Ключевая идея:** Декоратор создаёт новую функцию и **заменяет** ею исходную.

---

## 🏗️ Чертёж декоратора (шаблон)

```python
import functools

def decorator(func):
    @functools.wraps(func)  # Сохраняет __name__ и __doc__
    def wrapper(*args, **kwargs):
        # КОД ДО вызова декорируемой функции
        value = func(*args, **kwargs)
        # КОД ПОСЛЕ вызова функции
        # Или декорируем возвращаемое значение
        return value
    return wrapper
```

**⚠️ ВАЖНО:** Всегда используй `@functools.wraps(func)` чтобы сохранить метаданные функции!

## 🎯 Зачем нужны декораторы?

### 1. **DRY (Don't Repeat Yourself)**

Вместо копирования кода в каждую функцию:

```python
# ❌ БЕЗ декоратора - повторяем код
def function1():
    print("Начало работы")
    # основная логика
    print("Конец работы")

def function2():
    print("Начало работы")
    # основная логика
    print("Конец работы")

# ✅ С декоратором - пишем один раз
@log_execution
def function1():
    # только основная логика
    pass

@log_execution
def function2():
    # только основная логика
    pass
```

### 2. **Разделение ответственности**

Функция делает свою задачу, декоратор добавляет дополнительный функционал:

```python
@cache           # кэширование
@timer           # измерение времени
@validate_input  # проверка данных
def calculate(x, y):
    return x ** y  # только вычисления
```

### 3. **Легко включить/выключить функционал**

```python
# Нужно кэширование? Добавь декоратор:
@cache
def expensive_function(n):
    return sum(range(n))

# Не нужно? Просто убери @cache
def expensive_function(n):
    return sum(range(n))
```

## 📝 Базовая структура декоратора

### Минимальный декоратор

```python
def my_decorator(func):
    def wrapper():
        # код ДО вызова функции
        result = func()
        # код ПОСЛЕ вызова функции
        return result
    return wrapper

@my_decorator
def say_hello():
    return "Hello"

say_hello()  # вызовется wrapper, внутри которого вызовется say_hello
```

### С аргументами

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):  # принимаем любые аргументы
        print(f"Вызов {func.__name__} с {args}, {kwargs}")
        result = func(*args, **kwargs)
        print(f"Результат: {result}")
        return result
    return wrapper

@my_decorator
def add(a, b):
    return a + b

add(5, 3)
# Вывод:
# Вызов add с (5, 3), {}
# Результат: 8
```

## 🔧 functools.wraps - сохранение метаданных

### ❌ Проблема: потеря атрибутов **name** и **doc**

```python
def bold(func):
    def wrapper(*args, **kwargs):
        return '<b>' + func(*args, **kwargs) + '</b>'
    return wrapper

@bold
def greet(name):
    '''Функция приветствия пользователя.'''
    return f'Hello {name}!'

print(greet.__name__)  # → 'wrapper' ❌ (не 'greet')
print(greet.__doc__)   # → None ❌ (потерялось!)
```

### ✅ Решение: @functools.wraps

```python
import functools

def bold(func):
    @functools.wraps(func)  # ← ВАЖНО!
    def wrapper(*args, **kwargs):
        return '<b>' + func(*args, **kwargs) + '</b>'
    return wrapper

@bold
def greet(name):
    '''Функция приветствия пользователя.'''
    return f'Hello {name}!'

print(greet.__name__)  # → 'greet' ✅
print(greet.__doc__)   # → 'Функция приветствия пользователя.' ✅
```

**💡 Правило:** Всегда используй `@functools.wraps(func)` в своих декораторах!

## 🔍 Декоратор vs Новая функция

### ❌ НЕ декорирование (создание новой функции)

```python
def make_upper(func):
    def wrapper():
        return func().upper()
    return wrapper

def beegeek():
    return 'beegeek'

# Создаём НОВУЮ функцию
new_beegeek = make_upper(beegeek)

beegeek()      # → 'beegeek' (оригинал не изменился)
new_beegeek()  # → 'BEEGEEK' (новая функция)
```

**Это НЕ декорирование!** Старая функция осталась, создали новую.

---

### ✅ Декорирование (замена исходной функции)

```python
def make_upper(func):
    def wrapper():
        return func().upper()
    return wrapper

def beegeek():
    return 'beegeek'

# ЗАМЕНЯЕМ исходную функцию
beegeek = make_upper(beegeek)

beegeek()  # → 'BEEGEEK' (функция заменена!)
```

**Это декорирование!** Исходная функция заменена на новую версию.

---

### ✅ Синтаксический сахар @ (стандартный способ)

```python
@make_upper
def beegeek():
    return 'beegeek'

# Это ТОЧНО ТО ЖЕ САМОЕ, что:
# beegeek = make_upper(beegeek)

beegeek()  # → 'BEEGEEK'
```

**Это декорирование!** `@` — просто красивая запись.

## 📊 Сравнительная таблица

|Способ|Код|Это декоратор?|Старая функция?|
|---|---|---|---|
|**Новая переменная**|`new = decorator(old)`|❌ Нет|✅ Сохранена|
|**Замена**|`old = decorator(old)`|✅ Да|❌ Потеряна|
|**@ синтаксис**|`@decorator`|✅ Да|❌ Потеряна|

## 🎯 Практические примеры

### 1. Измерение времени выполнения

```python
import functools
import time

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        value = func(*args, **kwargs)
        end = time.perf_counter()
        work_time = end - start
        print(f'Время выполнения {func.__name__}: {round(work_time, 4)} сек.')
        return value
    return wrapper

@timer
def test(n):
    return sum([(i/99)**2 for i in range(n)])

@timer
def sleep(n):
    time.sleep(n)

res1 = test(10000)  # → Время выполнения test: 0.0015 сек.
res2 = sleep(2)     # → Время выполнения sleep: 2.0012 сек.
```

### 2. Подсчёт вызовов функции

```python
import functools

def counter(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        wrapper.calls += 1
        print(f'Вызов {func.__name__}: {wrapper.calls}')
        return func(*args, **kwargs)
    wrapper.calls = 0  # Атрибут функции!
    return wrapper

@counter
def greet(name):
    return f'Hello {name}!'

greet('Alice')  # → Вызов greet: 1
greet('Bob')    # → Вызов greet: 2
greet('Charlie')# → Вызов greet: 3

print(f'Всего вызовов: {greet.calls}')  # → 3
```

### 3. Замедление выполнения

```python
import functools
import time

def slow_down(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        time.sleep(1)  # Задержка 1 секунда
        return func(*args, **kwargs)
    return wrapper

@slow_down
def countdown(number):
    if number < 1:
        print('Конец!')
    else:
        print(number)
        countdown(number - 1)

countdown(5)
# → 5, 4, 3, 2, 1, Конец! (с задержкой в 1 сек)
```

### 4. Логирование вызовов

```python
import functools

def log_calls(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"→ Вызов {func.__name__}")
        print(f"  Аргументы: {args}, {kwargs}")
        result = func(*args, **kwargs)
        print(f"← {func.__name__} вернула: {result}")
        return result
    return wrapper

@log_calls
def multiply(a, b):
    return a * b

multiply(3, 4)
# Вывод:
# → Вызов multiply
#   Аргументы: (3, 4), {}
# ← multiply вернула: 12
```

### 5. Проверка типов аргументов

```python
import functools

def validate_types(*expected_types):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for arg, expected_type in zip(args, expected_types):
                if not isinstance(arg, expected_type):
                    raise TypeError(
                        f"Ожидался {expected_type.__name__}, "
                        f"получен {type(arg).__name__}"
                    )
            return func(*args, **kwargs)
        return wrapper
    return decorator

@validate_types(int, int)
def add(a, b):
    return a + b

add(5, 3)       # ✅ Работает
# add(5, "3")   # ❌ TypeError
```

### 6. Повтор при ошибке (retry)

```python
import functools
import time

def retry(max_attempts=3, delay=1):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            attempts = 0
            while attempts < max_attempts:
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    attempts += 1
                    print(f"Попытка {attempts} не удалась: {e}")
                    if attempts < max_attempts:
                        time.sleep(delay)
                    else:
                        raise
        return wrapper
    return decorator

@retry(max_attempts=3, delay=2)
def unstable_function():
    import random
    if random.random() < 0.7:
        raise Exception("Случайная ошибка")
    return "Успех!"
```

### Пример 4: Таймер с усреднением

```python
import functools
import time

def timer(iters=1):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            total = 0
            for i in range(iters):
                start = time.perf_counter()
                value = func(*args, **kwargs)
                end = time.perf_counter()
                total += end - start
            avg = total / iters
            print(f'Среднее время {func.__name__}: {round(avg, 4)} сек.')
            return value
        return wrapper
    return decorator

@timer(iters=1000)
def test(n):
    return sum([(i/99)**2 for i in range(n)])

test(10000)  # → Среднее время test: 0.0028 сек.
```

## 💾 Кэширование с @lru_cache

### Что такое кэширование?

**Кэширование** — сохранение результатов функции для повторного использования.

**Мемоизация** — кэширование возвращаемых значений функций.

### Декоратор @lru_cache

```python
from functools import lru_cache

@lru_cache(maxsize=128)  # Кэш на 128 элементов
def fibonacci(n):
    if n <= 2:
        return 1
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(100))  # Быстро! Кэш работает
```

**Параметры:**

- `maxsize=128` — размер кэша (None = бесконечный)
- `typed=False` — различать типы аргументов

### Пример: typed=False

```python
from functools import lru_cache

@lru_cache(typed=False)
def concat(text, num):
    return text + ' ' + str(num)

concat('hi', 4)    # Вычисляем
concat('hi', 4.0)  # Из кэша! (4 == 4.0)
```

### Пример: typed=True

```python
from functools import lru_cache

@lru_cache(typed=True)
def concat(text, num):
    return text + ' ' + str(num)

concat('hi', 4)    # Вычисляем
concat('hi', 4.0)  # Вычисляем снова! (4 != 4.0 по типу)
```

### Методы lru_cache

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def concat(text, num):
    return text + ' ' + str(num)

concat('hi', 1)
concat('hi', 1)   # Из кэша
concat('hi', 2)

# Статистика кэша
print(concat.cache_info())
# → CacheInfo(hits=1, misses=2, maxsize=128, currsize=2)

# Очистка кэша
concat.cache_clear()
print(concat.cache_info())
# → CacheInfo(hits=0, misses=0, maxsize=128, currsize=0)
```

### Декоратор @cache (Python 3.9+)

```python
from functools import cache

@cache  # = @lru_cache(maxsize=None)
def fibonacci(n):
    if n <= 2:
        return 1
    return fibonacci(n - 1) + fibonacci(n - 2)
```

**Отличия:**

- `@cache` — без ограничений, быстрее
- `@lru_cache` — с ограничением, меньше памяти

### ⚠️ Ограничения кэширования

**Можно кэшировать:**

- ✅ Чистые функции (без побочных эффектов)
- ✅ С неизменяемыми аргументами (int, str, tuple)

**Нельзя кэшировать:**

- ❌ Функции с побочными эффектами
- ❌ С изменяемыми аргументами (list, dict, set)

```python
@lru_cache
def bad_example(items):  # ❌ list не хешируемый
    return sum(items)

# bad_example([1, 2, 3])  # TypeError!

@lru_cache
def good_example(items):  # ✅ tuple хешируемый
    return sum(items)

good_example((1, 2, 3))  # ✅ Работает
```

## 🎨 Декораторы классов

```python
import functools

def singleton(cls):
    """Декоратор для создания Singleton"""
    instances = {}
    
    @functools.wraps(cls)
    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    
    return get_instance

@singleton
class Database:
    def __init__(self):
        print("Подключение к БД")

db1 = Database()  # → Подключение к БД
db2 = Database()  # Ничего не выводит
print(db1 is db2)  # → True (один объект!)
```

## ⚠️ Частые ошибки

### 1. Забыть вернуть wrapper

```python
# ❌ НЕПРАВИЛЬНО
def bad_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    # Забыли return wrapper!

@bad_decorator
def my_func():
    pass

# my_func теперь None!
```

### 2. Не использовать *args, **kwargs

```python
# ❌ НЕПРАВИЛЬНО
def bad_decorator(func):
    def wrapper():  # Не принимает аргументы!
        return func()
    return wrapper

@bad_decorator
def add(a, b):  # TypeError при вызове!
    return a + b
```

### 3. Забыть @functools.wraps

```python
# ❌ Потеря метаданных
def decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@decorator
def greet(name):
    """Приветствие"""
    return f"Hello {name}"

print(greet.__name__)  # → wrapper ❌
print(greet.__doc__)   # → None ❌

# ✅ С @functools.wraps
import functools

def decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@decorator
def greet(name):
    """Приветствие"""
    return f"Hello {name}"

print(greet.__name__)  # → greet ✅
print(greet.__doc__)   # → Приветствие ✅
```

### 4. Забыть скобки у декоратора с параметрами

```python
def repeater(times=1):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(times):
                func(*args, **kwargs)
        return wrapper
    return decorator

# ❌ НЕПРАВИЛЬНО
@repeater
def func():
    pass

# ✅ ПРАВИЛЬНО
@repeater()  # Нужны скобки!
def func():
    pass

@repeater(times=3)  # С аргументом
def func():
    pass
```

## 🎯 Когда использовать декораторы

**✅ Используй декораторы когда:**

- Нужно добавить один функционал ко многим функциям
- Хочешь разделить основную логику и вспомогательную
- Нужно легко включать/выключать функционал
- Работаешь с фреймворками (Flask, Django и т.д.)
- Нужно логирование, кэширование, проверка прав

**❌ НЕ используй декораторы когда:**

- Логика слишком сложная и запутывает код
- Функционал нужен только одной функции
- Проще написать напрямую

## 📚 Встроенные декораторы Python

```python
# @property - превращает метод в свойство
class Circle:
    def __init__(self, radius):
        self._radius = radius
    
    @property
    def radius(self):
        return self._radius
    
    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("Радиус >= 0")
        self._radius = value

# @staticmethod - статический метод
class Math:
    @staticmethod
    def add(a, b):
        return a + b

# @classmethod - метод класса
class Person:
    count = 0
    
    @classmethod
    def increment_count(cls):
        cls.count += 1

# @lru_cache - кэширование
from functools import lru_cache

@lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

# @cache - кэширование без ограничений (3.9+)
from functools import cache

@cache
def expensive_calc(n):
    return n ** 2
```

## 💭 Итоговая шпаргалка

### Базовый шаблон (ЗАПОМНИ!)

```python
import functools

def decorator(func):
    @functools.wraps(func)  # ← ОБЯЗАТЕЛЬНО!
    def wrapper(*args, **kwargs):
        # КОД ДО
        result = func(*args, **kwargs)
        # КОД ПОСЛЕ
        return result
    return wrapper

@decorator
def function():
    pass
```

### С параметрами

```python
import functools

def decorator(param):
    def actual_decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            # Используй param
            return func(*args, **kwargs)
        return wrapper
    return actual_decorator

@decorator("value")
def function():
    pass
```

### Ключевые правила

✅ **ВСЕГДА** используй `@functools.wraps(func)`  
✅ **ВСЕГДА** используй `*args, **kwargs` в wrapper  
✅ **ВСЕГДА** возвращай wrapper из decorator  
✅ **Декоратор с параметрами** = 3 уровня вложенности  
✅ **Множественные декораторы** применяются снизу вверх
