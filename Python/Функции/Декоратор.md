# 🎨 Декораторы в Python

## 💡 Что такое декоратор?

**Декоратор** — это функция, которая принимает другую функцию, **расширяет её поведение** и возвращает новую функцию.

```python
# Простой пример
@my_decorator
def hello():
    return "Hello"

# Это то же самое, что:
def hello():
    return "Hello"
hello = my_decorator(hello)
```

**Ключевая идея:** Декоратор **заменяет** исходную функцию на расширенную версию.

---

## 🎯 Зачем нужны декораторы?

### 1. **DRY (Don't Repeat Yourself)**

Вместо копирования кода в каждую функцию:

```python
# ❌ БЕЗ декоратора - повторяем код
def function1():
    print("Начало работы")
    # основная логика
    print("Конец работы")

def function2():
    print("Начало работы")
    # основная логика
    print("Конец работы")

# ✅ С декоратором - пишем один раз
@log_execution
def function1():
    # только основная логика
    pass

@log_execution
def function2():
    # только основная логика
    pass
```

### 2. **Разделение ответственности**

Функция делает свою задачу, декоратор добавляет дополнительный функционал:

```python
@cache           # кэширование
@timer           # измерение времени
@validate_input  # проверка данных
def calculate(x, y):
    return x ** y  # только вычисления
```

### 3. **Легко включить/выключить функционал**

```python
# Нужно кэширование? Добавь декоратор:
@cache
def expensive_function(n):
    return sum(range(n))

# Не нужно? Просто убери @cache
def expensive_function(n):
    return sum(range(n))
```

## 📝 Базовая структура декоратора

### Минимальный декоратор

```python
def my_decorator(func):
    def wrapper():
        # код ДО вызова функции
        result = func()
        # код ПОСЛЕ вызова функции
        return result
    return wrapper

@my_decorator
def say_hello():
    return "Hello"

say_hello()  # вызовется wrapper, внутри которого вызовется say_hello
```

### С аргументами

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):  # принимаем любые аргументы
        print(f"Вызов {func.__name__} с {args}, {kwargs}")
        result = func(*args, **kwargs)
        print(f"Результат: {result}")
        return result
    return wrapper

@my_decorator
def add(a, b):
    return a + b

add(5, 3)
# Вывод:
# Вызов add с (5, 3), {}
# Результат: 8
```

## 🔍 Декоратор vs Новая функция

### ❌ НЕ декорирование (создание новой функции)

```python
def make_upper(func):
    def wrapper():
        return func().upper()
    return wrapper

def beegeek():
    return 'beegeek'

# Создаём НОВУЮ функцию
new_beegeek = make_upper(beegeek)

beegeek()      # → 'beegeek' (оригинал не изменился)
new_beegeek()  # → 'BEEGEEK' (новая функция)
```

**Это НЕ декорирование!** Старая функция осталась, создали новую.

---

### ✅ Декорирование (замена исходной функции)

```python
def make_upper(func):
    def wrapper():
        return func().upper()
    return wrapper

def beegeek():
    return 'beegeek'

# ЗАМЕНЯЕМ исходную функцию
beegeek = make_upper(beegeek)

beegeek()  # → 'BEEGEEK' (функция заменена!)
```

**Это декорирование!** Исходная функция заменена на новую версию.

---

### ✅ Синтаксический сахар @ (стандартный способ)

```python
@make_upper
def beegeek():
    return 'beegeek'

# Это ТОЧНО ТО ЖЕ САМОЕ, что:
# beegeek = make_upper(beegeek)

beegeek()  # → 'BEEGEEK'
```

**Это декорирование!** `@` — просто красивая запись.

## 📊 Сравнительная таблица

|Способ|Код|Это декоратор?|Старая функция?|
|---|---|---|---|
|**Новая переменная**|`new = decorator(old)`|❌ Нет|✅ Сохранена|
|**Замена**|`old = decorator(old)`|✅ Да|❌ Потеряна|
|**@ синтаксис**|`@decorator`|✅ Да|❌ Потеряна|

## 🎯 Практические примеры

### 1. Измерение времени выполнения

```python
import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} выполнилась за {end - start:.4f} сек")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)
    return "Done"

slow_function()
# Вывод: slow_function выполнилась за 1.0012 сек
```

### 2. Кэширование результатов

```python
def cache(func):
    cached_results = {}
    
    def wrapper(*args):
        if args in cached_results:
            print(f"Из кэша: {args}")
            return cached_results[args]
        
        print(f"Вычисляем: {args}")
        result = func(*args)
        cached_results[args] = result
        return result
    
    return wrapper

@cache
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

fibonacci(5)
# Вывод:
# Вычисляем: (5,)
# Вычисляем: (4,)
# Вычисляем: (3,)
# Вычисляем: (2,)
# Вычисляем: (1,)
# Вычисляем: (0,)
# Из кэша: (1,)
# Из кэша: (2,)
# Из кэша: (3,)
```

**Встроенный вариант:**

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

### 3. Логирование вызовов

```python
def log_calls(func):
    def wrapper(*args, **kwargs):
        print(f"→ Вызов {func.__name__}")
        print(f"  Аргументы: {args}, {kwargs}")
        result = func(*args, **kwargs)
        print(f"← {func.__name__} вернула: {result}")
        return result
    return wrapper

@log_calls
def multiply(a, b):
    return a * b

multiply(3, 4)
# Вывод:
# → Вызов multiply
#   Аргументы: (3, 4), {}
# ← multiply вернула: 12
```

### 4. Проверка типов аргументов

```python
def validate_types(*expected_types):
    def decorator(func):
        def wrapper(*args, **kwargs):
            # Проверяем типы аргументов
            for arg, expected_type in zip(args, expected_types):
                if not isinstance(arg, expected_type):
                    raise TypeError(
                        f"Ожидался {expected_type.__name__}, "
                        f"получен {type(arg).__name__}"
                    )
            return func(*args, **kwargs)
        return wrapper
    return decorator

@validate_types(int, int)
def add(a, b):
    return a + b

add(5, 3)       # ✅ Работает
# add(5, "3")   # ❌ TypeError: Ожидался int, получен str
```

### 5. Подсчёт вызовов

```python
def count_calls(func):
    def wrapper(*args, **kwargs):
        wrapper.calls += 1
        print(f"Вызов #{wrapper.calls} функции {func.__name__}")
        return func(*args, **kwargs)
    
    wrapper.calls = 0
    return wrapper

@count_calls
def say_hello():
    return "Hello"

say_hello()  # Вызов #1
say_hello()  # Вызов #2
say_hello()  # Вызов #3
print(f"Всего вызовов: {say_hello.calls}")  # → 3
```

### 6. Повтор при ошибке (retry)

```python
import time

def retry(max_attempts=3, delay=1):
    def decorator(func):
        def wrapper(*args, **kwargs):
            attempts = 0
            while attempts < max_attempts:
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    attempts += 1
                    print(f"Попытка {attempts} не удалась: {e}")
                    if attempts < max_attempts:
                        time.sleep(delay)
                    else:
                        raise
        return wrapper
    return decorator

@retry(max_attempts=3, delay=2)
def unstable_function():
    import random
    if random.random() < 0.7:
        raise Exception("Случайная ошибка")
    return "Успех!"
```

### 7. Проверка прав доступа

```python
def require_permission(permission):
    def decorator(func):
        def wrapper(user, *args, **kwargs):
            if permission not in user.get("permissions", []):
                raise PermissionError(
                    f"Нет прав: {permission}"
                )
            return func(user, *args, **kwargs)
        return wrapper
    return decorator

@require_permission("admin")
def delete_user(user, user_id):
    print(f"Удаление пользователя {user_id}")
    return True

user = {"name": "John", "permissions": ["admin"]}
delete_user(user, 123)  # ✅ Работает

user = {"name": "Bob", "permissions": ["read"]}
# delete_user(user, 123)  # ❌ PermissionError
```

### 8. Отладочный декоратор

```python
def debug(func):
    def wrapper(*args, **kwargs):
        args_str = ", ".join(repr(arg) for arg in args)
        kwargs_str = ", ".join(f"{k}={v!r}" for k, v in kwargs.items())
        all_args = ", ".join(filter(None, [args_str, kwargs_str]))
        
        print(f"Вызов {func.__name__}({all_args})")
        result = func(*args, **kwargs)
        print(f"{func.__name__} → {result!r}")
        return result
    return wrapper

@debug
def make_greeting(name, prefix="Mr"):
    return f"{prefix} {name}"

make_greeting("John", prefix="Dr")
# Вывод:
# Вызов make_greeting('John', prefix='Dr')
# make_greeting → 'Dr John'
```

## 🔧 Декораторы с параметрами

### Структура декоратора с параметрами

```python
def decorator_with_params(param1, param2):
    def decorator(func):
        def wrapper(*args, **kwargs):
            # используем param1, param2
            print(f"Параметры: {param1}, {param2}")
            return func(*args, **kwargs)
        return wrapper
    return decorator

@decorator_with_params("hello", "world")
def my_function():
    return "Done"
```

**Три уровня вложенности:**

1. Внешняя функция — принимает параметры декоратора
2. Средняя функция — принимает декорируемую функцию
3. Внутренняя функция — wrapper, вызывается при вызове функции

### Пример: Повтор N раз

```python
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            results = []
            for _ in range(times):
                result = func(*args, **kwargs)
                results.append(result)
            return results
        return wrapper
    return decorator

@repeat(3)
def greet(name):
    return f"Hello, {name}!"

greet("Alice")
# → ['Hello, Alice!', 'Hello, Alice!', 'Hello, Alice!']
```

## 🎓 Продвинутые техники

### 1. Сохранение метаданных функции

**Проблема:**

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def add(a, b):
    """Складывает два числа"""
    return a + b

print(add.__name__)  # → 'wrapper' ❌
print(add.__doc__)   # → None ❌
```

**Решение: `functools.wraps`**

```python
from functools import wraps

def my_decorator(func):
    @wraps(func)  # сохраняет метаданные
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def add(a, b):
    """Складывает два числа"""
    return a + b

print(add.__name__)  # → 'add' ✅
print(add.__doc__)   # → 'Складывает два числа' ✅
```

### 3. Декораторы классов

```python
def singleton(cls):
    """Декоратор для создания Singleton"""
    instances = {}
    
    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    
    return get_instance

@singleton
class Database:
    def __init__(self):
        print("Подключение к БД")

db1 = Database()  # Вывод: Подключение к БД
db2 = Database()  # Ничего не выводит
print(db1 is db2)  # → True (один и тот же объект)
```

### 4. Декораторы методов класса

```python
class MyClass:
    def __init__(self):
        self.value = 0
    
    @property  # превращает метод в свойство
    def value_doubled(self):
        return self.value * 2
    
    @staticmethod  # статический метод
    def static_method():
        return "Статический метод"
    
    @classmethod  # метод класса
    def class_method(cls):
        return f"Метод класса {cls.__name__}"

obj = MyClass()
obj.value = 5
print(obj.value_doubled)  # → 10 (без скобок!)
```

## 🔨 Создание универсального декоратора

```python
from functools import wraps

def universal_decorator(func):
    """Шаблон универсального декоратора"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        # КОД ДО ВЫЗОВА ФУНКЦИИ
        print(f"Вызов {func.__name__}")
        
        # ВЫЗОВ ФУНКЦИИ
        try:
            result = func(*args, **kwargs)
        except Exception as e:
            # ОБРАБОТКА ОШИБОК
            print(f"Ошибка: {e}")
            raise
        
        # КОД ПОСЛЕ ВЫЗОВА ФУНКЦИИ
        print(f"Результат: {result}")
        
        return result
    
    return wrapper
```

## ⚠️ Частые ошибки

### 1. Забыть вернуть wrapper

```python
# ❌ НЕПРАВИЛЬНО
def bad_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    # забыли return wrapper!

@bad_decorator
def my_func():
    pass

# my_func теперь None!
```

### 2. Не использовать *args, **kwargs

```python
# ❌ НЕПРАВИЛЬНО - не работает с аргументами
def bad_decorator(func):
    def wrapper():  # не принимает аргументы!
        return func()
    return wrapper

@bad_decorator
def add(a, b):  # TypeError при вызове!
    return a + b
```

### 3. Забыть @wraps

```python
# ❌ Потеря метаданных
def decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

# ✅ С сохранением метаданных
from functools import wraps

def decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

### 4. Изменяемые значения по умолчанию

```python
# ❌ ОПАСНО
def bad_cache(func):
    cache = {}  # создаётся один раз
    def wrapper(x):
        if x not in cache:
            cache[x] = func(x)
        return cache[x]
    return wrapper

# ✅ ПРАВИЛЬНО - очищаем через атрибут
def good_cache(func):
    def wrapper(x):
        if x not in wrapper.cache:
            wrapper.cache[x] = func(x)
        return wrapper.cache[x]
    wrapper.cache = {}
    wrapper.clear_cache = lambda: wrapper.cache.clear()
    return wrapper
```

## 🎯 Когда использовать декораторы

**✅ Используй декораторы когда:**

- Нужно добавить один функционал ко многим функциям
- Хочешь разделить основную логику и вспомогательную (логирование, кэш)
- Нужно легко включать/выключать функционал
- Работаешь с фреймворками (Flask, Django и т.д.)

**❌ НЕ используй декораторы когда:**

- Логика слишком сложная и запутывает код
- Функционал нужен только одной функции
- Проще написать напрямую

```python
# @property - превращает метод в свойство
class Circle:
    def __init__(self, radius):
        self._radius = radius
    
    @property
    def radius(self):
        return self._radius
    
    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("Радиус не может быть отрицательным")
        self._radius = value

# @staticmethod - статический метод
class Math:
    @staticmethod
    def add(a, b):
        return a + b

# @classmethod - метод класса
class Person:
    count = 0
    
    @classmethod
    def increment_count(cls):
        cls.count += 1

# @lru_cache - кэширование
from functools import lru_cache

@lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

## 💭 Итоговая шпаргалка

```python
# Базовый шаблон
def decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        # до
        result = func(*args, **kwargs)
        # после
        return result
    return wrapper

# С параметрами
def decorator(param):
    def actual_decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            # используй param
            return func(*args, **kwargs)
        return wrapper
    return actual_decorator

# Применение
@decorator
def function():
    pass

# Это то же самое:
function = decorator(function)
```
