# 📖 Аннотации типов

> Python — язык с динамической типизацией: тип переменной определяется автоматически при присваивании значения. В Python 3.5 появились аннотации типов (PEP 484) — специальный синтаксис для явного указания типов параметров функций, возвращаемых значений и переменных. Аннотации не влияют на выполнение программы, но помогают IDE, статическим анализаторам (mypy, pyright) и другим разработчикам понять код.

## 🎯 Зачем нужна типизация

Python — язык с динамической типизацией: при определении переменных не нужно явно задавать тип данных, он устанавливается автоматически при присваивании значения. Это удобно для написания кода, но затрудняет поддержку большой кодовой базы — не всегда понятно, объекты каких типов используются в программе.

**В Python 3.5** появились аннотации типов — специальный синтаксис для явного указания типа данных параметров функций и их возвращаемых значений (PEP 484). **В Python 3.6** добавили возможность указывать аннотации в любом месте программы (PEP 526). С каждой новой версией языка эта функциональность улучшается.

Аннотации типов облегчают чтение кода и позволяют выявлять ошибки на ранних стадиях разработки. Большинство IDE подсвечивают места, где передается аргумент не того типа, который указан в аннотации, и дают подсказки об атрибутах и методах объекта.

**Важно:** аннотации типов — это **возможность, а не обязанность**. Программист сам решает, включать информацию о типах в свой код или нет. Python сохраняет динамическую типизацию, но при этом даёт возможность для статического анализа.

Аннотации **никак не влияют на выполнение программы**. Python не использует их напрямую, а лишь сохраняет в специальных атрибутах функций и переменных, делая доступными для сторонних утилит (mypy, IDE и т.д.).

```python
def greet(name: str) -> str:
    return f'Hello {name}!'

# Аннотация не мешает вызвать функцию с любым типом!
greet(123)  # Выполнится, хотя mypy укажет на ошибку
```

Если функция указана как принимающая строковый аргумент, это не мешает вызвать её с целым числом или списком. В зависимости от тела функции она может выполнить свою задачу или завершиться с ошибкой — возможность вызова с любыми типами аргументов никак не ограничивается.

## 🔤 Простые типы

Рассмотрим функцию без аннотаций:

```python
def greet(name):
    return f'Hello {name}!'
```

С аннотациями типов:

```python
def greet(name: str) -> str:
    return f'Hello {name}!'
```

Типы параметров указываются после имени параметра через двоеточие и пробел, а тип возвращаемого значения — после стрелки `->`. Теперь по одной сигнатуре сразу понятно, что функция принимает строку и возвращает строку.

**Необязательные параметры** — аннотация указывается между именем параметра и значением по умолчанию. По PEP 8 слева и справа от `=` ставится ровно один пробел:

```python
def greet(name: str = 'world') -> str:
    return f'Hello, {name}!'
```

Можно использовать любые базовые типы: `int`, `float`, `bool`, `str`. Для обозначения константы `None` используется сама константа, а не её тип `NoneType`.

**Пример 1.** Функция принимает `str` и `bool`, возвращает `None`:

```python
def print_hello(name: str, upper: bool = False) -> None:
    if upper:
        name = name.upper()
    print(f'Hello, {name}')
```

**Пример 2.** Функция принимает три `int`, возвращает `float`:

```python
def avg(num1: int, num2: int, num3: int) -> float:
    return (num1 + num2 + num3) / 3
```

## 💾 Аннотирование переменных

В Python 3.6+ можно объявлять типы переменных в любом месте программы:

```python
name: str = 'Timur'
age: int = 33
height: float = 171.5
is_married: bool = False
```

Можно аннотировать переменные, не назначая им сразу значения:

```python
surname: str
gender: bool
```

**Когда аннотировать переменные не нужно:** обычно в коде это не требуется, так как при явном указании значения сразу понятен и его тип.

**Когда аннотировать нужно:** если переменная получает результат выполнения функции из **сторонней библиотеки**, разработчики которой могли не указать тип возвращаемых данных.

**Когда аннотация избыточна:** если присваиваешь переменной результат выполнения **собственной** функции — аннотацию можно не указывать, поскольку она уже определена в самой функции, дублировать её не нужно.

## 📦 Типы коллекций

Можно добавлять аннотации типов, используя сами имена классов `list`, `tuple`, `set`, `dict`:

```python
def sum_square(nums: list) -> int:
    total = 0
    for i in nums:
        total += i ** 2
    return total
```

Это не слишком информативно — непонятно, элементы какого типа хранит список.

### Тип list

Чтобы указать тип элементов, нужный тип указывается в квадратных скобках:

```python
def sum_square(nums: list[int]) -> int:
    total = 0
    for i in nums:
        total += i ** 2
    return total
```

Список — изменяемая коллекция, и это один из случаев, когда переменную стоит аннотировать: даже с говорящим именем (`numbers`) непонятно, какого типа данные должен хранить пустой список.

### Тип tuple

Кортежи обычно хранят **фиксированное** количество элементов, необязательно одного типа. Например, кортеж из двух элементов может обозначать координаты точки, а из трёх — широту, долготу и направление.

```python
def func(arg: tuple[int]) -> None:
    pass
```

Функция ожидает кортеж из **одного** элемента типа `int`. Чтобы указать другое количество элементов, перечисляем типы в скобках:

```python
coordinates: tuple[float, float, str]
```

Ожидается кортеж из трёх элементов: два `float` и `str`.

Если нужен кортеж **произвольной** длины одного типа, вторым элементом указывается `Ellipsis` (`...`):

```python
def sum_square(nums: tuple[int, ...]) -> int:
    total = 0
    for i in nums:
        total += i ** 2
    return total
```

### Типы set и dict

Аннотации множеств похожи на списки — один тип в скобках означает множество произвольного количества элементов этого типа. Для словарей указываются **два** типа: для ключей и для значений.

```python
answers: set[float]
prices: dict[str, int]
```

`answers` — множество вещественных чисел произвольного количества. `prices` — словарь, где ключи `str`, значения `int`.

> Указание **точного** количества элементов множества или словаря аннотациями не предусмотрено — они неупорядоченные коллекции. Однако в Python 3.8 в модуль `typing` добавили тип `TypedDict` для точного указания ключей словаря.

> Аннотирование коллекций встроенными типами (`list[int]`) доступно с **Python 3.9**. В более ранних версиях нужно использовать `List`, `Tuple`, `Set`, `Dict` из модуля `typing`.

## 👤 Пользовательские типы

Чтобы в аннотациях указать экземпляр пользовательского типа данных, используется имя этого типа:

```python
class Cat:
    pass

cat: Cat
```

Для переменной `cat` ожидается значение в виде экземпляра класса `Cat`.

**Однако** при попытке указать похожую аннотацию **внутри метода этого же класса** возникает ошибка:

```python
class Cat:
    def method(self) -> Cat:  # ❌ NameError!
        return self
```

```
NameError: name 'Cat' is not defined
```

**Почему так происходит:** весь код в теле класса выполняется **до создания самого класса**. Сначала создаются все атрибуты и методы, затем создаётся класс, после чего атрибуты и методы связываются с ним. Аннотации вычисляются сразу при создании объектов — поэтому когда мы указываем такую аннотацию, класс ещё не существует. Аннотация `Cat` в данном случае является **опережающей ссылкой**.

## ⏪ Forward reference — опережающие ссылки

**Опережающая ссылка** (forward reference) — ссылка, указывающая на объект, который определён в коде **позже** самой ссылки.

**Решение:** имя объекта заключается в кавычки, полученная строка указывается как аннотация. Строка должна содержать корректное выражение Python, вычислимое функцией `eval()`.

```python
class Cat:
    def method(self) -> 'Cat':  # ✅ Работает!
        return self
```

-----

## 🪞 Тип Self

В **Python 3.11** в модуль `typing` добавили тип `Self`, который можно использовать вместо явного указания имени класса:

```python
from typing import Self

class Cat:
    def method(self) -> Self:
        return self
```

## 🏭 Аннотация самого типа — type[Cat]

Чтобы в аннотациях указать **непосредственно сам класс**, а не его экземпляр, используется тип `type`, в квадратных скобках указывается имя класса:

```python
class Cat:
    pass

def create_cat(cls: type[Cat]) -> Cat:
    return Cat()
```

Если аннотацию нужно указать **внутри метода класса**, имя также заключается в кавычки:

```python
class Cat:
    def create_cat(cls: type['Cat']) -> 'Cat':
        return Cat()
```

Функция `create_cat()` ожидает на вход **сам класс** `Cat`, а возвращает **экземпляр** класса `Cat`.

> Аннотации самих типов данных пригождаются в метапрограммировании и при разработке фреймворков.

## ⭐ Аннотации *args и **kwargs

Синтаксис аннотаций для `*args` и `**kwargs` не отличается от обычного:

```python
def func(*args: str, **kwargs: int) -> None:
    pass
```

Функция ожидает произвольное количество позиционных аргументов типа `str` и произвольное количество именованных аргументов типа `int`.

**Важный нюанс:** в теле функции для переменных `args` и `kwargs` типы данных определяются как `tuple[str, ...]` и `dict[str, int]` соответственно (не `str` и `int` напрямую!).

## 📚 Типы из модулей и библиотек

Стандартная библиотека Python содержит модули с дополнительными типами данных. Например, `datetime` включает типы для работы с датой и временем:

```python
from datetime import date

def func(arg: date) -> None:
    pass
```

Можно использовать полное имя типа, начиная с имени модуля:

```python
import datetime

def func(arg: datetime.date) -> None:
    pass
```

Также можно использовать типы данных **сторонних библиотек**:

```python
import pandas as pd

def func(arg: pd.DataFrame) -> None:
    pass
```

Функция ожидает на вход объект `DataFrame` из библиотеки `pandas`.

## 🔍 Атрибут **annotations**

Доступ к аннотациям функции можно получить через атрибут `__annotations__` — словарь, где ключи — имена параметров, значения — их типы. Возвращаемое значение хранится под ключом `'return'`.

```python
def avg(num1: int, num2: int, num3: int) -> float:
    return (num1 + num2 + num3) / 3

print(avg.__annotations__)
```

Выводит:

```python
{'num1': <class 'int'>, 'num2': <class 'int'>, 'num3': <class 'int'>, 'return': <class 'float'>}
```

### Вложенные аннотации коллекций

Аннотации коллекций могут быть вложенными:

```python
def create_matrix(n: int) -> list[list[int]]:
    return [[i + j for j in range(n)] for i in range(n)]
```

### Python 3.14: отложенное вычисление аннотаций

В **Python 3.14** вычисление аннотаций стало **отложенным**. Теперь они вычисляются не в момент создания объекта, а только при обращении к аннотации (например, через `__annotations__`). Это позволяет указывать аннотации более гибко — например, использовать имя класса внутри его же методов **без forward reference**:

```python
class Cat:
    def method(self) -> Cat:  # без кавычек!
        return self

print(Cat.method.__annotations__)
```

**В Python 3.13** это приводит к ошибке:

```
NameError: name 'Cat' is not defined
```

**В Python 3.14** выводит:

```python
{'return': <class '__main__.Cat'>}
```

## 💻 Примеры

### Пример 1: Полный набор простых типов

```python
def print_hello(name: str, upper: bool = False) -> None:
    """Приветствие с опцией верхнего регистра"""
    if upper:
        name = name.upper()
    print(f'Hello, {name}')

def avg(num1: int, num2: int, num3: int) -> float:
    """Среднее значение трёх чисел"""
    return (num1 + num2 + num3) / 3

# Использование
print_hello('Alice')          # Hello, Alice
print_hello('Bob', True)      # Hello, BOB
result = avg(1, 2, 3)         # 2.0
```

### Пример 2: Коллекции

```python
def get_coordinates() -> tuple[float, float]:
    """Координаты точки"""
    return (3.14, 2.71)

def count_words(text: str) -> dict[str, int]:
    """Подсчёт слов в тексте"""
    result: dict[str, int] = {}
    for word in text.split():
        result[word] = result.get(word, 0) + 1
    return result

def unique_letters(word: str) -> set[str]:
    """Уникальные буквы в слове"""
    return set(word)

x, y = get_coordinates()
counts = count_words("the cat sat on the mat")
letters = unique_letters("hello")
```

### Пример 3: Пользовательский класс с forward reference и Self

```python
from typing import Self

class Node:
    def __init__(self, value: int) -> None:
        self.value = value
        self.next: 'Node | None' = None  # forward reference
    
    def set_next(self, node: 'Node') -> Self:
        """Установить следующий узел (chaining)"""
        self.next = node
        return self

# Использование
node1 = Node(1)
node2 = Node(2)
node1.set_next(node2)
```

### Пример 4: Фабричный метод с type[Cat]

```python
class Animal:
    def __init__(self, name: str) -> None:
        self.name = name

def create_animal(cls: type[Animal], name: str) -> Animal:
    """Создать животное через переданный класс"""
    return cls(name)

# Использование
animal = create_animal(Animal, "Rex")
```

### Пример 5: *args и **kwargs с типами

```python
def log_message(*args: str, **kwargs: int) -> None:
    """Логировать сообщения с числовыми параметрами"""
    message = ' '.join(args)
    params = ', '.join(f"{k}={v}" for k, v in kwargs.items())
    print(f"{message} [{params}]")

# Использование
log_message("Error", "occurred", code=500, retries=3)
# Error occurred [code=500, retries=3]
```

### Пример 6: Типы из модулей

```python
from datetime import date
import pandas as pd

def days_until(target: date) -> int:
    """Количество дней до целевой даты"""
    return (target - date.today()).days

def load_dataframe(path: str) -> pd.DataFrame:
    """Загрузить DataFrame из файла"""
    return pd.read_csv(path)
```

### Пример 7: Работа с **annotations**

```python
def process(data: list[int], threshold: float = 0.5) -> dict[str, int]:
    return {"count": len(data)}

print(process.__annotations__)
# {'data': list[int], 'threshold': <class 'float'>, 'return': dict[str, int]}

print(process.__annotations__['data'])   # list[int]
print(process.__annotations__['return']) # dict[str, int]
```

## ⚠️ Частые ошибки

```python
# ❌ Забыли стрелку для возвращаемого значения
def add(a: int, b: int) int:  # SyntaxError!
    return a + b

# ✅ Правильно
def add(a: int, b: int) -> int:
    return a + b

# ❌ Использование имени класса внутри его же методов без кавычек (до 3.14)
class Cat:
    def method(self) -> Cat:  # ❌ NameError!
        return self

# ✅ Forward reference
class Cat:
    def method(self) -> 'Cat':
        return self

# ❌ Забыли второй тип для Dict
prices: dict[str] = {}  # Error!

# ✅ Правильно
prices: dict[str, int] = {}

# ❌ Забыли, что *args превращается в tuple, а **kwargs в dict
def func(*args: str) -> None:
    total: str = args  # ❌ Type mismatch! args это tuple[str, ...]

# ✅ Правильно
def func(*args: str) -> None:
    total: tuple[str, ...] = args

# ❌ Думать, что аннотация проверяется при выполнении
def add(a: int, b: int) -> int:
    return a + b

add('hello', 'world')  # ❌ Type checker скажет "ошибка"
                       # ✅ Но Python выполнит: 'helloworld'
```

## ✅ Главные правила

✅ **Начните с функций** — аннотируйте сигнатуры в первую очередь  
✅ **Аннотации не влияют на выполнение** — это только для анализа и читаемости  
✅ **Всегда указывайте тип элементов коллекций** — `list[int]`, а не просто `list`  
✅ **Forward reference для своего класса внутри методов** — используйте кавычки `'ClassName'`  
✅ **Self лучше явного имени класса** — особенно для method chaining  
✅ **type[Cat] для самого класса**, `Cat` — для экземпляра  
✅ **`*args`/`**kwargs` типизируют отдельные элементы**, а не сам tuple/dict  
✅ **Модули и сторонние библиотеки** можно использовать в аннотациях напрямую  
✅ **`__annotations__`** — словарь с типами, `'return'` — для возвращаемого значения  
✅ **Python 3.14** — аннотации вычисляются лениво, forward reference не всегда нужен
