## 🎯 Главная идея

`print()` — это не «магическая команда вывода на экран». Это **функция-обёртка** вокруг `.write()` объекта-потока.

Что делает `print()` под капотом:

1. Берёт переданные объекты (`*args`).
2. Преобразует каждый в строку через `str()`.
3. Соединяет их через `sep`.
4. Добавляет в конец `end`.
5. Передаёт готовую строку в `file.write()`.

```text
print(obj1, obj2)
  ↓
str(obj1), str(obj2)
  ↓
соединение через sep
  ↓
добавление end
  ↓
file.write(готовая_строка)
```

По умолчанию `file = sys.stdout` — подробнее про сами потоки см. [[00 — 📥 stdin, stdout, stderr]].

## 📐 Сигнатура print()

```python
print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)
```

| Параметр | По умолчанию | Что делает |
|---|---|---|
| `*objects` | — | произвольное количество объектов для вывода |
| `sep` | `' '` | разделитель между объектами |
| `end` | `'\n'` | что добавляется в конце вывода |
| `file` | `sys.stdout` | куда писать (любой объект с `.write()`) |
| `flush` | `False` | принудительно сбросить буфер сразу |

```python
print(10)
# str(10) → '10'
# sys.stdout.write('10' + '\n')
```

```python
sys.stdout.write(10)  # ❌ TypeError: write() argument must be str, not int
sys.stdout.write('10')  # ✅
```

**Важное:** `print()` сам вызывает `str()` за тебя — поэтому работает с любыми объектами (числами, списками, своими классами). А `.write()` принимает только строки.

## 🔗 sep и end

`*objects` — это произвольное число позиционных аргументов. `sep` склеивает их между собой, `end` добавляется в самом конце один раз.

```python
print('A', 'B', 'C', sep='-', end='!')
# 1. str() для каждого: 'A', 'B', 'C'
# 2. соединение через sep: 'A-B-C'
# 3. добавление end: 'A-B-C!'
# 4. sys.stdout.write('A-B-C!')
```

```python
print(10, 20, sep='-', end='!')
# равнозначно:
sys.stdout.write('10-20!')
```

**Почему `end='\n'` по умолчанию:** именно поэтому каждый `print()` начинается с новой строки. Если поставить `end=''`, следующий вывод окажется на той же строке.

```python
print('Hello', end='')
print('World')
# HelloWorld -- вывод склеился в одну строку
```

**Важное:** если передан **один** объект, `sep` ни на что не влияет — разделять нечего.

## 🔡 str() vs repr()

`print()` вызывает **`str()`**, а не `repr()`. Это объясняет, почему строки выводятся без кавычек.

```python
s = 'hello'
print(s)        # hello       -- str('hello') без кавычек
print(repr(s))  # 'hello'     -- repr('hello') с кавычками
```

Если в классе не определён `__str__()`, Python использует `__repr__()` как запасной вариант.

```python
class Point:
    def __repr__(self):
        return f'Point(x, y)'

p = Point()
print(p)  # Point(x, y) -- сработал __repr__, потому что __str__ не определён
```

**Важное:** для собственных классов всегда стоит определять `__repr__()` (для отладки/однозначности) и при необходимости `__str__()` (для удобного вывода пользователю). `print()` использует второй, если он есть.

## 📤 file и утиная типизация

По умолчанию `print()` пишет в `sys.stdout`, но можно передать любой объект через `file=`:

```python
with open('log.txt', 'w') as f:
    print('Hello', file=f)
# равнозначно: f.write('Hello' + '\n')
```

**Главное:** `print()` не проверяет, является ли `file` "настоящим файлом". Ему важно только одно — есть ли у объекта метод `.write()`.

```python
class MyOutput:
    def write(self, text):
        print(f'Получено: {text}', end='')

print('Hello', file=MyOutput())
# Получено: Hello
```

Это пример **утиной типизации** (duck typing):

> Если объект умеет `.write(строка)` — для `print()` он подходит как поток вывода. Не важно, файл это, сетевой сокет или собственный класс.

**Важное:** перенаправление через `sys.stdout = ...` влияет на *все* вызовы `print()` в программе. `file=` влияет только на конкретный вызов — это безопаснее. Подробнее про способы перенаправления — [[01 — 🔄 Перенаправление потоков]].

## 📊 Общая схема

```text
print(10, 20, sep='-', end='!')
  ↓
str(10) → '10'
str(20) → '20'
  ↓
соединение через sep → '10-20'
  ↓
добавление end → '10-20!'
  ↓
file.write('10-20!')
  ↓
по умолчанию file = sys.stdout
```

## ✅ Главные правила

✅ `print()` сам вызывает `str()` для каждого аргумента — поэтому работает с числами, списками, объектами
✅ `sep` соединяет аргументы между собой, `end` добавляется один раз в конце
✅ `end='\n'` по умолчанию — поэтому каждый `print()` начинается с новой строки
✅ `print()` использует `__str__()`, а если его нет — `__repr__()`
✅ `file=` принимает любой объект с `.write()` — утиная типизация
✅ `file=` действует только на один вызов, в отличие от смены `sys.stdout`
