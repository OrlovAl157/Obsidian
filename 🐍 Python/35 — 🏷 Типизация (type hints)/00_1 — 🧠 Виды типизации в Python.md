# 🧠 Виды типизации в Python

> Типизация описывает, когда и как язык проверяет соответствие типов. Python использует динамическую, сильную и неявную типизацию. Type hints добавляют возможность статического анализа, но не меняют сам язык. Понимание видов типизации помогает осознать, почему Python устроен так, как он устроен.

## 📊 Статическая vs динамическая

### Статическая типизация

Типы проверяются **ДО запуска программы** (при компиляции или анализе):

```java
// Java (статический язык)
int data = 17;
data = "Hello";  // ❌ ОШИБКА КОМПИЛЯЦИИ — видна сразу!
```

Переменная `data` получает тип `int` и больше не может быть строкой. Ошибка ловится до выполнения.

### Динамическая типизация

Типы проверяются **ВО ВРЕМЯ ВЫПОЛНЕНИЯ**:

```python
# Python (динамический язык)
data = 17         # data хранит int
data = "Hello"    # теперь data хранит str — нет ошибки!

# Ошибка произойдёт только если попытаешься операцию, недопустимую для типа:
result = "Hello" + 5  # ❌ TypeError при выполнении
```

Тип связан со **значением**, а не с переменной. Переменная может содержать что угодно.

**В контексте Python:** сам язык динамический, но type hints позволяют **проверять статически** с помощью mypy.

## 💪 Сильная vs слабая типизация

### Сильная типизация

Язык **не любит** неявные опасные преобразования:

```python
# Python (сильный язык)
"hello" + 5  # ❌ TypeError — не будет складывать строку и число!
```

### Слабая типизация

Язык **с удовольствием** преобразует типы:

```javascript
// JavaScript (слабый язык)
"hello" + 5        // "hello5" — число приведено к строке!
4 * "3"            // 12 — строка приведена к числу!
2 + true           // 3 — true приведено к 1!
false - 4          // -4 — false приведено к 0!
```

**Python почти всегда сильный**, но есть одно исключение:

```python
True + 1           # 2 — bool наследует от int!
False * 10         # 0 — false приведено к 0!
```

## 👁️ Явная vs неявная типизация

### Явная типизация

Типы пишутся **В КОДЕ**:

```java
// Java
int age = 25;
String name = "Alice";
List<Integer> numbers = new ArrayList<>();
```

### Неявная типизация

Типы **ВЫВОДЯТСЯ АВТОМАТИЧЕСКИ**:

```python
# Python
age = 25           # ясно, что int
name = "Alice"     # ясно, что str
numbers = [1, 2, 3]  # ясно, что list[int]
```

Программист не пишет типы, язык сам их определяет по значениям.

## 🐍 Python и его типизация

Python использует **комбинацию**:

- ✅ **Динамический** — тип проверяется во время выполнения
- ✅ **Сильный** — редко неявных преобразований
- ✅ **Неявно типизирован** — типы выводятся по значениям
- ✅ **Поддерживает аннотации** — можно добавить подсказки типов
- ✅ **Статический анализ** — mypy/pyright проверяют аннотации заранее

## 💻 Примеры

### Пример 1: Динамическая типизация в действии

```python
def process(data):
    return len(data)

# Работает со ВСЕМ, что имеет __len__():
print(process([1, 2, 3]))        # 3 (список)
print(process("hello"))          # 5 (строка)
print(process({"a": 1, "b": 2})) # 2 (словарь)
print(process(123))              # ❌ TypeError — int нет __len__()

# Это нормально для Python — гибкость!
```

### Пример 2: Сильная типизация (защита от ошибок)

```python
# ❌ Python отказывается складывать разные типы
numbers = [1, 2, 3]
try:
    result = sum(numbers) + "extra"  # TypeError!
except TypeError as e:
    print(f"Ошибка: {e}")
    # can only concatenate str (not "int") to str

# Это хорошо — Python предотвращает ошибки
```

### Пример 3: Неявная типизация (удобство)

```python
x = 5          # Не пишу "int x = 5", Python сам понимает
name = "Bob"   # Не пишу 'str name = "Bob"'
values = []    # Не пишу 'list[int] values = []'

# IDE и статические анализаторы ВСЁ РАВНО понимают типы
# (если код понятен — например, явное присваивание)
```

### Пример 4: Type hints добавляют статичность

```python
# Без аннотаций (полная динамичность)
def add(a, b):
    return a + b

add(1, 2)       # 3
add("1", "2")   # "12" (конкатенация строк)

# С аннотациями (статическая проверка возможна)
def add_typed(a: int, b: int) -> int:
    return a + b

add_typed(1, 2)        # ✅ OK
add_typed("1", "2")    # ❌ mypy скажет "ошибка типа!"
                       # ✅ Но Python выполнит нормально
```

## ⚠️ Частые ошибки

```python
# ❌ Думать, что Python слабо типизирован
result = "hello" + 5  # TypeError — нет!

# ✅ Python сильно типизирован (с редкими исключениями)

# ❌ Думать, что type hints обязательны
def add(a, b):
    return a + b  # Это нормально! Аннотации опциональны

# ✅ Аннотации — это выбор разработчика

# ❌ Путать "динамическая типизация" с "нет проверки типов"
x = 5
print(x.upper())  # ❌ AttributeError — проверка ВСЕ РАВНО есть!

# ✅ Динамическая типизация = проверка во время выполнения

# ❌ Думать, что type hints влияют на выполнение программы
def func(x: int) -> int:
    return x + 1

func("string")  # ❌ mypy скажет "ошибка"
                # ✅ Python выполнит и выдаст TypeError при '+1'
```

## ✅ Главные правила

✅ **Python динамический** — тип определяется значением, проверяется при выполнении  
✅ **Python сильный** — редко неявных преобразований (с исключениями)  
✅ **Python неявно типизирован** — типы выводятся автоматически  
✅ **Type hints опциональны** — это выбор, не требование  
✅ **Type hints = статический анализ** — mypy проверяет их заранее  
✅ **Type hints не влияют на выполнение** — только на анализ  
✅ **Аннотации помогают IDE** — автодополнение, подсказки, рефакторинг
