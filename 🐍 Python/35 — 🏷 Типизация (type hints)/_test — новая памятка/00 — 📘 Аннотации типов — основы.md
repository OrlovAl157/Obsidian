# 📘 Аннотации типов в Python — основы

> [!info] Коротко
> Аннотации типов в Python нужны не для того, чтобы язык стал "строго типизированным", а для того, чтобы код было проще читать, проверять и поддерживать. Они помогают IDE, `mypy`, `pyright` и самому разработчику понимать, какие значения ожидаются на входе и выходе.

## Справка по теме

| Что | Синтаксис | Пример | Зачем |
|---|---|---|---|
| Параметр функции | `name: Type` | `name: str` | показать ожидаемый тип |
| Возврат функции | `-> Type` | `-> int` | показать результат |
| Переменная | `var: Type = value` | `age: int = 18` | улучшить читаемость |
| Список | `list[Type]` | `list[str]` | тип элементов |
| Словарь | `dict[K, V]` | `dict[str, int]` | тип ключей и значений |
| Неизвестный тип | `Any` | `data: Any` | временная гибкость |
| Значение или `None` | `T | None` | `str | None` | nullable-значения |

## Что такое аннотации типов

Аннотация типа — это подсказка, прикрепленная к переменной, параметру, атрибуту или возвращаемому значению функции. Она не меняет природу Python: язык остается динамическим. Переменная не "фиксируется" навсегда одним типом только потому, что мы рядом написали `: int`.

Главная польза аннотаций в другом:

- код становится самодокументируемым;
- статические анализаторы умеют заранее находить ошибки;
- редактор лучше подсказывает методы и сигнатуры;
- рефакторинг становится спокойнее.

Пример без аннотаций:

```python
def repeat(value, count):
    return value * count
```

Сама функция короткая, но неясно, что именно она принимает. Строку? Число? Список?

Тот же пример с аннотациями:

```python
def repeat(value: str, count: int) -> str:
    return value * count
```

Теперь контракт читается сразу: функция берет строку и число, а возвращает строку.

> [!note] Важно
> Аннотации — это договоренность и источник информации. Они не заменяют знание языка и не отменяют обычные ошибки времени выполнения.

## Что именно можно аннотировать

Обычно аннотируют четыре вещи:

1. Параметры функций.
2. Возвращаемые значения.
3. Переменные.
4. Атрибуты классов и объектов.

Примеры:

```python
name: str = "Timur"
is_active: bool = True

def greet(user: str) -> str:
    return f"Hello, {user}"

class User:
    id: int
    email: str
```

Можно аннотировать и переменную без немедленного присваивания:

```python
total: float
message: str
```

Это удобно, когда значение появится позже, а тип уже хочется зафиксировать для читаемости и проверки.

Также аннотируют `*args` и `**kwargs`:

```python
def add_all(*args: int) -> int:
    return sum(args)

def print_user(**kwargs: str) -> None:
    for key, value in kwargs.items():
        print(key, value)
```

Здесь `*args: int` означает: каждый позиционный аргумент должен быть `int`. А `**kwargs: str` означает: каждое значение в именованных аргументах должно быть строкой.

## Базовые типы

Чаще всего начинают со встроенных типов Python:

```python
age: int = 25
price: float = 99.9
name: str = "Alice"
is_admin: bool = False
nothing: None = None
```

Это самый понятный и полезный слой типизации. Даже если больше ничего не писать, аннотации функций и базовых переменных уже сильно улучшают код.

Для функций это выглядит так:

```python
def square(x: int) -> int:
    return x * x

def format_price(value: float) -> str:
    return f"{value:.2f}"
```

В учебных и рабочих проектах именно сигнатуры функций обычно дают максимум пользы при минимуме усилий.

> [!tip] Практический смысл
> Если не знаешь, с чего начать типизацию, начинай с функций: входные параметры и возврат. Это самое выгодное место.

## Коллекции и вложенные типы

Аннотации особенно полезны для контейнеров, потому что простого `list` почти всегда мало. Важно знать, что именно лежит внутри.

```python
names: list[str] = ["Ann", "Bob", "Clara"]
scores: list[int] = [10, 20, 30]
point: tuple[int, int] = (3, 7)
tags: set[str] = {"python", "typing"}
ages: dict[str, int] = {"Ann": 20, "Bob": 25}
```

Можно и глубже:

```python
users: list[dict[str, str]] = [
    {"name": "Ann", "email": "ann@example.com"},
    {"name": "Bob", "email": "bob@example.com"},
]
```

Когда типы вложенные, аннотации начинают реально экономить время: без них структуру приходится угадывать по месту использования.

Если нужно поддерживать старые версии Python, можно встретить старый синтаксис из `typing`:

```python
from typing import List, Dict, Tuple

names: List[str] = ["Ann", "Bob"]
ages: Dict[str, int] = {"Ann": 20}
point: Tuple[int, int] = (1, 2)
```

Но в современном Python обычно пишут короче: `list[str]`, `dict[str, int]`, `tuple[int, int]`.

## Пользовательские типы и прямые ссылки

Аннотации могут ссылаться не только на встроенные типы, но и на собственные классы:

```python
class Cat:
    def __init__(self, name: str) -> None:
        self.name = name


def print_cat(cat: Cat) -> None:
    print(cat.name)
```

Иногда возникает проблема, если тип упоминается раньше, чем класс объявлен. Тогда используют строковую аннотацию:

```python
class Node:
    def __init__(self, value: int, next_node: "Node | None" = None) -> None:
        self.value = value
        self.next_node = next_node
```

Это называется forward reference — отложенная ссылка на тип.

В современных версиях Python тема стала удобнее, но сам принцип полезно понимать: иногда аннотация — это не только тип, но и зависимость от порядка объявления.

## Атрибут __annotations__

Python хранит аннотации в специальном атрибуте `__annotations__`.

Пример:

```python
value: int = 10

def greet(name: str) -> str:
    return f"Hello, {name}"
```

У функции:

```python
print(greet.__annotations__)
```

Результат будет примерно таким:

```python
{'name': <class 'str'>, 'return': <class 'str'>}
```

У классов и модулей аннотации тоже можно исследовать. Это полезно знать, потому что типы в Python — не просто комментарий в воздухе, а реально сохраненная метаинформация.

## Что аннотации не делают

Вот важное ограничение, о которое часто спотыкаются:

```python
def divide(a: int, b: int) -> float:
    return a / b

divide("10", 2)
```

Сам Python не обязан остановить такой вызов заранее из-за аннотации. Ошибка проявится только при выполнении операции.

То есть аннотации:

- не превращают Python в Java;
- не запрещают присваивать "не тот" тип во время выполнения;
- не заменяют `isinstance()`, валидацию и здравый смысл;
- не чинят код сами по себе.

Чтобы аннотации начали приносить максимум пользы, рядом нужны инструменты вроде `mypy` или `pyright`.

> [!warning] Частая ошибка
> Новички иногда думают, что `x: int = "hello"` немедленно вызовет исключение только из-за аннотации. В обычном Python это не так.

---

## Главные правила

- Аннотируй в первую очередь сигнатуры функций.
- Для коллекций почти всегда указывай тип элементов, а не только `list` или `dict`.
- Предпочитай современный синтаксис `list[str]`, `dict[str, int]`, `str | None`, если версия Python это позволяет.
- Помни, что аннотации помогают анализу кода, но не заменяют runtime-проверки.
- Если тип слишком сложный и мешает чтению, его лучше вынести в отдельный псевдоним.
