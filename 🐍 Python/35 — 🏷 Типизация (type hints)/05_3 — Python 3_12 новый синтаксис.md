# ⚡ Python 3.12 новый синтаксис Generic

> Python 3.12 добавил **PEP 695** — новый встроенный синтаксис для параметров типов. Вместо использования `TypeVar`, `Generic[T]` и сложных конструкций, теперь можно писать `class Box[T]:` и `def func[T](x: T) -> T:` напрямую. Это делает generic типы намного проще и понятнее.

## Справка

|Старый синтаксис (Python < 3.12)|Новый синтаксис (Python 3.12+)      |Что это                           |
|--------------------------------|------------------------------------|----------------------------------|
|`T = TypeVar('T')`              |Не нужно!                           |Просто пиши `[T]` в функции       |
|`def func(x: T) -> T:`          |`def func[T](x: T) -> T:`           |Параметр типа в сигнатуре         |
|`class Box(Generic[T]):`        |`class Box[T]:`                     |Параметр типа в определении класса|
|`T = TypeVar('T', bound=int)`   |`def func[T: int](x: T) -> T:`      |Bound через `:` в параметре       |
|`T = TypeVar('T', int, str)`    |`def func[T: int | str](x: T) -> T:`|Constraints через `|`             |

## 🎯 Generic функции

### Старый синтаксис (до 3.12)

```python
from typing import TypeVar

T = TypeVar('T')

def identity(x: T) -> T:
    return x

def first_element(seq: list[T]) -> T:
    return seq[0]
```

### Новый синтаксис (3.12+)

```python
def identity[T](x: T) -> T:
    return x

def first_element[T](seq: list[T]) -> T:
    return seq[0]

# Использование (идентично старому синтаксису)
result = identity(42)           # int
value = first_element([1, 2, 3])  # int
```

**Преимущества:**

- ✅ Параметры типа явно видны в сигнатуре
- ✅ Не нужно создавать `TypeVar` отдельно
- ✅ Читается как обычный параметр функции

## 🏗️ Generic классы

### Старый синтаксис (до 3.12)

```python
from typing import TypeVar, Generic

T = TypeVar('T')

class Box(Generic[T]):
    def __init__(self, item: T):
        self.item = item
    
    def get(self) -> T:
        return self.item
```

### Новый синтаксис (3.12+)

```python
class Box[T]:
    def __init__(self, item: T):
        self.item = item
    
    def get(self) -> T:
        return self.item

# Использование
int_box: Box[int] = Box(42)
value = int_box.get()  # int

str_box: Box[str] = Box("hello")
message = str_box.get()  # str
```

**Что изменилось:**

- ❌ Больше не нужен `Generic` импорт
- ❌ Больше не нужен `TypeVar`
- ✅ Параметры типа прямо в скобках класса

## 🔀 Несколько параметров типа

### Старый синтаксис (до 3.12)

```python
from typing import TypeVar, Generic

K = TypeVar('K')
V = TypeVar('V')

class Pair(Generic[K, V]):
    def __init__(self, key: K, value: V):
        self.key = key
        self.value = value
```

### Новый синтаксис (3.12+)

```python
class Pair[K, V]:
    def __init__(self, key: K, value: V):
        self.key = key
        self.value = value

# Использование
pair: Pair[str, int] = Pair("age", 30)
```

## 📏 Type bounds в новом синтаксисе

### Старый синтаксис (до 3.12)

```python
from typing import TypeVar

NumT = TypeVar('NumT', bound=int)

def double(x: NumT) -> NumT:
    return x * 2
```

### Новый синтаксис (3.12+)

Bounds указываются через `:` (как в Java):

```python
def double[T: int](x: T) -> T:
    return x * 2

# Использование
result1 = double(5)    # ✅ int
result2 = double(5.5)  # ✅ float
# result3 = double("5")  # ❌ Error: str не совместим с int
```

## 💻 Примеры

### Пример 1: Простая Generic функция

```python
def swap[T, U](a: T, b: U) -> tuple[U, T]:
    """Поменять два значения местами"""
    return b, a

# Использование
result = swap(42, "hello")
print(result)  # ("hello", 42)
```

### Пример 2: Generic класс со stack

```python
class Stack[T]:
    """Стек элементов"""
    
    def __init__(self):
        self.items: list[T] = []
    
    def push(self, item: T) -> None:
        self.items.append(item)
    
    def pop(self) -> T | None:
        return self.items.pop() if self.items else None
    
    def is_empty(self) -> bool:
        return len(self.items) == 0

# Использование
int_stack: Stack[int] = Stack()
int_stack.push(1)
int_stack.push(2)
print(int_stack.pop())  # 2

str_stack: Stack[str] = Stack()
str_stack.push("hello")
print(str_stack.pop())  # "hello"
```

### Пример 3: Generic с несколькими bounds

```python
from typing import Protocol

class Comparable[T]:
    """Всё, что можно сравнивать"""
    
    def compare(self, other: T) -> int:
        """Сравнить с другим (возвращает -1, 0 или 1)"""
        raise NotImplementedError

class Person(Comparable["Person"]):
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age
    
    def compare(self, other: "Person") -> int:
        if self.age < other.age:
            return -1
        elif self.age > other.age:
            return 1
        return 0

# Использование
alice = Person("Alice", 30)
bob = Person("Bob", 25)
print(alice.compare(bob))  # 1 (alice старше)
```

### Пример 4: Generic функция с constraint

```python
def process_number[T: int | float](value: T) -> T:
    """Обработать число (int или float)"""
    return value * 2

# Использование
print(process_number(5))     # 10 (int)
print(process_number(5.5))   # 11.0 (float)
# print(process_number("5"))  # ❌ Error!
```

### Пример 5: Наследование Generic класса

```python
class Container[T]:
    """Контейнер с базовой функциональностью"""
    
    def __init__(self):
        self.items: list[T] = []
    
    def add(self, item: T) -> None:
        self.items.append(item)
    
    def get_all(self) -> list[T]:
        return self.items

class FilteredContainer[T](Container[T]):
    """Контейнер с фильтрацией"""
    
    def get_filtered(self, predicate: callable[[T], bool]) -> list[T]:
        return [item for item in self.items if predicate(item)]

# Использование
container: FilteredContainer[int] = FilteredContainer()
container.add(1)
container.add(2)
container.add(3)
container.add(4)

even_numbers = container.get_filtered(lambda x: x % 2 == 0)
print(even_numbers)  # [2, 4]
```

## ⚠️ Частые ошибки

```python
# ❌ Перепутали синтаксис (старый синтаксис в Python 3.12)
from typing import TypeVar, Generic

T = TypeVar('T')
class Box(Generic[T]):  # Работает, но устаревший стиль!
    pass

# ✅ Используй новый синтаксис в 3.12+
class Box[T]:
    pass

# ❌ Забыли параметр типа в функции
def identity(x):  # Потеряется информация о типе!
    return x

# ✅ Правильно
def identity[T](x: T) -> T:
    return x

# ❌ Неправильный синтаксис bound (Java стиль)
def func[T extends int](x: T) -> T:  # ❌ Error!
    pass

# ✅ Правильно (Python синтаксис)
def func[T: int](x: T) -> T:
    pass

# ❌ Constraints как Union (неправильный синтаксис)
def process[T: int | str](value: T) -> T:  # ❌ Это bound, не constraints!
    pass

# ✅ Для constraints нужна сложнее конструкция (нет простого синтаксиса)
# Используй Union вместо этого:
def process[T](value: T) -> T:
    return value
```

## 🔄 Миграция со старого синтаксиса

### Шаг 1: Функция без параметров типа

**Было:**

```python
def add(a: int, b: int) -> int:
    return a + b
```

**Стало:** (не меняется)

```python
def add(a: int, b: int) -> int:
    return a + b
```

### Шаг 2: Функция с TypeVar

**Было:**

```python
from typing import TypeVar

T = TypeVar('T')

def identity(x: T) -> T:
    return x
```

**Стало:**

```python
def identity[T](x: T) -> T:
    return x
```

### Шаг 3: Generic класс

**Было:**

```python
from typing import TypeVar, Generic

T = TypeVar('T')

class Box(Generic[T]):
    def __init__(self, item: T):
        self.item = item
```

**Стало:**

```python
class Box[T]:
    def __init__(self, item: T):
        self.item = item
```

## ✅ Главные правила

✅ **Python 3.12+ новый синтаксис проще** — `class Box[T]:` вместо `class Box(Generic[T]):`  
✅ **Параметры типа в сигнатуре** — `def func[T](x: T) -> T:` вместо отдельного `TypeVar`  
✅ **Bounds через `:`** — `def func[T: int](x: T):` вместо `TypeVar('T', bound=int)`  
✅ **Constraints можно через `|`** — `def func[T: int | str](x: T):` (но это bound, не constraints)  
✅ **Старый синтаксис всё ещё работает** — но новый нагляднее  
✅ **Требует Python 3.12+** — проверь версию!
