# 📋 Протоколы и абстрактные классы

> **Протоколы** — способ определить “утиную” типизацию (duck typing) явно через тип. Если класс имеет все методы из Protocol, он автоматически совместим. **Абстрактные базовые классы (ABC)** — способ определить интерфейс, который **обязательно** имплементировать. Протоколы лучше для структурной типизации, ABC лучше для явных контрактов.

## Справка

|Подход                |Синтаксис                      |Когда                |
|----------------------|-------------------------------|---------------------|
|**Protocol**          |`class MyProtocol(Protocol):`  |Структурная типизация|
|**ABC**               |`class MyABC(ABC):`            |Явный контракт       |
|**@abstractmethod**   |`@abstractmethod def method():`|Метод, который нужен |
|**@runtime_checkable**|`@runtime_checkable`           |Проверка в runtime   |

## 🦆 Протоколы

Протокол определяет, какие методы должны быть у класса:

```python
from typing import Protocol

class Drawable(Protocol):
    """Все, что можно рисовать"""
    
    def draw(self) -> str:
        """Должен быть метод draw"""
        ...

class Circle:
    def draw(self) -> str:
        return "● (circle)"

class Square:
    def draw(self) -> str:
        return "■ (square)"

def render(shape: Drawable) -> None:
    print(shape.draw())

# Использование
render(Circle())   # ✅ Circle имеет draw()
render(Square())   # ✅ Square имеет draw()
```

Circle и Square **не наследуются** от Drawable, но mypy считает их совместимыми, потому что они имеют нужные методы.

## 🏛️ Абстрактные базовые классы

ABC заставляет подклассы имплементировать абстрактные методы:

```python
from abc import ABC, abstractmethod

class Animal(ABC):
    """Базовый класс для животных"""
    
    @abstractmethod
    def speak(self) -> str:
        """Все животные должны издавать звуки"""
        pass
    
    @abstractmethod
    def move(self) -> str:
        """Все животные должны двигаться"""
        pass

class Dog(Animal):
    def speak(self) -> str:
        return "Woof!"
    
    def move(self) -> str:
        return "Running on four legs"

class Bird(Animal):
    def speak(self) -> str:
        return "Tweet!"
    
    def move(self) -> str:
        return "Flying"

# Использование
dog = Dog()
dog.speak()  # "Woof!"

# ❌ Это ошибка — Animal не может быть создан (он абстрактный)
# animal = Animal()  # TypeError
```

## 🤝 Протоколы vs ABC

|Критерий              |Protocol               |ABC               |
|----------------------|-----------------------|------------------|
|**Наследование**      |Не требуется           |**Требуется**     |
|**Сигнатура**         |Структурная (утиная)   |Явная (контракт)  |
|**Runtime check**     |С `@runtime_checkable` |По умолчанию      |
|**Гибкость**          |Больше                 |Меньше            |
|**Явность**           |Меньше                 |Больше            |
|**Когда использовать**|Внешний код, библиотеки|Собственный проект|

## 💻 Примеры

### Пример 1: Protocol для сравнения

```python
from typing import Protocol

class Comparable(Protocol):
    """Всё, что можно сравнивать"""
    
    def __lt__(self, other: object) -> bool:
        ...
    
    def __eq__(self, other: object) -> bool:
        ...

class Person:
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age
    
    def __lt__(self, other: object) -> bool:
        if isinstance(other, Person):
            return self.age < other.age
        return NotImplemented
    
    def __eq__(self, other: object) -> bool:
        if isinstance(other, Person):
            return self.age == other.age
        return NotImplemented

def find_min(items: list[Comparable]) -> Comparable:
    return min(items)  # type: ignore

# Использование
people = [Person("Alice", 30), Person("Bob", 25)]
youngest = find_min(people)
```

### Пример 2: ABC для обработчиков

```python
from abc import ABC, abstractmethod

class EventHandler(ABC):
    """Базовый класс для обработчиков событий"""
    
    @abstractmethod
    def handle(self, event: dict) -> None:
        """Обработать событие"""
        pass
    
    @abstractmethod
    def can_handle(self, event: dict) -> bool:
        """Может ли обработать это событие"""
        pass

class ClickHandler(EventHandler):
    def handle(self, event: dict) -> None:
        print(f"Click handled: {event}")
    
    def can_handle(self, event: dict) -> bool:
        return event.get("type") == "click"

class KeyHandler(EventHandler):
    def handle(self, event: dict) -> None:
        print(f"Key pressed: {event}")
    
    def can_handle(self, event: dict) -> bool:
        return event.get("type") == "key"

# Использование
handlers = [ClickHandler(), KeyHandler()]
for handler in handlers:
    event = {"type": "click"}
    if handler.can_handle(event):
        handler.handle(event)
```

### Пример 3: Protocol для сериализации

```python
from typing import Protocol

class Serializable(Protocol):
    """Всё, что можно сериализовать"""
    
    def to_dict(self) -> dict:
        ...
    
    @classmethod
    def from_dict(cls, data: dict) -> "Serializable":
        ...

class User:
    def __init__(self, name: str, email: str):
        self.name = name
        self.email = email
    
    def to_dict(self) -> dict:
        return {"name": self.name, "email": self.email}
    
    @classmethod
    def from_dict(cls, data: dict) -> "User":
        return cls(data["name"], data["email"])

def save(obj: Serializable) -> None:
    data = obj.to_dict()
    print(f"Saved: {data}")

# Использование
user = User("Alice", "alice@example.com")
save(user)  # ✅ User имеет нужные методы
```

### Пример 4: Runtime checkable protocol

```python
from typing import Protocol, runtime_checkable

@runtime_checkable
class Drawable(Protocol):
    def draw(self) -> str:
        ...

class Circle:
    def draw(self) -> str:
        return "● "

class Square:
    def draw(self) -> str:
        return "■ "

class Point:
    pass

# Runtime проверка
objects = [Circle(), Square(), Point()]
for obj in objects:
    if isinstance(obj, Drawable):  # ✅ Runtime проверка!
        print(obj.draw())
```

## ⚠️ Частые ошибки

```python
from typing import Protocol
from abc import ABC, abstractmethod

# ❌ Protocol без методов (неинформативно)
class Empty(Protocol):
    pass

# ✅ Protocol с методами
class Drawable(Protocol):
    def draw(self) -> str:
        ...

# ❌ Забыли @abstractmethod
class Animal(ABC):
    def speak(self):  # ❌ Это не обязательно имплементировать!
        pass

# ✅ Правильно
class Animal(ABC):
    @abstractmethod
    def speak(self) -> str:
        ...

# ❌ Думать, что Protocol требует наследования
class MyProtocol(Protocol):
    def method(self) -> None:
        ...

class MyClass:  # ❌ Забыли наследоваться?
    def method(self) -> None:
        pass

# ✅ Protocol НЕ требует наследования (это структурная типизация!)
class MyClass:
    def method(self) -> None:
        pass

# ✅ mypy считает совместимым
obj: MyProtocol = MyClass()
```

-----

## ✅ Главные правила

✅ **Protocol для структурной типизации** — не требует наследования  
✅ **ABC для явного контракта** — требует наследования и имплементации  
✅ **Protocol когда не контролируешь код** — например, внешние библиотеки  
✅ **ABC когда контролируешь код** — в своём проекте  
✅ **@runtime_checkable для runtime проверок** — когда нужна проверка при выполнении  
✅ **@abstractmethod обязателен** — для обозначения необходимых методов
