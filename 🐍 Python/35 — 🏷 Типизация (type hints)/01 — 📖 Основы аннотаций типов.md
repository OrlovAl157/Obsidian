# 📖 Основы аннотаций типов в Python

## 📌 Коротко

> Аннотации типов показывают, какие данные ожидаются в переменных, параметрах и возвращаемых значениях функций. Они не заставляют Python автоматически проверять типы во время выполнения, но делают код понятнее для человека и инструментов анализа.

## 🎯 Зачем нужны аннотации

Аннотации типов решают в первую очередь не задачу выполнения программы, а задачу чтения и сопровождения кода.

Они помогают:

- быстрее понять контракт функции;
- раньше замечать ошибки через IDE и статические анализаторы;
- упростить рефакторинг;
- сделать большой код предсказуемее.

Важно: аннотации сами по себе не запрещают передать "не тот" тип.

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"

print(greet(100))
```

Такой вызов Python всё равно попытается выполнить. Ошибка или корректный результат будут зависеть уже от логики внутри функции.

## 🧱 Базовый синтаксис

### Функции

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"
```

- тип параметра пишется после имени через `:`
- тип возвращаемого значения пишется после `->`

### Параметр по умолчанию

```python
def greet(name: str = "world") -> str:
    return f"Hello, {name}!"
```

### Переменные

```python
name: str = "Timur"
age: int = 33
height: float = 171.5
is_admin: bool = False
```

Можно аннотировать и без значения:

```python
surname: str
is_married: bool
```

---

## 📊 Краткая справка

| Что аннотируем | Синтаксис | Пример |
| --- | --- | --- |
| Параметр функции | `name: Type` | `name: str` |
| Возвращаемое значение | `-> Type` | `-> int` |
| Переменная | `var: Type = value` | `age: int = 33` |
| Список | `list[T]` | `list[int]` |
| Кортеж | `tuple[T1, T2]` | `tuple[str, int]` |
| Словарь | `dict[K, V]` | `dict[str, float]` |
| Сам класс | `type[Class]` | `type[Cat]` |
| Экземпляр класса | `Class` | `Cat` |

## 🔢 Простые типы

Для простых значений обычно используют:

- `int`
- `float`
- `bool`
- `str`
- `None`

### Примеры

```python
def print_hello(name: str, upper: bool = False) -> None:
    if upper:
        name = name.upper()
    print(f"Hello, {name}")


def avg(num1: int, num2: int, num3: int) -> float:
    return (num1 + num2 + num3) / 3
```

### Таблица

| Аннотация | Когда используется | Пример |
| --- | --- | --- |
| `int` | целые числа | `count: int = 10` |
| `float` | вещественные числа | `price: float = 99.9` |
| `bool` | логические значения | `is_active: bool = True` |
| `str` | строки | `name: str = "Alice"` |
| `None` | функция ничего не возвращает | `def log() -> None:` |

## 🗃 Аннотации коллекций

Современный Python использует встроенные обобщённые типы:

- `list[int]`
- `tuple[int, str]`
- `set[float]`
- `dict[str, int]`

### Списки

```python
def sum_square(nums: list[int]) -> int:
    total = 0
    for num in nums:
        total += num ** 2
    return total
```

Здесь важно не просто то, что это `list`, а то, что это список целых чисел.

### Кортежи

```python
point: tuple[float, float] = (10.5, 20.0)
user: tuple[str, int] = ("timur", 33)
coordinates: tuple[float, float, str] = (55.75, 37.61, "north")
```

Кортеж фиксированной формы обычно аннотируют перечислением типов по позициям.

Если нужен кортеж произвольной длины, но из одного типа:

```python
nums: tuple[int, ...] = (1, 2, 3, 4)
```

### Множества и словари

```python
answers: set[float] = {1.5, 2.5, 3.5}
prices: dict[str, int] = {"apple": 100, "banana": 80}
```

### Вложенные аннотации

```python
def create_matrix(n: int) -> list[list[int]]:
    return [[i + j for j in range(n)] for i in range(n)]
```

## 🏛 Пользовательские типы

Если у тебя есть свой класс, его имя тоже можно использовать в аннотациях.

```python
class Cat:
    pass


cat: Cat
```

### Проблема внутри класса

Иногда при ссылке на текущий класс внутри него возникает опережающая ссылка.

```python
class Cat:
    def clone(self) -> "Cat":
        return self
```

Такой строковый вариант долгое время был стандартным способом избежать ошибки.

Почему вообще возникает `NameError`: тело класса выполняется до того, как сам класс окончательно создаётся. В момент вычисления аннотации имя `Cat` внутри этого же класса ещё может быть недоступно.

### Современный вариант

```python
from typing import Self


class Cat:
    def clone(self) -> Self:
        return self
```

`Self` здесь означает “экземпляр того же самого класса”. Это удобно, когда метод возвращает `self` или нужен fluent-style код с цепочкой вызовов.

### Что изменилось в Python 3.14

Начиная с Python 3.14 вычисление аннотаций стало отложенным. Из-за этого аннотации со ссылкой на сам класс стали работать гибче, и запись вроде `-> Cat` теперь во многих случаях возможна без кавычек.

### `Self` и `type[Cat]` - это не одно и то же

- `Self` используют, когда метод возвращает экземпляр текущего класса
- `type[Cat]` используют, когда функция или метод принимает сам класс `Cat`, а не его экземпляр

```python
class Cat:
    pass


def clone_cat(cat: Self) -> Self:
    return cat


def create_cat(cls: type[Cat]) -> Cat:
    return cls()
```

### Если нужен сам класс, а не экземпляр

```python
class Cat:
    pass


def create_cat(cls: type[Cat]) -> Cat:
    return cls()
```

## ⚙️ *args, **kwargs и внешние типы

### Переменное количество аргументов

```python
def func(*args: str, **kwargs: int) -> None:
    pass
```

Это означает:

- `args` содержит позиционные аргументы типа `str`
- `kwargs` содержит именованные аргументы со значениями типа `int`

Внутри функции это соответствует примерно таким типам:

- `args -> tuple[str, ...]`
- `kwargs -> dict[str, int]`

### Типы из модулей и библиотек

Можно использовать типы стандартной библиотеки:

```python
from datetime import date


def is_holiday(day: date) -> bool:
    return False
```

И типы сторонних библиотек:

```python
import pandas as pd


def normalize_frame(frame: pd.DataFrame) -> pd.DataFrame:
    return frame
```

## 🔍 Атрибут __annotations__

Python сохраняет аннотации в специальном атрибуте `__annotations__`.

Важно: `__annotations__` — это атрибут, а не метод. Обычно это словарь, в котором Python хранит аннотации как метаданные.

### Где встречается

`__annotations__` можно встретить:

- у функции
- у класса
- у модуля

Примеры:

```python
func.__annotations__
User.__annotations__
__annotations__
```

### Что хранит

Обычно:

- ключ — имя параметра, атрибута или переменной
- значение — объект типа или аннотация, которая была указана в коде

Например:

```python
class User:
    name: str
    age: int
```

Тогда:

```python
print(User.__annotations__)
```

даст примерно:

```python
{
    "name": str,
    "age": int,
}
```

```python
def avg(num1: int, num2: int, num3: int) -> float:
    return (num1 + num2 + num3) / 3


print(avg.__annotations__)
```

Результат будет примерно таким:

```python
{
    "num1": int,
    "num2": int,
    "num3": int,
    "return": float,
}
```

### Это обычный атрибут

К нему можно обратиться напрямую:

```python
User.__annotations__
```

Его даже можно изменить:

```python
User.__annotations__["name"] = int
User.__annotations__ = {}
```

Python не запрещает это делать, потому что `__annotations__` — обычный атрибут со служебными данными.

### Что это не делает

Изменение `__annotations__` не меняет реальные объекты и их значения.

```python
x: int = 5
__annotations__["x"] = str
```

После этого `x` всё равно останется числом `5`. Изменится только описание в словаре аннотаций.

### Когда создаётся

У функции `__annotations__` появляется при создании функции. Даже если аннотаций нет, часто это просто пустой словарь:

```python
def func():
    pass

print(func.__annotations__)  # {}
```

У класса `__annotations__` появляется при создании класса, если на уровне класса есть аннотированные атрибуты:

```python
class User:
    name: str

print(User.__annotations__)  # {'name': str}
```

У модуля `__annotations__` появляется, если в модуле есть аннотированные переменные:

```python
x: int
```

Если таких переменных нет, имени `__annotations__` может вообще не быть.

### Где лучше писать аннотации атрибутов

Обычно лучше так:

```python
class User:
    name: str
```

чем только так:

```python
self.name: str = value
```

Когда аннотация находится на уровне класса, структура данных лучше видна через `User.__annotations__`, а значит её легче использовать IDE, анализаторам типов, `dataclass` и другим библиотекам.

### Самая важная мысль

`__annotations__` — это словарь с метаданными об аннотациях. Python не "исполняет" их напрямую и не превращает автоматически в runtime-проверку типов.

## 💡 Практические замечания

- Аннотируй в первую очередь публичные функции, методы и сложные структуры данных.
- Если тип и так очевиден из литерала, аннотация переменной часто не обязательна.
- Для коллекций почти всегда полезнее указывать внутренний тип, а не просто `list` или `dict`.
- Если функция принимает сам класс, а не объект этого класса, используй `type[Cat]`, а не просто `Cat`.
- Для нового кода лучше использовать современный синтаксис: `list[int]`, `str | None`, `type Alias = ...`.

## ⚠️ Частые ошибки

### ❌ Аннотация без внутреннего типа коллекции

```python
def process(items: list) -> None:
    pass
```

Непонятно, какие элементы ожидаются.

### ✅ Лучше так

```python
def process(items: list[str]) -> None:
    pass
```

### ❌ Путать `None` и `NoneType`

```python
def log() -> NoneType:
    pass
```

### ✅ Правильно

```python
def log() -> None:
    pass
```

### ❌ Думать, что аннотация автоматически проверяет типы

```python
def add(a: int, b: int) -> int:
    return a + b


add("1", "2")
```

Аннотация сама по себе не остановит вызов.
