# 🧠 Виды типизации в Python

## 📌 Коротко

> Типизация описывает, как язык связывает данные с их типами и когда он проверяет, что операции с этими данными допустимы.

Python относится к языкам с динамической и сильной типизацией, а `type hints` добавляют к нему удобный слой подсказок для людей и инструментов.

## 📚 Что такое типизация

Типизация в программировании отвечает за то, как язык работает с типами данных:

- как он понимает, что значение — это число, строка, список или что-то другое;
- когда он проверяет совместимость типов;
- разрешает ли он смешивать разные типы в выражениях;
- помогает ли он программисту заранее заметить ошибку.

В разных языках это устроено по-разному. Поэтому полезно понимать не только "есть типизация или нет", а именно какой она бывает.

## 🧭 Основные виды

### Статическая и динамическая

Это самая важная пара понятий — **когда** проверяются типы.

**Статическая типизация**:

- типы проверяются **до запуска программы**;
- ошибки типов ловятся на этапе компиляции или анализа;
- переменная получает тип один раз и больше его не меняет;
- код часто становится более предсказуемым и быстрым.

Пример (Java):
```java
int data = 17;
data = "Hello";  // ❌ Ошибка компиляции!
```

**Динамическая типизация**:

- тип связан со **значением**, а не с переменной;
- типы проверяются, когда программа реально доходит до операции;
- переменная может хранить значения разных типов в разные моменты времени;
- гибче, но требует внимательности.

Пример (Python):
```python
data = 17      # data хранит int
data = "Hello" # теперь data хранит str — нет ошибки!
```

### Сильная и слабая

Это уже не про **момент** проверки, а про **жёсткость**: насколько язык терпимо смешивает типы.

**Сильная типизация**:

- язык не любит неявные опасные преобразования;
- если операция бессмысленна, будет ошибка;
- надёжнее, но требует явных преобразований.

Пример (Python):
```python
data = "python"
new_data = data + 2022  # ❌ TypeError!
```

**Слабая типизация**:

- язык может **сам привести типы** друг к другу;
- удобнее для коротких скриптов, но может привести к неожиданным результатам.

Примеры (JavaScript):
```javascript
5 + '6'        // '56' (число приведено к строке!)
4 * '3'        // 12 (строка приведена к числу)
2 + true       // 3 (true приведено к 1)
false - 4      // -4 (false приведено к 0)
```

### Явная и неявная

**Явная типизация**:

- типы пишутся **прямо в коде**;
- программист сам показывает, что ожидается;
- дольше писать, но ясно читается.

Пример (Java):
```java
int data = 17;
String name = "Alice";
List<Integer> numbers = new ArrayList<>();
```

**Неявная типизация**:

- тип **выводится** по значению;
- язык или инструмент сам догадывается;
- код компактнее, но иногда неясно.

Пример (Python):
```python
data = 17          # ясно, что int
name = "Alice"     # ясно, что str
```

## 📊 Типизация в популярных языках

| Язык | Статическая / Динамическая | Сильная / Слабая | Явная / Неявная |
|------|---|---|---|
| **Python** | Динамическая | Сильная | Неявная |
| **JavaScript** | Динамическая | Слабая | Неявная |
| **Ruby** | Динамическая | Сильная | Неявная |
| **Java** | Статическая | Сильная | Явная |
| **C#** | Статическая | Сильная | Явная |
| **C** | Статическая | Слабая | Явная |
| **C++** | Статическая | Слабая | Явная |
| **Go** | Статическая | Сильная | Явная |
| **Haskell** | Статическая | Сильная | Неявная |
| **PHP** | Динамическая | Слабая | Неявная |

## 🐍 Python и типизация

Python:

- **динамический** — тип проверяется во время выполнения;
- **сильный** — редко и неохотно приводит типы неявно;
- **неявно типизирован** — типы выводятся по значениям;
- **поддерживает явные аннотации** — можно добавить подсказки;
- позволяет очень гибко работать с данными;
- может быть проверен внешними type checker-ами (mypy, pyright).

### Нюанс: Python не совсем "сильный"

Технически Python считается сильно типизированным, но есть исключение. `bool` наследует от `int`, поэтому:

```python
True + 1         # = 2 (True приведено к 1)
False * 10       # = 0 (False приведено к 0)
```

Это редкий случай неявного преобразования, но стоит помнить.

## 📖 История: Почему type hints появились

**Давно (Python 2.x)**:
- Python использовался в основном для маленьких скриптов;
- можно было писать быстро и без документации;
- не было встроенных проверок типов.

**Проблема растёт (2010-е)**:
- начинают писать большие системы (100k+ строк кода);
- без информации о типах сложно понять, что функция ожидает;
- много ошибок обнаруживалось только в runtime;
- рефакторинг становится опасным.

**Решение (Python 3.5+)**:
- добавлены type hints (PEP 484);
- типы пишутся как подсказки, не обязательно;
- язык остаётся динамическим;
- но внешние инструменты (mypy) могут проверять статически.

```python
# Python 3.5+: можно добавлять подсказки типов
def greet(name: str) -> str:
    return f'Hello, {name}!'
```

Это компромисс: **динамичность Python + надёжность статической типизации**.

## 🔄 Динамическая типизация в статических языках

Интересный факт: даже строгие языки позволяют имитировать динамическую типизацию:

**C#** — тип `dynamic`:
```csharp
dynamic x = 5;
x = "Hello";  // Разрешено!
```

**F#** — оператор `?`:
```fsharp
let x ? = 5
x <- "Hello"  // Работает
```

**Haskell** — модуль `Data.Dynamic`:
```haskell
import Data.Dynamic
let x = toDyn (5 :: Int)
```

**Delphi** — тип `Variant`:
```delphi
var x: Variant;
x := 5;
x := 'Hello';  // Нормально
```

Это показывает, что понятия не абсолютны — любой язык может расширить свои возможности.

## 🧩 Как это выглядит в коде

### Без аннотаций

```python
def greet(name):
    return f'Hello, {name}!'
```

Работает, но непонятно, что передавать.

### С аннотациями

```python
def greet(name: str) -> str:
    return f'Hello, {name}!'
```

Здесь:
- `name: str` — параметр ожидает строку
- `-> str` — функция возвращает строку

Сам Python это не проверяет, но mypy заметит ошибку.

### Пример с ошибкой типов

```python
def multiply(x: int, y: int) -> int:
    return x * y

result = multiply(3, 5)        # ✅ Нормально
result = multiply("3", 5)      # ❌ mypy предупредит!
```

Python запустит оба варианта, но `mypy` подсветит проблему на этапе разработки.

### Аннотации переменных

```python
age: int = 25
name: str = "Alice"
numbers: list[int] = [1, 2, 3]
data: dict[str, int] = {"a": 1}
prices: tuple[str, ...] = ("apple", "banana", "orange")
```

## 🚦 Статическая проверка в Python

Python сам по себе **не запрещает** ошибки типов. Для этого используют отдельные инструменты:

- **mypy** — самый популярный, разработан автором Python Гвидо ван Россумом;
- **pyright** — от Microsoft, очень быстрый;
- **pyre** — от Facebook;
- **pytype** — от Google.

Они ловят:

- несовместимые типы аргументов;
- ошибки в возвращаемых значениях;
- проблемы с коллекциями;
- несоответствие аннотаций фактическому коду.

## 💡 Практический смысл

Типизация нужна не ради формальности, а чтобы код был **понятнее и надёжнее**:

- меньше скрытых ошибок;
- проще рефакторить большой код;
- удобнее работать в больших проектах;
- лучше автодополнение в IDE;
- проще читать и понимать чужой код.

**Вывод**: используй type hints в production коде, особенно если работаешь в команде.

## 🧠 Как запомнить

```
Когда?
├─ Динамическая  → тип проверяется во время ВЫПОЛНЕНИЯ
└─ Статическая   → тип проверяется ЗАРАНЕЕ (компиляция)

Как сильно?
├─ Сильная       → редко неявных преобразований
└─ Слабая        → часто неявных преобразований

Как видно?
├─ Явная         → типы ПИШУТСЯ в коде
└─ Неявная       → типы ВЫВОДЯТСЯ автоматически
```
