# 🧠 Виды типизации в Python

## 📌 Коротко

> Типизация описывает, как язык связывает данные с их типами и когда он проверяет, что операции с этими данными допустимы.

Python относится к языкам с динамической и сильной типизацией, а `type hints` добавляют к нему удобный слой подсказок для людей и инструментов.

## 📚 Что такое типизация

Типизация в программировании отвечает за то, как язык работает с типами данных:

- как он понимает, что значение — это число, строка, список или что-то другое;
- когда он проверяет совместимость типов;
- разрешает ли он смешивать разные типы в выражениях;
- помогает ли он программисту заранее заметить ошибку.

В разных языках это устроено по-разному. Поэтому полезно понимать не только “есть типизация или нет”, а именно какой она бывает.

## 🧭 Основные виды

### Статическая и динамическая

Это самая важная пара понятий.

**Статическая типизация**:

- типы проверяются до запуска программы;
- ошибки типов ловятся на этапе компиляции или анализа;
- код часто становится более предсказуемым.

**Динамическая типизация**:

- тип связан со значением во время выполнения;
- типы проверяются, когда программа реально доходит до операции;
- переменная может хранить значения разных типов в разные моменты времени.

Пример динамического поведения в Python:

```python
data = 17
data = "Hello"
```

Одна и та же переменная сначала хранит `int`, потом `str`. Для Python это нормально.

### Сильная и слабая

Это уже не про момент проверки, а про то, насколько язык терпимо смешивает типы.

**Сильная типизация**:

- язык не любит неявные опасные преобразования;
- если операция бессмысленна, чаще будет ошибка.

**Слабая типизация**:

- язык может сам привести типы друг к другу;
- иногда это удобно, но иногда приводит к неожиданным результатам.

Пример для Python:

```python
data = "python"
new_data = data + 2022
```

Это вызовет `TypeError`, потому что строку и число Python не складывает неявно.

### Явная и неявная

**Явная типизация**:

- типы пишутся прямо в коде;
- программист сам показывает, что ожидается.

**Неявная типизация**:

- тип можно вывести по значению;
- язык или инструмент сам догадается.

В Python часто используют явные аннотации, но сам язык остаётся динамическим.

## 🐍 Python и типизация

Python:

- динамический;
- сильный;
- поддерживает явные аннотации типов;
- допускает очень гибкую работу с данными;
- при этом может быть проверен внешними type checker-ами.

Это означает важную вещь: `type hints` не превращают Python в статически типизированный язык. Они только добавляют подсказки и помогают проверяющим инструментам.

## 🧩 Как это выглядит в коде

### Без аннотаций

```python
def greet(name):
    return f'Hello, {name}!'
```

### С аннотациями

```python
def greet(name: str) -> str:
    return f'Hello, {name}!'
```

Здесь `name: str` говорит, что ожидается строка, а `-> str` показывает, что функция возвращает строку.

### Пример с ошибкой типов

```python
def multiply(x: int, y: int) -> int:
    return x * y

result = multiply(3, 5)
result = multiply("3", 5)
```

Сам Python может запустить такой код, но `mypy` или `pyright` уже могут подсветить проблему.

## 🔧 Где здесь type hints

`type hints` — это не сама типизация языка, а способ явно описать ожидаемые типы в Python.

Они полезны для:

- параметров функций;
- возвращаемых значений;
- переменных;
- коллекций;
- сложных структур данных;
- документации кода.

Примеры:

```python
age: int = 25
name: str = "Alice"
numbers: list[int] = [1, 2, 3]
data: dict[str, int] = {"a": 1}
```

## 🚦 Статическая проверка

Python сам по себе не запрещает все ошибки типов. Для этого используют отдельные инструменты:

- `mypy`;
- `pyright`;
- `pyre`;
- `pytype`.

Они помогают ловить:

- несовместимые типы аргументов;
- ошибки в возвращаемых значениях;
- проблемы с коллекциями;
- несоответствие аннотаций фактическому коду.

## 💡 Практический смысл

Типизация нужна не ради формальности, а чтобы код был понятнее и надёжнее:

- меньше скрытых ошибок;
- проще рефакторить;
- удобнее работать в больших проектах;
- лучше автодополнение в IDE;
- проще читать чужой код.

## 🧠 Как запомнить

- **динамическая** — тип проверяется во время выполнения;
- **статическая** — тип проверяется заранее;
- **сильная** — плохие смешивания типов не прощаются;
- **слабая** — язык чаще сам приводит типы;
- **явная** — тип пишется в коде;
- **неявная** — тип выводится автоматически.
