# 📖 Основы типизации в Python

> Type hints (аннотации типов) — это синтаксис для явного указания ожидаемых типов переменных, параметров функций и возвращаемых значений. Они помогают писать понятный и надёжный код, хотя Python их не проверяет при выполнении. Статические анализаторы как mypy используют эти подсказки для поиска ошибок до запуска программы.

## Справка по основам

|Что                                  |Синтаксис              |Пример                            |
|-------------------------------------|-----------------------|----------------------------------|
|**Параметр функции**                 |`param: Type`          |`def add(a: int, b: int)`         |
|**Возвращаемое значение**            |`-> Type`              |`def add(a: int) -> int:`         |
|**Переменная**                       |`var: Type = value`    |`age: int = 25`                   |
|**Параметр с значением по умолчанию**|`param: Type = default`|`def greet(name: str = 'world')`  |
|**Функция без возврата**             |`-> None`              |`def print_msg(msg: str) -> None:`|

## 🎯 Зачем нужна типизация

Type hints нужны для нескольких целей:

**Самодокументирование кода** — сразу видно, какие типы ожидаются. Вместо того чтобы читать документацию или внутренность функции, можно посмотреть сигнатуру и всё понять.

**IDE понимает код лучше** — автодополнение, подсказки о методах и атрибутах работают качественнее, когда IDE знает типы.

**Поиск ошибок** — статические анализаторы как mypy или pyright проверяют типы и находят ошибки **до запуска программы**. Python сам типы не проверяет, но эти инструменты помогают.

**Рефакторинг безопаснее** — когда меняешь код, type checker сразу скажет, что сломалось в других местах.

**Важный момент:** типизация опциональна. Python остаётся динамически типизированным языком. Аннотации — это только подсказки для людей и инструментов, не требования для интерпретатора.

## 📝 Аннотирование функций

### Базовый синтаксис

Типы параметров указываются после имени через `:`, тип возвращаемого значения после `->`:

```python
def multiply(x: int, y: int) -> int:
    return x * y

def greet(name: str) -> str:
    return f'Hello, {name}!'

def is_even(num: int) -> bool:
    return num % 2 == 0
```

### Параметры с значениями по умолчанию

Аннотация указывается **между именем и значением по умолчанию**:

```python
def greet(name: str = 'world') -> str:
    return f'Hello, {name}!'

def power(x: int, exponent: int = 2) -> int:
    return x ** exponent

def configure(debug: bool = False, timeout: int = 30) -> None:
    print(f'Debug: {debug}, Timeout: {timeout}')
```

### Функции без возвращаемого значения

Когда функция ничего не возвращает (или возвращает None неявно), указывай `-> None`:

```python
def print_message(msg: str) -> None:
    print(msg)

def setup_database() -> None:
    # инициализация
    pass
```

## 💾 Аннотирование переменных

В Python 3.6+ можно аннотировать переменные в любом месте кода:

```python
# С присваиванием значения
name: str = 'Alice'
age: int = 25
height: float = 1.75
is_active: bool = True

# Без значения (только аннотация, обычно не используется)
username: str
password: str
```

Обычно переменные не аннотируют, если тип очевиден. Но полезно для:

- Пустых контейнеров: `users: list[str] = []` (не понятно, какой тип внутри)
- Результатов функций без типов: `data: dict = external_function()`
- Явного указания намерения

## 🔤 Встроенные базовые типы

```python
# Числа
count: int = 42
price: float = 19.99
result: complex = 1 + 2j

# Строки и байты
message: str = 'hello'
data: bytes = b'data'

# Логические значения
is_valid: bool = True

# Ничего (отсутствие значения)
nothing: None = None

def cleanup() -> None:
    print('Cleaning up')
```

## 💻 Примеры

### Пример 1: Простые функции

```python
def add(a: int, b: int) -> int:
    """Сумма двух чисел"""
    return a + b

def divide(dividend: float, divisor: float) -> float:
    """Деление с результатом типа float"""
    if divisor == 0:
        raise ValueError('Divisor cannot be zero')
    return dividend / divisor

def is_palindrome(text: str) -> bool:
    """Проверка, является ли строка палиндромом"""
    clean = text.lower().replace(' ', '')
    return clean == clean[::-1]

# Использование
result = add(10, 20)           # 30
quotient = divide(10.5, 2.5)   # 4.2
check = is_palindrome('racecar')  # True
```

### Пример 2: Функции с опциональными параметрами

```python
def greet(name: str = 'stranger', formal: bool = False) -> str:
    """Приветствие"""
    greeting = "Good day" if formal else "Hi"
    return f"{greeting}, {name}!"

def create_list(size: int = 0, fill_value: str = '') -> list[str]:
    """Создать список со значениями"""
    return [fill_value] * size

# Использование
print(greet())                          # Hi, stranger!
print(greet('Alice', formal=True))      # Good day, Alice!
print(create_list(3, 'hello'))          # ['hello', 'hello', 'hello']
```

### Пример 3: Функции с None

```python
def find_user(user_id: int) -> str | None:
    """Может вернуть строку или None"""
    users = {1: "Alice", 2: "Bob"}
    return users.get(user_id)

def get_first_element(items: list) -> int | None:
    """Получить первый элемент или None"""
    return items[0] if items else None

# Использование
user = find_user(1)
if user is not None:
    print(f"Found: {user}")
else:
    print("Not found")
```

### Пример 4: Переменные в коде

```python
# Пустые контейнеры — аннотируем для ясности
numbers: list[int] = []
user_ages: dict[str, int] = {}
active_tags: set[str] = set()

# Результаты от функций без типов (старый код)
config: dict = load_config()
data: list = external_api.fetch_data()

# Явное указание значений
MAX_USERS: int = 100
API_KEY: str = 'secret123'
DEBUG_MODE: bool = False
```

## ⚠️ Частые ошибки

```python
# ❌ Забыли стрелку для возвращаемого значения
def add(a: int, b: int) int:  # SyntaxError!
    return a + b

# ✅ Правильно
def add(a: int, b: int) -> int:
    return a + b

# ❌ Неправильный порядок (аннотация после значения)
def greet(name = 'world': str) -> str:  # SyntaxError!
    return f'Hello, {name}!'

# ✅ Правильно
def greet(name: str = 'world') -> str:
    return f'Hello, {name}!'

# ❌ Забыли None для функции без возврата
def print_msg(msg: str):  # Неявный возврат
    print(msg)

# ✅ Правильно
def print_msg(msg: str) -> None:
    print(msg)

# ❌ Думать, что аннотация проверяется при выполнении
def add(a: int, b: int) -> int:
    return a + b

add('hello', 'world')  # ❌ Type checker скажет "ошибка"
                       # ✅ Но Python выполнит: 'helloworld'
```

## ✅ Главные правила

✅ **Начните с функций** — аннотируйте сигнатуры в первую очередь  
✅ **Используйте type checker** — mypy или pyright найдут ошибки до запуска  
✅ **Аннотации не влияют на выполнение** — это только для анализа  
✅ **Добавляйте типы постепенно** — не переусложняйте сразу  
✅ **None для функций без возврата** — это важно  
✅ **Комментарии + типы** — вместе документируют код лучше  
✅ **IDE благодарна** — автодополнение работает намного лучше
