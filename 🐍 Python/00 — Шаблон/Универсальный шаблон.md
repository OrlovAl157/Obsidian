## 📝 Инструкция по использованию шаблона

### Структура файла (что обязательно):

```
✅ ОБЯЗАТЕЛЬНО:
├── frontmatter (теги)
├── # Заголовок с эмодзи
├── 📌 Коротко (1 предложение)
├── 📖 Теория
│   ├── Основы
│   └── Синтаксис/Как работает (если нужно)
└── 🔗 Связанные темы

❓ ОПЦИОНАЛЬНО (добавляйте по необходимости):
├── 💡 Полезные советы (только если есть что сказать)
├── 🚀 Быстрая справка (только для тем с методами/операциями)
├── 💻 Примеры кода (только если тема практическая)
└── ❓ Вопросы (для личных заметок)
```

### Когда использовать опциональные секции:

**💡 Полезные советы** — включайте если:

- Есть важные моменты, которые нужно знать заранее
- Есть частые ошибки новичков
- Есть best practices
- Есть неочевидные особенности

**НЕ включайте** если нечего добавить сверх того, что в теории!

**🚀 Быстрая справка** — включайте если:

- Тема содержит методы (типы данных, модули)
- Есть операции с разной сложностью
- Нужна таблица для быстрого поиска

**НЕ включайте** для концептуальных тем (ООП, паттерны, теория)!

**💻 Примеры кода** — включайте если:

- Тема практическая (синтаксис, методы, функции)
- Нужны примеры использования
- Есть практические кейсы

**НЕ включайте** для чисто теоретических тем!

### Примеры применения шаблона:

#### Для ТИПОВ ДАННЫХ (списки, словари):

```
✅ Используем: Полезные советы + Быстрая справка + Примеры кода
📖 Теория — концептуально (устройство)
💻 Примеры — всё остальное
```

#### Для КОНЦЕПТУАЛЬНЫХ ТЕМ (ООП, принципы):

```
✅ Используем: только Теорию
❌ Убираем: Быстрая справка, Полезные советы (если нечего сказать)
💻 Примеры — только если нужно показать синтаксис
```

#### Для МОДУЛЕЙ (collections, itertools):

```
✅ Используем: Полезные советы + Быстрая справка + Примеры кода
📖 Теория — что это, зачем
💻 Примеры — функции/классы с примерами
```

#### Для СИНТАКСИСА (циклы, условия):

```
✅ Используем: Полезные советы + Примеры кода
❌ Убираем: Быстрая справка (нет методов)
💻 Примеры — разные варианты синтаксиса
```

### Общие принципы:

**Эмодзи:**

- Используйте для визуального разделения
- Одинаковые эмодзи для одинаковых секций во всех файлах

**Таблицы:**

- Для справочной информации
- Для сравнения вариантов
- Для списка методов/операций

**Примеры кода:**

- Должны быть рабочими
- С комментариями к неочевидным вещам
- От простого к сложному

**Связанные темы:**

- Обязательно ссылки на связанные файлы
- Используйте [[WikiLinks]]

**Статусы в тегах:**

- статус/не_начат
- статус/в_процессе
- статус/завершён
