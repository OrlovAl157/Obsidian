# 🌿 Git — Ветки и слияние

## 📌 Коротко

> Ветка — это отдельная линия разработки. Как черновик: делаешь изменения не трогая основной код, а потом сливаешь.

---

## 🌿 Работа с ветками

| Команда | Что делает |
|---|---|
| `git branch` | Показать все локальные ветки |
| `git branch <имя>` | Создать новую ветку |
| `git checkout <ветка>` | Переключиться на ветку |
| `git checkout -b <имя>` | Создать ветку и сразу переключиться |
| `git switch <ветка>` | Современный способ переключения (Git 2.23+) |
| `git switch -c <имя>` | Создать и переключиться (современный способ) |
| `git branch -d <имя>` | Удалить ветку (безопасно — только если слита) |
| `git branch -D <имя>` | Удалить ветку принудительно |

---

## 🔀 Слияние

| Команда | Что делает |
|---|---|
| `git merge <ветка>` | Слить ветку в текущую |
| `git rebase <ветка>` | Перенести коммиты на другую ветку (линейная история) |

### Разница merge vs rebase

```
merge:   A---B---C---M    (M = merge commit, история разветвлялась)
rebase:  A---B---C---D    (D = твои изменения, история линейная)
```

> 💡 `merge` — безопасно, сохраняет историю. `rebase` — чище, но переписывает историю. На shared-ветках rebase лучше не использовать.
