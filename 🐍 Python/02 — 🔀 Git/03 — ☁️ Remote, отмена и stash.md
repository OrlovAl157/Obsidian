# ☁️ Git — Remote, отмена и stash

## 📌 Коротко

> Remote — удалённый репозиторий (GitHub, GitLab). Stash — временное хранилище незакоммиченных изменений.

---

## ☁️ Remote (GitHub / GitLab)

| Команда | Что делает |
|---|---|
| `git remote -v` | Показать удалённые репозитории |
| `git remote add origin <url>` | Привязать удалённый репозиторий |
| `git push origin <ветка>` | Отправить ветку на GitHub |
| `git push -u origin main` | Push + запомнить ветку (потом просто `git push`) |
| `git pull` | Скачать изменения и слить с текущей веткой |
| `git fetch` | Скачать изменения БЕЗ слияния |

> 💡 `pull` = `fetch` + `merge`

---

## ↩️ Отмена изменений

| Команда | Что делает |
|---|---|
| `git restore <файл>` | Отменить изменения в файле (до коммита) |
| `git restore --staged <файл>` | Убрать файл из staging (изменения сохранятся) |
| `git revert <hash>` | Отменить коммит, создав новый (безопасно) |
| `git reset --soft HEAD~1` | Отменить последний коммит, сохранив изменения |
| `git reset --hard HEAD~1` | ⚠️ Удалить последний коммит И изменения |

> ⚠️ `reset --hard` — опасная команда. Данные удаляются без возможности восстановления.

> 💡 **hash** — первые 7 символов из `git log --oneline`

---

## 📦 Stash (временное хранилище)

> Аналогия: как поставить вещи на полку — убрал с рабочего стола, но не выбросил

| Команда | Что делает |
|---|---|
| `git stash` | Спрятать незакоммиченные изменения |
| `git stash pop` | Вернуть спрятанные изменения |
| `git stash list` | Показать список всех stash |
| `git stash drop` | Удалить последний stash |

---

## 🏷 Теги

| Команда | Что делает |
|---|---|
| `git tag v1.0` | Поставить тег на текущий коммит |
| `git push origin --tags` | Отправить все теги на GitHub |
