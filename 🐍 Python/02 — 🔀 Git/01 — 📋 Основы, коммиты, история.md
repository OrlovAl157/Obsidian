# 📋 Git — Основы: настройка, коммиты, история

## 📌 Коротко

> Git — система контроля версий. Сохраняет историю изменений кода, позволяет откатиться назад и работать в команде.

---

## 🔧 Настройка (один раз при установке)

```bash
git config --global user.name "Твоё Имя"
git config --global user.email "email@example.com"
git config --list   # проверить все настройки
```

---

## 🆕 Создание репозитория

| Команда | Что делает |
|---|---|
| `git init` | Создать новый репозиторий в текущей папке |
| `git clone <url>` | Скопировать репозиторий с GitHub/GitLab |

---

## 🔍 Состояние

| Команда | Что делает |
|---|---|
| `git status` | Показать что изменено, что готово к коммиту |
| `git diff` | Изменения в файлах (ещё не добавленные в staging) |
| `git diff --staged` | Изменения уже добавленные в staging |

---

## ➕ Add и Commit

> Аналогия: `add` — сложить вещи в коробку, `commit` — запечатать и подписать коробку

| Команда | Что делает |
|---|---|
| `git add файл` | Добавить файл в зону подготовки (staging) |
| `git add .` | Добавить ВСЕ изменённые файлы |
| `git commit -m "текст"` | Сохранить снимок с сообщением |
| `git commit -am "текст"` | Add + commit одной командой (только tracked файлы) |

---

## 📜 История

| Команда | Что делает |
|---|---|
| `git log` | Полная история коммитов |
| `git log --oneline` | Краткая история — одна строка на коммит |
| `git log --oneline --graph` | История в виде дерева веток |

> 💡 **HEAD** — указатель на последний коммит текущей ветки
