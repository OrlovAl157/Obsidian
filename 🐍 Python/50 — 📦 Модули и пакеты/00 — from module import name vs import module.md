# 📦 `from module import name` vs `import module`

## 📌 Коротко

> `from` — копирует ссылку в локальную область. `import` — даёт доступ к самому модулю и его атрибутам.

---

## 📖 Теория

### `from sys import stdout`

```python
from sys import stdout   # stdout — это локальная копия ссылки

stdout = что_угодно      # меняем только локальную переменную
                         # sys.stdout НЕ изменится!
```

### `import sys`

```python
import sys               # sys — это сам модуль

sys.stdout = что_угодно  # меняем атрибут модуля
                         # изменение видно везде!
```

---

### Аналогия через списки

```python
a = [1, 2, 3]
b = a             # b — копия ссылки на тот же список

b = [4, 5, 6]    # переназначаем b — a не изменился!
print(a)          # [1, 2, 3]

b = a
b[:] = [4, 5, 6]  # меняем содержимое через срез — a изменился!
print(a)           # [4, 5, 6]
```

---

### Практический пример — подмена `sys.stdout`

```python
# ❌ Не работает — меняем локальную копию
from sys import stdout
stdout = my_object        # sys.stdout остался прежним!

# ✅ Работает — меняем атрибут модуля
import sys
sys.stdout = my_object    # изменение видно везде
```

---

## 🚀 Быстрая справка

||`from sys import stdout`|`import sys`|
|---|---|---|
|Что получаем|локальная копия ссылки|сам модуль|
|Изменение через `=`|не влияет на оригинал|влияет на оригинал|
|Когда использовать|просто читать значение|нужно менять атрибут модуля|
