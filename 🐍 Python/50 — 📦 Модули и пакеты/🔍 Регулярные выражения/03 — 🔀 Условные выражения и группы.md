## 🔀 Условные выражения `(?(n)yes|no)`

Если группа n нашла совпадение - применяется паттерн `yes`, иначе - `no`.

### 📋 Синтаксис

```
(?(n)yes-pattern|no-pattern)
(?(name)yes-pattern|no-pattern)  # для именованных групп
```

`no-pattern` можно опустить: `(?(n)yes-pattern|)`

-----

### 💻 Примеры

**Пример 1: Простое условие**

```python
import re

regex = r"(a)?(?(1)b|c)"

# Логика:
# - Группа (a)? ищет 'a' (может не быть)
# - Если группа 1 нашла 'a' → ищет 'b'
# - Если группа 1 пустая → ищет 'c'

text = "ab ac c cb"
re.findall(regex, text)  # ['a', '', '']  (найдёт: ab, c)
```

**Пример 2: Скобки**

```python
import re

# Цифра в скобках или без
regex = r"^(\[)?\d(?(1)\]|)$"

# Логика:
# - (\[)? - может быть '['
# - \d - обязательная цифра
# - (?(1)\]|) - если была '[', то ищем ']', иначе ничего

re.match(regex, "1")     # ✅ Match
re.match(regex, "[2]")   # ✅ Match
re.match(regex, "3]")    # ❌ None (нет '[')
re.match(regex, "[4")    # ❌ None (нет ']')
```

### Парные скобки

```python
import re

# Цифра в скобках или без
regex = r"^(\[)?\d(?(1)\]|)$"

# Разбор:
# (\[)?      - Группа 1: может быть '['
# \d         - цифра
# (?(1)\]|)  - если была '[' → ищем ']', иначе ничего
# ^ и $      - обязательны для корректной работы!

re.match(regex, "1")      # ✅ Match
re.match(regex, "[2]")    # ✅ Match
re.match(regex, "3]")     # ❌ None (нет '[')
re.match(regex, "[4")     # ❌ None (нет ']')
```

### Именованные группы

```python
import re

text = "2024-01-15"
pattern = r'^(?P<year>\d{4})-\d{2}-(?(year)\d{2}|XX)$'

# Если group 'year' нашла цифры → ищем \d{2}, иначе → 'XX'

re.match(pattern, text)  # ✅ Match
```

### Множественные условия

```python
import re

# Условия можно использовать последовательно
regex = r"^(a)?(b)?(?(1)A|)(?(2)B|)$"

# (a)?       - Группа 1
# (b)?       - Группа 2
# (?(1)A|)   - если есть 'a' → нужна 'A'
# (?(2)B|)   - если есть 'b' → нужна 'B'

re.match(regex, "")       # ✅ Match
re.match(regex, "aA")     # ✅ Match
re.match(regex, "bB")     # ✅ Match
re.match(regex, "abAB")   # ✅ Match
re.match(regex, "a")      # ❌ None (нет 'A')
```

## ⚠️ Важно

**1. Всегда используйте якоря/границы:**

```python
# ❌ БЕЗ якорей - некорректно
r"(\[)?\d(?(1)\]|)"  
re.search(r"(\[)?\d(?(1)\]|)", "[5 text")  # Match (ошибка!)

# ✅ С якорями - правильно
r"^(\[)?\d(?(1)\]|)$"
```

**2. Группа должна существовать:**

```python
r"(a)?(?(2)b|c)"  # ❌ Группы 2 нет!
r"(a)?(?(1)b|c)"  # ✅ Группа 1 есть
```

**3. Условие НЕ создаёт группу:**

```python
match = re.search(r"(a)?(?(1)b|c)", "ab")
match.groups()  # ('a',) - только одна группа
```

## 🎯 Практический пример

```python
import re

# Телефон с опциональным кодом
regex = r"^(\+7\s?)?(\()?\d{3}(?(2)\)\s?|[\s-]?)\d{3}[\s-]?\d{2}[\s-]?\d{2}$"

phones = [
    "+7 (123) 456-78-90",  # ✅
    "(123) 456-78-90",     # ✅
    "+7 123-456-78-90",    # ✅
    "(123 456-78-90",      # ❌ скобка не закрыта
]

for phone in phones:
    result = '✅' if re.match(regex, phone) else '❌'
    print(f"{phone:25} → {result}")
```

-----

## 📌 Когда использовать

✅ **Используйте:**

- Парные элементы (скобки, кавычки, теги)
- Зависимость между частями паттерна
- Опциональные элементы влияют на формат

❌ **Не используйте:**

- Когда можно обойтись простыми группами
- Для сложной логики (лучше несколько regex)
