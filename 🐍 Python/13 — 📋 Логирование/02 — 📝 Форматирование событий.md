## Справка по форматированию

| Заполнитель | Описание | Пример |
|------------|---------|--------|
| `%(name)s` | Имя логгера | `__main__`, `utils` |
| `%(levelname)s` | Имя уровня | `DEBUG`, `INFO`, `WARNING` |
| `%(levelno)d` | Номер уровня | `10`, `20`, `30` |
| `%(message)s` | Сообщение | `Ошибка при загрузке` |
| `%(asctime)s` | Дата и время | `2025-04-25 14:39:03,568` |
| `%(filename)s` | Имя файла | `main.py` |
| `%(funcName)s` | Имя функции | `process_data` |
| `%(lineno)d` | Номер строки | `42` |
| `%(module)s` | Имя модуля | `utils` |
| `%(pathname)s` | Полный путь файла | `/home/user/project/main.py` |

## 🎯 Что такое форматирование

**Форматирование** — это определение того, как будет выглядеть сообщение в логах.

### Без форматирования

```python
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

logger.info('Приложение запущено')
logger.error('Ошибка подключения')
```

Вывод:
```
Приложение запущено
Ошибка подключения
```

**Проблема:** непонятно когда произошло, какой уровень, откуда событие.

### С форматированием

```python
import logging

fmt = '%(asctime)s | %(levelname)s | %(name)s | %(message)s'
logging.basicConfig(level=logging.INFO, format=fmt)
logger = logging.getLogger(__name__)

logger.info('Приложение запущено')
logger.error('Ошибка подключения')
```

Вывод:
```
2025-04-25 14:39:03,568 | INFO | __main__ | Приложение запущено
2025-04-25 14:39:03,569 | ERROR | __main__ | Ошибка подключения
```

**Намного лучше!** Видно дату, время, уровень, модуль, сообщение.

## 🔤 Стандартный формат

### По умолчанию

Если вызвать `basicConfig()` без параметра `format`, используется стандартный:

```python
%(levelname)s:%(name)s:%(message)s
```

Пример вывода:
```
INFO:__main__:Приложение запущено
ERROR:__main__:Ошибка подключения
```

### Как использовать свой формат

```python
import logging

# Способ 1: простой формат
logging.basicConfig(
    level=logging.INFO,
    format='[%(levelname)s] %(message)s'
)
logger = logging.getLogger(__name__)
logger.info('Привет')

# Вывод: [INFO] Привет
```

### Многострочный формат

```python
import logging

fmt = '''
Уровень: %(levelname)s
Дата: %(asctime)s
Модуль: %(name)s
Сообщение: %(message)s
'''

logging.basicConfig(level=logging.INFO, format=fmt)
logger = logging.getLogger(__name__)
logger.info('Событие')
```

Вывод:
```
Уровень: INFO
Дата: 2025-04-25 14:39:03,568
Модуль: __main__
Сообщение: Событие
```

## 📋 Встроенные заполнители

### name — имя логгера

```python
import logging

logging.basicConfig(format='%(name)s: %(message)s')

logger1 = logging.getLogger('database')
logger2 = logging.getLogger('network')
logger3 = logging.getLogger('__main__')

logger1.warning('Warning 1')
logger2.warning('Warning 2')
logger3.warning('Warning 3')
```

Вывод:
```
database: Warning 1
network: Warning 2
__main__: Warning 3
```

### levelname — имя уровня

```python
import logging

logging.basicConfig(format='[%(levelname)s] %(message)s', level=logging.DEBUG)
logger = logging.getLogger(__name__)

logger.debug('Отладка')
logger.info('Информация')
logger.warning('Предупреждение')
logger.error('Ошибка')
logger.critical('Критическая')
```

Вывод:
```
[DEBUG] Отладка
[INFO] Информация
[WARNING] Предупреждение
[ERROR] Ошибка
[CRITICAL] Критическая
```

### levelno — номер уровня

```python
import logging

logging.basicConfig(format='Level %(levelno)d: %(message)s', level=logging.DEBUG)
logger = logging.getLogger(__name__)

logger.debug('Отладка')     # 10
logger.info('Информация')    # 20
logger.warning('Предупреждение')  # 30
```

Вывод:
```
Level 10: Отладка
Level 20: Информация
Level 30: Предупреждение
```

### message — само сообщение

```python
import logging

logging.basicConfig(format='MESSAGE: %(message)s')
logger = logging.getLogger(__name__)

logger.info('Привет мир')
logger.error('Что-то пошло не так')
```

Вывод:
```
MESSAGE: Привет мир
MESSAGE: Что-то пошло не так
```

### filename — имя файла

```python
import logging

logging.basicConfig(format='%(filename)s: %(message)s')
logger = logging.getLogger(__name__)

logger.info('Событие')
```

Вывод:
```
main.py: Событие
```

### funcName — имя функции

```python
import logging

logging.basicConfig(format='%(funcName)s(): %(message)s', level=logging.INFO)
logger = logging.getLogger(__name__)

def process_data():
    logger.info('Обработка данных')

process_data()
```

Вывод:
```
process_data(): Обработка данных
```

### lineno — номер строки

```python
import logging

logging.basicConfig(format='Line %(lineno)d: %(message)s')
logger = logging.getLogger(__name__)

logger.info('Событие на строке 6')  # линия 6
```

Вывод:
```
Line 6: Событие на строке 6
```

### pathname — полный путь

```python
import logging

logging.basicConfig(format='%(pathname)s: %(message)s')
logger = logging.getLogger(__name__)

logger.info('Событие')
```

Вывод:
```
C:\Projects\myapp\main.py: Событие
```

### module — имя модуля

```python
import logging

logging.basicConfig(format='%(module)s - %(message)s')
logger = logging.getLogger(__name__)

logger.info('Событие')
```

Вывод (если файл `utils.py`):
```
utils - Событие
```

## ⏰ Форматирование даты и времени

### Параметр asctime

По умолчанию дата/время в формате: `2025-04-25 14:39:03,568`

```python
import logging

fmt = '%(asctime)s - %(message)s'
logging.basicConfig(level=logging.INFO, format=fmt)
logger = logging.getLogger(__name__)

logger.info('Событие')
```

Вывод:
```
2025-04-25 14:39:03,568 - Событие
```

### Параметр datefmt — свой формат даты

```python
import logging

fmt = '%(asctime)s - %(message)s'
datefmt = '%d.%m.%Y %H:%M:%S'  # день.месяц.год час:минута:секунда

logging.basicConfig(level=logging.INFO, format=fmt, datefmt=datefmt)
logger = logging.getLogger(__name__)

logger.info('Событие')
```

Вывод:
```
25.04.2025 14:39:03 - Событие
```

### Основные спецификаторы даты/времени

```python
%Y  # год (4 цифры): 2025
%y  # год (2 цифры): 25
%m  # месяц (01-12): 04
%d  # день (01-31): 25
%H  # час (00-23): 14
%M  # минута (00-59): 39
%S  # секунда (00-59): 03
%A  # полный день недели: Friday
%a  # сокращённо: Fri
%B  # полный месяц: April
%b  # сокращённо: Apr
```

### Примеры datefmt

```python
# Вариант 1: европейский
datefmt = '%d.%m.%Y %H:%M:%S'  # 25.04.2025 14:39:03

# Вариант 2: американский
datefmt = '%m/%d/%Y %I:%M:%S %p'  # 04/25/2025 02:39:03 PM

# Вариант 3: компактный
datefmt = '%Y-%m-%d %H:%M'  # 2025-04-25 14:39

# Вариант 4: с днём недели
datefmt = '%A, %d %B %Y %H:%M:%S'  # Friday, 25 April 2025 14:39:03
```

## 🎨 Стили шаблонных строк

### Стиль % (по умолчанию)

```python
import logging

fmt = '%(asctime)s - %(levelname)s - %(message)s'
logging.basicConfig(format=fmt)
logger = logging.getLogger(__name__)

logger.info('Событие')
```

### Стиль { } (как f-строки)

```python
import logging

fmt = '{asctime} - {levelname} - {message}'
logging.basicConfig(format=fmt, style='{')  # style='{'
logger = logging.getLogger(__name__)

logger.info('Событие')
```

Вывод (одинаковый):
```
2025-04-25 14:39:03,568 - INFO - Событие
```

### Когда какой стиль использовать?

```python
# ✅ Стиль % — классический и надёжный
fmt = '%(levelname)s: %(message)s'

# ✅ Стиль {} — более красивый и понятный
fmt = '{levelname}: {message}'

# ❌ Не используй f-строки (без префикса f)
# ✅ Но можешь использовать форматирование внутри {}
fmt = '{levelname:10}: {message}'  # выравнивание
```

### Форматирование в стиле {}

```python
import logging

# Выравнивание текста
fmt = '{levelname:10} | {name:^15} | {message}'
logging.basicConfig(format=fmt, style='{', level=logging.INFO)
logger = logging.getLogger(__main__)

logger.info('Событие')
logger.warning('Предупреждение')
```

Вывод (с выравниванием):
```
INFO       |       __main__       | Событие
WARNING    |       __main__       | Предупреждение
```

## 👤 Пользовательские заполнители

### Что это?

Иногда нужно добавить в логи информацию, которой нет в стандартных заполнителях (например, ID пользователя, номер заказа и т.д.).

### Как использовать?

Используй параметр `extra` при логировании:

```python
import logging

fmt = 'user_id={user_id}, action={action}: {message}'
logging.basicConfig(format=fmt, style='{')
logger = logging.getLogger(__name__)

extra_data = {'user_id': 123, 'action': 'login'}
logger.info('Пользователь вошёл', extra=extra_data)
```

Вывод:
```
user_id=123, action=login: Пользователь вошёл
```

### Пример: логирование с данными о пользователе

```python
import logging

logging.basicConfig(
    format='[{asctime}] {levelname:8} | user={user_id} | {message}',
    style='{',
    datefmt='%H:%M:%S',
    level=logging.INFO
)
logger = logging.getLogger(__name__)

def login_user(user_id, username):
    extra = {'user_id': user_id}
    logger.info(f'Пользователь {username} вошёл', extra=extra)

def logout_user(user_id, username):
    extra = {'user_id': user_id}
    logger.info(f'Пользователь {username} вышел', extra=extra)

login_user(123, 'alice')
logout_user(123, 'alice')
```

Вывод:
```
[14:39:03] INFO     | user=123 | Пользователь alice вошёл
[14:39:04] INFO     | user=123 | Пользователь alice вышел
```

### Несколько полей в extra

```python
import logging

logging.basicConfig(
    format='[{levelname}] user={user_id} order={order_id} status={status}: {message}',
    style='{',
    level=logging.INFO
)
logger = logging.getLogger(__name__)

def process_order(order_id, user_id, status):
    extra = {
        'user_id': user_id,
        'order_id': order_id,
        'status': status
    }
    logger.info('Обработка заказа', extra=extra)

process_order(order_id=5001, user_id=123, status='pending')
process_order(order_id=5002, user_id=456, status='completed')
```

Вывод:
```
[INFO] user=123 order=5001 status=pending: Обработка заказа
[INFO] user=456 order=5002 status=completed: Обработка заказа
```

### Опциональные поля (defaults в Formatter)

Если некоторые поля могут отсутствовать, используй `defaults` (об этом позже с Formatter).

```python
import logging

logger = logging.getLogger(__name__)
extra = {'user_id': 123}  # нет order_id!

# Это вызовет ошибку
logger.info('Событие', extra=extra)
# ValueError: Formatting field not found in record: 'order_id'
```

**Решение:** использовать Formatter с `defaults` (см. тему "Обработчики и форматеры").

## 📝 Примечания

### Примечание 1: НЕЛЬЗЯ переписывать встроенные заполнители

```python
import logging

logging.basicConfig(format='{levelname}: {message}', style='{')
logger = logging.getLogger(__name__)

# ❌ ОШИБКА! Нельзя переписать levelname в extra
extra = {'levelname': 'FAKE'}
logger.info('Событие', extra=extra)

# KeyError: "Attempt to overwrite 'levelname' in LogRecord"
```

**Встроенные заполнители нельзя переписывать!**

### Примечание 2: лишние ключи в extra игнорируются

```python
import logging

logging.basicConfig(format='user={user_id}: {message}', style='{')
logger = logging.getLogger(__name__)

# age не используется в формате, просто игнорируется
extra = {'user_id': 123, 'age': 30, 'city': 'Moscow'}
logger.info('Событие', extra=extra)

# Вывод: user=123: Событие
```

### Примечание 3: пустая строка как сообщение

Если нужны только заполнители extra без сообщения, можно передать пустую строку:

```python
import logging

logging.basicConfig(
    format='user={user_id} action={action} | {message}',
    style='{',
    level=logging.INFO
)
logger = logging.getLogger(__name__)

extra = {'user_id': 123, 'action': 'click_button'}
logger.info('', extra=extra)  # пустая строка!

# Вывод: user=123 action=click_button |
```

## ✅ Главные правила

✅ Используй параметр `format` для красивого вывода  
✅ Используй `datefmt` для своего формата даты  
✅ Стиль `%` — классический и всегда работает  
✅ Стиль `{}` — красивый, как f-строки  
✅ Встроенные заполнители всегда доступны  
✅ Пользовательские поля передавай через `extra={'key': value}`  
✅ НЕ переписывай встроенные заполнители в extra  
✅ Лишние ключи в extra игнорируются  
✅ Сообщение (msg) — всегда обязательно  
✅ Для опциональных полей используй `defaults` в Formatter (далее)
