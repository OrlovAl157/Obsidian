## 🎯 Что такое фильтр

**Фильтр** — это дополнительное условие для логирования. Если фильтр вернул `False`, событие не будет записано.

### Иерархия: логгер → фильтры → обработчики

```
Logger
  ↓
Фильтры логгера (если есть)
  ↓
Handler 1 → Фильтры обработчика 1
Handler 2 → Фильтры обработчика 2
Handler 3 → Фильтры обработчика 3
```

### Зачем нужны?

Иногда уровня логирования (DEBUG, INFO, WARNING и т.д.) недостаточно.

**Примеры:**

```python
# ❌ Уровень логирования не помогает:
# Нужно логировать все INFO, но ТОЛЬКО от модуля 'auth'
# Нужно логировать все события, но ТОЛЬКО в рабочее время (09:00-18:00)
# Нужно логировать ERROR, но НЕ от системы авторизации

# ✅ Для этого используют фильтры!
```

### Простой пример

```python
import logging

class InfoOnlyFilter(logging.Filter):
    def filter(self, record):
        return record.levelno == logging.INFO  # только INFO!

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()
filt = InfoOnlyFilter()
handler.addFilter(filt)
logger.addHandler(handler)

logger.debug('DEBUG')      # ❌ отфильтровано
logger.info('INFO')        # ✅ пропущено фильтром
logger.warning('WARNING')  # ❌ отфильтровано
logger.error('ERROR')      # ❌ отфильтровано
```

Вывод:
```
INFO
```

**Только INFO событие выведено!**

## 📋 Объект LogRecord

### Что это?

`LogRecord` — это объект, который содержит всю информацию о событии логирования. Создаётся когда вызываешь `logger.info()`, `logger.error()` и т.д.

### Как получить доступ?

Доступ к LogRecord только в методе `filter()`:

```python
import logging

class MyFilter(logging.Filter):
    def filter(self, record):
        print(type(record))  # <class 'logging.LogRecord'>
        return True  # пропустить событие

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()
handler.addFilter(MyFilter())
logger.addHandler(handler)

logger.warning('Событие')
```

Вывод:
```
<class 'logging.LogRecord'>
```

### Основные атрибуты LogRecord

```python
import logging

class MyFilter(logging.Filter):
    def filter(self, record):
        print(f"name: {record.name}")           # имя логгера
        print(f"levelname: {record.levelname}") # имя уровня
        print(f"levelno: {record.levelno}")     # номер уровня
        print(f"message: {record.getMessage()}")# сообщение
        print(f"filename: {record.filename}")   # имя файла
        print(f"funcName: {record.funcName}")   # имя функции
        print(f"lineno: {record.lineno}")       # номер строки
        print(f"created: {record.created}")     # время (timestamp)
        return True

logger = logging.getLogger('myapp')
handler = logging.StreamHandler()
handler.addFilter(MyFilter())
logger.addHandler(handler)

logger.warning('Событие')
```

Вывод (примерно):
```
name: myapp
levelname: WARNING
levelno: 30
message: Событие
filename: main.py
funcName: <module>
lineno: 19
created: 1747047588.0334811
```

## 🏗️ Создание фильтра

### Способ 1: класс наследованный от Filter

```python
import logging

class MyFilter(logging.Filter):
    def filter(self, record):
        # Возвращаем True если событие нужно логировать
        # Возвращаем False если событие нужно пропустить
        return True

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()

filt = MyFilter()
handler.addFilter(filt)
logger.addHandler(handler)
```

### Способ 2: класс БЕЗ наследования (Python 3.2+)

```python
import logging

class MyFilter:  # без наследования!
    def filter(self, record):
        return True

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()

filt = MyFilter()
handler.addFilter(filt)
logger.addHandler(handler)
```

### Способ 3: обычная функция

```python
import logging

def my_filter(record):
    return True

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()

handler.addFilter(my_filter)
logger.addHandler(handler)
```

## 📂 Практические примеры фильтров

### Пример 1: фильтр по уровню

```python
import logging

class LevelFilter(logging.Filter):
    def __init__(self, level):
        super().__init__()
        self.level = level

    def filter(self, record):
        return record.levelno >= self.level

logger = logging.getLogger(__name__)
logger.setLevel(logging.DEBUG)

handler = logging.StreamHandler()
handler.addFilter(LevelFilter(logging.WARNING))
logger.addHandler(handler)

logger.debug('DEBUG')       # ❌ не выводится (уровень < WARNING)
logger.info('INFO')         # ❌ не выводится
logger.warning('WARNING')   # ✅ выводится
logger.error('ERROR')       # ✅ выводится
```

### Пример 2: фильтр по имени модуля

```python
import logging

class ModuleFilter(logging.Filter):
    def __init__(self, module_name):
        super().__init__()
        self.module_name = module_name

    def filter(self, record):
        return record.name.startswith(self.module_name)

logger1 = logging.getLogger('auth.login')
logger2 = logging.getLogger('auth.logout')
logger3 = logging.getLogger('database.query')

handler = logging.StreamHandler()
handler.addFilter(ModuleFilter('auth'))
logger1.addHandler(handler)
logger2.addHandler(handler)
logger3.addHandler(handler)

logger1.warning('Попытка входа')      # ✅ (auth.login)
logger2.warning('Выход')              # ✅ (auth.logout)
logger3.warning('Запрос к БД')        # ❌ (database.query)
```

Вывод:
```
Попытка входа
Выход
```

### Пример 3: фильтр по времени (рабочие часы)

```python
import logging
from datetime import datetime, time

class WorkingHoursFilter(logging.Filter):
    def filter(self, record):
        current_time = datetime.fromtimestamp(record.created).time()
        start_time = time(9, 0, 0)   # 09:00
        end_time = time(18, 0, 0)    # 18:00
        return start_time <= current_time <= end_time

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()
handler.addFilter(WorkingHoursFilter())
logger.addHandler(handler)

logger.warning('Событие (если в рабочее время)')  # может выведиться или нет
```

### Пример 4: фильтр по содержимому сообщения

```python
import logging

class KeywordFilter(logging.Filter):
    def __init__(self, keyword):
        super().__init__()
        self.keyword = keyword

    def filter(self, record):
        return self.keyword in record.getMessage()

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()
handler.addFilter(KeywordFilter('ERROR'))  # только сообщения с ERROR
logger.addHandler(handler)

logger.warning('Это просто предупреждение')
logger.warning('Это ERROR в сообщении')
logger.warning('ERROR найден!')
```

Вывод:
```
Это ERROR в сообщении
ERROR найден!
```

## ➕ Несколько фильтров

### Один обработчик → несколько фильтров

**Все фильтры должны вернуть True!**

```python
import logging
from datetime import datetime, time

class LevelFilter(logging.Filter):
    def __init__(self, min_level):
        super().__init__()
        self.min_level = min_level

    def filter(self, record):
        return record.levelno >= self.min_level

class WorkingHoursFilter(logging.Filter):
    def filter(self, record):
        current_time = datetime.fromtimestamp(record.created).time()
        return time(9, 0) <= current_time <= time(18, 0)

logger = logging.getLogger(__name__)
logger.setLevel(logging.DEBUG)

handler = logging.StreamHandler()

# Добавляем два фильтра
level_filter = LevelFilter(logging.INFO)
time_filter = WorkingHoursFilter()

handler.addFilter(level_filter)
handler.addFilter(time_filter)

logger.addHandler(handler)

logger.debug('DEBUG')       # ❌ уровень < INFO
logger.info('INFO')         # ✅ если рабочее время
logger.warning('WARNING')   # ✅ если рабочее время
```

**Логика:** Event проходит ТОЛЬКО если ВСЕ фильтры вернули True.

## 🔀 Разные фильтры для разных обработчиков

### Разные условия для консоли и файла

```python
import logging
import sys

class ErrorFilter(logging.Filter):
    def filter(self, record):
        return record.levelno >= logging.ERROR

class DebugFilter(logging.Filter):
    def filter(self, record):
        return record.levelno >= logging.DEBUG

logger = logging.getLogger(__name__)
logger.setLevel(logging.DEBUG)

# КОНСОЛЬ: только ERROR и выше
console = logging.StreamHandler(sys.stdout)
console.addFilter(ErrorFilter())
logger.addHandler(console)

# ФАЙЛ: DEBUG и выше
file = logging.FileHandler('app.log')
file.addFilter(DebugFilter())
logger.addHandler(file)

logger.debug('DEBUG')      # ✅ только в файл
logger.info('INFO')        # ✅ только в файл
logger.warning('WARNING')  # ✅ только в файл
logger.error('ERROR')      # ✅ консоль И файл
logger.critical('CRITICAL')# ✅ консоль И файл
```

**На консоль:**
```
ERROR
CRITICAL
```

**В файле (app.log):**
```
DEBUG
INFO
WARNING
ERROR
CRITICAL
```

## 🔧 Изменение LogRecord

### Можно добавлять или менять атрибуты

```python
import logging

class EnrichFilter(logging.Filter):
    def filter(self, record):
        # Добавляем новые атрибуты
        record.user_id = 123
        record.session_id = 'abc456'
        record.request_time = 1.5
        return True

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()

# Форматер может использовать эти атрибуты
fmt = 'user={user_id} session={session_id} time={request_time}s: {message}'
formatter = logging.Formatter(fmt, style='{')
handler.setFormatter(formatter)

handler.addFilter(EnrichFilter())
logger.addHandler(handler)

logger.info('Обработка запроса')
```

Вывод:
```
user=123 session=abc456 time=1.5s: Обработка запроса
```

### Можно менять существующие атрибуты

```python
import logging

class MaskPasswordFilter(logging.Filter):
    def filter(self, record):
        # Меняем сообщение (удаляем пароли)
        msg = record.getMessage()
        if 'password=' in msg:
            msg = msg.replace('password=***', 'password=***')
        record.msg = msg
        return True

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()
handler.addFilter(MaskPasswordFilter())
logger.addHandler(handler)

logger.warning('Попытка входа: login=admin password=secret123')
```

Вывод (пароль скрыт):
```
Попытка входа: login=admin password=***
```

## 🌳 Фильтры на логгере vs на обработчике

### На логгере

Фильтр выполняется ДО передачи события обработчикам:

```python
import logging

def my_filter(record):
    return record.levelno >= logging.WARNING

logger = logging.getLogger(__name__)
logger.addFilter(my_filter)  # фильтр на ЛОГГЕРЕ

handler = logging.StreamHandler()
logger.addHandler(handler)

logger.debug('DEBUG')      # отфильтруется ДО обработчика
logger.warning('WARNING')  # пройдёт
```

### На обработчике

Фильтр выполняется ПОСЛЕ прохождения уровня логгера:

```python
import logging

def my_filter(record):
    return record.levelno >= logging.WARNING

logger = logging.getLogger(__name__)
logger.setLevel(logging.DEBUG)

handler = logging.StreamHandler()
handler.addFilter(my_filter)  # фильтр на ОБРАБОТЧИКЕ
logger.addHandler(handler)

logger.debug('DEBUG')      # пройдёт логгер, отфильтруется обработчиком
logger.warning('WARNING')  # пройдёт оба
```

## 📝 Примечания

### Примечание 1: фильтры как функции

```python
import logging

# Вместо класса можно просто функцию
def only_errors(record):
    return record.levelno >= logging.ERROR

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()
handler.addFilter(only_errors)
logger.addHandler(handler)

logger.info('INFO')     # ❌ не выводится
logger.error('ERROR')   # ✅ выводится
```

### Примечание 2: список фильтров

```python
import logging

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()

filt1 = logging.Filter()
filt2 = logging.Filter()

handler.addFilter(filt1)
handler.addFilter(filt2)

print(handler.filters)  # список фильтров обработчика
print(logger.filters)   # список фильтров логгера
```

### Примечание 3: удаление фильтра

```python
import logging

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()

filt = logging.Filter()
handler.addFilter(filt)
print(handler.filters)      # [filt]

handler.removeFilter(filt)
print(handler.filters)      # []
```

### Примечание 4: повторное добавление фильтра

Если попытаться добавить одинаковый фильтр дважды, второй раз проигнорируется:

```python
import logging

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()

filt = logging.Filter()
handler.addFilter(filt)
handler.addFilter(filt)

print(len(handler.filters))  # 1 (не 2!)
```

### Примечание 5: пользовательские атрибуты в extra

Фильтр может проверять пользовательские атрибуты из `extra`:

```python
import logging

class UserFilter(logging.Filter):
    def filter(self, record):
        # getattr для безопасной проверки (может не быть)
        user_id = getattr(record, 'user_id', None)
        return user_id == 123

logger = logging.getLogger(__name__)
handler = logging.StreamHandler()
handler.addFilter(UserFilter())
logger.addHandler(handler)

logger.info('Событие 1', extra={'user_id': 123})   # ✅ выводится
logger.info('Событие 2', extra={'user_id': 456})   # ❌ не выводится
logger.info('Событие 3')                             # ❌ не выводится
```

## ✅ Главные правила

✅ Фильтр — это дополнительное условие (кроме уровня)  
✅ Наследуй от `logging.Filter` или просто создай класс с `filter()`  
✅ Метод `filter()` возвращает True (пропустить) или False (отфильтровать)  
✅ LogRecord содержит всю информацию о событии  
✅ Несколько фильтров на одном обработчике — И логика (все должны вернуть True)  
✅ Разные фильтры на разных обработчиках — независимо  
✅ Фильтр может менять атрибуты LogRecord  
✅ На логгере фильтр срабатывает ДО обработчиков  
✅ На обработчике фильтр срабатывает ПОСЛЕ уровня логгера  
✅ Можно использовать функции вместо классов
