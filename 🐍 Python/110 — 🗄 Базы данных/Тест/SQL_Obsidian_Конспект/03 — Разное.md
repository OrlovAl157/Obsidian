# Разное

> Источник: лист Excel-конспекта. Перенесено в Markdown для Obsidian.

| Функция | Что делает | Пример | Результат | Колонка 5 | Оператор  | Описание |
| --- | --- | --- | --- | --- | --- | --- |
| Разное SQL |  |  |  |  | Порядок записи запросов |  |
| = | равно |  |  |  | SELECT  | Данные для извлечения |
| !=<br><> | не равно |  |  |  | FROM  | Таблица для извлечения данных |
| < | меньше |  |  |  |     INNER JOIN ... ON | сопостовляем две таблицы по условию |
| <= | меньше или равно |  |  |  |     LEFT JOIN ... ON | присоединяем к FROM таблице новую |
| > | больше |  |  |  |     RIGHT JOIN ... ON | присоединяем к новой таблице из FROM |
| >= | больше или равно |  |  |  |     CROSS JOIN ... ON | создаём возможные пересечения строк двух таблиц |
| <=> | эквивалентно | SELECT 'x' != NULL, 10 > NULL, NULL = NULL; | +--------------+---------------+--------------------+<br>\| 'x' != NULL \| 10 > NULL \| NULL = NULL \|<br>+--------------+---------------+--------------------+<br>\| NULL         \| NULL        \| NULL               \|<br>+--------------+---------------+--------------------+ |  | WINDOW | Cоздание оконной функции |
|  |  | SELECT 'x' <=> NULL, '5' <=> NULL, NULL <=> NULL; | +------------------+------------------+----------------------+<br>\| 'x' <=> NULL \| '5' <=> NULL \| NULL <=> NULL \|<br>+------------------+------------------+----------------------+<br>\| 0                     \| 0                   \| 1                         \|<br>+------------------+------------------+----------------------+ |  | WHERE  | Фильтрация на уровне записей |
| BETWEEN | указание диапазона (включительно) | BETWEEN 0 AND 100 | 56 |  | GROUP BY  | Создание групп |
| IS NULL | проверяет, что значение NULL | WHERE row IS NULL | NULL |  | HAVING  | Фильтрация на уровне групп |
| IS NOT NULL | проверяет, что значение НЕ NULL | WHERE row IS NOT NULL | ' ' |  | ORDER BY  | Порядок сортировки результатов |
| AND | И | WHERE number > 0<br>AND number < 10 | 5 |  | LIMIT  | Ограничение количества записей |
| OR | ИЛИ | WHERE country = 'USA'<br>OR country = 'Russia' | USA<br>Russia |  | OFFSET | Начало извлечения записей. Нумерация идёт c 0. |
| IN | в диапазоне | numbers IN '50', '55', '53', '54' | 50<br>53<br>54<br>55 |  | UNION | присоединяем к результату нижний результат, по умолчанию удаляет одинаковые графы (DISTINCT), чтобы добавить все, нужно дописать ALL (UNION ALL) |
| NOT | НЕТ | WHERE NOT country = 'USA' | Russia |  |  |  |
| NOT IN | НЕТ в диапазоне | numbers NOT IN '50', '55', '53', '54' | 49<br>51<br>52<br>56 |  |  |  |
| LIKE | как | LIKE '_90%' | A90BCDEFG |  |  |  |
| DISTINCT  | уникальные значения | SELECT DISTINCT name | John, Mary, Paul |  |  |  |
| LIMIT | ограничение вывода | LIMIT 2 | 1<br>2 |  |  |  |
| OFFSET | с какой позиции выводить | OFFSET 3 LIMIT 2 | 3<br>4 |  |  |  |
| AS | КАК, создание псевдонима графы | SELECT nums AS numbers | numbers |  |  |  |
| DESC | сортировать в обратном порядке | ORDER BY numbers DESC | 5<br>4<br>3<br>2<br>1<br>0 |  |  |  |
| FIELD() | Функция FIELD() возвращает индекс одного из аргументов (начиная со второго аргумента, который имеет индекс 1) при совпадении первого аргумента с ним, или 0 при отсутствии совпадений. | ORDER BY FIELD('C', 'D', 'A') | C<br>D<br>A |  |  |  |
| EXISTS() | Фильтрация в таблице, у которой  значение поля (НЕ) существует в другой таблице. Например, вывести значения, если id таблицы есть в другой таблице. Можно заменть на IN.  | SELECT title<br>FROM Books<br>WHERE EXISTS (SELECT *<br>              FROM BooksRental<br>              WHERE book_id = Books.id) | Harry Potter<br>Fight Club<br>The Green Mile<br>It  |  |  |  |
| ALL() | Участвуют все значения из подзапроса. | SELECT num<br>FROM Numbers<br>WHERE num > ALL (SELECT num<br>                                   FROM numbers<br>                                   WHERE num < 6) | 6<br>7<br>8<br>9 |  |  |  |
| ANY() | Участвует любое значение из подзапроса. Лучше использовать IN. | SELECT num<br>FROM Numbers<br>WHERE num > ANY (SELECT num<br>                                   FROM numbers<br>                                   WHERE num < 6) | 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9 |  |  |  |
| DESCRIBE | Получает информацию об всех полях таблицы. | DESCRIBE Books | +--------------+-----------------+---------+--------+------------+--------------------+<br>\| Field          \| Type             \| Null     \| Key  \| Default  \| Extra                  \|<br>+--------------+-----------------+---------+--------+------------+--------------------+<br>\| id               \| int                \| NO      \| PRI   \| NULL     \| auto_increment  \|<br>\| title            \| varchar(40)  \| YES    \|          \| NULL     \|                           \|<br>\| author_id   \| int                \| YES    \| MUL  \| NULL     \|                           \|<br>+--------------+-----------------+---------+--------+------------+----------------------+ |  |  |  |
| IGNORE | Не выводить ошибки при изменении данных в таблице (добавление, обновлении, удалении, замене) и идти дальше по запросу. Ошибочные значения не изменяются. | UPDATE IGNORE FROM Books<br>SET id = 'NULL'<br>WHERE id LIKE '%' | В таблице ничего не изменится |  |  |  |
| WITH | Обобщённое табличное выражение. Создаём для упрощёния временную таблицу, к которой обращаемся один раз. |  |  |  |  |  |
| WITH RECURSIVE | Рекурсивное обобщённое табличное выражение. Создаём для упрощёния временную таблицу, к которой обращаемся один раз. |  |  |  |  |  |
| cte_max_recursion_depth | Увеличение значение системной переменной, отвечающей за максимально допустимое количество итераций рекурсивного запроса | SET @@cte_max_recursion_depth := 10000; |  |  |  |  |
