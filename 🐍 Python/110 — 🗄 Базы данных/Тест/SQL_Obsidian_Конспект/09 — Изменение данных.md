# Изменение данных

> Источник: лист Excel-конспекта. Перенесено в Markdown для Obsidian.

| Колонка 1 | Колонка 2 |
| --- | --- |
| Обновление данных: | Пример: |

| Колонка 1 | Колонка 2 |
| --- | --- |
| UPDATE IGNORE ...<br>INNER JOIN ...<br>SET price = 9.99,<br>        amount = 50<br>WHERE ...<br>ORDER BY ...<br>LIMIT... | UPDATE Books<br>SET price = CASE<br>                WHEN amount < 5 THEN price * 0.9<br>                WHEN amount BETWEEN 5 AND 10 THEN price * 0.7<br>                ELSE price * 0.5<br>            END; |

## Удаление данных:

```sql
DELETE IGNORE FROM ...
USING ... INNER JOIN ON 
WHERE ...
ORDER BY ...
LIMIT ...
```

| Колонка 1 | Колонка 2 |
| --- | --- |
| TRUNCATE ... | Работает эффективнее, удяляет все значения из таблицы. Нельзя бездумно удалять с помощью TRUNCATE, так как эта запись не попадает в журнал и не подлежит восстановлению, а после DELETE можно восстановить таблицу. |

## Вставка данных:

```sql
INSERT IGNORE INTO Books (id, title, author, price)
VALUES (1, 'Animal Farm', 'George Orwell', 9.99),
               (6, 'Lord of the Flies', 'William Golding', 5.99);
```

```sql
INSERT INTO Books
SET id = 6,
        title = 'Animal Farm',
        author = 'George Orwell',
        price = 9.99;
```

```sql
INSERT INTO Books (id, title, author, price)
SELECT id, title, author, price
FROM NewBooks;
```

```sql
REPLACE INTO Books (id, title, author, price)
VALUES (1, 'Animal Farm', 'George Orwell', 9.99),
               (6, 'Lord of the Flies', 'William Golding', 5.99);
```
