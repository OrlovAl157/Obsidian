# 🗄 Базы данных

## 📌 Коротко

> База данных — организованная и структурированная коллекция данных для хранения, управления и обработки информации.

---

## 📖 Теория

### Что такое база данных?

**База данных (БД)** — в широком смысле любой набор взаимосвязанной информации. В контексте информатики — структурированная коллекция данных с механизмами хранения и доступа.

**Данные** — собранная и сохранённая информация о сущности (человеке, месте, предмете) и её атрибутах.

### Где используются БД

- **Бизнес** — клиенты, продажи, маркетинг
- **Медицина** — истории болезни, результаты анализов
- **Финансы** — счета, транзакции, кредитные истории

### Возможности баз данных

- Хранение и организация больших объёмов данных
- Структурирование данных и определение связей между ними
- Поддержка языков запросов для доступа к данным

---

## 📖 Модели баз данных

### 📁 Плоские файлы

Данные хранятся в виде текстовых записей без структуры и связей.

> 💡 Пример: файлы CSV — используются и по сей день.

---

### 🌲 Иерархическая модель (1960-е)

Данные представлены в виде **древовидной структуры** — каждая запись имеет одну родительскую запись.

```
Компания
├── Отдел A
│   ├── Сотрудник 1
│   └── Сотрудник 2
└── Отдел B
    └── Сотрудник 3
```

| ✅ Плюсы | ❌ Минусы |
|---|---|
| Эффективна для иерархических данных | Неуниверсальна |
| Быстрый поиск по дереву | Реальный мир сложнее дерева |

---

### 🕸 Сетевая модель (1960–1970-е)

Расширение иерархической — каждая запись может быть связана с **несколькими** другими записями.

| ✅ Плюсы | ❌ Минусы |
|---|---|
| Универсальна, богатые связи | Сложна в реализации |
| Гибкое моделирование | Трудно менять структуру |

---

### 📊 Реляционная модель (1970-е — наши дни)

Данные хранятся в **таблицах** со строками и столбцами. Таблицы связаны через ключи.

> 💡 Самая распространённая модель по сей день. Подробнее — [[02 — 🔗 Реляционные базы данных]]

---

### 🔑 Модель ключ — значение (начало 2000-х)

Данные хранятся в виде пар **ключ → значение** (хеш-таблица).

```
"user:1" → {"name": "Саша", "age": 25}
"user:2" → {"name": "Иван", "age": 30}
```

| ✅ Плюсы | ❌ Минусы |
|---|---|
| Очень быстрый доступ | Не подходит для сложных запросов |
| Простая структура | Ограниченный анализ данных |

> 💡 Применение: кэширование, сессии, очереди.

---

### 📄 Документная модель (начало 2000-х)

Данные хранятся в виде **документов** (JSON, XML). Каждый документ — самодостаточная единица данных.

```json
{
  "id": 1,
  "name": "Ноутбук",
  "specs": { "ram": "16GB", "cpu": "Intel i7" },
  "tags": ["электроника", "компьютеры"]
}
```

| ✅ Плюсы | ❌ Минусы |
|---|---|
| Гибкая структура документов | Сложные связи между документами |
| Высокая масштабируемость | Нет строгой схемы |

> 💡 Применение: каталоги товаров, социальные сети, веб-приложения.

---

## 💡 Полезные советы

- **SQL vs NoSQL** — реляционные БД используют SQL, нереляционные — NoSQL. Выбор зависит от задачи.
- **Нереляционные модели** — иерархическая, сетевая, ключ-значение, документная — часто называют NoSQL.
- Также существуют: **графовые**, **колоночные**, **объектно-ориентированные** модели.
