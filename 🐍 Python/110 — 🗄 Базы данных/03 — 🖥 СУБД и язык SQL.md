# 🖥 СУБД и язык SQL

## 📌 Коротко

> СУБД — программное обеспечение-посредник между базой данных и пользователем. SQL — стандартный язык запросов для управления реляционными БД.

---

## 📖 Теория

### Что такое СУБД?

**СУБД (система управления базами данных)** — программное обеспечение, которое служит интерфейсом между базой данных и разработчиком, пользователем или программой.

> ⚠️ База данных и СУБД — не одно и то же. БД — это данные. СУБД — это программа для работы с ними.

**Возможности СУБД:**
- Извлечение, добавление, изменение и удаление данных
- Резервное копирование и восстановление данных
- Многопользовательский доступ (несколько пользователей одновременно)
- Защита данных: аутентификация, авторизация, шифрование
- Инструменты анализа и визуализации данных

**Запрос** — каждое обращение к базе данных через СУБД.

---

### Типы СУБД

**По расположению данных:**

| Тип | Описание |
|---|---|
| **Локальная** | База данных на одном сервере/устройстве |
| **Распределённая** | Данные на нескольких серверах в разных местах |

**По способу хранения:**

| Тип | Описание | Пример |
|---|---|---|
| **Клиент-серверная** | СУБД и БД на сервере, доступ с любого устройства | PostgreSQL, MySQL |
| **Файл-серверная** | БД на сервере, СУБД на каждом устройстве | MS Access |
| **Встраиваемая** | Встроена в приложение, работает только внутри него | SQLite |

**По языку запросов:**

| Тип | Язык | Применение |
|---|---|---|
| **SQL-ориентированные** | SQL | Реляционные БД |
| **NoSQL-ориентированные** | Разные языки | Нереляционные БД, большие объёмы |

---

### Язык SQL

**SQL (Structured Query Language)** — язык структурированных запросов, специально разработанный для работы с реляционными базами данных.

**Особенности SQL:**
- **Декларативный** — описываешь *что* нужно получить, а не *как*. СУБД сама решает как выполнить запрос.
- **Слабоструктурированный** — запросы напоминают обычные предложения английского языка
- **Стандартный** — единый язык для всех реляционных СУБД (с диалектами)

```sql
-- Пример SQL-запроса
SELECT title
FROM Movies
WHERE rating BETWEEN 7 AND 9
```

> 💡 Исторически язык назывался **SEQUEL** (Structured English Query Language), но название сократили из-за торговой марки.

> ⚠️ Каждая СУБД может иметь свой **диалект** SQL — небольшие отличия в синтаксисе. Но большинство запросов переносимы между СУБД.

---

### Популярные реляционные СУБД

| СУБД | Тип | Особенности | Применение |
|---|---|---|---|
| **PostgreSQL** | Клиент-серверная | Мощная, бесплатная, без ограничений на размер БД | Крупные проекты |
| **MySQL** | Клиент-серверная | Простая, быстрая, широкое сообщество | Средние и малые проекты |
| **SQLite** | Встраиваемая | Один файл, без сервера, минимальная настройка | Небольшие приложения |

**PostgreSQL** — поддерживает расширения на Python, Perl, Java. Нет ограничений на размер БД.

**MySQL** — используется на платформе Stepik. Поддерживает Windows, Linux, Unix. API для C++, Python, Ruby, Java.

**SQLite** — работает на компьютерах, смартфонах, ТВ, приставках. Не требует администрирования.

---

## 💡 Полезные советы

- **Языки запросов** — часто называют языками программирования для СУБД, но SQL скорее декларативный язык, а не императивный.
- **Денормализация** — намеренное нарушение нормальных форм ради ускорения чтения. Иногда оправдано.
- **NoSQL** — не значит "против SQL". Это просто другая модель хранения данных для других задач.
