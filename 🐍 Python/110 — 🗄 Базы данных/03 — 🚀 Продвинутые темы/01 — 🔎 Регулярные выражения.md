## Справка по метасимволам

| Метасимвол | Что делает | Пример | Результат |
| --- | --- | --- | --- |
| `.` | один любой символ | `.......` | beegeek |
| `\` | экранирование | `pygen\.ru` | pygen.ru |
| `*` | 0 и более раз | `aaa\d*` | aaa, aaa8657 |
| `+` | 1 и более раз | `\d+` | 19587792597 |
| `?` | 0 или 1 раз | `https?://` | http:// или https:// |
| `{n}` | ровно `n` раз | `be{5}ge{4}k` | beeeeegeeeek |
| `{n,k}` | от `n` до `k` раз | `a{3,5}` | aaa, aaaa, aaaaa |
| `{n,}` | не менее `n` раз | `a{4,}` | aaaa, aaaaa... |
| `^` | начало строки | `^[Hh]ello` | Hello... |
| `$` | конец строки | `[Gg]oodbye$` | ...Goodbye |
| `\|` | ИЛИ | `[b-k]\|[4-8]` | d |
| `[...]` | один из набора | `[bd]` | b или d |
| `[^...]` | НЕ из диапазона | `[^7-9]` | 6 |
| `(...)` | группировка | `(SELECT){2}` | SELECTSELECT |

## ⚙️ Метасимволы символов

**Назначение:** искать конкретные типы символов.

| Метасимвол | Что делает | Пример |
| --- | --- | --- |
| `.` | один любой символ (кроме `\n`) | `.......` находит любую 7-символьную строку |
| `\d` | цифра 0-9 | `\d\d\d` находит 123, 456, 999 |
| `\D` | не цифра | `bee\Dgee\D` находит bee-geek, bee_geek |
| `\w` | буква, цифра или `_` | `\w\w\w\w` находит _x1_, test, abc9 |
| `\W` | не буква, не цифра, не `_` | `\W\W` находит !?, ->, #@ |
| `\s` | пробельный символ (пробел, табуляция) | `...\s....` находит bee geek |
| `\S` | не пробельный символ | `...\S....` находит bee-geek, bee_geek |
| `\` | экранирование специальных | `pygen\.ru` находит pygen.ru |

**Примеры:**

```sql
SELECT REGEXP_LIKE('test123', '\d+');      -- 1 (есть цифры)
SELECT REGEXP_LIKE('hello world', '\s');   -- 1 (есть пробел)
SELECT REGEXP_LIKE('email@domain.com', '\w+@\w+\.\w+');  -- 1 (email)
```

**Важное:** `\d`, `\w`, `\s` очень удобны для паттернов, используй их часто.

## 🔢 Метасимволы количества

**Назначение:** указать сколько раз должна повторяться предыдущая часть.

| Метасимвол | Что делает | Пример | Находит |
| --- | --- | --- | --- |
| `*` | 0 и более раз | `aaa\d*` | aaa, aaa8, aaa8657 |
| `+` | 1 и более раз | `\d+` | 1, 19, 19587792597 |
| `?` | 0 или 1 раз | `https?://` | http://, https:// |
| `{n}` | ровно `n` раз | `be{5}` | beeeee |
| `{n,k}` | от `n` до `k` раз | `a{3,5}` | aaa, aaaa, aaaaa |
| `{n,}` | не менее `n` раз | `a{4,}` | aaaa, aaaaa, aaaaaa... |
| `{,k}` | не более `k` раз | `a{,3}` | пусто, a, aa, aaa |

**Примеры:**

```sql
SELECT REGEXP_LIKE('phone: 123456789', '\d{9}');      -- 1 (9 цифр подряд)
SELECT REGEXP_LIKE('beeeeek', 'e{4,}');               -- 1 (4+ буквы e)
SELECT REGEXP_LIKE('color', 'colou?r');               -- 1 (u опционально)
SELECT REGEXP_LIKE('http://site.ru', 'https?://');   -- 1 (http или https)
```

**Важное:** `+` и `*` — самые частые, `?` — для опциональных частей, `{n,m}` — для точных диапазонов.

## 🎯 Метасимволы позиций

**Назначение:** искать в начале, конце или границах слов.

| Метасимвол | Что делает | Пример | Находит |
| --- | --- | --- | --- |
| `^` | начало строки | `^[Hh]ello` | Hello в начале строки |
| `$` | конец строки | `goodbye$` | goodbye в конце строки |
| `\b` | граница слова | `\bcat\b` | cat как отдельное слово |
| `\B` | не граница слова | `\Bcat` | scat, ducati (cat внутри слова) |

**Примеры:**

```sql
-- Строка начинается с числа
SELECT REGEXP_LIKE('123 street', '^\d+');              -- 1

-- Строка заканчивается на .com
SELECT REGEXP_LIKE('example.com', '\.com$');           -- 1

-- Слово cat отдельно
SELECT REGEXP_LIKE('concatenate', '\bcat\b');          -- 0 (не отдельно)
SELECT REGEXP_LIKE('the cat sat', '\bcat\b');          -- 1 (отдельно)

-- cat не в начале слова
SELECT REGEXP_LIKE('scat', '\Bcat');                   -- 1 (внутри слова)
```

**Важное:** `\b` полезна для поиска целых слов, `^` и `$` — для проверки начала/конца.

## 📍 Наборы и диапазоны

**Назначение:** выбрать один символ из набора или диапазона.

| Конструкция | Что делает | Пример | Находит |
| --- | --- | --- | --- |
| `[abc]` | один из символов | `[aeiou]` | a, e, i, o или u |
| `[a-z]` | диапазон строчных букв | `[a-z]` | любую букву от a до z |
| `[A-Z]` | диапазон заглавных букв | `[A-Z]` | любую букву от A до Z |
| `[0-9]` | диапазон цифр | `[0-9]` | любую цифру |
| `[a-zA-Z0-9]` | буквы и цифры | `[a-zA-Z0-9]` | буква или цифра |
| `[^abc]` | НЕ один из символов | `[^0-9]` | что угодно кроме цифры |

**Примеры:**

```sql
-- Гласные буквы
SELECT REGEXP_LIKE('hello', '[aeiou]{2}');             -- 1 (две гласные подряд)

-- ID формата id_123 (буквы, подчеркивание, цифры)
SELECT REGEXP_LIKE('user_id_42', '[a-z_0-9]+');        -- 1

-- НЕ буква
SELECT REGEXP_LIKE('test-123', '[^a-z]');              -- 1 (есть тире)

-- Буквы и цифры
SELECT REGEXP_LIKE('abc123XYZ', '[a-zA-Z0-9]+');       -- 1
```

**Важное:** диапазоны работают только с ASCII символами, для кириллицы используй явные символы.

## 🔤 Специальные последовательности

**Назначение:** использовать группировку и ссылки на группы.

| Конструкция | Что делает | Пример |
| --- | --- | --- |
| `(...)` | группировка | `(SELECT){2}` находит SELECTSELECT |
| `(...)\1` | ссылка на группу 1 | `(\b\w+\b) \1` находит повторяющиеся слова |
| `(?P<name>...)` | именованная группа | `(?P<word>\w+) (?P=word)` повторяющиеся слова |

**Примеры:**

```sql
-- Повторяющиеся слова (hello hello)
SELECT REGEXP_LIKE('say hello hello to world', '(\w+) \1');    -- 1

-- Две одинаковые части подряд
SELECT REGEXP_LIKE('testtest', '(test){2}');                   -- 1
```

**Важное:** ссылки на группы редко используются в SQL, но полезны для сложных паттернов.

## ⚡ Жадный и ленивый поиск

**Назначение:** контролировать как много матчить: максимально или минимально.

| Тип | Синтаксис | Поведение |
| --- | --- | --- |
| Жадный | `.*`, `.+`, `.*.*` | матчит как МОЖНО БОЛЬШЕ |
| Ленивый | `.*?`, `.+?` | матчит как МОЖНО МЕНЬШЕ |

**Примеры:**

```sql
-- Жадный: от первого <B> до ПОСЛЕДНЕГО </B>
SELECT REGEXP_SUBSTR('<B>Russia</B> and <B>Belarus</B>', '<B>.*</B>');
-- Результат: <B>Russia</B> and <B>Belarus</B> (всё!)

-- Ленивый: от первого <B> до ПЕРВОГО </B>
SELECT REGEXP_SUBSTR('<B>Russia</B> and <B>Belarus</B>', '<B>.*?</B>');
-- Результат: <B>Russia</B> (только первый матч)

-- Практический пример: извлечение содержимого
SELECT REGEXP_SUBSTR(html, '<body>.*?</body>') FROM pages;
-- Ленивый поиск нужен здесь!
```

**Важное:** по умолчанию используется жадный поиск, добавь `?` для ленивого.

## 🔍 Функции регулярных выражений

**Назначение:** работать с регулярными выражениями в SQL.

| Функция | Что делает | Синтаксис |
| --- | --- | --- |
| `REGEXP_LIKE` | проверяет совпадение (TRUE/FALSE) | `REGEXP_LIKE(str, pattern)` |
| `REGEXP_INSTR` | найти позицию совпадения | `REGEXP_INSTR(str, pattern)` |
| `REGEXP_SUBSTR` | извлечь подстроку | `REGEXP_SUBSTR(str, pattern)` |
| `REGEXP_REPLACE` | заменить по шаблону | `REGEXP_REPLACE(str, pattern, repl)` |
| `REGEXP` / `RLIKE` | альтернативный синтаксис | `str REGEXP pattern` |

**Примеры:**

```sql
-- REGEXP_LIKE: проверка
SELECT REGEXP_LIKE('beegeek', 'bee');          -- 1 (есть совпадение)

-- REGEXP_INSTR: позиция
SELECT REGEXP_INSTR('beegeek', 'e{2}');        -- 2 (позиция двух e подряд)

-- REGEXP_SUBSTR: извлечение
SELECT REGEXP_SUBSTR('phone: 123456789', '\d+');  -- 123456789

-- REGEXP_REPLACE: замена
SELECT REGEXP_REPLACE('hello123world', '\d+', 'XXX');  -- helloXXXworld

-- REGEXP оператор
SELECT * FROM users WHERE email REGEXP '[a-z]+@[a-z]+\.[a-z]+';
```

**Важное:** `REGEXP_LIKE` самая частая, остальные для специальных задач.

## 🚩 Флаги

**Назначение:** изменить поведение поиска.

| Флаг | Что дает |
| --- | --- |
| `c` | учитывает регистр (case-sensitive) |
| `i` | игнорирует регистр (case-insensitive) |
| `m` | многострочный режим (^ и $ работают на строки внутри) |
| `n` | точка `.` матчит переводы строк `\n` |

**Примеры:**

```sql
-- С учётом регистра
SELECT REGEXP_LIKE('Beegeek', 'bee', 'c');     -- 0 (разные регистры)

-- Без учёта регистра (по умолчанию)
SELECT REGEXP_LIKE('Beegeek', 'bee');          -- 1 (находит)

-- Многострочный режим
SELECT REGEXP_LIKE('Foo\nBar', '^Bar', 'm');   -- 1 (Bar в начале строки 2)
```

**Важное:** без флага `c` поиск игнорирует регистр по умолчанию.

## 💡 Практические замечания

- **Экранирование точки:** `.` — любой символ, `\.` — буквально точка
- **Ленивый поиск:** используй `.*?` вместо `.*` когда нужно найти первое совпадение
- **Границы слов:** `\b` для целых слов, `\B` для внутри слов
- **Производительность:** сложные regex могут быть медленными на больших данных
- **Тестирование:** тестируй regex отдельно перед использованием в WHERE
- **NULL:** regex с NULL всегда возвращают NULL

## ⚠️ Частые ошибки

**❌ ОШИБКА:**
```sql
SELECT * FROM users WHERE email REGEXP '@';
```
❌ Точка в email не экранирована!

**✅ ВЕРНО:**
```sql
SELECT * FROM users WHERE email REGEXP '.*@.*\..*';
```

---

**❌ ОШИБКА:**
```sql
SELECT REGEXP_SUBSTR(text, '<div>.*</div>');
```
❌ Жадный поиск! Если несколько div, возьмёт от первого до последнего

**✅ ВЕРНО:**
```sql
SELECT REGEXP_SUBSTR(text, '<div>.*?</div>');
```

---

**❌ ОШИБКА:**
```sql
SELECT * FROM users WHERE username REGEXP '^[a-z]+$';
-- НО: находит john_123 (содержит совпадение подстроки)
```

❌ Нужны границы слова!

**✅ ВЕРНО:**
```sql
SELECT * FROM users WHERE username REGEXP '^[a-z]+$';
-- Теперь совпадение ВСЕЙ строки, не подстроки
```
