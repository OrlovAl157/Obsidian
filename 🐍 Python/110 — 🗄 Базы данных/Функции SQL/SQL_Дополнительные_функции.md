# 🔧 Памятка: Дополнительные функции SQL

Вспомогательные функции для работы с `NULL`, условной логикой и приведением типов.

### 💡 Ключевые особенности

- **`ISNULL()`** — возвращает `1` если значение `NULL`, иначе `0`.

- **`IF()`** — SQL-аналог тернарного оператора: выбирает одно из двух значений по условию.

- **`COALESCE()`** — возвращает первый не-`NULL` аргумент из списка. Принимает любое количество аргументов.

- **`IFNULL()`** — частный случай `COALESCE()` только для двух аргументов: заменяет `NULL` на альтернативу.

- **`NULLIF()`** — возвращает `NULL` если два значения равны. Полезно для защиты от деления на ноль.

- **`CONVERT()` и `CAST()`** — приводят значение к нужному типу. Делают одно и то же, отличается только синтаксис.

### 💡 Полезные советы

- **Предпочитайте `COALESCE()` вместо `IFNULL()`**: они делают одно и то же, но `COALESCE()` принимает любое количество аргументов — это стандарт SQL.

- **`NULLIF()` для безопасного деления**: `value / NULLIF(divisor, 0)` — если делитель `0`, вернётся `NULL` вместо ошибки.

- **`CONVERT()` vs `CAST()`**: синтаксис разный, поведение одинаковое. `CAST` — стандарт SQL, `CONVERT` — MySQL-специфика.

- **Несовместимые типы**: при приведении к дате → `NULL`, при приведении к числу → `0`.

- **Автоприведение типов**: MySQL сам приводит типы при операциях (`1 + '15'` → `16.0`), но лучше делать это явно.

### 🚀 Быстрая справка

| **Функция** | **Что делает** | **Пример** | **Результат** |
|---|---|---|---|
| `ISNULL(val)` | проверка на NULL | `ISNULL(NULL)` | `1` |
| `ISNULL(val)` | проверка на NULL | `ISNULL(0)` | `0` |
| `IF(cond, v1, v2)` | выбор по условию | `IF(1>0, 'да', 'нет')` | `'да'` |
| `COALESCE(a, b, c)` | первый не-NULL | `COALESCE(NULL, 2, 3)` | `2` |
| `IFNULL(val, alt)` | замена NULL | `IFNULL(NULL, 'нет')` | `'нет'` |
| `NULLIF(v1, v2)` | NULL если равны | `NULLIF('a', 'a')` | `NULL` |
| `NULLIF(v1, v2)` | NULL если равны | `NULLIF('a', 'b')` | `'a'` |
| `CONVERT(val, type)` | приведение типа | `CONVERT(100.78, DECIMAL(4,1))` | `100.8` |
| `CAST(val AS type)` | приведение типа | `CAST(100.78 AS DECIMAL(4,1))` | `100.8` |

## 🔍 ISNULL()

```sql
-- Проверяет значение на NULL
-- Возвращает: 1 если NULL, 0 если не NULL

SELECT ISNULL(0),       -- → 0
       ISNULL(1),       -- → 0
       ISNULL(1.1),     -- → 0
       ISNULL(''),      -- → 0  (пустая строка ≠ NULL!)
       ISNULL('bee'),   -- → 0
       ISNULL(NULL);    -- → 1
```

## 🔀 IF()

```sql
-- Тернарный оператор в SQL
-- IF(условие, значение_если_истина, значение_если_ложь)

SELECT IF(1 > 0, 'bee', 'geek'),   -- → 'bee'   (условие истинно)
       IF(1 < 0, 'bee', 'geek');   -- → 'geek'  (условие ложно)

-- Практический пример
SELECT name,
       IF(score >= 90, 'отлично', 'не отлично') AS grade
FROM students;
```

## 🔎 COALESCE()

```sql
-- Возвращает первый аргумент, не равный NULL
-- Принимает любое количество аргументов

SELECT COALESCE(1, 2, 3),          -- → 1    (первый не-NULL)
       COALESCE(NULL, 2, 3),       -- → 2    (первый не-NULL)
       COALESCE(1, NULL, 3),       -- → 1    (первый не-NULL)
       COALESCE(NULL, NULL, 3);    -- → 3    (первый не-NULL)

-- Если все аргументы NULL:
SELECT COALESCE(NULL),             -- → NULL
       COALESCE(NULL, NULL);       -- → NULL

-- Практический пример: подставить значение по умолчанию
SELECT name,
       COALESCE(phone, email, 'нет контакта') AS contact
FROM users;
```

## 🔄 IFNULL()

```sql
-- Заменяет NULL на альтернативное значение
-- IFNULL(значение, альтернатива)
-- ❗ Принимает только два аргумента — используй COALESCE() для большего числа

SELECT IFNULL(NULL, 'geek'),       -- → 'geek'  (NULL → альтернатива)
       IFNULL('bee', 'geek');      -- → 'bee'   (не NULL → исходное)

-- Практический пример
SELECT name,
       IFNULL(discount, 0) AS discount   -- NULL заменяем на 0
FROM orders;
```

## ❌ NULLIF()

```sql
-- Возвращает NULL если два значения равны, иначе первое значение
-- NULLIF(значение1, значение2)

SELECT NULLIF('bee', 'bee'),       -- → NULL   (равны → NULL)
       NULLIF('bee', 'geek');      -- → 'bee'  (не равны → первое)

-- ⭐ Главный use-case: безопасное деление
-- Защита от деления на ноль:
SELECT total / NULLIF(count, 0) AS average
FROM stats;
-- Если count = 0 → NULLIF вернёт NULL → деление вернёт NULL (не ошибку)
```

## 🔁 CONVERT() и CAST()

```sql
-- Приводят значение к нужному типу
-- Делают одно и то же, разный синтаксис:

-- CONVERT(значение, тип)
SELECT CONVERT(100.78, DECIMAL(4, 1)),   -- → 100.8
       CONVERT(100.78, DECIMAL(3, 0));   -- → 101

-- CAST(значение AS тип)
SELECT CAST(100.78 AS DECIMAL(4, 1)),    -- → 100.8
       CAST(100.78 AS DECIMAL(3, 0));    -- → 101
```

### Поддерживаемые типы

| **Тип** | **Описание** |
|---|---|
| `DATE` | дата |
| `DATETIME` | дата и время |
| `TIME` | время |
| `DECIMAL(M, D)` | число с плавающей точкой (M знаков, D после запятой) |
| `CHAR(N)` | строка фиксированной длины |
| `BINARY` | бинарные данные |
| `SIGNED` | BIGINT со знаком |
| `UNSIGNED` | BIGINT без знака |
| `YEAR` | год |

### Поведение при несовместимых типах

```sql
-- Приведение к дате: несовместимое → NULL
SELECT CONVERT('2023-01-01', DATE),    -- → 2023-01-01  ✅
       CONVERT('beegeek', DATE);       -- → NULL         ❌ (не подходит формат)

-- Приведение к числу: несовместимое → 0
SELECT CONVERT('beegeek', SIGNED),     -- → 0
       CONVERT('beegeek', UNSIGNED),   -- → 0
       CONVERT('beegeek', DECIMAL(5,2)); -- → 0.00

-- Автоматическое приведение (лучше делать явно!)
SELECT 1 + '15';                       -- → 16.0  (строка → число автоматически)
```

## ⚠️ Важные особенности

```sql
-- ❗ Пустая строка ≠ NULL
ISNULL('')    -- → 0  (это не NULL, это пустая строка!)
ISNULL(NULL)  -- → 1

-- ❗ COALESCE предпочтительнее IFNULL
-- IFNULL(a, b)          → только 2 аргумента
-- COALESCE(a, b, c...)  → любое количество, стандарт SQL

-- ❗ NULLIF защищает от деления на ноль
value / NULLIF(divisor, 0)
-- divisor = 5  → вернёт divisor как есть, деление выполнится
-- divisor = 0  → вернёт NULL, результат деления тоже NULL (не ошибка!)

-- ❗ DECIMAL(M, D):
-- M — общее количество значащих цифр
-- D — количество цифр после запятой
CONVERT(100.78, DECIMAL(4, 1))  -- → 100.8  (4 цифры всего, 1 после запятой)
CONVERT(100.78, DECIMAL(3, 0))  -- → 101    (3 цифры всего, 0 после запятой)
```
