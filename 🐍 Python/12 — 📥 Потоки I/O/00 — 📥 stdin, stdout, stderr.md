# 📥 stdin, stdout, stderr

## 🎯 ЧТО ЭТО ВООБЩЕ?

Когда ты запускаешь программу в консоли, происходит взаимодействие между тобой и программой через **три трубопровода** (потоки):

- **stdin** (стандартный ввод) — то, что ты ВВОДИШЬ в программу
- **stdout** (стандартный вывод) — результаты ВЫВОДИТ программа
- **stderr** (стандартный поток ошибок) — ОШИБКИ выводит программа

Каждый поток — это просто файловый дескриптор с номером:

```
stdin  = 0  (ввод)
stdout = 1  (вывод)
stderr = 2  (ошибки)
```

---

## 📍 ДОСТУП К ПОТОКАМ ЧЕРЕЗ sys

В Python модуль `sys` предоставляет все три потока готовыми к использованию:

```python
import sys

sys.stdin   # поток ввода
sys.stdout  # поток вывода
sys.stderr  # поток ошибок
```

Это не простые переменные — это **файловые объекты**! С ними можно работать как с файлами.

---

## 📥 STDIN: ЧИТАЕМ ВВОД ПОЛЬЗОВАТЕЛЯ

### Что это?

`sys.stdin` — это объект, из которого можно **ЧИТАТЬ** то, что вводит пользователь с клавиатуры.

### Способ 1: next()

```python
import sys

line = next(sys.stdin)  # читаем одну строку
print(line)
```

Если пользователь введёт "Привет" и нажмёт Enter:
```
Привет

```

**Заметь:** выводится пустая строка! Потому что пользователь нажимает Enter, добавляется `\n`, и `print()` добавляет ещё один `\n`.

### Способ 2: Цикл for (лучше!)

```python
import sys

for line in sys.stdin:
    print(line.rstrip('\n'))  # удаляем \n перед выводом

print('Завершение программы.')
```

Пользователь вводит:
```
Привет!
Как дела?
```

Нажимает Ctrl+D (Linux/Mac) или Ctrl+Z+Enter (Windows) для завершения.

Программа выводит:
```
Привет!
Как дела?
Завершение программы.
```

Цикл `for` автоматически обрабатывает конец ввода!

### Способ 3: read() — всё сразу

```python
import sys

all_input = sys.stdin.read()  # читаем ВСЁ
print(all_input)
```

Программа ждёт, пока пользователь завершит ввод (Ctrl+D или Ctrl+Z+Enter), потом выводит всё.

### Важно: ВСЁ ПРИХОДИТ КАК СТРОКИ

Даже если пользователь вводит числа или списки, всё будет строками:

```python
import sys

for line in sys.stdin:
    print(type(line))  # всегда <class 'str'>
```

Пользователь вводит:
```
1
[1, 2, 3]
{'key': 'value'}
```

Выводит:
```
<class 'str'>
<class 'str'>
<class 'str'>
```

### Ошибка: попытка писать в stdin

stdin — только для ЧТЕНИЯ! Если попытаться писать:

```python
import sys

sys.stdin.write('Привет!')  # ❌ ошибка!
```

Выдаст:
```
io.UnsupportedOperation: not writable
```

## 📤 STDOUT: ВЫВОДИМ НОРМАЛЬНУЮ ИНФОРМАЦИЮ

### Что это?

`sys.stdout` — объект, в который можно **ПИСАТЬ** информацию. По умолчанию это экран.

### write() и writelines()

```python
import sys

sys.stdout.write('Привет!\n')
sys.stdout.write('Как дела?\n')
```

Выводит:
```
Привет!
Как дела!
```

**Важно:** `write()` НЕ добавляет `\n` автоматически! Ты должен добавить его сам!

Можно писать несколько строк сразу:

```python
import sys

lines = ['Привет!\n', 'Как дела?\n', 'Что делаешь?\n']
sys.stdout.writelines(lines)
```

### Обычно используешь print()

На самом деле `print()` просто пишет в `sys.stdout`:

```python
print("Привет")
# По сути это:
# sys.stdout.write("Привет\n")
```

### Ошибка 1: только строки

```python
import sys

sys.stdout.write(123)  # ❌ ошибка!
```

Выдаст:
```
TypeError: write() argument must be str, not int
```

Только строки!

### Ошибка 2: попытка читать из stdout

stdout — только для ПИСЬМА! Если попытаться читать:

```python
import sys

text = sys.stdout.read()  # ❌ ошибка!
```

Выдаст:
```
io.UnsupportedOperation: not readable
```

## 🔴 STDERR: ВЫВОДИМ ОШИБКИ

### Что это?

`sys.stderr` — объект для **ПИСАНИЯ ОШИБОК И ПРЕДУПРЕЖДЕНИЙ**.

Работает так же, как stdout, но предназначен для ошибок.

```python
import sys

sys.stderr.write('Произошла ошибка!\n')
sys.stderr.write('Ещё одна ошибка!\n')
```

Выводит:
```
Произошла ошибка!
Ещё одна ошибка!
```

В консоли текст часто выделяется **красным цветом**!

### Зачем отдельный поток для ошибок?

**Представь:** программа обрабатывает файлы и выводит результаты.

Ты можешь сказать операционной системе:
- stdout → сохранить в `results.txt`
- stderr → сохранить в `errors.txt`

Результаты и ошибки не смешиваются! 📊

### Пример: функция с предупреждением

```python
import sys

def add(a, b):
    if a > 100:
        sys.stderr.write('⚠️ Предупреждение: функция может работать нестабильно!\n')
    return a + b

result = add(150, 20)
print(result)  # stdout
```

Выводит:
```
⚠️ Предупреждение: функция может работать нестабильно!
170
```

Предупреждение красное, результат — чёрный (на обычный stdout)!

### Важно: stderr НЕ участвует в обработке исключений

```python
import sys

sys.stderr.write('Ошибка!\n')
# программа ПРОДОЛЖАЕТ работать!
print('Я всё ещё здесь!')
```

Выводит:
```
Ошибка!
Я всё ещё здесь!
```

stderr просто пишет информацию, не останавливает программу!

## 📊 СРАВНЕНИЕ

| Поток | Читать | Писать | По умолчанию | Использование |
|---|---|---|---|---|
| **stdin** | ✅ | ❌ | клавиатура | ввод пользователя |
| **stdout** | ❌ | ✅ | экран | вывод результатов |
| **stderr** | ❌ | ✅ | экран | ошибки и предупреждения |

## ✅ ГЛАВНЫЕ ПРАВИЛА

✅ stdin — только для ЧТЕНИЯ  
✅ stdout и stderr — только для ПИСЬМА  
✅ write() НЕ добавляет `\n` (добавляй сам!)  
✅ writelines() пишет список строк  
✅ ВСЕ ДАННЫЕ ИЗ stdin — это строки (str)  
✅ В write() можно писать только строки  
✅ stderr — для ошибок, stdout — для результатов  
✅ Используй цикл `for` для чтения stdin (удобнее всего)
