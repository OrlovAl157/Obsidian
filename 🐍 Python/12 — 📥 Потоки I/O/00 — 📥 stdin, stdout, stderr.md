## 🎯 Что такое потоки

**Потоки ввода, вывода и ошибок** — это **три канала связи** между программой и пользователем (или операционной системой). Они работают как трубопроводы, по которым течёт информация.

Когда ты запускаешь программу, все три потока открываются автоматически:

```
Пользователь/ОС
    ↓ ↑ ↑
    │ │ └─── stderr (ошибки, красный текст)
    │ └───── stdout (результаты, чёрный текст)
    └─────── stdin (ввод с клавиатуры)
    
   Программа
```

**Каждый поток имеет номер** (файловый дескриптор):
- **stdin = 0** (стандартный ввод)
- **stdout = 1** (стандартный вывод)
- **stderr = 2** (стандартный поток ошибок)

Эти номера используются не только в Python, но и в операционных системах (Windows, Linux, macOS).

## 📥 stdin — поток ввода

### Что это?

`sys.stdin` — это **файловый объект**, из которого программа **ЧИТАЕТ** данные, которые вводит пользователь с клавиатуры.

### Доступ к stdin

```python
import sys

sys.stdin  # вот это и есть поток ввода
```

### Способ 1: Функция next()

Читаем одну строку за раз:

```python
import sys

line = next(sys.stdin)  # ждёт, пока пользователь введёт и нажмёт Enter
print("Ты ввёл:", line)
```

Пользователь вводит:
```
Привет
```

Выводит:
```
Ты ввёл: Привет

```

**Проблема:** есть пустая строка! Потому что пользователь нажимает Enter, добавляется `\n`, и `print()` добавляет ещё один `\n`.

**Решение:**

```python
import sys

line = next(sys.stdin).rstrip('\n')  # удаляем \n в конце
print("Ты ввёл:", line)
```

Выводит:
```
Ты ввёл: Привет
```

### Способ 2: Цикл for (самый удобный!)

Читаем ВСЕ строки до конца ввода:

```python
import sys

for line in sys.stdin:
    print("Строка:", line.rstrip('\n'))

print('Завершение программы.')
```

Пользователь вводит:
```
Привет!
Как дела?
```

Нажимает **Ctrl+D** (Linux/Mac) или **Ctrl+Z+Enter** (Windows).

Программа выводит:
```
Строка: Привет!
Строка: Как дела?
Завершение программы.
```

Цикл `for` **автоматически обрабатывает конец ввода**!

### Способ 3: Метод read() — все данные сразу

```python
import sys

all_data = sys.stdin.read()  # читаем ВСЁ до конца
print(all_data)
```

Программа ждёт (Ctrl+D или Ctrl+Z+Enter), потом выводит всё.

### Важно: ВСЁ приходит как строки

Даже если пользователь вводит числа, списки или словари, **всё будет строками (str)**:

```python
import sys

for line in sys.stdin:
    print(type(line))  # всегда <class 'str'>
```

Пользователь вводит:
```
1
[1, 2, 3]
{'key': 'value'}
```

Выводит:
```
<class 'str'>
<class 'str'>
<class 'str'>
```

Если нужно число, приводи к типу:

```python
number = int(input("Число: "))  # или используй int(sys.stdin.readline())
```

### Ошибка: попытка писать в stdin

stdin — **только для ЧТЕНИЯ**! Попытка писать вызывает ошибку:

```python
import sys

sys.stdin.write('Привет!')  # ❌ ОШИБКА!
```

Выдаст:
```
io.UnsupportedOperation: not writable
```

## 📤 stdout — поток вывода

### Что это?

`sys.stdout` — это **файловый объект**, в который программа **ПИШЕТ** результаты. По умолчанию это экран.

### Метод write()

```python
import sys

sys.stdout.write('Привет!\n')
sys.stdout.write('Как дела?\n')
```

Выводит:
```
Привет!
Как дела!
```

**Важно:** `write()` НЕ добавляет `\n` автоматически! Ты добавляешь сам!

### Метод writelines()

Пишем несколько строк сразу:

```python
import sys

lines = ['Привет!\n', 'Как дела?\n', 'Что делаешь?\n']
sys.stdout.writelines(lines)
```

Выводит:
```
Привет!
Как дела!
Что делаешь!
```

### Обычно используешь print()

На самом деле `print()` просто пишет в `sys.stdout`:

```python
print("Привет")
# Под капотом это примерно:
# sys.stdout.write("Привет\n")
```

`print()` удобнее, потому что добавляет `\n` автоматически и может форматировать.

### Ошибка 1: только строки

```python
import sys

sys.stdout.write(123)  # ❌ ОШИБКА!
```

Выдаст:
```
TypeError: write() argument must be str, not int
```

Только строки!

### Ошибка 2: попытка читать из stdout

stdout — **только для ПИСЬМА**! Попытка читать вызывает ошибку:

```python
import sys

text = sys.stdout.read()  # ❌ ОШИБКА!
```

Выдаст:
```
io.UnsupportedOperation: not readable
```

## 🔴 stderr — поток ошибок

### Что это?

`sys.stderr` — это **файловый объект** для **ПИСЬМА ошибок и предупреждений**. Работает как stdout, но предназначен специально для ошибок.

### Писать в stderr

```python
import sys

sys.stderr.write('Произошла ошибка!\n')
sys.stderr.write('Ещё одна ошибка!\n')
```

Выводит:
```
Произошла ошибка!
Ещё одна ошибка!
```

В консоли текст часто выделяется **красным цветом**!

### Пример: функция с предупреждением

```python
import sys

def add(a, b):
    if a > 100:
        sys.stderr.write('⚠️ Предупреждение: большое значение a!\n')
    return a + b

result = add(150, 20)
print(result)
```

Выводит:
```
⚠️ Предупреждение: большое значение a!
170
```

Предупреждение выводится в stderr (красное), результат в stdout (чёрное).

### Важно: stderr НЕ участвует в исключениях

Когда ты пишешь в stderr, программа **продолжает работать**:

```python
import sys

sys.stderr.write('Ошибка!\n')
print('Я всё ещё здесь!')  # выполняется!
```

Выводит:
```
Ошибка!
Я всё ещё здесь!
```

stderr просто выводит информацию, не останавливает программу!

## 🔀 Разница между stdout и stderr

### Зачем разные потоки?

**stdout** — это нормальный вывод результатов:
```python
print('Результат: 42')
```

**stderr** — это ошибки и предупреждения:
```python
print('⚠️ Ошибка при обработке файла', file=sys.stderr)
```

### Операционная система позволяет их разделять!

В терминале (bash/cmd) можно перенаправлять их в разные файлы:

```bash
python program.py > results.txt 2> errors.txt
```

- `> results.txt` — перенаправляет stdout в results.txt
- `2> errors.txt` — перенаправляет stderr (номер 2) в errors.txt

**Результат:** результаты и ошибки в разных файлах! 📊

## ⚠️ Ошибки и ограничения

### Ошибка: забыл добавить \n

```python
import sys

sys.stdout.write('Текст 1')
sys.stdout.write('Текст 2')
```

Выводит без переноса:
```
Текст 1Текст 2
```

**Решение:** добавь `\n`:

```python
sys.stdout.write('Текст 1\n')
sys.stdout.write('Текст 2\n')
```

Выводит:
```
Текст 1
Текст 2
```

### StopIteration при раннем конце ввода

```python
import sys

print(next(sys.stdin))  # ожидает ввод
print(next(sys.stdin))  # если ввод закончился...
print(next(sys.stdin))  # ❌ StopIteration!
```

**Решение:** используй цикл `for`, он обрабатывает StopIteration:

```python
import sys

for line in sys.stdin:
    print(line.rstrip('\n'))
```

### Буферизация: данные не выводятся сразу

Когда пишешь в stdout/stderr, данные могут скапливаться в **буфере**:

```python
import time
import sys

for i in range(5, 0, -1):
    sys.stdout.write(str(i) + '...')
    time.sleep(1)
```

Ничего не выводится 5 секунд, потом всё сразу!

**Решение:** используй `flush()`:

```python
import time
import sys

for i in range(5, 0, -1):
    sys.stdout.write(str(i) + '...')
    sys.stdout.flush()  # очистить буфер!
    time.sleep(1)
```

Теперь выводит каждую секунду.

## ✅ Главные правила

✅ stdin — только для ЧТЕНИЯ  
✅ stdout и stderr — только для ПИСЬМА  
✅ write() НЕ добавляет `\n` (добавляй сам!)  
✅ ВСЕ данные из stdin — это строки (str)  
✅ В write() только строки!  
✅ stderr — для ошибок, stdout — для результатов  
✅ Используй цикл `for` для чтения stdin (удобнее)  
✅ Помни про буферизацию и flush()!
