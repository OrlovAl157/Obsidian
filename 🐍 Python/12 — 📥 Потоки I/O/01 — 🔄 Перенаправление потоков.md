## 🎯 Зачем нужно перенаправление

**Задача:** сохранять вывод программы в файлы вместо того, чтобы выводить на экран.

**Примеры:**
- Программа обрабатывает данные → результаты в `results.txt`
- Программа выполняет операции → ошибки в `errors.txt`
- Логирование: сохраняем всё в журнал для анализа

**Зачем отдельно?** Потому что результаты и ошибки часто обрабатывают по-разному:

```
stdout (результаты) → results.txt
stderr (ошибки)    → errors.txt
```

## 📝 Способ 1: параметр file в print()

### Самый простой способ

```python
import sys

with open('output.txt', 'w', encoding='utf-8') as file:
    print('Результат 1', file=file)
    print('Результат 2', file=file)

print('Это на экран')
```

Создаёт файл `output.txt`:
```
Результат 1
Результат 2
```

На экран выводит:
```
Это на экран
```

### Можно перенаправлять куда угодно

```python
import sys

file = open('data.txt', 'w')

print('В файл', file=file)
print('На экран')
print('В stderr', file=sys.stderr)

file.close()
```

### Ограничение: нужен контроль над кодом

```python
def foo():
    print('Это выводит функция')
    print('Но я не могу добавить file=file')

foo()  # выводит на экран, ничего не поделаешь!
```

**Решение:** использовать способ 2 или 3.

## 🔄 Способ 2: менять sys.stdout напрямую

### Идея: переменить сам stdout

```python
import sys

def foo():
    print('Привет!')
    print('Как дела?')
    print('Что делаешь?')

# Открываем файл
file = open('output.txt', 'w', encoding='utf-8')

# Сохраняем оригинальный stdout!
orig_stdout = sys.stdout

# МЕНЯЕМ stdout на файл!
sys.stdout = file

foo()  # внутри foo используется print()
       # но теперь он пишет в файл!

# ВОССТАНАВЛИВАЕМ оригинальный stdout!
sys.stdout = orig_stdout

file.close()

print('На экран!')
```

Файл `output.txt`:
```
Привет!
Как дела?
Что делаешь?
```

На экран:
```
На экран!
```

### Как это работает?

`sys.stdout` — это просто **переменная**, которая указывает, куда писать!

```python
sys.stdout = file  # ← теперь пишем в файл
```

Когда `print()` выводит что-то, он обращается к текущему `sys.stdout`:

```python
print("текст")
# Под капотом:
sys.stdout.write("текст\n")
```

Если ты сменил `sys.stdout`, то `print()` будет писать туда!

### ⚠️ ВАЖНО: восстанавливать ПЕРЕД закрытием!

```python
# ❌ НЕПРАВИЛЬНО
sys.stdout = file
print('В файл')
file.close()
print('На экран')  # ❌ ОШИБКА!
```

Выдаст:
```
ValueError: I/O operation on closed file
```

**Почему?** Потому что `sys.stdout` ещё указывает на закрытый файл!

**Правильно:**

```python
# ✅ ПРАВИЛЬНО
orig_stdout = sys.stdout
sys.stdout = file
print('В файл')
sys.stdout = orig_stdout  # восстановили ПЕРЕД закрытием!
file.close()
print('На экран')  # ✅ работает!
```

### Точно так же для stderr

```python
import sys

def add(a, b):
    if a > 100:
        print('⚠️ Предупреждение!', file=sys.stderr)
    return a + b

file = open('errors.txt', 'w', encoding='utf-8')
orig_stderr = sys.stderr
sys.stderr = file

result = add(150, 20)
print(result)  # это stdout, на экран

sys.stderr = orig_stderr
file.close()

print('Завершение')  # на экран
```

## 🎯 Способ 3: contextlib (умнее!)

### Проблема способа 2

Легко забыть восстановить `sys.stdout` или `sys.stderr`!

### Решение: контекстный менеджер

Python предоставляет модуль `contextlib` с готовыми менеджерами:

```python
from contextlib import redirect_stdout

def foo():
    print('Привет!')
    print('Как дела?')
    print('Что делаешь?')

with open('output.txt', 'w', encoding='utf-8') as file:
    with redirect_stdout(file):
        foo()  # всё пишется в файл

print('На экран!')  # автоматически восстановилось!
```

**Что происходит:**

1. `with redirect_stdout(file):` — "направь stdout в файл"
2. Весь код внутри блока `with` пишет в файл
3. При выходе из блока `with` — **автоматически восстановится** оригинальный stdout!

**Даже если произойдёт ошибка:**

```python
from contextlib import redirect_stdout

with open('output.txt', 'w') as file:
    with redirect_stdout(file):
        print('Текст')
        1 / 0  # ❌ ОШИБКА!
        # stdout всё равно восстановится!

print('На экран')  # ✅ работает, несмотря на ошибку выше!
```

### redirect_stderr

Работает точно так же, но для stderr:

```python
import sys
from contextlib import redirect_stderr

def add(a, b):
    if a > 100:
        print('⚠️ Предупреждение!', file=sys.stderr)
    return a + b

with open('errors.txt', 'w', encoding='utf-8') as file:
    with redirect_stderr(file):
        result = add(150, 20)
        print(result)  # это stdout, на экран

print('На экран')
```

### ✅ Почему это лучше?

- ✅ Не нужно помнить про восстановление
- ✅ Если произойдёт ошибка, stdout восстановится
- ✅ Чище и понятнее код
- ✅ Безопаснее!

**Используй этот способ! 👍**

## ⚠️ Буферизация и flush()

### Проблема: данные не выводятся сразу

Когда пишешь в stdout/stderr, данные скапливаются в **буфере**:

```python
import time
import sys

for i in range(5, 0, -1):
    sys.stdout.write(str(i) + '...')
    time.sleep(1)
```

**Что видишь на экране:** ничего 5 секунд, потом всё сразу: `5...4...3...2...1...`

Это потому что цикл завершается, и только потом буфер очищается.

### Решение 1: flush()

```python
import time
import sys

for i in range(5, 0, -1):
    sys.stdout.write(str(i) + '...')
    sys.stdout.flush()  # очистить буфер!
    time.sleep(1)
```

**Что видишь:** каждую секунду: `5...` → `4...` → `3...` → `2...` → `1...`

Метод `flush()` сбрасывает буфер и выводит текст сразу.

### Решение 2: добавить \n

Перенос строки **автоматически очищает буфер**:

```python
import time
import sys

for i in range(5, 0, -1):
    sys.stdout.write(str(i) + '...\n')  # с переносом!
    time.sleep(1)
```

**Что видишь:** каждую секунду с переносом:
```
5...
4...
3...
2...
1...
```

### Решение 3: параметр flush в print()

```python
import time

for i in range(5, 0, -1):
    print(str(i) + '...', end='', flush=True)
    time.sleep(1)
```

Параметр `flush=True` сбрасывает буфер после каждого `print()`.

## 🔍 Как работает буферизация

### По умолчанию: строчная буферизация

```python
import sys

sys.stdout.write('A')  # в буфере
sys.stdout.write('B')  # в буфере
sys.stdout.write('\n') # бах! вывод!

# На экране: AB
```

Буфер очищается при переносе строки `\n`.

### Методы для очистки буфера

**write() + flush():**
```python
sys.stdout.write('текст')
sys.stdout.flush()  # сброс буфера
```

**write() + \n:**
```python
sys.stdout.write('текст\n')  # автоматически очистит буфер
```

**print() с flush=True:**
```python
print('текст', flush=True)  # очистит буфер
```

## ✅ Главные правила

✅ Способ 1 (`file=`) — когда контролируешь код  
✅ Способ 3 (`contextlib`) — **ВСЕГДА** используй этот!  
✅ Способ 2 (`sys.stdout =`) — только если нужна гибкость (рискованно!)  
✅ Восстанавливай stdout/stderr ПЕРЕД закрытием файла!  
✅ Помни про буферизацию — используй `flush()` или `\n`  
✅ stdout и stderr работают независимо  
✅ contextlib автоматически восстанавливает потоки — это безопасно!
