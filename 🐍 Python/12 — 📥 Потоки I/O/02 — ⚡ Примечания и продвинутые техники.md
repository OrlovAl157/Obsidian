## 📌 Примечание 1: warnings для предупреждений

### Проблема: print в stderr неправильно

```python
import sys

def add(a, b):
    if a > 100:
        print('⚠️ Предупреждение!', file=sys.stderr)
    return a + b

print(add(150, 20))
```

Это работает, но это **не Python-way**!

### Решение: модуль warnings

```python
import warnings

def add(a, b):
    if a > 100:
        warnings.warn('Функция может работать нестабильно при a > 100!')
    return a + b

print(add(150, 20))
print('Завершение')
```

Выводит:
```
C:\main.py:6: UserWarning: Функция может работать нестабильно!
  warnings.warn('...')
150
Завершение
```

### Преимущества warnings

✅ Показывает **файл и номер строки** — сразу видно откуда ошибка  
✅ Можно **отключить все предупреждения** глобально  
✅ Можно **отфильтровать** конкретные предупреждения  
✅ **Стандартный способ** в Python для предупреждений  
✅ IDE понимает и подсвечивает  

### Типы предупреждений

```python
import warnings

warnings.warn('Это UserWarning по умолчанию')
warnings.warn('Это ошибка!', DeprecationWarning)  # устарелая функция
warnings.warn('Это синтаксис!', SyntaxWarning)    # проблема синтаксиса
```

### Отключить все предупреждения

```python
import warnings

warnings.simplefilter("ignore")  # игнорировать все

def add(a, b):
    if a > 100:
        warnings.warn('Предупреждение!')  # не выведется
    return a + b

print(add(150, 20))  # 150 (без предупреждения)
```

### Фильтровать конкретные типы

```python
import warnings

warnings.filterwarnings("ignore", category=DeprecationWarning)
# игнорируем только DeprecationWarning
```

## 📌 Примечание 2: сохранение traceback

### Проблема: как сохранить полный traceback в файл?

Когда программа падает, выводится красный текст ошибки. Нужно его сохранить.

### Способ 1: использовать redirect_stderr

```python
from contextlib import redirect_stderr

def foo():
    raise ValueError('Произошла ошибка')

with open('error.txt', 'w') as file:
    with redirect_stderr(file):
        foo()  # ❌ ОШИБКА, но stderr перенаправлен
```

**Проблема:** `redirect_stderr` откатит stderr обратно и пробросит исключение!

Файл `error.txt` останется пустым. ❌

### Способ 2: обработать исключение + traceback.print_exc()

```python
import traceback

def foo():
    raise ValueError('Произошла ошибка')

with open('error.txt', 'w') as file:
    try:
        foo()
    except ValueError:
        traceback.print_exc(file=file)  # пишет traceback в файл!
```

Файл `error.txt`:
```
Traceback (most recent call last):
  File "main.py", line 9, in <module>
    foo()
  File "main.py", line 5, in foo
    raise ValueError('Произошла ошибка')
ValueError: Произошла ошибка
```

**Это работает! ✅**

### Способ 3: с redirect_stderr и обработкой

```python
import traceback
from contextlib import redirect_stderr

def foo():
    raise ValueError('Произошла ошибка')

with open('error.txt', 'w') as file:
    with redirect_stderr(file):
        try:
            foo()
        except ValueError:
            traceback.print_exc()  # пишет в текущий stderr (файл)
```

**Тоже работает! ✅**

### Когда использовать

**Способ 2** — проще и понятнее, если ты **обрабатываешь конкретное исключение**.

**Способ 3** — если ты хочешь **перехватить stderr сторонней библиотеки**, которая выводит ошибки, которые не контролируешь.

### Получить traceback как строку

```python
import traceback
import io

def foo():
    raise ValueError('Ошибка')

try:
    foo()
except ValueError:
    # Получить traceback как строку
    error_text = traceback.format_exc()
    print("Ошибка:")
    print(error_text)
```

## 📌 Примечание 3: опасность open(0/1/2)

### Можно открыть потоки через номера дескрипторов

```python
stdin = open(0, encoding='utf-8')   # поток ввода
stdout = open(1, 'w', encoding='utf-8')  # поток вывода
stderr = open(2, 'w', encoding='utf-8')  # поток ошибок
```

### ⚠️ ОПАСНОСТЬ: случайно закроешь поток!

```python
with open(1, 'w'):
    pass

print('Это не выведется!')  # ❌ ОШИБКА!
```

Выдаст:
```
OSError: [Errno 9] Bad file descriptor
```

**Почему?** `with` закроет поток при выходе, и `print()` не сможет писать!

### Другой пример опасности

```python
stdout_backup = open(1, 'w')
stdout_backup.close()

print('Это не выведется!')  # ❌ ОШИБКА!
```

### ✅ Совет

**Избегай открывать потоки через `open(0/1/2)`!**

Используй вместо этого:
- `sys.stdin`, `sys.stdout`, `sys.stderr` — готовые объекты
- `contextlib.redirect_stdout()`, `redirect_stderr()` — безопасно
- `print(..., file=file)` — просто

## 📌 Примечание 4: разделение потоков в shell

### Зачем разделять stdout и stderr?

Потому что **оперативная система позволяет перенаправлять их отдельно!**

### В Linux/macOS/PowerShell

```bash
# Пишет результаты в results.txt
python program.py > results.txt

# Пишет ошибки в errors.txt (2 — это номер stderr!)
python program.py 2> errors.txt

# Пишет результаты И ошибки в один файл
python program.py > output.txt 2>&1

# stdout в файл, stderr на экран
python program.py > results.txt

# stderr в файл, stdout на экран
python program.py 2> errors.txt
```

### В cmd.exe (Windows)

```cmd
REM Результаты в файл
python program.py > results.txt

REM Ошибки в файл
python program.py 2> errors.txt

REM Оба в один файл
python program.py > output.txt 2>&1
```

### Пример программы

```python
import sys

print('Это результат')
print('Это ошибка', file=sys.stderr)
print('Ещё результат')
```

Команда:
```bash
python program.py > results.txt 2> errors.txt
```

Файл `results.txt`:
```
Это результат
Ещё результат
```

Файл `errors.txt`:
```
Это ошибка
```

## 🎯 Best Practices

### ✅ Что ДЕЛАЙ

1. **Используй `contextlib.redirect_stdout()` и `redirect_stderr()`** — безопасно и удобно
2. **Используй `warnings.warn()` для предупреждений** — стандартный способ
3. **Используй `print(..., file=sys.stderr)` для ошибок** — когда контролируешь код
4. **Разделяй результаты и ошибки** — в разные файлы
5. **Сохраняй `traceback.print_exc()`** — для логирования

### ❌ Что НЕ ДЕЛАЙ

1. **Не открывай потоки через `open(0/1/2)`** — слишком рискованно
2. **Не меняй `sys.stdout`/`sys.stderr` вручную** — используй contextlib
3. **Не забывай `flush()`** — когда данные не выводятся сразу
4. **Не используй `from module import *`** — может затереть stdout/stderr

---

## ✅ Главные правила

✅ Используй `warnings.warn()` для предупреждений  
✅ Используй `contextlib.redirect_stdout()` для безопасности  
✅ Используй `traceback.print_exc(file=file)` для сохранения ошибок  
✅ Помни про буферизацию — используй `flush()` или `\n`  
✅ stdout и stderr работают независимо — разделяй их!  
✅ Избегай `open(0/1/2)` — очень рискованно!  
✅ Разделяй результаты и ошибки в разные файлы
