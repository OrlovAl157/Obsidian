# 🔬 Памятка: memoryview и буферный протокол

`memoryview` — обёртка, которая даёт прямой доступ к памяти объекта без её копирования. Особенно полезна при работе с большими данными и частыми срезами.

### 💡 Ключевые особенности

- **Буферный протокол** — механизм прямого доступа к внутренней памяти объекта без копирования. Поддерживается `bytes`, `bytearray`, `array.array`, `numpy` и другими.
- **`memoryview`** — Python-обёртка над буферным протоколом. Не копирует данные — только хранит ссылку и параметры среза.
- **`view.obj`** — атрибут, указывающий на исходный объект. У среза `view[1:7]` он тот же, что и у родителя.
- **`view.tobytes()`** — только этот метод создаёт реальную копию данных в виде `bytes`.
- **Скорость срезов** — `memoryview` при срезах не создаёт новые объекты, что даёт кратное ускорение на больших данных.

### 💡 Полезные советы

- Используй `memoryview` когда нужно много срезов от большого `bytes`/`bytearray` — это в разы быстрее прямых срезов.
- `tobytes()` вызывай только когда реально нужна копия — иначе теряется весь смысл.
- `memoryview` поддерживает и более сложные структуры: `array.array`, многомерные `numpy`-массивы.

### 🚀 Быстрая справка

| **Операция** | **Синтаксис** | **Копирует данные?** |
|---|---|---|
| создать view | `memoryview(bee)` | нет |
| исходный объект | `view.obj` | нет |
| срез | `view[1:7]` | нет |
| получить bytes | `view.tobytes()` | да |
| проверить идентичность | `view.obj is bee` | — |

## 🧠 Буферный протокол

**Назначение:** низкоуровневый механизм, позволяющий объектам предоставлять прямой доступ к своей памяти без копирования.

| Область применения | Почему важно |
|---|---|
| научные вычисления | большие массивы данных |
| обработка изображений | пиксельные буферы |
| работа с файлами | потоковое чтение |
| сетевой ввод-вывод | передача без копирования |

**Объекты, поддерживающие буферный протокол:**

```python
# Встроенные
bytes
bytearray

# Из стандартной библиотеки
array.array
io.BytesIO

# Сторонние
numpy.ndarray   # многомерные массивы
```

**Важное:** обычные `list`, `str`, `dict` буферный протокол **не поддерживают**.

## 🔭 Тип memoryview

**Назначение:** Python-обёртка над буферным протоколом — даёт доступ к памяти объекта без копирования.

**Примеры:**

```python
bee = bytes('Beegeek 🐝', encoding='utf-8')
view = memoryview(bee)

print(view)         # <memory at 0x000002570...>
print(type(view))   # <class 'memoryview'>
```

```python
# view.obj — ссылка на исходный объект, не копия
print(view.obj)          # b'Beegeek \xf0\x9f\x90\x9d'
print(view.obj is bee)   # True -- та же область памяти
```

```python
# tobytes() — единственный метод, который создаёт реальную копию
print(view.tobytes())    # b'Beegeek \xf0\x9f\x90\x9d'
```

**Важное:** `memoryview` использует внутреннюю структуру Си с указателями — все вычисления адресов памяти без накладных расходов Python.

## ✂️ Срезы без копирования

**Назначение:** главная фишка `memoryview` — срез не создаёт новый объект, а только запоминает начало, конец и шаг.

**Примеры:**

```python
bee = bytes('Beegeek 🐝', encoding='utf-8')
view = memoryview(bee)
subview = view[1:7]

# obj у среза тот же, что и у родителя
print(view.obj)               # b'Beegeek \xf0\x9f\x90\x9d'
print(subview.obj)            # b'Beegeek \xf0\x9f\x90\x9d'
print(subview.obj is view.obj)  # True -- одна и та же память!

# tobytes() -- только здесь происходит реальное копирование
print(view.tobytes())         # b'Beegeek \xf0\x9f\x90\x9d'
print(subview.tobytes())      # b'eegeek'
```

```python
# Сравнение: bytes при срезе ВСЕГДА создаёт новый объект
bee = b'Beegeek'
sliced = bee[1:7]    # новый объект bytes в памяти

view = memoryview(bee)
sub = view[1:7]      # новый объект memoryview, но данные НЕ копируются
```

**Важное:** `bytes[1:7]` — новый объект в памяти. `memoryview[1:7]` — новый объект Python, но данные те же.

## ⚡ Производительность

**Назначение:** показать, насколько `memoryview` быстрее при частых срезах больших данных.

**Пример теста:**

```python
from time import perf_counter

def slice_time_test(data):
    start = perf_counter()
    while data:
        data = data[1:]
    return f'{perf_counter() - start:.5f}'

numbers = [100_000, 200_000, 300_000, 400_000, 500_000]

for n in numbers:
    data = b'x' * n
    view = memoryview(b'x' * n)
    print(f'{n = :_}: bytes — {slice_time_test(data)} с., '
          f'memoryview — {slice_time_test(view)} с.')
```

**Результаты:**

| Размер данных | `bytes` (сек.) | `memoryview` (сек.) | Ускорение |
|---|---|---|---|
| 100 000 | 0.10170 | 0.00547 | ~19× |
| 200 000 | 0.74611 | 0.00960 | ~78× |
| 300 000 | 0.84645 | 0.01577 | ~54× |
| 400 000 | 1.49646 | 0.01847 | ~81× |
| 500 000 | 2.94509 | 0.02367 | ~124× |

**Важное:** чем больше данных — тем заметнее разрыв. `bytes` при каждом срезе копирует всё оставшееся, `memoryview` только смещает указатель.

## ⚠️ Важные особенности

```python
# ❗ memoryview работает только с объектами, поддерживающими буферный протокол
memoryview([1, 2, 3])   # ❌ TypeError: list не поддерживает буферный протокол
memoryview('abc')        # ❌ TypeError: str не поддерживает буферный протокол

# ❗ tobytes() — только здесь создаётся копия
view = memoryview(b'Beegeek')
sub = view[1:4]          # ещё нет копии
data = sub.tobytes()     # b'eeg' — вот здесь копия

# ❗ obj всегда указывает на исходный объект
bee = bytearray(b'Beegeek')
view = memoryview(bee)
sub = view[1:4]
print(sub.obj is bee)    # True

# ❗ Для изменяемых bytearray можно изменять через memoryview
bee = bytearray(b'Beegeek')
view = memoryview(bee)
view[0] = 98             # меняем 'B' (66) → 'b' (98)
print(bee)               # bytearray(b'beegeek') -- исходный объект изменился!
```
