# ⚙️ Статический анализ с mypy

## 📌 Коротко

> `mypy` — статический анализатор, который проверяет, согласуются ли аннотации типов с тем, как используется код. Он помогает ловить ошибки до запуска программы.

## 🎯 Зачем нужен mypy

Аннотации типов становятся особенно полезными, когда их проверяет отдельный инструмент.

`mypy` помогает:

- находить неправильные аргументы функций;
- замечать несовпадение возвращаемого типа;
- видеть отсутствующие аннотации в важном коде;
- делать рефакторинг спокойнее.

## 🔍 Какие ошибки он ловит

### Неправильные аргументы

```python
def add(a: int, b: int) -> int:
    return a + b


print(add("1", "2") / 2)
```

`mypy` укажет, что в функцию передаются строки вместо `int`.

### Неверный возвращаемый тип

```python
def add(a: int, b: int) -> str:
    return a + b
```

Здесь аннотация обещает `str`, а фактически выражение возвращает `int`.

### Отсутствующие аннотации

По умолчанию:

```python
def add(a, b):
    return a + b
```

обычно не вызывает ошибку.

Но если нужен более строгий режим:

```bash
mypy main.py --disallow-untyped-defs
```

Тогда `mypy` пожалуется на функцию без аннотаций.

---

## 📊 Что проверяет mypy

| Ситуация | Пример | Что делает mypy |
| --- | --- | --- |
| Неверный тип аргумента | `add("1", "2")` | Сообщает об ошибке |
| Неверный return type | `-> str`, но возвращается `int` | Сообщает об ошибке |
| Нет аннотаций | `def add(a, b): ...` | По умолчанию молчит |
| Нет аннотаций в strict-режиме | `--disallow-untyped-defs` | Сообщает об ошибке |

## 🎯 Type narrowing

Статические анализаторы умеют сужать тип после проверок.

```python
def print_name(name: str | None) -> None:
    if name is not None:
        print(name.upper())
```

После `is not None` анализатор уже понимает, что внутри блока `name` — это `str`.

То же самое работает с isinstance():

```python
def process(value: int | str) -> str:
    if isinstance(value, str):
        return value.upper()
    return str(value)
```

## 🆚 mypy и pyright

Оба инструмента проверяют аннотации без запуска программы.

| Инструмент | Что полезно знать |
| --- | --- |
| `mypy` | классический и очень распространённый type checker |
| `pyright` | обычно быстрее, часто хорошо ощущается в IDE |

Базовые команды:

```bash
mypy .
pyright .
```

Полезные флаги `mypy`:

```bash
mypy --strict .
mypy --ignore-missing-imports .
```

### `# type: ignore`

Иногда точечно приходится подавить предупреждение:

```python
value = legacy_function()  # type: ignore
result = risky_call()      # type: ignore[assignment]
```

Использовать это стоит редко, иначе типизация быстро теряет смысл.

## ⚠️ Что важно понимать

### `mypy` не запускает программу

Он не проверяет поведение кода на реальных данных, а анализирует его статически.

### Он лучше ловит явные нарушения контракта

```python
def add(a: int, b: int) -> int:
    if not isinstance(a, int) or not isinstance(b, int):
        raise TypeError
    return a + b


print(add("1", "2") / 2)
```

Даже если внутри есть `isinstance`, `mypy` всё равно прав: вызов уже нарушает сигнатуру.

### Динамические данные сложнее анализировать

```python
def add(a: int, b: int) -> int:
    if not isinstance(a, int) or not isinstance(b, int):
        raise TypeError
    return a + b


def to_str(a, b):
    return str(a), str(b)


a, b = to_str(1, 2)
print(add(a, b) / 2)
```

Без аннотаций во вспомогательных функциях `mypy` может не увидеть проблему.

## 💡 Практические замечания

- `mypy` особенно полезен на функциях, модулях и публичных API.
- Чем лучше аннотирован код, тем полезнее вывод анализатора.
- Не нужно ждать полного покрытия аннотациями, можно усиливать проверку постепенно.
- Для современного Python вместе с `mypy` часто используют ещё и `pyright`.

---

## ⚠️ Частые ошибки

### ❌ Думать, что `mypy` заменяет тесты

`mypy` не проверяет бизнес-логику и не гарантирует корректное поведение программы.

### ❌ Надеяться, что `isinstance()` внутри функции отменяет плохой вызов

Плохой вызов всё равно остаётся плохим с точки зрения статического анализа.

### ❌ Оставлять всё без аннотаций и ждать пользы

Без типов статический анализ быстро теряет силу.
