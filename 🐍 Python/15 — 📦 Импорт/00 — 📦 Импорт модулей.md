## Справка по импорту

| Способ | Синтаксис | Создаёт | Обращение | Когда использовать |
|--------|-----------|---------|-----------|-------------------|
| Весь модуль | `import math` | одно имя `math` | `math.pi`, `math.sqrt()` | Обычно — самый безопасный способ |
| Конкретный объект | `from math import sqrt` | имя `sqrt` | `sqrt(4)` | Когда используешь один-два объекта часто |
| Все объекты | `from math import *` | много имён | `pi`, `sqrt()` | Редко — опасно, может затереть имена |

## 📦 Что такое модуль

**Модуль в Python** — это файл с расширением `.py`, который содержит код на языке Python. Внутри модуля может быть:

- Функции
- Классы
- Переменные и константы
- Вызовы функций
- Логика приложения

**Имя модуля** — это имя файла без расширения `.py`. Если у тебя есть файл `utils.py`, то имя модуля — `utils`.

**Зачем нужны модули?** Они позволяют структурировать код, делать его переиспользуемым и организованным. Вместо того чтобы писать весь код в одном файле, можно разделить его на несколько логических частей — отдельных модулей.

**Типы модулей:**
- **Встроенные** — поставляются с Python (`math`, `os`, `sys`, `json`)
- **Сторонние** — устанавливаются через pip (`requests`, `numpy`, `pandas`)
- **Собственные** — модули, которые ты создаёшь сам в своем проекте

Когда ты импортируешь модуль, Python **загружает его в память** и дает тебе доступ к его содержимому.

## 🔄 Три основных способа импорта

### 1️⃣ import module — весь модуль

```python
import math
```

**Что происходит:**
- Python загружает модуль `math`
- создаёт в текущей области видимости одно **имя** `math`
- это имя ссылается на **объект типа module** (не на экземпляр класса, а именно на объект типа `module`)
- внутри этого объекта живут все атрибуты: `pi`, `sqrt`, `sin`, `cos` и т.д.

**Как это выглядит в памяти:**

```
Твоя область видимости:
math ────────────┐
                 ▼
          <module 'math'>
            ├── pi (3.14159...)
            ├── sqrt (функция)
            ├── sin (функция)
            └── cos (функция)
```

**Как обращаться к объектам:**

```python
import math

print(math.pi)        # 3.141592653589793
print(math.sqrt(9))   # 3.0
print(math.sin(0))    # 0.0
```

**Почему это хорошо:**
- Сразу видно откуда пришло имя (через точку)
- Безопасно — не затирает случайно твои переменные
- В коде ясно, что это из внешнего модуля

---

### 2️⃣ from module import object — конкретный объект

```python
from math import sqrt
```

**Что происходит:**
- Python загружает модуль `math`
- ищет внутри него объект `sqrt` (это функция)
- берёт этот объект и создаёт в твоей области видимости **имя** `sqrt`
- это имя ссылается на **тот же объект функции**, что и `math.sqrt`

**Важно:** это не копирует объект, а создаёт ещё одну ссылку на один и тот же объект в памяти.

**Как это выглядит в памяти:**

```
<module 'math'>
  └── sqrt ─────────┐
                    │ (одна и та же функция)
Твоя область:      │
  sqrt ────────────┘
```

**Это эквивалентно:**

```python
import math
sqrt = math.sqrt
```

Обе строки создают одну и ту же ситуацию: имя `sqrt` ссылается на один и тот же объект-функцию в памяти.

**Как обращаться:**

```python
from math import sqrt

print(sqrt(9))    # 3.0
print(sqrt(16))   # 4.0
```

Можно вызвать функцию напрямую, без префикса `math.`.

**Когда использовать:**
- Если часто обращаешься к одному-двум объектам из модуля
- Например: `from datetime import datetime` — если часто используешь `datetime`

**Можно импортировать несколько объектов сразу:**

```python
from math import sqrt, sin, cos, pi

print(pi)          # 3.141592653589793
print(sqrt(4))     # 2.0
print(sin(0))      # 0.0
```

### 3️⃣ from module import * — все объекты

```python
from math import *
```

**Что происходит:**
- Python загружает модуль `math`
- берёт ВСЕ его атрибуты
- создаёт в твоей области видимости множество отдельных имён: `pi`, `sqrt`, `sin`, `cos` и т.д.

**Как это выглядит в памяти:**

```
<module 'math'>
  ├── pi ───────┐
  ├── sqrt ─────┼─→ Твоя область
  ├── sin ──────┤
  └── cos ──────┘
```

**Как обращаться:**

```python
from math import *

print(pi)           # 3.141592653589793
print(sqrt(9))      # 3.0
print(sin(0))       # 0.0
```

Можно обращаться к любому объекту напрямую.

**⚠️ Почему это опасно:** смотри раздел ниже.

## 🧠 Как импорт работает в памяти

Важно понимать, что при импорте **не создаются копии объектов**. Вместо этого **создаются новые имена (ссылки) на уже существующие объекты**.

**Пример:**

```python
import math
from math import sqrt

# Сейчас в памяти:
# - один модуль math (загружен один раз)
# - одна функция sqrt (существует в памяти один раз)
# - два имена, указывающие на эту функцию: math.sqrt и sqrt

print(math.sqrt is sqrt)  # True — это один и тот же объект!
```

**Что это означает:**
- Если модуль занимает 1 МБ памяти, он займёт 1 МБ, неважно, сколько раз ты его импортируешь
- Изменение объекта в одном месте отразится везде, где на него ссылаются
- Объект не копируется и не создаётся заново при каждом импорте

**Как Python "запоминает" загруженные модули:**

Python хранит все загруженные модули в словаре `sys.modules`:

```python
import sys
import math

print('math' in sys.modules)  # True

# При повторном импорте Python ищет модуль в кэше
import math  # не загружает снова, а берёт из кэша
```

При первом импорте модуль загружается, при повторном — берётся из кэша. Это повышает производительность.

## 🛣️ Абсолютный и относительный импорт

Когда ты импортируешь модули внутри пакета, у тебя есть две стратегии: абсолютный и относительный импорт.

### Абсолютный импорт — полный путь от корня проекта

**Синтаксис:** пишешь полный путь от корня проекта или пакета.

```python
from shop.utils import format_price
from shop.helpers import validate_email
```

**Как это работает:**
- Python смотрит на полный путь `shop.utils`
- идёт от корня проекта
- ищет пакет `shop` → модуль `utils` → функцию `format_price`

**Аналогия из реальной жизни:** полный адрес дома.

```
Россия → Челябинск → улица Пушкина → дом 5
shop   → utils      → format_price
```

**Пример структуры проекта:**

```
project/
├── main.py
└── shop/
    ├── __init__.py
    ├── cart.py
    └── utils.py
```

Из `main.py`:

```python
# Абсолютный импорт
from shop.utils import format_price
```

### Относительный импорт — путь от текущего файла

**Синтаксис:** используешь точки `.` и `..` для обозначения текущего пакета и уровней выше.

- **`.`** означает "текущий пакет"
- **`..`** означает "пакет на уровень выше"
- **`...`** означает "ещё на уровень выше" и т.д.

```python
from .utils import format_price      # в текущем пакете, модуль utils
from ..helpers import validate       # в пакете выше
```

**Как это работает:**
- Python смотрит, где находится текущий файл
- `.` говорит "остаюсь в текущем пакете"
- `..` говорит "поднимаюсь на один пакет выше"

**Аналогия:** "иди оттуда, где я сейчас".

**Пример структуры:**

```
project/
└── shop/
    ├── cart.py
    ├── utils.py
    └── services/
        ├── api.py
        └── payment.py
```

Из `shop/cart.py`:

```python
# Относительный импорт — в текущем пакете shop
from .utils import format_price
```

Из `shop/services/api.py`:

```python
# Относительный импорт — поднимаюсь в shop, потом в utils
from ..utils import format_price
```

**Абсолютный импорт из `shop/services/api.py`:**

```python
from shop.utils import format_price
```

### Сравнение

| Вид | Пример | Откуда считает путь | Плюсы | Минусы |
|-----|--------|-------------------|-------|--------|
| Абсолютный | `from shop.utils import func` | От корня проекта | Ясно, понятно читается, можно запускать отдельно | Длиннее писать |
| Относительный | `from .utils import func` | От текущего пакета | Короче писать, удобно внутри пакета | Может быть запутанным, если много уровней |

**Важный нюанс:**

Если ты запустишь файл с относительным импортом как скрипт напрямую:

```bash
python shop/cart.py
```

То можешь получить ошибку:

```
ImportError: attempted relative import with no known parent package
```

Потому что Python видит это как отдельный скрипт, а не как модуль пакета. **Абсолютный импорт более устойчив к такому**.

## ⚠️ Почему `import module` лучше

Хотя все три способа рабочие, `import math` считается лучшей практикой. Вот почему:

**1. Видно откуда пришло имя**

Когда ты видишь `math.sqrt()`, сразу ясно, что `sqrt` — это функция из модуля `math`. Когда видишь просто `sqrt()`, приходится гадать, откуда она пришла.

```python
# Ясно
print(math.pi)

# Непонятно откуда это
print(pi)
```

**2. Безопаснее — не затирает случайно имена**

```python
# Опасно
from math import *
pi = 5  # Ой, мы затёрли константу числа Пи из модуля math!

# Безопасно
import math
pi = 5  # Мы создали свою переменную, math.pi осталась нетронутой
```

**3. Меньше имён в области видимости**

```python
import math
# В твоей области видимости: одно имя math

from math import *
# В твоей области видимости: pi, sqrt, sin, cos, tan, log, exp, ...
```

Слишком много имён может привести к конфликтам и сделать код менее читаемым.

**PEP8 рекомендует:** используй абсолютный импорт конкретных объектов или импорт всего модуля, избегай `from module import *`.

## 🚫 Опасность `from module import *`

`from module import *` — это самый опасный способ импорта. Почему?

**Проблема 1: непредсказуемость**

Ты не знаешь, какие именно имена попадут в твой код. Модуль может экспортировать десятки объектов, и ты не в курсе.

```python
from math import *
# Что именно было импортировано? pi? sqrt? sin? Не ясно!
```

**Проблема 2: затирание имён**

Какой-то импортированный объект может случайно перекрыть твою переменную или встроенную функцию.

```python
# Встроенная функция sum()
result = sum([1, 2, 3, 4])  # 10

# Опасный импорт
from numpy import *  # может быть `sum` из numpy!

result = sum([1, 2, 3, 4])  # теперь это numpy.sum, а не встроенный sum!
```

**Проблема 3: сложнее читать и отлаживать код**

Когда видишь `sqrt()`, непонятно, откуда она. Нужно копаться в импортах и искать.

## 🔍 Как Python находит модули

Когда ты пишешь `import math`, Python ищет этот модуль в определённых местах. Порядок поиска задан в переменной `sys.path`:

```python
import sys
print(sys.path)
```

Выведет что-то вроде:

```
[
    'C:\\Projects\\myapp',           # текущий каталог проекта
    'C:\\Python312\\python312.zip',  # встроенные модули
    'C:\\Python312\\Lib',            # стандартная библиотека
    'C:\\Python312\\Lib\\site-packages',  # сторонние пакеты (pip)
]
```

**Порядок поиска модулей:**

1. **Текущий каталог проекта** — сначала ищет здесь
2. **Встроенные модули Python** — потом встроенные модули стандартной библиотеки
3. **Каталог site-packages** — потом сторонние модули, установленные через pip

**Опасность:** если ты создашь свой модуль с именем встроенного, например `math.py`, то Python найдёт твой модуль в проекте и будет использовать его вместо встроенного!

```python
# Если создать файл math.py в своем проекте
import math
print(math.sqrt(9))  # ошибка! твой math.py не содержит sqrt
```

**Совет:** никогда не давай модулям имена встроенных модулей.
