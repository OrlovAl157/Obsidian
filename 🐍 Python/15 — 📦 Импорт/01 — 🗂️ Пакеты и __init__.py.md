## Справка по пакетам

| Элемент | Назначение | Пример |
|---------|-----------|--------|
| Пакет (папка) | Группирует логически связанные модули | `shop/`, `api/`, `utils/` |
| `__init__.py` | Инициализация пакета; делает папку пакетом | `shop/__init__.py` |
| Пустой `__init__.py` | Пакет инициализируется, модули доступны через полный путь | `from shop.cart import add_item` |
| Заполненный `__init__.py` | Модули добавляются в пространство пакета; доступны напрямую | `from shop import add_item` (если добавлено в `__init__.py`) |
| Модуль в пакете | Файл `.py` внутри пакета | `shop/cart.py`, `shop/utils.py` |

## 📦 Что такое пакет

**Пакет в Python** — это папка, которая содержит Python модули (файлы `.py`) и **обязательно** файл `__init__.py`.

Именно наличие файла `__init__.py` отличает обычную папку от пакета Python. Без этого файла Python не будет рассматривать папку как пакет.

**Зачем нужны пакеты?**
- Структурировать большой проект
- Группировать логически связанные модули
- Избежать конфликтов имён (например, разные пакеты могут иметь модули с одинаковыми именами)
- Сделать код более организованным и читаемым

**Аналогия:** если модули — это книги, то пакеты — это полки в библиотеке. Книги на одной полке логически связаны между собой.

**Пример простого пакета:**

```
project/
└── shop/
    ├── __init__.py
    ├── cart.py
    ├── order.py
    └── utils.py
```

Здесь `shop` — это пакет. Он содержит три модуля: `cart`, `order` и `utils`.

**Пакеты могут быть вложенными:**

```
project/
└── shop/
    ├── __init__.py
    ├── cart.py
    └── payments/
        ├── __init__.py
        ├── stripe.py
        └── paypal.py
```

Здесь `shop` — это пакет, а `shop.payments` — это подпакет (пакет внутри пакета).

## 🗂️ Структура пакета

Минимальная структура пакета:

```
project/
└── mypackage/
    ├── __init__.py
    └── module.py
```

**Что нужно:**
- Папка с названием пакета (`mypackage`)
- Файл `__init__.py` внутри этой папки (может быть пустым)
- Один или несколько модулей (файлы `.py`)

**Как создать пакет:**

1. Создаёшь папку: `mkdir mypackage`
2. Создаёшь файл `__init__.py` в этой папке (может быть пустым)
3. Добавляешь модули (`.py` файлы) в папку
4. Теперь это пакет, и ты можешь его импортировать

**После создания пакета:**

```python
# Это работает!
import mypackage
from mypackage import module
from mypackage.module import function
```

**Без файла `__init__.py`:**

```python
# Это НЕ работает (папка не будет пакетом)
import mypackage  # ошибка!
```

Поэтому файл `__init__.py` — это критически важный маркер.

## 📝 Файл __init__.py — зачем нужен

**Файл `__init__.py` имеет два назначения:**

### 1️⃣ Маркер пакета

Наличие `__init__.py` говорит Python: "Это папка — пакет, а не просто директория".

Начиная с Python 3.3, этот файл технически не обязателен (есть "namespace packages"), но его все равно рекомендуется создавать для совместимости и ясности.

### 2️⃣ Инициализация пакета

Файл `__init__.py` **всегда выполняется** при импорте пакета. Это позволяет:
- Добавить модули в пространство имён пакета
- Запустить инициализационный код
- Определить, какие объекты экспортирует пакет (через `__all__`)

**Пример:**

```python
# shop/__init__.py

print("Пакет shop загружается!")
```

```python
# main.py

import shop  # выведет: "Пакет shop загружается!"
```

При каждом импорте пакета код из `__init__.py` выполняется один раз (Python кэширует модули).

## ⚪ Пустой __init__.py

**Пустой файл `__init__.py`** — это минималь, который просто говорит Python, что это пакет.

```python
# shop/__init__.py
# Совершенно пустой файл
```

**Структура проекта:**

```
project/
├── main.py
└── shop/
    ├── __init__.py
    ├── cart.py
    └── utils.py
```

**Содержимое модулей:**

```python
# shop/cart.py
def add_item(item):
    return f"Добавлен {item}"
```

```python
# shop/utils.py
def format_price(price):
    return f"{price} руб."
```

**Как импортировать с пустым `__init__.py`:**

```python
# main.py

# Нужно указывать полный путь до модуля
from shop.cart import add_item
from shop.utils import format_price

print(add_item("яблоко"))       # Добавлен яблоко
print(format_price(100))        # 100 руб.
```

**Важно:** с пустым `__init__.py` ты **не можешь** обращаться к модулям через пакет:

```python
# Это НЕ работает с пустым __init__.py!
from shop import cart  # ошибка: AttributeError
import shop
shop.cart.add_item()   # ошибка: AttributeError
```

Потому что пакет не "знает" о модулях — нужно явно добавить их.

## 🟢 Заполненный __init__.py

**Заполненный `__init__.py`** содержит импорты модулей, которые становятся частью пакета.

**Структура проекта (та же):**

```
project/
├── main.py
└── shop/
    ├── __init__.py
    ├── cart.py
    └── utils.py
```

**Заполненный `__init__.py`:**

```python
# shop/__init__.py

from . import cart
from . import utils
```

Теперь модули `cart` и `utils` добавлены в пространство пакета.

**Как импортировать с заполненным `__init__.py`:**

```python
# main.py

# Теперь можешь обращаться к модулям через пакет!
from shop import cart
from shop import utils

print(cart.add_item("яблоко"))   # Добавлен яблоко
print(utils.format_price(100))  # 100 руб.
```

**Или ещё лучше** — добавить функции прямо в пространство пакета:

```python
# shop/__init__.py

from .cart import add_item
from .utils import format_price
```

Теперь можешь импортировать функции напрямую:

```python
# main.py

from shop import add_item, format_price

print(add_item("яблоко"))    # Добавлен яблоко
print(format_price(100))     # 100 руб.
```

**Ещё вариант** — определить функции прямо в `__init__.py`:

```python
# shop/__init__.py

def add_item(item):
    return f"Добавлен {item}"

def format_price(price):
    return f"{price} руб."
```

Тогда в `main.py`:

```python
from shop import add_item, format_price

print(add_item("яблоко"))    # Добавлен яблоко
print(format_price(100))     # 100 руб.
```

## 🔌 Импорт модулей в __init__.py

При добавлении модулей в `__init__.py` используются **относительные импорты** с точками:

```python
# shop/__init__.py

from . import cart          # точка = текущий пакет shop
from . import utils
from .cart import add_item  # добавить функцию из модуля
```

**Или абсолютный импорт:**

```python
# shop/__init__.py

from shop import cart
from shop.cart import add_item
```

**Относительный импорт лучше**, потому что:
- Если пакет переименуешь, всё ещё будет работать
- Код более гибкий
- Соответствует PEP8

**Важный нюанс:** если в `__init__.py` импортировать модули БЕЗ префикса (неправильно):

```python
# shop/__init__.py

import cart       # НЕПРАВИЛЬНО! Это скажет Python искать cart как отдельный модуль
import utils      # НЕПРАВИЛЬНО!
```

То Python не найдёт эти модули, потому что ищет их не внутри пакета, а по всему пути `sys.path`.

**Правильно:**

```python
# shop/__init__.py

from . import cart   # правильно! Ищет cart в текущем пакете
from . import utils  # правильно!
```

## 💡 Пример реальной структуры

**Большой проект с вложенными пакетами:**

```
project/
├── main.py
├── config.py
└── app/
    ├── __init__.py
    ├── models.py
    ├── views.py
    └── services/
        ├── __init__.py
        ├── email.py
        └── payment.py
```

**Содержимое файлов:**

```python
# app/__init__.py

from .models import User, Product
from .views import render_home
```

```python
# app/services/__init__.py

from .email import send_email
from .payment import process_payment
```

```python
# main.py

# Импортировать из пакета
from app import User, Product, render_home

# Импортировать из подпакета
from app.services import send_email, process_payment

user = User("John")
send_email(user.email)
```

**Как Python это обрабатывает:**
1. Импортирует пакет `app` → выполняет `app/__init__.py`
2. `app/__init__.py` импортирует модули и добавляет объекты в пространство `app`
3. `main.py` получает доступ к `User`, `Product`, `render_home` как будто они определены прямо в пакете

**Это удобно:**
- Пакет выглядит как один модуль с хорошим API
- Пользователю не нужно знать внутреннюю структуру
- Если переместить модули внутри пакета, импорты не сломаются

## ⚠️ Важные нюансы

### ❗ Нюанс 1: `__init__.py` выполняется один раз

```python
# app/__init__.py

print("Инициализация app")
```

```python
# main.py

import app           # выведет: Инициализация app
import app           # не выведет ничего! (кэш)
from app import *    # не выведет ничего! (кэш)
```

Python запоминает загруженные модули в `sys.modules`, поэтому `__init__.py` выполняется только при первом импорте.

### ❗ Нюанс 2: Циклические импорты

```python
# app/__init__.py
from .models import User

# app/models.py
from app import something  # Ошибка! Циклический импорт
```

Избегай ситуаций, когда пакет импортирует модуль, а модуль импортирует пакет. Используй абсолютные импорты в модулях или импортируй конкретные функции.

### ❗ Нюанс 3: `__all__` контролирует `from package import *`

```python
# app/__init__.py

from .models import User, Product
from .views import render_home

__all__ = ['User', 'Product']  # только эти объекты экспортируются
```

```python
# main.py

from app import *  # импортирует User и Product, но НЕ render_home
```

Используй `__all__`, чтобы явно указать, какие объекты экспортирует пакет.

### ❗ Нюанс 4: Пакеты vs обычные модули

Не путай:
- **Пакет** — папка с `__init__.py`
- **Модуль** — файл `.py`
- **Подпакет** — пакет внутри пакета

Пакет может содержать модули и подпакеты.

### ❗ Нюанс 5: Относительный импорт только в пакетах

Относительные импорты (с точками) работают **только в модулях, которые являются частью пакета**:

```python
# app/models.py (модуль в пакете app)

from . import views  # работает!
```

```python
# script.py (отдельный скрипт)

from . import something  # ошибка! это не модуль пакета
```

Если запустишь модуль как скрипт и в нём используется относительный импорт, будет ошибка.

## 📚 Итог

| Ситуация | Что делать |
|----------|-----------|
| Создаёшь пакет | Создай папку и файл `__init__.py` |
| Хочешь, чтобы модули были доступны из пакета | Добавь импорты в `__init__.py` |
| Хочешь обращаться к функциям как `from package import func` | Импортируй функции в `__init__.py` |
| Хочешь контролировать, что экспортирует пакет | Используй `__all__` в `__init__.py` |
| Импортируешь модули внутри пакета | Используй относительный импорт (точки) |
