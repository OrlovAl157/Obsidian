# 📚 Справочник: Итераторы, генераторы и коллекции

## 📌 Коротко

> Справочник основных терминов: Collection, Sequence, Iterable, Iterator, Generator, Range. Определения, схемы связей и иерархия типов для понимания всей экосистемы.

---

## 🗂️ Основные понятия

### Collection (Коллекция)

```
Коллекция — объект, который хранит в себе набор значений 
одного или нескольких типов и позволяет обращаться к своим значениям
```

**Примеры коллекций:**
- `list` — упорядоченная, изменяемая
- `tuple` — упорядоченная, неизменяемая
- `set` — неупорядоченная, уникальные значения
- `dict` — пары ключ-значение
- `str` — последовательность символов

### Sequence (Последовательность)

```
Sequence — коллекция, элементы которой находятся в строгом порядке
и имеют числовые индексы (0, 1, 2, ...)
```

**Примеры последовательностей:**
- `list` → можно индексировать, изменять
- `tuple` → можно индексировать, неизменяемая
- `str` → можно индексировать, неизменяемая
- `range` → можно индексировать, специальный объект

**Не последовательности (неупорядоченные):**
- `set` → нет индексов
- `dict` → ключи не числовые индексы

### Iterable (Итерируемый объект)

```
Iterable — объект, на основе которого можно создать iterator.
То, по чему можно "ходить" циклом for.
```

**Как проверить:**
```python
from collections.abc import Iterable

numbers = [1, 2, 3]
print(isinstance(numbers, Iterable))  # True
```

**Все итерируемые объекты:**
- ✅ Все коллекции (list, tuple, set, dict, str)
- ✅ range object
- ✅ Все итераторы (сам итератор — итерируемый!)
- ✅ Файловые объекты
- ✅ Генераторы

**Правило:** Если у объекта есть метод `__iter__()`, он итерируемый.

### Iterator (Итератор)

```
Iterator — специальный объект, который выдает следующее значение
в результате применения функции next() и возбуждает StopIteration
когда значения закончились.
```

**Характеристики:**
- Создается из итерируемого через `iter()`
- Выдает элементы через `next()` по одному
- Одноразовый (истощается)
- Ленивый (вычисляет по требованию)

**Примеры итераторов:**
- Результат `iter(list)` → `list_iterator`
- Результат `map()` → `map` object
- Результат `filter()` → `filter` object
- Результат `zip()` → `zip` object
- Файловый объект при чтении

**Правило:** Если у объекта есть методы `__iter__()` И `__next__()`, он итератор.

### Generator (Генератор)

```
Generator — частный случай итератора, созданный с помощью функции 
с yield или генераторного выражения. Подкласс Iterator.
```

**Два типа генераторов:**
1. **Функция-генератор** — функция с `yield`
2. **Генераторное выражение** — `(x for x in ...)`

**Особенности генератора:**
- Это ТАКЖЕ итератор (имеет `__iter__()` и `__next__()`)
- Имеет дополнительные методы: `.send()`, `.throw()`, `.close()`
- Можно передавать данные В генератор через `.send()`

### Range object (Объект диапазона)

```
Range object — особый неизменяемый объект, который является чем-то 
средним между sequence и iterator.
```

**Свойства как Sequence:**
- ✅ Можно обращаться по индексам: `r[0]`, `r[5]`
- ✅ Можно брать срезы: `r[1:5]`
- ✅ Можно получить длину: `len(r)`
- ✅ Многоразовый (не истощается)

**Свойства как Iterator:**
- ✅ Значения вычисляются лениво
- ✅ Не хранит все значения в памяти
- ✅ Эффективен для больших диапазонов

```python
# range не хранит все числа!
r = range(10000000000000)  # Триллион чисел
print(len(r))  # 10000000000000 (мгновенно!)
print(r[0])    # 0 (мгновенно!)
print(r[-1])   # 9999999999999 (мгновенно!)

# Список хранит все числа!
l = list(range(1000000))  # Миллион чисел
# MemoryError для триллиона!
```

## 🔗 Схема связей

```
                              range object
                                   |
                                   | (особый объект между
                                   |  sequence и iterator)
                                   ↓
       set ──┐
             │
      dict ──┼──→ Collections ─────→ Iterable ←──────→ iterator
             │         ↑                ↑              (функция next())
       list ─┤         │                │
             │         │                │
      tuple ─┤    Sequence              │
             │         ↑                │
     string ─┘         │                │
                       │                │
                  (упорядоченные    (на основе любого
                   с индексами)      итерируемого объекта
                                     можно создать итератор)
```

## 📊 Иерархия типов

### Collections (Коллекции)

```
Collections (коллекции - хранят данные)
    │
    ├── Sequence (последовательности - упорядоченные с индексами)
    │      ├── list   (список - изменяемый)
    │      ├── tuple  (кортеж - неизменяемый)
    │      └── string (строка - неизменяемая)
    │
    ├── set    (множество - неупорядоченные, уникальные)
    └── dict   (словарь - пары ключ-значение)
```

### Iterable (Итерируемые объекты)

```
Iterable (по ним можно ходить циклом for)
    │
    ├── Collections (все коллекции итерируемые!)
    │    ├── Sequence (list, tuple, str)
    │    ├── set
    │    └── dict
    │
    ├── range object (специальный объект)
    │
    ├── iterator (сам итератор тоже итерируемый!)
    │    ├── list_iterator
    │    ├── map object
    │    ├── filter object
    │    ├── zip object
    │    ├── generator (ГЕНЕРАТОР!)
    │    └── custom iterator (классы с __iter__ и __next__)
    │
    └── file objects (файловые объекты)
```

### Iterator (Итераторы - выдают по одному)

```
Iterator (вызываем next() чтобы получить элемент)
    │
    ├── generator (функция с yield или выражение)
    │    ├── function generator (def с yield)
    │    └── generator expression ((x for x in ...))
    │
    ├── built-in iterators (встроенные)
    │    ├── map object
    │    ├── filter object
    │    ├── zip object
    │    ├── enumerate object
    │    └── reversed object
    │
    ├── custom iterator (класс с __iter__ и __next__)
    │
    └── file iterator (при чтении файла)
```

## 🔄 Поток работы с итераторами

```
Создание итератора:
    
    iterable_obj (list, tuple, string, generator, etc.)
          │
          │ iter()
          ↓
       iterator
          │
          │ next()
          ↓
    следующий элемент
          │
          │ next()
          ↓
    следующий элемент
          │
          │ next() (когда элементы закончились)
          ↓
    StopIteration (исключение)
```

**Пример:**
```python
# Шаг 1: Итерируемый объект
my_list = [1, 2, 3]

# Шаг 2: Создание итератора
my_iter = iter(my_list)

# Шаг 3: Получение элементов
print(next(my_iter))  # → 1
print(next(my_iter))  # → 2
print(next(my_iter))  # → 3
print(next(my_iter))  # → StopIteration!
```

## 📋 Ключевые отличия

### Collection vs Sequence

```
Collection          → любой набор значений (set, dict, list, tuple)
Sequence            → упорядоченный набор с индексами (list, tuple, str)

Пример:
set({1, 2, 3})      # Collection, но НЕ Sequence (нет индексов)
list([1, 2, 3])     # Collection И Sequence (есть индексы)
```

### Iterable vs Iterator

```
Iterable            → можно перебирать (for loop)
Iterator            → выдает элементы по одному (next())

Пример:
list                # Iterable, но НЕ Iterator (нельзя вызвать next)
iter(list)          # Iterator (можно вызвать next)
                    # Iterator также Iterable!
```

### Iterator vs Generator

```
Iterator            → протокол (интерфейс), класс с __iter__ и __next__
Generator           → конкретная реализация, функция с yield
                    → Generator ЯВЛЯЕТСЯ Iterator (подкласс)

Пример:
class MyIter:       # Iterator (написали класс)
    def __iter__(): ...
    def __next__(): ...

def my_gen():       # Generator (написали функцию с yield)
    yield 1
```

### Range vs List

```
range(1000000)      # Range object - ленивый, 48 байт
list(range(1000000)) # List - все в памяти, 8+ MB

range[0]            # ✅ O(1) - мгновенно
list[0]             # ✅ O(1) - мгновенно

range[999999]       # ✅ O(1) - мгновенно! (вычислит значение)
list[999999]        # ✅ O(1) - уже в памяти
```

## 🎯 Практические примеры

### Iterable vs Iterator

```python
# ITERABLE - нельзя вызвать next()
my_list = [1, 2, 3]
# next(my_list)  # ❌ TypeError

# ITERATOR - можно вызвать next()
my_iter = iter(my_list)
print(next(my_iter))  # ✅ 1
print(next(my_iter))  # ✅ 2
```

### Range особенность

```python
# Range - ленивый, многоразовый
r = range(5)
print(list(r))  # [0, 1, 2, 3, 4]
print(list(r))  # [0, 1, 2, 3, 4] (снова!)

# Iterator - одноразовый
it = iter([1, 2, 3])
print(list(it))  # [1, 2, 3]
print(list(it))  # [] (пуст!)
```

### Generator vs Iterator

```python
# Generator (функция с yield)
def my_gen():
    yield 1
    yield 2

gen = my_gen()
print(next(gen))  # 1

# Генератор ТАКЖЕ является итератором!
print(isinstance(gen, Iterator))  # True
```
