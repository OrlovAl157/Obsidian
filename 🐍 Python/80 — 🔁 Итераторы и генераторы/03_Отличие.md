# 🔄 Отличие: Итератор vs Генератор

## 📌 Коротко

> Генератор ЯВЛЯЕТСЯ подтипом итератора. Итератор — протокол (интерфейс), генератор — конкретная реализация. Главное отличие: наличие `yield` и дополнительные методы `.send()`, `.throw()`, `.close()`.

---

## 💡 Полезные советы

- **Генератор = частный случай итератора** — у генератора есть всё, что у итератора, + больше
- **Главное отличие — `yield`** — ключевое слово генератора, у итератора его нет
- **`.send()` уникален для генератора** — можно отправлять данные в генератор
- **Вопрос на собеседовании может быть некорректен** — это как сравнивать протокол и реализацию
- **Используйте генератор** — проще писать, понятнее читать
- **Используйте класс-итератор** — если нужна сложная логика или состояние

---

## 🎯 Две парадигмы

### Итератор — Протокол (интерфейс)

```
Iterator (протокол) = правила реализации
├── Методы:
│   ├── __iter__() — возвращает self
│   └── __next__() — возвращает следующий элемент или StopIteration
└── Примеры реализации:
    ├── Классы с __iter__ и __next__
    ├── Встроенные: map, filter, zip, enumerate
    └── Генераторы (тоже реализуют этот протокол)
```

### Генератор — Конкретная реализация

```
Generator (реализация) = функция с yield
├── Автоматически реализует Iterator протокол
├── Дополнительные методы:
│   ├── .send(value) — отправка данных
│   ├── .throw(exc) — отправка исключения
│   └── .close() — завершение
└── Типы:
    ├── Функция-генератор (def с yield)
    └── Генераторное выражение ((x for x in ...))
```

## 📊 Сравнительная таблица

| Характеристика | Итератор | Генератор |
|---|---|---|
| **Что это?** | Протокол (правила) | Конкретная реализация |
| **Создание** | Класс с `__iter__` и `__next__` | Функция с `yield` или `()` |
| **Код (примерно)** | ~15-20 строк | ~5-10 строк |
| **`yield`** | ❌ Нет (не нужен) | ✅ **Главное отличие!** |
| **Дополнительные методы** | Только `__iter__`, `__next__` | + `.send()`, `.throw()`, `.close()` |
| **Отношение** | Базовый протокол | Подкласс итератора |
| **Типы** | Любой класс с протоколом | Функция-генератор или выражение |
| **Сложность реализации** | Более сложно | Просто и элегантно |
| **Передача данных В объект** | ❌ Нельзя | ✅ Через `.send()` |
| **Состояние** | Хранится в атрибутах | Автоматически в локальных переменных |

## 💻 Практический пример: Счётчик

### Вариант 1: Итератор (класс)

```python
class Counter:
    def __init__(self, low, high):
        self.current = low
        self.high = high
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current > self.high:
            raise StopIteration
        else:
            self.current += 1
            return self.current - 1

# Использование
counter = Counter(1, 5)
print(list(counter))  # [1, 2, 3, 4, 5]

# Подсчёт строк: 17 строк!
```

### Вариант 2: Генератор (функция)

```python
def counter(low, high):
    while low <= high:
        yield low
        low += 1

# Использование
gen = counter(1, 5)
print(list(gen))  # [1, 2, 3, 4, 5]

# Подсчёт строк: 5 строк!
```

**Генератор в 3-4 раза компактнее!**

## 🔍 Проверка: Генератор ЯВЛЯЕТСЯ итератором

```python
import typing

# Создаём генератор
_gen = (i for i in range(5))

# Создаём итератор
_iter = iter(range(5))

print(_gen)   # <generator object <genexpr> at 0x...>
print(_iter)  # <range_iterator object at 0x...>

# Проверка наследования
print(issubclass(_gen.__class__, typing.Iterator))
# True ✅ (Генератор ЯВЛЯЕТСЯ Iterator)

print(issubclass(_iter.__class__, typing.Generator))
# False ❌ (Iterator НЕ ЯВЛЯЕТСЯ Generator)
```

**Вывод: Генератор — подкласс итератора, но не наоборот!**

## 🎁 Уникальная возможность: .send()

### Итератор не может получать данные

```python
class SimpleIterator:
    def __init__(self):
        self.value = 0
    
    def __iter__(self):
        return self
    
    def __next__(self):
        self.value += 1
        if self.value > 3:
            raise StopIteration
        return self.value

it = SimpleIterator()
print(next(it))  # 1
# it.send(100)  # ❌ AttributeError!
```

### Генератор может получать данные через .send()

```python
def echo():
    value = None
    while True:
        value = yield value  # Получаем значение!

gen = echo()
next(gen)  # Инициализация

print(gen.send("Hello"))   # Hello
print(gen.send("World"))   # World
print(gen.send("Python"))  # Python

# ✅ Это работает!
```

**Это мощная особенность, которой нет у обычных итераторов!**

## 💻 Практический пример: .send()

### Калькулятор с .send()

```python
def calculator():
    total = 0
    while True:
        x = yield total  # Получить значение и вернуть сумму
        if x is not None:
            total += x

calc = calculator()
next(calc)  # Инициализация

print(calc.send(5))    # Добавить 5, вернуть 5
print(calc.send(3))    # Добавить 3, вернуть 8
print(calc.send(2))    # Добавить 2, вернуть 10
print(calc.send(10))   # Добавить 10, вернуть 20
```

### Процессор обработки данных

```python
def uppercase_processor():
    while True:
        text = yield  # Получить текст
        if text:
            print(text.upper())

processor = uppercase_processor()
next(processor)  # Инициализация

processor.send("hello")   # HELLO
processor.send("world")   # WORLD
processor.close()
```

## ⚠️ Ошибка: Забыли инициализировать

```python
def gen():
    value = yield 1
    print(f"Получено: {value}")
    yield 2

g = gen()
# g.send("Hello")  # ❌ TypeError! (не инициализирован)

# Правильно:
g = gen()
next(g)  # или g.send(None) — инициализация
g.send("Hello")  # ✅ Теперь работает
```

## 📊 Таблица методов

| Метод | Итератор | Генератор | Описание |
|---|---|---|---|
| `__iter__()` | ✅ Да | ✅ Да | Возвращает self |
| `__next__()` | ✅ Да | ✅ Да | Следующий элемент |
| `.send(value)` | ❌ Нет | ✅ Да | Отправить данные |
| `.throw(exc)` | ❌ Нет | ✅ Да | Отправить исключение |
| `.close()` | ❌ Нет | ✅ Да | Завершить генератор |

## 🎯 Когда использовать что

### Используйте ГЕНЕРАТОР, если:

✅ **Простая логика** (1-2 циклов, 1-2 условия)
✅ **Последовательная выдача значений** (как 95% случаев)
✅ **Нужна компактность и читаемость**
✅ **Нужны методы .send()/.throw()/.close()**

### Используйте ИТЕРАТОР (класс), если:

✅ **Сложная логика** (много состояния, методов)
✅ **Нужны дополнительные атрибуты/методы**
✅ **Нужна сложная инициализация**
✅ **Реализуете другие протоколы (например, `__len__`)**

## 📝 Ответы на собеседовании

### Вариант 1 (простой)

> "Генератор — это удобный способ создания итераторов с помощью функции и `yield`. Генератор ЯВЛЯЕТСЯ итератором, но итератор НЕ ЯВЛЯЕТСЯ генератором."

### Вариант 2 (технический)

> "Генератор — подкласс итератора. Итератор — это протокол, генератор — конкретная реализация. Главное отличие: наличие `yield` в генераторе и дополнительные методы `.send()`, `.throw()`, `.close()`, которых нет у обычных итераторов."

### Вариант 3 (с примером)

> "Оба реализуют протокол Iterator (имеют `__iter__()` и `__next__()`). Но генератор создаётся функцией с `yield`, а итератор — классом. Генератор имеет уникальные методы `.send()` для отправки данных и `.throw()` для отправки исключений."

### Вариант 4 (философский)

> "Вопрос некорректен — это как сравнивать автомобиль и зубную щётку. Генератор расширяет возможности итератора. Думайте о них как об отце (итератор) и сыне (генератор), где сын имеет все качества отца + дополнительные."

## 💡 Практический пример: Выбор способа реализации

### Задача: Простая фильтрация

```python
# Вариант 1: Генератор (лучше!)
def positive_numbers(numbers):
    for num in numbers:
        if num > 0:
            yield num

# Вариант 2: Класс-итератор (лишнее!)
class PositiveIterator:
    def __init__(self, numbers):
        self.numbers = numbers
        self.index = 0
    
    def __iter__(self):
        return self
    
    def __next__(self):
        while self.index < len(self.numbers):
            num = self.numbers[self.index]
            self.index += 1
            if num > 0:
                return num
        raise StopIteration

# Генератор в 4 строки! Класс в 15 строк!
```

### Задача: Сложный обработчик состояния

```python
# Вариант 1: Генератор (не подходит)
def processor():
    # Сложная логика со множеством состояний
    # Много условий, переменных...
    pass  # Будет нечитаемо

# Вариант 2: Класс-итератор (подходит!)
class Processor:
    def __init__(self):
        self.state = 0
        self.buffer = []
        self.config = {}
    
    def __iter__(self):
        return self
    
    def __next__(self):
        # Сложная логика...
        pass

# Класс лучше для сложной логики!
```

## 🔗 Внешние примеры из сообщества

### Из реального опыта

> **"После года опыта я понял:** Генератор — подкласс итератора. По принципу Open/Closed генератор расширяет возможности итератора, как сын расширяет возможности отца."

### Из документации Python

> **"Python's generators provide a convenient way to implement the iterator protocol"** — официальная документация Python

### Из глоссария Python

> **Generator** — функция, которая возвращает generator iterator. Выглядит как обычная функция, но содержит `yield`.
>
> **Iterator** — объект, представляющий поток данных. Повторные вызовы `__next__()` возвращают элементы. Когда данных больше нет — возбуждается `StopIteration`.
