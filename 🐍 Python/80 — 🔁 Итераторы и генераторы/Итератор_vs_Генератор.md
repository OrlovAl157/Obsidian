# 🆚 Итератор vs Генератор

## 📌 Коротко

> Генератор — это подкласс итератора и удобный способ его реализации через `yield`; итератор — протокол, генератор — конкретная реализация этого протокола.

---

## 📖 Теория

### Вопрос с собеседования

**"Чем генератор отличается от итератора?"**

Этот вопрос часто задают на собеседованиях джунов, и даже опытные разработчики спорят о правильном ответе.

---

### Обсуждение в сообществе

#### Ответ 1 — Краткий

> **Если говорить кратко:**
>
> - **Итератор** — механизм поэлементного обхода данных
> - **Генератор** — позволяет отложено создавать результат при итерации
>
> **Генератор является подтипом итератора. Но не наоборот.**

**Доказательство кодом:**

```python
import typing

_gen  = (i for i in range(5))   # генератор
_iter = iter(range(5))          # итератор

print(_gen)   # <generator object <genexpr> at 0x...>
print(_iter)  # <range_iterator object at 0x...>

print(issubclass(_gen.__class__,  typing.Iterator))   # True ✅
print(issubclass(_iter.__class__, typing.Generator))  # False ❌
```

**Вывод:** Генератор — это подкласс итератора!

---

#### Ответ 2 — Контраргумент

> А разве итератор не позволяет отложено создавать результат? 🤔

**Что у них общего:**

```python
from itertools import tee

gen_  = (i for i in range(5))
iter_ = iter(range(5))

# 1. Генератор тоже истощается, как итератор
list(gen_)  # Использовали
list(gen_)  # [] Пуст!

# 2. У генератора есть те же методы
iter1, iter2 = tee(gen_)  # Работает без ошибок

# 3. iter() возвращает себя же
print(gen_ == iter(gen_))  # True

# 4. next() работает
print(next(gen_))  # Без ошибок
```

> **"Генератор — это удобный СПОСОБ создания итераторов"**
>
> Во всём остальном это те же итераторы.

---

#### Ответ 3 — Из теории

> **Итерируемый объект и итератор — это протоколы** (правила реализации).
>
> **Генератор — это не протокол, а конкретная реализация.**
>
> Специальный механизм в Python, который позволяет создать однократно перебираемую функцию, логика которой легко и красиво описывается.

---

#### Ответ 4 — Практический

> **Для себя я сформулировал так:**
>
> - **Итератор** — универсальный интерфейс доступа к коллекциям
> - **Генераторная функция** — конкретная реализация коллекции с описанным протоколом доступа
>
> **Примеры:**
> - `range()` — генератор именно целых чисел и никак иначе
> - `enumerate()` — генератор кортежей

---

#### Ответ 5 — От мидла на собеседовании

> **"Разница в наличии `yield` у генератора. Всё."**

---

#### Ответ 6 — Из документации Python

> **"Python's generators provide a convenient way to implement the iterator protocol"**
>
> **Краткий ответ:** Генераторы — это удобный способ реализации протокола итератора.

---

#### Ответ 7 — Из официального глоссария Python

> **Generator** — функция, которая возвращает generator iterator. Выглядит как обычная функция, но содержит `yield` для создания последовательности значений.
>
> **Iterator** — объект, представляющий поток данных. Повторные вызовы `__next__()` возвращают элементы. Когда данных больше нет — возбуждается `StopIteration`.
>
> **Важно из PEP 342:**
>
> У генератора есть возможность **получать входные данные**, замораживая своё состояние. Это позволяло создавать сопрограммы (корутины) ДО появления `async/await`.
>
> **Дополнительные методы генератора:**
> - `.send()` — отправка данных В генератор
> - `.throw()` — отправка исключений
> - `.close()` — завершение работы
>
> Возможность отправки данных в генератор — существенное отличие!

---

#### Ответ 8 — Финальный, с опытом

> После года практики:
>
> **Итератор** — объект, который полностью удовлетворяет протоколу Iterator.
>
> **Генератор** — подкласс/наследник итератора. По принципу Open/Closed генератор **расширяет** возможности итератора.
>
> **Их сравнение — как сравнение отца и сына.**
>
> В былые времена у всех был припасён "свой собственный класс/метакласс для создания итераторов". Из-за этого и ввели генераторы.
>
> **Преимущества генераторов:**
> 1. ✅ Удобный синтаксис создания
> 2. ✅ Возможность управления из коробки (`.send()`, `.throw()`, `.close()`)
> 3. ✅ Почти полная обратная совместимость с итераторами (принцип Liskov)
>
> **Про корутины:**
> До появления `async/await` корутины создавали через проинициированные генераторы.

---

### Итоговая таблица различий

| Характеристика | Итератор | Генератор |
|---|---|---|
| **Что это?** | Протокол (интерфейс) | Конкретная реализация |
| **Создание** | Класс с `__iter__` и `__next__` | Функция с `yield` |
| **Сложность кода** | ~15-20 строк | ~5-7 строк |
| **yield** | ❌ Нет | ✅ **Главное отличие!** |
| **Дополнительные методы** | Только `__iter__`, `__next__` | + `.send()`, `.throw()`, `.close()` |
| **Передача данных В объект** | ❌ Нельзя | ✅ Через `.send()` |
| **Отношение** | Базовый протокол | Подкласс итератора |
| **Корутины** | ❌ Не поддерживает | ✅ Поддерживал до async/await |

## 💻 Примеры кода

### Итератор — много кода

```python
class Counter:
    def __init__(self, low, high):
        self.current = low
        self.high = high

    def __iter__(self):
        return self

    def __next__(self):
        if self.current > self.high:
            raise StopIteration
        self.current += 1
        return self.current - 1

counter = Counter(1, 5)
print(list(counter))  # [1, 2, 3, 4, 5]
# 17 строк кода!
```

### Генератор — компактно

```python
def counter(low, high):
    while low <= high:
        yield low
        low += 1

gen = counter(1, 5)
print(list(gen))  # [1, 2, 3, 4, 5]
# 5 строк кода!
```

### Уникальная возможность генератора — .send()

```python
def echo():
    value = None
    while True:
        # Получаем данные извне!
        value = yield value

gen = echo()
next(gen)  # Инициализация

print(gen.send("Hello"))  # Hello
print(gen.send("World"))  # World

# Итератор так не умеет!
```

### Краткие ответы на собеседовании

**Вариант 1 — простой:**
> "Генератор — это удобный способ создания итераторов с помощью функции и `yield`"

**Вариант 2 — технический:**
> "Генератор — подкласс итератора. Итератор — это протокол, генератор — конкретная реализация этого протокола"

**Вариант 3 — с деталями:**
> "Главное отличие — наличие `yield`. Генератор имеет дополнительные методы `.send()`, `.throw()`, `.close()` для управления выполнением, чего нет у обычных итераторов"

**Вариант 4 — философский:**
> "Вопрос некорректен — это как сравнивать автомобиль и зубную щётку. Генератор расширяет итератор, как сын расширяет возможности отца"
