## Справка по типам

| Группа | Типы | Когда использовать |
| --- | --- | --- |
| Строки фиксированной длины | `CHAR(n)` | Коды одинаковой длины: `ISO`, `UUID` в фиксированном виде, статусы. |
| Строки переменной длины | `VARCHAR(n)` | Имена, email, заголовки, короткий текст. |
| Большой текст | `TEXT`, `MEDIUMTEXT`, `LONGTEXT` | Описания, статьи, длинные заметки. |
| Целые числа | `TINYINT`, `SMALLINT`, `MEDIUMINT`, `INT`, `BIGINT` | Количество, идентификаторы, счётчики. |
| Дробные приближённые | `FLOAT`, `DOUBLE` | Научные расчёты, где допустима погрешность. |
| Дробные точные | `DECIMAL(M, D)`, `NUMERIC(M, D)` | Деньги, цены, проценты, точные значения. |
| Дата и время | `DATE`, `TIME`, `DATETIME`, `TIMESTAMP`, `YEAR` | Даты, время, события, временные метки. |

## 🔤 Строковые типы данных

**Назначение:** хранить текст, коды, названия и длинные описания.

| Тип | Размер / лимит | Что важно помнить |
| --- | --- | --- |
| `CHAR(n)` | до 255 символов | Фиксированная длина. Подходит для значений одинакового размера. |
| `VARCHAR(n)` | до 65535 байт строки | Переменная длина. Чаще всего используется для короткого текста. |
| `TINYTEXT` | до 255 байт | Очень короткий текст. |
| `TEXT` | до 65535 байт | Обычный длинный текст. |
| `MEDIUMTEXT` | до 16777215 байт | Очень длинный текст. |
| `LONGTEXT` | до 4294967295 байт | Максимально длинный текст. |

**Примеры:**

```sql
CREATE TABLE Users (
    code CHAR(6),
    email VARCHAR(255),
    bio TEXT
);
```

**Важное:** лимиты для строковых типов в MySQL зависят от байтов, а не только от количества символов. В `utf8mb4` один символ может занимать до 4 байт.

## 🔢 Целые числа

**Назначение:** хранить целые значения: идентификаторы, количество, возраст, счётчики.

| Тип | Размер | Диапазон со знаком | Диапазон `UNSIGNED` |
| --- | --- | --- | --- |
| `TINYINT` | 1 байт | -128 ... 127 | 0 ... 255 |
| `SMALLINT` | 2 байта | -32768 ... 32767 | 0 ... 65535 |
| `MEDIUMINT` | 3 байта | -8388608 ... 8388607 | 0 ... 16777215 |
| `INT` | 4 байта | -2147483648 ... 2147483647 | 0 ... 4294967295 |
| `BIGINT` | 8 байт | -2^63 ... 2^63 - 1 | 0 ... 2^64 - 1 |

**Примеры:**

```sql
CREATE TABLE Orders (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    quantity INT UNSIGNED,
    rating TINYINT
);
```

**Важное:** если значение не может быть отрицательным, можно использовать `UNSIGNED`. Для первичных ключей часто берут `INT UNSIGNED` или `BIGINT UNSIGNED`.

## 🧮 Числа с плавающей точкой

**Назначение:** хранить приближённые дробные значения.

| Тип | Размер | Что важно помнить |
| --- | --- | --- |
| `FLOAT` | 4 байта | Меньше места, ниже точность. |
| `DOUBLE` | 8 байт | Больше точность, но всё равно возможна погрешность. |

**Примеры:**

```sql
CREATE TABLE Measurements (
    temperature FLOAT,
    distance DOUBLE
);
```

**Важное:** `FLOAT` и `DOUBLE` не подходят для денег, потому что могут давать погрешность округления.

## 💰 Числа с фиксированной точкой

**Назначение:** хранить точные дробные значения.

| Тип | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `DECIMAL(M, D)` | `DECIMAL(10, 2)` | `M` — всего цифр, `D` — цифр после точки. |
| `NUMERIC(M, D)` | `NUMERIC(10, 2)` | В MySQL аналогичен `DECIMAL`. |

**Примеры:**

```sql
CREATE TABLE Products (
    price DECIMAL(10, 2),
    discount NUMERIC(5, 2)
);
```

**Важное:** для цены `99999999.99` подойдёт `DECIMAL(10, 2)`: всего 10 цифр, из них 2 после точки.

## 📅 Дата и время

**Назначение:** хранить даты, время и временные метки событий.

| Тип | Размер | Диапазон / смысл |
| --- | --- | --- |
| `DATE` | 3 байта | Дата: `1000-01-01` ... `9999-12-31`. |
| `TIME` | 3 байта | Время или длительность. Для времени суток обычно `00:00:00` ... `23:59:59`. |
| `DATETIME` | 8 байт | Дата + время: `1000-01-01 00:00:00` ... `9999-12-31 23:59:59`. |
| `TIMESTAMP` | 4 байта | Временная метка, часто для `created_at` / `updated_at`. |
| `YEAR` | 1 байт | Год: обычно `1901` ... `2155`. |

**Примеры:**

```sql
CREATE TABLE Events (
    event_date DATE,
    starts_at DATETIME,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Важное:** `DATETIME` хранит дату и время как значение, а `TIMESTAMP` часто используют для временных меток с учётом настроек часового пояса.

## 🧱 Пример выбора типов

```sql
CREATE TABLE Books (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    price DECIMAL(10, 2),
    amount INT UNSIGNED,
    published_at DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Важное:** хороший выбор типа — это баланс между точностью, размером данных и тем, как поле будет использоваться в запросах.
