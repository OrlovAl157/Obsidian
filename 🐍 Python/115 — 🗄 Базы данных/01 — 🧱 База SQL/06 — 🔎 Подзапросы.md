## Справка по типам подзапросов

| Тип | Где используется | Возвращает | Пример |
| --- | --- | --- | --- |
| Скалярный | SELECT, WHERE | 1 значение | `(SELECT MAX(salary) FROM Employees)` |
| Табличный | FROM | Несколько строк/столбцов | `(SELECT * FROM Orders WHERE status = 'done')` |
| Список значений | IN, NOT IN | Несколько значений 1 столбца | `(SELECT user_id FROM Premium)` |
| EXISTS / NOT EXISTS | WHERE | TRUE/FALSE | `EXISTS (SELECT 1 FROM Orders WHERE user_id = u.id)` |

## 📍 Подзапросы в WHERE

**Назначение:** фильтровать строки, используя результат другого запроса.

| Конструкция | Что делает |
| --- | --- |
| `WHERE col = (SELECT ...)` | Скалярный подзапрос (1 значение) |
| `WHERE col IN (SELECT ...)` | Несколько значений |
| `WHERE col > (SELECT ...)` | Сравнение с одним значением |

**Примеры:**

```sql
-- Найти всех пользователей старше среднего возраста
SELECT name, age
FROM Users
WHERE age > (SELECT AVG(age) FROM Users);

-- Результат:
-- name    | age
-- --------+-----
-- Bob     | 34
-- Frank   | 45
-- Jack    | 38
```

```sql
-- Найти пользователей, у которых есть заказы
SELECT name
FROM Users
WHERE user_id IN (SELECT DISTINCT user_id FROM Orders);

-- Результат: только пользователи с заказами
-- name
-- ------
-- Alice
-- Bob
-- Diana
```

```sql
-- Найти продукты дороже минимальной цены в категории Electronics
SELECT product_name, price
FROM Products
WHERE price > (SELECT AVG(price) FROM Products WHERE category = 'Electronics');

-- Результат:
-- product_name | price
-- -------------+-------
-- Gaming Laptop | 1499.99
-- Smartphone   | 999.99
```

**Важное:** 
- Подзапрос в WHERE выполняется для каждой строки
- Должен возвращать одно значение (скалярный)
- Производительность может быть ниже чем JOIN

## 🔄 Подзапросы в FROM (табличные)

**Назначение:** использовать результат запроса как таблицу-источник.

| Конструкция | Что делает |
| --- | --- |
| `FROM (SELECT ...) AS alias` | Подзапрос возвращает таблицу |

**Примеры:**

```sql
-- Найти города с количеством пользователей > 3
SELECT city, user_count
FROM (
    SELECT city, COUNT(*) as user_count
    FROM Users
    GROUP BY city
) AS city_stats
WHERE user_count > 3;

-- Результат:
-- city     | user_count
-- ---------+------------
-- New York | 4
-- Tokyo    | 5
```

```sql
-- Сложная аналитика: расход по категориям
SELECT 
    category,
    total_revenue,
    ROUND(total_revenue * 100 / (SELECT SUM(total_revenue) FROM revenue_by_category), 2) as percentage
FROM (
    SELECT 
        p.category,
        SUM(oi.quantity * oi.price) as total_revenue
    FROM Products p
    JOIN OrderItems oi ON p.id = oi.product_id
    GROUP BY p.category
) AS revenue_by_category;

-- Результат:
-- category    | total_revenue | percentage
-- ------------+---------------+------------
-- Electronics | 45000.00      | 45.23
-- Books       | 30000.00      | 30.15
-- Clothing    | 24500.00      | 24.62
```

**Важное:** 
- Подзапрос в FROM ВСЕГДА должен иметь алиас (AS)
- Может быть медленнее чем JOIN на больших таблицах
- Полезен для упрощения логики сложных запросов

## 📊 Подзапросы в SELECT (скалярные)

**Назначение:** вывести в результате вычисленное значение из другой таблицы.

| Конструкция | Что делает |
| --- | --- |
| `SELECT col, (SELECT ...) as alias` | Скалярный подзапрос для каждой строки |

**Примеры:**

```sql
-- Для каждого пользователя показать количество его заказов
SELECT 
    name,
    (SELECT COUNT(*) FROM Orders WHERE user_id = u.id) as order_count
FROM Users u;

-- Результат:
-- name    | order_count
-- --------+-------------
-- Alice   | 3
-- Bob     | 1
-- Charlie | 0
-- Diana   | 2
```

```sql
-- Для каждого продукта показать его цену и среднюю цену категории
SELECT 
    product_name,
    price,
    (SELECT AVG(price) FROM Products p2 WHERE p2.category = p.category) as category_avg_price
FROM Products p;

-- Результат:
-- product_name | price  | category_avg_price
-- -------------+--------+--------------------
-- Laptop       | 999.99 | 899.50
-- Mouse        | 29.99  | 45.20
-- Programming Book | 49.99 | 34.50
```

**Важное:** 
- Подзапрос в SELECT выполняется для КАЖДОЙ строки результата
- Может быть очень медленным на больших таблицах
- Используй JOIN или подзапрос в FROM если медленно

## ❓ EXISTS и NOT EXISTS

**Назначение:** проверить, существуют ли строки в другой таблице (TRUE/FALSE).

| Конструкция | Что делает |
| --- | --- |
| `EXISTS (SELECT ...)` | TRUE если подзапрос вернул строки |
| `NOT EXISTS (SELECT ...)` | TRUE если подзапрос вернул 0 строк |

**Примеры:**

```sql
-- Найти пользователей с заказами (EXISTS вместо IN)
SELECT name
FROM Users u
WHERE EXISTS (
    SELECT 1 FROM Orders o WHERE o.user_id = u.id
);

-- Результат: только пользователи с заказами
-- name
-- ------
-- Alice
-- Bob
-- Diana
```

```sql
-- Найти пользователей БЕЗ заказов
SELECT name
FROM Users u
WHERE NOT EXISTS (
    SELECT 1 FROM Orders o WHERE o.user_id = u.id
);

-- Результат: только пользователи без заказов
-- name
-- -------
-- Charlie
-- Eve
```

```sql
-- Найти заказы, где есть товары в наличии
SELECT order_id, total_amount
FROM Orders o
WHERE EXISTS (
    SELECT 1 FROM OrderItems oi 
    JOIN Products p ON oi.product_id = p.id
    WHERE oi.order_id = o.id AND p.quantity > 0
);
```

**Важное:** 
- EXISTS часто быстрее IN на больших таблицах
- Не важно что возвращает подзапрос (SELECT 1 оптимально)
- Подзапрос останавливается как только найдена первая строка

## 🔗 Коррелированные подзапросы

**Назначение:** подзапрос ссылается на столбцы внешнего запроса.

| Конструкция | Что делает |
| --- | --- |
| Коррелированный подзапрос | Выполняется для каждой строки внешнего запроса |

**Примеры:**

```sql
-- Для каждого сотрудника показать его зарплату и среднюю по отделу
SELECT 
    name,
    salary,
    (SELECT AVG(salary) FROM Employees e2 WHERE e2.department = e.department) as dept_avg
FROM Employees e;

-- Результат:
-- name  | salary | dept_avg
-- ------+--------+----------
-- Alice | 70000  | 65000
-- Bob   | 60000  | 65000
-- Carol | 80000  | 65000
```

```sql
-- Найти сотрудников с зарплатой выше средней по своему отделу
SELECT 
    name,
    salary,
    department
FROM Employees e
WHERE salary > (
    SELECT AVG(salary) 
    FROM Employees e2 
    WHERE e2.department = e.department
);

-- Результат:
-- name  | salary | department
-- ------+--------+------------
-- Carol | 80000  | HR
-- Frank | 95000  | IT
```

**Важное:** 
- Коррелированные подзапросы медленнее (выполняются для каждой строки)
- Часто можно заменить на JOIN
- Используй если логика очень сложная

## 📌 IN, ANY, ALL операторы

**Назначение:** сравнить значение со списком или результатом подзапроса.

| Оператор | Что делает | Пример |
| --- | --- | --- |
| `IN (SELECT ...)` | Проверяет вхождение в список | `user_id IN (1, 2, 3)` |
| `NOT IN (SELECT ...)` | Исключает значения из списка | `status NOT IN ('cancelled', 'pending')` |
| `ANY (SELECT ...)` | Сравнивает с ЛЮБЫМ значением | `price > ANY (SELECT price FROM Products)` |
| `ALL (SELECT ...)` | Сравнивает со ВСЕМИ значениями | `price > ALL (SELECT price FROM Products)` |

**Примеры:**

```sql
-- IN: найти пользователей из списка городов
SELECT name
FROM Users
WHERE city IN (
    SELECT city FROM Cities WHERE population > 1000000
);

-- Эквивалент:
SELECT name
FROM Users
WHERE city IN ('New York', 'London', 'Tokyo');
```

```sql
-- ANY: продукты дороже любого книги
SELECT product_name, price
FROM Products
WHERE price > ANY (
    SELECT price FROM Products WHERE category = 'Books'
);

-- Результат: продукты дороже МИНИМАЛЬНОЙ цены книг
```

```sql
-- ALL: продукты дороже всех книг
SELECT product_name, price
FROM Products
WHERE price > ALL (
    SELECT price FROM Products WHERE category = 'Books'
);

-- Результат: продукты дороже МАКСИМАЛЬНОЙ цены книг
```

**Важное:** 
- `IN` — наиболее читаемый и используемый
- `ANY` эквивалентен `IN` с `=`
- `ALL` часто можно заменить на `> MAX(...)`
- `NOT IN` может быть медленным с NULL значениями (используй `NOT EXISTS`)

## ⚡ Быстрые примеры

```sql
-- Пользователи старше среднего возраста
SELECT * FROM Users
WHERE age > (SELECT AVG(age) FROM Users);

-- Пользователи с заказами (IN)
SELECT * FROM Users
WHERE id IN (SELECT DISTINCT user_id FROM Orders);

-- Пользователи с заказами (EXISTS, быстрее)
SELECT u.* FROM Users u
WHERE EXISTS (SELECT 1 FROM Orders WHERE user_id = u.id);

-- Пользователи БЕЗ заказов
SELECT u.* FROM Users u
WHERE NOT EXISTS (SELECT 1 FROM Orders WHERE user_id = u.id);

-- Статистика по городам
SELECT city, user_count FROM (
    SELECT city, COUNT(*) as user_count
    FROM Users
    GROUP BY city
) stats
WHERE user_count > 5;

-- Для каждого заказа его сумму и среднюю сумму по пользователю
SELECT 
    order_id,
    total,
    (SELECT AVG(total) FROM Orders WHERE user_id = o.user_id) as user_avg
FROM Orders o;
```

## 💡 Практические замечания

- **JOIN vs подзапрос** — обычно JOIN быстрее на больших таблицах
- **EXISTS vs IN** — EXISTS часто быстрее, особенно если много совпадений
- **Производительность** — подзапросы в SELECT медленные на больших таблицах
- **Читаемость** — подзапросы более понятны чем сложный JOIN
- **NULL в IN** — `NOT IN` с NULL может быть опасен (используй NOT EXISTS)
- **Коррелированные подзапросы** — выполняются для каждой строки, медленнее
- **Оптимизация** — переписывай медленные подзапросы в JOIN когда возможно

## ⚠️ Частые ошибки

**❌ ОШИБКА:**
```sql
SELECT * FROM Users
WHERE name IN (
    SELECT id FROM Orders
);
```
❌ Типы не совпадают! name (строка) vs id (число)

**✅ ВЕРНО:**
```sql
SELECT * FROM Users
WHERE id IN (
    SELECT DISTINCT user_id FROM Orders
);
```

---

**❌ ОШИБКА:**
```sql
SELECT * FROM Users
WHERE city NOT IN (
    SELECT city FROM Products WHERE price > 100
);
```
❌ Если подзапрос вернёт NULL — результат всегда FALSE!

**✅ ВЕРНО:**
```sql
SELECT * FROM Users
WHERE id NOT IN (
    SELECT user_id FROM Orders WHERE status IS NOT NULL
);
-- Или используй NOT EXISTS:
SELECT * FROM Users u
WHERE NOT EXISTS (
    SELECT 1 FROM Orders WHERE user_id = u.id
);
```

---

**❌ ОШИБКА:**
```sql
SELECT name, 
    (SELECT COUNT(*) FROM Orders WHERE user_id = u.id) as cnt
FROM Users u;
```
❌ Медленно! Подзапрос выполняется для каждой строки

**✅ ВЕРНО (если возможно):**
```sql
SELECT u.name, COUNT(o.id) as cnt
FROM Users u
LEFT JOIN Orders o ON u.id = o.user_id
GROUP BY u.id;
```

---

**❌ ОШИБКА:**
```sql
SELECT * FROM Orders
WHERE user_id = (SELECT id FROM Users WHERE status = 'active');
```
❌ Если подзапрос вернёт несколько строк — ошибка!

**✅ ВЕРНО:**
```sql
SELECT * FROM Orders
WHERE user_id IN (
    SELECT id FROM Users WHERE status = 'active'
);
```
