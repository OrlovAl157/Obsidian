## ➗ Деление и остаток

**Назначение:** делить числа, получать целую часть результата и остаток.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `DIV` | `a DIV b` | Возвращает только целую часть деления. |
| `MOD` | `MOD(a, b)` | Возвращает остаток от деления. |

**Примеры:**

```sql
SELECT 11 DIV 3;
-- 3

SELECT 11 MOD 3;
-- 2

SELECT 11 % 3;
-- 2

SELECT MOD(11, 3);
-- 2
```

**Важное:** `MOD()` и оператор `%` дают одинаковый результат для остатка.




## 🔄 Округление и модуль

**Назначение:** менять точность числа, округлять вверх или вниз и получать абсолютное значение.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `ABS` | `ABS(x)` | Убирает знак числа. |
| `ROUND` | `ROUND(x[, d])` | Округляет до нужного количества знаков. |
| `FLOOR` | `FLOOR(x)` | Округляет вниз. |
| `CEILING` | `CEILING(x)` | Округляет вверх. |

**Примеры:**

```sql
SELECT ABS(-2.1234);
-- 2.1234

SELECT ROUND(23.1653, 2);
-- 23.17

SELECT FLOOR(-11.22);
-- -12

SELECT CEILING(18.39);
-- 19
```

**Важное:** `FLOOR()` и `CEILING()` особенно заметно ведут себя на отрицательных числах.




## ⚡ Степени и корни

**Назначение:** работать со степенями, корнями и случайными числами.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `POW` | `POW(x, y)` | Возводит число `x` в степень `y`. |
| `SQRT` | `SQRT(x)` | Возвращает квадратный корень. |
| `RAND` | `RAND()` | Возвращает случайное число от `0` до `1`. |

**Примеры:**

```sql
SELECT POW(2, 3);
-- 8

SELECT SQRT(9);
-- 3

SELECT RAND();
-- 0.8266121575856694
```

**Важное:** `RAND()` удобно использовать для генерации случайных значений, но результат меняется от вызова к вызову.




## 📐 Тригонометрия и углы

**Назначение:** выполнять тригонометрические вычисления и переводить градусы в радианы и обратно.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `DEGREES` | `DEGREES(x)` | Переводит радианы в градусы. |
| `RADIANS` | `RADIANS(x)` | Переводит градусы в радианы. |
| `PI` | `PI()` | Возвращает число Пи. |
| `SIN` / `COS` / `TAN` / `COT` | `SIN(x)`, `COS(x)` ... | Тригонометрические функции. |

**Примеры:**

```sql
SELECT DEGREES(3.14);
-- 179.9087476710785

SELECT RADIANS(180);
-- 3.141592653589793

SELECT PI();
-- 3.141593

SELECT SIN(PI());
-- 0

SELECT COS(PI());
-- -1

SELECT TAN(PI());
-- 0

SELECT COT(PI());
-- -8165619676597685
```

**Важное:** тригонометрические функции в MySQL принимают аргументы в радианах.




## 🎯 Минимум, максимум и форматирование

**Назначение:** выбирать крайние значения и форматировать числа для чтения.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `LEAST` | `LEAST(a, b, ...)` | Возвращает минимальное значение из списка. |
| `GREATEST` | `GREATEST(a, b, ...)` | Возвращает максимальное значение из списка. |
| `FORMAT` | `FORMAT(x, d)` | Округляет и форматирует число. |

**Примеры:**

```sql
SELECT LEAST(2, 1, 3, 5, 4);
-- 1

SELECT GREATEST(2, 1, 3, 5, 4);
-- 5

SELECT FORMAT(1000000.1653, 2);
-- 1,000,000.17
```

**Важное:** `FORMAT()` не просто округляет, но и добавляет разделители тысяч.




## 📊 Логарифмы и системы счисления

**Назначение:** считать логарифмы и переводить числа между системами счисления.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `CONV` | `CONV(n, from_base, to_base)` | Переводит число из одной системы счисления в другую. |
| `LOG2` | `LOG2(x)` | Возвращает двоичный логарифм. |
| `LOG10` | `LOG10(x)` | Возвращает логарифм по основанию `10`. |

**Примеры:**

```sql
SELECT CONV('10F', 16, 2);
-- 100001111

SELECT LOG2(65536);
-- 16

SELECT LOG10(100);
-- 2
```

**Важное:** `CONV()` удобно использовать для работы с двоичными, шестнадцатеричными и другими представлениями чисел.

---

## 💡 Практические замечания

- Для работы с деньгами и точностью используй `ROUND()` с правильным количеством знаков.
- `GREATEST()` и `LEAST()` полезны для ограничений типа "минимум 0, максимум 100".
- `FORMAT()` удобен для вывода в отчёты, но не для сохранения в БД (результат — строка).
- `RAND()` используй осторожно в WHERE — может снизить производительность из-за отсутствия индекса.
- Тригонометрия редко нужна в SQL, но встречается в геолокации и специальных вычислениях.
- `CONV()` помогает при работе с битовыми флагами и нестандартными представлениями.
- Большинство числовых функций при `NULL` возвращают `NULL`.
