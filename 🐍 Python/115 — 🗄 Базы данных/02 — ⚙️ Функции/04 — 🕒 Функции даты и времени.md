## 🕐 Текущие дата и время

**Назначение:** получить текущие дату и время в локальной зоне или в UTC.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `NOW` | `NOW()` | Текущие дата и время. |
| `CURDATE` | `CURDATE()` | Только дата. |
| `CURTIME` | `CURTIME()` | Только время. |
| `UTC_TIMESTAMP` / `UTC_DATE` / `UTC_TIME` | `UTC_TIMESTAMP()` | Возвращают значения в UTC. |

**Примеры:**

```sql
SELECT NOW();
-- 2023-06-27 10:14:48, дата + время

SELECT CURDATE();
-- 2023-06-27, дата

SELECT CURTIME();
-- 10:23:15, время

SELECT UTC_TIMESTAMP();
-- 2023-06-28 05:44:53, UTC
```

**Важное:** UTC-функции полезны, когда нужно хранить или сравнивать время независимо от локального часового пояса.




## 🔨 Создание даты и времени

**Назначение:** создать дату или время из составных частей.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `MAKEDATE` | `MAKEDATE(year, dayofyear)` | Формирует дату по году и номеру дня. |
| `MAKETIME` | `MAKETIME(h, m, s)` | Формирует время по часам, минутам и секундам. |

**Примеры:**

```sql
SELECT MAKEDATE(2023, 365);
-- 2023-12-31

SELECT MAKEDATE(2023, 366);
-- 2024-01-01  (день > 365 → переходит в следующий год)

SELECT MAKEDATE(2023, 732);
-- 2025-01-01  (каждые 365-366 дней = 1 год)

SELECT MAKETIME(21, 11, 42);
-- 21:11:42

SELECT MAKETIME(222, 50, 11);
-- 222:50:11  (часы могут быть > 24)

SELECT MAKETIME(10, 60, 12);
-- NULL  ❗ минуты > 59 → NULL

SELECT MAKETIME(5, 33, 79);
-- NULL  ❗ секунды > 59 → NULL
```

**Важное:** такие функции удобны, когда дата и время приходят в виде отдельных числовых компонентов.

> [!warning] MAKETIME и NULL
> Часы могут быть любыми (даже 222), но минуты и секунды **не могут превышать 59** — функция вернёт `NULL`.




## 🔄 Преобразование времени и секунд

**Назначение:** переводить время в секунды и обратно.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `TIME_TO_SEC` | `TIME_TO_SEC(time)` | Переводит время в секунды. |
| `SEC_TO_TIME` | `SEC_TO_TIME(sec)` | Переводит секунды во время. |

**Примеры:**

```sql
SELECT TIME_TO_SEC('01:00:00');
-- 3600

SELECT SEC_TO_TIME(3600);
-- 01:00:00
```

**Важное:** удобно для расчётов с продолжительностями и временем выполнения.




## 🛠 Служебные функции даты

**Назначение:** получать служебные числовые представления даты и последний день месяца.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `TO_DAYS` | `TO_DAYS(date)` | Количество дней от нулевого года. |
| `TO_SECONDS` | `TO_SECONDS(date)` | Количество секунд от нулевого года. |
| `LAST_DAY` | `LAST_DAY(date)` | Последний день месяца. |

**Примеры:**

```sql
SELECT LAST_DAY('2023-02-14');
-- 2023-02-28

SELECT TO_DAYS('2023-04-01');
-- 737764
```

**Важное:** `LAST_DAY()` особенно удобна в отчётах и при работе с границами месяца.




## ✂️ Извлечение частей даты и времени

**Назначение:** получать отдельные компоненты даты и времени.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `DATE` / `TIME` | `DATE(expr)`, `TIME(expr)` | Извлекают дату или время. |
| `YEAR` / `MONTH` / `DAY` | `YEAR(date)` | Извлекают части даты. |
| `HOUR` / `MINUTE` / `SECOND` / `MICROSECOND` | `HOUR(time)` | Извлекают части времени. |

**Примеры:**

```sql
SELECT DATE('2023-12-31 10:00:20');
-- 2023-12-31

SELECT YEAR('2027-12-31');
-- 2027

SELECT MONTH('2023-10-20');
-- 10

SELECT HOUR('18:20:00');
-- 18

SELECT MINUTE('12:30:10.1415');
-- 30
```

**Важное:** эти функции удобны при группировке, фильтрации и форматировании по отдельным частям даты и времени.




## 📅 Дни недели, недели и кварталы

**Назначение:** работать с календарными характеристиками даты.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `DAYOFYEAR` | `DAYOFYEAR(date)` | День года. |
| `WEEK` | `WEEK(date)` | Номер недели. |
| `WEEKDAY` | `WEEKDAY(date)` | День недели, где понедельник = `0`. |
| `QUARTER` | `QUARTER(date)` | Квартал. |
| `DAYNAME` / `MONTHNAME` | `DAYNAME(date)` | Названия дня недели и месяца. |

**Примеры:**

```sql
SELECT DAYOFYEAR('2023-12-31');
-- 365

SELECT WEEK('2023-12-31');
-- 53

SELECT WEEKDAY('2023-10-16');
-- 0, понедельник

SELECT QUARTER('2023-10-01');
-- 4

SELECT DAYNAME('2023-10-16');
-- Monday
```

**Важное:** `DAYNAME()` и `MONTHNAME()` зависят от локали сервера или текущих настроек языка.

## ➕ Сложение и вычитание интервалов

**Назначение:** прибавлять и вычитать интервалы из даты, времени или timestamp.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `INTERVAL` | `date + INTERVAL expr unit` | Позволяет добавлять интервалы напрямую. |
| `ADDDATE` / `SUBDATE` | `ADDDATE(date, INTERVAL ...)` | Работают с датой и datetime. |
| `ADDTIME` / `SUBTIME` | `ADDTIME(expr1, expr2)` | Работают со временем и datetime. |

**Формат составных интервалов:**

| Единица | Формат величины | Пример |
| --- | --- | --- |
| `YEAR_MONTH` | `'годы-месяцы'` | `INTERVAL '10-2' YEAR_MONTH` |
| `DAY_HOUR` | `'дни часы'` | `INTERVAL '4 2' DAY_HOUR` |
| `HOUR_MINUTE` | `'часы:минуты'` | `INTERVAL '01:30' HOUR_MINUTE` |
| `DAY_SECOND` | `'дни часы:минуты:секунды'` | `INTERVAL '1 02:30:00' DAY_SECOND` |

**Примеры:**

```sql
SELECT '2023-01-01' + INTERVAL '10-2' YEAR_MONTH;
-- 2033-03-01

SELECT '2023-01-01 14:00:00' - INTERVAL '01:30' HOUR_MINUTE;
-- 2023-01-01 12:30:00

SELECT '2023-01-01' - INTERVAL 1 HOUR;
-- 2022-12-31 23:00:00  (дата без времени → получает 00:00:00, результат становится datetime)

SELECT ADDDATE('2023-10-20 07:30:00', INTERVAL 4 HOUR);
-- 2023-10-20 11:30:00

SELECT SUBDATE('2023-10-20', INTERVAL 4 DAY);
-- 2023-10-16

SELECT ADDTIME('2023-10-20 21:00:00', '06:00:00');
-- 2023-10-21 03:00:00
```

**Важное:** здесь важно не только значение интервала, но и его единица: `DAY`, `MONTH`, `YEAR_MONTH` и так далее.

> [!warning] Составные интервалы — формат строки
> Для составных единиц (`YEAR_MONTH`, `DAY_HOUR` и др.) величина передаётся как **строка** в определённом формате, а не как число. Неправильный формат даст неожиданный результат.




## 📏 Разница между датами

**Назначение:** считать разницу между датами и временем разными способами.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `DATEDIFF` | `DATEDIFF(a, b)` | Считает разницу в днях, время игнорируется. |
| `TIMEDIFF` | `TIMEDIFF(a, b)` | Считает разницу между временем или datetime. |
| `TIMESTAMPDIFF` | `TIMESTAMPDIFF(unit, a, b)` | Считает разницу в выбранной единице. Только одиночные единицы. |
| `TIMESTAMP` | `TIMESTAMP(date, time)` | Объединяет дату и время. |

**Примеры:**

```sql
-- DATEDIFF игнорирует время — считает только даты
SELECT DATEDIFF('2023-10-20 07:30:00', '2023-10-20 23:59:59');
-- 0  ❗ одна и та же дата, несмотря на разное время

SELECT DATEDIFF('2023-10-26 07:30:00', '2023-10-20 08:00:00');
-- 6

SELECT TIMEDIFF('2023-10-25 09:00:00', '2023-10-20 07:00:00');
-- 122:00:00

SELECT TIMESTAMPDIFF(HOUR, '2023-10-20 07:00:00', '2023-10-20 12:00:00');
-- 5

SELECT TIMESTAMPDIFF(MONTH, '2023-10-20', '2023-12-31');
-- 2
```

**Важное:** `DATEDIFF()` игнорирует время суток и считает только дни.

> [!warning] TIMESTAMPDIFF — только одиночные единицы
> Составные единицы (`DAY_HOUR`, `YEAR_MONTH` и др.) вызовут **ошибку синтаксиса**.
> ```sql
> TIMESTAMPDIFF(DAY_HOUR, '2023-10-20', '2023-10-25')  -- ❌ ERROR!
> TIMESTAMPDIFF(DAY, '2023-10-20', '2023-10-25')        -- ✅ 5
> ```




## 🎨 Форматирование даты и времени

**Назначение:** преобразовывать дату и время в строку и обратно по заданному шаблону.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `DATE_FORMAT` | `DATE_FORMAT(date, format)` | Форматирует дату и время в строку. |
| `TIME_FORMAT` | `TIME_FORMAT(time, format)` | Форматирует время. |
| `STR_TO_DATE` | `STR_TO_DATE(str, format)` | Преобразует строку в дату/время. |
| `GET_FORMAT` | `GET_FORMAT(type, locale)` | Возвращает шаблон формата. |

**Примеры:**

```sql
SELECT DATE_FORMAT('2023-10-20 07:30:00', '%H:%i, %d.%m.%Y');
-- 07:30, 20.10.2023

SELECT STR_TO_DATE('07:30, 20.10.2023', '%H:%i, %d.%m.%Y');
-- 2023-10-20 07:30:00

SELECT STR_TO_DATE('20.10.2023', '%d.%m.%Y');
-- 2023-10-20  (только дата)

SELECT STR_TO_DATE('20.10', '%d.%m');
-- NULL  ❗ нет года → не может построить дату

SELECT STR_TO_DATE('Time: 07', 'Time: %H');
-- 07:00:00  (для времени достаточно одного компонента, остальные = 0)

SELECT GET_FORMAT(DATE, 'EUR');
-- %d.%m.%Y
```

**Важное:** `STR_TO_DATE()` полезна, когда дата приходит как текст из CSV, формы или внешней системы.

> [!warning] STR_TO_DATE — правило про NULL
> Для построения **даты** нужны все три компонента: год, месяц, день — иначе `NULL`.
> Для построения **времени** достаточно одного любого компонента, остальные примут значение `0`.




## 📊 Спецификаторы форматирования

| Спецификатор | Описание |
| --- | --- |
| `%M` | Полное название месяца: January..December |
| `%b` | Сокращённое название месяца: Jan..Dec |
| `%W` | Полное название дня недели: Sunday..Saturday |
| `%a` | Сокращённое название дня недели: Sun..Sat |
| `%Y` | Год: 0000..9999 |
| `%m` | Номер месяца: 00..12 |
| `%d` | День месяца: 00..31 |
| `%w` | Номер дня недели: 0..6 |
| `%H` | Часы: 00..23 |
| `%i` | Минуты: 00..59 |
| `%S` | Секунды: 00..59 |
| `%f` | Микросекунды: 000000..999999 |
| `%T` | Время в формате HH:MM:SS |

## 💡 Практические замечания

- Для хранения времени в базе используй UTC, конвертируй в локальное время только при выводе.
- `NOW()` вызывается один раз в начале выполнения запроса — если нужно время до миллисекунды, используй `MICROMICROSECOND()`.
- `DATE_FORMAT()` полезна для отчётов, но медленнее при работе с большими объёмами данных.
- `DATEDIFF()` работает только с полными днями, для точной разницы используй `TIMESTAMPDIFF()`.
- При сравнении дат помни о часовых поясах если они в столбце.
- `STR_TO_DATE()` может быть медленной на больших таблицах — лучше нормализуй данные при загрузке.
- `LAST_DAY()` удобна для рассчётов счётов, квартальных отчётов и планов.
