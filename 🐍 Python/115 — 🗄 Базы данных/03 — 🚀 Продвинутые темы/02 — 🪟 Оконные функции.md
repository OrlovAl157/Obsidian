## Справка по оконным функциям

| Функция | Что делает | Синтаксис | Категория |
| --- | --- | --- | --- |
| `ROW_NUMBER()` | Нумерует строки (1, 2, 3...) | `ROW_NUMBER() OVER (...)` | Ранжирование |
| `RANK()` | Ранг с пропусками (1, 2, 2, 4...) | `RANK() OVER (...)` | Ранжирование |
| `DENSE_RANK()` | Ранг без пропусков (1, 2, 2, 3...) | `DENSE_RANK() OVER (...)` | Ранжирование |
| `NTILE(n)` | Разбивает на n групп | `NTILE(4) OVER (...)` | Ранжирование |
| `SUM()` | Сумма по окну | `SUM(salary) OVER (...)` | Агрегат |
| `AVG()` | Среднее по окну | `AVG(salary) OVER (...)` | Агрегат |
| `COUNT()` | Подсчёт по окну | `COUNT(*) OVER (...)` | Агрегат |
| `MIN()` / `MAX()` | Мин/макс по окну | `MIN(price) OVER (...)` | Агрегат |
| `FIRST_VALUE()` | Первое значение окна | `FIRST_VALUE(col) OVER (...)` | Смещение |
| `LAST_VALUE()` | Последнее значение окна | `LAST_VALUE(col) OVER (...)` | Смещение |
| `NTH_VALUE()` | N-ое значение окна | `NTH_VALUE(col, 2) OVER (...)` | Смещение |
| `LAG()` | Значение n строк назад | `LAG(col, 1) OVER (...)` | Смещение |
| `LEAD()` | Значение n строк вперёд | `LEAD(col, 1) OVER (...)` | Смещение |

## 🎯 Синтаксис OVER

**Назначение:** определить окно (набор строк) для оконной функции.

| Клауза | Что делает | Пример |
| --- | --- | --- |
| `OVER ()` | Окно = вся таблица | `SUM(salary) OVER ()` |
| `OVER (ORDER BY col)` | Окно от начала до текущей строки | `ROW_NUMBER() OVER (ORDER BY salary)` |
| `OVER (PARTITION BY col)` | Отдельное окно для каждого значения | `SUM(salary) OVER (PARTITION BY dept)` |
| `OVER (PARTITION BY col ORDER BY col)` | Комбо: разделить + упорядочить | `ROW_NUMBER() OVER (PARTITION BY dept ORDER BY salary)` |

**Примеры:**

```sql
-- Окно = вся таблица
SELECT salary, SUM(salary) OVER () as total_payroll
FROM Employees;

-- Окно по отделам
SELECT name, department, salary,
       SUM(salary) OVER (PARTITION BY department) as dept_payroll
FROM Employees;

-- Ранжирование в каждом отделе
SELECT name, department, salary,
       ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) as rank
FROM Employees;
```

**Важное:** 
- `PARTITION BY` разделяет на окна, в каждом считается отдельно
- `ORDER BY` упорядочивает строки внутри окна
- `ROWS` и `RANGE` определяют границы окна

## 🔢 Ранжирующие функции

**Назначение:** нумеровать или ранжировать строки.

| Функция | Поведение | Пример результата |
| --- | --- | --- |
| `ROW_NUMBER()` | Уникальная нумерация | 1, 2, 3, 4, 5 |
| `RANK()` | Ранг с пропусками | 1, 2, 2, 4, 5 |
| `DENSE_RANK()` | Ранг без пропусков | 1, 2, 2, 3, 4 |
| `NTILE(n)` | Разбивает на n групп | 1, 1, 1, 2, 2, 3, 3 |

**Примеры:**

```sql
-- ROW_NUMBER: строго по порядку
SELECT id, full_name, salary,
       ROW_NUMBER() OVER (ORDER BY salary DESC) AS row_num
FROM Employees;

-- Результат:
-- id | full_name     | salary | row_num
-- ---+---------------+--------+----------
-- 1  | Sergey Brin   | 10000  | 1
-- 2  | Alice Johnson | 10000  | 2       ← разные номера, одинаковая зарплата
-- 3  | Susan Wojcicki| 8000   | 3
```

```sql
-- RANK: одинаковые = одинаковый ранг, потом пропуск
SELECT id, full_name, salary,
       RANK() OVER (ORDER BY salary DESC) AS rank
FROM Employees;

-- Результат:
-- id | full_name     | salary | rank
-- ---+---------------+--------+------
-- 1  | Sergey Brin   | 10000  | 1
-- 2  | Alice Johnson | 10000  | 2    ← НО выводится как 2, следующий будет 3 (не 2)
-- 3  | Susan Wojcicki| 8000   | 3
```

```sql
-- DENSE_RANK: одинаковые = ранг, нет пропусков
SELECT id, full_name, salary,
       DENSE_RANK() OVER (ORDER BY salary DESC) AS dense_rank
FROM Employees;

-- Результат:
-- id | full_name     | salary | dense_rank
-- ---+---------------+--------+------------
-- 1  | Sergey Brin   | 10000  | 1
-- 2  | Alice Johnson | 10000  | 2
-- 3  | Susan Wojcicki| 8000   | 3    ← следующий ранг сразу 3
```

```sql
-- NTILE: разбить на 3 группы поровну
SELECT id, full_name,
       NTILE(3) OVER (ORDER BY id) AS quartile
FROM Employees;

-- Результат:
-- id | full_name      | quartile
-- ---+----------------+----------
-- 1  | Sergey Brin    | 1
-- 2  | John Doerr     | 1
-- 3  | Larry Page     | 1
-- 4  | Eric Schmidt   | 1
-- 5  | Sundar Pichai  | 2
-- 6  | Marissa Mayer  | 2
-- 7  | Susan Wojcicki | 2
-- 8  | John Smith     | 3
```

**Важное:** 
- `ROW_NUMBER()` и `RANK()` отличаются только при одинаковых значениях
- `DENSE_RANK()` полезен когда нет "дыр" в рангах важна
- `NTILE()` для квартилей, терцилей и т.д.

## 📊 Агрегатные функции как оконные

**Назначение:** вычислить статистику для окна, не теряя отдельные строки (как с GROUP BY).

| Функция | Что считает |
| --- | --- |
| `SUM()` | Сумма значений в окне |
| `AVG()` | Среднее значение |
| `COUNT()` | Количество строк |
| `MIN()` / `MAX()` | Минимум и максимум |

**Примеры:**

```sql
-- Для каждого сотрудника: его зарплата и средняя по отделу
SELECT name, department, salary,
       AVG(salary) OVER (PARTITION BY department) as dept_avg_salary
FROM Employees;

-- Результат:
-- name       | department | salary | dept_avg_salary
-- -----------+------------+--------+-----------------
-- Alice      | HR         | 70000  | 65000
-- Bob        | HR         | 60000  | 65000
-- Carol      | HR         | 65000  | 65000
-- Dan        | IT         | 80000  | 80000
-- Eve        | IT         | 80000  | 80000
```

```sql
-- Сумма зарплат по отделам (все сотрудники видят одно число)
SELECT name, department, salary,
       SUM(salary) OVER (PARTITION BY department) as dept_payroll
FROM Employees;

-- Результат:
-- name  | department | salary | dept_payroll
-- ------+------------+--------+--------------
-- Alice | HR         | 70000  | 195000     ← все HR видят 195000
-- Bob   | HR         | 60000  | 195000
-- Carol | HR         | 65000  | 195000
-- Dan   | IT         | 80000  | 160000     ← все IT видят 160000
-- Eve   | IT         | 80000  | 160000
```

**Важное:** 
- Агрегатные функции в OVER не групмпируют строки (есть все строки)
- Каждому сотруднику видна статистика его окна
- По умолчанию окно = от начала до текущей строки (если есть ORDER BY)
