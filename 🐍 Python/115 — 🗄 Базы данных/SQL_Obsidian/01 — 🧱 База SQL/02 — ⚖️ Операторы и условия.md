## Справка по операторам

| Группа | Оператор / конструкция | Что делает |
| --- | --- | --- |
| Сравнение | `=`, `!=`, `<>`, `<`, `<=`, `>`, `>=` | Сравнивает значения. |
| NULL-safe | `<=>` | Безопасно сравнивает значения с `NULL`. |
| Логика | `AND`, `OR`, `NOT` | Объединяет или инвертирует условия. |
| Списки | `IN`, `NOT IN` | Проверяет вхождение значения в список. |
| Диапазоны | `BETWEEN` | Проверяет попадание в диапазон включительно. |
| NULL | `IS NULL`, `IS NOT NULL` | Проверяет, является ли значение `NULL`. |
| Шаблоны | `LIKE` | Ищет по шаблону с `_` и `%`. |
| Подзапросы | `EXISTS`, `ALL`, `ANY` | Работает с результатами подзапросов. |
| Вывод | `DISTINCT`, `LIMIT`, `OFFSET` | Управляет результатом выборки. |
| Сортировка | `ORDER BY`, `FIELD()` | Задаёт порядок строк. |
| Объединение | `UNION`, `UNION ALL` | Объединяет результаты запросов. |

## ⚖️ Операторы сравнения

**Назначение:** сравнить значения в `WHERE`, `JOIN ... ON`, `HAVING` и других условиях.

| Оператор | Значение | Пример |
| --- | --- | --- |
| `=` | равно | `WHERE id = 5` |
| `!=` или `<>` | не равно | `WHERE status != 'closed'` |
| `<` | меньше | `WHERE price < 100` |
| `<=` | меньше или равно | `WHERE age <= 18` |
| `>` | больше | `WHERE created_at > '2024-01-01'` |
| `>=` | больше или равно | `WHERE score >= 10` |

**Примеры:**

```sql
SELECT *
FROM Products
WHERE price >= 100;

SELECT *
FROM Users
WHERE status <> 'blocked';
```

**Важное:** обычные операторы сравнения с `NULL` не возвращают `TRUE` или `FALSE`; результатом будет `NULL`.

## 🧠 NULL и безопасное сравнение

**Назначение:** правильно работать с отсутствующими значениями.

| Оператор | Что делает | Пример |
| --- | --- | --- |
| `IS NULL` | проверяет, что значение отсутствует | `WHERE deleted_at IS NULL` |
| `IS NOT NULL` | проверяет, что значение есть | `WHERE email IS NOT NULL` |
| `<=>` | безопасное сравнение с `NULL` | `SELECT NULL <=> NULL` |

**Примеры:**

```sql
SELECT 'x' != NULL, 10 > NULL, NULL = NULL;
-- NULL | NULL | NULL

SELECT 'x' <=> NULL, '5' <=> NULL, NULL <=> NULL;
-- 0 | 0 | 1

SELECT *
FROM Users
WHERE email IS NOT NULL;
```

**Важное:** для проверки `NULL` используй `IS NULL` / `IS NOT NULL`, а не `= NULL`.

## 🔗 Логические операторы

**Назначение:** объединять несколько условий.

| Оператор | Что делает | Пример |
| --- | --- | --- |
| `AND` | оба условия истинны | `price > 100 AND amount > 0` |
| `OR` | хотя бы одно условие истинно | `city = 'Moscow' OR city = 'Kazan'` |
| `NOT` | отрицание условия | `NOT active` |

**Примеры:**

```sql
SELECT *
FROM Products
WHERE price BETWEEN 10 AND 100
  AND amount > 0;

SELECT *
FROM Users
WHERE role = 'admin'
   OR role = 'moderator';
```

**Важное:** если смешиваешь `AND` и `OR`, лучше ставить скобки, чтобы явно показать порядок логики.

## 🔍 Фильтрация списков, диапазонов и шаблонов

**Назначение:** удобно описывать часто встречающиеся условия.

| Конструкция | Что делает | Пример |
| --- | --- | --- |
| `IN` | значение входит в список | `name IN ('USA', 'Russia', 'China')` |
| `NOT IN` | значение не входит в список | `num NOT IN (50, 53, 54)` |
| `BETWEEN` | значение в диапазоне включительно | `price BETWEEN 10 AND 100` |
| `LIKE` | соответствие шаблону | `phone LIKE '_90%'` |
| `EXISTS` | подзапрос вернул строки | `WHERE EXISTS (...)` |
| `ALL` | условие истинно для всех значений подзапроса | `num > ALL (...)` |
| `ANY` | условие истинно хотя бы для одного значения | `num > ANY (...)` |

**Примеры:**

```sql
SELECT *
FROM products
WHERE price BETWEEN 10 AND 100;

SELECT *
FROM users
WHERE phone LIKE '_90%';

SELECT *
FROM countries
WHERE name IN ('USA', 'Russia', 'China');

SELECT *
FROM numbers
WHERE num NOT IN (50, 53, 54, 55);
```

```sql
SELECT title
FROM Books
WHERE EXISTS (
    SELECT *
    FROM BooksRental
    WHERE book_id = Books.id
);
```

```sql
SELECT num
FROM Numbers
WHERE num > ALL (SELECT num FROM Numbers WHERE num < 6);
-- Результат: 6, 7, 8, 9

SELECT num
FROM Numbers
WHERE num > ANY (SELECT num FROM Numbers WHERE num < 6);
-- Результат: 1, 2, 3, 4, 5, 6, 7, 8, 9
```

**Важное:** `BETWEEN` включает обе границы. Для `LIKE`: `_` означает один символ, `%` — любое количество символов.

## 📌 DISTINCT, LIMIT, OFFSET, AS, ORDER BY

**Назначение:** управлять тем, какие строки и столбцы попадут в результат и в каком порядке.

| Конструкция | Что делает | Пример |
| --- | --- | --- |
| `DISTINCT` | убирает дубликаты | `SELECT DISTINCT name FROM users` |
| `LIMIT` | ограничивает количество строк | `LIMIT 2` |
| `OFFSET` | пропускает указанное количество строк | `OFFSET 3` |
| `AS` | задаёт псевдоним | `nums AS numbers` |
| `ORDER BY` | сортирует результат | `ORDER BY name DESC` |
| `FIELD()` | задаёт пользовательский порядок | `ORDER BY FIELD(status, 'new', 'done')` |

**Примеры:**

```sql
SELECT DISTINCT name
FROM users;
-- John, Mary, Paul

SELECT *
FROM users
LIMIT 2;
-- первые 2 записи

SELECT *
FROM users
OFFSET 3 LIMIT 2;
-- 4-я и 5-я записи, потому что OFFSET начинается с 0
```

```sql
SELECT nums AS numbers
FROM table_name;

SELECT *
FROM users
ORDER BY name;
-- по возрастанию, ASC по умолчанию

SELECT *
FROM users
ORDER BY name DESC;
-- по убыванию
```

```sql
SELECT *
FROM Tasks
ORDER BY FIELD(status, 'urgent', 'normal', 'later');
```

**Важное:** `LIMIT` без `ORDER BY` может давать нестабильный порядок строк, потому что база не обязана возвращать строки одинаково каждый раз.

## 🧩 UNION и UNION ALL

**Назначение:** объединять результаты нескольких запросов с одинаковым количеством столбцов.

| Конструкция | Что делает | Что важно помнить |
| --- | --- | --- |
| `UNION` | объединяет и удаляет дубликаты | По умолчанию работает как `UNION DISTINCT`. |
| `UNION ALL` | объединяет и оставляет все строки | Быстрее, если дубликаты не нужно удалять. |

**Примеры:**

```sql
SELECT name FROM table1
UNION
SELECT name FROM table2;
-- удаляет дубликаты

SELECT name FROM table1
UNION ALL
SELECT name FROM table2;
-- оставляет все записи
```

**Важное:** типы столбцов в объединяемых запросах должны быть совместимы.

## 🛠 Служебные операторы и конструкции

**Назначение:** получать информацию о таблицах и управлять поведением запросов.

### DESCRIBE

```sql
DESCRIBE Books;
-- Результат:
-- +------------+-------------+------+-----+---------+----------------+
-- | Field      | Type        | Null | Key | Default | Extra          |
-- +------------+-------------+------+-----+---------+----------------+
-- | id         | int         | NO   | PRI | NULL    | auto_increment |
-- | title      | varchar(40) | YES  |     | NULL    |                |
-- | author_id  | int         | YES  | MUL | NULL    |                |
-- +------------+-------------+------+-----+---------+----------------+
```

### IGNORE

```sql
UPDATE IGNORE Books
SET id = 'NULL'
WHERE id LIKE '%';
-- Ошибочные значения не изменяются, запрос продолжается
```

## 🧭 Порядок выполнения запроса

| № | Ключевое слово | Описание |
| --- | --- | --- |
| 1 | `FROM` | Таблица для извлечения данных |
| 2 | `JOIN ... ON` | Соединение таблиц |
| 3 | `WHERE` | Фильтрация строк |
| 4 | `GROUP BY` | Создание групп |
| 5 | `HAVING` | Фильтрация групп |
| 6 | `SELECT` | Выбор выражений для результата |
| 7 | `DISTINCT` | Удаление дубликатов |
| 8 | `ORDER BY` | Сортировка результата |
| 9 | `LIMIT` / `OFFSET` | Ограничение и смещение результата |

**Важное:** в SQL логический порядок выполнения отличается от порядка записи запроса. Поэтому псевдонимы из `SELECT` не всегда доступны в `WHERE`.
