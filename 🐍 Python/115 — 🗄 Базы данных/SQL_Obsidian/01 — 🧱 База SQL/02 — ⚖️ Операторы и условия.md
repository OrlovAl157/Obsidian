# ⚖️ Операторы и условия SQL

## Операторы сравнения

| Оператор | Значение | Пример |
|----------|----------|--------|
| `=` | равно | `WHERE id = 5` |
| `!=` или `<>` | не равно | `WHERE status != 'closed'` |
| `<` | меньше | `WHERE price < 100` |
| `<=` | меньше или равно | `WHERE age <= 18` |
| `>` | больше | `WHERE created_at > '2024-01-01'` |
| `>=` | больше или равно | `WHERE score >= 10` |
| `<=>` | безопасное сравнение с `NULL` | `SELECT NULL <=> NULL` — результат `1` |

### Пример `<=>`

```sql
SELECT 'x' != NULL, 10 > NULL, NULL = NULL;
-- Результат: NULL | NULL | NULL

SELECT 'x' <=> NULL, '5' <=> NULL, NULL <=> NULL;
-- Результат: 0 | 0 | 1
```

## Логические операторы

| Оператор | Значение |
|----------|----------|
| `AND` | И: оба условия истинны |
| `OR` | ИЛИ: хотя бы одно условие истинно |
| `NOT` | НЕ: отрицание |
| `IN` | входит в список |
| `NOT IN` | не входит в список |
| `BETWEEN` | в диапазоне, включительно |
| `IS NULL` | является `NULL` |
| `IS NOT NULL` | не является `NULL` |
| `LIKE` | соответствует шаблону |
| `EXISTS()` | существует в подзапросе |
| `ALL()` | для всех значений подзапроса |
| `ANY()` | для любого значения подзапроса |

### Примеры

```sql
-- BETWEEN: включительно
SELECT * FROM products WHERE price BETWEEN 10 AND 100;

-- LIKE: _ — один символ, % — любое количество символов
SELECT * FROM users WHERE phone LIKE '_90%';

-- IN
SELECT * FROM countries WHERE name IN ('USA', 'Russia', 'China');

-- NOT IN
SELECT * FROM numbers WHERE num NOT IN (50, 53, 54, 55);

-- EXISTS
SELECT title FROM Books
WHERE EXISTS (SELECT * FROM BooksRental WHERE book_id = Books.id);

-- ALL: все значения из подзапроса
SELECT num FROM Numbers
WHERE num > ALL (SELECT num FROM numbers WHERE num < 6);
-- Результат: 6, 7, 8, 9

-- ANY: часто лучше использовать IN
SELECT num FROM Numbers
WHERE num > ANY (SELECT num FROM numbers WHERE num < 6);
-- Результат: 1, 2, 3, 4, 5, 6, 7, 8, 9
```

## DISTINCT

Уникальные значения.

```sql
SELECT DISTINCT name FROM users;
-- John, Mary, Paul
```

## LIMIT и OFFSET

Ограничение вывода.

```sql
SELECT * FROM users LIMIT 2;
-- первые 2 записи

SELECT * FROM users OFFSET 3 LIMIT 2;
-- 3-я и 4-я записи, OFFSET с 0
```

## AS

Создание псевдонима.

```sql
SELECT nums AS numbers FROM table;
```

## ORDER BY

Сортировка.

```sql
SELECT * FROM users ORDER BY name;
-- по возрастанию, ASC по умолчанию

SELECT * FROM users ORDER BY name DESC;
-- по убыванию
```

## FIELD()

Возвращает индекс аргумента.

```sql
ORDER BY FIELD('C', 'D', 'A');
-- C, D, A: индекс 1 у C
```

## UNION / UNION ALL

Объединение результатов.

```sql
SELECT name FROM table1
UNION
SELECT name FROM table2;
-- UNION удаляет дубликаты, DISTINCT по умолчанию

SELECT name FROM table1
UNION ALL
SELECT name FROM table2;
-- UNION ALL оставляет все записи
```

## DESCRIBE

Информация о полях таблицы.

```sql
DESCRIBE Books;
-- Результат:
-- +------------+-------------+------+-----+---------+----------------+
-- | Field      | Type        | Null | Key | Default | Extra          |
-- +------------+-------------+------+-----+---------+----------------+
-- | id         | int         | NO   | PRI | NULL    | auto_increment |
-- | title      | varchar(40) | YES  |     | NULL    |                |
-- | author_id  | int         | YES  | MUL | NULL    |                |
-- +------------+-------------+------+-----+---------+----------------+
```

## IGNORE

Игнорирование ошибок при изменении данных.

```sql
UPDATE IGNORE Books
SET id = 'NULL'
WHERE id LIKE '%';
-- Ошибочные значения не изменяются, запрос продолжается
```

## Порядок выполнения запроса

| № | Ключевое слово | Описание |
|---|----------------|----------|
| 1 | `SELECT` | Данные для извлечения |
| 2 | `FROM` | Таблица для извлечения данных |
| 3 | `INNER JOIN ... ON` | Сопоставление двух таблиц по условию |
| 4 | `LEFT JOIN ... ON` | Присоединение к `FROM`-таблице новой |
| 5 | `RIGHT JOIN ... ON` | Присоединение к новой таблице из `FROM` |
| 6 | `CROSS JOIN ... ON` | Создание возможных пересечений строк двух таблиц |
| 7 | `WINDOW` | Создание оконной функции |
| 8 | `WHERE` | Фильтрация на уровне записей |
| 9 | `GROUP BY` | Создание групп |
| 10 | `HAVING` | Фильтрация на уровне групп |
| 11 | `ORDER BY` | Порядок сортировки результатов |
| 12 | `LIMIT` | Ограничение количества записей |
| 13 | `OFFSET` | Начало извлечения записей, нумерация с 0 |
