## Справка по функциям

| Функция | Что делает | Синтаксис |
| --- | --- | --- |
| `CONCAT` | Соединяет строки | `CONCAT(str1, str2, ...)` |
| `CONCAT_WS` | Соединяет строки через разделитель | `CONCAT_WS(sep, str1, str2, ...)` |
| `GROUP_CONCAT` | Собирает значения из нескольких строк в одну строку | `GROUP_CONCAT(expr SEPARATOR ', ')` |
| `CHAR_LENGTH` | Возвращает длину в символах | `CHAR_LENGTH(str)` |
| `LOCATE` | Ищет позицию подстроки | `LOCATE(substr, str[, start])` |
| `LEFT` | Берет первые `n` символов | `LEFT(str, n)` |
| `RIGHT` | Берет последние `n` символов | `RIGHT(str, n)` |
| `SUBSTRING` | Возвращает подстроку по позиции | `SUBSTRING(str, start[, len])` |
| `SUBSTRING_INDEX` | Возвращает часть строки по разделителю | `SUBSTRING_INDEX(str, delim, count)` |
| `LOWER` | Переводит в нижний регистр | `LOWER(str)` |
| `UPPER` | Переводит в верхний регистр | `UPPER(str)` |
| `ORD` | Возвращает код первого символа | `ORD(str)` |
| `CHAR` | Собирает символы по кодам | `CHAR(code1, code2, ...)` |
| `LPAD` | Дополняет строку слева | `LPAD(str, len, pad)` |
| `RPAD` | Дополняет строку справа | `RPAD(str, len, pad)` |
| `LTRIM` | Удаляет пробелы слева | `LTRIM(str)` |
| `RTRIM` | Удаляет пробелы справа | `RTRIM(str)` |
| `TRIM` | Удаляет пробелы или указанные символы по краям | `TRIM([LEADING \| TRAILING \| BOTH] [remstr FROM] str)` |
| `REPLACE` | Заменяет фрагменты строки | `REPLACE(str, from, to)` |
| `REVERSE` | Переворачивает строку | `REVERSE(str)` |
| `REPEAT` | Повторяет строку `n` раз | `REPEAT(str, n)` |

## Соединение строк

**Назначение:** собрать несколько значений в одну строку.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `CONCAT` | `CONCAT(str1, str2, ...)` | Если хотя бы один аргумент `NULL`, результат тоже `NULL`. |
| `CONCAT_WS` | `CONCAT_WS(separator, str1, str2, ...)` | Пропускает `NULL`-значения и не вставляет для них разделитель. |
| `GROUP_CONCAT` | `GROUP_CONCAT(expr [ORDER BY ...] [SEPARATOR str])` | Агрегатная функция: собирает значения из нескольких строк результата. |

**Примеры:**

```sql
SELECT CONCAT('John', ' ', 'Doe');
-- John Doe

SELECT CONCAT_WS('-', 'USA', 'NY', 'Manhattan');
-- USA-NY-Manhattan

SELECT CONCAT_WS(', ', 'Apple', NULL, 'Banana');
-- Apple, Banana

SELECT GROUP_CONCAT(name SEPARATOR ', ')
FROM users;
-- Peter, Gwen, Flash, Mary

SELECT GROUP_CONCAT(name ORDER BY name SEPARATOR ' | ')
FROM users;
```

**Важное:** `GROUP_CONCAT` часто используют с `GROUP BY`, но не только. Еще у него есть ограничение на максимальную длину результата: `group_concat_max_len`.

## Поиск подстроки и длина

**Назначение:** найти подстроку и понять размер строки.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `LOCATE` | `LOCATE(substr, str[, start])` | Возвращает позицию первого вхождения, нумерация идет с `1`. Если не найдено, вернет `0`. |
| `CHAR_LENGTH` | `CHAR_LENGTH(str)` | Возвращает количество символов, а не байтов. |

**Примеры:**

```sql
SELECT CHAR_LENGTH('beegeek');
-- 7

SELECT LOCATE('geek', 'beegeek');
-- 4

SELECT LOCATE('e', 'beegeek', 3);
-- 4

SELECT LOCATE('xyz', 'beegeek');
-- 0
```

**Важное:** чувствительность `LOCATE` к регистру зависит от collation строки. Если нужна длина в байтах, а не в символах, смотри `LENGTH()`.

## Извлечение части строки

**Назначение:** получить фрагмент строки по позиции, длине или разделителю.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `LEFT` | `LEFT(str, n)` | Возвращает первые `n` символов. |
| `RIGHT` | `RIGHT(str, n)` | Возвращает последние `n` символов. |
| `SUBSTRING` | `SUBSTRING(str, start[, len])` | Поддерживает отрицательный `start`, тогда отсчет идет с конца строки. |
| `SUBSTRING_INDEX` | `SUBSTRING_INDEX(str, delim, count)` | Работает по разделителю, а не по числовой позиции. |

**Примеры:**

```sql
SELECT LEFT('beegeek', 3);
-- bee

SELECT RIGHT('beegeek', 3);
-- eek

SELECT SUBSTRING('beegeek', 4, 2);
-- ge

SELECT SUBSTRING('beegeek', -4);
-- geek

SELECT SUBSTRING_INDEX('g-e-e-k', '-', 2);
-- g-e

SELECT SUBSTRING_INDEX('g-e-e-k', '-', -1);
-- k
```

**Важное:** `SUBSTRING()` удобно использовать для позиций, `SUBSTRING_INDEX()` для работы со строками вида `a/b/c`, email, доменами и путями.

## Изменение регистра

**Назначение:** привести строку к нижнему или верхнему регистру.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `LOWER` | `LOWER(str)` | Переводит буквы в нижний регистр. |
| `UPPER` | `UPPER(str)` | Переводит буквы в верхний регистр. |

**Примеры:**

```sql
SELECT LOWER('BeeGeek SQL');
-- beegeek sql

SELECT UPPER('BeeGeek SQL');
-- BEEGEEK SQL

SELECT LOWER(NULL);
-- NULL
```

**Важное:** выражения вроде `LOWER(name)` удобны для обучения и нормализации, но в фильтрах могут мешать использованию индекса по столбцу.

## Коды символов и CHAR

**Назначение:** работать с кодами символов и собирать строку по числовым кодам.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `ORD` | `ORD(str)` | Возвращает числовой код первого символа. |
| `CHAR` | `CHAR(code1, code2, ...)` | Возвращает строку, собранную из переданных кодов. |

**Примеры:**

```sql
SELECT ORD('A');
-- 65

SELECT ORD('Z');
-- 90

SELECT CHAR(65, 66, 67);
-- ABC
```

**Важное:** в учебных примерах это часто называют “ASCII”, но на практике поведение связано еще и с кодировкой/charset.

## Выравнивание и дополнение

**Назначение:** искусственно довести строку до нужной длины.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `LPAD` | `LPAD(str, len, pad)` | Добавляет символы слева до длины `len`. |
| `RPAD` | `RPAD(str, len, pad)` | Добавляет символы справа до длины `len`. |

**Примеры:**

```sql
SELECT LPAD('bee', 6, '-');
-- ---bee

SELECT RPAD('bee', 6, '-');
-- bee---

SELECT LPAD('bee', 6, '-+-');
-- -+-bee

SELECT LPAD('verylong', 3, '-');
-- ver
```

**Важное:** если `len` меньше длины исходной строки, MySQL не дополняет, а обрезает результат.

## Удаление пробелов и символов по краям

**Назначение:** очистить строку от лишних пробелов или одинаковых символов по краям.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `LTRIM` | `LTRIM(str)` | Удаляет пробелы только слева. |
| `RTRIM` | `RTRIM(str)` | Удаляет пробелы только справа. |
| `TRIM` | `TRIM([LEADING \| TRAILING \| BOTH] [remstr FROM] str)` | Удаляет пробелы по умолчанию или указанный фрагмент с края строки. |

**Примеры:**

```sql
SELECT LTRIM('   bee');
-- bee

SELECT RTRIM('bee   ');
-- bee

SELECT TRIM('   bee   ');
-- bee

SELECT TRIM(LEADING '-' FROM '--bee--');
-- bee--

SELECT TRIM(TRAILING '-' FROM '--bee--');
-- --bee

SELECT TRIM(BOTH '-' FROM '--bee--');
-- bee
```

**Важное:** `TRIM` не заменяет `REPLACE`. Он работает только с краями строки, а не с серединой.

## Преобразование и модификация

**Назначение:** изменить содержимое строки или порядок символов.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `REPLACE` | `REPLACE(str, from, to)` | Меняет все вхождения `from` на `to`. |
| `REVERSE` | `REVERSE(str)` | Разворачивает строку задом наперед. |
| `REPEAT` | `REPEAT(str, n)` | Повторяет строку `n` раз. |

**Примеры:**

```sql
SELECT REPLACE('beegEek', 'e', 'i');
-- biigEik

SELECT REPLACE('Hello World', 'World', 'SQL');
-- Hello SQL

SELECT REVERSE('beegeek');
-- keegeeb

SELECT REPEAT('bee', 3);
-- beebeebee

SELECT REPEAT('-', 5);
-- -----
```

**Важное:** `REPLACE` не обязан быть нечувствительным к регистру: итог зависит от collation и контекста сравнения, поэтому это лучше проверять на своей базе.

## Практические замечания

- Для очистки данных часто используют связку `TRIM()` + `LOWER()` или `UPPER()`.
- Для разбора строк полезно комбинировать `LOCATE()` и `SUBSTRING()`.
- Для строк с разделителями вроде `name@domain.com` или `a/b/c` особенно удобен `SUBSTRING_INDEX()`.
- `CONCAT_WS()` часто практичнее, чем `CONCAT()`, потому что аккуратно пропускает `NULL`.
- `CHAR_LENGTH()` считает символы, а `LENGTH()` байты.
- Большинство строковых функций при `NULL` возвращают `NULL`, но исключения и нюансы лучше помнить отдельно.
