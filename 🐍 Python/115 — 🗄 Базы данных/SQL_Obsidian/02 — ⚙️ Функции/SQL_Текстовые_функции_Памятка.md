## 📖 УРОВЕНЬ 2 — Подробные объяснения

### Соединение строк

**Назначение:** Объединение нескольких строк в одну.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **CONCAT** | `CONCAT(str1, str2, str3, ...)` | Соединяет все аргументы подряд. Если хотя бы один `NULL` — результат `NULL`. |
| **CONCAT_WS** | `CONCAT_WS(separator, str1, str2, ...)` | Соединяет с разделителем. Пропускает `NULL` значения (не вставляет для них разделитель). |
| **GROUP_CONCAT** | `GROUP_CONCAT(col SEPARATOR ',')` | Объединяет значения одного столбца из разных строк результата. Используется с `GROUP BY`. |

**Примеры:**

```sql
SELECT CONCAT('John', ' ', 'Doe');
-- John Doe

SELECT CONCAT_WS('-', 'USA', 'NY', 'Manhattan');
-- USA-NY-Manhattan

SELECT CONCAT_WS(', ', 'Apple', NULL, 'Banana');
-- Apple, Banana  (NULL пропущен)

SELECT GROUP_CONCAT(name SEPARATOR ', ') FROM users;
-- Peter, Gwen, Flash, Mary
```

**Важное:** `CONCAT` + любой `NULL` = `NULL`. `CONCAT_WS` игнорирует `NULL`.

### Поиск подстроки и анализ длины

**Назначение:** Определить позицию подстроки в строке и вычислить длину строки.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **LOCATE** | `LOCATE(substr, str, [start])` | Возвращает позицию первого вхождения подстроки (нумерация с 1). Если не найдена — 0. Поиск без учёта регистра. |
| **CHAR_LENGTH** | `CHAR_LENGTH(str)` | Возвращает количество символов в строке. Для `NULL` возвращает `NULL`. |

**Примеры:**

```sql
SELECT CHAR_LENGTH('beegeek');
-- 7

SELECT LOCATE('geek', 'beegeek');
-- 4

SELECT LOCATE('e', 'beegeek', 3);  -- поиск с 3-й позиции
-- 4

SELECT LOCATE('xyz', 'beegeek');  -- не найдена
-- 0
```

**Важное:** Нумерация позиций начинается с 1, не с 0. Если подстрока не найдена — возвращается 0.

### Извлечение части строки

**Назначение:** Получить фрагмент строки по позиции, длине или разделителю.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **LEFT** | `LEFT(str, n)` | Возвращает первые n символов. Если n > длины — вся строка. Если n ≤ 0 — пустая строка. |
| **RIGHT** | `RIGHT(str, n)` | То же самое, но последние n символов. |
| **SUBSTRING** | `SUBSTRING(str, start, [len])` | Подстрока начиная с позиции start длиной len. Допускается отрицательный start (отсчёт с конца). |
| **SUBSTRING_INDEX** | `SUBSTRING_INDEX(str, delim, k)` | Часть строки до k-го появления разделителя. Отрицательное k — с конца строки. |

**Примеры:**

```sql
SELECT LEFT('beegeek', 3);
-- bee

SELECT RIGHT('beegeek', 3);
-- eek

SELECT SUBSTRING('beegeek', 4, 2);
-- ge

SELECT SUBSTRING('beegeek', -4);  -- последние 4 символа
-- geek

SELECT SUBSTRING_INDEX('g-e-e-k', '-', 2);
-- g-e

SELECT SUBSTRING_INDEX('g-e-e-k', '-', -1);  -- с конца
-- k
```

**Важное:** SUBSTRING может работать с отрицательными индексами (считает с конца). SUBSTRING_INDEX работает с разделителем, а не позициями.

### Изменение регистра

**Назначение:** Привести текст к одному регистру — нижнему или верхнему.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **LOWER** | `LOWER(str)` | Преобразует все буквы в нижний регистр. Для `NULL` возвращает `NULL`. |
| **UPPER** | `UPPER(str)` | Преобразует все буквы в верхний регистр. Для `NULL` возвращает `NULL`. |

**Примеры:**

```sql
SELECT LOWER('BeeGeek SQL');
-- beegeek sql

SELECT UPPER('BeeGeek SQL');
-- BEEGEEK SQL

SELECT LOWER(NULL);
-- NULL
```

**Важное:** Часто используется для сравнения строк без учёта регистра: `WHERE LOWER(name) = 'john'`

### Работа с ASCII

**Назначение:** Получить ASCII-код символа и наоборот.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **ORD** | `ORD(char)` | Возвращает ASCII-код первого символа строки. Для пустой строки или `NULL` — `NULL`. |
| **CHAR** | `CHAR(code1, code2, ...)` | Возвращает символ(ы) по ASCII-коду. Несколько кодов создают строку. |

**Примеры:**

```sql
SELECT ORD('A');
-- 65

SELECT ORD('Z');
-- 90

SELECT CHAR(65, 66, 67);
-- ABC
```

**Важное:** ORD работает только с первым символом. CHAR часто требует CONVERT для правильного отображения.

### Выравнивание и дополнение

**Назначение:** Искусственно изменить длину строки за счёт добавления символов.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **LPAD** | `LPAD(str, len, pad)` | Дополняет строку слева символами из pad до длины len. Если >len — обрезает справа. |
| **RPAD** | `RPAD(str, len, pad)` | То же самое, но справа добавляет. |

**Примеры:**

```sql
SELECT LPAD('bee', 6, '-');
-- ---bee

SELECT RPAD('bee', 6, '-');
-- bee---

SELECT LPAD('bee', 6, '-+-');
-- -+-bee

SELECT LPAD('verylong', 3, '-');  -- обрезание
-- ver
```

**Важное:** Если len меньше длины исходной строки — происходит обрезание. Pad может быть строка, не только символ.

### Удаление пробелов

**Назначение:** Очистить строки от лишних символов в начале и/или конце.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **LTRIM** | `LTRIM(str)` | Удаляет пробелы только из начала строки. |
| **RTRIM** | `RTRIM(str)` | Удаляет пробелы только из конца строки. |
| **TRIM** | `TRIM([LEADING\|TRAILING\|BOTH] [char FROM] str)` | По умолчанию удаляет пробелы с обоих концов. С параметрами можно удалять другие символы с определённой стороны. |

**Примеры:**

```sql
SELECT LTRIM('   bee');
-- bee

SELECT RTRIM('bee   ');
-- bee

SELECT TRIM('   bee   ');
-- bee

SELECT TRIM(LEADING '-' FROM '--bee--');
-- bee--

SELECT TRIM(TRAILING '-' FROM '--bee--');
-- --bee

SELECT TRIM(BOTH '-' FROM '--bee--');
-- bee
```

**Важное:** Базовый TRIM удаляет только пробелы. Расширенный синтаксис с LEADING/TRAILING может удалять любую подстроку, но только с краёв (не внутри строки).

### Преобразование и модификация

**Назначение:** Изменить структуру или порядок символов в строке.

| Функция | Синтаксис | Описание |
|---------|-----------|----------|
| **REPLACE** | `REPLACE(str, from, to)` | Заменяет все вхождения подстроки from на to. Регистр важен. Если from не найдена — возвращает исходную строку. |
| **REVERSE** | `REVERSE(str)` | Возвращает строку с обратным порядком символов. |
| **REPEAT** | `REPEAT(str, n)` | Повторяет строку n раз. Если n ≤ 0 — пустая строка. |

**Примеры:**

```sql
SELECT REPLACE('beegEek', 'e', 'i');
-- biigEik

SELECT REPLACE('Hello World', 'World', 'SQL');
-- Hello SQL

SELECT REVERSE('beegeek');
-- keegeeb

SELECT REPEAT('bee', 3);
-- beebeebee

SELECT REPEAT('-', 5);
-- -----
```

**Важное:** REPLACE различает регистр: 'e' и 'E' — это разные символы. REVERSE переворачивает весь порядок. REPEAT создаёт много повторений — будь осторожен с большими n.

---

## 💡 Общие советы

- **Для очистки данных:** TRIM() + LOWER()/UPPER() для стандартизации
- **Для анализа:** LOCATE() + SUBSTRING() для работы с частями строк
- **Для форматирования:** LPAD()/RPAD() для выравнивания, REPEAT() для разделителей
- **Для объединения:** CONCAT_WS() чаще удобнее чем CONCAT (автоматически обрабатывает NULL)
- **Помни про NULL:** Большинство функций с NULL возвращают NULL. Исключение — CONCAT_WS
