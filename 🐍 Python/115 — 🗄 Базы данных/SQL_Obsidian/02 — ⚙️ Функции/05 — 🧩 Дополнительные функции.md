## 🔍 Работа с NULL

**Назначение:** проверять `NULL`, подставлять значения по умолчанию и выбирать первое непустое значение.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `ISNULL` | `ISNULL(expr)` | Проверяет, является ли значение `NULL`. |
| `COALESCE` | `COALESCE(a, b, ...)` | Возвращает первое не-`NULL` значение. |
| `IFNULL` | `IFNULL(expr1, expr2)` | Если первое значение `NULL`, возвращает второе. |
| `NULLIF` | `NULLIF(a, b)` | Если значения равны, возвращает `NULL`. |

**Примеры:**

```sql
SELECT ISNULL(NULL);
-- 1

SELECT ISNULL('value');
-- 0

SELECT COALESCE(NULL, NULL, 2, 3);
-- 2

SELECT IFNULL(NULL, 'geek');
-- geek

SELECT NULLIF('bee', 'bee');
-- NULL

SELECT NULLIF('bee', 'geek');
-- bee
```

**Важное:** `COALESCE()` особенно полезна, когда нужно перебрать сразу несколько возможных источников значения. Работает быстрее вложенных `IFNULL()`.




## ❓ Условные функции

**Назначение:** возвращать разные результаты в зависимости от условия.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `IF` | `IF(cond, a, b)` | Возвращает одно из двух значений по условию. |

**Примеры:**

```sql
SELECT IF(1 < 0, 'bee', 'geek');
-- geek

SELECT IF(age >= 18, 'adult', 'minor');
-- разные результаты в зависимости от возраста

SELECT IF(salary IS NULL, 0, salary);
-- подставить 0 если зарплата не указана
```

**Важное:** `IF()` удобна для коротких условий, когда не нужен полный `CASE`. Для множества условий используй `CASE`.




## 🔄 Приведение типов

**Назначение:** явно преобразовывать значение к нужному типу.

| Функция | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `CONVERT` | `CONVERT(expr, type)` | Преобразует значение к типу в стиле MySQL. |
| `CAST` | `CAST(expr AS type)` | Явное преобразование в SQL-стиле. |

**Примеры:**

```sql
SELECT CONVERT(100.78, DECIMAL(4, 1));
-- 100.8

SELECT CAST(100.78 AS DECIMAL(3, 0));
-- 101

SELECT CAST('2023-10-20' AS DATE);
-- 2023-10-20

SELECT CONVERT('123', SIGNED);
-- 123 (число)
```

**Важное:** `CAST()` и `CONVERT()` похожи по назначению, но синтаксис у них разный. `CAST()` предпочтительнее для переносимости кода.




## 📋 Конструкция CASE

**Назначение:** описать несколько условий и выбрать результат по одному из них.

| Вариант | Синтаксис | Что важно помнить |
| --- | --- | --- |
| `CASE expr` | `CASE expr WHEN ... THEN ... END` | Удобен, когда сравнивается одно выражение с разными значениями. |
| `CASE` | `CASE WHEN cond THEN ... END` | Удобен для произвольных логических условий. |

**Примеры:**

Вариант 1: сравнение с выражением

```sql
SELECT
    product_name,
    CASE quantity
        WHEN 0 THEN 'Out of stock'
        WHEN 1 THEN 'One in stock'
        WHEN 2 THEN 'Little in stock'
        WHEN 3 THEN 'Little in stock'
        WHEN 4 THEN 'Little in stock'
        WHEN 5 THEN 'Little in stock'
        ELSE 'A lot in stock'
    END AS availability
FROM products;
```

Вариант 2: логические условия

```sql
SELECT
    product_name,
    CASE
        WHEN quantity = 0 THEN 'Out of stock'
        WHEN quantity = 1 THEN 'One in stock'
        WHEN quantity BETWEEN 2 AND 5 THEN 'Little in stock'
        ELSE 'A lot in stock'
    END AS availability
FROM products;
```

Результат выполнения

```text
+--------------------+
| availability       |
+--------------------+
| A lot in stock     |
| One in stock       |
| Little in stock    |
| Little in stock    |
| Out of stock       |
+--------------------+
```

**Важное:** `CASE` удобнее `IF()`, когда условий больше двух или когда логика должна оставаться читаемой. `CASE` может быть в SELECT, WHERE, ORDER BY и даже в GROUP BY.

## 💡 Практические замечания

- Для обработки `NULL` в выводе используй `COALESCE()` вместо вложенных `IFNULL()` — это читаемее и быстрее.
- `CASE WHEN` работает в индексах и часто оптимизируется лучше, чем `IF()` на больших выборках.
- При приведении типов помни о потере точности: `CAST('abc' AS INT)` вернёт 0, не ошибку.
- `IF()` вычисляет оба варианта — для сложных вычислений лучше использовать `CASE`.
- Используй `NULLIF()` для избежания ошибок деления на ноль: `a / NULLIF(b, 0)`.
- `CAST()` более портативна, чем `CONVERT()` — используй её если планируешь переносить запрос на другие СУБД.
- В больших таблицах `CASE` часто работает быстрее `IF()` благодаря лучшей оптимизации.
