# 📖 Хеш-функции — теория

## 📌 Коротко

> Хеш-функция — преобразование данных **произвольного размера** в число **фиксированного размера** по определённому алгоритму. Результат — хеш, хеш-значение или хеш-код.

---

## 📖 Теория

### Принцип работы

```
"beegeek"  →  d7dd6be4b4a8a123f201ecbcacaf5883f1cec5e0  (SHA-1)
"Beegeek"  →  4f27087e433ab69ec25abfee3eb192980c0887eb  (SHA-1)
```

Одна буква изменилась — хеш кардинально другой. Это **лавинообразный эффект**.

SHA-1 всегда возвращает строку из 40 символов — независимо от размера входных данных.

---

### Коллизия

Коллизия — два **разных** набора данных дают **одинаковый** хеш.

```
hash("abc") → 42
hash("xyz") → 42   ← коллизия!
```

> 💡 Коллизии **неизбежны** — хешей конечное количество, данных бесконечно много. Для хороших функций коллизии сведены к минимуму.

> ⚠️ Из равенства хешей **нельзя** заключить что данные равны. Но если хеши разные — данные точно разные.

---

### Детерминированность

Одни и те же данные → **всегда** одинаковый хеш:

```
hash("hello") → 123  (всегда, при любом запуске)
hash("hello") → 123  (повторно — тот же результат)
```

Следствия:

- `a == b` → `hash(a) == hash(b)` (обязательно)
- `hash(a) == hash(b)` → `a == b` (необязательно, может быть коллизия)

---

## ✅ Характеристики хорошей хеш-функции

|Характеристика|Описание|
|---|---|
|**Лавинность**|Малое изменение входа → большое изменение хеша|
|**Быстрое вычисление**|Работает быстро даже на огромных данных|
|**Минимум коллизий**|Разные данные → разные хеши (насколько возможно)|
|**Равномерное распределение**|Хеши равномерно заполняют весь диапазон|
|**Детерминированность**|Одни данные → всегда один хеш|
|**Ограниченность**|Хеш всегда в заданных пределах|
|**Необратимость**|По хешу нельзя восстановить данные|

> 💡 Реально идеальных хеш-функций не существует — всегда приходится чем-то жертвовать.

---

## 🌍 Области применения

### Проверка целостности данных

Отправитель передаёт данные + хеш. Получатель хеширует данные и сравнивает хеши.

```
Отправитель: данные + hash(данные)
Получатель:  hash(полученные данные) == hash(данные)?
             ДА  → данные не изменились ✅
             НЕТ → данные повреждены или подделаны ❌
```

Хеш используемый для проверки целостности называется **контрольной суммой**.

### Хранение паролей

Сервисы хранят не пароль, а его хеш. При входе сравниваются хеши.

```
Регистрация: пароль → hash(пароль) → БД
Вход:        hash(введённый пароль) == hash из БД?
```

> ⚠️ Если при восстановлении пароля сервис прислал **старый пароль в открытом виде** — он не хеширует пароли. Это очень плохо.

### Ускорение работы с большими данными

Вместо посимвольного сравнения строк — сравнение хешей. Быстрее в разы.

```
Миллион строк → вычислить hash каждой → сравнивать числа, не строки
```

### Хеш-таблицы (dict, set)

Поиск за O(1) через хеш вместо перебора O(n).

## 📉 Сужение диапазона через `%`

Хеши могут быть огромными. Для сужения диапазона используется `%`:

```python
def hash_function(obj):
    return hash(obj) % 100   # диапазон [0; 100)
```

> 💡 В Python `%` с положительным делителем всегда возвращает неотрицательный результат — даже для отрицательных хешей.

> ⚠️ Меньший диапазон → больше коллизий. В качестве делителя рекомендуется большое **простое число**.

## 🔨 Создание своей хеш-функции (эволюция)

### Версия 1 — сумма кодов символов

```python
def hash_function(obj):
    return sum(ord(character) for character in obj)

hash_function('Python')  # → 642
```

**Проблемы:**

- Работает только со строками
- Нечувствительна к порядку символов → коллизии анаграмм

```python
hash_function('Python')  # → 642
hash_function('yPthon')  # → 642  ← коллизия!
hash_function('thonPy')  # → 642  ← коллизия!
```

---

### Версия 2 — через `str()`

```python
def hash_function(obj):
    return sum(ord(character) for character in str(obj))

hash_function('Python')  # → 642
hash_function(123)       # → 150
hash_function(None)      # → 400
```

**Улучшение:** работает с любым типом.

**Проблемы остались:**

- Анаграммы дают одинаковый хеш
- Малое изменение → малое изменение хеша (нет лавинности)

```python
hash_function('Python')  # → 642
hash_function('Pythoo')  # → 643  ← почти то же самое!
hash_function('Pythop')  # → 644
```

---

### Версия 3 — с учётом позиции символа

```python
def hash_function(obj):
    return sum(
        index * ord(character)
        for index, character in enumerate(str(obj), start=1)
        # нумерация с 1, а не с 0 — иначе первый символ всегда отбрасывается
    )

hash_function('Python')  # → 2301
hash_function('yPthon')  # → 2260  ← уже не коллизия!
hash_function('thonPy')  # → 2223
```

**Улучшение:** анаграммы дают разные хеши.

**Проблема:** хеш растёт неограниченно с размером входных данных.

```python
hash_function('Python')        # → 2301
hash_function('Beegeek'*1000)  # → 2380463000
hash_function('Stepik'*10000)  # → 187200003610000000
```

---

### Версия 4 — с ограничением диапазона через `%`

```python
def hash_function(obj):
    return sum(
        index * ord(character)
        for index, character in enumerate(str(obj), start=1)
    ) % 123456761   # большое простое число


hash_function('Python')        # → 2301
hash_function('Beegeek'*1000)  # → 34784541
hash_function('Stepik'*10000)  # → 116170254
```

**Итог:** универсальная, устойчива к анаграммам, ограниченный диапазон.

**Остаётся проблема:** медленная работа на очень больших данных (O(n) по длине).

## ⚡ Встроенная `hash()` vs своя функция

||`hash()`|Наша функция (v4)|
|---|---|---|
|Лавинность|✅|❌ слабая|
|Скорость|✅ очень быстро|❌ O(n)|
|Равномерность|✅|✅ приемлемо|
|Коллизии|минимум|больше|
|Криптостойкость|❌|❌|

> 💡 Для криптографии используют специальные функции из модуля `hashlib` (SHA-256, SHA-3 и др.) — они криптостойкие в отличие от встроенной `hash()`.

---

## ⚠️ Важные ограничения `hash()` в Python

- Хеши строк **меняются между запусками** (рандомизация для защиты от DoS)
- **Никогда не сохраняй** хеши в БД — они нестабильны между запусками и версиями Python
- `hash()` **не криптостойкая** — не используй для паролей
