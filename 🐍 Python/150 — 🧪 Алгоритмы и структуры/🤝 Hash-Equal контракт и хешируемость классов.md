# 🤝 Hash-Equal контракт и хешируемость классов

## 📌 Коротко

> Если два объекта равны — их хеши обязаны совпадать. Переопределил `__eq__` — переопредели `__hash__`. Не сделал этого — Python сам ставит `__hash__ = None` и объект становится нехешируемым.

---

## 📖 Теория

### Поведение по умолчанию

По умолчанию в пользовательском классе:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p1 = Point(1, 2)
p2 = Point(1, 2)

print(p1 == p2)          # → False  (сравнение по id!)
print(p1 is p2)          # → False
print(hash(p1))          # → id(p1) // 16
print(hash(p2))          # → id(p2) // 16  (разные!)
```

- `==` работает как `is` — сравнивает идентичность
- `hash` считается от `id` объекта
- Контракт выполняется автоматически ✅

---

### Hash-Equal контракт

```
если a == b  →  обязательно hash(a) == hash(b)
если hash(a) != hash(b)  →  обязательно a != b

НО:
hash(a) == hash(b)  →  НЕ ОЗНАЧАЕТ a == b  (коллизия!)
```



> 💡 Это фундаментальное требование для работы словарей и множеств. Нарушение контракта — ломает поиск.


1. Если __eq__ и __hash__ не определены — сравнение и хеш
   основаны на идентичности объекта (id // 16)

2. Если определён __eq__, но не определён __hash__ — 
   сравнение работает по своей логике, а __hash__ 
   автоматически становится None, чтобы не нарушить контракт

3. Если определены оба — равные объекты обязаны 
   иметь равные хеш-значения (Hash-Equal контракт)

---

### Почему переопределение `__eq__` ломает контракт

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if isinstance(other, Point):
            return self.x == other.x and self.y == other.y
        return NotImplemented
    # __hash__ не переопределён!

p1 = Point(1, 2)
p2 = Point(1, 2)

print(p1 == p2)              # → True   (наш __eq__ — равны!)
print(hash(p1) == hash(p2))  # → False  (hash от id — разные!)
```

**Контракт нарушен** — равные объекты имеют разные хеши!

```
                    ДО переопределения __eq__:
== → сравнивает id
hash → считается от id
→ контракт ✅

                    ПОСЛЕ переопределения __eq__:
== → сравнивает значения (p1 == p2 → True)
hash → всё ещё от id (hash(p1) != hash(p2))
→ контракт нарушен ❌
```

---

### Что делает Python если переопределён `__eq__` но не `__hash__`

Python **автоматически** ставит `__hash__ = None`:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if isinstance(other, Point):
            return self.x == other.x and self.y == other.y
        return NotImplemented
    # __hash__ не определён!

print(Point.__hash__)   # → None
hash(Point(1, 2))       # → TypeError: unhashable type: 'Point'
```

> 💡 Python защищает от случайного нарушения контракта — "ты переопределил равенство но не сказал как считать хеш, я не могу гарантировать контракт, поэтому запрещаю хешировать вообще".

Можно явно сделать класс нехешируемым:

```python
class Point:
    __hash__ = None   # явно запрещаем хеширование
```

---

### Правильная реализация `__hash__`

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if isinstance(other, Point):
            return self.x == other.x and self.y == other.y
        return NotImplemented

    def __hash__(self):
        return hash((self.x, self.y))   # хеш кортежа из атрибутов


p1 = Point(1, 2)
p2 = Point(1, 2)

print(p1 == p2)              # → True
print(hash(p1) == hash(p2))  # → True  ← контракт выполняется ✅

# Теперь можно использовать как ключ:
d = {p1: 'point'}
print(d[p2])   # → 'point' ✅
```

> 💡 Хеш кортежа из атрибутов — стандартная практика. Альтернатива — `@property` возвращающий кортеж атрибутов:

```python
@property
def _fields(self):
    return self.x, self.y

def __eq__(self, other):
    return self._fields == other._fields

def __hash__(self):
    return hash(self._fields)
```

---

### Три правила хешируемости

**Правило 1 — не переопределил `__eq__`:**

```
Не трогай __hash__ — всё работает по умолчанию ✅
```

**Правило 2 — изменяемый класс переопределил `__eq__`:**

```
НЕ переопределяй __hash__!
Python сам поставит __hash__ = None → объект нехешируемый
Причина: хеш изменяемого объекта нестабилен → сломает словарь
```

**Правило 3 — неизменяемый класс переопределил `__eq__`:**

```
Переопредели __hash__ → объект хешируемый и правильно работает в dict/set ✅
```

---

### Почему изменяемый объект не должен быть хешируемым

```python
p = Point(1, 2)
d = {p: 'value'}    # добавили — hash((1, 2)) → корзина 5

p.x = 99            # изменили атрибут!

d[p]                # ищем — hash((99, 2)) → корзина 7 → KeyError! ❌
# Пара осталась в словаре но найти её невозможно — утечка памяти!
```

> ⚠️ Изменение хешируемого объекта после добавления в словарь — пара "теряется". Найти её уже нельзя.

---

### Изменяемость пользовательского класса

**Изменяемый** — методы изменяют текущий экземпляр, `id` не меняется:

```python
class Cat:
    def __init__(self):
        self.things = []

    def append(self, thing):
        self.things.append(thing)   # меняем текущий объект

cat = Cat()
id1 = id(cat)
cat.append('meow')
id2 = id(cat)
print(id1 == id2)   # → True  ← тот же объект
```

**Неизменяемый** — методы возвращают новый экземпляр, `id` меняется:

```python
class Dog:
    def __init__(self, things=None):
        self.things = things or []

    def append(self, thing):
        return Dog(self.things + [thing])   # возвращаем НОВЫЙ объект

dog = Dog()
id1 = id(dog)
dog = dog.append('bark')
id2 = id(dog)
print(id1 == id2)   # → False  ← новый объект
```

> ⚠️ Настоящая неизменяемость — только у встроенных типов (str, tuple). В пользовательских классах это **соглашение**, которое нужно соблюдать самому.

## 📊 Итоговая таблица

|Ситуация|`__eq__`|`__hash__`|Результат|
|---|---|---|---|
|Ничего не переопределено|от `object` (по `id`)|от `object` (`id // 16`)|хешируемый ✅|
|Только `__eq__` переопределён|своя логика|`None` (автоматически)|нехешируемый ❌|
|Оба переопределены (неизменяемый)|своя логика|своя логика|хешируемый ✅|
|Изменяемый + только `__eq__`|своя логика|`None`|нехешируемый ❌ (правильно!)|

## 💻 Быстрая шпаргалка

```python
class MyClass:
    def __init__(self, value):
        self.value = value

    def __eq__(self, other):
        if isinstance(other, MyClass):
            return self.value == other.value
        return NotImplemented

    def __hash__(self):
        return hash(self.value)   # или hash((self.attr1, self.attr2))


# Контракт:
# a == b  →  hash(a) == hash(b)  ← ОБЯЗАТЕЛЬНО
# hash(a) == hash(b)  →  a == b  ← НЕ ОБЯЗАТЕЛЬНО (коллизия!)

# Нельзя хешировать изменяемый объект:
# class Mutable:
#     def __eq__(self): ...
#     # __hash__ = None  ← Python поставит автоматически
```
