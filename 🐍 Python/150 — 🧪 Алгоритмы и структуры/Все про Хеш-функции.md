# #️⃣ Хеш-функции

## 📌 Коротко

> Хеш-функция — **одностороннее преобразование** данных любого размера в число фиксированного размера. Используется в `dict` и `set` для поиска за O(1). Обратно восстановить исходные данные **невозможно**.

-----

## 📖 Теория

### Что такое хеш-функция

Хеш-функция берёт данные любого размера и возвращает **целое число фиксированного размера**:

```
"hello"           → 1234567890
"hello world"     → 9876543210
[1, 2, 3, ...]    → 1111111111
```

Размер входа не важен — хеш всегда одного размера. В CPython на 64-битной системе — 64-битное целое число.

-----

### Ключевые свойства

|Свойство                |Описание                                      |
|------------------------|----------------------------------------------|
|**Одностороннее**       |По хешу нельзя восстановить исходные данные   |
|**Детерминированное**   |Одни и те же данные → всегда один и тот же хеш|
|**Фиксированный размер**|Хеш всегда одного размера, независимо от входа|
|**Лавинный эффект**     |Маленькое изменение входа → сильно другой хеш |

```python
hash("hello")    # → 1234567890  (всегда одно и то же в рамках сессии)
hash("hellо")    # → 9999999999  (одна буква другая → совсем другой хеш)
```

> ⚠️ В Python хеш **меняется между запусками** — это защита от атак (hash randomization). Внутри одной сессии — стабильный.

-----

### Коллизия

Коллизия — два разных объекта дают **одинаковый хеш**. Это нормально и неизбежно — хешей конечное количество, объектов бесконечно много.

```
hash("abc") % 8 = 3
hash("xyz") % 8 = 3  ← коллизия!
```

Python решает коллизию через **зондирование** (probing) — ищет следующий свободный слот по специальной формуле, пока не найдёт место.

-----

### Хешируемые и нехешируемые объекты

Объект хешируемый если он **неизменяемый**. Если объект изменится после добавления в словарь — его хеш изменится и Python не сможет его найти.

|Хешируемые ✅                             |Нехешируемые ❌                  |
|-----------------------------------------|--------------------------------|
|`int`, `float`, `bool`                   |`list`                          |
|`str`                                    |`dict`                          |
|`tuple` (если содержит только хешируемые)|`set`                           |
|`frozenset`                              |`tuple` с изменяемыми элементами|

```python
hash(42)          # ✅ → число
hash("hello")     # ✅ → число
hash((1, 2, 3))   # ✅ → число
hash([1, 2, 3])   # ❌ → TypeError: unhashable type: 'list'
```

## 🗃 Как хеш используется в dict

### Структура словаря

Словарь — это **хеш-таблица** с двумя уровнями:

```
┌─────────────────────────────────────────┐
│  УРОВЕНЬ 1: Индексная таблица           │
│  слот: 0    1    2    3    4    5    6   │
│        [-1] [2] [-1] [0] [-1] [-1] [1]  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  УРОВЕНЬ 2: Массив записей              │
│  поз. │    хеш     │  ключ  │ значение  │
│   0   │ hash(name) │ 'name' │ 'John'    │
│   1   │ hash(age)  │ 'age'  │  25       │
│   2   │ hash(city) │ 'city' │ 'Moscow'  │
└─────────────────────────────────────────┘
```

### Добавление ключа

```
d['name'] = 'John'

Шаг 1: hash('name') → большое число, например 12345678
Шаг 2: 12345678 % размер_таблицы → индекс слота = 3
Шаг 3: слот 3 свободен?
        ДА → записать индекс → в массив записей добавить [hash | 'name' | 'John']
        НЕТ → коллизия → зондирование → следующий свободный слот
```

### Поиск ключа — почему O(1)

```
d['age']

Шаг 1: hash('age') % размер_таблицы → индекс слота = 6
Шаг 2: индексная_таблица[6] = 1
Шаг 3: массив_записей[1] → проверяем хеши → совпадают
Шаг 4: 'age' == 'age' через __eq__ → True
Шаг 5: возвращаем 25 ✅
```

> 💡 Прямой доступ по индексу — как в массиве. Не нужно перебирать все элементы → O(1).

> ⚠️ Сначала сравниваются **хеши** (быстро, числа), только потом `__eq__` (медленнее). Оптимизация скорости.

### Состояния слотов

|Значение|Смысл                           |
|--------|--------------------------------|
|`-1`    |пустой, никогда не использовался|
|`-2`    |**dummy** — был удалён          |
|`N`     |индекс в массиве записей        |


> ⚠️ При удалении ставится `-2` (dummy), а не `-1`! Иначе цепочка зондирования прервётся и поиск не найдёт ключи сдвинутые из-за коллизий.

### Перестройка (resize)

Когда заполненность достигает **~⅔ от размера таблицы** — Python создаёт новую таблицу большего размера и перехеширует все элементы. Dummy-слоты при этом исчезают.

## 🧩 Как хеш используется в set

`set` устроен аналогично `dict` — та же хеш-таблица, только без значений. Хранит только ключи.

```python
s = {1, 2, 3}

# Проверка вхождения — O(1):
1 in s   # hash(1) → слот → проверка → True

# Добавление — O(1):
s.add(4)  # hash(4) → свободный слот → записать

# Операции над множествами используют хеши для быстрого сравнения:
{1, 2, 3} & {2, 3, 4}   # → {2, 3}
```

> 💡 Именно поэтому элементы `set` должны быть хешируемыми — `set` из списков невозможен.

## 🔧 hash() в Python

```python
# Встроенная функция hash()
hash(42)          # → 42  (для int хеш == само число, почти всегда)
hash(3.14)        # → число
hash("hello")     # → число (меняется между запусками!)
hash((1, 2))      # → число
hash(True)        # → 1
hash(False)       # → 0

# Важно: hash(True) == hash(1) == 1
# Поэтому True и 1 — один ключ в словаре!
d = {True: "yes", 1: "no"}
print(d)  # → {True: 'no'}  ← 1 перезаписал True!
```

### **hash** и **eq**

Если определяешь `__eq__` в своём классе — **обязательно** определи и `__hash__`. Иначе Python автоматически ставит `__hash__ = None` и объект становится нехешируемым.

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

    def __hash__(self):
        return hash((self.x, self.y))  # кортеж из атрибутов — простой способ


p1 = Point(1, 2)
p2 = Point(1, 2)

print(p1 == p2)       # → True
print(hash(p1) == hash(p2))  # → True
d = {p1: "point"}
print(d[p2])          # → "point" ✅
```

> ⚠️ Правило: если `a == b`, то обязательно `hash(a) == hash(b)`. Обратное не обязано быть правдой (коллизии допустимы).

## ⚡ Производительность

|Операция      |dict / set|list                |
|--------------|----------|--------------------|
|Поиск элемента|O(1)      |O(n)                |
|Добавление    |O(1)      |O(1) амортизированно|
|Удаление      |O(1)      |O(n)                |


> 💡 Именно поэтому `x in set` работает мгновенно даже для миллиона элементов, а `x in list` — медленно.

## 💻 Быстрая шпаргалка

```python
# Хеш любого хешируемого объекта
hash("hello")        # строка
hash(42)             # число
hash((1, 2, 3))      # кортеж
hash(frozenset([1])) # frozenset

# Нельзя хешировать изменяемые:
hash([1, 2])   # ❌ TypeError
hash({1: 2})   # ❌ TypeError
hash({1, 2})   # ❌ TypeError

# Использование как ключ словаря:
d = {(1, 2): "point"}  # ✅ кортеж — ок
d = {[1, 2]: "point"}  # ❌ список — нельзя

# Элемент множества:
s = {(1, 2), (3, 4)}   # ✅
s = {[1, 2], [3, 4]}   # ❌

# Свой класс с хешем:
class MyClass:
    def __init__(self, val):
        self.val = val
    def __eq__(self, other):
        return self.val == other.val
    def __hash__(self):
        return hash(self.val)
```
