# 🔍 Пробирование и удаление — пошагово

-----

## Шаг 1 — Вставка нового элемента

```
1. Вычисляем индекс: hash(key) % размер_таблицы
2. Смотрим на ячейку по этому индексу:
   - Свободна (None/EMPTY)? → вставляем сюда ✅
   - Занята? → переходим к шагу 3
```

-----

## Шаг 2 — Коллизия → пробирование

```
Ячейка занята — ищем следующую свободную:

следующий_индекс = (текущий + 1) % размер_таблицы

Повторяем пока не найдём свободную ячейку.
Движение циклично — дойдя до конца, идём в начало.

[None, None, None, 1, 2, 3, 4, None, None]
                   ↑  ↑  ↑  ↑  ↑
                   3  4  5  6  7 — проверяем по очереди
                                ↑
                             свободно → вставляем 5
```

-----

## Шаг 3 — Поиск элемента

```
1. Вычисляем индекс: hash(key) % размер_таблицы
2. Смотрим на ячейку:
   - None (EMPTY)  → элемента нет, возвращаем False ❌
   - DUMMY         → здесь что-то было, но удалено → продолжаем поиск
   - Занята        → сравниваем ключи через __eq__:
                       совпали → нашли! ✅
                       не совпали → продолжаем поиск
```

-----

## Шаг 4 — Удаление элемента (почему не None)

```
❌ Наивный способ — заменить на None:

До:    [None, None, None, 1, 2, 3, 4, 5, None]
После: [None, None, None, 1, 2, None, 4, 5, None]
                               ↑
                          разрыв цепочки!

Теперь ищем 4:
Индекс 3 → не то
Индекс 4 → не то
Индекс 5 → None → стоп, "не найдено" ❌
Но 4 есть в таблице! Цепочка сломана.
```

```
✅ Правильный способ — заменить на DUMMY:

После: [None, None, None, 1, 2, DUMMY, 4, 5, None]
                               ↑
                          не останавливаемся!

Теперь ищем 4:
Индекс 3 → не то
Индекс 4 → не то
Индекс 5 → DUMMY → продолжаем!
Индекс 6 → нашли 4 ✅
```

-----

## Шаг 5 — Три состояния ячейки

```
None / EMPTY  →  никогда не использовалась
                 при поиске — СТОП, элемента нет

DUMMY         →  была удалена
                 при поиске — ПРОДОЛЖАЕМ

Занятая       →  содержит данные
                 при поиске — сравниваем __eq__
```

-----

## Шаг 6 — Перестройка таблицы (уборка DUMMY)

```
При заполнении ~⅔ таблицы Python перестраивает её:
1. Создаёт новую таблицу большего размера
2. Перехеширует все живые элементы заново
3. DUMMY-слоты исчезают — таблица чистая
```

-----

## ⚠️ Минус линейного пробирования — кластеры

```
[None, None, 1, 2, 3, 4, 5, None, None]
             ↑_______________↑
                  кластер

Чем больше кластер → тем дольше поиск
В худшем случае → O(n) вместо O(1)

CPython использует улучшенную формулу пробирования
чтобы минимизировать кластеры.
```
