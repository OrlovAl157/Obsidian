# 🧩 DefaultObject — объект с динамическими атрибутами и значением по умолчанию

## 📌 Суть

> Класс принимает произвольное количество атрибутов через `**kwargs` и возвращает значение по умолчанию для любого несуществующего атрибута вместо `AttributeError`.

---

## 💻 Код

```python
class DefaultObject:
    def __init__(self, default=None, **kwargs):
        self.default = default
        for name, value in kwargs.items():
            setattr(self, name, value)       # динамически устанавливаем атрибуты

    def __getattribute__(self, attr):
        return object.__getattribute__(self, attr)

    def __getattr__(self, attr):
        return self.default                  # запасной план — атрибут не найден

    def __setattr__(self, attr, value):
        return object.__setattr__(self, attr, value)
```

---

## 🔍 Разбор по частям

### `__init__` — динамические атрибуты через `**kwargs`

```python
obj = DefaultObject(default=0, name='Саша', age=25, city='Москва')
```

`**kwargs` собирает все именованные аргументы в словарь:

```python
kwargs = {'name': 'Саша', 'age': 25, 'city': 'Москва'}
```

Затем `setattr` в цикле устанавливает каждый как атрибут объекта:

```python
for name, value in kwargs.items():
    setattr(self, name, value)
# эквивалентно:
# self.name = 'Саша'
# self.age = 25
# self.city = 'Москва'
```

> 💡 `setattr(obj, 'name', value)` — это то же что `obj.name = value`, но имя атрибута передаётся строкой. Позволяет устанавливать атрибуты динамически.

---

### `__getattribute__` — стандартное поведение

```python
def __getattribute__(self, attr):
    return object.__getattribute__(self, attr)
```

Просто делегирует в `object` — стандартное поведение без изменений. Определён явно для наглядности.

---

### `__getattr__` — значение по умолчанию

```python
def __getattr__(self, attr):
    return self.default
```

Вызывается **только если атрибут не найден**. Вместо `AttributeError` возвращает `self.default`.

```python
obj = DefaultObject(default=0, name='Саша')

print(obj.name)      # → 'Саша'  (__getattribute__ нашёл атрибут)
print(obj.age)       # → 0       (__getattr__ — атрибута нет, вернул default)
print(obj.anything)  # → 0       (__getattr__ — атрибута нет, вернул default)
```

---

### `__setattr__` — стандартное поведение

```python
def __setattr__(self, attr, value):
    return object.__setattr__(self, attr, value)
```

Тоже делегирует в `object`. Определён явно — показывает что установка атрибутов контролируется.

---

## 🔄 Цепочка вызовов

```
obj.name
  ↓
__getattribute__("name")
  ↓
атрибут найден в __dict__? → ДА → вернуть 'Саша'
                           → НЕТ → AttributeError
                                     ↓
                                 __getattr__("name")
                                     ↓
                                 вернуть self.default
```

## 🎯 Практическое применение

```python
# Конфигурация с дефолтами:
config = DefaultObject(
    default='не задано',
    host='localhost',
    port=8080
)

print(config.host)     # → 'localhost'
print(config.port)     # → 8080
print(config.timeout)  # → 'не задано'
print(config.debug)    # → 'не задано'

# JSON-подобный объект:
data = DefaultObject(default=None, name='Alice', role='admin')
print(data.name)       # → 'Alice'
print(data.email)      # → None  (вместо KeyError как в словаре)
```

## 🔀 Альтернативные реализации `__init__`

Все три варианта делают одно и то же — устанавливают атрибуты из `kwargs`:

```python
# Вариант 1 — через setattr() (встроенная функция):
def __init__(self, default=None, **kwargs):
    self.default = default
    for name, value in kwargs.items():
        setattr(self, name, value)          # вызывает __setattr__

# Вариант 2 — через self.__setattr__() (явный вызов магического метода):
def __init__(self, default=None, **kwargs):
    self.default = default
    for a, v in kwargs.items():
        self.__setattr__(a, v)              # то же самое, но напрямую

# Вариант 3 — через __dict__.update() (самый быстрый):
def __init__(self, default=None, **kwargs):
    self.default = default
    self.__dict__.update(kwargs)            # обходит __setattr__ полностью!
```

|Вариант|Вызывает `__setattr__`?|Скорость|Когда использовать|
|---|---|---|---|
|`setattr()`|✅ да|средняя|стандартный случай|
|`self.__setattr__()`|✅ да|средняя|явный вызов для читаемости|
|`__dict__.update()`|❌ нет|быстрее|нет кастомной логики в `__setattr__`|

> ⚠️ `__dict__.update()` обходит `__setattr__` — если там есть валидация или логирование, она не сработает!

