

## 🎯 Суть

Есть два главных паттерна:

1. **Срезы с присваиванием** (`list[:] = new_data`) — переписываем содержимое списка, не переопределяя сам объект
2. **Методы, очищающие и расширяющие** (`clear()` + `extend()`) — изменяем список изнутри через встроенные методы

---

## 🔄 Способ 1: Срез `[:]` для переписывания содержимого

### 📌 Как это работает

```python
old_list = [1, 2, 3, 4, 5]
new_list = ['a', 'b', 'c']

old_list[:] = new_list
# Результат: old_list = ['a', 'b', 'c']
```

### 🧩 Цепочка вызовов

```
old_list[:] = new_list
    ↓
вызывается __setitem__(slice(None, None, None), ['a', 'b', 'c'])
    ↓
список ПЕРЕПИСЫВАЕТСЯ на месте
    ↓
прежнее содержимое полностью заменяется
    ↓
объект старается остаётся тем же! id(old_list) не меняется
```

### 🔑 Ключевое отличие от переприсваивания

```python
# ❌ Переприсваивание — создаём новый объект:
old_list = [1, 2, 3, 4, 5]
new_list = ['a', 'b', 'c']
old_list = new_list  # old_list ↦ другой объект!
print(id(old_list))  # id изменился

# ✅ Срез — модифицируем существующий объект:
old_list = [1, 2, 3, 4, 5]
new_list = ['a', 'b', 'c']
old_list[:] = new_list  # старый объект переписан!
print(id(old_list))  # id ТОТ ЖЕ
```

### 💡 Почему это полезно?

Если другие переменные ссылаются на старый список — они **видят новое содержимое**:

```python
main_list = [1, 2, 3]
backup = main_list  # обе переменные ↦ один объект

main_list[:] = ['x', 'y', 'z']

print(main_list)  # ['x', 'y', 'z']
print(backup)     # ['x', 'y', 'z'] ← тоже изменился!
print(id(main_list) == id(backup))  # True
```

### 🎯 Практические примеры

```python
# Пример 1: Обновление списка в функции
def refill_data(target_list, new_data):
    target_list[:] = new_data

original = [1, 2, 3]
refill_data(original, [10, 20, 30])
print(original)  # [10, 20, 30] ← исходный список изменён!

# Пример 2: Реверс на месте
numbers = [1, 2, 3, 4, 5]
numbers[:] = numbers[::-1]
print(numbers)  # [5, 4, 3, 2, 1]

# Пример 3: Фильтр на месте
data = [1, 2, 3, 4, 5, 6]
data[:] = [x for x in data if x % 2 == 0]
print(data)  # [2, 4, 6]

# Пример 4: Трансформация на месте
items = [1, 2, 3]
items[:] = [x * 10 for x in items]
print(items)  # [10, 20, 30]
```

### ⚡ Под капотом

Срез вызывает `__setitem__`:

```python
class TrackedList(list):
    def __setitem__(self, key, value):
        print(f"__setitem__ вызван: key={key}, value={value}")
        super().__setitem__(key, value)

tl = TrackedList([1, 2, 3])
tl[:] = [10, 20, 30]

# Выведет:
# __setitem__ вызван: key=slice(None, None, None), value=[10, 20, 30]
```

## 🔧 Способ 2: clear() + extend() для трансформации списка

### 📌 Как это работает

```python
class AdvancedList(list):
    def __init__(self, iterable=(), default=None):
        super().__init__(item for item in iterable)
        self._default = default

    def extend_self(self, data):
        """Очищаем и расширяем — переписываем список на месте"""
        self.clear()    # удаляем всё содержимое
        self.extend(data)  # добавляем новое

    def map(self, func):
        """Преобразуем каждый элемент"""
        new_data = list(func(item) for item in self)
        self.extend_self(new_data)  # ← переписываем!

    def filter(self, predicate):
        """Оставляем только подходящие элементы"""
        new_data = list(filter(predicate, self))
        self.extend_self(new_data)  # ← переписываем!
```

### 🧩 Цепочка вызовов

```
self.extend_self([new_data])
    ↓
self.clear()
    ↓
удаляются все элементы (но объект остаётся)
    ↓
self.extend([new_data])
    ↓
добавляются новые элементы в пустой список
    ↓
результат: переписанный на месте список
```

### 🎯 Практические примеры

```python
# Инициализация
nums = AdvancedList([1, 2, 3, 4, 5])

# Пример 1: map — преобразование всех элементов
nums.map(lambda x: x ** 2)
print(nums)  # [1, 4, 9, 16, 25]

# Пример 2: filter — оставляем только чётные
nums.filter(lambda x: x % 2 == 0)
print(nums)  # [4, 16]

# Пример 3: цепочка операций
data = AdvancedList([1, 2, 3, 4, 5])
data.map(lambda x: x * 10)      # [10, 20, 30, 40, 50]
print(data)
data.filter(lambda x: x > 25)   # [30, 40, 50]
print(data)
```

### 💡 Почему clear() + extend() вместо присваивания?

```python
# ❌ Если бы использовали присваивание:
def map_wrong(self, func):
    new_data = list(func(item) for item in self)
    self = new_data  # ← НЕ работает! self = новый объект в localsope
    # исходный список не изменился!

# ✅ clear() + extend() — работает:
def map_right(self, func):
    new_data = list(func(item) for item in self)
    self.clear()      # ← модифицируем существующий объект
    self.extend(new_data)  # ← добавляем новое в него
```

## 📊 Сравнение методов

|Способ|Синтаксис|Когда использовать|Тип изменения|
|---|---|---|---|
|Срез `[:]`|`list[:] = new_data`|Быстро переписать содержимое|Полная замена|
|clear() + extend()|`list.clear(); list.extend(data)`|Сложная логика, методы класса|Плавная замена|
|Присваивание `=`|`list = new_data`|Создать новую ссылку|Создание нового объекта|
|Список методы|`list.append()`, `list.pop()`|Добавление/удаление элементов|Пошаговое изменение|

## 🎨 Расширенные варианты

### Вариант 1: Список с "историей" изменений

```python
class HistoryList(list):
    def __init__(self, iterable=()):
        super().__init__(iterable)
        self.history = [list(self)]  # сохраняем начальное состояние

    def transform(self, func):
        """Трансформируем и сохраняем в историю"""
        new_data = list(func(item) for item in self)
        self.clear()
        self.extend(new_data)
        self.history.append(list(self))  # сохраняем новое состояние

    def undo(self):
        """Вернуться к предыдущему состоянию"""
        if len(self.history) > 1:
            self.history.pop()  # удаляем текущее
            state = self.history[-1]  # берём предыдущее
            self[:] = state  # переписываем на месте

# Использование
hl = HistoryList([1, 2, 3, 4, 5])
hl.transform(lambda x: x ** 2)
print(hl)  # [1, 4, 9, 16, 25]
print(hl.history)  # [[1,2,3,4,5], [1,4,9,16,25]]

hl.undo()
print(hl)  # [1, 2, 3, 4, 5]
```

### Вариант 2: Ленивый список (изменения по требованию)

```python
class LazyList(list):
    def __init__(self, iterable=()):
        super().__init__(iterable)
        self.pending_transforms = []

    def queue_transform(self, func):
        """Добавляем трансформацию в очередь"""
        self.pending_transforms.append(func)

    def apply_all(self):
        """Применяем все накопленные трансформации"""
        current = list(self)
        for func in self.pending_transforms:
            current = [func(x) for x in current]
        self[:] = current  # переписываем на месте
        self.pending_transforms.clear()

# Использование
ll = LazyList([1, 2, 3, 4, 5])
ll.queue_transform(lambda x: x * 2)      # не применили!
ll.queue_transform(lambda x: x + 10)     # добавили в очередь
print(ll)  # [1, 2, 3, 4, 5] ← ещё не изменился

ll.apply_all()
print(ll)  # [12, 14, 16, 18, 20]
```

### Вариант 3: Валидирующий список (проверка перед изменением)

```python
class ValidatedList(list):
    def __init__(self, iterable=(), validator=None):
        super().__init__(iterable)
        self.validator = validator or (lambda x: True)

    def update_with_validation(self, new_data):
        """Переписываем только если все элементы валидны"""
        if not all(self.validator(item) for item in new_data):
            raise ValueError("Некоторые элементы не прошли валидацию")
        self[:] = new_data

    def safe_transform(self, func):
        """Трансформируем с проверкой результата"""
        new_data = [func(x) for x in self]
        self.update_with_validation(new_data)

# Использование
def only_positive(x):
    return x > 0

vl = ValidatedList([1, 2, 3], validator=only_positive)

# ✅ Работает
vl.safe_transform(lambda x: x * 2)
print(vl)  # [2, 4, 6]

# ❌ Ошибка — результат содержит отрицательное число
try:
    vl.safe_transform(lambda x: x - 10)
except ValueError as e:
    print(f"Ошибка: {e}")
```

### Вариант 4: Список с "snapshots"

```python
class SnapshotList(list):
    def __init__(self, iterable=()):
        super().__init__(iterable)
        self.snapshots = {}

    def save_snapshot(self, name):
        """Сохраняем текущее состояние под именем"""
        self.snapshots[name] = list(self)

    def restore_snapshot(self, name):
        """Восстанавливаем сохранённое состояние"""
        if name not in self.snapshots:
            raise KeyError(f"Снимок '{name}' не найден")
        self[:] = self.snapshots[name]

    def list_snapshots(self):
        """Показываем все сохранённые снимки"""
        return list(self.snapshots.keys())

# Использование
sl = SnapshotList([1, 2, 3, 4, 5])
sl.save_snapshot('original')

sl[:] = [x * 2 for x in sl]
print(sl)  # [2, 4, 6, 8, 10]
sl.save_snapshot('doubled')

sl[:] = [x + 100 for x in sl]
print(sl)  # [102, 104, 106, 108, 110]

sl.restore_snapshot('original')
print(sl)  # [1, 2, 3, 4, 5]
print(sl.list_snapshots())  # ['original', 'doubled']
```

### Вариант 5: Декоратор для auto-transform

```python
def auto_update(method):
    """Декоратор — автоматически обновляет список после метода"""
    def wrapper(self, *args, **kwargs):
        result = method(self, *args, **kwargs)
        if result is not None:  # если метод вернул новое содержимое
            self[:] = result
        return self  # возвращаем сам список для цепочки
    return wrapper

class FluentList(list):
    @auto_update
    def squared(self):
        return [x ** 2 for x in self]

    @auto_update
    def filtered(self, predicate):
        return [x for x in self if predicate(x)]

    @auto_update
    def mapped(self, func):
        return [func(x) for x in self]

# Использование — цепочка методов!
fl = FluentList([1, 2, 3, 4, 5])
fl.squared().filtered(lambda x: x > 10).mapped(lambda x: x // 4)
print(fl)  # [0, 1, 1, 1] — результат трёх операций подряд
```

## 🔗 Связь со скриптом об `__getattribute__`

Оба подхода используют **базовые методы object**:

```python
# В памятке об object.__getattribute__:
object.__getattribute__(self, '__dict__').update(kwargs)
                                     # ↑ тоже модифицируем на месте!

# Здесь:
self[:] = new_data  # вызывает __setitem__ → базовую реализацию
self.clear()        # вызывает clear() → базовую реализацию list
```

**Суть одна**: модифицируем существующий объект вместо создания нового.

## 📋 Шпаргалка

```python
# ✅ Переписать список на месте
my_list[:] = new_data

# ✅ Очистить и расширить
my_list.clear()
my_list.extend(new_data)

# ✅ В методе класса
def update(self, data):
    self[:] = data  # или self.clear(); self.extend(data)

# ❌ Не работает (создаёт новый объект):
my_list = new_data

# ❌ Не работает в методе (меняет локальную переменную):
def update_wrong(self, data):
    self = data  # ← только в этой функции!
```

## ❓ Резюме

|Вопрос|Ответ|
|---|---|
|Зачем `[:]` вместо `=`?|Чтобы изменить содержимое объекта, не переопределяя саму ссылку|
|Когда использовать clear() + extend()?|Когда нужна логика в методе класса или сложные трансформации|
|Что быстрее?|`[:]` немного быстрее (одна операция), но различие минимально|
|Применим ли на кортежах?|Нет! Кортежи неизменяемы. Только на list|
|А на dict или set?|Нет встроенной операции, нужно переписывать по-другому|
