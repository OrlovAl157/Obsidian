# 🌍 Памятка: Unicode, UTF-8 и кодировки

`Unicode` — это всемирная таблица символов (символ ↔ номер). `UTF-8` — один из способов превратить этот номер в последовательность байт для хранения и передачи.

### 💡 Ключевые особенности

- **Unicode** — это просто таблица: `символ ↔ номер (code point)`. Не знает ничего о байтах.
- **UTF-8** — способ кодирования: превращает Unicode-номер в последовательность от 1 до 4 байт.
- **`ord()`** — символ → Unicode-номер. **`chr()`** — номер → символ.
- **`encode()`** — строка → байты. **`decode()`** — байты → строка.
- **`len()` строки** считает символы (code points), а не байты — это разные числа для не-ASCII текста.
- **Неправильная кодировка** при decode может дать как "кракозябры", так и `UnicodeDecodeError` — в зависимости от того, является ли байтовая последовательность вообще валидной в этой кодировке.

### 💡 Полезные советы

- UTF-8 — стандарт по умолчанию в Python 3 и в интернете. Используй его, если нет особых причин для другого.
- Для подсчёта реального размера текста в памяти/при передаче — используй `len(text.encode('utf-8'))`, а не `len(text)`.
- При работе с файлами всегда указывай кодировку явно: `open(file, encoding='utf-8')`, не полагайся на умолчания ОС.

### 🚀 Быстрая справка

| **Функция/метод** | **Что делает** | **Пример** |
|---|---|---|
| `ord(c)` | символ → номер | `ord('A')` → `65` |
| `chr(n)` | номер → символ | `chr(65)` → `'A'` |
| `str.encode(enc)` | строка → байты | `'A'.encode('utf-8')` → `b'A'` |
| `bytes.decode(enc)` | байты → строка | `b'A'.decode('utf-8')` → `'A'` |
| `len(str)` | количество символов | `len('😀')` → `1` |
| `len(bytes)` | количество байт | `len('😀'.encode())` → `4` |

## 🔤 Зачем нужны кодировки

Компьютер хранит всё как биты и байты: `01010101`. Но человеку нужны буквы, цифры, символы и эмодзи. Кодировки — это правила, которые переводят между текстом и числами: **символ ↔ число**.

---

## 📌 ASCII

**Назначение:** одна из первых таблиц символов, создана в 1960-х для английского текста.

| Параметр | Значение |
|---|---|
| Размер | 7 бит |
| Диапазон | 0–127 |
| Покрытие | только латиница, цифры, пунктуация, служебные символы |

**Примеры:**

```python
ord('A')   # → 65
ord('a')   # → 97
ord('0')   # → 48
chr(65)    # → 'A'
```

| Символ | Код ASCII |
|---|---|
| `A` | 65 |
| `a` | 97 |
| `0` | 48 |
| пробел | 32 |
| `!` | 33 |

**Важное:** ASCII не знает ни кириллицы, ни акцентов, ни эмодзи — отсюда и возникла проблема, решённая позже Unicode.

## ⚠️ Проблема старых кодировок

**Назначение:** понять, откуда берутся "кракозябры".

| Кодировка | Для чего |
|---|---|
| `Windows-1251` | русский |
| `KOI8-R` | русский (альтернатива) |
| `ISO-8859` | европейские языки |
| `Big5` | китайский |

Разные страны создавали свои таблицы символов. Одинаковые байты означали разные символы в разных кодировках — если текст закодирован в одной, а декодирован в другой, получается бессмыслица.

**Примеры:**

```python
text = 'Привет'

data = text.encode('utf-8')
print(data)   # b'\xd0\x9f\xd1\x80...'

print(data.decode('cp1251'))
# → ÐŸÑ€Ð¸Ð²ÐµÑ‚   (кракозябры — это "Привет", прочитанный в неправильной кодировке)
```

**Важное:** кракозябры — это не ошибка данных, а ошибка **интерпретации** одних и тех же байт.

## 🌐 Unicode

**Назначение:** единая таблица символов, покрывающая все языки мира.

**Главная идея:**

> Unicode — это НЕ кодировка. Unicode — это просто таблица: `символ ↔ номер`.

| Символ | Unicode |
|---|---|
| `A` | U+0041 |
| `в` | U+0432 |
| `😀` | U+1F600 |
| `∞` | U+221E |
| `中` | U+4E2D |

**Примеры:**

```python
ord('A')    # → 65
ord('в')    # → 1074
ord('😀')   # → 128512
```

```python
# Номер символа в Unicode называется code point
code = ord('в')
print(code)        # → 1074
print(hex(code))   # → 0x432
print(chr(code))   # → 'в'
```

**Важное:** диапазон code point — от `0` до `1114111` (`0x10FFFF`).

## 🔤 UTF-8, UTF-16, UTF-32

**Назначение:** превратить Unicode-номер (code point) в реальные байты для хранения.

`UTF (Unicode Transformation Format)` — это способ переделать Unicode-номер в последовательность байт.

| Кодировка | Принцип | Размер на символ |
|---|---|---|
| `UTF-8` | переменное число байт | 1–4 байта |
| `UTF-16` | переменное число байт | 2 или 4 байта |
| `UTF-32` | фиксированное число байт | всегда 4 байта |

### UTF-8 — диапазоны code point

| Диапазон code point | Размер в UTF-8 |
|---|---|
| `0 – 127` (U+0000–U+007F) | 1 байт |
| `128 – 2047` (U+0080–U+07FF) | 2 байта |
| `2048 – 65535` (U+0800–U+FFFF) | 3 байта |
| `65536 – 1114111` (U+10000–U+10FFFF) | 4 байта |

**Примеры:**

```python
'A'.encode('utf-8')    # → b'A'                       (1 байт, code point 65)
'в'.encode('utf-8')    # → b'\xd0\xb2'                 (2 байта, code point 1074)
'中'.encode('utf-8')   # → b'\xe4\xb8\xad'              (3 байта, code point 20013)
'😀'.encode('utf-8')   # → b'\xf0\x9f\x98\x80'          (4 байта, code point 128512)
```

**Плюсы UTF-8:**
- экономит память — английский текст занимает 1 байт на символ
- полностью совместим с ASCII
- стандарт интернета
- поддерживает все языки и эмодзи

```python
'Hello'.encode('utf-8')     # → b'Hello'  (5 байт для 5 символов)
'Привет'.encode('utf-8')    # → 12 байт для 6 символов (кириллица — по 2 байта)
```

```python
# UTF-16 и UTF-32 — для сравнения
'A'.encode('utf-16')   # → b'\xff\xfeA\x00'         (с пометкой порядка байт + 2 байта)
'A'.encode('utf-32')   # → b'\xff\xfe\x00\x00A\x00\x00\x00'  (всегда 4 байта)
```

**Важное:** UTF-32 простой, но расточительный по памяти — используется редко. UTF-16 используется, например, в Java и внутри Windows.

## 🔧 Функции ord, chr, encode, decode

**Назначение:** базовый набор инструментов для перехода между символом, номером и байтами.

| Функция/метод | Направление |
|---|---|
| `ord(c)` | символ → Unicode-номер |
| `chr(n)` | Unicode-номер → символ |
| `str.encode(enc)` | строка (Unicode) → байты |
| `bytes.decode(enc)` | байты → строка (Unicode) |

**Примеры:**

```python
ord('A')      # → 65
ord('в')      # → 1074
ord('😀')     # → 128512

chr(65)       # → 'A'
chr(1074)     # → 'в'
chr(128512)   # → '😀'
```

```python
text = 'Привет'

encoded = text.encode('utf-8')
print(encoded)
# b'\xd0\x9f\xd1\x80\xd0\xb8\xd0\xb2\xd0\xb5\xd1\x82'

decoded = encoded.decode('utf-8')
print(decoded)   # → 'Привет'  ✅ верная кодировка

print(encoded.decode('cp1251'))
# → ÐŸÑ€Ð¸Ð²ÐµÑ‚  ❌ неверная кодировка, но валидные байты → кракозябры
```

## 📏 len и количество байт

**Назначение:** понять разницу между длиной строки и размером в байтах — частый источник ошибок.

**Примеры:**

```python
text = '😀'

print(len(text))                  # → 1   (один символ / code point)
print(len(text.encode('utf-8')))  # → 4   (четыре байта в UTF-8)
```

```python
text = 'Привет'

print(len(text))                  # → 6   (шесть символов)
print(len(text.encode('utf-8')))  # → 12  (кириллица — по 2 байта на символ)
```

**Важное:** `len()` строки **всегда** считает количество символов (code points), независимо от кодировки. Размер в байтах зависит от кодировки и нужно считать отдельно через `len(text.encode(...))`.

## 📊 Главная схема

**Кодирование:**

```
СИМВОЛ (текст)
    ↓
Unicode code point (1074)
    ↓
UTF-8 алгоритм
    ↓
БАЙТЫ (b'\xd0\xb2')
```

**Декодирование:**

```
БАЙТЫ (b'\xd0\xb2')
    ↓
decode('utf-8')
    ↓
Unicode code point (1074)
    ↓
СИМВОЛ (текст: 'в')
```

**Разница Unicode vs UTF-8:**

| | Unicode | UTF-8 |
|---|---|---|
| Что это | таблица символ↔номер | способ хранения номера в байтах |
| Знает о байтах | нет | да |
| Аналогия | каталог товаров | упаковка товара в коробку |

## ⚠️ Важные особенности

```python
# ❗ Неправильная кодировка не всегда даёт кракозябры —
# может вызвать исключение, если байты невалидны для этой кодировки
b'\xd0\x9f'.decode('ascii')
# → UnicodeDecodeError: 'ascii' codec can't decode byte 0xd0 in position 0

# ❗ len() считает символы, не байты
len('😀')                    # → 1
len('😀'.encode('utf-8'))    # → 4

# ❗ Не все code point из Unicode используются — есть зарезервированные
# и суррогатные диапазоны, не предназначенные для прямого использования

# ❗ UTF-8 всегда обратно совместим с ASCII:
# любой ASCII-текст в UTF-8 выглядит так же побайтово
'Hello'.encode('ascii') == 'Hello'.encode('utf-8')   # → True
```
