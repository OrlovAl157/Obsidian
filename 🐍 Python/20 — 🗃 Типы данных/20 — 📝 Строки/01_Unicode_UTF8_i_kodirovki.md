# 🌍 Unicode, UTF-8 и кодировки

## Зачем вообще нужны кодировки?

Компьютер хранит всё как биты и байты: `01010101`. Но человеку нужны буквы, цифры, символы и эмодзи. Кодировки — это правила, которые переводят между текстом и числами: **символ ↔ число**.

---

## 📌 ASCII: СТАРАЯ ТАБЛИЦА СИМВОЛОВ

### Что это?

ASCII (American Standard Code for Information Interchange) — одна из первых таблиц символов. Создана в 1960-х для английского текста.

### Содержит

- Английские буквы (A-Z, a-z)
- Цифры (0-9)
- Знаки пунктуации
- Служебные символы (перенос строки, табуляция и т.д.)

### Примеры

| Символ | Код ASCII |
|---|---|
| A | 65 |
| a | 97 |
| 0 | 48 |
| пробел | 32 |
| ! | 33 |

### Особенности

- Использует 7 бит
- Диапазон: 0–127
- **ТОЛЬКО латиница**
- Нет кириллицы, нет акцентов, нет эмодзи

```python
ord('A')  # → 65
chr(65)   # → 'A'
```

### Проблема ASCII

ASCII работал только для английского. Что делать со всеми остальными языками мира?

---

## ⚠️ ПРОБЛЕМА СТАРЫХ КОДИРОВОК

### Что произошло

Разные страны создавали свои таблицы символов:

- **Windows-1251** — для русского
- **KOI8-R** — для русского (альтернатива)
- **ISO-8859** — для европейских языков
- **Big5** — для китайского
- И ещё десятки других

### Результат: кракозябры

Одинаковые байты означали разные символы в разных кодировках. Если текст был закодирован в одной, а декодирован в другой — получалась чушь:

```
ÐŸÑ€Ð¸Ð²ÐµÑ‚
```

Это просто "Привет", прочитанный в неправильной кодировке!

### Пример проблемы

```python
text = 'Привет'

# Кодируем в UTF-8
data = text.encode('utf-8')
print(data)  # → b'\xd0\x9f\xd1\x80...'

# Декодируем в НЕПРАВИЛЬНОЙ кодировке
print(data.decode('cp1251'))
# → ÐŸÑ€Ð¸Ð²ÐµÑ‚ (кракозябры!)
```

## 🌐 UNICODE: УНИВЕРСАЛЬНАЯ ТАБЛИЦА

### Что это?

Unicode — это огромная таблица символов, которая включает символы ВСЕХ языков мира. Каждому символу дан уникальный номер.

### Примеры

| Символ | Unicode |
|---|---|
| A | U+0041 |
| в | U+0432 |
| 😀 | U+1F600 |
| ∞ | U+221E |
| 中 | U+4E2D |

### Главная идея

**Unicode — это НЕ кодировка. Unicode — это просто таблица: символ ↔ номер.**

```python
# ord() показывает Unicode-номер
ord('A')     # → 65
ord('в')     # → 1074
ord('😀')    # → 128512
```

### Code Point

Номер символа в Unicode называется **Code Point**. Это просто число, которое однозначно определяет символ.

```python
code = ord('в')
print(code)        # → 1074
print(hex(code))   # → 0x432
print(chr(code))   # → 'в'
```

## 🔤 КОДИРОВКИ UNICODE: UTF-8, UTF-16, UTF-32

### Проблема: как хранить номер в памяти?

Unicode говорит: "в" = 1074.

Но как хранить число 1074 в компьютере? В байтах! Вот для этого нужны кодировки.

**UTF (Unicode Transformation Format) — это способ переделать Unicode-номер в последовательность байт.**

### UTF-8: САМАЯ ПОПУЛЯРНАЯ

**Идея:** используй переменное количество байт в зависимости от символа.

| Символ | Размер | Пример |
|---|---|---|
| A | 1 байт | `b'A'` |
| в | 2 байта | `b'\xd0\xb2'` |
| 😀 | 4 байта | `b'\xf0\x9f\x98\x80'` |

**Плюсы UTF-8:**
- ✅ Экономит память (английский текст занимает 1 байт на символ)
- ✅ Полностью совместим с ASCII
- ✅ Стандарт интернета
- ✅ Поддерживает ВСЕ языки и эмодзи

```python
'A'.encode('utf-8')        # → b'A' (1 байт)
'в'.encode('utf-8')        # → b'\xd0\xb2' (2 байта)
'😀'.encode('utf-8')       # → b'\xf0\x9f\x98\x80' (4 байта)
```

### UTF-16

**Используется:** Java, Windows

Обычно 2 байта на символ, иногда 4.

```python
'A'.encode('utf-16')       # → b'\xff\xfe A\x00' (2-4 байта)
```

### UTF-32

Каждый символ — ровно 4 байта. Простая логика, но расходует много памяти.

```python
'A'.encode('utf-32')       # → b'\xff\xfe\x00\x00A\x00\x00\x00'
```

## 🔧 ФУНКЦИИ РАБОТЫ С ТЕКСТОМ И КОДИРОВКАМИ

### ord() — символ в номер

Преобразует символ в его Unicode-номер:

```python
ord('A')      # → 65
ord('в')      # → 1074
ord('😀')     # → 128512
```

### chr() — номер в символ

Преобразует номер обратно в символ:

```python
chr(65)       # → 'A'
chr(1074)     # → 'в'
chr(128512)   # → '😀'
```

### encode() — текст в байты

Преобразует строку (Unicode) в байты с помощью выбранной кодировки:

```python
text = 'Привет'

# В UTF-8
encoded = text.encode('utf-8')
print(encoded)
# → b'\xd0\x9f\xd1\x80\xd0\xb8\xd0\xb2\xd0\xb5\xd1\x82'

# В UTF-16
encoded = text.encode('utf-16')
print(encoded)
# → b'\xff\xfe...'
```

### decode() — байты в текст

Преобразует байты обратно в текст (выбираешь правильную кодировку!):

```python
data = b'\xd0\x9f\xd1\x80\xd0\xb8\xd0\xb2\xd0\xb5\xd1\x82'

# Правильная кодировка
text = data.decode('utf-8')
print(text)  # → 'Привет' ✅

# Неправильная кодировка
text = data.decode('cp1251')
print(text)  # → ÐŸÑ€Ð¸Ð²ÐµÑ‚ (кракозябры!) ❌
```

## 📊 ГЛАВНАЯ СХЕМА

### Процесс кодирования

```
СИМВОЛ (текст)
    ↓
Unicode code point (1074)
    ↓
UTF-8 алгоритм
    ↓
БАЙТЫ (b'\xd0\xb2')
```

### Процесс декодирования

```
БАЙТЫ (b'\xd0\xb2')
    ↓
decode('utf-8')
    ↓
Unicode code point (1074)
    ↓
СИМВОЛ (текст: 'в')
```

## 🔑 КАК РАБОТАЕТ UTF-8 ПОД КАПОТОМ

### Простой пример: символ 'в'

1. **Символ:** 'в'
2. **Unicode nomber:** 1074 (или U+0432)
3. **UTF-8 правило:** числа 0–127 = 1 байт, 128–2047 = 2 байта
4. **UTF-8 кодирование:** 1074 → два байта → `\xd0\xb2`

```python
'в'.encode('utf-8')  # → b'\xd0\xb2'
```

### Почему UTF-8 гениальный

**Английский текст очень компактный:**
```python
'Hello'.encode('utf-8')
# → b'Hello' (5 байт для 5 символов!)
```

**Но поддерживает все языки:**
```python
'Привет'.encode('utf-8')
# → 12 байт для 6 символов (каждый русский = 2 байта)
```

## 🔥 РАЗНИЦА МЕЖДУ UNICODE И UTF-8

**Unicode:**
- Это просто таблица
- Говорит: 'в' = 1074
- Не знает о байтах

**UTF-8:**
- Это способ кодирования
- Говорит: как хранить 1074 в памяти?
- Превращает число в байты

**Аналогия:** Unicode это каталог товаров, UTF-8 это способ упаковать товар в коробку.
