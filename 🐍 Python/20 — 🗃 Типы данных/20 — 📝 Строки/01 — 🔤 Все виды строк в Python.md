# 🔤 Все виды строк в Python

## Почему так много видов строк?

Python предоставляет разные способы создания строк для разных задач. Каждый вид решает свою проблему: одна экономит время на форматирование, другая не обрабатывает escape-последовательности, третья работает с байтами.

---

## 📝 ОБЫЧНЫЕ СТРОКИ

### Что это?

Самый простой способ создать строку. Используй одинарные или двойные кавычки — нет разницы.

```python
name = "Гвидо"
greeting = 'Привет'
```

### Escape-последовательности

Обычные строки обрабатывают специальные комбинации:

```python
# Перенос строки
text = "Hello\nWorld"
print(text)
# → Hello
#   World

# Табуляция
text = "Name\tAge"
print(text)
# → Name    Age

# Экранирование кавычки
text = "Он сказал \"Привет\""
print(text)
# → Он сказал "Привет"
```

### Когда использовать

Используй обычные строки для основного текста, когда нужна обработка специальных символов.

---

## 🎯 RAW-STRINGS (r"...")

### Что это?

Raw-string — это строка, где Python **НЕ обрабатывает** escape-последовательности. Backslash остаётся просто backslash.

```python
path = r"C:\new\test"
print(path)
# → C:\new\test (не пытается обработать \n или \t)
```

### Почему нужны?

Когда работаешь с путями, регулярными выражениями или другим текстом с большим количеством backslash.

Без raw-string:
```python
# ❌ Много экранирования
path = "C:\\new\\test\\file.txt"
```

С raw-string:
```python
# ✅ Понятнее
path = r"C:\new\test\file.txt"
```

### Ограничение: не может заканчиваться одним backslash

```python
# ❌ Ошибка
path = r"C:\new\"

# ✅ Правильно (два backslash в конце)
path = r"C:\new\\"

# ✅ Или обычная строка
path = "C:\\new\\"
```

**Почему?** Python видит `\"` и думает, что это экранирование кавычки, но в raw-string это не работает по замыслу.

### Часто используется

```python
# Регулярные выражения
pattern = r"\d{3}-\d{2}-\d{4}"  # SSN pattern

# Пути в Windows
file = r"D:\Documents\Report.pdf"
```

## 🚀 F-STRINGS (f"...")

### Что это?

F-string — это строка, где можно вставлять переменные и выражения прямо в текст. Введены в Python 3.6 и очень удобны.

```python
name = "Гвидо"
age = 67

# Вставка переменных
message = f"Привет, {name}! Тебе {age} лет"
print(message)
# → Привет, Гвидо! Тебе 67 лет
```

### Вычисления внутри строки

```python
x = 10
y = 20

result = f"Сумма: {x + y}"
print(result)
# → Сумма: 30

# Даже функции
import math
circle = f"Радиус 5: длина окружности = {2 * math.pi * 5:.2f}"
print(circle)
# → Радиус 5: длина окружности = 31.42
```

### Форматирование чисел

```python
price = 1234.5678

# Два знака после запятой
f"{price:.2f}"  # → '1234.57'

# Отступ
f"{name:>20}"  # выравнивание вправо на 20 символов
```

### Когда использовать

Когда нужно вставить переменные или выражения в строку. Это современный и удобный способ форматирования.

```python
# ❌ Старый способ
message = "Hello, {}! You are {} years old".format(name, age)

# ✅ F-string (новый)
message = f"Hello, {name}! You are {age} years old"
```

## 🌍 ЮНИКОД-СТРОКИ (u"...")

### Что это?

В Python 3 все строки по умолчанию — юникод. Префикс `u` оставлен для совместимости с Python 2.

```python
# Оба варианта одинаковые в Python 3
text1 = "Привет 😀"
text2 = u"Привет 😀"

# Это одно и то же
print(text1 == text2)  # → True
```

### Зачем нужен?

В Python 2 обычные строки были байтами, а `u"..."` были юникод-строками. Теперь это не нужно, но встречается в старом коде.

### Когда использовать

В Python 3 **не используй** префикс `u`. Просто пиши обычные строки — они уже юникод.

```python
# ✅ В Python 3 просто так
emoji = "😀🎉🚀"
```

## 💾 БАЙТОВЫЕ СТРОКИ (b"...")

### Что это?

Байтовая строка — это не текст, а последовательность байтов. Это разные типы данных!

```python
text = "Привет"      # str (текст)
data = b"Привet"     # bytes (байты)

print(type(text))    # → <class 'str'>
print(type(data))    # → <class 'bytes'>
```

### Когда нужны?

Когда работаешь с бинарными данными: файлы, сетевые протоколы, хеширование, кодирование.

```python
# Чтение файла в бинарном режиме
with open("file.png", "rb") as f:
    data = f.read()  # → bytes

# Хеширование
import hashlib
hash_value = hashlib.md5(b"password").hexdigest()

# Кодирование строки в байты
text = "Привет"
encoded = text.encode('utf-8')
print(encoded)  # → b'\xd0\x9f\xd1\x80\xd0\xb8\xd0\xb2\xd0\xb5\xd1\x82'

# Декодирование обратно
decoded = encoded.decode('utf-8')
print(decoded)  # → Привет
```

### Важно

Байтовые строки — это просто данные. Нельзя смешивать с текстом:

```python
# ❌ Ошибка
message = "Hello " + b"world"

# ✅ Правильно
message = "Hello " + "world"
message = b"Hello " + b"world"
```

## 📚 DOCSTRINGS (тройные кавычки)

### Что это?

Docstring — это строка документации функции, класса или модуля. Пишется в первой строке определения.

```python
def greet(name):
    """Приветствует человека по имени.
    
    Args:
        name (str): Имя человека
        
    Returns:
        str: Приветственное сообщение
    """
    return f"Привет, {name}!"
```

### Зачем нужны?

Для документации кода. Python и IDE используют docstring для справки:

```python
help(greet)
# Выведет документацию функции

# Или через __doc__
print(greet.__doc__)
```

### Тройные кавычки

Позволяют писать текст на несколько строк без `\n`:

```python
# Обычная строка
text = "Строка 1\nСтрока 2\nСтрока 3"

# Docstring (удобнее)
text = """Строка 1
Строка 2
Строка 3"""
```

### Форматы docstring

**NumPy style:**
```python
def add(a, b):
    """Складывает два числа.
    
    Parameters
    ----------
    a : int
        Первое число
    b : int
        Второе число
        
    Returns
    -------
    int
        Сумма чисел
    """
    return a + b
```

**Google style:**
```python
def add(a, b):
    """Складывает два числа.
    
    Args:
        a: Первое число
        b: Второе число
        
    Returns:
        Сумма чисел
    """
    return a + b
```

## 📊 СРАВНИТЕЛЬНАЯ ТАБЛИЦА

| Тип | Синтаксис | Escape | F-string | Использование |
|---|---|---|---|---|
| Обычная | `"..."` | ✅ | ✅ | Основной текст |
| Raw | `r"..."` | ❌ | ❌ | Пути, regex |
| F-string | `f"..."` | ✅ | ✅ | Форматирование |
| Юникод | `u"..."` | ✅ | ✅ | Для совместимости (редко) |
| Байты | `b"..."` | ✅ | ❌ | Бинарные данные |
| Docstring | `"""..."""` | ✅ | ✅ | Документация |

---

## ✅ ГЛАВНЫЕ ПРАВИЛА

✅ В Python 3 все строки по умолчанию юникод  
✅ Raw-string не может заканчиваться одним `\`  
✅ F-string — лучший способ форматирования (с Python 3.6)  
✅ Байтовые строки (`b"..."`) это НЕ текст, это данные  
✅ Docstring пиши в первой строке функции/класса  
✅ Используй тройные кавычки для многострочного текста
