# 🏗 Памятка: Числовая иерархия и особенности типов

### 💡 Ключевые особенности

- **Числовая иерархия** выражена через абстрактные базовые классы модуля `numbers`: `Number → Complex → Real → Rational → Integral`.

- **`Decimal` не входит в иерархию `Real`** — несмотря на то что это число с плавающей точкой, смешивать `Decimal` и `float` нельзя.

- **`float`** нельзя точно сравнивать через `==` — большинство дробей не представимы в двоичном виде точно.

- **`Decimal`** нужно создавать только из строк или целых чисел — создание из `float` уже содержит ошибку округления.

- **`bool`** — наследник `int`. `True == 1`, `False == 0`. Все операции `int` работают с `bool`.

- **`int`** в Python не имеет ограничений на размер — поддерживает длинную арифметику.

### 💡 Полезные советы

- **Для точных вычислений с дробями** — используй `Decimal('0.1')`, не `Decimal(0.1)`.

- **Для проверки числового типа** — используй `isinstance(x, numbers.Number)` вместо проверки конкретного класса.

- **`isinstance(True, int)`** → `True` — помни об этом при проверке типов, если важно различать `bool` и `int`.

- **`{True, 1}`** → `{True}` — множество и словарь считают `True` и `1` одним и тем же ключом.

- **NumPy и другие C-библиотеки** — имеют ограничения на размер чисел в отличие от встроенного `int`.

### 🚀 Быстрая справка

| **Тип** | **Модуль** | **Абстрактный родитель** | **Особенность** |
|---|---|---|---|
| `int` | встроенный | `Integral` | длинная арифметика, нет ограничений |
| `bool` | встроенный | `Integral` | подтип `int`, `True==1`, `False==0` |
| `float` | встроенный | `Real` | двоичное представление, потеря точности |
| `complex` | встроенный | `Complex` | комплексные числа |
| `Decimal` | `decimal` | `Number` (только!) | точная арифметика, несовместим с `float` |
| `Fraction` | `fractions` | `Rational` | рациональные дроби, точная арифметика |

## 🏛️ Числовая иерархия

```python
from numbers import Number, Complex, Real, Rational, Integral

# Иерархия наследования:
# Number
#   └── Complex       ← complex
#         └── Real    ← float
#               └── Rational  ← Fraction
#                     └── Integral  ← int, bool

# Проверка принадлежности
isinstance(3.14, Real)       # → True
isinstance(3.14, Rational)   # → False  (float не является рациональным)
isinstance(42, Integral)     # → True
isinstance(True, Integral)   # → True   (bool — наследник int)

# Decimal — стоит отдельно
from decimal import Decimal
isinstance(Decimal('3.14'), Number)   # → True
isinstance(Decimal('3.14'), Real)     # → False  ❗ не входит в общую иерархию
```

```python
# Decimal и float несовместимы — нельзя смешивать
Decimal('3.14') + 2.71   # ❌ TypeError!
Decimal('3.14') + 2      # ✅ int можно
```

## ⚠️ float — потеря точности

```python
# Большинство дробей не представимы в двоичном виде точно
# 0.1 в двоичном → 0.000110011001100... (бесконечная дробь)
# 0.3 в двоичном → 0.0100110011001100... (бесконечная дробь)

0.1 * 3             # → 0.30000000000000004  (не 0.3!)
0.1 * 3 == 0.3      # → False  ❗ никогда не сравнивай float через ==

# Правильный способ сравнения float
import math
math.isclose(0.1 * 3, 0.3)     # → True  ✅
abs(0.1 * 3 - 0.3) < 1e-9      # → True  ✅
```

## ✅ Decimal — точная арифметика

```python
from decimal import Decimal

# ❗ НЕЛЬЗЯ создавать из float — ошибка уже внутри float
Decimal(0.1)                    # → Decimal('0.1000000000000000055511...')
Decimal(0.1) * 3 == Decimal(0.3)   # → False  ❌

# ✅ ПРАВИЛЬНО — создавать из строки или int
Decimal('0.1') * 3              # → Decimal('0.3')
Decimal('0.1') * 3 == Decimal('0.3')   # → True  ✅
Decimal(1) / Decimal(10)        # → Decimal('0.1')  ✅
```

## 🔵 bool — подтип int

```python
# bool наследуется от int
issubclass(bool, int)           # → True
isinstance(True, int)           # → True  ❗ помни при проверке типов!

# True == 1, False == 0
True == 1                       # → True
False == 0                      # → True
True + 3                        # → 4
False + 3                       # → 3

# Строковое представление разное
repr(True)                      # → 'True'
repr(1)                         # → '1'

# Побочный эффект — множество и словарь считают True/1 и False/0 одним ключом
{True, 1, False, 0}             # → {True, False}  ❗ (True вытесняет 1)
{True: 'a', 1: 'b'}            # → {True: 'b'}    ❗ (значение перезаписывается)

# От bool нельзя наследоваться
class MyBool(bool): pass        # ❌ TypeError: type 'bool' is not an acceptable base type
```

## 🔢 Falsy-объекты

```python
# Объекты, которые bool() интерпретирует как False:

bool(None)          # → False
bool(False)         # → False

# Нули всех числовых типов
bool(0)             # → False
bool(0.0)           # → False
bool(0j)            # → False

from decimal import Decimal
from fractions import Fraction
bool(Decimal(0))    # → False
bool(Fraction(0,1)) # → False

# Пустые коллекции и последовательности
bool('')            # → False
bool(())            # → False
bool([])            # → False
bool({})            # → False
bool(set())         # → False
bool(range(0))      # → False

# Всё остальное — truthy
bool('0')           # → True   (непустая строка!)
bool([0])           # → True   (непустой список!)
bool(0.0001)        # → True
```

## ⚠️ Важные особенности

```python
# ❗ int в Python не имеет ограничений на размер
10 ** 1000          # ✅ работает (очень большое число)

# ❗ Но NumPy и C-библиотеки — имеют ограничения
import numpy as np
np.int8(200)        # ❌ overflow — тип int8 максимум 127

# ❗ isinstance учитывает наследование
isinstance(True, int)    # → True  (bool — наследник int)
isinstance(True, bool)   # → True
type(True) == int        # → False  (type не учитывает наследование)
type(True) == bool       # → True

# ❗ Decimal нельзя складывать с float
from decimal import Decimal
Decimal('1.5') + 1.5    # ❌ TypeError
Decimal('1.5') + 1      # ✅ int совместим
```
