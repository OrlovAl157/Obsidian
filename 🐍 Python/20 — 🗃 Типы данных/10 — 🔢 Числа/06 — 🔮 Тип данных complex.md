# 🔮 Тип данных complex

## 📌 Коротко

> `complex` — комплексное число вида `a + bj`, где `a` — вещественная часть, `b` — мнимая. Встроенный тип, не требует импорта. Используется в DSP, физике, математике.

---

## 📖 Теория

### Немного математики (только суть)

В математике мнимая единица `i` — это число такое что `i² = -1`. Это позволяет работать с корнями из отрицательных чисел и описывать точки на плоскости одним числом.

Комплексное число: `z = a + bi`, где:
- `a` — **вещественная** (действительная) часть
- `b` — **мнимая** часть

**В Python используется буква `j` вместо `i`** — это традиция инженеров и физиков (в электротехнике `i` — это ток).

```
Комплексная плоскость:

  Im (мнимая ось)
  ↑
  |      z = 3 + 4j
4 |. . . . *
  |        |
  |        |   |z| = √(3²+4²) = 5
  |        |
  +--------+------→ Re (вещественная ось)
  0        3
```

### Ключевые особенности

- ✅ Встроенный тип, `import` не нужен
- ✅ Обе части хранятся как `float`
- ✅ Поддерживает все арифметические операции
- ✅ Можно смешивать с `int` и `float`
- ⚠️ Нет операторов `<`, `>` (комплексные нельзя упорядочить)
- ⚠️ Для расширенных функций нужен модуль `cmath`

## 💻 Примеры кода

### Создание complex

```python
# Через литерал — j обязателен для мнимой части
z1 = 5 + 7j
z2 = 1j              # только мнимая часть (не просто j — это переменная!)
z3 = -3 + 5j
z4 = 1.5 + 3.2j
z5 = 3 + 0j          # вещественное как complex

print(type(z1))      # → <class 'complex'>

# Через функцию complex(real, imag)
z1 = complex(6, -8)      # → (6-8j)
z2 = complex(0, 2.5)     # → 2.5j
z3 = complex(5, 0)       # → (5+0j)
z4 = complex(3)          # → (3+0j)  (imag по умолчанию 0)

# Из строки — без пробелов!
z5 = complex('3+4j')     # → (3+4j)
# complex('3 + 4j')      # → ValueError! Пробелы не допускаются

# ⚠️ Фишка: 1j — это мнимая единица
print(1j)           # → 1j
print(1j * 1j)      # → (-1+0j)  т.е. i² = -1 ✅
# print(j)          # → NameError! j — это просто имя переменной
```

> [!important] Пиши `1j`, не `j`. Просто `j` — это имя переменной, Python его не знает.

### Свойства — real и imag

```python
z = 3 + 4j

print(z.real)   # → 3.0  (всегда float!)
print(z.imag)   # → 4.0  (всегда float!)

# Даже если задать как int
z2 = complex(3, 4)
print(z2.real)  # → 3.0  (float, не int)
print(z2.imag)  # → 4.0
```

### Арифметика

```python
z1 = 1 + 3j
z2 = -3 + 2j

print(z1 + z2)      # → (-2+5j)
print(z1 - z2)      # → (4+1j)
print(z1 * z2)      # → (-9-7j)    (1+3j)(-3+2j) = -3+2j-9j+6j² = -3-7j-6 = -9-7j
print(z1 / z2)      # → (0.2307...-0.8461...j)
print(z1 ** 2)      # → (-8+6j)

# Смешивание с int и float — работает
z1 + 5              # → (6+3j)
z1 * 2.5            # → (2.5+7.5j)
z1 - 1.0            # → (0.0+3.0j)

# ❌ Операции сравнения < > — не работают!
# z1 > z2            # → TypeError! Комплексные нельзя упорядочить
z1 == z2            # → False  (равенство — ок)
z1 != z2            # → True
```

### Метод conjugate() — сопряжённое число

Сопряжённое к `a + bj` — это `a - bj`. Мнимая часть меняет знак:

```python
z = 3 + 4j
print(z.conjugate())    # → (3-4j)

# Произведение числа на сопряжённое = сумма квадратов частей (всегда вещественное!)
z * z.conjugate()       # → (25+0j)  = 3² + 4² = 9 + 16 = 25
# Это используется при делении комплексных чисел
```

### Модуль — abs()

Модуль комплексного числа — это расстояние от точки до начала координат:

```python
z = 3 + 4j
print(abs(z))       # → 5.0   (√(3² + 4²) = √25 = 5)

z2 = 1j
print(abs(z2))      # → 1.0   (расстояние до начала = 1)

# Египетский треугольник: 3-4-5
abs(3 + 4j)         # → 5.0  ✅
abs(5 + 12j)        # → 13.0
abs(8 + 15j)        # → 17.0
```


### Модуль cmath — математика для комплексных чисел

```python
import cmath

z = 2 + 3j

# Полярные координаты
print(cmath.phase(z))       # → 0.9827...  (угол в радианах)
print(cmath.polar(z))       # → (3.6055..., 0.9827...)  (модуль, угол)

# Из полярных обратно в прямоугольные
r, phi = cmath.polar(z)
print(cmath.rect(r, phi))   # → (2+3j)

# Стандартные функции — возвращают complex
cmath.sqrt(-1)              # → 1j    (√-1 = i ✅)
cmath.sqrt(-4)              # → 2j    (√-4 = 2i)
cmath.sqrt(2 + 3j)          # → (1.674...+0.895...j)

cmath.exp(1j * cmath.pi)    # → (-1+1.2e-16j) ≈ -1  (формула Эйлера e^(πi) = -1)
cmath.log(z)                # → натуральный логарифм
cmath.sin(z)                # → синус комплексного числа

# Проверка
cmath.isnan(z)              # → False
cmath.isinf(z)              # → False
cmath.isfinite(z)           # → True
```

> [!tip] Формула Эйлера: `e^(iπ) + 1 = 0` — связывает e, π, i, 1, 0. Проверь:
> ```python
> import cmath
> print(cmath.exp(1j * cmath.pi) + 1)  # → 1.2246...e-16j ≈ 0 ✅
> ```

### Практические примеры

#### Вращение точки на плоскости

```python
import cmath

# Комплексное число как точка (x, y) = (real, imag)
point = complex(3, 0)       # точка (3, 0)

# Поворот на угол — умножение на e^(iθ)
angle = cmath.pi / 2        # 90 градусов
rotation = cmath.exp(1j * angle)

rotated = point * rotation
print(round(rotated.real, 10))  # → 0.0  (почти 0)
print(round(rotated.imag, 10))  # → 3.0  (точка переместилась в (0, 3))

# Поворот на 45°
angle45 = cmath.pi / 4
rotated45 = point * cmath.exp(1j * angle45)
print(rotated45)            # → (2.121...+2.121...j)
```

#### Нахождение корней из отрицательных чисел

```python
import cmath

# В math.sqrt(-1) → ValueError!
# В cmath.sqrt(-1) → 1j ✅

print(cmath.sqrt(-9))       # → 3j
print(cmath.sqrt(-2))       # → 1.414...j
print(cmath.sqrt(2 + 2j))   # → (1.5537...+0.6435...j)
```

#### Решение квадратного уравнения (все корни)

```python
import cmath

def solve_quadratic(a, b, c):
    """Решить ax² + bx + c = 0, включая комплексные корни"""
    discriminant = b**2 - 4*a*c
    sqrt_d = cmath.sqrt(discriminant)
    x1 = (-b + sqrt_d) / (2 * a)
    x2 = (-b - sqrt_d) / (2 * a)
    return x1, x2

# Вещественные корни
print(solve_quadratic(1, -5, 6))    # → (3+0j, 2+0j)

# Комплексные корни (D < 0)
print(solve_quadratic(1, 0, 1))     # → (1j, -1j)   → x² + 1 = 0
print(solve_quadratic(1, 2, 5))     # → (-1+2j, -1-2j)
```

### Частые ошибки

```python
# ❌ Просто j — это имя переменной, не мнимая единица
# z = 3 + j   # NameError если j не определена

# ✅ Нужно 1j или complex(0, 1)
z = 3 + 1j      # ✅
z = 3 + complex(0, 1)  # ✅

# ❌ Пробелы в строке для complex()
# complex('3 + 4j')   # → ValueError!

# ✅ Без пробелов
complex('3+4j')     # → (3+4j)

# ❌ Сравнение > <
# (1+2j) > (3+4j)    # → TypeError!

# ✅ Сравнение равенства — ок
(1+2j) == (1+2j)    # → True

# ❌ math.sqrt для отрицательных
import math
# math.sqrt(-1)   # → ValueError!

# ✅ cmath.sqrt
import cmath
cmath.sqrt(-1)    # → 1j ✅

# ⚠️ real и imag всегда float
z = 3 + 4j
type(z.real)    # → float (не int!)
```

### Когда используется complex

**✅ Используй complex когда:**
- Математика: квадратные уравнения с D < 0, теория чисел
- Физика: волны, переменный ток, колебания
- Обработка сигналов (DSP): преобразование Фурье
- Графика и анимация: вращение точек на плоскости
- Фракталы: множество Мандельброта

**❌ Не нужен когда:**
- Работаешь только с вещественными числами

## 📊 Методы и функции

```python
z = 3 + 4j

# Свойства
z.real          # → 3.0  (вещественная часть)
z.imag          # → 4.0  (мнимая часть)

# Методы
z.conjugate()   # → (3-4j)  (сопряжённое)

# Встроенные функции
abs(z)          # → 5.0   (модуль = расстояние до нуля)
complex(3, 4)   # → (3+4j) (создание)
complex('3+4j') # → (3+4j) (из строки)

# Модуль cmath
import cmath
cmath.phase(z)  # полярный угол
cmath.polar(z)  # (модуль, угол)
cmath.rect(r, φ) # из полярных в прямоугольные
cmath.sqrt(z)   # квадратный корень
cmath.exp(z)    # e^z
cmath.log(z)    # ln(z)
cmath.sin(z)    # sin(z)
cmath.pi        # π
cmath.e         # e
```
