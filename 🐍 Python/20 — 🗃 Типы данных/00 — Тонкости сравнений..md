## 🔗 1. Chaining comparisons — цепочки сравнений

> Любая цепочка операторов сравнения разбивается через `and`

**Правило:**

```
a op b op c  →  (a op b) and (b op c)
```

**Примеры:**

```python
print(False == False in [False])
# → (False == False) and (False in [False])
# → True and True → True

print(1 == 1 == True)
# → (1 == 1) and (1 == True)
# → True and True → True

print(2 == 2 == True)
# → (2 == 2) and (2 == True)
# → True and False → False  ⚠️ True это именно 1, не любое ненулевое!

print(1 in [1, 2] == True)
# → (1 in [1, 2]) and ([1, 2] == True)
# → True and False → False
```

> 💡 Средний элемент цепочки участвует в **обоих** сравнениях.

## 🔢 2. True == 1, False == 0

> `True` и `False` — это числа `1` и `0`, ничего больше

```python
print(True == 1)     # → True
print(False == 0)    # → True
print(True == 2)     # → False  ⚠️ True это ТОЛЬКО 1
print(False == '')   # → False  ⚠️ bool('') == False, но '' != False
print(False == [])   # → False
print(None == False) # → False  ⚠️ None это не False!
```

> ⚠️ `bool(x) == False` не означает `x == False`! Приведение к булевому и сравнение — разные вещи.

---

## 🔍 3. is vs ==

> `==` сравнивает значения, `is` — идентичность объекта в памяти

|Оператор|Что сравнивает|Пример|
|---|---|---|
|`==`|Значение|`[1,2] == [1,2]` → `True`|
|`is`|Идентичность (id)|`[1,2] is [1,2]` → `False`|

```python
a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)   # → True  (одинаковые значения)
print(a is b)   # → False (разные объекты в памяти)
```

**None всегда проверять через is:**

```python
x = None
print(x == None)  # → True, но не рекомендуется!
print(x is None)  # → True ✅ правильный способ
```

> 💡 `None` — синглтон. `is None` надёжнее — `__eq__` можно переопределить, `is` — нельзя.

## ⚡ 4. or / and возвращают объект, не булево

> Логические операторы возвращают один из операндов, а не `True`/`False`

**or — первый истинный или последний:**

```python
print(0 or 'привет')         # → 'привет'
print(5 or 'привет')         # → 5       (первый истинный)
print(0 or '' or [])         # → []      (все ложные → последний)
print(0 or '' or 'привет')   # → 'привет'
```

**and — первый ложный или последний:**

```python
print(5 and 'привет')        # → 'привет' (все истинные → последний)
print(0 and 'привет')        # → 0        (первый ложный)
print(5 and 0 and 'привет')  # → 0        (первый ложный)
```

> 💡 **Короткое замыкание** — Python останавливается как только результат известен.

---

## 🔄 5. not not — приведение к булевому

> Двойное отрицание = `bool(x)`

```python
print(not not 'привет')  # → True
print(not not 0)         # → False
print(not not [])        # → False
print(not not [1, 2])    # → True
```

> 💡 `not not x` эквивалентно `bool(x)`.

---

## 🚫 6. Сравнение разных типов

> `==` между разными типами почти всегда `False`

```python
print('1' == 1)    # → False  (str vs int)
print('1' == True) # → False
print([] == False) # → False
print({} == False) # → False
```

> ⚠️ Не путать `x == False` с `bool(x) == False` — это разные проверки!

---

## ❌ 7. None нельзя упорядочить

> `==` с None безопасен, `>` и `<` — нет

```python
print(None == 0)   # → False     ✅ OK
print(None == '')  # → False     ✅ OK
print(None > 0)    # → TypeError ❌
print(None < 0)    # → TypeError ❌
```

> 💡 `==` просто проверяет равенство → безопасно. `>` пытается упорядочить → Python не знает как → `TypeError`.

---

## 🔎 8. in — скорость зависит от коллекции

> Выбор коллекции влияет на скорость поиска

|Коллекция|Сложность|Почему|
|---|---|---|
|`list`|O(n)|перебор всех элементов|
|`tuple`|O(n)|перебор всех элементов|
|`set`|O(1)|хеш-таблица|
|`dict`|O(1)|проверяет ключи по хешу|

```python
x in [1, 2, 3]   # O(n) — медленно на больших данных
x in {1, 2, 3}   # O(1) — мгновенно ✅
```

> 💡 Если часто проверяешь наличие элемента — используй `set`.

## 🧠 9. Интернирование объектов

> Python кэширует часто используемые объекты — они указывают на один адрес в памяти

**Числа от -5 до 256:**

```python
a = 256
b = 256
print(a is b)   # → True  ✅ (закэшировано)

a = 257
b = 257
print(a is b)   # → False (новые объекты)
```

**Короткие строки без пробелов:**

```python
a = 'hello'
b = 'hello'
print(a is b)         # → True  (интернировано)

a = 'hello world'
b = 'hello world'
print(a is b)         # → False (пробел — не интернируется)
```

> ⚠️ В IDE результат может отличаться из-за оптимизации. Не полагайся на `is` для значений — только для `None`, `True`, `False`.

## 💻 Быстрая шпаргалка

```python
# 1. Цепочки → разбить через and
False == False in [False]   # → True

# 2. True/False это 1/0
True == 1    # → True
True == 2    # → False ⚠️

# 3. None через is
x is None    # ✅

# 4. or/and возвращают объект
0 or 'default'   # → 'default'
5 and 'ok'       # → 'ok'

# 5. not not = bool()
not not []   # → False

# 6. Разные типы
'1' == 1     # → False

# 7. None не упорядочить
None > 0     # → TypeError ❌

# 8. in — множество быстрее
x in {1,2,3}  # O(1) ✅

# 9. Интернирование
256 is 256   # → True
257 is 257   # → False (обычно)
```
