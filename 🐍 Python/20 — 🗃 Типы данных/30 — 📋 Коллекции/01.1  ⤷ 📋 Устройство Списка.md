## 🎯 Главный вопрос: Почему id() не меняется?

**Краткий ответ:** `id()` возвращает адрес **объекта-списка**, а не адрес **массива элементов**. При переполнении перемещается только внутренний массив, а сам объект остаётся на месте.

## 🔄 Что происходит при append()

### Шаг 1: Создание списка

python

```python
lst = [1, 2, 3, 4]

┌─────────────────────────────┐
│ Объект списка               │  id = 0x1000 ✓
│ size: 4                     │
│ allocated: 6 (4 * 1.5)      │
│ ob_item → 0x2000            │
└─────────────────────────────┘
         ↓
    ┌───────────────────┐
    │ Массив (0x2000)   │
    │ [1][2][3][4][_][_]│
    └───────────────────┘
```

---

### Шаг 2: Добавление в пределах резерва

python

```python
lst.append(5)  # Влезает!

┌─────────────────────────────┐
│ Объект списка               │  id = 0x1000 ✓ (НЕ изменился)
│ size: 5  ← изменилось       │
│ allocated: 6                │
│ ob_item → 0x2000 ✓          │  ← указатель НЕ изменился
└─────────────────────────────┘
         ↓
    ┌───────────────────┐
    │ Массив (0x2000)   │  ← ТОТ ЖЕ массив
    │ [1][2][3][4][5][_]│
    └───────────────────┘
```

---

### Шаг 3: Переполнение резерва

python

```python
lst.append(6)
lst.append(7)  # ⚠️ Переполнение!

┌─────────────────────────────┐
│ Объект списка               │  id = 0x1000 ✓ (ВСЁЁ ЕЩЁ тот же!)
│ size: 7  ← новый размер     │
│ allocated: 10 (7 * 1.5)     │
│ ob_item → 0x3000 ✗          │  ← ТОЛЬКО указатель изменился!
└─────────────────────────────┘
         ↓
    ┌──────────────────────────────────┐
    │ НОВЫЙ массив (0x3000)            │  ← Новое место в памяти
    │ [1][2][3][4][5][6][7][_][_][_]   │
    └──────────────────────────────────┘

    ┌───────────────────┐
    │ Старый (0x2000)   │  ← Удалён сборщиком мусора
    │ [освобождено]     │
    └───────────────────┘
```

## 📊 Демонстрация в коде

python

```python
import sys

lst = [1, 2, 3, 4]

print(f"Исходный список:")
print(f"  ID списка: {id(lst)}")
print(f"  Размер: {len(lst)}")
print(f"  Выделено байт: {sys.getsizeof(lst)}")
print()

# Добавляем в пределах резерва
lst.append(5)
print(f"После append(5):")
print(f"  ID списка: {id(lst)}")  # ← ТОТ ЖЕ!
print(f"  Размер: {len(lst)}")
print(f"  Выделено байт: {sys.getsizeof(lst)}")  # может НЕ измениться
print()

# Вызываем переполнение
lst.extend([6, 7, 8, 9, 10])
print(f"После extend([6,7,8,9,10]):")
print(f"  ID списка: {id(lst)}")  # ← ВСЁЁ ЕЩЁ ТОТ ЖЕ!
print(f"  Размер: {len(lst)}")
print(f"  Выделено байт: {sys.getsizeof(lst)}")  # ← УВЕЛИЧИЛОСЬ
```

**Вывод:**

```

  ID списка: 140234567890
  Размер: 4
  Выделено байт: 120

После append(5):
  ID списка: 140234567890  ← НЕ ИЗМЕНИЛСЯ
  Размер: 5
  Выделено байт: 120       ← Влезло в резерв

После extend([6,7,8,9,10]):
  ID списка: 140234567890  ← ВСЁ ЕЩЁ НЕ ИЗМЕНИЛСЯ!
```

```python
import ctypes

def get_array_address(lst):

    # ОПАСНО: зависит от реализации CPython

print(f"ID объекта списка: {id(lst)}")
```

## 💡 Почему так сделано?

### Преимущества двухуровневой структуры:

1. **Стабильность ссылок**

python

```python
   my_list = [1, 2, 3]
   reference = my_list  # Ссылка на объект
   
   my_list.append(4)
   my_list.extend([5, 6, 7, 8])  # Много добавлений
   
   # reference всё ещё указывает на тот же объект!
   print(reference is my_list)  # True
   print(id(reference) == id(my_list))  # True
```

2. **Эффективная работа словарей**

python

```python
   data = {}
   my_list = [1, 2, 3]
   
   data[id(my_list)] = "some value"
   
   my_list.append(4)  # Внутренний массив переехал
   
   # Но ключ словаря всё ещё работает!
   print(data[id(my_list)])  # "some value"
```

3. **Консистентность в многопоточности**
    - ID объекта не меняется
    - Проще синхронизировать доступ

## 📈 Стратегия роста списков

Python использует формулу роста (упрощённо):

python

```python
new_allocated = (current_size + (current_size >> 3) + 6)
```

**Практически это даёт:**

|Текущий размер|Новая ёмкость|Рост|
|---|---|---|
|0|4|—|
|4|8|2x|
|8|16|2x|
|16|25|1.56x|
|25|35|1.4x|
|35|46|1.31x|
|64|80|1.25x|
|128|161|1.26x|

Со временем коэффициент роста **стремится к ~1.125x** для больших списков.

## 🔄 Когда ID ДЕЙСТВИТЕЛЬНО меняется?

**Только при создании НОВОГО объекта:**

python

```python
lst = [1, 2, 3]
print(f"ID: {id(lst)}")  # 0x1000

# Создаём НОВЫЙ список
lst = lst + [4, 5]       # ← Оператор + создаёт новый объект!
print(f"ID: {id(lst)}")  # 0x2000 ← ДРУГОЙ ID!

# VS

lst = [1, 2, 3]
print(f"ID: {id(lst)}")  # 0x1000

lst.extend([4, 5])       # ← Модифицирует существующий!
print(f"ID: {id(lst)}")  # 0x1000 ← ТОТ ЖЕ ID!
```

---

## 📌 Ключевые выводы

1. **Список = Объект + Массив**
    - `id()` указывает на объект (фиксированный)
    - Массив данных может перемещаться
2. **При переполнении:**
    - ✅ Объект остаётся на месте → `id()` не меняется
    - ✗ Внутренний массив переезжает в новое место
    - Старый массив освобождается
3. **Рост списка:**
    - Изначально резерв ~1.5x
    - При переполнении новый резерв ~1.125x от текущего размера
    - Это амортизирует стоимость копирования
4. **ID меняется только:**
    - При присваивании нового объекта: `lst = [...]`
    - При операциях создающих новый список: `lst + [...]`, `lst * 2`
    - НО НЕ при: `append()`, `extend()`, `insert()`, `lst[i] = x`

## 🧪 Эксперимент: отслеживание переполнений

python

```python
import sys

lst = []
prev_size = sys.getsizeof(lst)

print(f"{'Размер':<10} {'Память (байт)':<15} {'Изменение':<10} {'ID'}")
print("-" * 60)

for i in range(20):
    lst.append(i)
    current_size = sys.getsizeof(lst)
    
    if current_size != prev_size:
        print(f"{len(lst):<10} {current_size:<15} {'← РОСТ':^10} {id(lst)}")
        prev_size = current_size
    else:
        print(f"{len(lst):<10} {current_size:<15} {'':<10} {id(lst)}")
```
