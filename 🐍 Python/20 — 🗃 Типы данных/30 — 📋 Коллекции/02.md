# 🔒 Кортежи (tuple)

## 📌 Коротко

> Упорядоченная коллекция неизменяемых элементов. Сохраняет порядок, быстрее списков, можно использовать как ключи в словарях. После создания нельзя изменять.

-----

```python
# 🔒 КОРТЕЖ (tuple)
(элемент, элемент, элемент)

# Характеристики:
- Упорядоченность: сохраняет порядок элементов
- Неизменяемость: нельзя добавлять/удалять элементы
- Индексируется: есть доступ по индексу [0], [1], ...
- Хешируется: можно использовать как ключ словаря
- Память: меньше чем список (оптимизация неизменяемости)
- Скорость: быстрее списков
```

### 💡 Ключевые особенности

- **Неизменяемость**: Нельзя изменить, добавить или удалить элементы после создания
- **Упорядоченность**: Сохраняет порядок элементов (в отличие от множеств)
- **Индексируемость**: Доступ по индексу `tuple[0]`, `tuple[-1]`
- **Хешируемость**: Можно использовать как ключи в словарях (в отличие от списков)
- **Производительность**: Работают быстрее списков благодаря неизменяемости

### 💡 Полезные советы

- **Скобки необязательны**: `1, 2, 3` это тоже кортеж (tuple packing)
- **Один элемент**: обязательна запятая — `(5,)` это кортеж, `(5)` это просто число
- **Отрицательные индексы**: `tuple[-1]` — последний элемент, `tuple[-2]` — предпоследний
- **Распаковка**: удобный способ присвоения нескольких переменных одновременно
- **Как ключи**: можно использовать в словарях (неизменяемые!)

## 🚀 Быстрая справка

|Задача                |Метод/Оператор    |Сложность |Пример             |
|----------------------|------------------|----------|-------------------|

|Длина                 |`len()`           |O(1)      |`len(my_tuple)`    |
|Найти индекс          |`.index()`        |O(n)      |`my_tuple.index(5)`|
|Подсчитать элементы   |`.count()`        |O(n)      |`my_tuple.count(5)`|
|Проверить наличие     |`in`              |O(n)      |`5 in my_tuple`    |
|Минимум               |`min()`           |O(n)      |`min(my_tuple)`    |
|Максимум              |`max()`           |O(n)      |`max(my_tuple)`    |

## 📝 Создание кортежей

```python
# Пустой кортеж
empty = ()
empty = tuple()

# С элементами
numbers = (1, 2, 3, 4, 5)
mixed = (1, "text", True, 3.14)
fruits = ("apple", "banana", "cherry")

# Без скобок (tuple packing) - скобки опциональны
coordinates = 10, 20, 30                # → (10, 20, 30)
person = "John", 25, "Engineer"         # → ("John", 25, "Engineer")

# Один элемент (запятая ОБЯЗАТЕЛЬНА!)
single = (5,)                           # → (5,) это кортеж
not_tuple = (5)                         # → 5 это просто число, не кортеж!
single = 5,                             # → (5,) с скобками не нужны

# Из других последовательностей
from_list = tuple([1, 2, 3])           # → (1, 2, 3)
from_string = tuple("hello")            # → ('h', 'e', 'l', 'l', 'o')
from_range = tuple(range(5))            # → (0, 1, 2, 3, 4)
from_set = tuple({3, 1, 2})             # → (1, 2, 3) порядок может отличаться

# Вложенные кортежи
nested = ((1, 2), (3, 4), (5, 6))
matrix = (
    (1, 2, 3),
    (4, 5, 6),
    (7, 8, 9)
)

# Кортеж смешанных типов
mixed_nested = (1, "hello", (10, 20), [1, 2, 3])  # ✅ список можно, он в кортеже
```

## 🔍 Доступ к элементам

```python
fruits = ("apple", "banana", "cherry", "date", "elderberry")

# Индексация (прямая)
fruits[0]                       # → "apple" (первый)
fruits[1]                       # → "banana"
fruits[4]                       # → "elderberry" (последний)

# Индексация (отрицательная)
fruits[-1]                      # → "elderberry" (последний)
fruits[-2]                      # → "date" (второй с конца)
fruits[-5]                      # → "apple" (пятый с конца = первый)

# Срезы (slicing)
fruits[1:3]                     # → ("banana", "cherry") [1:3)
fruits[:3]                      # → ("apple", "banana", "cherry")
fruits[2:]                      # → ("cherry", "date", "elderberry")
fruits[::2]                     # → ("apple", "cherry", "elderberry") с шагом 2
fruits[::-1]                    # → обратный порядок
fruits[1::2]                    # → ("banana", "date") начиная со второго, шаг 2

# Вложенные кортежи
nested = ((1, 2), (3, 4), (5, 6))
nested[0]                       # → (1, 2) первый подкортеж
nested[0][1]                    # → 2 второй элемент первого подкортежа
nested[1][0]                    # → 3 первый элемент второго подкортежа
```

## 🎯 Распаковка кортежей

```python
# Базовая распаковка
point = (10, 20, 30)
x, y, z = point                 # → x=10, y=20, z=30

# Распаковка в цикле
for x, y in ((1, 2), (3, 4), (5, 6)):
    print(f"x={x}, y={y}")

# Обмен значений
a, b = 5, 10
a, b = b, a                     # → a=10, b=5 (элегантный обмен!)

# Распаковка с остатком (extended unpacking, Python 3+)
numbers = (1, 2, 3, 4, 5)
first, *rest = numbers          # → first=1, rest=[2, 3, 4, 5]
first, *middle, last = numbers  # → first=1, middle=[2, 3, 4], last=5
*start, last = numbers          # → start=[1, 2, 3, 4], last=5
first, *middle, last = (1, 2, 3)  # → first=1, middle=[2], last=3

# Игнорирование значений (используйте _)
person = ("John", 25, "Engineer")
name, _, profession = person    # → name="John", profession="Engineer"
first, *_, last = (1, 2, 3, 4, 5)  # → first=1, last=5

# Вложенная распаковка
nested = ((1, 2), (3, 4))
(a, b), (c, d) = nested        # → a=1, b=2, c=3, d=4

# Неправильная распаковка
# a, b = (1, 2, 3)             # ❌ ValueError: too many values to unpack
# a, b, c = (1, 2)             # ❌ ValueError: not enough values to unpack
```

## 🔄 Итерация

```python
fruits = ("apple", "banana", "cherry")

# Обычный цикл
for fruit in fruits:
    print(fruit)                # → apple, banana, cherry

# С индексом (enumerate)
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")      # → 0: apple, 1: banana, 2: cherry

# С индексом, начиная с 1
for i, fruit in enumerate(fruits, start=1):
    print(f"{i}: {fruit}")      # → 1: apple, 2: banana, 3: cherry

# Две последовательности одновременно (zip)
prices = (100, 200, 150)
for fruit, price in zip(fruits, prices):
    print(f"{fruit}: {price}₽") # → apple: 100₽, banana: 200₽, ...

# Вложенные кортежи
pairs = ((1, 2), (3, 4), (5, 6))
for a, b in pairs:
    print(f"a={a}, b={b}")      # → a=1, b=2; a=3, b=4; ...

# Обратная итерация
for fruit in reversed(fruits):
    print(fruit)                # → cherry, banana, apple

# В отсортированном порядке
for fruit in sorted(fruits):
    print(fruit)                # → apple, banana, cherry
```

## 🔄 Преобразование типов

```python
# Кортеж → Список
my_tuple = (1, 2, 3, 4, 5)
my_list = list(my_tuple)        # → [1, 2, 3, 4, 5]

# Список → Кортеж
my_list = [1, 2, 3, 4, 5]
my_tuple = tuple(my_list)       # → (1, 2, 3, 4, 5)

# Строка → Кортеж (каждый символ отдельно)

# Множество → Кортеж (порядок не гарантирован)
my_set = {3, 1, 2}

sorted_tuple = tuple(sorted(my_set))  # → (1, 2, 3) гарантированно

# Словарь → Кортеж (только ключи)
my_dict = {"a": 1, "b": 2, "c": 3}
keys = tuple(my_dict)           # → ("a", "b", "c")
keys = tuple(my_dict.keys())    # → то же самое
values = tuple(my_dict.values())  # → (1, 2, 3)
items = tuple(my_dict.items())  # → (("a", 1), ("b", 2), ("c", 3))

# Диапазон → Кортеж
my_range = range(5)
my_tuple = tuple(my_range)      # → (0, 1, 2, 3, 4)
```

## ⚠️ Важные отличия от списков

```python
# 🔒 КОРТЕЖ - НЕИЗМЕНЯЕМЫЙ
my_tuple = (1, 2, 3)

# ❌ Нельзя изменять элементы
# my_tuple[0] = 10              # TypeError: 'tuple' object does not support item assignment

# ❌ Нельзя добавлять элементы
# my_tuple.append(4)            # AttributeError: 'tuple' object has no attribute 'append'

# ❌ Нельзя удалять элементы
# my_tuple.remove(1)            # AttributeError: 'tuple' object has no attribute 'remove'

# ✅ Можно создать новый кортеж с изменениями
my_tuple = (1, 2, 3)
my_tuple = my_tuple + (4,)      # → (1, 2, 3, 4) новый кортеж
my_tuple = (0,) + my_tuple      # → (0, 1, 2, 3, 4) новый кортеж

# Список - ИЗМЕНЯЕМЫЙ
my_list = [1, 2, 3]
my_list[0] = 10                 # ✅ Работает
my_list.append(4)               # ✅ Работает
my_list.remove(1)               # ✅ Работает

# Чтобы "изменить" кортеж, нужно создать новый
my_tuple = (1, 2, 3)
temp_list = list(my_tuple)
temp_list[0] = 10
temp_list.append(4)
my_tuple = tuple(temp_list)     # → (10, 2, 3, 4)

# ⚠️ Важно: элементы кортежа могут быть изменяемы!
mutable_tuple = ([1, 2], [3, 4])  # кортеж неизменяемый
mutable_tuple[0].append(100)       # ✅ это работает! список внутри изменяется
# → ([1, 2, 100], [3, 4])

# mutable_tuple[0] = [5, 6]        # ❌ это НЕ работает - нельзя переприсвоить элемент кортежа
```

## 🎯 Когда использовать кортежи

**Используйте кортежи когда:**

- Данные не должны изменяться (координаты, RGB цвета, конфигурация)
- Нужно использовать как ключ в словаре
- Возвращаете несколько значений из функции
- Нужна небольшая оптимизация по памяти и скорости
- Хотите защитить данные от случайного изменения
- Работаете с неизменяемыми наборами данных (дни недели, направления)

**Используйте списки когда:**

- Нужно добавлять/удалять элементы
- Данные часто изменяются
- Нужны методы вроде `.append()`, `.sort()`, `.remove()`, `.insert()`
- Работаете с коллекцией однородных элементов
- Порядок и индексация важны, но изменяемость критична

## 🔍 Методы кортежей (всего 2!)

```python
my_tuple = (1, 2, 3, 2, 4, 2, 5)

# Метод 1: подсчитать вхождения элемента
my_tuple.count(2)               # → 3 (2 встречается 3 раза)
my_tuple.count(10)              # → 0 (нет в кортеже)

# Метод 2: найти индекс элемента
my_tuple.index(3)               # → 2 (3 находится на индексе 2)
my_tuple.index(2)               # → 1 (первое вхождение 2)

# Используют диапазон для поиска
my_tuple.index(2, 2)            # → 3 (найти 2 начиная с индекса 2)
my_tuple.index(2, 2, 6)         # → 3 (найти 2 в диапазоне [2:6])

# ❌ Неправильное использование
# my_tuple.index(10)             # ValueError: 10 is not in tuple

# Всё остальное - встроенные функции
len(my_tuple)                   # → 7 (длина)
min(my_tuple)                   # → 1 (минимум)
max(my_tuple)                   # → 5 (максимум)
sum(my_tuple)                   # → 19 (сумма)
sorted(my_tuple)                # → [1, 2, 2, 2, 3, 4, 5] (список!)
```
