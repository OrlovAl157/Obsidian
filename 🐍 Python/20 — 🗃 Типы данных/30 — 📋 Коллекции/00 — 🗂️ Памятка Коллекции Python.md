## ⚡ Big O — производительность

|Операция|`list`|`tuple`|`set`|`dict`|`deque`|`array`|
|---|:-:|:-:|:-:|:-:|:-:|:-:|
|Доступ по индексу|O(1)|O(1)|❌|❌|**O(n)** ⚠️|O(1)|
|Поиск `x in`|**O(n)** ⚠️|**O(n)** ⚠️|O(1)|O(1)|**O(n)** ⚠️|**O(n)** ⚠️|
|Добавить в конец|O(1)|❌|O(1)|O(1)|O(1)|O(1)|
|Добавить в начало|**O(n)** ⚠️|❌|O(1)|❌|O(1)|**O(n)** ⚠️|
|Удалить с конца|O(1)|❌|O(1)|O(1)|O(1)|O(1)|
|Удалить с начала|**O(n)** ⚠️|❌|O(1)|❌|O(1)|**O(n)** ⚠️|
|Удалить элемент|O(n)|❌|O(1)|O(1)|O(n)|O(n)|
|Сортировка|O(n log n)|❌|❌|❌|O(n log n)|O(n log n)|

> ⚠️ Жирные — потенциально медленные операции, на которые стоит обратить внимание при выборе коллекции.

## 💾 Память

|Коллекция|Относительно|Комментарий|
|---|:-:|---|
|`tuple`|🟢 минимум|нет резервирования, компактнее list|
|`array`|🟢 минимум|в 3–4 раза меньше list для чисел|
|`list`|🟡 средняя|резервирует место с запасом для роста|
|`deque`|🟡 средняя|блоки памяти вместо непрерывного массива|
|`str`|🟡 средняя|оптимизирована для текста|
|`set`|🔴 высокая|хеш-таблица требует дополнительную память|
|`dict`|🔴 высокая|хеш-таблица + ключи + значения|

```python
import sys
from array import array

sys.getsizeof([1,2,3,4,5])          # ~120 байт
sys.getsizeof((1,2,3,4,5))          # ~80 байт
sys.getsizeof({1,2,3,4,5})          # ~224 байт
sys.getsizeof(array('i',[1,2,3,4,5])) # ~80 байт
```

## 🃏 Карточки коллекций

---

### 📋 list

> Универсальная коллекция. Подходит для большинства задач.

```python
lst = [1, 'hello', True, 3.14]  # любые типы, дубликаты ок

# Добавление
lst.append(4)           # в конец        — O(1) ✅
lst.insert(0, 99)       # в начало       — O(n) ⚠️
lst.extend([5, 6])      # несколько в конец

# Удаление
lst.pop()               # с конца        — O(1) ✅
lst.pop(0)              # с начала       — O(n) ⚠️
lst.remove(3)           # по значению    — O(n)
lst.clear()             # очистить всё

# Поиск
lst.index(2)            # индекс значения — O(n)
lst.count(2)            # количество вхождений
2 in lst                # проверка — O(n) ⚠️

# Сортировка
lst.sort()              # на месте, изменяет список
lst.sort(reverse=True)
sorted(lst)             # возвращает новый список

# Копирование
lst.copy()              # поверхностная копия
```

|||
|---|---|
|✅ **Использовать**|порядок важен, данные меняются, нужен индекс|
|❌ **Не использовать**|данные фиксированы → `tuple`; нужна уникальность → `set`; много вставок в начало → `deque`; большие числа → `array`|

---

### 📌 tuple

> Неизменяемый список. Быстрее и компактнее. Можно использовать как ключ словаря.

```python
t = (1, 2, 3)
t = 1, 2, 3             # скобки необязательны
t = (42,)               # ⚠️ один элемент — запятая обязательна!
t = (42)                # ❌ это просто число, не кортеж

# Только два метода
t.count(2)
t.index(2)

# Распаковка
a, b, c = (1, 2, 3)
first, *rest = (1, 2, 3, 4)    # → first=1, rest=[2,3,4]

# Ключ словаря — потому что хешируемый
d = {(0, 0): 'начало', (1, 2): 'точка A'}

# Возврат нескольких значений из функции
def min_max(lst):
    return min(lst), max(lst)   # неявно возвращает tuple
```

|||
|---|---|
|✅ **Использовать**|данные не меняются, ключ словаря, возврат нескольких значений, координаты `(x, y)`, RGB `(r, g, b)`|
|❌ **Не использовать**|данные будут меняться → `list`|

### 🔵 set

> Только уникальные элементы. Молниеносный поиск O(1).

```python
s = {1, 2, 3, 3, 2}    # → {1, 2, 3}  дубликаты удалены
s = set([1, 2, 2, 3])  # из списка
s = set()              # ⚠️ пустое множество — не {}, это словарь!

# Изменение
s.add(4)               # O(1)
s.remove(3)            # KeyError если нет
s.discard(99)          # без ошибки если нет
s.pop()                # случайный элемент

# Математика множеств
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b                  # объединение      → {1,2,3,4,5,6}
a & b                  # пересечение      → {3,4}
a - b                  # разность         → {1,2}
a ^ b                  # симм. разность   → {1,2,5,6}

a.issubset(b)          # a ⊆ b?
a.issuperset(b)        # a ⊇ b?
a.isdisjoint(b)        # нет общих?

# Удаление дубликатов из списка
unique = list(set([1, 2, 2, 3, 3]))   # → [1, 2, 3]
```

> ⚠️ Элементы должны быть **хешируемыми** — нельзя хранить `list` и `dict`.

|||
|---|---|
|✅ **Использовать**|нужна уникальность, быстрый `x in s`, операции над множествами|
|❌ **Не использовать**|важен порядок, нужны дубликаты, нужен индекс|

### 🧊 frozenset

> Неизменяемый set. Может быть ключом словаря или элементом другого set.

```python
fs = frozenset([1, 2, 3, 2])   # → frozenset({1, 2, 3})

fs | {4, 5}            # все операции set — ✅
fs.add(4)              # изменение — ❌ AttributeError

# Ключ словаря
d = {frozenset({1, 2}): "пара A", frozenset({3, 4}): "пара B"}

# Элемент другого set
s = {frozenset({1, 2}), frozenset({3, 4})}
```

|||
|---|---|
|✅ **Использовать**|нужно множество как ключ словаря или элемент другого множества|

### 📖 dict

> Пары ключ-значение. Быстрый поиск по ключу O(1).

```python
d = {'name': 'Иван', 'age': 25}
d = dict(name='Иван', age=25)

# Чтение
d['name']                    # KeyError если нет ⚠️
d.get('name')                # None если нет ✅
d.get('email', 'н/д')        # дефолт если нет ✅

# Запись
d['email'] = 'a@b.com'       # добавить / изменить
d.update({'age': 26})        # обновить несколько

# Удаление
del d['age']                 # KeyError если нет
d.pop('age')                 # вернёт значение
d.pop('age', None)           # без ошибки ✅

# Перебор
d.keys()                     # ключи
d.values()                   # значения
d.items()                    # пары (ключ, значение)

# Проверка
'name' in d                  # O(1) ✅

# Удобные методы
d.setdefault('hits', 0)      # установить если ключа нет

# Слияние (Python 3.9+)
merged = d1 | d2
d1 |= d2                     # обновить на месте

# Генератор
squares = {x: x**2 for x in range(5)}
```

> 💡 Ключи должны быть **хешируемыми**: `str`, `int`, `tuple` — ✅; `list`, `dict` — ❌

|||
|---|---|
|✅ **Использовать**|связь ключ-значение, быстрый поиск, подсчёт, кэш, конфиги|
|❌ **Не использовать**|простой список → `list`; нужна уникальность → `set`|

### 🔄 deque

> Быстрые операции с обоих концов O(1). Из `collections`.

```python
from collections import deque

d = deque([1, 2, 3])
d = deque([1, 2, 3], maxlen=5)  # при переполнении — старые удаляются автоматически

d.append(4)            # в конец     — O(1) ✅
d.appendleft(0)        # в начало    — O(1) ✅
d.pop()                # с конца     — O(1) ✅
d.popleft()            # с начала    — O(1) ✅
d.extend([4, 5])
d.extendleft([0, -1])  # ⚠️ разворачивает порядок!
d.rotate(2)            # сдвиг вправо
d.rotate(-2)           # сдвиг влево

d[0]                   # индекс      — O(n) ⚠️
```

```python
# Очередь FIFO
queue = deque()
queue.append('first')
queue.popleft()        # → 'first'

# Стек LIFO
stack = deque()
stack.append('first')
stack.pop()            # → 'first'

# История последних N действий
history = deque(maxlen=10)
# при переполнении старые удаляются автоматически
```

|||
|---|---|
|✅ **Использовать**|FIFO/LIFO очереди, частые операции в начале, история с лимитом, скользящее окно|
|❌ **Не использовать**|нужен быстрый доступ по индексу → `list`|

### 🔢 array

> Компактный массив одного типа. Из модуля `array`.

```python
from array import array

arr = array('i', [1, 2, 3])   # signed int
arr = array('f', [1.0, 2.5])  # float
arr = array('d', [1.0, 2.5])  # double
```

|Код|Тип|Байт|
|:-:|---|:-:|
|`'b'`|signed int|1|
|`'B'`|unsigned int|1|
|`'i'`|signed int|2|
|`'l'`|signed long|4|
|`'f'`|float|4|
|`'d'`|double|8|

```python
arr.append(6)
arr.pop()
arr[0]                 # O(1)
arr.tolist()           # → обычный list
```

|||
|---|---|
|✅ **Использовать**|большие числовые данные, экономия памяти, бинарные данные|
|❌ **Не использовать**|смешанные типы → `list`; маленькие данные → `list`|

## 📦 collections — дополнительные коллекции

### Counter — счётчик

```python
from collections import Counter

c = Counter(['a', 'b', 'a', 'c', 'b', 'a'])
# → Counter({'a': 3, 'b': 2, 'c': 1})

c['a']                 # → 3  (не KeyError — вернёт 0 если нет)
c.most_common(2)       # → [('a', 3), ('b', 2)]
c.total()              # сумма всех (Python 3.10+)

# Арифметика
c1 = Counter(a=3, b=2)
c2 = Counter(a=1, b=4)
c1 + c2                # → Counter({'b': 6, 'a': 4})
c1 - c2                # → Counter({'a': 2})  (отрицательные отбрасываются)
```

✅ **Использовать:** подсчёт частоты, топ-N элементов

---

### defaultdict — словарь с дефолтом

```python
from collections import defaultdict

# Группировка
d = defaultdict(list)
for word in ['apple', 'ant', 'banana', 'bear']:
    d[word[0]].append(word)
# → {'a': ['apple', 'ant'], 'b': ['banana', 'bear']}

# Подсчёт
d = defaultdict(int)
for char in "abracadabra":
    d[char] += 1       # не нужна проверка — нет KeyError

# Вложенные структуры
d = defaultdict(lambda: defaultdict(int))
d['user1']['login'] += 1
```

> 💡 При обращении к несуществующему ключу создаёт его с дефолтным значением — в отличие от обычного `dict` который бросает `KeyError`.

✅ **Использовать:** группировка, накопление, избежание KeyError

---

### namedtuple — именованный кортеж

```python
from collections import namedtuple

Point = namedtuple('Point', ['x', 'y'])
p = Point(10, 20)

p.x                    # → 10  (по имени)
p[0]                   # → 10  (по индексу)
p._asdict()            # → {'x': 10, 'y': 20}
p._replace(x=100)      # новый экземпляр с изменённым полем

Color = namedtuple('Color', ['r', 'g', 'b'])
red = Color(255, 0, 0)
```

✅ **Использовать:** простые структуры (точки, координаты, RGB), читаемость кода

---

### OrderedDict — упорядоченный словарь

```python
from collections import OrderedDict

d = OrderedDict()
d.move_to_end('key')             # переместить в конец
d.move_to_end('key', last=False) # переместить в начало
d.popitem(last=True)             # удалить последний
d.popitem(last=False)            # удалить первый
```

> 💡 В Python 3.7+ обычный `dict` уже сохраняет порядок. `OrderedDict` нужен только когда нужны методы `move_to_end` и `popitem`.

---

### ChainMap — цепочка словарей

```python
from collections import ChainMap

defaults = {'color': 'red', 'size': 'medium'}
user = {'color': 'blue'}

config = ChainMap(user, defaults)
config['color']   # → 'blue'    (из user — приоритет)
config['size']    # → 'medium'  (из defaults)

# Изменения идут только в первый словарь
config['theme'] = 'dark'   # → попадёт в user, defaults не тронут
```

✅ **Использовать:** объединение конфигов без копирования, приоритет настроек

## 🎯 Когда что выбрать

|Задача|Выбор|
|---|---|
|Универсальная коллекция|`list`|
|Данные не меняются|`tuple`|
|Только уникальные элементы|`set`|
|Быстрый поиск|`set` или `dict`|
|Пары ключ-значение|`dict`|
|Очередь FIFO|`deque`|
|Стек LIFO|`list` или `deque`|
|Большие числовые данные|`array`|
|Подсчёт частоты|`Counter`|
|Группировка без KeyError|`defaultdict`|
|История с лимитом|`deque(maxlen=n)`|
|Структура данных с именами|`namedtuple`|
|Объединение конфигов|`ChainMap`|
|Множество как ключ словаря|`frozenset`|

## ⚠️ Частые ошибки

```python
# 1. Медленный поиск в list — используй set
if x in [1, 2, 3, 4, 5]:   # ❌ O(n)
if x in {1, 2, 3, 4, 5}:   # ✅ O(1)

# 2. Вставки в начало list — используй deque
lst.insert(0, x)            # ❌ O(n) каждый раз
dq.appendleft(x)            # ✅ O(1)

# 3. Пустое {} — это словарь, не множество!
s = {}                      # ❌ dict
s = set()                   # ✅ set

# 4. Кортеж из одного элемента — нужна запятая
t = (42)                    # ❌ просто int
t = (42,)                   # ✅ tuple

# 5. Изменение list во время итерации
for x in lst:
    if x % 2 == 0:
        lst.remove(x)       # ❌ пропускает элементы!
lst = [x for x in lst if x % 2 != 0]  # ✅

# 6. list для больших числовых данных
nums = [0] * 1_000_000      # ❌ ~8 MB
nums = array('i', [0] * 1_000_000)  # ✅ ~4 MB
```
