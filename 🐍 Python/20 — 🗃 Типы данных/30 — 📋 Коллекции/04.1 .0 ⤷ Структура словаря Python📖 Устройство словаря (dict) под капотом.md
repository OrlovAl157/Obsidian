# 📖 Устройство словаря (dict) под капотом

## 📌 Коротко

Словарь в Python — это **хеш-таблица** с двухуровневой структурой (с Python 3.6+). Поиск по ключу работает за O(1) благодаря хешированию.

-----

## 🏗 Двухуровневая структура

```
┌─────────────────────────────────────────┐
│  УРОВЕНЬ 1: Индексная таблица           │
│  Маленькая, хранит только числа         │
│                                         │
│  слот: 0    1    2    3    4    5    6   │
│        [-1] [2] [-1] [0] [-1] [-1] [1]  │
│              ↑        ↑         ↑       │
│           'city'   'name'    'age'      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  УРОВЕНЬ 2: Массив записей              │
│  Плотный, по порядку добавления         │
│                                         │
│  поз. │    хеш     │  ключ  │ значение  │
│  ─────┼────────────┼────────┼────────── │
│   0   │ hash(name) │ 'name' │ 'John'    │
│   1   │ hash(age)  │ 'age'  │  25       │
│   2   │ hash(city) │ 'city' │ 'Moscow'  │
└─────────────────────────────────────────┘
```

> 💡 Индексная таблица хранит **номер строки** в массиве записей — как оглавление книги.

-----

## ➕ Добавление ключа

```python
d['name'] = 'John'
```

```
Шаг 1: hash('name') → большое число, например 12345678
Шаг 2: 12345678 % размер_таблицы → индекс слота, например 3
Шаг 3: слот 3 свободен (-1)?
        ДА → записать индекс в индексную таблицу
        НЕТ → коллизия, искать следующий свободный слот
Шаг 4: добавить запись в конец массива записей
        [hash | 'name' | 'John']
```

-----

## 🔍 Поиск ключа

```python
d['age']  # → 25
```

```
Шаг 1: hash('age') % размер_таблицы → индекс слота = 6
Шаг 2: индексная таблица[6] = 1
Шаг 3: массив записей[1] → проверяем хеши → совпадают
Шаг 4: проверяем 'age' == 'age' через __eq__ → True
Шаг 5: возвращаем 25 ✅
```

> ⚠️ Сначала сравниваются **хеши** (быстро, это просто числа), и только потом `__eq__` (медленнее). Оптимизация скорости.

-----

## 💥 Коллизия

Коллизия — два разных ключа дают одинаковый индекс слота.

```
hash('a') % 8 = 3  → слот 3 занят!
→ ищем следующий свободный слот по формуле
→ слот 7 свободен → записываем туда
```

При поиске Python идёт по той же цепочке пока не найдёт нужный ключ или пустой слот.

-----

## 🗑 Удаление ключа

Слоты в индексной таблице имеют три состояния:

|Значение|Смысл                           |
|--------|--------------------------------|
|`-1`    |пустой, никогда не использовался|
|`-2`    |**dummy** — был удалён          |
|`N`     |индекс в массиве записей        |


> ⚠️ При удалении ставится `-2` (dummy), а не `-1`! Иначе поиск остановится на этом слоте и не найдёт ключи которые были сдвинуты из-за коллизии.

```
Ищем 'b', была коллизия → слот 3 → слот 7
         ↑
    если здесь -1 → стоп, "ключа нет" ❌
    если здесь -2 → продолжаем поиск ✅
```

-----

## 📈 Рост словаря

Когда заполняется **~2/3 слотов** индексной таблицы:

```
1. Создаётся новая таблица — в 2 раза больше
2. Все записи перехешируются заново
3. Старая таблица удаляется
```

Порог 2/3 — баланс между памятью и количеством коллизий.

-----

## 🔑 Правило хешируемости

Ключом словаря может быть только **неизменяемый** объект:

```python
d['name'] = 1    # ✅ строка
d[42] = 1        # ✅ число
d[(1,2)] = 1     # ✅ кортеж
d[[1,2]] = 1     # ❌ список — нельзя!
```

Правило: если `a == b`, то `hash(a) == hash(b)`. Список изменяемый — его хеш менялся бы, словарь сломался бы.

-----

## 💻 Быстрая шпаргалка

```python
# Проверить хеш
hash('name')      # число
hash(42)          # 42
hash(1) == hash(1.0)  # True! (1 == 1.0)

# Список нельзя хешировать
hash([1,2,3])     # TypeError!

# Поиск O(1) — не зависит от размера словаря
d = {i: i for i in range(1000000)}
d[999999]  # мгновенно!
```
