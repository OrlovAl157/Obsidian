## 🎯 Главная идея

Словарь в Python использует **две отдельные структуры данных**:

1. **`dk_indices`** — хэш-таблица (карта индексов)
2. **`entries`** — массив записей (хранилище данных)

-----

## ⚡ Зачем это нужно?

✅ **Быстрый поиск** — через хэш-таблицу O(1)  
✅ **Сохранение порядка** — через массив записей  
✅ **Экономия памяти** — индексы занимают меньше места, чем полные записи  
✅ **Эффективное изменение размера** — при росте словаря переносятся только индексы

-----

## 📊 Пример: словарь `{"A": 10, "B": 20, "C": 30}`

### 1️⃣ Массив `entries` (хранилище в порядке вставки)

```
index |     hash       | key | value
----------------------------------------
  0   | 6385194080     | "A" |  10
  1   | 6385194081     | "B" |  20
  2   | 6385194082     | "C" |  30
```

**Важно:** Хэши реальные, на вашем Python могут отличаться!

-----

### 2️⃣ Хэш-таблица `dk_indices` (карта индексов)

Размер таблицы: 8 слотов (степень двойки)

```
slot | entry index | что это означает
--------------------------------------
 0   |  -1         | пустой слот
 1   |   2         | → entries[2] = "C"
 2   |  -1         | пустой слот
 3   |   0         | → entries[0] = "A"
 4   |  -1         | пустой слот
 5   |  -1         | пустой слот
 6   |   1         | → entries[1] = "B"
 7   |  -1         | пустой слот
```

**Ключевой момент:** `-1` означает “пустой слот”, числа — это **индексы в массиве entries**

-----

## 🔍 КРИТИЧЕСКИ ВАЖНО

### ❌ Хэш-таблица НЕ содержит:

- ❌ Хэши ключей
- ❌ Сами ключи
- ❌ Значения

### ✅ Хэш-таблица содержит ТОЛЬКО:

- ✅ Индексы (маленькие числа: int16 или int32)
- ✅ Указатели на записи в `entries`

### 💾 Где хранятся хэши?

**В массиве `entries`!** Каждая запись содержит:

- `me_hash` — хэш ключа (64-битное число)
- `me_key` — сам ключ
- `me_value` — значение

-----

## 🎨 Визуальная схема

```
         dk_indices (хэш-таблица)
         ┌─────────────────────┐
slot 0:  │  -1                 │
slot 1:  │   2  ───────────────┼─────┐
slot 2:  │  -1                 │     │
slot 3:  │   0  ───────────┐   │     │
slot 4:  │  -1             │   │     │
slot 5:  │  -1             │   │     │
slot 6:  │   1  ──────┐    │   │     │
slot 7:  │  -1        │    │   │     │
         └────────────┼────┼───┼─────┘
                      ▼    ▼   ▼
         entries (массив записей)
         ┌──────────────────────────┐
index 0: │ {6385194080, "A", 10}    │ ← порядок
index 1: │ {6385194081, "B", 20}    │   вставки
index 2: │ {6385194082, "C", 30}    │   сохранён!
         └──────────────────────────┘
```

-----

## 🔄 Как работает поиск ключа?

### Пример: `d["B"]`

1. **Вычислить хэш:** `hash("B")` → `6385194081`
2. **Найти слот:** `6385194081 & (8 - 1)` → `6` (побитовое И с маской)
3. **Прочитать индекс:** `dk_indices[6]` → `1`
4. **Получить запись:** `entries[1]` → `{hash, "B", 20}`
5. **Проверить ключ:** `"B" == "B"` ✅
6. **Вернуть значение:** `20`

-----

## 🔄 Как работает обход словаря?

### Пример: `for key in d:`

```python
# Обход идёт ПО ПОРЯДКУ entries!
for i in range(len(entries)):
    yield entries[i].key
```

**Результат:** `"A"` → `"B"` → `"C"` (порядок вставки сохранён!)

-----

## 💡 Почему такая структура?

### 1. **Экономия памяти**

- Индекс (int16): 2 байта
- Хэш (int64): 8 байт
- Экономия: **в 4 раза** на больших словарях!

### 2. **Быстрое изменение размера**

При росте словаря:

- ❌ Не нужно копировать все `{hash, key, value}`
- ✅ Достаточно пересчитать индексы

### 3. **Сохранение порядка**

- Обход словаря идёт по `entries` (порядок вставки)
- Поиск идёт по `dk_indices` (быстрый доступ)

### 4. **Меньше коллизий**

Хэш-таблица остаётся компактной и эффективной

-----

## 🧪 Пример с кодом

```python
# Создаём словарь
d = {"A": 10, "B": 20, "C": 30}

# Внутренняя структура (концептуально):

# entries (порядок вставки):
entries = [
    {"hash": hash("A"), "key": "A", "value": 10},  # index 0
    {"hash": hash("B"), "key": "B", "value": 20},  # index 1
    {"hash": hash("C"), "key": "C", "value": 30},  # index 2
]

# dk_indices (хэш-таблица с индексами):
dk_indices = [-1, 2, -1, 0, -1, -1, 1, -1]
#             ↑   ↑      ↑           ↑
#           пусто C    пусто A     пусто B

# Поиск d["B"]:
slot = hash("B") & 7  # допустим → 6
index = dk_indices[6]  # → 1
entry = entries[1]     # → {"B": 20}
result = entry["value"]  # → 20
```

-----

## 📌 Итоговая шпаргалка

|Компонент         |Что хранит                 |Зачем нужен               |
|------------------|---------------------------|--------------------------|
|**`dk_indices`**  |Индексы (int16/int32)      |Быстрый поиск по хэшу     |
|**`entries`**     |`{hash, key, value}`       |Хранение данных + порядок |
|**Размер таблицы**|Степень двойки (8, 16, 32…)|Быстрое вычисление слота  |
|**`-1` в таблице**|Пустой слот                |Показывает свободные места|

-----

## 🎯 Главное правило

> **Хэш-таблица НЕ хранит хэши!**  
> Она хранит **индексы**, указывающие на записи,  
> где хэши **действительно** находятся.

-----

## 🔑 Формулы

### Вычисление слота:

```python
slot = hash(key) & (table_size - 1)
# Эквивалентно: hash(key) % table_size
# Но побитовое И быстрее!
```

### Разрешение коллизий (linear probing):

```python
slot = (slot + 1) & (table_size - 1)
# Переходим к следующему слоту
```

## 📚 Дополнительная информация

**📌 Версии Python:**

- До Python 3.6: порядок не гарантировался
- Python 3.6: порядок как “деталь реализации”
- Python 3.7+: порядок гарантирован официально

**🗂️ Типы индексов:**

- Маленькие словари (< 256 элементов): `int8`
- Средние (< 65536): `int16`
- Большие: `int32` или `int64`

## 🎓 Проверьте себя

1. **Где хранится хэш ключа?** → В `entries`
2. **Что содержит `dk_indices`?** → Индексы
3. **Как сохраняется порядок вставки?** → Через массив `entries`
4. **Почему это эффективно?** → Индексы занимают меньше памяти
