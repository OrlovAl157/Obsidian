# 📦 Атрибуты объектов и классов

## 📌 Коротко

> Атрибуты — это данные, хранящиеся в объектах и классах. Атрибуты экземпляра у каждого свои, атрибуты класса — общие для всех.

---

## 📖 Теория

### Всё в Python — объект

Python поддерживает ООП на 100%: все данные в нём являются объектами. Числа, строки, списки, словари, функции, модули и даже сами типы данных — всё это объекты. Поэтому их можно присваивать переменным, помещать в списки, хранить в словарях, передавать в функции в качестве аргументов.

**Пример 1: Встроенные типы**

```python
num = 10
flag = True
text = 'OOP'

print(type(num))    # → <class 'int'>
print(type(flag))   # → <class 'bool'>
print(type(text))   # → <class 'str'>
```

**Пример 2: Сложные объекты**

```python
import math

def cube(num):
    return num ** 3

print(type(math))             # → <class 'module'>
print(type(cube))             # → <class 'function'>
print(type(print))            # → <class 'builtin_function_or_method'>
print(type(int))              # → <class 'type'>
```

> 💡 Тип данных и класс — это одно и то же.

### Создание класса и объекта

Классы создаются с помощью инструкции `class`, за которой следует имя класса и двоеточие. Тело класса пишется с отступом.

```python
class <НазваниеКласса>:
    <тело класса>
```

**Минимальный класс:**

```python
class Cat:
    pass

# Создание объекта (экземпляра класса)
cat = Cat()
```

Объект класса также называют экземпляром класса — это одно и то же. Термин "экземпляр" используется чаще, так как сам класс тоже является объектом.

**Соглашения об именовании (PEP 8):**

|Что|Стиль|Пример|
|---|---|---|
|Класс|UpperCamelCase|`BankAccount`, `MyClass`|
|Атрибут / метод|snake_case|`paws_count`, `night_vision`|

> ⚠️ Встроенные классы (`int`, `str`, `list` и др.) соглашению UpperCamelCase не следуют — это историческое исключение.

### Атрибуты объекта (экземпляра)

Атрибуты экземпляра принадлежат конкретному объекту и хранят его индивидуальное состояние. Каждый объект имеет свой собственный набор атрибутов.

**Создание и присвоение атрибутов:**

```python
class Cat:
    pass

cat1 = Cat()
cat2 = Cat()

# Устанавливаем атрибуты каждому объекту отдельно
cat1.breed = 'Британский'
cat1.name = 'Кемаль'
cat1.age = 1

cat2.name = 'Роджер'
cat2.age = 5
# cat2.breed не задан — у каждого объекта свой набор атрибутов!
```

**Чтение и изменение атрибутов:**

```python
cat = Cat()
cat.breed = 'Британский'
cat.name = 'Кемаль'
cat.age = 1

print(cat.breed, cat.name)   # → Британский Кемаль

cat.age += 2                 # изменение атрибута
print(cat.age)               # → 3
```

**Ошибка при обращении к несуществующему атрибуту:**

```python
print(cat.color)             # ❌ AttributeError: 'Cat' object has no attribute 'color'
```

### Атрибуты класса

Атрибуты класса задаются в теле класса и являются общими для всех его экземпляров. Их значения одинаковы для всех объектов этого класса.

**Определение и доступ:**

```python
class Cat:
    night_vision = True    # атрибут класса
    paws_count = 4         # атрибут класса

cat1 = Cat()
cat2 = Cat()

# Доступ через экземпляр
print(cat1.night_vision)   # → True
print(cat2.paws_count)     # → 4

# Доступ напрямую через класс
print(Cat.night_vision)    # → True
print(Cat.paws_count)      # → 4
```

---

### ⚠️ Тонкость: изменение атрибута класса

Это одна из самых частых ловушек в ООП на Python.

**Изменение через класс — затрагивает все экземпляры:**

```python
class Cat:
    night_vision = True

cat1 = Cat()
cat2 = Cat()

Cat.night_vision = False       # меняем через класс

print(cat1.night_vision)       # → False
print(cat2.night_vision)       # → False
```

**Изменение через экземпляр — создаёт новый атрибут только у этого экземпляра:**

```python
class Cat:
    night_vision = True

cat1 = Cat()
cat2 = Cat()

cat1.night_vision = False      # кажется, что меняем атрибут класса...

print(cat1.night_vision, cat1.__dict__)   # → False {'night_vision': False}
print(cat2.night_vision, cat2.__dict__)   # → True  {}
# cat2 не изменился! У cat1 появился собственный атрибут с тем же именем
```

> 💡 Порядок поиска атрибута: сначала Python ищет в `__dict__` экземпляра, затем в `__dict__` класса. Поэтому атрибут экземпляра "перекрывает" атрибут класса с тем же именем.

---

### Атрибут __dict__

Все пользовательские атрибуты хранятся в специальном словаре `__dict__`. Это обычный словарь Python, с которым можно работать напрямую.

**__dict__ экземпляра** — содержит только атрибуты этого объекта:

```python
class Cat:
    pass

cat = Cat()
cat.breed = 'Британский'
cat.name = 'Кемаль'
cat.age = 1

print(cat.__dict__)
# → {'breed': 'Британский', 'name': 'Кемаль', 'age': 1}
```

**__dict__ класса** — содержит атрибуты класса и служебные атрибуты:

```python
class Cat:
    night_vision = True
    paws_count = 4

print(Cat.__dict__)
# → {'__module__': '__main__', 'night_vision': True, 'paws_count': 4,
#    '__dict__': ..., '__weakref__': ..., '__doc__': None}
```

> 💡 Помимо пользовательских атрибутов (`night_vision`, `paws_count`), `Cat.__dict__` содержит служебные атрибуты — `__module__`, `__doc__` и другие.

**Дополнительные полезные атрибуты:**

```python
class Cat:
    """Класс, описывающий кошку"""
    pass

cat = Cat()

# Тип объекта
print(type(cat))          # → <class '__main__.Cat'>
print(cat.__class__)      # → <class '__main__.Cat'>
print(type(cat) == cat.__class__)  # → True

# Строка документации класса
print(Cat.__doc__)        # → Класс, описывающий кошку
```

### Функция dir()

Возвращает список всех атрибутов и методов объекта — пользовательских, унаследованных и служебных (dunder-методов).

```python
from datetime import date

today = date(2022, 9, 25)
print(dir(today))
# → ['__add__', '__class__', ..., 'day', 'month', 'year', 'isoweekday', ...]
```

```python
class Cat:
    night_vision = True

cat = Cat()
cat.name = 'Кемаль'

print(dir(cat))
# → [..., '__class__', '__dict__', '__init__', ..., 'name', 'night_vision']
```

> 💡 `dir()` — удобный инструмент для исследования объекта в интерактивном режиме или при отладке.

### Функции для работы с атрибутами

Альтернатива точечной нотации. Особенно полезны, когда имя атрибута хранится в переменной или передаётся динамически.

**getattr(obj, name, default)**

Возвращает значение атрибута. Если атрибута нет — возвращает `default`. Если `default` не указан — выбрасывает `AttributeError`.

```python
class Cat:
    night_vision = True

cat = Cat()
cat.name = 'Кемаль'

print(getattr(cat, 'name'))           # → Кемаль
print(getattr(cat, 'age', None))      # → None (нет атрибута, но нет и ошибки)
print(getattr(cat, 'age'))            # ❌ AttributeError

# Работает и с атрибутами класса
print(getattr(cat, 'night_vision'))   # → True
```

**setattr(obj, name, value)**

Устанавливает атрибут. Если атрибут уже есть — перезаписывает значение.

```python
cat = Cat()
cat.name = 'Кемаль'

setattr(cat, 'age', 1)           # добавляем новый атрибут
setattr(cat, 'name', 'Роджер')   # перезаписываем существующий

print(cat.age)    # → 1
print(cat.name)   # → Роджер
```

**hasattr(obj, name)**

Возвращает `True` если атрибут существует, `False` если нет. Видит и атрибуты класса.

```python
class Cat:
    night_vision = True

cat = Cat()
cat.name = 'Кемаль'

print(hasattr(cat, 'name'))          # → True
print(hasattr(cat, 'age'))           # → False
print(hasattr(cat, 'night_vision'))  # → True  (атрибут класса тоже виден)
```

**delattr(obj, name)**

Удаляет атрибут у объекта. Если атрибута нет — выбрасывает `AttributeError`.

```python
cat = Cat()
cat.name = 'Кемаль'
cat.age = 1

print(cat.__dict__)    # → {'name': 'Кемаль', 'age': 1}

delattr(cat, 'name')
delattr(cat, 'age')

print(cat.__dict__)    # → {}

delattr(cat, 'name')   # ❌ AttributeError — атрибута уже нет
```

> ⚠️ `delattr()` работает только с атрибутами экземпляра. Удалить атрибут класса через экземпляр не получится — будет `AttributeError`.

**Сравнение: точечная нотация vs функции**

|Задача|Точечная нотация|Функция|
|---|---|---|
|Получить|`cat.name`|`getattr(cat, 'name', None)`|
|Установить|`cat.name = 'Рекс'`|`setattr(cat, 'name', 'Рекс')`|
|Проверить|`'name' in cat.__dict__`|`hasattr(cat, 'name')`|
|Удалить|`del cat.name`|`delattr(cat, 'name')`|

**Когда использовать функции вместо точечной нотации:**

```python
# Имя атрибута хранится в переменной — точечная нотация не сработает
attr_name = 'name'
# cat.attr_name  ← ищет атрибут с именем "attr_name", а не "name"!
getattr(cat, attr_name)  # ✅ работает правильно

# Динамическое создание атрибутов из списка
fields = ['name', 'age', 'breed']
values = ['Кемаль', 1, 'Британский']

cat = Cat()
for field, value in zip(fields, values):
    setattr(cat, field, value)

print(cat.__dict__)  # → {'name': 'Кемаль', 'age': 1, 'breed': 'Британский'}
```

## Быстрая шпаргалка

```python
class Cat:
    # Атрибут класса — общий для всех
    paws_count = 4

cat = Cat()

# Атрибут экземпляра — только у этого объекта
cat.name = 'Кемаль'

# Доступ
cat.name              # точечная нотация
Cat.paws_count        # атрибут класса через класс

# Исследование
cat.__dict__          # {'name': 'Кемаль'}
Cat.__dict__          # {'paws_count': 4, ...служебные...}
Cat.__doc__           # строка документации
cat.__class__         # <class '__main__.Cat'>
dir(cat)              # всё: атрибуты + методы + унаследованное

# Функции
getattr(cat, 'name', None)    # безопасное чтение → 'Кемаль'
setattr(cat, 'age', 3)        # установка
hasattr(cat, 'name')          # проверка → True
delattr(cat, 'name')          # удаление
```
