# 🏛 Абстракция

## 📌 Коротко

Абстракция — принцип ООП, который описывает **что** объект должен делать, скрывая **как** он это делает. В Python реализуется через абстрактные классы (`ABC`) и декоратор `@abstractmethod`.

-----

## 🧠 Суть принципа

Абстракция — это **контракт для потомков**. Абстрактный класс говорит:

> “Вот что ты **обязан** уметь делать — а как именно, решай сам.”

Аналогия: птицы умеют летать и есть — это абстракция. Машет ли птица крыльями или планирует — это конкретная реализация в подклассе.

|Абстрактный класс       |Подкласс                |
|------------------------|------------------------|
|Описывает **что** делать|Реализует **как** делать|
|Не создаёт экземпляры   |Создаёт экземпляры      |
|Задаёт контракт         |Выполняет контракт      |

-----

## ⚠️ Важные отличия

> ⚠️ **Абстракция ≠ Инкапсуляция**
> 
> - Абстракция скрывает **детали реализации** (пользователь не знает как работает метод внутри)
> - Инкапсуляция скрывает **внутренние данные** (через приватные атрибуты и интерфейсы)

> ⚠️ **Абстрактный класс ≠ просто базовый класс**
> 
> - Обычный базовый класс: можно создать экземпляр, потомки могут не переопределять методы
> - Абстрактный класс: нельзя создать экземпляр, `@abstractmethod` обязателен к реализации

-----

## 💻 Синтаксис

```python
from abc import ABC, abstractmethod

class Calculator(ABC):
    def __init__(self):
        self.history = []  # обычный атрибут — наследуется всеми потомками

    @abstractmethod
    def add(self, a, b): pass  # → обязан быть реализован в потомке

    @abstractmethod
    def multiply(self, a, b): pass  # → обязан быть реализован в потомке


class SimpleCalculator(Calculator):
    def __init__(self):
        super().__init__()  # → вызываем __init__ родителя чтобы получить self.history

    def add(self, a, b):
        result = a + b
        self.history.append(result)  # → используем атрибут из абстрактного класса
        return result

    def multiply(self, a, b):
        return a * b


# calc = Calculator()  # → TypeError: Can't instantiate abstract class
calc = SimpleCalculator()  # → OK
print(calc.add(2, 3))  # → 5
```

-----

## 🔍 Что будет если не реализовать абстрактный метод

```python
class BrokenCalculator(Calculator):
    def add(self, a, b):
        return a + b
    # multiply не реализован!

b = BrokenCalculator()  # → TypeError: Can't instantiate abstract class
                        #   BrokenCalculator with abstract method multiply
```

> 💡 Ошибка возникает при создании **экземпляра**, не при объявлении класса.

## 📐 Реальные примеры использования

```python
# Пример 1 — Животные
class Animal(ABC):
    @abstractmethod
    def speak(self): pass  # → каждое животное издаёт свой звук

class Dog(Animal):
    def speak(self):
        return "Гав!"  # → конкретная реализация

class Cat(Animal):
    def speak(self):
        return "Мяу!"

# Полиморфизм через абстракцию
animals = [Dog(), Cat()]
for animal in animals:
    print(animal.speak())  # → Гав! Мяу!


# Пример 2 — Банк
class BankAccount(ABC):
    @abstractmethod
    def deposit(self, amount): pass

    @abstractmethod
    def withdraw(self, amount): pass

class SavingsAccount(BankAccount):
    def deposit(self, amount):
        ...
    def withdraw(self, amount):
        ...
```

## 💻 Быстрая шпаргалка

```python
from abc import ABC, abstractmethod

class MyAbstract(ABC):          # наследуемся от ABC
    @abstractmethod
    def must_implement(self):   # обязательный метод
        pass

    def optional(self):         # обычный метод — наследуется как есть
        return "я уже реализован"

# MyAbstract()  # → ошибка!

class Concrete(MyAbstract):
    def must_implement(self):   # реализуем обязательный метод
        return "готово"

obj = Concrete()  # → OK
```
