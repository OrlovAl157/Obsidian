# 🐍 Магические методы: `__new__`, `__init__`, `__del__`

## 📌 Что такое магические методы?

Магические методы (dunder-методы, от **D**ouble **Under**score) — специальные методы в Python, обрамлённые двойным подчёркиванием: `__init__()`, `__new__()`, `__del__()`.

- Не вызываются напрямую — срабатывают **автоматически** при определённых действиях
- Добавляют в класс **специальный функционал**
- Создавать новые нельзя — можно только **реализовывать существующие**

---

## 🏗️ `__new__()` — создание объекта

**Первый шаг** при конструировании экземпляра. Создаёт и **возвращает** новый пустой объект.

```python
class MyClass:
    def __new__(cls, *args, **kwargs):
        instance = object.__new__(cls)  # или super().__new__(cls)
        return instance
```

### Важные особенности

|Факт|Пояснение|
|---|---|
|Первый аргумент — `cls`|Принимает сам класс, а не экземпляр|
|Технически — статический метод|Но `@staticmethod` не нужен — Python делает это сам|
|Должен вернуть экземпляр своего класса|Иначе `__init__` **не будет вызван**|
|Аргументы при вызове класса|Передаются **и в `__new__`**, и в `__init__`**|

```python
# ❌ __new__ вернул не тот тип → __init__ не вызовется
class MyClass:
    def __new__(cls, *args, **kwargs):
        return 'instance'  # возвращаем str, а не MyClass → __init__ пропускается

    def __init__(self):
        print('Этот принт не выведется')
```

---

## ⚙️ `__init__()` — инициализация объекта

**Второй шаг**. Получает уже созданный объект и устанавливает ему атрибуты.

```python
class Cat:
    def __init__(self, name):
        self.name = name  # устанавливаем атрибуты
```

### Правило

> `__init__()` **всегда должен возвращать `None`**. Явный `return self` вызовет `TypeError`.

---

## 🔄 Порядок вызовов

```
MyClass()  →  __new__() создаёт объект  →  __init__() инициализирует  →  готовый объект
```

```python
class Cat:
    def __new__(cls, *args, **kwargs):
        print('1. Создание')
        return object.__new__(cls)

    def __init__(self, name):
        print('2. Инициализация')
        self.name = name

# Ручное воспроизведение того же процесса:
cat = Cat.__new__(Cat)
Cat.__init__(cat, 'Кемаль')
```

## 🔗 `super()` vs `object.__new__(cls)`

```python
# Оба варианта равнозначны:
instance = object.__new__(cls)
instance = super().__new__(cls)
```

- **`super()`** — используй при наследовании и множественном наследовании
- **`object.__new__(cls)`** — явное обращение к базовому классу, удобно при паттернах проектирования

## 🎯 Паттерн Singleton

Объект существует **только в одном экземпляре**.

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = object.__new__(cls)
        return cls._instance

first = Singleton()
second = Singleton()
print(first is second)  # True — один и тот же объект
```

---

## 🎯 Паттерн Monostate ⭐ (бонус для собеса)

Несколько объектов, но **одно общее состояние**. Не требует `__new__`.

```python
class Monostate:
    _shared_state = {}

    def __init__(self):
        self.__dict__ = self._shared_state
        if not self._shared_state:
            self._do_work()        # выполняется только 1 раз
            self.empty = False

    def _do_work(self):
        print('Ресурсоёмкая задача')

one = Monostate()
two = Monostate()
print(one is two)        # False — разные объекты
one.data = 20
two.data = 55
print(one.__dict__)      # {'empty': False, 'data': 55} — состояние общее!
```

||Singleton|Monostate|
|---|---|---|
|Кол-во объектов|1|Много|
|Состояние|Общее|Общее|
|Метод|`__new__`|`__init__` + `__dict__`|

## 🗑️ `__del__()` — финализатор

Вызывается **автоматически**, когда счётчик ссылок на объект становится равным нулю. Используется для финальных действий перед уничтожением объекта.

```python
class MyClass:
    def __del__(self):
        print('Bye')  # закрыть файл, соединение, логировать и т.д.

obj = MyClass()
del obj  # → 'Bye'
```

> **`del obj`** — удаляет не объект, а **связь между именем и объектом** (уменьшает счётчик ссылок).

### Счётчик ссылок

```python
obj1 = MyClass()
obj2 = obj1       # счётчик = 2

del obj1          # счётчик = 1, __del__ НЕ вызывается
del obj2          # счётчик = 0, __del__ вызывается → 'Bye'
```

### Локальные переменные функции

```python
def func():
    obj = MyClass()   # создаётся при вызове
# После завершения func() → obj удаляется → счётчик = 0 → __del__

func()  # выведет 'Bye'
```

## ⚠️ Важные нюансы

### Почему `__del__` может не вызваться

```python
class MyClass:
    def __new__(cls):
        return cls      # ❌ возвращаем сам класс, не экземпляр

cl = MyClass()
del cl          # 'Bye' НЕ выведется!
```

Причина: `cl = MyClass` (ссылка на класс), экземпляр не создавался → `__del__` некому вызывать.

### `__del__` — НЕ сборщик мусора

- `__del__` — это **финализатор**: выполняет финальные действия _перед_ удалением
- Само удаление делает внутренний механизм CPython (garbage collector)
- `__del__` может не вызваться в некоторых ситуациях (циклические ссылки, завершение интерпретатора)

### Практические применения `__del__`

- Закрытие файлов / сетевых соединений
- Логирование удаления объекта
- Удаление объекта из кэша
- В играх: действие юнита перед смертью

---

## 🧩 Шпаргалка

```
object.__new__(cls)  →  создаёт пустой экземпляр
__new__(cls, ...)    →  вызывается первым, возвращает объект
__init__(self, ...) →  вызывается вторым, инициализирует атрибуты, возвращает None
__del__(self)        →  финализатор, вызывается при счётчик_ссылок == 0
del obj              →  удаляет ссылку, уменьшает счётчик (не удаляет объект напрямую)
```
