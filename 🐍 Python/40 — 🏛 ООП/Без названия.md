# 🏗 Порядок создания экземпляра — `type.__call__`

## 📌 Коротко

> `Line()` → `type.__call__` → `__new__` (создаёт) → `__init__` (инициализирует) → возвращает экземпляр. `l()` → ищет `__call__` через MRO класса — если нет нигде, `TypeError`.

---

## 🔍 Что делает `type.__call__` внутри

```python
# Точнее type.__call__ выглядит так:
def __call__(cls, *args, **kwargs):
    obj = cls.__new__(cls, *args, **kwargs)    # 1. создаём объект

    if isinstance(obj, cls):                   # 2. ⚠️ важная проверка!
        cls.__init__(obj, *args, **kwargs)     # 3. наполняем атрибутами

    return obj                                 # 4. возвращаем готовый объект
```

> ⚠️ `__init__` вызывается **только если** `__new__` вернул экземпляр именно этого класса! Если `__new__` вернул что-то другое — `__init__` не вызовется.

```python
# Пример — __new__ возвращает не экземпляр cls:
class A:
    def __new__(cls):
        return 42        # возвращаем не экземпляр A!

    def __init__(self):
        print('__init__ вызван')   # это НЕ вызовется!

a = A()
print(a)   # → 42  (__init__ пропущен!)
```

### `__new__` vs `__init__`

||`__new__`|`__init__`|
|---|---|---|
|Принимает|`cls`|`self`|
|Роль|**фабрика** — создаёт и возвращает экземпляр|**инициализатор** — наполняет атрибутами|
|До/после|до `__init__` — объекта ещё нет|после `__new__` — объект уже есть|
|Возвращает|**новый объект**|`None`|

**Почему `cls` а не `self`:**

- `__new__` получает `cls` — объекта ещё нет, нужно знать **какой класс** создаёт объект
- `__init__` получает `self` — объект уже создан `__new__`, `self` это и есть тот самый объект

```python
class Point:
    def __new__(cls, x, y):
        print(f'__new__ вызван, cls = {cls}')
        obj = super().__new__(cls)   # создаём пустой объект
        return obj                   # передаём в __init__

    def __init__(self, x, y):
        print(f'__init__ вызван, self = {self}')
        self.x = x
        self.y = y

p = Point(1, 2)
# → __new__ вызван, cls = <class 'Point'>
# → __init__ вызван, self = <Point object>
```

## 📖 Два разных `__call__`

### 1. `Line()` — создание экземпляра

```
Line()
  │
  ▼
Line — объект метакласса type
  │
  ▼
type.__call__(Line)
  │
  ├── obj = Line.__new__(Line)        ← создаёт объект
  ├── isinstance(obj, Line)?
  │     ДА → Line.__init__(obj)       ← инициализирует
  │     НЕТ → __init__ пропускается
  └── return obj → l
```

### 2. `l()` — вызов экземпляра

```
l()
  │
  ▼
Python ищет __call__ через MRO класса Line:
Line.__dict__ → Geom.__dict__ → object.__dict__
  │
  ├── найден → вызываем Line.__call__(l) ✅
  │
  └── не найден нигде → TypeError ❌
```

> 💡 Поиск `__call__` идёт через MRO класса, а не "у экземпляра напрямую".

---

## 📖 Два уровня `__call__`

|Вызов|Чей `__call__`|Где ищем|
|---|---|---|
|`Line()`|`type.__call__`|метакласс `type`|
|`l()`|`Line.__call__`|через MRO: `Line → Geom → object`|

> 💡 `object` не реализует `__call__` — поэтому экземпляры по умолчанию **не вызываемы**. Нужно явно определить `__call__` в классе.

## 🗃 Что хранит `type.__dict__`

```python
print(type.__dict__.keys())
```

Ключевые методы:

|Метод|Назначение|
|---|---|
|`__call__`|вызывается при `Line()` — запускает создание экземпляра|
|`__new__`|выделяет память под новый экземпляр|
|`__init__`|инициализация самого **класса** (не экземпляра!)|
|`__getattribute__`|доступ к атрибутам класса|
|`__setattr__`|установка атрибутов класса|
|`__delattr__`|удаление атрибутов класса|
|`mro`|возвращает порядок разрешения методов|
|`__subclasses__`|список дочерних классов|
|`__bases__`|кортеж родительских классов|
|`__name__`|имя класса|
|`__dict__`|словарь атрибутов класса|

## 💻 Примеры

### `__call__` у экземпляра

```python
class Line:
    def __call__(self):
        print("вызвали экземпляр!")

l = Line()
l()   # → "вызвали экземпляр!"
```

### Без `__call__` — ошибка

```python
class Line:
    pass

l = Line()
l()   # → TypeError: 'Line' object is not callable
```

### Проверить вызываемость

```python
callable(Line)   # → True  (у type есть __call__)
callable(l)      # → False (у Line нет __call__)
```
