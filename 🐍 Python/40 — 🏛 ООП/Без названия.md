# 🏛 Когда использовать ООП

## 📌 Коротко

ООП оправдано когда у сущности есть **состояние** (данные) и **поведение** (методы) связанные между собой. Если этого нет — достаточно функций.

-----

## ✅ Используй ООП когда

- Есть **состояние** — данные которые живут и меняются между вызовами
- Есть **поведение** тесно связанное с этими данными
- Нужно создавать **множество похожих объектов**
- Нужно **наследование** и переиспользование кода
- Сущность моделирует реальный объект с характеристиками и действиями

```python
# ✅ Хороший кандидат для класса
class BankAccount:
    def __init__(self, balance):
        self.balance = balance  # состояние

    def deposit(self, amount):  # поведение
        self.balance += amount

    def withdraw(self, amount):  # поведение
        self.balance -= amount
```

-----

## ❌ Используй функции когда

- Простая задача — одно действие
- Нет состояния которое нужно хранить между вызовами
- Функции не связаны между собой
- Код небольшой и однозадачный

```python
# ✅ Достаточно функции
def calculate_tax(income, rate):
    return income * rate
```

-----

## 🔑 Ключевой критерий — состояние

> 💡 Задай себе вопрос: **“Нужно ли мне хранить данные между вызовами?”**
> 
> - ДА → скорее всего нужен класс
> - НЕТ → достаточно функции

-----

## 📊 Сравнение

|Критерий         |Функции   |ООП    |
|-----------------|----------|-------|
|Состояние        |❌ нет     |✅ есть |
|Похожие объекты  |❌         |✅      |
|Наследование     |❌         |✅      |
|Простота         |✅ проще   |сложнее|
|Переиспользование|ограничено|✅      |
