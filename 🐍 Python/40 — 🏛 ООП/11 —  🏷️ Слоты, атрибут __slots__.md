# 🏷️ Слоты, атрибут __slots__

## 📌 Коротко

> `__slots__` — способ ограничить набор допустимых атрибутов экземпляров и отказаться от `__dict__` ради памяти и скорости доступа. После объявления слотов:
> - экземпляры не получают `__dict__` (если специально не включить),
> - нельзя динамически добавлять атрибуты, не перечисленные в `__slots__`,
> - наследники могут добавить свои слоты (и тогда тоже обходятся без `__dict__`), а могут оставить словарь.

---

## 📖 Теория

### Зачем нужны слоты

По умолчанию атрибуты экземпляра хранятся в словаре `__dict__`. Это гибко, но дорого по памяти (и слегка медленнее по доступу), особенно когда экземпляров много.

Если заранее известен фиксированный набор атрибутов — можно заменить словарь более компактной структурой через `__slots__`.

### Атрибут `__slots__`

Слоты — это атрибут *класса*:

```python
class Point:
    __slots__ = ("x", "y")

    def __init__(self, x, y):
        self.x = x
        self.y = y
```

Имя слота — строка, удовлетворяющая правилам идентификаторов Python. Значение `__slots__` обычно задают кортежем, но формально может быть любая итерируемая коллекция строк.

Последствия:

```python
point = Point(1, 2)

print(Point.__slots__)   # ('x', 'y')
print(point.__slots__)   # ('x', 'y')

print(point.__dict__)    # ❌ AttributeError: 'Point' object has no attribute '__dict__'
point.z = 3              # ❌ AttributeError: 'Point' object has no attribute 'z'
```

### Слоты при наследовании

Если базовый класс со слотами, а наследник слотов не объявляет — у наследника появится `__dict__` для своих новых атрибутов:

```python
class Point2D:
    __slots__ = ("x", "y")

class Point3D(Point2D):
    def __init__(self, x, y, z):
        super().__init__(x, y)
        self.z = z

point = Point3D(1, 2, 3)
print(point.__slots__)  # ('x', 'y')
print(point.__dict__)   # {'z': 3}
```

Чтобы и наследник тоже работал через слоты, объявляем `__slots__` в нём (перечисляя *только новые* атрибуты — базовые слоты наследуются автоматически):

```python
class Point3D(Point2D):
    __slots__ = ("z",)

point = Point3D(1, 2, 3)
print(point.__dict__)   # ❌ AttributeError
```

### Сравнение производительности

Примерное сравнение (зависит от окружения, размера кэша и т.п.) показывает ускорение доступа к атрибутам порядка 15–20%:

```python
from timeit import repeat

def get_set_test(cls):
    obj = cls(0, 0)

    def get_set():
        obj.x += 1
        obj.y += 2

    return get_set

print(max(repeat(get_set_test(PointWithoutSlots))))
print(max(repeat(get_set_test(PointWithSlots))))
# 0.11013359995558858
# 0.09572770004160702
```

По памяти (например, через `pympler.asizeof`) экземпляр со слотами обычно существенно легче за счёт отсутствия `__dict__`.

## 💬 Полезное (из обсуждений)

- Хотите и слоты, и возможность добавлять произвольные атрибуты — включите `__dict__` в `__slots__`.
- Нужна поддержка слабых ссылок (`weakref`) — добавьте `'__weakref__'` в `__slots__`.
- Эффект по памяти заметен только при *большом числе* экземпляров; ради 1–2 объектов использовать `__slots__` чаще всего не стоит.
