# 🎨 Декораторы функций, методов и классы-декораторы

## 💡 Декоратор — что это?

**Декоратор** — это функция, которая принимает другую функцию (или метод), добавляет ей новый функционал и возвращает модифицированную версию.

```python
@my_decorator
def hello():
    return "Hello"

# Это то же самое, что:
def hello():
    return "Hello"
hello = my_decorator(hello)
```

**Главная идея:** Декоратор создаёт **wrapper** (обёртку) вокруг функции.

---

## 🏗️ Базовый шаблон декоратора

```python
import functools

def my_decorator(func):
    @functools.wraps(func)  # Сохраняет __name__, __doc__ и другие метаданные
    def wrapper(*args, **kwargs):
        # КОД ДО вызова функции
        print(f"Вызываю {func.__name__}")
        
        result = func(*args, **kwargs)  # Вызов исходной функции
        
        # КОД ПОСЛЕ вызова функции
        print(f"{func.__name__} завершена")
        return result
    
    return wrapper

@my_decorator
def greet(name):
    return f"Hello, {name}!"

greet("Alice")
# Вывод:
# Вызываю greet
# Hello, Alice!
# greet завершена
```

---

## ⚠️ Важно: functools.wraps

### ❌ БЕЗ @functools.wraps — потеря метаданных

```python
def bad_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@bad_decorator
def greet(name):
    """Функция приветствия"""
    return f"Hello, {name}!"

print(greet.__name__)  # → 'wrapper' ❌
print(greet.__doc__)   # → None ❌
```

### ✅ С @functools.wraps — метаданные сохранены

```python
import functools

def good_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@good_decorator
def greet(name):
    """Функция приветствия"""
    return f"Hello, {name}!"

print(greet.__name__)  # → 'greet' ✅
print(greet.__doc__)   # → 'Функция приветствия' ✅
```

**💡 Правило:** Всегда используй `@functools.wraps(func)` в своих декораторах!

## 🎯 Простой декоратор: do_twice

Декоратор, который вызывает функцию дважды:

```python
import functools

def do_twice(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        for _ in range(2):
            value = func(*args, **kwargs)
        return value
    return wrapper

@do_twice
def greet(name):
    print(f'Привет, {name}!')

greet('Кемаль')
# Вывод:
# Привет, Кемаль!
# Привет, Кемаль!
```

**Как это работает:**
1. `@do_twice` применяется к функции `greet`
2. На месте `greet` теперь находится функция `wrapper`
3. При вызове `greet('Кемаль')` вызывается `wrapper('Кемаль')`
4. `wrapper` два раза вызывает оригинальную функцию `greet`

## 🔧 Декоратор с аргументами: do_n_times

Декоратор, который принимает параметр и вызывает функцию N раз:

```python
import functools

def do_n_times(n):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                value = func(*args, **kwargs)
            return value
        return wrapper
    return decorator

@do_n_times(4)
def greet(name):
    print(f'Привет, {name}!')

greet('Кемаль')
# Вывод:
# Привет, Кемаль!
# Привет, Кемаль!
# Привет, Кемаль!
# Привет, Кемаль!
```

**Три уровня вложенности:**

```
do_n_times(4)              ← принимает ПАРАМЕТР
    ↓
decorator                  ← получает ФУНКЦИЮ
    ↓
wrapper                    ← оборачивает вызовы
```

**Эквивалент без @ синтаксиса:**

```python
decorator = do_n_times(4)
greet = decorator(greet)
```

**💡 Откровение:** Декоратор с параметром — это просто функция, которая возвращает другую функцию (декоратор), которая уже работает с исходной функцией!

## 🏛️ Декорирование методов класса

Свои декораторы можно применять к методам класса:

```python
import functools

def do_n_times(n):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                value = func(*args, **kwargs)
            return value
        return wrapper
    return decorator

class Cat:
    @do_n_times(4)
    def say(self):
        print('Мяу')

cat = Cat()
cat.say()
# Вывод:
# Мяу
# Мяу
# Мяу
# Мяу
```

**Важно:** Первый аргумент `wrapper` это `self` (неявно передаётся).

## 🏭 Классы-декораторы: класс как декоратор

Вместо функции можно использовать **класс** как декоратор. Для этого класс должен иметь:
- `__init__(self, func)` — принимает декорируемую функцию
- `__call__(self, *args, **kwargs)` — делает экземпляр вызываемым

### Простой класс-декоратор: do_twice

```python
import functools

class do_twice:
    def __init__(self, func):
        functools.update_wrapper(self, func)  # Сохраняем метаданные функции
        self.func = func
    
    def __call__(self, *args, **kwargs):
        for _ in range(2):
            value = self.func(*args, **kwargs)
        return value

@do_twice
def greet(name):
    """Функция приветствия"""
    print(f'Привет, {name}!')

greet('Кемаль')
# Вывод:
# Привет, Кемаль!
# Привет, Кемаль!

print(greet.__name__)  # → 'greet' ✅
print(greet.__doc__)   # → 'Функция приветствия' ✅
```

**Как это работает:**
1. `@do_twice` вызывает `do_twice(greet)` → создаёт экземпляр класса `do_twice`
2. Теперь `greet` — это объект класса `do_twice`
3. При вызове `greet(...)` Python вызывает метод `__call__` этого объекта

**⚠️ Важно:** Для классов-декораторов используй `functools.update_wrapper(self, func)`, а не `@functools.wraps(func)`!

---

### Класс-декоратор с параметром: do_n_times

```python
import functools

class do_n_times:
    def __init__(self, n):
        self.n = n  # Сохраняем параметр
    
    def __call__(self, func):  # Получаем функцию во втором вызове
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(self.n):
                value = func(*args, **kwargs)
            return value
        return wrapper

@do_n_times(4)
def greet(name):
    print(f'Привет, {name}!')

greet('Кемаль')
# Вывод:
# Привет, Кемаль!
# Привет, Кемаль!
# Привет, Кемаль!
# Привет, Кемаль!
```

**Два вызова:**
```
@do_n_times(4)          ← первый вызов: __init__
def greet(name):
    ...

# Эквивалент:
decorator = do_n_times(4)    # __init__(n=4)
greet = decorator(greet)     # __call__(func)
```

## 🔄 mappingproxy и изменение атрибутов класса

### 📖 Что такое mappingproxy?

Атрибут `__dict__` класса — это **не обычный словарь**, а объект типа `mappingproxy`:

```python
class MyClass:
    pass

print(type(MyClass.__dict__))  # → <class 'mappingproxy'>
print(type({}))                # → <class 'dict'>
```

**mappingproxy** — это "защищённый словарь" без метода `__setitem__`:

```python
class MyClass:
    pass

# ❌ НЕЛЬЗЯ — TypeError
MyClass.__dict__['new_attr'] = 'value'

# ✅ МОЖНО — используем setattr
setattr(MyClass, 'new_attr', 'value')
```

### Работа с классом-декоратором через setattr/delattr/getattr

Когда декоратор **модифицирует класс**, используй:

- `setattr(cls, name, value)` — добавить/изменить атрибут
- `delattr(cls, name)` — удалить атрибут
- `getattr(cls, name)` — получить атрибут

**Пример: переименование методов в snake_case**

```python
import re
import functools

def snake_case(attrs=False):
    def wrapper(cls):
        # Берём КОПИЮ ключей, чтобы изменение словаря не помешало итерации
        for el in list(cls.__dict__):
            # Пропускаем магические методы (__init__, __str__, и т.д.)
            if el.startswith('__'):
                continue
            
            # Преобразуем имя: camelCase → snake_case
            snake_name = re.sub(r'(?<!_)(?<!^)([A-Z])', r'_\1', el).lower()
            
            # Если attrs=True: переименовываем ВСЕ атрибуты
            if attrs:
                original = getattr(cls, el)
                setattr(cls, snake_name, original)
                delattr(cls, el)
            # Иначе: переименовываем только методы (callable)
            elif callable(getattr(cls, el)):
                original = getattr(cls, el)
                setattr(cls, snake_name, original)
                delattr(cls, el)
        
        return cls
    return wrapper

@snake_case(attrs=False)
class MyClass:
    def myMethod(self):
        return "hello"
    
    myAttribute = 42

obj = MyClass()
obj.my_method()        # ✅ Работает (переименован)
print(obj.myAttribute) # ✅ Работает (не переименован, attrs=False)
```

**Что происходит в цикле:**

```python
for el in list(cls.__dict__):
    # list(cls.__dict__) создаёт КОПИЮ ключей
    # Важно, потому что мы удаляем элементы через delattr
```

**Почему `list(cls.__dict__)`?**

Если просто писать `for el in cls.__dict__:`, то при удалении элемента через `delattr` может возникнуть ошибка "dictionary changed during iteration". Поэтому берём копию ключей через `list()`.

## 📊 Таблица: функция-декоратор vs класс-декоратор

| Свойство | Функция-декоратор | Класс-декоратор |
|---|---|---|
| **Синтаксис** | `def decorator(func):` | `class Decorator:` |
| **Сохранение метаданных** | `@functools.wraps(func)` | `functools.update_wrapper(self, func)` |
| **Вложенность** | 2 уровня (без параметров) | 1 уровень `__init__` + `__call__` |
| **Читаемость** | Иногда сложно следить | Яснее благодаря `__init__` и `__call__` |
| **Состояние** | В замыкании | В атрибутах класса |

## ⚠️ Частые ошибки

### 1. Забыть @functools.wraps или update_wrapper

```python
# ❌ НЕПРАВИЛЬНО
def decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@decorator
def my_func():
    """docstring"""
    pass

print(my_func.__name__)  # → 'wrapper' ❌
print(my_func.__doc__)   # → None ❌
```

### 2. Забыть вернуть wrapper или __call__

```python
# ❌ НЕПРАВИЛЬНО
def decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    # Забыли return wrapper!

# ❌ Класс-декоратор без __call__
class BadDecorator:
    def __init__(self, func):
        self.func = func
    # Забыли __call__!
```

### 3. Не использовать *args, **kwargs

```python
# ❌ НЕПРАВИЛЬНО
def decorator(func):
    def wrapper():  # Не принимает аргументы!
        return func()
    return wrapper

@decorator
def add(a, b):
    return a + b

add(5, 3)  # ❌ TypeError: wrapper() takes 0 positional arguments but 2 were given
```

### 4. Не использовать list() при итерации по __dict__

```python
# ❌ НЕПРАВИЛЬНО (может привести к ошибке)
for el in cls.__dict__:
    if condition:
        delattr(cls, el)  # Изменяем словарь во время итерации!

# ✅ ПРАВИЛЬНО
for el in list(cls.__dict__):  # Копия ключей
    if condition:
        delattr(cls, el)  # Безопасно
```

## 🎯 Когда использовать?

### ✅ Функция-декоратор
- Логика простая и понятная
- Нужна минимальная вложенность
- Стандартный выбор для большинства случаев

### ✅ Класс-декоратор
- Нужно сохранять состояние между вызовами
- Логика сложная (много операций)
- Нужны дополнительные методы и атрибуты

## 💭 Итоговая шпаргалка

### Функция-декоратор (без параметров)

```python
import functools

def decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        # код до
        result = func(*args, **kwargs)
        # код после
        return result
    return wrapper

@decorator
def my_func():
    pass
```

### Функция-декоратор (с параметром)

```python
import functools

def decorator(param):
    def actual_decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            # используй param
            return func(*args, **kwargs)
        return wrapper
    return actual_decorator

@decorator("value")
def my_func():
    pass
```

### Класс-декоратор (без параметров)

```python
import functools

class Decorator:
    def __init__(self, func):
        functools.update_wrapper(self, func)
        self.func = func
    
    def __call__(self, *args, **kwargs):
        return self.func(*args, **kwargs)

@Decorator
def my_func():
    pass
```

### Класс-декоратор (с параметром)

```python
import functools

class Decorator:
    def __init__(self, param):
        self.param = param
    
    def __call__(self, func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            # используй self.param
            return func(*args, **kwargs)
        return wrapper

@Decorator("value")
def my_func():
    pass
```

## ✅ Ключевые правила

✅ Всегда используй `@functools.wraps(func)` для функций  
✅ Всегда используй `functools.update_wrapper(self, func)` для классов  
✅ Всегда используй `*args, **kwargs` в wrapper  
✅ Всегда возвращай результат из wrapper/\_\_call\_\_  
✅ При изменении `cls.__dict__` используй `setattr/delattr/getattr`  
✅ При итерации по `cls.__dict__` с изменениями используй `list(cls.__dict__)`
