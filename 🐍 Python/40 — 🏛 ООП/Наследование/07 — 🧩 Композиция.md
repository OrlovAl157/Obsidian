# 🧩 Композиция

## 📌 Коротко

> Композиция — экземпляр одного класса содержит экземпляр другого класса. Описывается словом **имеет**. Более гибкий и слабосвязанный подход чем наследование.

---

## 📖 Является vs Имеет

| Отношение | Подход | Пример |
|---|---|---|
| **является** | Наследование | Кошка является животным |
| **имеет** | Композиция | Автомобиль имеет двигатель |

```python
# Наследование — IS-A
class Cat(Animal): pass        # Cat ЯВЛЯЕТСЯ Animal

# Композиция — HAS-A
class Car:
    def __init__(self):
        self.engine = Engine() # Car ИМЕЕТ Engine
```

## 📖 Пример: Square и Rectangle

```python
class Rectangle:
    def __init__(self, length, width):
        self.length = length
        self.width = width

    def perimeter(self):
        return (self.length + self.width) * 2

    def area(self):
        return self.length * self.width

class Square:
    def __init__(self, side):
        self.value = Rectangle(side, side)  # HAS-A Rectangle

    def perimeter(self):
        return self.value.perimeter()  # делегируем

    def area(self):
        return self.value.area()       # делегируем

square = Square(2)
print(square.area())       # 4
print(square.perimeter())  # 8
```

> [!tip] Почему не наследование?
> Квадрат **не является** прямоугольником в полном смысле — у прямоугольника стороны независимы, у квадрата нет. Наследование привело бы к проблемам при изменении сторон.

## 📖 Пример: Employee и Salary

```python
class Salary:
    def __init__(self, pay, bonus):
        self.pay = pay
        self.bonus = bonus

    def annual(self):
        return 12 * self.pay + self.bonus

    def semi_annual(self):
        return 6 * self.pay  # без премии

class Employee:
    def __init__(self, name, pay, bonus):
        self.name = name
        self.salary = Salary(pay, bonus)  # HAS-A Salary

    def total_salary(self):
        return self.salary.annual()  # делегируем

employee = Employee('Гвидо', 100000, 10000)
print(employee.total_salary())  # 1210000
```

Вынесение логики в `Salary` — разделение ответственности. `Employee` хранит данные, `Salary` считает. Нужно добавить полугодовую зарплату? Просто добавляем метод в `Salary`.

## 📖 Композиция vs Агрегация

Оба варианта — "has-a", но разница в жизненном цикле компонента:

```python
# Композиция — компонент создаётся ВНУТРИ составного
class Employee:
    def __init__(self, name, pay, bonus):
        self.salary = Salary(pay, bonus)  # создаём сами, живёт вместе с Employee

# Агрегация — компонент создаётся СНАРУЖИ и передаётся
class Employee:
    def __init__(self, name, salary):
        self.salary = salary  # получаем извне, живёт независимо

salary = Salary(100000, 10000)
employee = Employee('Гвидо', salary)
```

| | Композиция | Агрегация |
|---|---|---|
| Компонент создаётся | Внутри составного | Снаружи |
| Зависимость | Жёсткая | Слабая |
| Пример | Сердце в теле | Водитель в машине |

## 📖 Когда что выбирать

> [!tip] Правило
> Если можешь сказать **"X является Y"** → наследование.
> Если можешь сказать **"X имеет Y"** → композиция.

**Наследование** — когда нужно получить весь функционал родителя и расширить или изменить его.

**Композиция** — когда нужно использовать функционал другого класса без его изменения.

---

## 📖 Принцип "предпочитай композицию наследованию"

Композиция создаёт **меньше зависимостей**:

```python
# Наследование — хрупко:
class Animal:
    def breathe(self):
        return 'дышу через лёгкие'  # изменили метод

class Cat(Animal):
    pass  # Cat автоматически получает изменение — хорошо или плохо?

# Композиция — устойчиво:
class Engine:
    def start(self):
        return 'запускаю по-новому'  # изменили

class Car:
    def __init__(self):
        self.engine = Engine()

    def start(self):
        return self.engine.start()  # Car сам решает использовать ли изменение
```

> [!warning] Важно
> Изменение компонентного класса **редко** влияет на составной.
> Изменение родительского класса **может сломать** всех наследников.

## 📖 UserDict как пример композиции

`UserDict` из модуля `collections` — классический пример:

```python
from collections import UserDict

# UserDict внутренне содержит обычный dict:
class UserDict:
    def __init__(self):
        self.data = {}  # HAS-A dict — композиция!

    def __setitem__(self, key, value):
        self.data[key] = value  # делегирует в self.data
```

Не наследуется от `dict` — содержит его. Это позволяет надёжно перехватывать все операции.
