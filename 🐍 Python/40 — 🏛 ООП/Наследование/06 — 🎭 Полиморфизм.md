# 🎭 Полиморфизм

## 📌 Коротко

> Полиморфизм — способность объектов разных типов реагировать на одни и те же операции по-разному. В Python реализуется через утиную типизацию и магические методы.

---

## 📖 Что такое полиморфизм

Буквально: **поли** — много, **морфос** — форма. Один интерфейс — разное поведение.

```python
# Один оператор + — разное поведение:
print(1 + 2)          # 3       — сложение чисел
print('a' + 'b')      # 'ab'    — конкатенация строк
print([1] + [2])      # [1, 2]  — объединение списков
```

---

## 📖 Утиная типизация

> "Если это выглядит как утка и крякает как утка — это утка."

Python не проверяет **тип** объекта — он проверяет наличие **нужного метода**:

```python
class Duck:
    def sound(self): return 'кря'

class Dog:
    def sound(self): return 'гав'

class Robot:
    def sound(self): return 'бип'

# Функция не знает тип — просто вызывает метод
def make_sound(obj):
    print(obj.sound())

make_sound(Duck())   # кря
make_sound(Dog())    # гав
make_sound(Robot())  # бип
```

## 📖 Полиморфизм через магические методы

`len()` не знает что за объект — просто вызывает `__len__()`:

```python
class MyCollection:
    def __init__(self, items):
        self.items = items

    def __len__(self):
        return len(self.items)

obj = MyCollection([1, 2, 3])
print(len(obj))  # 3 — полиморфизм!
```

Встроенные функции которые используют полиморфизм:

| Функция | Метод |
|---|---|
| `len(obj)` | `__len__()` |
| `str(obj)` | `__str__()` |
| `repr(obj)` | `__repr__()` |
| `bool(obj)` | `__bool__()` |
| `iter(obj)` | `__iter__()` |
| `abs(obj)` | `__abs__()` |

## 📖 Полиморфизм через наследование

```python
class Animal:
    def sound(self):
        raise NotImplementedError

class Cat(Animal):
    def sound(self):
        return 'мяу'

class Dog(Animal):
    def sound(self):
        return 'гав'

animals = [Cat(), Dog(), Cat()]
for animal in animals:
    print(animal.sound())  # мяу, гав, мяу
```

## 📖 Утиная типизация vs isinstance

```python
# ❌ Не полиморфизм — явная проверка типа (антипаттерн)
def process(obj):
    if isinstance(obj, int):
        return obj * 2
    elif isinstance(obj, str):
        return obj.upper()
    # хочешь добавить новый тип? — лезь внутрь функции

# ✅ Полиморфизм — объект сам знает что делать
def process(obj):
    return obj.process()
    # хочешь добавить новый тип? — просто создай класс с методом process()
```

> [!warning] Важно
> Проверка через `isinstance` внутри функции — это антипаттерн. Полиморфизм означает что функция **не знает** с каким типом работает.

---

## 📖 Перегрузка методов vs Полиморфизм

В Python нет классической перегрузки как в Java/C++ — нельзя определить два метода с одним именем. Второй перезапишет первый.

`@singledispatchmethod` — обходной путь, но не встроенный полиморфизм.

---

## 📖 Формы полиморфизма в Python

| Форма | Пример |
|---|---|
| Операторы | `+` для int, str, list |
| Встроенные функции | `len()`, `str()`, `bool()` |
| Утиная типизация | любой объект с нужным методом |
| Наследование | переопределение методов |
