## 🎯 Блок 1: Введение в наследование

### Основные понятия

**Наследование** — механизм получения новых классов из уже существующих. Класс-наследник приобретает все атрибуты и методы класса-родителя.

- **Родительский класс (базовый, суперкласс)** — класс, от которого наследуются
- **Дочерний класс (наследник, подкласс)** — класс, который наследуется

### Синтаксис наследования

```python
class ParentClass:
    pass

class ChildClass(ParentClass):
    pass
```

### Наследование методов и атрибутов

Дочерний класс автоматически наследует все методы и атрибуты родителя:

```python
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def sleep(self):
        return f'{self.name} спит zZ'

class Cat(Animal):
    pass

cat = Cat('Кемаль', 1)
print(cat.name)      # Кемаль
print(cat.sleep())   # Кемаль спит zZ
```

> **✅ Вывод:** дочерний класс **автоматически** получает конструктор и методы родителя

### Переопределение методов

Дочерний класс может **полностью заменить** метод родителя:

```python
class Cat(Animal):
    def sleep(self):
        return f'{self.name} очень крепко спит zZ'

cat = Cat('Кемаль', 1)
print(cat.sleep())  # Кемаль очень крепко спит zZ
```

**⚠️ Важно:** При переопределении метод родителя становится **недоступен** для экземпляров дочернего класса.

### Переопределение конструктора без дублирования

Когда дочерний класс переопределяет `__init__()`, часто дублируется код инициализации родителя. **Лучше вызвать родительский конструктор:**

```python
class Cat(Animal):
    def __init__(self, name, age, eyecolor):
        Animal.__init__(self, name, age)  # Явный вызов родительского __init__
        self.eyecolor = eyecolor

cat = Cat('Кемаль', 1, 'синий')
print(cat.name, cat.age, cat.eyecolor)  # Кемаль 1 синий
```

### Наследование атрибутов класса

Дочерний класс наследует **атрибуты класса** (не экземпляра):

```python
class Animal:
    en_name = 'animal'
    ru_name = 'животное'

class Cat(Animal):
    pass

class Dog(Animal):
    en_name = 'dog'
    ru_name = 'собака'

print(Cat.en_name)   # animal — унаследовано
print(Dog.en_name)   # dog — переопределено
```

> **Важно:** Cat использует ссылку на атрибут Animal, не копию. Если Animal.en_name изменится, Cat увидит новое значение (если не переопределён).

### Неявное наследование от object

**Все классы в Python наследуются от класса `object`** (автоматически):

```python
class MyClass:
    pass

# Равнозначно:
class MyClass(object):
    pass
```

Поэтому любой класс имеет методы, унаследованные от `object`: `__init__()`, `__str__()`, `__repr__()`, `__eq__()` и т.д.

### Многоуровневое наследование

Классы образуют **иерархию** — цепь наследования:

```python
class A:
    def method_A(self):
        return 'A'

class B(A):
    def method_B(self):
        return 'B'

class C(B):
    pass

c = C()
print(c.method_A())  # A — унаследовано от A через B
print(c.method_B())  # B — унаследовано от B
```

**Цепочка наследования:** C → B → A → object

### Порядок поиска методов (MRO — Method Resolution Order)

При обращении к методу Python ищет его в этом порядке:

```
C → B → A → object
```

Сперва в текущем классе, потом в родителях **снизу вверх**.

```python
# При обращении к методу класса C:
# 1. Ищет в C
# 2. Ищет в B (родитель C)
# 3. Ищет в A (родитель B)
# 4. Ищет в object (родитель A)
```

> **Совет:** Используй `ClassName.__mro__` или `ClassName.mro()` чтобы увидеть порядок поиска.

### Проверка наследования

```python
print(issubclass(Cat, Animal))   # True — Cat наследник Animal
print(issubclass(Cat, Cat))      # True — класс наследник самого себя!

cat = Cat('Кемаль', 1)
print(isinstance(cat, Cat))      # True
print(isinstance(cat, Animal))   # True — через наследование
print(isinstance(cat, object))   # True — все классы наследники object
```

## 🎯 Блок 2: super() и расширение методов

### Проблема дублирования кода

При переопределении `__init__()` в дочернем классе часто повторяешь код родителя:

```python
class Animal:
    def __init__(self, name, age, eyecolor):
        self.name = name
        self.age = age
        self.eyecolor = eyecolor

class Cat(Animal):
    def __init__(self, name, age, eyecolor, breed):
        self.name = name           # ❌ Дублирование
        self.age = age             # ❌ Дублирование
        self.eyecolor = eyecolor   # ❌ Дублирование
        self.breed = breed
```

### Решение: вызов родительского метода напрямую

Можно явно обратиться к методу родителя по его имени:

```python
class Cat(Animal):
    def __init__(self, name, age, eyecolor, breed):
        Animal.__init__(self, name, age, eyecolor)  # Вызов родительского __init__
        self.breed = breed
```

**Но это хрупко:** если название класса изменится, нужно обновить везде.

### Функция super()

**`super()`** — встроенная функция, которая **неявно** обращается к родительскому классу:

```python
class Cat(Animal):
    def __init__(self, name, age, eyecolor, breed):
        super().__init__(name, age, eyecolor)  # Вызов родительского __init__
        self.breed = breed

cat = Cat('Кемаль', 1, 'синий', 'манчкин')
print(cat.name, cat.breed)  # Кемаль манчкин
```

> **✅ Преимущество:** если родительский класс изменится, код продолжит работать.

### Как работает super()

`super().__init__(name, age, eyecolor)` равнозначно `Animal.__init__(self, name, age, eyecolor)`, но:

- **Автоматически определяет** текущий класс и родителя
- **Не требует** передавать `self` явно (super() это делает за тебя)
- **Безопаснее** при многоуровневом наследовании

### Явная форма super()

Можно указать класс и объект явно (редко используется):

```python
class Cat(Animal):
    def __init__(self, name, age, eyecolor, breed):
        super(Cat, self).__init__(name, age, eyecolor)  # Явно указываем Cat и self
        self.breed = breed
```

Результат одинаков, но первая форма `super()` — предпочтительная и чище.

### super() и многоуровневое наследование

При наследовании через несколько уровней `super()` **автоматически** ищет метод во всей иерархии:

```python
class Animal:
    def __init__(self, name):
        print('Инициализация Animal')
        self.name = name

class Cat(Animal):
    pass

class Kitten(Cat):
    def __init__(self, name, breed):
        print('Инициализация Kitten')
        super().__init__(name)  # super() найдёт метод в Cat, потом в Animal
        self.breed = breed

kitten = Kitten('Кемаль', 'манчкин')
# Вывод:
# Инициализация Kitten
# Инициализация Animal
```

**Порядок поиска:** Kitten → Cat (нет `__init__`) → Animal (нашёл!)

### Расширение методов vs полная замена

**Расширение** — добавить логику к родительскому методу:

```python
class Cat(Animal):
    def sleep(self):
        return super().sleep() + ' очень крепко'  # Используем родительский метод

cat = Cat('Кемаль', 1)
print(cat.sleep())  # Кемаль спит zZ очень крепко
```

**Полная замена** — игнорировать родительский метод:

```python
class Cat(Animal):
    def sleep(self):
        return f'{self.name} мяукает и спит!'  # Не используем super()

cat = Cat('Кемаль', 1)
print(cat.sleep())  # Кемаль мяукает и спит!
```

> **Выбор:** если новый метод использует логику родителя — `super()`, если полностью другой — не используй.

### Использование методов наследников в базовом классе

Родительский класс может вызвать метод, который переопределён в дочернем:

```python
class Animal:
    def __init__(self, name):
        self.name = name
    
    def info(self):
        return f'Имя: {self.name}, звук: {self.sound()}'

class Cat(Animal):
    def sound(self):
        return 'мяу'

class Dog(Animal):
    def sound(self):
        return 'гав'

cat = Cat('Кемаль')
dog = Dog('Роджер')
print(cat.info())  # Имя: Кемаль, звук: мяу
print(dog.info())  # Имя: Роджер, звук: гав
```

**Механизм:** `info()` вызывает `self.sound()`. Python ищет `sound()` в классе текущего объекта (Cat или Dog), а не в Animal.

## 🎯 Блок 3: Наследование от неизменяемых типов

### Процесс создания объекта: **new**() и **init**()

Когда ты создаёшь объект `obj = MyClass(args)`, Python выполняет **два шага**:

1. **`__new__(cls, ...)`** — создаёт **пустой** объект (выделяет память)
2. **`__init__(self, ...)`** — инициализирует объект (добавляет атрибуты)

```python
class Cat:
    def __new__(cls, *args, **kwargs):
        print('1. Создание объекта')
        return super().__new__(cls)
    
    def __init__(self, name):
        print('2. Инициализация объекта')
        self.name = name

cat = Cat('Кемаль')
# Вывод:
# 1. Создание объекта
# 2. Инициализация объекта
```

**Порядок:** сперва `__new__()`, потом `__init__()`.

### Почему **new**() важен для неизменяемых типов

Неизменяемые типы (int, float, str, tuple) **заморожены после создания** — значение устанавливается в `__new__()`, не в `__init__()`.

```python
class Distance(float):
    def __init__(self, value, unit):
        super().__init__()  # ❌ Поздно! Значение уже установлено
        self.unit = unit

distance = Distance(1, 'Meters')  # TypeError!
```

**Ошибка:** `float.__new__()` ожидает **максимум 1 аргумент**, но мы передали 2 (value, unit).

### Решение: переопределить **new**()

```python
class Distance(float):
    def __new__(cls, value, unit):
        instance = super().__new__(cls, value)  # Создаём float с значением value
        instance.unit = unit                    # Добавляем атрибут ДО возврата
        return instance

distance = Distance(1, 'Meters')
print(distance)          # 1.0
print(distance.unit)     # Meters
```

**Логика:**

1. `super().__new__(cls, value)` вызывает `float.__new__()` с одним значением
2. Получаем готовый объект типа Distance со значением 1.0
3. Добавляем к нему атрибут `unit` пока он ещё в `__new__()`
4. Возвращаем готовый объект

> **Важно:** неизменяемой является только **числовая часть** (1.0). Атрибут `unit` можно менять.

```python
distance = Distance(1, 'Meters')
distance.unit = 'Kilometers'  # ✅ Атрибут изменяемый
print(distance.unit)  # Kilometers
```

### Пример: наследование от str

```python
class UpperString(str):
    def __new__(cls, value):
        instance = super().__new__(cls, value.upper())
        return instance

us = UpperString('hello')
print(us)  # HELLO
```

### Когда **new**() может вернуть другой тип

`__new__()` может вернуть экземпляр **другого класса** — тогда `__init__()` не вызывается:

```python
class Cat:
    def __new__(cls):
        return 42  # Возвращаем число, а не Cat

cat = Cat()
print(cat, type(cat))  # 42 <class 'int'>
```

## 🎯 Блок 4: Наследование от изменяемых типов

### Проблема: методы не вызывают **setitem**()

При наследовании от `dict` переопределяешь `__setitem__()`, чтобы контролировать добавление элементов:

```python
class UpperDict(dict):
    def __setitem__(self, key, value):
        key = str(key).upper()
        super().__setitem__(key, value)

ud = UpperDict()
ud['name'] = 'Tim'  # ✅ Работает — вызывает __setitem__
print(ud)  # {'NAME': 'Tim'}
```

**Но вот проблема:**

```python
ud.update({'city': 'NY'})  # ❌ Не вызывает __setitem__!
print(ud)  # {'NAME': 'Tim', 'city': 'NY'} — 'city' в нижнем регистре
```

**Почему?** Метод `update()` встроенного `dict` работает **напрямую с внутренним хранилищем**, минуя твой `__setitem__()`.

### Аналогичные проблемы с list

```python
class StringList(list):
    def __setitem__(self, index, item):
        super().__setitem__(index, str(item))

sl = StringList([1, 2, 3])
sl[0] = 4  # ✅ Работает — вызывает __setitem__
print(sl)  # ['4', '2', '3']

sl.append(5)  # ❌ Не вызывает __setitem__!
print(sl)  # ['4', '2', '3', 5] — число, а не строка
```

### Решение 1: переопределить все методы

```python
class UpperDict(dict):
    def __init__(self, items=(), **kwargs):
        self.update(items)
        self.update(kwargs)
    
    def update(self, items):
        if isinstance(items, dict):
            items = items.items()
        for key, value in items:
            self[key] = value  # Теперь вызывает наш __setitem__!
    
    def setdefault(self, key, value):
        if str(key).upper() not in self:
            self[key] = value
    
    def __setitem__(self, key, value):
        key = str(key).upper()
        super().__setitem__(key, value)
```

**Проблема:** много кода, нужно помнить все методы.

### Решение 2: использовать UserDict и UserList

Классы `UserDict` и `UserList` из модуля `collections` — это **обёртки**, которые гарантируют, что все методы вызывают твои переопределённые методы:

```python
from collections import UserDict

class UpperDict(UserDict):
    def __setitem__(self, key, value):
        key = str(key).upper()
        self.data[key] = value  # self.data — это обычный dict

ud = UpperDict({'name': 'Tim'})
ud['city'] = 'NY'
ud.update({'age': 30})
ud.setdefault('country', 'USA')

print(ud)  # {'NAME': 'TIM', 'CITY': 'NY', 'AGE': 30, 'COUNTRY': 'USA'}
```

**✅ Всё работает!** `update()`, `setdefault()` и др. теперь вызывают твой `__setitem__()`.

### Как работает UserDict

`UserDict` содержит обычный `dict` в атрибуте `self.data`. Все методы работают через этот атрибут и вызывают переопределённые методы:

```python
class UserDict:
    def __init__(self, *args, **kwargs):
        self.data = {}
        self.update(*args, **kwargs)
    
    def __setitem__(self, key, value):
        self.data[key] = value
    
    def update(self, other=None, **kwargs):
        if isinstance(other, dict):
            for key in other:
                self[key] = other[key]  # Вызывает self[key], что вызывает наш __setitem__!
```

### UserList для списков

```python
from collections import UserList

class StringList(UserList):
    def __init__(self, iterable):
        super().__init__(str(item) for item in iterable)
    
    def __setitem__(self, index, item):
        self.data[index] = str(item)
    
    def append(self, item):
        self.data.append(str(item))
    
    def extend(self, other):
        self.data.extend(str(item) for item in other)

sl = StringList([1, 2, 3])
sl.append(4)
sl.extend([5, 6])
print(sl)  # ['1', '2', '3', '4', '5', '6']
```

### UserString для строк

```python
from collections import UserString

class MutableString(UserString):
    def lower(self):
        self.data = self.data.lower()
    
    def upper(self):
        self.data = self.data.upper()
    
    def __setitem__(self, index, value):
        self.data = self.data[:index] + value + self.data[index+1:]

ms = MutableString('hello')
ms.upper()
print(ms)  # HELLO
```

> **Ключ:** `self.__class__` в методах типа `__add__()` гарантирует, что результат будет типа текущего класса, не просто UserString.

### Когда наследоваться от dict/list нормально

Наследование от встроенных типов OK, если **только один метод** переопределяется:

```python
class DefaultDict(dict):
    def __init__(self, *args, default=None, **kwargs):
        super().__init__(*args, **kwargs)
        self.default = default
    
    def __missing__(self, key):
        return self.default

dd = DefaultDict({'a': 1}, default='not found')
print(dd['a'])  # 1
print(dd['b'])  # not found
```

**Почему OK:** `__missing__()` — это специальный метод, который вызывается только `__getitem__()`. Других проблем нет.

## 📊 Quick Reference: когда что использовать

|Ситуация|Решение|Пример|
|---|---|---|
|Простое наследование, добавить методы|Наследуй от класса|`class Dog(Animal)`|
|Переопределить несколько методов, расширить|Используй `super()`|`super().__init__()`|
|Изменить значение неизменяемого типа|Переопредели `__new__()`|`Distance(float)`|
|Контролировать все добавления в dict|Используй `UserDict`|`class UpperDict(UserDict)`|
|Контролировать все добавления в list|Используй `UserList`|`class StringList(UserList)`|
|Создать изменяемую строку|Используй `UserString`|`class MyString(UserString)`|
|Только один метод переопределить в dict|Наследь от dict|`class DefaultDict(dict)`|

## ❓ Три важных вопроса и ответы

### Q1: Почему в super().**new**(cls) не передаём *args, **kwargs?

**A:** Потому что `object.__new__(cls)` **не принимает** дополнительные аргументы. Он создаёт пустой объект — всё.

`*args, **kwargs` в сигнатуре `__new__()` нужны для **гибкости** — чтобы принять то, что передали при вызове класса. Но когда вызываем `object.__new__()`, мы знаем, что ему нужен только `cls`.

```python
def __new__(cls, *args, **kwargs):  # Принимаем любое количество аргументов
    return super().__new__(cls)      # Передаём только cls
```

### Q2: Классы — это экземпляры type? А type наследник object?

**A:** Да. Иерархия:

```
MyClass — экземпляр type (мета-класса)
type — экземпляр type (сам себя)
object — экземпляр type (и родитель всех классов)

issubclass(type, object)  # True
```

Когда создаёшь класс `class MyClass`, Python вызывает `type.__new__()`, который создаёт класс. `type` наследует от `object`.

### Q3: Что значит "значение определяется при создании в **new**()"?

**A:** Для неизменяемых типов **значение задаётся один раз** в `__new__()` и **не может быть изменено** позже.

```python
class Distance(float):
    def __init__(self, value, unit):
        # ❌ Поздно! float уже создан с некоторым значением
        self.unit = unit

distance = Distance(1, 'Meters')
# При вызове float.__new__() ожидает максимум 1 аргумент
# Но мы передали 2 → TypeError
```

**Решение:** передай значение в `__new__()`:

```python
class Distance(float):
    def __new__(cls, value, unit):
        instance = super().__new__(cls, value)  # Значение устанавливается ТУТ
        instance.unit = unit
        return instance
```

Теперь `instance` — это готовый float с значением, и атрибут `unit` добавлен.

## 📚 Практический пример: класс MutableString

```python
from collections import UserString

class MutableString(UserString):
    """Изменяемая строка."""
    
    def lower(self):
        """Преобразовать в нижний регистр."""
        self.data = self.data.lower()
    
    def upper(self):
        """Преобразовать в верхний регистр."""
        self.data = self.data.upper()
    
    def sort(self, key=None, reverse=False):
        """Отсортировать символы."""
        self.data = ''.join(sorted(self.data, key=key, reverse=reverse))
    
    def __setitem__(self, index, value):
        """Изменить символ по индексу."""
        data_list = list(self.data)
        data_list[index] = value
        self.data = ''.join(data_list)
    
    def __delitem__(self, index):
        """Удалить символ по индексу."""
        self.data = self.data[:index] + self.data[index+1:]

# Использование
ms = MutableString('hello')
ms.upper()
print(ms)  # HELLO

ms[0] = 'J'
print(ms)  # JELLO

ms.sort()
print(ms)  # ELLJO

del ms[0]
print(ms)  # LLJO
```
