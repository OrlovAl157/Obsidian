# 🏗 Наследование от неизменяемых типов

## 📌 Коротко

> При наследовании от неизменяемых типов (`int`, `float`, `str`, `tuple`) значение объекта устанавливается в `__new__()`, а не в `__init__()`. Поэтому переопределять нужно `__new__()`, а не только `__init__()`.

---

## 📖 Протокол создания объекта

Когда пишешь `obj = MyClass(args)`, Python выполняет **два шага**:

1. **`__new__(cls, ...)`** — создаёт новый пустой объект (выделяет память)
2. **`__init__(self, ...)`** — инициализирует объект (добавляет атрибуты)

```python
class Cat:
    def __new__(cls, *args, **kwargs):
        print('1. Создание объекта')
        return super().__new__(cls)  # object.__new__(Cat) — создаёт экземпляр

    def __init__(self, name):
        print('2. Инициализация объекта')
        self.name = name

cat = Cat('Кемаль')
# 1. Создание объекта
# 2. Инициализация объекта
```

> [!tip] Зачем super().__new__(cls)?
> `object.__new__()` — это C-реализация, которая физически создаёт объект. Это терминальная функция — никакой бесконечной рекурсии нет. Именно она отвечает за создание **всех** объектов в Python.

---

## 📖 Сигнатура __new__()

`__new__()` принимает `*args, **kwargs` — чтобы не ограничивать сигнатуру `__init__()`. Все аргументы, переданные при создании экземпляра, попадают **и в `__new__()`, и в `__init__()`**:

```python
class Cat:
    def __new__(cls, *args, **kwargs):
        print(args, kwargs)        # ('Кемаль',) {'age': 2}
        return super().__new__(cls)

    def __init__(self, name, age=1):
        self.name = name
        self.age = age

cat = Cat('Кемаль', age=2)
```

## 📖 Особые случаи поведения __new__()

### Если __new__() вернул экземпляр другого класса

`__init__()` вызван **не будет** — Python вызывает `__init__()` только если `__new__()` вернул экземпляр **того же класса**:

```python
class Animal:
    pass

class Cat:
    def __new__(cls, *args, **kwargs):
        print('__new__ Cat вызван')
        return Animal()  # Возвращаем другой тип!

    def __init__(self, name):
        print('__init__ Cat вызван')  # Не будет выведено
        self.name = name

cat = Cat('Кемаль')
print(type(cat))  # <class 'Animal'>
# __init__ Cat вызван — НЕ выводится
```

> [!warning] Важно
> Если `__new__()` вернул `None` — `__init__()` тоже не вызывается.

### __init__() должен возвращать None

```python
class Cat:
    def __init__(self, name):
        self.name = name
        return self  # ❌ TypeError: __init__() should return None, not 'Cat'
```

## 📖 Почему нужен __new__() для неизменяемых типов

Неизменяемые типы (`int`, `float`, `str`, `tuple`) **заморожены после создания** — значение устанавливается в `__new__()`. В `__init__()` менять его уже поздно.

**Проблема:**

```python
class Distance(float):
    def __init__(self, value, unit):
        super().__init__()  # ❌ float уже создан — значение не передать
        self.unit = unit

distance = Distance(1, 'Meters')
# TypeError: float expected at most 1 argument, got 2
```

Ошибка возникает потому что `float.__new__()` ожидает **максимум 1 аргумент** — числовое значение. Мы передаём 2.

**Решение — переопределить `__new__()`:**

```python
class Distance(float):
    def __new__(cls, value, unit):
        instance = super().__new__(cls, value)  # float.__new__(Distance, value)
        instance.unit = unit                    # Добавляем атрибут до возврата
        return instance

distance = Distance(1, 'Meters')
print(distance)        # 1.0  — числовое значение
print(distance.unit)   # Meters — наш атрибут
```

> [!tip] Неизменяемо только число
> Числовая часть `1.0` — неизменяема. Атрибут `unit` — обычный атрибут экземпляра, его можно менять:
> ```python
> distance.unit = 'Kilometers'  # ✅ Работает
> ```

## 📖 Пример: наследование от str

Когда нужно расширить строку **без переопределения** процесса создания — `__new__()` трогать не нужно, `str.__new__()` сам подхватит значение:

```python
class WordCountString(str):
    def __str__(self):
        return f'{super().__str__()}, слов: {self.words()}'

    def words(self):
        return len(self.split())

wcs = WordCountString('I Love Beegeek')
print(wcs.words())  # 3
print(wcs)          # I Love Beegeek, слов: 3
```

> [!warning] Почему super().__str__(), а не просто self?
> Если написать `f'{self}, слов: ...'` — Python вызовет `str(self)` → `__str__(self)` → снова `str(self)` → бесконечная рекурсия.
> `super().__str__()` обращается к методу `str` напрямую, минуя наш переопределённый `__str__()`.

> [!tip] Как строка попала в объект без __init__?
> `str.__new__()` сохраняет строковое значение при создании объекта. `__init__()` для этого не нужен — значение уже в объекте после `__new__()`.

---

## 📖 Когда нужно переопределять __new__() при наследовании от str

Когда нужно **изменить** само строковое значение при создании:

```python
class UpperString(str):
    def __new__(cls, value):
        instance = super().__new__(cls, value.upper())  # Меняем значение в __new__
        return instance

us = UpperString('hello')
print(us)  # HELLO
```

## 📖 Ограничения: от каких типов нельзя наследоваться

`bool` и `NoneType` — **нельзя** делать родительскими классами:

```python
class MyBool(bool):
    pass
# TypeError: type 'bool' is not an acceptable base type
```

> [!tip] Как запретить наследование от своего класса?
> Через `__init_subclass__()` или метакласс. Пример через `__init_subclass__`:
> ```python
> class Final:
>     def __init_subclass__(cls, **kwargs):
>         raise TypeError(f"Нельзя наследоваться от {Final.__name__}")
>
> class Child(Final):  # TypeError!
>     pass
> ```

---

## 📊 Quick Reference

| Тип | Нужен __new__? | Причина |
|---|---|---|
| Обычный класс | Нет | `__init__()` достаточно |
| `float`, `int` | Да | Числовое значение устанавливается в `__new__()` |
| `str`, `bytes` | Да (если менять значение) | Строковое значение устанавливается в `__new__()` |
| `tuple` | Да | Содержимое кортежа устанавливается в `__new__()` |
| `bool`, `NoneType` | — | Нельзя наследоваться |
