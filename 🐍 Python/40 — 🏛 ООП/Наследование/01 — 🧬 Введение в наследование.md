# 🧬 Введение в наследование

## 📌 Коротко

> Наследование — механизм получения новых классов из уже существующих. Дочерний класс автоматически получает все атрибуты и методы родительского.

---

## 📖 Основные понятия

**Родительский класс (базовый, суперкласс)** — класс, от которого наследуются.  
**Дочерний класс (наследник, подкласс)** — класс, который наследуется.

Аналогия из биологии: ребёнок наследует рост, цвет глаз и фамилию родителя — но может добавлять собственные черты.

---

## 📖 Синтаксис

```python
class ParentClass:
    pass

class ChildClass(ParentClass):  # В скобках — родительский класс
    pass
```

---

## 📖 Наследование методов и атрибутов экземпляра

Дочерний класс **автоматически** получает все методы родителя — включая `__init__()`:

```python
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def sleep(self):
        return f'{self.name} спит zZ'

class Cat(Animal):
    pass  # Пустой класс, но наследует всё от Animal

cat = Cat('Кемаль', 1)
print(cat.name)     # Кемаль
print(cat.sleep())  # Кемаль спит zZ
```

---

## 📖 Переопределение методов

Дочерний класс может **полностью заменить** метод родителя:

```python
class Cat(Animal):
    def sleep(self):
        return f'{self.name} очень крепко спит zZ'

cat = Cat('Кемаль', 1)
print(cat.sleep())  # Кемаль очень крепко спит zZ
```

> [!warning] Важно
> При переопределении метод родителя становится **недоступен** для экземпляров дочернего класса. Это полная замена, а не дополнение.

Переопределение `__init__()` — вызывай родительский конструктор, чтобы не дублировать код:

```python
class Cat(Animal):
    def __init__(self, name, age, eyecolor):
        Animal.__init__(self, name, age)  # Вызов родительского __init__
        self.eyecolor = eyecolor

cat = Cat('Кемаль', 1, 'синий')
print(cat.name, cat.age, cat.eyecolor)  # Кемаль 1 синий
```

> [!tip] Почему работает передача self
> `Animal.__init__(self, ...)` — `self` здесь это экземпляр `Cat`. Python запишет атрибуты `name` и `age` **на объект кота**, несмотря на то что метод взят из `Animal`.

## 📖 Наследование атрибутов класса

```python
class Animal:
    en_name = 'animal'
    ru_name = 'животное'

class Cat(Animal):      # Наследует — использует ссылку на атрибут Animal
    pass

class Dog(Animal):      # Переопределяет — создаёт собственный атрибут
    en_name = 'dog'
    ru_name = 'собака'

Animal.en_name = 'Animal (updated)'
print(Cat.en_name)  # Animal (updated) — изменилось вместе с родителем
print(Dog.en_name)  # dog              — независимо
```

> [!warning] Важно
> `Cat` хранит **ссылку** на атрибут `Animal`, а не копию. `Dog` независим — у него собственный атрибут.

## 📖 Неявное наследование от object

Все классы в Python автоматически наследуются от `object`:

```python
class MyClass:
    pass

# Полностью равнозначно:
class MyClass(object):
    pass
```

Поэтому любой «пустой» класс уже имеет методы `__init__`, `__str__`, `__repr__`, `__eq__`, `__hash__` и др. — все унаследованы от `object`.

> [!tip] Интересный факт
> Декораторы работают в том числе благодаря наследованию от `object` — именно оттуда приходят нужные магические методы.

---

## 📖 Многоуровневое наследование

```python
class A:
    def method_a(self):
        return 'A'

class B(A):
    def method_b(self):
        return 'B'

class C(B):
    pass

c = C()
print(c.method_a())  # A — унаследовано через B
print(c.method_b())  # B — унаследовано от B
```

Цепочка: `C → B → A → object`

---

## 📖 Порядок поиска методов (MRO)

При обращении к методу Python ищет его **снизу вверх по иерархии**, останавливаясь на первом совпадении:

```python
class A:
    def method(self):
        return 'метод A'

class B(A):
    def method(self):
        return 'метод B'

class C(B):
    pass

c = C()
print(c.method())  # метод B — нашли в B, до A не дошли
```

Посмотреть MRO явно:

```python
print(C.__mro__)
# (<class 'C'>, <class 'B'>, <class 'A'>, <class 'object'>)

print(C.mro())
# [<class 'C'>, <class 'B'>, <class 'A'>, <class 'object'>]

print(C.__base__)  # <class 'B'> — только непосредственный родитель
```

## 📖 Проверка наследования и принадлежности

### issubclass() — проверка классов

```python
class A: pass
class B(A): pass
class C(B): pass

print(issubclass(C, B))  # True  — прямой наследник
print(issubclass(C, A))  # True  — косвенный наследник
print(issubclass(A, C))  # False — обратное неверно
print(issubclass(C, C))  # True  — класс — наследник самого себя!
```

### isinstance() — проверка экземпляров

```python
obj = C()
print(isinstance(obj, C))       # True
print(isinstance(obj, B))       # True — через наследование
print(isinstance(obj, A))       # True — через наследование
print(isinstance(obj, object))  # True — все объекты
```

> [!tip] Рекомендация
> Для проверки принадлежности используй `isinstance()` — она учитывает всю иерархию наследования, в отличие от `type(obj) == C`.

---

## 📖 Любопытные факты

### bool наследует от int

```python
print(issubclass(bool, int))  # True
print(isinstance(True, int))  # True
print(True + True)            # 2
```

### Иерархия исключений

Все исключения — классы с иерархией наследования. На вершине — `BaseException`:

```
BaseException
 ├── SystemExit
 ├── KeyboardInterrupt
 └── Exception
      ├── ArithmeticError
      │    └── ZeroDivisionError
      ├── LookupError
      │    ├── IndexError
      │    └── KeyError
      ├── TypeError
      └── ValueError
```

Поэтому `except LookupError` поймает и `IndexError`, и `KeyError`.

---

## 📊 Quick Reference

| Что проверяем | Инструмент | Пример |
|---|---|---|
| Класс — наследник другого | `issubclass(Child, Parent)` | `issubclass(Cat, Animal)` |
| Объект — экземпляр класса | `isinstance(obj, Class)` | `isinstance(cat, Animal)` |
| Непосредственный родитель | `Class.__base__` | `Cat.__base__` → Animal |
| Вся цепочка наследования | `Class.__mro__` | `Cat.__mro__` |
