# 🪄 Магические методы: `__new__`, `__init__`, `__del__`

## 📌 Коротко

> Магические методы (dunder-методы) — специальные методы Python, обрамлённые двойным подчёркиванием. Они не вызываются напрямую, а срабатывают **автоматически** при определённых действиях. Создавать новые нельзя — можно только реализовывать существующие.

---

## 📖 Теория

### 🏗️ Процесс создания объекта

При вызове класса запускается двухшаговый процесс:

```
MyClass()  →  __new__() создаёт пустой объект  →  __init__() инициализирует его
```

- **`__new__(cls)`** — первый шаг, создаёт и **возвращает** новый пустой экземпляр
- **`__init__(self)`** — второй шаг, получает уже созданный объект и устанавливает атрибуты

```python
class Cat:
    def __new__(cls, *args, **kwargs):
        print('1. Создание экземпляра')
        return object.__new__(cls)

    def __init__(self, name):
        print('2. Инициализация экземпляра')
        self.name = name

cat = Cat('Кемаль')
# 1. Создание экземпляра
# 2. Инициализация экземпляра
```

> 💡 Весь этот процесс можно воспроизвести вручную: `cat = Cat.__new__(Cat)` → `Cat.__init__(cat, 'Кемаль')`

---

### 📋 Особенности `__new__()`

|Факт|Пояснение|
|---|---|
|Первый аргумент — `cls`|Принимает сам класс, не экземпляр|
|Технически — статический метод|Но `@staticmethod` не нужен — Python делает это сам|
|Должен вернуть экземпляр своего класса|Иначе `__init__` **не будет вызван**|
|Аргументы при вызове класса|Передаются **и в `__new__`**, и в `__init__`**|

```python
# ❌ __new__ вернул чужой тип → __init__ не вызовется
class MyClass:
    def __new__(cls, *args, **kwargs):
        return 'instance'              # возвращаем str, а не MyClass

    def __init__(self):
        print('Этот принт не выведется')

obj = MyClass()
print(obj)   # → 'instance'  (просто строка, не объект MyClass)
```

```python
# ❌ __init__ не должен возвращать значение
class MyClass:
    def __init__(self):
        return self   # TypeError: __init__() should return None, not 'MyClass'
```

### 🔗 `super()` vs `object.__new__(cls)`

Оба варианта создают объект одинаково:

```python
instance = object.__new__(cls)   # явное обращение к базовому классу
instance = super().__new__(cls)  # через прокси-объект
```

- **`super()`** — используй при наследовании и множественном наследовании, дающий доступ к методам **родительского класса**.
- **`object.__new__(cls)`** — явное и понятное обращение, удобно в паттернах проектирования

### 🎯 Паттерн Singleton

Объект существует **только в одном экземпляре**. При повторном вызове класса возвращается тот же самый объект.

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = object.__new__(cls)
        return cls._instance


first = Singleton()
second = Singleton()
print(first is second)   # → True — это один и тот же объект
```

---

### 🎯 Паттерн Monostate ⭐

Несколько объектов, но **одно общее состояние**. В отличие от Singleton, объектов может быть много, но все они разделяют один `__dict__`.

```python
class Monostate:
    _shared_state = {}

    def __init__(self):
        self.__dict__ = self._shared_state
        if not self._shared_state:
            self._do_work()        # ресурсоёмкая задача — только 1 раз
            self.empty = False

    def _do_work(self):
        print('Выполняется ресурсоёмкая задача')


one = Monostate()
two = Monostate()
print(one is two)        # → False — разные объекты
one.data = 20
two.data = 55
print(one.__dict__)      # → {'empty': False, 'data': 55}  — состояние общее!
print(two.__dict__)      # → {'empty': False, 'data': 55}
```

||Singleton|Monostate|
|---|---|---|
|Кол-во объектов|1|Много|
|Состояние|Общее|Общее|
|Реализация|`__new__`|`__init__` + `__dict__`|

> 💡 Если на собесе рассказать про Monostate после Singleton — будет большой плюс.

### 🗑️ Метод `__del__()` — финализатор

Вызывается **автоматически**, когда счётчик ссылок на объект становится равным нулю.

```python
class MyClass:
    def __del__(self):
        print('Bye')

obj = MyClass()
del obj   # → 'Bye'
```

### 🔢 Оператор `del` и счётчик ссылок

**`del obj`** — удаляет **связь между именем и объектом**, но не сам объект. `__del__` вызывается только когда счётчик ссылок достигает нуля.

```python
obj1 = MyClass()
obj2 = obj1       # счётчик = 2

del obj1          # счётчик = 1, __del__ НЕ вызывается
del obj2          # счётчик = 0, __del__ вызывается → 'Bye'
```

```python
# Локальные переменные удаляются при выходе из функции
def func():
    obj = MyClass()

func()   # → 'Bye'  (obj удалён после завершения функции)
```

---

### ⚠️ Важные нюансы `__del__`

#### `__del__` — НЕ сборщик мусора

```
счётчик ссылок = 0  →  __del__() выполняет финальные действия  →  внутренний механизм удаляет объект
```

- `__del__` — это **финализатор**: выполняет финальные действия _перед_ удалением
- Само удаление делает внутренний механизм CPython — `__del__` его **не заменяет**
- `__del__` может **не вызваться** при циклических ссылках или аварийном завершении интерпретатора

#### Почему `__del__` может не вызваться

```python
class MyClass:
    def __new__(cls):
        return cls      # ❌ возвращаем сам класс, не экземпляр

    def __del__(self):
        print('Bye')

cl = MyClass()
del cl          # 'Bye' НЕ выведется!
```

Причина: `cl = MyClass` (ссылка на класс), экземпляр не создавался → `__del__` некому вызывать.

#### Практические применения `__del__`

- Закрытие файлов / сетевых соединений
- Логирование удаления объекта
- Удаление объекта из кэша
- В играх: действие юнита перед смертью

## 🧩 Шпаргалка

```
object.__new__(cls)   →  создаёт пустой экземпляр
__new__(cls, ...)     →  вызывается первым, возвращает объект
__init__(self, ...)   →  вызывается вторым, инициализирует атрибуты, возвращает None
__del__(self)         →  финализатор, вызывается при счётчик_ссылок == 0
del obj               →  удаляет ссылку, уменьшает счётчик (не удаляет объект напрямую)
```

```python
class Cat:
    def __new__(cls, *args, **kwargs):
        print('1. Создание')
        return object.__new__(cls)   # или super().__new__(cls)

    def __init__(self, name):
        print('2. Инициализация')
        self.name = name             # __init__ всегда возвращает None

    def __del__(self):
        print(f'Объект {self.name} удалён')


cat = Cat('Кемаль')
# 1. Создание
# 2. Инициализация

del cat   # → Объект Кемаль удалён
```
