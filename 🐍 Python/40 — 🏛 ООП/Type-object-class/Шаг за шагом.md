## 🧩 Суть:

> В Python **всё наследуется от `object`**.
> 
> Поэтому **каждый класс** берёт **основную структуру (каркас)** от `object`  
> и **добавляет свои уникальные атрибуты и методы**, чтобы реализовать нужное поведение.

---

## 🔹 Шаг за шагом:

### 1️⃣ `object` — это **универсальный базовый класс**

Он задаёт “скелет” всех объектов:

- каждый объект имеет тип (`__class__`);
    
- может хранить атрибуты (`__dict__`);
    
- может быть выведен (`__repr__`, `__str__`);
    
- может сравниваться (`__eq__`, `__ne__`), и т.д.
    

---

### 2️⃣ Наследник (например, `int`)

`int` берёт этот “скелет” и **расширяет его своими особенностями**:

- умеет хранить целое число;
    
- знает, как складываться (`__add__`);
    
- знает, как сравниваться (`__lt__`, `__gt__`);
    
- умеет показывать себя (`__str__`).
    

---

### 3️⃣ Аналогия 💡

🦴 `object` — это **скелет** (форма, общая структура).  
🧠 `int`, `str`, `list` и другие — это **конкретные “живые организмы”**,  
которые “наделяют” этот скелет собственными органами (методами) и способностями (атрибутами).

---

### 4️⃣ Пример в коде:

`print(issubclass(int, object))  # True — int наследует object print(dir(object))              # базовые атрибуты print(dir(int))                 # базовые + уникальные для int`

`dir(object)` даёт основу,  
а `dir(int)` — ту же основу **плюс** дополнительные методы `int`.

---

✅ Итак, ты абсолютно точно понял:

> Все классы в Python “рождаются” от `object` и строятся по его шаблону,  
> а затем дополняются своими уникальными свойствами.
