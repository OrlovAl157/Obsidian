# 🔍 Протокол дескрипторов

## 📌 Коротко

> Дескриптор — это объект у которого определён хотя бы один из методов `__get__`, `__set__`, `__delete__`. Он позволяет перехватывать доступ к атрибуту и добавлять логику — как `@property`, но **переиспользуемый** для любого количества атрибутов.

---

## ⚖️ Дескриптор vs `@property`

```python
# @property — дублирование кода для каждого атрибута
class Cat:
    @property
    def name(self): return self._name
    @name.setter
    def name(self, v): ...   # одна и та же логика

    @property
    def breed(self): return self._breed
    @breed.setter
    def breed(self, v): ...  # снова та же логика


# Дескриптор — один раз написал, используй везде
class Cat:
    name  = CatDescriptor()   # одна строка!
    breed = CatDescriptor()   # одна строка!
    color = CatDescriptor()   # одна строка!
```

**Когда использовать дескриптор:**

- много атрибутов с одинаковой логикой
- логику нужно переиспользовать в разных классах

---

## 🔑 Базовый шаблон

### Новый способ (Python 3.6+)

```python
class CatDescriptor:
    def __set_name__(self, cls, attr):
        self._attr = attr

    def __get__(self, obj, cls):
        if obj is None:
            return self
        return obj.__dict__[self._attr]

    def __set__(self, obj, value):
        obj.__dict__[self._attr] = value

    def __delete__(self, obj):
        del obj.__dict__[self._attr]

class Cat:
    name  = CatDescriptor()
    breed = CatDescriptor()
    color = CatDescriptor()

    def __init__(self, name, breed, color):
        self.name  = name
        self.breed = breed
        self.color = color
```

### Старый способ (до Python 3.6)

```python
class CatDescriptor:
    def __init__(self, attr):
        self._attr = attr       # имя передаём вручную

    def __get__(self, obj, cls):
        if obj is None:
            return self
        return obj.__dict__[self._attr]

    def __set__(self, obj, value):
        obj.__dict__[self._attr] = value

    def __delete__(self, obj):
        del obj.__dict__[self._attr]

class Cat:
    name = CatDescriptor('name')  # имя передаём явно!
```

## 📖 Теория

### Цепочка поиска атрибута

```
cat.name
    ↓
object.__getattribute__(cat, 'name')
    ↓
1. Смотрит в Cat.__dict__ (словарь КЛАССА)
   └── есть там объект с именем 'name'?
       └── да, есть — это дескриптор
           └── у него есть __get__?
               └── да → вызываем Cat.__dict__['name'].__get__(cat, Cat)
                        и возвращаем результат  ← СТОП, дальше не идём

2. Если в классе не нашли дескриптор —
   смотрим в cat.__dict__ (словарь ЭКЗЕМПЛЯРА)
   └── есть там ключ 'name'?
       └── да → возвращаем cat.__dict__['name']

3. Если и там нет — AttributeError
```

### `__getattribute__` под капотом

```python
obj = Cat.__dict__.get('name')
if obj is not None and hasattr(obj, '__get__'):
    return obj.__get__(cat, Cat)   # дескриптор нашли → вызываем __get__

if 'name' in cat.__dict__:
    return cat.__dict__['name']

raise AttributeError
```

### `__set_name__` — как дескриптор узнаёт своё имя

```
class Cat:
    name  = NonEmptyString()   ← Python создаёт класс
    breed = NonEmptyString()      и сразу говорит каждому
    color = NonEmptyString()      дескриптору его имя

          ↓ автоматически вызывается

NonEmptyString.__set_name__(Cat, 'name')   → self._attr = 'name'
NonEmptyString.__set_name__(Cat, 'breed')  → self._attr = 'breed'
NonEmptyString.__set_name__(Cat, 'color')  → self._attr = 'color'

          ↓ теперь каждый дескриптор знает своё имя

def __get__(self, obj, cls):
    return obj.__dict__[self._attr]
                         ↑
                    знает благодаря __set_name__
```

### Параметры `__set_name__`

```python
def __set_name__(self, cls, attr):
#                 ↑     ↑    ↑
#    объект дескриптора  |    имя переменной ('name', 'breed', 'color')
#    (NonEmptyString())  |
#                     класс где объявлен дескриптор (Cat)

class Cat:
    name = NonEmptyString()
    ↑              ↑
   attr           self

   cls = Cat
```

> **Зачем `cls`?** В большинстве случаев не используется. Может пригодиться при наследовании или если нужно знать в каком классе объявлен дескриптор.

### Порядок `__init__` и `__set_name__`

```python
class LimitedTakes:
    def __init__(self, times):
        # ШАГ 1 — вызывается первым при LimitedTakes(3)
        # создаётся обычный объект
        self.times = times
        self.count = 0
        # self._attr ещё не существует!

    def __set_name__(self, cls, attr):
        # ШАГ 2 — вызывается вторым
        # объект становится дескриптором
        self._attr = attr


class Cat:
    name = LimitedTakes(3)

# Порядок:
# 1. LimitedTakes(3)  → __init__     — создаём объект
# 2. name = ...       → __set_name__ — регистрируем как дескриптор
```

### `if obj is None` — обращение через класс

```python
def __get__(self, obj, cls):
    if obj is None:                    # обращение через КЛАСС
        return self                    # возвращаем сам дескриптор
    return obj.__dict__[self._attr]    # обращение через ЭКЗЕМПЛЯР

cat.name   # obj = cat  → возвращает значение атрибута
Cat.name   # obj = None → возвращает сам дескриптор

# без проверки — ошибка:
# None.__dict__ → AttributeError!
```

> **Зачем `cls` в `__get__`?** Аналогично `__set_name__` — в большинстве случаев не используется. Передаётся на случай если нужно знать класс при обращении.

### Дескриптор данных vs не-данных

```python
# Дескриптор НЕ-ДАННЫХ — только __get__
# приоритет НИЖЕ __dict__ экземпляра

# Дескриптор ДАННЫХ — __get__ + __set__ (или __delete__)
# приоритет ВЫШЕ __dict__ экземпляра


# Полный порядок поиска:
# 1. Дескриптор ДАННЫХ     (класс)   — есть __set__ или __delete__
# 2. __dict__ экземпляра
# 3. Дескриптор НЕ-ДАННЫХ  (класс)   — только __get__
# 4. __dict__ класса
# 5. AttributeError


cat.__dict__['x'] = 'экземпляр'   # напрямую в словарь
cat.__dict__['y'] = 'экземпляр'

print(cat.x)   # → 'дескриптор' — DataDesc выиграл (есть __set__)
print(cat.y)   # → 'экземпляр'  — __dict__ выиграл (только __get__)
```

### Последовательность поиска для не-данных дескриптора

```
obj.x
    ↓
__getattribute__
    ↓
смотрит в MyClass.__dict__ — находит Descriptor
    ↓
проверяет — есть __set__ или __delete__?
    ↓
НЕТ → дескриптор не-данных → низкий приоритет
    ↓
смотрит в obj.__dict__ — есть 'x'?
    ↓
ДА → возвращает значение из экземпляра
```

### Где хранятся дескрипторы и значения

```python
Cat.__dict__ = {
    'name':  <CatDescriptor object>,  # _attr = 'name'
    'breed': <CatDescriptor object>,  # _attr = 'breed'
    'color': <CatDescriptor object>,  # _attr = 'color'
}

cat.__dict__ = {
    'name':  'Кемаль',   # реальные значения
    'breed': 'Сиам',
    'color': 'белый',
}
# дескриптор в классе — один для всех экземпляров!
# значения в экземпляре — у каждого свои
```

## ⚠️ Нюансы

### Рекурсия — почему `obj.__dict__` напрямую

```python
def __get__(self, obj, cls):
    return obj.name        # ❌ рекурсия! → снова вызовет __get__

def __get__(self, obj, cls):
    return obj.__dict__[self._attr]   # ✅ напрямую в словарь
```

### Не храни значения в самом дескрипторе!

```python
# ❌ Неправильно — дескриптор один для всех экземпляров!
class Descriptor:
    def __init__(self):
        self.value = None   # общее для всех экземпляров!

    def __set__(self, obj, value):
        self.value = value   # cat1 и cat2 перезапишут друг друга!

# ✅ Правильно — храни в obj.__dict__
class Descriptor:
    def __set__(self, obj, value):
        obj.__dict__[self._attr] = value   # у каждого экземпляра своё!
```
