# 📋 Протокол последовательностей

## 📌 Коротко

> Последовательность — объект который поддерживает индексацию, срезы, проверку принадлежности, длину и итерирование. Обязательны только `__len__` и `__getitem__` — остальное Python может сделать сам.

---

## 📖 Утиная типизация

> "Если объект ведёт себя как последовательность — он последовательность, независимо от типа."

Не проверяем тип — проверяем поведение.

---

## ✅ Какие методы обязательны, а какие нет

|Метод|Обязателен?|Почему|
|---|---|---|
|`__len__`|✅ **Да**|без него нет длины|
|`__getitem__`|✅ **Да**|без него нет индексации и срезов|
|`__iter__`|❌ Нет|если есть `__len__` + `__getitem__` — Python итерирует сам через индексы|
|`__contains__`|❌ Нет|если нет — Python перебирает через `__iter__` или индексы|
|`__reversed__`|❌ Нет|если есть `__len__` + `__getitem__` — `reversed()` работает сам|

> 💡 Минимум для последовательности: **`__len__` + `__getitem__`** — всё остальное Python умеет сделать сам!

---

## ⚠️ Зачем тогда определять необязательные методы?

**`__iter__`** — явно определённый быстрее чем автоматический перебор по индексам:

```python
# Без __iter__ — Python делает так:
i = 0
while True:
    try:
        yield obj[i]
        i += 1
    except IndexError:
        break
```

**`__contains__`** — можно сделать O(1) если внутри `set`, иначе Python перебирает через `__iter__`:

```python
# Без __contains__:
for item in self:   # перебирает всё — O(n)
    if item == target:
        return True
return False
```

**`__reversed__`** — кастомная логика разворота если нужна.

## 🔧 Методы протокола

### `__len__` — длина

```python
def __len__(self):
    return len(self.cart)
```

> ⚠️ Если `__bool__` не определён — `__len__` используется для логических приведений! `len == 0` → `False`.

### `__getitem__` — индексация и срезы

```python
def __getitem__(self, key):
    if isinstance(key, slice):                   # ← сначала проверяем срез!
        return Order(self.cart[key], self.customer)
    if not isinstance(key, int):
        raise TypeError('Индекс должен быть целым числом')
    if key < ~len(self.cart) or key >= len(self.cart):   # ← поддержка отрицательных!
        raise IndexError('Неверный индекс')
    return self.cart[key]
```

> 💡 `key < ~len(self.cart)` — поддержка отрицательных индексов через `~`. `~n = -(n+1)`.

### `__setitem__` и `__delitem__` — изменяемая последовательность

```python
def __setitem__(self, key, value):   # obj[i] = value
    key = self.check_key(key)
    self.cart[key] = value

def __delitem__(self, key):          # del obj[i]
    key = self.check_key(key)
    del self.cart[key]
```

### `__contains__` — проверка принадлежности

```python
def __contains__(self, item):
    return item in self.cart   # делегируем списку
```

### `__iter__` — итерирование

```python
def __iter__(self):
    yield from self.cart   # или return iter(self.cart)
```

### `__reversed__` — разворот

```python
def __reversed__(self):
    return reversed(self.cart)
```

## 🍕 Срезы — объект `slice`

При `obj[1:3]` Python автоматически создаёт `slice(1, 3)` и передаёт в `__getitem__`:

```python
obj[1:]      → slice(1, None, None)
obj[:3]      → slice(None, 3, None)
obj[1:3]     → slice(1, 3, None)
obj[1:4:2]   → slice(1, 4, 2)
```

Атрибуты: `start`, `stop`, `step` — неуказанные равны `None`.

#### Метод `indices()` — удобная обработка None

```python
s = slice(1, None, None)   # obj[1:]
s.indices(5)               # → (1, 5, 1)  — None заменён на реальные значения
```

## 📦 Полный шаблон

### Неизменяемая последовательность

```python
class MySequence:
    def __init__(self, data):
        self._data = list(data)

    def __len__(self):
        return len(self._data)

    def __getitem__(self, key):
        if isinstance(key, slice):
            return MySequence(self._data[key])
        if not isinstance(key, int):
            raise TypeError('Индекс должен быть целым числом')
        if key < ~len(self._data) or key >= len(self._data):
            raise IndexError('Неверный индекс')
        return self._data[key]

    def __contains__(self, item):        # необязателен, но ускоряет
        return item in self._data

    def __iter__(self):                  # необязателен, но явно быстрее
        yield from self._data

    def __reversed__(self):              # необязателен
        return reversed(self._data)
```

### Изменяемая последовательность (+ setitem и delitem)

```python
    def _check_key(self, key):
        if not isinstance(key, int):
            raise TypeError('Индекс должен быть целым числом')
        if key < ~len(self._data) or key >= len(self._data):
            raise IndexError('Неверный индекс')
        return key

    def __setitem__(self, key, value):
        key = self._check_key(key)
        self._data[key] = value

    def __delitem__(self, key):
        key = self._check_key(key)
        del self._data[key]
```

## 🎁 Бонусы от реализации протокола

```python
# __len__ + __getitem__ дают:
len(obj)          # длина
obj[i]            # индексация
obj[1:3]          # срезы
for x in obj      # итерирование (автоматически!)
reversed(obj)     # разворот (автоматически!)
from random import choice
choice(obj)       # случайный элемент ✅
```
