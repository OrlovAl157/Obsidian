```markdown
# Универсальная формула контекстного менеджера

## ШАГ 1 — создаётся объект

Вызывается выражение после `with`. Это может быть:
- вызов класса: `ContextManager()`
- вызов функции: `open('file.txt')`
- готовый объект: `context` (уже созданный)

## ШАГ 2 — вызывается `__enter__()`

Python автоматически вызывает его на объекте.
Внутри `__enter__` происходит подготовка:
- открытие файла
- создание копии
- установка флага
- захват блокировки

`__enter__` может вернуть:
- `self` → сам объект менеджера
- `self._file` → другой объект (как в `File`)
- копию → как в `DefenedVector`
- `None` → если `as` не нужен

## ШАГ 3 — результат `__enter__` → переменная после `as`

with ContextManager() as obj:
#                      ↑
#         сюда попадает то, что вернул __enter__

Если `as` нет — результат просто игнорируется:

with ContextManager():   # тоже валидно

## ШАГ 4 — выполняется тело блока

Здесь работаем с `obj`. Может возникнуть исключение или нет.

## ШАГ 5 — вызывается `__exit__()`

Python автоматически вызывает его **всегда**:
- и когда всё хорошо
- и когда была ошибка

Параметры `__exit__`:

| Параметр   | Если ошибки не было | Если была ошибка   |
|------------|---------------------|--------------------|
| `exc_type` | `None`              | тип исключения     |
| `exc_val`  | `None`              | объект ошибки      |
| `exc_tb`   | `None`              | traceback          |

## ШАГ 6 — логика внутри `__exit__`

def __exit__(self, exc_type, exc_val, exc_tb):
    if exc_type is None:
        # ошибки не было → сохраняем результат, закрываем ресурс
        pass
    else:
        # была ошибка → откатываем, логируем, игнорируем
        pass

## ШАГ 7 — возвращаемое значение

return False  # исключение НЕ подавляется → идёт дальше наружу
return True   # исключение подавляется → код продолжается как ни в чём не бывало
return None   # то же самое что False
```
