# 🔒 Контекстные менеджеры

## 📌 Коротко

> Контекстный менеджер — это инструмент который позволяет **гарантированно** выполнить действия до и после блока кода, независимо от того была ошибка или нет.

---

## 🚀 Быстрая справка

### Протокол

|Метод|Когда вызывается|Возвращает|
|---|---|---|
|`__enter__`|при входе в блок `with`|объект для `as` или `None`|
|`__exit__`|при выходе из блока `with` — всегда|`True` — подавить ошибку, `False` — пробросить|

### 4 паттерна использования

|Паттерн|Пример|
|---|---|
|Открытие/закрытие ресурса|файлы, соединения с БД, сокеты|
|Временное изменение состояния|`sys.stdout`, рекурсия, таймер|
|Транзакция|применить или откатить изменения|
|Блокировка|`threading.Lock`, многопоточность|

### 3 категории менеджеров

|Категория|Описание|Пример|
|---|---|---|
|Одноразовый|только один раз, потом пересоздавать|`open()`|
|Многоразовый|повторно в невложенных `with`|`Timer`|
|Реентерабельный|повторно во вложенных `with`|`Indenter`|

---

## 📖 Теория

### Протокол контекстного менеджера

Любой объект у которого есть два метода — `__enter__` и `__exit__` — может использоваться с `with`.

```python
class CustomContextManager:
    def __enter__(self):
        # подготовка — выполняется до блока
        return self   # или None, или другой объект

    def __exit__(self, exc_type, exc_val, exc_tb):
        # завершение — выполняется всегда после блока
        return False  # True — подавить ошибку, False — пробросить
```

### Параметры `__exit__`

```python
def __exit__(self, exc_type, exc_val, exc_tb):
    # если ошибки не было — все три параметра None
    # если была ошибка:
    #   exc_type — тип исключения  (например IndexError)
    #   exc_val  — объект ошибки   (например "list index out of range")
    #   exc_tb   — traceback        (информация о трассировке)
    
    if exc_type is None:
        pass  # всё хорошо
    
    return False   # False → ошибка идёт дальше
    # return True  → ошибка подавляется
```

> Если параметры не нужны — можно использовать `*exc_info`:
> 
> ```python
> def __exit__(self, *exc_info):
>     ...
> ```

## 🔑 Универсальная формула

```python
# ШАГ 1 — создаётся объект
# вызывается выражение после with:
#   - вызов класса:   ContextManager()
#   - вызов функции:  open('file.txt')
#   - готовый объект: context (уже созданный)

manager = ContextManager()


# ШАГ 2 — вызывается __enter__()
# Python автоматически вызывает его на объекте
# внутри __enter__ происходит подготовка:
#   - открытие файла
#   - создание копии
#   - установка флага
#   - захват блокировки

# __enter__ может вернуть:
#   - self        → сам объект менеджера
#   - self._file  → другой объект (как в File)
#   - копию       → как в DefenedVector
#   - None        → если as не нужен


# ШАГ 3 — результат __enter__ → переменная после as
# with ContextManager() as obj:
#                         ↑
#              сюда попадает то, что вернул __enter__

# если as нет — результат просто игнорируется
# with ContextManager():   ← тоже валидно


# ШАГ 4 — выполняется тело блока
# здесь работаем с obj
# может возникнуть исключение или нет


# ШАГ 5 — вызывается __exit__()
# Python автоматически вызывает его ВСЕГДА
# — и когда всё хорошо
# — и когда была ошибка

# параметры __exit__:
#   exc_type  — тип исключения  (None если ошибки не было)
#   exc_val   — объект ошибки   (None если ошибки не было)
#   exc_tb    — traceback        (None если ошибки не было)


# ШАГ 6 — логика внутри __exit__
def __exit__(self, exc_type, exc_val, exc_tb):
    if exc_type is None:
        # ошибки не было → сохраняем результат, закрываем ресурс
        pass
    else:
        # была ошибка → откатываем, логируем, игнорируем
        pass

    # ШАГ 7 — возвращаемое значение
    return False  # исключение НЕ подавляется → идёт дальше наружу
    # return True   → исключение подавляется
    # return None   → то же самое что False
```

## 💻 Примеры кода

### Trace — логирование входа и выхода

```python
class Trace:
    def __enter__(self):
        print('Начало выполнения блока with')

    def __exit__(self, exc_type, exc_value, traceback):
        if exc_value:
            print(f'Была ошибка: {exc_value}')
        print('Конец выполнения блока with')
        return True   # подавляем все исключения


with Trace():
    print('Python generation!')
    print(1 / 0)

# Начало выполнения блока with
# Python generation!
# Была ошибка: division by zero
# Конец выполнения блока with
```

### Timer — замер времени (многоразовый)

```python
from time import perf_counter, sleep

class Timer:
    def __enter__(self):
        self.start = perf_counter()
        return self

    def __exit__(self, *exc_info):
        self.elapsed = perf_counter() - self.start


timer = Timer()

with timer:
    sleep(1.5)
print('Затраченное время:', timer.elapsed)  # ~1.5

with timer:                                  # используем повторно!
    sleep(0.7)
print('Затраченное время:', timer.elapsed)  # ~0.7
```

### Timer с лямбдой — отложенное вычисление

```python
from time import perf_counter, sleep

class Timer:
    def __enter__(self):
        self.start = perf_counter()
        self.end = 0.0                        # страховка если ошибка в __enter__
        return lambda: self.end - self.start  # возвращаем функцию, не результат!

    def __exit__(self, *exc_info):
        self.end = perf_counter()             # записывается после блока


with Timer() as timer:
    sleep(0.7)

print('Затраченное время:', timer())          # вызываем лямбду — теперь self.end заполнен
```

### DefenedVector — транзакция (откат при ошибке)

```python
class DefenedVector:
    def __init__(self, v):
        self.__v = v                   # сохраняем ссылку на оригинал

    def __enter__(self):
        self.__temp = self.__v[:]      # создаём копию для работы
        return self.__temp

    def __exit__(self, exc_type, exc_val, exc_tb):
        if exc_type is None:
            self.__v[:] = self.__temp  # успех → применяем изменения
        return False                   # ошибка → откат (оригинал не тронут)


v1 = [1, 2, 3]
v2 = [2, 3, 4]

with DefenedVector(v1) as dv:
    for i, a in enumerate(dv):
        dv[i] = a + v2[i]

print(v1)  # [3, 5, 7]
```

### Indenter — реентерабельный (вложенные with)

```python
class Indenter:
    def __init__(self):
        self.level = -1

    def __enter__(self):
        self.level += 1
        return self

    def __exit__(self, *exc_info):
        self.level -= 1

    def print(self, text):
        print('    ' * self.level + text)


with Indenter() as indent:
    indent.print('python')
    with indent:
        indent.print('beegeek')
        with indent:
            indent.print('stepik')
        indent.print('pygen')
    indent.print('bye-bye')

# python
#     beegeek
#         stepik
#     pygen
# bye-bye
```

## 🔧 Под капотом

### Развёртка `with` в обычный код

```python
# with open('test.txt') as f:
#     f.write('Hello!')

# ШАГ 1 — создаётся объект
manager = open('test.txt')

# ШАГ 2 — достаём методы из класса
enter = type(manager).__enter__
exit  = type(manager).__exit__

# ШАГ 3 — вызываем __enter__, получаем объект для работы
f = enter(manager)

hit_except = False

try:
    # ШАГ 4 — выполняется тело блока
    f.write('Hello!')

except:
    # ШАГ 5 — если была ошибка
    hit_except = True
    if not exit(manager, *sys.exc_info()):
        raise                    # если False — пробрасываем дальше

finally:
    # ШАГ 6 — если ошибки не было
    if not hit_except:
        exit(manager, None, None, None)
```

### Две роли в контекстном менеджере

```python
import builtins

def open(path, mode='r', encoding=None):
    return File(path, mode, encoding)   # фабричная функция → создаёт объект


class File:
    def __init__(self, path, mode, encoding):
        # файл открывается здесь, при создании объекта
        self._file = builtins.open(path, mode, encoding=encoding)

    def __enter__(self):
        return self._file    # возвращаем файловый объект, не self!

    def __exit__(self, exc_type, exc_val, exc_tb):
        self._file.close()   # закрываем файл всегда
        return False


with open('test.txt', 'w', encoding='utf-8') as f:
    f.write('Hello!')
```

> **Важно:** контекстный менеджер не обязан быть тем же объектом с которым работаешь внутри `with`:
> 
> |Роль|Объект|
> |---|---|
> |Управление ресурсом|`File`|
> |Работа внутри блока|`_file`|
