# 🔄 Итерируемые объекты и итераторы

## 📌 Коротко

> Итерируемый объект — можно перебрать, есть `__iter__`. Итератор — выдаёт элементы по одному, есть `__iter__` и `__next__`. Любой итератор является итерируемым, но не наоборот.

---

## 🏗 Типовые шаблоны

### Итерируемый объект — 3 варианта `__iter__`

```python
class MyIterable:
    def __init__(self, data):
        self.data = data

    # Вариант 1 — делегируем встроенному iter():
    def __iter__(self):
        return iter(self.data)

    # Вариант 2 — через yield from:
    def __iter__(self):
        yield from self.data

    # Вариант 3 — через yield (явный):
    def __iter__(self):
        for el in self.data:
            yield el

    # Вариант 4 — генераторное выражение:
    def __iter__(self):
        return (el for el in self.data)
```

> 💡 Все четыре варианта эквивалентны. `yield from` — самый короткий. `iter()` — самый быстрый.

---

### Класс-итератор — шаблон

```python
class MyIterator:
    def __init__(self, data):
        self.data = data
        self.index = 0          # ← сохраняем состояние!

    def __iter__(self):
        return self             # ← возвращаем себя

    def __next__(self):
        if self.index >= len(self.data):
            raise StopIteration
        value = self.data[self.index]
        self.index += 1
        return value
```

### Бесконечный итератор — шаблон

```python
class InfiniteIterator:
    def __init__(self, start=0):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):
        value = self.current
        self.current += 1
        return value            # StopIteration не вызывается → бесконечный!
```

> ⚠️ Бесконечный итератор без `break` или `islice` → бесконечный цикл!

## 📖 Теория

### Два типа итерируемых объектов

```
Итерируемые объекты
├── Коллекции/последовательности (list, str, tuple, set, dict, range)
│   └── есть __iter__, НЕТ __next__
│   └── многоразовые — можно итерировать сколько угодно раз
│
└── Итераторы (list_iterator, enumerate, zip, генераторы...)
    └── есть __iter__ И __next__
    └── одноразовые — состояние сохраняется, элементы исчезают
```

---

### Протокол итерируемого объекта

Объект является итерируемым если у него есть `__iter__()`:

```python
words = ['hello', 'beegeek', 'python']

iterator = iter(words)        # вызывает words.__iter__()
print(type(words))            # → <class 'list'>
print(type(iterator))         # → <class 'list_iterator'>
```

> 💡 `iter(obj)` — вызывает `obj.__iter__()` под капотом.

---

### Протокол итератора

Итератор имеет **оба** метода:

|Метод|Что делает|
|---|---|
|`__iter__()`|возвращает **себя** (`return self`)|
|`__next__()`|возвращает следующий элемент, сохраняет состояние|

```python
iterator = iter([1, 2, 3])

print(next(iterator))   # → 1   (вызывает iterator.__next__())
print(next(iterator))   # → 2
print(next(iterator))   # → 3
print(next(iterator))   # → StopIteration!
```

> ⚠️ Когда элементы закончились — `StopIteration`. Итератор "мёртв", элементы не вернуть.

---

### Почему у итератора есть `__iter__`?

Цикл `for` **всегда** вызывает `__iter__` — он не знает что ему передали, итератор или итерируемый объект. Поэтому итератор тоже должен иметь `__iter__` для совместимости:

```python
iterator1 = iter([1, 2, 3])
iterator2 = iter(iterator1)    # передаём итератор в iter()

print(iterator1 is iterator2)  # → True  ← возвращает себя!
```

> 💡 Итератор возвращает `self` из `__iter__` — потому что он уже итератор, преобразовывать нечего. Это нужно только для совместимости с `for`.

---

### Итератор — одноразовый!

```python
it = iter([1, 2, 3])

print(list(it))   # → [1, 2, 3]
print(list(it))   # → []  ← пустой! состояние сохранилось

# Аналогично с in:
it = iter([1, 2, 3])
print(2 in it)    # → True  (прошёл мимо 1 и 2, остановился)
print(2 in it)    # → False (2 уже позади, итератор продвинулся дальше)
```

> ⚠️ Итератор помнит где остановился. Вернуться назад нельзя — только создать новый.

---

### Коллекция — многоразовая!

```python
lst = [1, 2, 3]

print(list(lst))   # → [1, 2, 3]
print(list(lst))   # → [1, 2, 3]  ← снова всё!

# Каждый раз создаётся новый итератор:
it1 = iter(lst)
it2 = iter(lst)
print(it1 is it2)  # → False  ← разные объекты!
```

---

### range — особый случай

`range` — **не итератор**, а **ленивая последовательность**:

```python
r = range(3)

print(next(r))     # → TypeError: 'range' object is not an iterator

# Но итерировать можно сколько угодно:
print(list(r))     # → [0, 1, 2]
print(list(r))     # → [0, 1, 2]  ← снова!

# Поддерживает индексацию и срезы:
r = range(5, 50, 5)
print(r[3])        # → 20
print(r[-1])       # → 45
print(r[4:8])      # → range(25, 45, 5)
print(len(r))      # → 9
```

> 💡 `range` похож на коллекцию но не хранит все элементы в памяти — вычисляет по необходимости.

---

### `iter()` с двумя аргументами

Особый режим — `iter(callable, sentinel)`:

```python
# Создаёт итератор который вызывает функцию до тех пор
# пока результат не равен sentinel:

zero_iterator = iter(int, -1)   # int() всегда возвращает 0, sentinel = -1

for i in range(5):
    print(next(zero_iterator))  # → 0, 0, 0, 0, 0  (бесконечно!)
```

### Три признака итерируемого/итератора

```python
# 1. Итерируемый объект — не вызывает TypeError при iter():
iter([1, 2, 3])    # ✅
iter(42)           # ❌ TypeError

# 2. Итератор — не вызывает TypeError при next():
next(iter([1]))    # ✅
next([1, 2, 3])    # ❌ TypeError

# 3. Итератор — возвращает себя при iter():
it = iter([1, 2, 3])
it is iter(it)     # → True ← это итератор!
lst = [1, 2, 3]
lst is iter(lst)   # → False ← это просто итерируемый
```

## 🧩 Примеры классов-итераторов

### Counter — конечный итератор

Генерирует числа от `low` до `high` включительно:

```python
class Counter:
    def __init__(self, low, high):
        self.low = low
        self.high = high

    def __iter__(self):
        return self              # возвращаем себя — мы итератор!

    def __next__(self):
        if self.low > self.high:
            raise StopIteration  # элементы закончились
        self.low += 1
        return self.low - 1      # возвращаем текущее, потом двигаемся


counter = Counter(3, 6)
for i in counter:
    print(i)   # → 3, 4, 5, 6
```

---

### EvenNumbers — бесконечный итератор

Генерирует чётные числа от `begin` до бесконечности:

```python
class EvenNumbers:
    def __init__(self, begin):
        self.begin = begin + begin % 2   # округляем до чётного вверх

    def __iter__(self):
        return self

    def __next__(self):
        value = self.begin
        self.begin += 2
        return value             # StopIteration не вызывается → бесконечный!


evens = EvenNumbers(10)
for index, num in enumerate(evens):
    if index > 4:
        break
    print(num)   # → 10, 12, 14, 16, 18
```

> ⚠️ Бесконечный итератор — не вызывает `StopIteration`. Без ограничения (`break`, `islice`) — бесконечный цикл!

---

### Factorials — бесконечный итератор факториалов

```python
class Factorials:
    def __init__(self):
        self.value = 1
        self.index = 1

    def __iter__(self):
        return self

    def __next__(self):
        self.value *= self.index
        self.index += 1
        return self.value


facts = Factorials()
for index, num in enumerate(facts, 1):
    if index > 5:
        break
    print(f'{index}! = {num}')
# → 1! = 1, 2! = 2, 3! = 6, 4! = 24, 5! = 120
```

### Три варианта реализации `__iter__` в итерируемом объекте

```python
class Order:
    def __init__(self, cart):
        self.cart = list(cart)

    # Вариант 1 — через yield from (короткий, pythonic):
    def __iter__(self):
        yield from self.cart

    # Вариант 2 — через yield (явный):
    def __iter__(self):
        for item in self.cart:
            yield item

    # Вариант 3 — через iter() (делегируем списку):
    def __iter__(self):
        return iter(self.cart)
```

> 💡 Все три варианта эквивалентны. `yield from` — самый короткий и pythonic.

## 🏗 Шаблон — как реализовать в своём классе

### Итерируемый объект (не итератор)

```python
class MyIterable:
    def __init__(self, data):
        self.data = data

    def __iter__(self):
        return iter(self.data)       # делегируем встроенному итератору
        # или:
        # yield from self.data       # через генератор
        # или:
        # for item in self.data:
        #     yield item
```

### Класс-итератор

```python
class MyIterator:
    def __init__(self, data):
        self.data = data
        self.index = 0              # сохраняем состояние!

    def __iter__(self):
        return self                 # возвращаем себя

    def __next__(self):
        if self.index >= len(self.data):
            raise StopIteration
        value = self.data[self.index]
        self.index += 1
        return value
```

## 📊 Сравнение

||Итерируемый объект|Итератор|range|
|---|---|---|---|
|`__iter__`|✅ (возвращает итератор)|✅ (возвращает self)|✅|
|`__next__`|❌|✅|❌|
|Многоразовый|✅|❌|✅|
|Индексация|зависит|❌|✅|
|`len()`|зависит|❌|✅|
|Пример|list, str, dict|enumerate, zip, генератор|range|

## ⚠️ Частые ошибки

```python
# ❌ range — не итератор:
next(range(3))          # TypeError!

# ❌ list — не итератор:
next([1, 2, 3])         # TypeError!

# ❌ Итератор одноразовый:
it = iter([1, 2, 3])
list(it)                # [1, 2, 3]
list(it)                # []  ← пустой!

# ✅ Правильно — создать новый итератор:
lst = [1, 2, 3]
list(iter(lst))         # [1, 2, 3]
list(iter(lst))         # [1, 2, 3]
```
