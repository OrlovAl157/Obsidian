# ⤷ Устройство @property

## 📌 Коротко

> `@property` создаёт объект-дескриптор с тремя слотами (`fget`, `fset`, `fdel`). Каждый вызов `.setter` / `.deleter` создаёт **новый** объект `property`, копируя старые слоты и добавляя новый.

---

## 📖 Теория

### Базовый шаблон (полная форма)

Это каркас, который нужно держать в голове. Всегда начинается с подчёркнутого хранилища.

```python
class ClassName:
    def __init__(self, value):
        self._value = value      # реальное хранилище (protected)

    @property
    def value(self):             # геттер
        """Документация для свойства"""
        return self._value

    @value.setter
    def value(self, new_value):  # сеттер
        self._value = new_value

    @value.deleter
    def value(self):             # делитер
        del self._value
```

---

### Механика: «Что вместо чего»

`@property` заменяет явные геттеры/сеттеры (Java-style) на естественный доступ к атрибутам.

| Было (методы) | Стало (property) |
|:---|:---|
| `obj.get_value()` | `obj.value` |
| `obj.set_value(10)` | `obj.value = 10` |
| `del obj.value` (если есть метод) | `del obj.value` |

> **Суть:** Property превращает методы в **дескриптор**, перехватывающий чтение, запись и удаление.

---

### Минимальные варианты (частые кейсы)

#### Только чтение (Read-only)

```python
@property
def total(self):
    return self._price + self._tax
# obj.total = 10 → AttributeError: can't set attribute
```

#### Getter + Setter (валидация)

```python
@property
def age(self):
    return self._age

@age.setter
def age(self, value):
    if not 0 <= value <= 150:
        raise ValueError("Некорректный возраст")
    self._age = value
```

---

### Тонкости и важные правила

#### ⚠️ Имена должны совпадать!

```python
@property
def age(self): ...

@age.setter    # ❌ Ошибка: если написать @year.setter — не привяжется
def age(self, value): ...
```

#### ⚠️ Подчеркивание — это не защита, а «знак вежливости»

```python
obj._value = 999  # ❌ Технически это сломает логику, но Python не запретит
```

> Никогда не обращайся к `_value` напрямую за пределами класса. Это соглашение "мы все взрослые люди".

#### ⚠️ Read-only = отсутствие setter

Если есть только `@property`, свойство становится неизменяемым. Удобно для констант или вычисляемых значений, которые не должны меняться вручную.

---

### Зачем это нужно? (Сценарии)

**1. Инкапсуляция и контроль** — скрываем реальное хранилище (`_value`), предоставляя чистый интерфейс (`value`).

**2. Валидация данных** — ловим некорректные значения в сеттере до того, как они испортят состояние объекта.

**3. Вычисляемые свойства** — позволяют обращаться к методу как к атрибуту (без скобок):

```python
@property
def full_name(self):
    return f"{self.first} {self.last}"
# Использование: obj.full_name, а не obj.full_name()
```

**4. Совместимость (рефакторинг)** — можно начать с простого атрибута `obj.age`, а потом добавить логику через property, не меняя код, который использует `obj.age`.

---

### Взгляд под капот (дескрипторы и контейнер)

#### Property как контейнер

Когда ты пишешь:

```python
@property
def name(self):
    return self._name
```

Python делает примерно так:

```python
name = property(fget=name)  # создаётся объект property
```

`property` — это **контейнер (дескриптор)**. Внутри хранится ссылка на функцию-геттер (`fget`). Пока сеттера и делитера нет → их значения `None`.

Можно представить как коробку с тремя слотами:

```
+----------------------+
| fget  | fset  | fdel |
+----------------------+
   ↓        ↓       ↓
  name    None    None
```

#### `.setter()` — добавляем функцию

Когда пишешь:

```python
@name.setter
def name(self, value):
    self._name = value
```

Что реально происходит:

1. `name` — это существующий объект `property`
2. У него вызывается метод `setter(self, fset)`
3. Метод **не меняет старый объект**, а создаёт **новый объект** `property`, в котором:
   - `fget` = старый геттер
   - `fset` = новая функция `name(self, value)`
   - `fdel` = старый делитер (или `None`)
4. Этот новый объект снова записывается в атрибут `name` класса

То же самое с `.deleter()` — создаётся новая "коробка", куда копируются старые `fget` и `fset`, а в `fdel` кладётся новая функция.

#### Что происходит при обращении `obj.value`?

Python вызывает механизм дескрипторов:

```python
ClassName.__dict__['value'].__get__(obj, ClassName)
```

Объект `property` смотрит в свой слот `fget` и вызывает ту функцию, которую ты туда положил.

---

### Итоговая формула (золотой стандарт)

```python
class MyClass:
    def __init__(self, x):
        self._x = x

    @property
    def x(self):
        """Getter"""
        return self._x

    @x.setter
    def x(self, value):
        """Setter с логикой"""
        # валидация, логирование и т.д.
        self._x = value

    @x.deleter
    def x(self):
        """Deleter (редко нужен)"""
        del self._x
```

---

## 💻 Быстрая шпаргалка

**Главные мысли:**

1. **Имена:** геттер и сеттер должны называться **одинаково**
2. **Данные:** хранятся в `_атрибут`, доступ через `атрибут`
3. **Контейнер:** `@property` создаёт коробку с тремя слотами (`fget`, `fset`, `fdel`). Методы `.setter` / `.deleter` создают **новую коробку**, копируя старые слоты
4. **Механика:** это дескриптор, перехватывающий `obj.attr`, `obj.attr = val`, `del obj.attr`
5. **Плюсы:** валидация, read-only свойства, вычисляемые поля, обратная совместимость
