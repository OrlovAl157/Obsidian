# 🧬 type, object, классы и экземпляры

## 📌 Коротко

> `type` создаёт классы, `object` — скелет базовых свойств. Все классы наследуются от object и создаются через type. Всё в Python — объект.

---

## 📖 Теория

### Два прародителя всего

В Python есть два фундаментальных объекта из которых строится всё остальное:

- **`object`** — скелет с базовыми свойствами. Все классы наследуются от него.
- **`type`** — фабрика/завод. Создаёт классы.

Они созданы одновременно на уровне C-кода интерпретатора (CPython). Это хардкод — в обычной логике их взаимозависимость была бы парадоксом.

---

### Наследник vs Экземпляр

Это самое важное различие которое часто путают.

||Наследник|Экземпляр|
|---|---|---|
|Что означает|"расширяет поведение"|"создан из"|
|Связь через|`__bases__` / `issubclass()`|`__class__` / `isinstance()`|
|Отношение|класс → класс|класс → объект|
|Аналогия|новый чертёж на основе старого|дом построенный по чертежу|

```python
class Animal:
    pass

class Dog(Animal):  # Dog — наследник Animal
    pass

rex = Dog()         # rex — экземпляр Dog

issubclass(Dog, Animal)   # True — наследник
isinstance(rex, Dog)      # True — экземпляр
isinstance(rex, Animal)   # True — экземпляр (через всю цепочку наследования!)
```

---

### object — скелет базовых свойств

Когда пишешь `class Dog:` Python автоматически делает `class Dog(object):`.

От `object` все классы получают базовые методы:

```python
d = Dog()

d.__str__()    # откуда? из object
d.__repr__()   # откуда? из object
d.__eq__()     # откуда? из object
d.__hash__()   # откуда? из object
```

Ты их не писал — они пришли через наследование от `object`.

---

### type — фабрика классов

`type` создаёт классы. Когда пишешь `class Dog:` Python вызывает `type('Dog', (object,), {})`.

```python
# Эти две записи эквивалентны:

class Dog:
    def bark(self):
        return "Woof"

Dog = type('Dog', (object,), {'bark': lambda self: 'Woof'})
```

### Полная цепочка на примере `a = 5`

```
type  ──────────────────────► создаёт все классы
 │
 │ наследуется от
 ▼
object  ◄──────────────────── все классы наследуются от него
 │
 │ наследуется от
 ▼
int  ──── type(int) = type,  int.__bases__ = (object,)
 │
 │ экземпляр
 ▼
a = 5
```

```python
a = 5

type(a)               # <class 'int'>
type(int)             # <class 'type'>
type(object)          # <class 'type'>
type(type)            # <class 'type'>  ← сам себе метакласс!

int.__bases__         # (<class 'object'>,)
type.__bases__        # (<class 'object'>,)  ← type наследник object!

isinstance(a, int)    # True
isinstance(a, object) # True
isinstance(int, type) # True
isinstance(int, object) # True  ← класс тоже объект!
```

---

### Парадокс type и object

```python
# type — наследник object
issubclass(type, object)   # True

# type — экземпляр object (как любой класс)
isinstance(type, object)   # True

# type — экземпляр самого себя
isinstance(type, type)     # True
```

`type` находится в двух деревьях одновременно:

```
ДЕРЕВО НАСЛЕДОВАНИЯ          ДЕРЕВО ЭКЗЕМПЛЯРОВ
(issubclass / __bases__)     (type() / isinstance())

    object                        type
      │                          /    \
      │                    создаёт  создаёт
      ▼                        │       │
     type                     int   object
      │                        │
      ▼                    создаёт
     int                       │
                               ▼
                             a = 5
```

### Аналогии для запоминания

**Гончар и глина:**

- `type` — это гончар (создаёт)
- `object` — это глина (из чего всё сделано)
- Все классы — горшки, слеплены гончаром из глины
- Сам гончар тоже сделан из той же глины (`isinstance(type, object) = True`)

**Чертёж и дом:**

- Наследование — архитектор взял старый чертёж и нарисовал новый на его основе (чертёж → чертёж)
- Экземпляр — строители построили реальный дом по чертежу (чертёж → объект)

---

### Всё есть объект

В Python класс — это просто объект который умеет создавать другие объекты. Поэтому классы можно передавать в функции, хранить в списках и т.д.:

```python
# Классы можно передавать как обычные объекты
def create(cls):
    return cls()

create(Dog)   # работает
create(int)   # работает

# Классы можно хранить в списке
zoo = [Dog, int, str, list]
```

---

### Итог одной фразой

> **`type`** создаёт классы. **`object`** — скелет с базовыми свойствами на который type наслаивает методы и атрибуты, получая разные классы. От классов создаются экземпляры — привычные нам объекты. Всё в Python является объектом.
