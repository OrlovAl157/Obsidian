# 🏛️ Декорирование классов

## 💡 Что такое декорирование класса?

**Декорирование класса** — это применение декоратора к целому классу, а не к функции или методу.

```python
@decorator
class MyClass:
    pass

# Это то же самое, что:
class MyClass:
    pass
MyClass = decorator(MyClass)
```

**Главное отличие от функций:**
- При декорировании **функции** мы заменяем её другой функцией
- При декорировании **класса** мы модифицируем сам класс и возвращаем его (или возвращаем что-то другое)

---

## 🏗️ Базовый шаблон

```python
def class_decorator(cls):
    # Модифицируем класс
    # ...
    return cls  # Возвращаем модифицированный класс
```

**Важно:** Декоратор класса получает сам класс (не экземпляр) как аргумент.

---

## 📝 Пример 1: Добавить атрибут классу

```python
def add_attr(cls):
    cls.attr = None  # Добавляем атрибут к классу
    return cls

@add_attr
class MyClass:
    pass

print(MyClass.attr)  # → None ✅
```

---

## 📝 Пример 2: Модифицировать __init__

Когда нужно добавить **поведение** к экземплярам класса, нужно **изменить `__init__`**:

```python
import functools

def add_attr_to_instances(cls):
    old_init = cls.__init__  # Сохраняем исходный инициализатор
    
    @functools.wraps(old_init)
    def new_init(self, *args, **kwargs):
        old_init(self, *args, **kwargs)  # Вызываем исходный __init__
        self.attr = None  # Добавляем новый атрибут
    
    cls.__init__ = new_init  # Заменяем __init__ новым
    return cls

@add_attr_to_instances
class MyClass:
    pass

obj = MyClass()
print(obj.attr)  # → None ✅
```

**Как это работает:**

```
1. @add_attr_to_instances вызывает старый __init__
              ↓
2. Старый __init__ выполняет исходную логику
              ↓
3. Новый код добавляет атрибут экземпляру
              ↓
4. __init__ завершён, экземпляр готов
```

## 📝 Пример 3: Декоратор с параметрами

```python
import functools

def add_attr_to_instances(**attrs):
    def decorator(cls):
        old_init = cls.__init__
        
        @functools.wraps(old_init)
        def new_init(self, *args, **kwargs):
            old_init(self, *args, **kwargs)
            self.__dict__.update(attrs)  # Добавляем все атрибуты сразу
        
        cls.__init__ = new_init
        return cls
    
    return decorator

@add_attr_to_instances(first_attr=1, second_attr=2)
class MyClass:
    pass

obj = MyClass()
print(obj.first_attr)   # → 1 ✅
print(obj.second_attr)  # → 2 ✅
```

**Три уровня вложенности:**

```
add_attr_to_instances(**attrs)    ← принимает ПАРАМЕТРЫ
            ↓
decorator(cls)                    ← получает КЛАСС
            ↓
new_init(self, ...)               ← вызывается вместо __init__
```

## 📝 Пример 4: Подсчёт экземпляров

```python
import functools

def count_instances(cls):
    old_init = cls.__init__
    cls.count = 0  # Счётчик на уровне класса
    
    @functools.wraps(old_init)
    def new_init(self, *args, **kwargs):
        old_init(self, *args, **kwargs)
        cls.count += 1  # Увеличиваем счётчик
    
    cls.__init__ = new_init
    return cls

@count_instances
class MyClass:
    pass

print(MyClass.count)  # → 0

obj1 = MyClass()
obj2 = MyClass()
obj3 = MyClass()

print(MyClass.count)  # → 3 ✅
```

## 🔍 Разбор кода: limiter

### Что это?

Декоратор класса `limiter` — это **система кэширования экземпляров** с ограничением по количеству.

**Задача:**
- Ограничить количество экземпляров класса (лимит)
- Если экземпляр с таким ID уже существует — вернуть существующий
- Если лимит превышен — вернуть первый или последний экземпляр

---

### 📋 Сигнатура

```python
def limiter(limit, unique, lookup):
    def wrapper(cls):
        # ... модифицируем класс ...
        return ...
    return wrapper
```

**Параметры:**
- `limit` — максимальное количество экземпляров
- `unique` — имя атрибута, по которому определяем уникальность
- `lookup` — `'FIRST'` или другое значение (для выбора какой экземпляр возвращать при превышении лимита)

---

### 🔬 Полный код с комментариями

```python
def limiter(limit, unique, lookup):
    """
    Декоратор класса, ограничивающий количество экземпляров.
    
    Args:
        limit: максимальное кол-во экземпляров
        unique: имя атрибута для уникальной идентификации
        lookup: 'FIRST' (первый) или 'LAST' (последний) при превышении лимита
    """
    
    def wrapper(cls):
        # Добавляем атрибуты класса для управления кэшем
        cls._limit = limit                  # Ограничение по количеству
        cls._instances = {}                 # Словарь кэшированных экземпляров
        cls._lookup = lookup                # Какой экземпляр возвращать
        
        def wrapper2(*args, **kwargs):
            # Это будет ЗА МЕСТО конструктора класса
            
            # 1️⃣ Создаём НОВЫЙ объект (вызываем оригинальный __init__)
            obj = cls(*args, **kwargs)
            
            # 2️⃣ Получаем значение уникального атрибута
            unique_value = getattr(obj, unique)
            
            # 3️⃣ Проверяем, есть ли уже экземпляр с таким ID
            if unique_value in cls._instances:
                # Экземпляр уже есть — возвращаем его (старый)
                return cls._instances[unique_value]
            
            # 4️⃣ Или проверяем лимит
            elif len(cls._instances) < cls._limit:
                # Место есть — добавляем новый экземпляр в кэш
                cls._instances[unique_value] = obj
                return obj
            
            # 5️⃣ Лимит превышен
            else:
                # lookup='FIRST' → первый (индекс 0)
                # lookup='LAST' (или другое) → последний (индекс -1)
                key_to_return = list(cls._instances)[0 if cls._lookup == 'FIRST' else -1]
                return cls._instances[key_to_return]
        
        # Заменяем конструктор класса на wrapper2
        return wrapper2
    
    return wrapper
```

class MyClass:

        ↓

        ↓

        ↓

        ↓
    return cls._instances[1] ← возвращаем obj1
    
    Новый obj с ID=3 и value=0 ИГНОРИРУЕТСЯ!
    obj4 это ТОЖЕ obj1
    obj4.value = 5  ← NOT 0!

───────────────────────────────────────────

print(obj4.value)  # → 5 (это obj1.value)

### 🎯 Практический пример в действии

```python
def limiter(limit, unique, lookup):
    def wrapper(cls):
        cls._limit = limit
        cls._instances = {}
        cls._lookup = lookup
        
        def wrapper2(*args, **kwargs):
            obj = cls(*args, **kwargs)
            
            if getattr(obj, unique) in cls._instances:
                return cls._instances[getattr(obj, unique)]
            elif len(cls._instances) < cls._limit:
                cls._instances[getattr(obj, unique)] = obj
                return obj
            else:
                return cls._instances[list(cls._instances)[0 if cls._lookup == 'FIRST' else -1]]
        
        return wrapper2
    return wrapper


@limiter(2, 'ID', 'FIRST')
class MyClass:
    def __init__(self, ID, value):
        self.ID = ID
        self.value = value
        print(f"Создан экземпляр: ID={ID}, value={value}")


print("=" * 50)
obj1 = MyClass(1, 5)
print(f"obj1: ID={obj1.ID}, value={obj1.value}")
print(f"_instances: {list(MyClass._instances.keys())}")

print("=" * 50)
obj2 = MyClass(2, 8)
print(f"obj2: ID={obj2.ID}, value={obj2.value}")
print(f"_instances: {list(MyClass._instances.keys())}")

print("=" * 50)
obj3 = MyClass(1, 20)
print(f"obj3: ID={obj3.ID}, value={obj3.value}")
print(f"obj3 это obj1? {obj3 is obj1}")
print(f"_instances: {list(MyClass._instances.keys())}")

print("=" * 50)
obj4 = MyClass(3, 0)
print(f"obj4: ID={obj4.ID}, value={obj4.value}")
print(f"obj4 это obj1? {obj4 is obj1}")
print(f"_instances: {list(MyClass._instances.keys())}")
```

**Вывод:**
```
==================================================
Создан экземпляр: ID=1, value=5
obj1: ID=1, value=5
_instances: [1]
==================================================
Создан экземпляр: ID=2, value=8
obj2: ID=2, value=8
_instances: [1, 2]
==================================================
Создан экземпляр: ID=1, value=20
obj3: ID=1, value=5
obj3 это obj1? True
_instances: [1, 2]
==================================================
Создан экземпляр: ID=3, value=0
obj4: ID=1, value=5
obj4 это obj1? True
_instances: [1, 2]
```

**Важные моменты:**
- При `MyClass(1, 20)` создаётся **новый объект**, но он **игнорируется** (если ID уже существует)
- При `MyClass(3, 0)` тоже создаётся новый объект, но **также игнорируется**
- `obj3` и `obj4` это **один и тот же объект** что и `obj1`
- `print(obj3.value)` выводит `5`, а не `20`
- `print(obj4.value)` выводит `5`, а не `0`

### 🔑 Ключевой момент: wrapper2 ЗАМЕНЯЕТ конструктор

```python
@limiter(2, 'ID', 'FIRST')
class MyClass:
    def __init__(self, ...):
        ...

# После декорирования MyClass теперь ссылается на wrapper2
# Оригинальный конструктор класса НЕДОСТУПЕН напрямую
# Вместо MyClass(...) теперь вызывается wrapper2(...)
```

## 🎨 Вариант с lookup='LAST'

Если изменить `lookup` на `'LAST'`, то при превышении лимита будет возвращаться **последний** добавленный экземпляр:

```python
@limiter(2, 'ID', 'LAST')
class MyClass:
    def __init__(self, ID, value):
        self.ID = ID
        self.value = value

obj1 = MyClass(1, 5)   # _instances = {1: obj1}
obj2 = MyClass(2, 8)   # _instances = {1: obj1, 2: obj2}
obj3 = MyClass(3, 0)   # Лимит превышен → return last
                       # last = list({1: obj1, 2: obj2})[-1] = 2
                       # return obj2

print(obj3 is obj2)    # → True
print(obj3.value)      # → 8 (NOT 0!)
```

## ⚠️ Практические применения limiter

### 1. Синглтон (только 1 экземпляр)

```python
@limiter(1, 'id', 'FIRST')
class Database:
    def __init__(self, id):
        self.id = id
        print("Подключение к БД...")

db1 = Database(1)
db2 = Database(2)  # Вернёт db1

print(db1 is db2)  # → True (один объект!)
```

### 2. Пул соединений (N экземпляров)

```python
@limiter(10, 'connection_id', 'FIRST')
class Connection:
    def __init__(self, connection_id):
        self.connection_id = connection_id
        print(f"Создано соединение {connection_id}")

# Можно создать максимум 10 уникальных соединений
# Остальные запросы будут переиспользовать существующие
```

## ⚠️ Частые ошибки

### 1. Забыть вернуть wrapper2

```python
# ❌ НЕПРАВИЛЬНО
def limiter(limit, unique, lookup):
    def wrapper(cls):
        # ... код ...
        # Забыли return wrapper2!
    return wrapper
```

### 2. Использовать wrapper2 как обычный метод

```python
# ❌ НЕПРАВИЛЬНО (wrapper2 не знает о self)
# wrapper2 это замена конструктора, не метода!
```

### 3. Не сохранять параметры в cls

```python
# ❌ НЕПРАВИЛЬНО
def wrapper(cls):
    limit = ...  # Локальная переменная, потеряется!
    
    def wrapper2(*args, **kwargs):
        if len(???) < limit:  # Откуда limit?
            ...
```

### 4. Не использовать getattr для получения unique

```python
# ❌ НЕПРАВИЛЬНО
unique_value = obj.unique  # Не работает, ошибка если атрибута нет

# ✅ ПРАВИЛЬНО
unique_value = getattr(obj, unique)  # Безопасно
```

## 💭 Итоговая шпаргалка

### Шаблон декоратора класса (простой)

```python
def class_decorator(cls):
    # Модифицируем класс
    cls.new_attr = "value"
    return cls

@class_decorator
class MyClass:
    pass
```

### Шаблон для модификации __init__

```python
import functools

def modify_init(cls):
    old_init = cls.__init__
    
    @functools.wraps(old_init)
    def new_init(self, *args, **kwargs):
        old_init(self, *args, **kwargs)
        # Дополнительный код
    
    cls.__init__ = new_init
    return cls

@modify_init
class MyClass:
    pass
```

### Шаблон с параметрами

```python
def decorator_with_params(param):
    def wrapper(cls):
        cls.param = param
        return cls
    return wrapper

@decorator_with_params("value")
class MyClass:
    pass
```

### Шаблон для замены конструктора (как limiter)

```python
def replace_constructor(cls):
    def new_constructor(*args, **kwargs):
        # Логика вместо __init__
        obj = cls(*args, **kwargs)
        # Постобработка
        return obj
    
    return new_constructor

@replace_constructor
class MyClass:
    pass
```

## ✅ Ключевые правила

✅ Декоратор класса получает **сам класс**, не экземпляр  
✅ Обычно модифицируешь класс и **возвращаешь его же**  
✅ Для изменения поведения экземпляров — модифицируй `__init__`  
✅ При замене `__init__` используй `@functools.wraps(old_init)`  
✅ Используй `setattr/delattr/getattr` для работы с атрибутами класса  
✅ Параметры декоратора сохраняй в `cls.__dict__` (как атрибуты класса)  
✅ `list(cls.__dict__)` нужен при удалении элементов во время итерации
