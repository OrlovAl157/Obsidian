Вот **главные признаки ООП в коде**:

---

### 🔹 1. Используются классы

Если ты создаёшь свой класс через `class`, это уже признак ООП.

`class Car:     pass`

---

### 🔹 2. Создаются объекты (экземпляры классов)

Когда ты используешь класс для создания объектов:

`my_car = Car()`

— значит, ты работаешь с объектами, а не просто с функциями или числами.

---

### 🔹 3. Методы и атрибуты

Если функции связаны с объектами и работают с их внутренними данными:

`class Car:     def __init__(self, color):         self.color = color          def drive(self):         print(f"{self.color} is driving")  my_car.drive()`

- `self.color` — атрибут
    
- `drive()` — метод объекта
    

---

### 🔹 4. Используется наследование

Когда один класс **расширяет или переопределяет** другой:

`class Vehicle:     def move(self):         print("Moving")  class Car(Vehicle):     pass`

---

### 🔹 5. Полиморфизм и инкапсуляция

- **Полиморфизм** — одинаковые методы для разных объектов:
    

`class Cat:     def speak(self): print("Meow") class Dog:     def speak(self): print("Woof")  for animal in [Cat(), Dog()]:     animal.speak()  # разный результат, один метод`

- **Инкапсуляция** — данные скрыты внутри объекта:
    

`class Car:     def __init__(self):         self.__speed = 0  # приватный атрибут`

---

### 🔹 6. Основной признак

Если твой код **оперирует объектами, создаёт их, вызывает их методы и использует их свойства**,  
а не просто функции и глобальные переменные — это **ООП-код**.

---

💡 **Итог:**

> В ООП стиль определяют **классы, объекты, методы, атрибуты и взаимодействие между ними**, а не то, что ты используешь Python. Даже в Python процедурный код всегда остаётся объектным под капотом, но по стилю — это **не ООП**, пока нет этих признаков.

### 🔹 Разбор

1. **Классы и объекты** — главный признак.
    
    - Если у тебя есть хотя бы **один класс и объекты**, это уже ООП-код.
        
2. **Методы и атрибуты**
    
    - Желательно использовать методы, а не только атрибуты, но если код очень простой, можно ограничиться только классом с атрибутами.
        
3. **Наследование, полиморфизм, инкапсуляция**
    
    - Это **дополнительные признаки**, которые делают код «чистым» ООП, но они **не обязательны** для того, чтобы код считался объектно-ориентированным.
        

---

### 🔹 Примеры

✅ **Простой ООП-код** — только класс и объект:

`class Car:     pass  my_car = Car()`

→ Уже ООП, хотя нет методов, наследования, полиморфизма.

✅ **Более «классический» ООП-код**:

`class Car:     def __init__(self, color):         self.color = color     def drive(self):         print(f"{self.color} car is driving")  my_car = Car("red") my_car.drive()`

→ Здесь и методы, и атрибуты — стиль более явно объектно-ориентированный.

---

💡 **Итог:**

> Достаточно одного признака (класс + объекты) — все остальные признаки — **дополнительно**, чтобы код был «чистым» ООП.
