## 2. ПОЛНАЯ ИЕРАРХИЯ (КРУПНЫМ ПЛАНОМ)

```
                        LOGGING
                           │
              ┌────────────┼────────────┐
              │            │            │
           LOGGER      FILTER      HANDLER
              │            │            │
              │            │      ┌─────┴─────┐
              │            │      │           │
              │            │  FORMATTER   LEVEL
              │            │
              └────────────┘
```

### Что к чему относится:

| Компонент | Относится к | Создается | Включает |
|-----------|-------------|-----------|----------|
| **Logger** | - | `logging.getLogger(name)` | Уровень, Фильтры, Обработчики |
| **Filter** | Logger или Handler | `logging.Filter(name)` | Условия фильтрации |
| **Handler** | Logger | `logging.StreamHandler()`<br>`logging.FileHandler()`<br>`logging.SMTPHandler()` и др. | Уровень, Фильтры, Форматтер |
| **Formatter** | Handler | `logging.Formatter(format)` | Шаблон вывода |

## 3. ПОШАГОВОЕ СОЗДАНИЕ (ОТ ПРОСТОГО К СЛОЖНОМУ)

### 🟢 Уровень 1: Базовый (только логгер)

```python
import logging

logger = logging.getLogger('my_app')
logger.warning('Простое предупреждение')
# Вывод: warning
```

### 🟡 Уровень 2: С хендлером

```python
import logging
import sys

logger = logging.getLogger('my_app')
handler = logging.StreamHandler(sys.stdout)  # Куда?
logger.addHandler(handler)                   # Привязываем

logger.info('Информация')  # Уже куда-то выведется
```

### 🟠 Уровень 3: С форматтером

```python
import logging
import sys

logger = logging.getLogger('my_app')
handler = logging.StreamHandler(sys.stdout)

# Как форматировать?
formatter = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
handler.setFormatter(formatter)  # Привязываем форматтер к хендлеру

logger.addHandler(handler)
logger.info('Информация')
# Вывод: 2026-07-22 10:00:00 - my_app - INFO - Информация
```

### 🔴 Уровень 4: С фильтром

```python
import logging
import sys

# 1. Создаем фильтр (может быть классом или функцией)
class MyFilter(logging.Filter):
    def filter(self, record):
        return 'error' in record.getMessage().lower()

logger = logging.getLogger('my_app')
handler = logging.StreamHandler(sys.stdout)
formatter = logging.Formatter('%(levelname)s - %(message)s')
handler.setFormatter(formatter)

# 2. Добавляем фильтр к логгеру ИЛИ хендлеру
logger.addFilter(MyFilter())  # Фильтрует все сообщения
# ИЛИ handler.addFilter(MyFilter())  # Фильтрует только для этого хендлера

logger.addHandler(handler)

logger.info('Это обычное сообщение')  # ❌ Не пройдет фильтр
logger.error('Это сообщение с error')  # ✅ Пройдет фильтр
```

### 🟣 Уровень 5: Полная конфигурация

```python
import logging
import sys

# ========== 1. СОЗДАЕМ ЛОГГЕР ==========
logger = logging.getLogger('my_app')
logger.setLevel(logging.DEBUG)  # Уровень логгера

# ========== 2. СОЗДАЕМ ХЕНДЛЕР ==========
# Хендлеры бывают разные:
console_handler = logging.StreamHandler(sys.stdout)  # В консоль
file_handler = logging.FileHandler('app.log')        # В файл

console_handler.setLevel(logging.INFO)   # Уровень хендлера
file_handler.setLevel(logging.DEBUG)

# ========== 3. СОЗДАЕМ ФОРМАТТЕРЫ ==========
console_formatter = logging.Formatter(
    '%(asctime)s - %(levelname)s - %(message)s'
)
file_formatter = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

# ========== 4. ПРИВЯЗЫВАЕМ ФОРМАТТЕР К ХЕНДЛЕРУ ==========
console_handler.setFormatter(console_formatter)
file_handler.setFormatter(file_formatter)

# ========== 5. ПРИВЯЗЫВАЕМ ХЕНДЛЕР К ЛОГГЕРУ ==========
logger.addHandler(console_handler)
logger.addHandler(file_handler)

# ========== 6. (ОПЦИОНАЛЬНО) ДОБАВЛЯЕМ ФИЛЬТР ==========
class ErrorOnlyFilter(logging.Filter):
    def filter(self, record):
        return record.levelno >= logging.ERROR

file_handler.addFilter(ErrorOnlyFilter())  # В файл только ошибки

# ========== 7. ИСПОЛЬЗУЕМ ==========
logger.debug('Отладка')    # Только в файл (DEBUG < INFO, но файл имеет DEBUG)
logger.info('Информация')  # В консоль и файл
logger.error('Ошибка')     # В консоль, файл (фильтр пропускает)
```

## 4. ИЕРАРХИЯ ЛОГГЕРОВ (ПО ИМЕНАМ)

```python
import logging

# Логгеры создают иерархию через точки в именах
root = logging.getLogger()           # Корневой логгер
app = logging.getLogger('app')        # Дочерний root
api = logging.getLogger('app.api')    # Дочерний app
db = logging.getLogger('app.db')      # Дочерний app
auth = logging.getLogger('app.api.auth')  # Дочерний api

# Иерархия:
# root
#   └── app
#        ├── api
#        │    └── auth
#        └── db
```

### Правила распространения:

```python
# Установка уровня для app повлияет на всех потомков
logging.getLogger('app').setLevel(logging.DEBUG)

# Теперь все дочерние логгеры (api, db, auth) наследуют DEBUG
# если у них самих не установлен другой уровень

# propagation (распространение):
# - Сообщение от auth → сначала проверяется auth
# - Потом api (если auth не обработал)
# - Потом app
# - Потом root
```

## 5. КОГДА ЧТО ИСПОЛЬЗОВАТЬ?

| Компонент | Когда использовать |
|-----------|-------------------|
| **Logger** | Всегда. Главная точка входа для логирования |
| **Handler** | Для разных назначений: консоль, файл, email, syslog, HTTP и т.д. |
| **Formatter** | Когда нужен разный формат для разных выводов (например, для консоли кратко, для файла подробно) |
| **Filter** | Для сложной фильтрации (по содержимому, по атрибутам, по контексту) |

## 6. БЫСТРЫЙ СТАРТ (ШАБЛОН)

```python
import logging
import sys

# Настройка (делается один раз в начале приложения)
def setup_logging():
    logger = logging.getLogger('my_app')
    logger.setLevel(logging.DEBUG)
    
    # Консоль
    console = logging.StreamHandler(sys.stdout)
    console.setLevel(logging.INFO)
    console.setFormatter(logging.Formatter(
        '%(asctime)s - %(levelname)s - %(message)s'
    ))
    logger.addHandler(console)
    
    # Файл
    file_handler = logging.FileHandler('app.log')
    file_handler.setLevel(logging.DEBUG)
    file_handler.setFormatter(logging.Formatter(
        '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    ))
    logger.addHandler(file_handler)
    
    return logger

# Использование в коде
logger = setup_logging()
logger.info('Приложение запущено')
logger.debug('Детальная отладка')
logger.error('Произошла ошибка', exc_info=True)
```

## 7. ЗОЛОТЫЕ ПРАВИЛА

1. **Logger** - один на модуль (`logger = logging.getLogger(__name__)`)
2. **Handler** - несколько для разных целей (консоль + файл)
3. **Formatter** - один для каждого Handler (или общий)
4. **Filter** - опционально, для сложной логики
5. **Уровни**: Logger > Handler (оба проверяются!)
6. **Propagation**: включен по умолчанию (сообщения идут к родителям)
