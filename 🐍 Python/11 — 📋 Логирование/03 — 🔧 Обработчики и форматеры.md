| Класс | Назначение | Использование |
|-------|-----------|----------------|

## 🎯 Что такое обработчик

**Обработчик (Handler)** — это то, как логирование выводится (куда и в каком формате).


### Архитектура логирования

```
Logger
  ↓
Event создан
  ↓
Фильтры логгера (если есть)
  ↓
↙          ↓          ↘
Handler 1   Handler 2   Handler 3
(файл)     (консоль)   (email)
```

**Каждый логгер может иметь несколько обработчиков!**

### Как это работает?

```python
import logging

# Логгер само по себе ничего не выводит!
logger = logging.getLogger(__name__)

# Нужен обработчик, чтобы что-то произошло
handler = logging.StreamHandler()  # создаём обработчик
logger.addHandler(handler)          # добавляем его в логгер

logger.warning('Событие')  # теперь выводится!
```

### По умолчанию при basicConfig()

```python
import logging

logging.basicConfig(level=logging.INFO)
root = logging.getLogger()

print(root.handlers)
```

Вывод:
```
[<StreamHandler <stderr> (NOTSET)>]
```

**`basicConfig()` автоматически создаёт StreamHandler!**

## 📤 StreamHandler — вывод в консоль

### Что это?

`StreamHandler` выводит логи в поток (stream). По умолчанию это `sys.stderr` (поток ошибок).

### Создание

```python
import logging

handler = logging.StreamHandler()
```

### Вывод в sys.stdout (вместо stderr)

```python
import logging
import sys

handler = logging.StreamHandler(stream=sys.stdout)
logger = logging.getLogger(__name__)
logger.addHandler(handler)

logger.warning('Это на stdout, не stderr')
```

### Полный пример

```python
import logging
import sys

# Создаём обработчик для консоли
handler = logging.StreamHandler(stream=sys.stdout)

# Создаём логгер
logger = logging.getLogger(__name__)

# Добавляем обработчик
logger.addHandler(handler)

# Логируем
logger.warning('Событие на консоль')
logger.error('Ошибка на консоль')
```

Вывод:
```
Событие на консоль
Ошибка на консоль
```

**Проблема:** нет форматирования! Нужно добавить Formatter.

## 📁 FileHandler — запись в файл

### Что это?

`FileHandler` записывает логи в файл.

### Создание

```python
import logging

# Простой способ
handler = logging.FileHandler('logs.log', encoding='utf-8')
```

### С параметром mode

```python
import logging

# mode='w' — перезаписать файл (по умолчанию mode='a' — добавить)
handler = logging.FileHandler('logs.log', mode='w', encoding='utf-8')
```

### Полный пример

```python
import logging

# Обработчик для файла
handler = logging.FileHandler('app.log', encoding='utf-8')

# Логгер
logger = logging.getLogger(__name__)
logger.addHandler(handler)

# Логируем
logger.warning('В файл')
logger.error('Тоже в файл')
```

Создаёт файл `app.log`:
```
В файл
Тоже в файл
```

**Проблема:** тоже нет форматирования!

## 🎨 Форматер — оформление

### Что это?

`Formatter` определяет, как будет выглядеть сообщение в логах. Устанавливается на каждый обработчик.

### Создание форматера

```python
import logging

# Создаём форматер
fmt = '%(asctime)s - %(levelname)s - %(message)s'
formatter = logging.Formatter(fmt)
```

### С datefmt

```python
import logging

fmt = '%(asctime)s - %(levelname)s - %(message)s'
datefmt = '%d.%m.%Y %H:%M:%S'
formatter = logging.Formatter(fmt, datefmt=datefmt)
```

### Со стилем {}

```python
import logging

fmt = '{asctime} - {levelname} - {message}'
formatter = logging.Formatter(fmt, style='{', datefmt='%H:%M:%S')
```

## ⚙️ Установка форматера

### На обработчик

```python
import logging

# 1. Создаём обработчик
handler = logging.StreamHandler()

# 2. Создаём форматер
fmt = '%(asctime)s | %(levelname)s | %(message)s'
formatter = logging.Formatter(fmt)

# 3. Устанавливаем форматер на обработчик
handler.setFormatter(formatter)

# 4. Добавляем обработчик в логгер
logger = logging.getLogger(__name__)
logger.addHandler(handler)

# 5. Используем
logger.info('Событие')
```

Вывод:
```
2025-04-25 14:39:03,568 | INFO | Событие
```

### Пример: консоль и файл с разными форматами

```python
import logging
import sys

# КОНСОЛЬ
console_handler = logging.StreamHandler(sys.stdout)
console_fmt = '[{levelname}] {message}'
console_formatter = logging.Formatter(console_fmt, style='{')
console_handler.setFormatter(console_formatter)

# ФАЙЛ
file_handler = logging.FileHandler('app.log', encoding='utf-8')
file_fmt = '{asctime} | {levelname:8} | {name} | {message}'
file_formatter = logging.Formatter(file_fmt, style='{', datefmt='%H:%M:%S')
file_handler.setFormatter(file_formatter)

# ЛОГГЕР
logger = logging.getLogger(__name__)
logger.addHandler(console_handler)
logger.addHandler(file_handler)

# ИСПОЛЬЗУЕМ
logger.info('Тестовое событие')
logger.error('Ошибка')
```

**На консоль:**
```
[INFO] Тестовое событие
[ERROR] Ошибка
```

**В файле `app.log`:**
```
14:39:03 | INFO     | __main__ | Тестовое событие
14:39:03 | ERROR    | __main__ | Ошибка
```

## 🔀 Несколько обработчиков

### Один логгер → несколько обработчиков

```python
import logging
import sys

logger = logging.getLogger(__name__)

# Обработчик 1: консоль (INFO и выше)
console = logging.StreamHandler(sys.stdout)
console.setLevel(logging.INFO)
console_fmt = logging.Formatter('[{levelname}] {message}', style='{')
console.setFormatter(console_fmt)

# Обработчик 2: файл (WARNING и выше)
file = logging.FileHandler('errors.log', encoding='utf-8')
file.setLevel(logging.WARNING)
file_fmt = logging.Formatter('{asctime} | {levelname} | {message}', style='{')
file.setFormatter(file_fmt)

# Добавляем оба
logger.addHandler(console)
logger.addHandler(file)
logger.setLevel(logging.DEBUG)  # логгер на DEBUG

# Логируем
logger.debug('DEBUG')       # нигде не выводится (обработчики выше)
logger.info('INFO')         # ✅ только в консоль
logger.warning('WARNING')   # ✅ консоль И файл
logger.error('ERROR')       # ✅ консоль И файл
```

**На консоль:**
```
[INFO] INFO
[WARNING] WARNING
[ERROR] ERROR
```

**В файле (errors.log):**
```
14:39:03 | WARNING | WARNING
14:39:03 | ERROR | ERROR
```

## 🏗️ Правильная архитектура логирования

### Маленький проект (с basicConfig)

```python
# main.py
import logging

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    datefmt='%d.%m.%Y %H:%M:%S'
)

logger = logging.getLogger(__name__)
logger.info('Приложение запущено')
```

### Большой проект (с обработчиками)

```python
# config/logging.py
import logging
import sys

def setup_logging():
    # КОНСОЛЬ
    console_handler = logging.StreamHandler(sys.stdout)
    console_handler.setLevel(logging.INFO)
    console_fmt = logging.Formatter(
        '[{levelname}] {name} - {message}',
        style='{'
    )
    console_handler.setFormatter(console_fmt)
    
    # ФАЙЛ
    file_handler = logging.FileHandler('logs/app.log', encoding='utf-8')
    file_handler.setLevel(logging.DEBUG)
    file_fmt = logging.Formatter(
        '{asctime} | {levelname:8} | {name:15} | {message}',
        style='{',
        datefmt='%d.%m.%Y %H:%M:%S'
    )
    file_handler.setFormatter(file_fmt)
    
    # КОРНЕВОЙ ЛОГГЕР
    root_logger = logging.getLogger()
    root_logger.setLevel(logging.DEBUG)
    root_logger.addHandler(console_handler)
    root_logger.addHandler(file_handler)

if __name__ == '__main__':
    setup_logging()
```

```python
# main.py
from config.logging import setup_logging
import logging

setup_logging()
logger = logging.getLogger(__name__)

logger.info('Приложение запущено')
logger.debug('Отладочная информация')
```

## 📝 Примечания

### Примечание 1: закрытие обработчиков

После завершения логирования нужно закрыть обработчики (освобождают файлы и ресурсы):

```python
import logging

handler = logging.FileHandler('app.log')
logger = logging.getLogger(__name__)
logger.addHandler(handler)

# ... использование ...

# Способ 1: закрыть каждый
handler.close()

# Способ 2: закрыть все (лучше)
logging.shutdown()
```

**Обычно НЕ нужно делать вручную** — Python автоматически закроет при выходе!

### Примечание 2: опять про propagate

Логгер может использовать обработчики родителя (по умолчанию):

```python
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger('my_app.module')

handler = logging.FileHandler('module.log')
logger.addHandler(handler)

logger.info('Событие')
```

Событие логируется:
- ✅ В `module.log` (обработчик module)
- ✅ На экран (обработчик root)

Чтобы отключить наследование:

```python
logger.propagate = False  # только свои обработчики
```

### Примечание 3: уровень обработчика

Обработчик может иметь свой уровень (независимо от логгера):

```python
import logging
import sys

logger = logging.getLogger(__name__)
logger.setLevel(logging.DEBUG)

# Консоль: только ERROR и выше
console = logging.StreamHandler(sys.stdout)
console.setLevel(logging.ERROR)
logger.addHandler(console)

logger.debug('DEBUG')       # не выведется (уровень консоли ERROR)
logger.info('INFO')         # не выведется
logger.error('ERROR')       # ✅ выведется
```

### Примечание 4: параметр defaults в Formatter

Если в extra некоторые ключи отсутствуют, используй `defaults`:

```python
import logging
import sys

handler = logging.StreamHandler(sys.stdout)
fmt = '{user_id} | {user_name} | {message}'
formatter = logging.Formatter(
    fmt,
    style='{',
    defaults={'user_id': 0, 'user_name': 'unknown'}  # значения по умолчанию
)
handler.setFormatter(formatter)

logger = logging.getLogger(__name__)
logger.addHandler(handler)

# В extra только user_id
logger.info('Событие', extra={'user_id': 123})
# Вывод: 123 | unknown | Событие
```

### Примечание 5: удаление обработчика

```python
import logging

handler = logging.StreamHandler()
logger = logging.getLogger(__name__)

logger.addHandler(handler)
print(logger.handlers)  # [handler]

logger.removeHandler(handler)
print(logger.handlers)  # []
```

### Примечание 6: другие типы обработчиков

```python
# SMTPHandler — отправка по email
handler = logging.handlers.SMTPHandler(
    mailhost='smtp.gmail.com',
    fromaddr='alert@example.com',
    toaddrs=['admin@example.com'],
    subject='Error Alert'
)

# RotatingFileHandler — файл с ограничением размера
handler = logging.handlers.RotatingFileHandler(
    'app.log',
    maxBytes=1000000,  # 1MB
    backupCount=5      # хранить 5 файлов
)

# SysLogHandler — в системный журнал
handler = logging.handlers.SysLogHandler('/dev/log')
```

## ✅ Главные правила

✅ Handler — это куда выводится логирование  
✅ Formatter — это как выглядит сообщение  
✅ На каждый handler — один Formatter  
✅ На один logger — несколько handlers  
✅ StreamHandler — в консоль  
✅ FileHandler — в файл  
✅ **Всегда устанавливай Formatter на Handler через setFormatter()**  
✅ basicConfig() автоматически создаёт StreamHandler  
✅ Для больших проектов — вручную создавай handlers  
✅ Handler может иметь свой уровень логирования  
✅ Используй defaults в Formatter для опциональных полей
