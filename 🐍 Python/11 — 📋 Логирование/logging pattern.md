# Паттерн создания и настройки Handlers в Python Logging

```python
import logging

# ШАГ 1: Создаю логгер
# logging.getLogger('myapp') — возвращает логгер с именем 'myapp'
# Если повторно вызвать с тем же именем — вернёт существующий логгер
logger = logging.getLogger('myapp')
logger.setLevel(logging.DEBUG)  # ПЕРВЫЙ фильтр — логгер пропускает сообщения от DEBUG и выше

# ШАГ 2: Создаю handlers (куда отправлять)
# StreamHandler() — пишет в консоль (stdout), сообщения видны в реальном времени
console_handler = logging.StreamHandler()

# FileHandler('app.log') — пишет в файл, если файла нет — создаёт, если есть — дописывает
file_handler = logging.FileHandler('app.log')

# ШАГ 3: Настраиваю УРОВЕНЬ для каждого handler (ВТОРОЙ фильтр)
# Консоль: только WARNING и выше (WARNING, ERROR, CRITICAL) — чтобы не шумела
console_handler.setLevel(logging.WARNING)

# Файл: всё от DEBUG и выше — полная история для анализа
file_handler.setLevel(logging.DEBUG)

# ШАГ 4: Создаю форматтер
# %(asctime)s — дата/время, %(name)s — имя логгера, %(levelname)s — уровень, %(message)s — сообщение
formatter = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

# ШАГ 5: Применяю форматтер к handlers
console_handler.setFormatter(formatter)
file_handler.setFormatter(formatter)

# ШАГ 6: Добавляю handlers к логгеру
logger.addHandler(console_handler)
logger.addHandler(file_handler)

# ==================== РЕЗУЛЬТАТЫ ====================
# Каждое сообщение проходит через цепочку фильтров:
# Логгер → Консоль (свой фильтр) → Файл (свой фильтр)

logger.debug("Отладочное сообщение")      # Логгер: ✓ | Консоль: ✗ | Файл: ✓ → только в файл
logger.info("Информационное сообщение")   # Логгер: ✓ | Консоль: ✗ | Файл: ✓ → только в файл
logger.warning("Предупреждение!")         # Логгер: ✓ | Консоль: ✓ | Файл: ✓ → в оба места
logger.error("Критическая ошибка")        # Логгер: ✓ | Консоль: ✓ | Файл: ✓ → в оба места
logger.critical("Система падает!")        # Логгер: ✓ | Консоль: ✓ | Файл: ✓ → в оба места

# Проверка конфигурации
print(f"\nНазвание логгера: {logger.name}")
print(f"Уровень логгера: {logger.level} (DEBUG=10)")
print(f"Handlers: {logger.handlers}")
print(f"Сообщения пишутся в файл: app.log")
```

## Объяснение каждого шага

### ШАГ 1: Создание логгера
- `logging.getLogger('myapp')` создаёт или возвращает существующий логгер
- `setLevel(DEBUG)` устанавливает минимальный уровень сообщений, которые логгер пропускает дальше
- Это **первый фильтр** — если логгер отклонит сообщение, handlers его не увидят

### ШАГ 2: Создание handlers
- **StreamHandler()** — пишет в консоль, для реального времени
- **FileHandler('app.log')** — пишет в файл, для долгосрочного хранения

### ШАГ 3: Уровень для каждого handler
- **Консоль**: WARNING и выше — только важное, чтобы не шумела
- **Файл**: DEBUG и выше — полная информация для анализа
- Это **второй независимый фильтр** для каждого handler

### ШАГ 4-5: Форматтер
Определяет как выглядит сообщение:
- `%(asctime)s` — дата и время
- `%(name)s` — имя логгера
- `%(levelname)s` — уровень (DEBUG, INFO, WARNING и т.д.)
- `%(message)s` — твоё сообщение

### ШАГ 6: Добавление handlers
- `logger.addHandler()` связывает handler с логгером
- Один логгер может иметь много handlers
- Сообщение пойдёт всем handlers одновременно

## Таблица фильтрации

| Вызов | Логгер | Консоль | Файл | Результат |
|-------|--------|---------|------|-----------|
| `logger.debug()` | ✓ | ✗ | ✓ | Только в файл |
| `logger.info()` | ✓ | ✗ | ✓ | Только в файл |
| `logger.warning()` | ✓ | ✓ | ✓ | В оба места |
| `logger.error()` | ✓ | ✓ | ✓ | В оба места |
| `logger.critical()` | ✓ | ✓ | ✓ | В оба места |

## Ключевые моменты

1. **Два независимых фильтра**: логгер и каждый handler имеют свой уровень
2. **Один к одному**: один логгер может иметь много handlers, один handler может быть у многих логгеров
3. **Форматтер**: решает как выглядит сообщение, применяется для каждого handler'а
4. **Propagate**: по умолчанию сообщения идут и родительским логгерам (если они есть)
