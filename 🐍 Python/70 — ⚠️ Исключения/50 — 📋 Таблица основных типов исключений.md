# ⚖️ Оператор assert в Python

## 📌 Коротко

> **assert** — оператор для проверки утверждений во время разработки. Используется для отладки и проверки внутренних инвариантов. Важно понимать, что assert — это инструмент отладки, а не обработки ошибок, и его можно отключить флагом `-O`.

---

## 💡 Полезные советы

- **Assert — только для отладки**, не для обработки ошибок пользователя
- **Код должен работать без assert** — они отключаются флагом `-O`
- **Используйте для внутренних проверок** — инварианты, предусловия
- **Не используйте для валидации пользовательского ввода** — используйте исключения
- **Избегайте побочных эффектов** — assert не должен изменять состояние
- **Скобки не нужны** — `assert condition` а не `assert(condition, "msg")`

---

## 🚀 Быстрая справка

| Конструкция | Синтаксис | Результат |
|---|---|---|
| **Базовый** | `assert условие` | Ошибка если условие ложно |
| **С сообщением** | `assert условие, "текст"` | Ошибка с описанием |
| **Отключение** | `python -O script.py` | Все assert игнорируются |
| **Эквивалент** | `if not condition: raise AssertionError("msg")` | То же самое |

# Памятка по оператору assert в Python

## Пример для быстрой справки

```python
def divide(a, b):
    assert b != 0, "Делитель не может быть нулем"
    return a / b

print(divide(10, 2))  # 5.0
print(divide(10, 0))  # AssertionError: Делитель не может быть нулем
```

|----------------|-------------------------|-------------------------|

|**С сообщением**|`assert условие, "текст"`|Ошибка с описанием       |

## Что такое assert?

**assert** — оператор для проверки утверждений во время разработки.

**Назначение:**

- 🐛 Отладка кода
- ✅ Проверка внутренних инвариантов
- 📝 Документирование предположений

**⚠️ Важно:** Assert — это инструмент **отладки**, а не обработки ошибок!

## Синтаксис

```python
# Базовая форма
assert условие

# С сообщением
assert условие, "сообщение об ошибке"

# Эквивалентно
if not условие:
    raise AssertionError("сообщение об ошибке")
```

## Базовые примеры

### Проверка значений

```python
age = 29
assert age > 0, 'Возраст должен быть положительным'

# Если age = -5, получим:
# AssertionError: Возраст должен быть положительным
```

### Проверка перед операцией

```python
num1 = 20
num2 = 0

assert num2 != 0, 'Делитель не может быть нулем'
print('Частное:', num1 / num2)
```

### Проверка диапазона

```python
def set_age(age):
    assert 0 < age < 150, f"Возраст {age} вне допустимого диапазона"
    print(f"Возраст: {age}")

set_age(25)   # ✅ OK
set_age(-5)   # ❌ AssertionError
```

## Assert — оператор, НЕ функция!

```python
# ❌ НЕПРАВИЛЬНО - создает кортеж, который всегда True!
assert(2 + 2 == 5, "Ошибка")  # Не выдаст ошибку!

# ✅ ПРАВИЛЬНО
assert 2 + 2 == 5, "Ошибка"  # AssertionError
```

## Практические примеры

### Проверка предусловий

```python
def get_discount(price, percent):
    assert price > 0, "Цена должна быть положительной"
    assert 0 <= percent <= 100, "Скидка от 0 до 100"
    return price * (1 - percent / 100)
```

### Проверка инвариантов класса

```python
class BankAccount:
    def __init__(self, balance=0):
        assert balance >= 0
        self.balance = balance
    
    def withdraw(self, amount):
        assert amount > 0
        assert amount <= self.balance, "Недостаточно средств"
        self.balance -= amount
        assert self.balance >= 0  # Инвариант
```

### Проверка типов и структуры

```python
def process_user(user):
    assert isinstance(user, dict)
    assert 'name' in user, "Отсутствует поле 'name'"
    assert 'age' in user, "Отсутствует поле 'age'"
    print(f"Обработка: {user['name']}")
```

## Отключение assert

Assert можно отключить флагом `-O`:

```bash
# С assert (проверки работают)
python script.py

# Без assert (все assert игнорируются)
python -O script.py
```

**⚠️ Код должен работать корректно даже с отключенным assert!**

## Когда использовать?

### ✅ Используйте assert для:

|Ситуация                     |Пример                            |
|-----------------------------|----------------------------------|
|**Внутренние инварианты**    |`assert balance >= 0`             |
|**Проверка после вычислений**|`assert 0 <= probability <= 1`    |
|**Документирование**         |`assert is_sorted(arr)`           |
|**Невозможные ситуации**     |`assert False, "Недостижимый код"`|

### ❌ НЕ используйте для:

|Ситуация                             |Вместо assert используйте               |
|-------------------------------------|----------------------------------------|
|**Валидация пользовательского ввода**|`if not valid: raise ValueError`        |
|**Проверка файлов**                  |`if not exists: raise FileNotFoundError`|
|**Критичная бизнес-логика**          |`if invalid: raise CustomError`         |
|**Операции с побочными эффектами**   |Выполните операцию, затем проверьте     |

## Assert vs исключения

```python
# ❌ Плохо - для пользовательских данных
def register(age):
    assert age >= 18, "Возраст должен быть 18+"

# ✅ Хорошо - используйте исключения
def register(age):
    if age < 18:
        raise ValueError("Возраст должен быть 18+")
```

**Правило:** Assert для программистских ошибок, исключения для ожидаемых ошибок.

## Распространенные ошибки

### ❌ Ошибка 1: Скобки как в функции

```python
# ❌ Неправильно - кортеж всегда True
assert(condition, "message")

# ✅ Правильно
assert condition, "message"
```

### ❌ Ошибка 2: Побочные эффекты

```python
# ❌ ОЧЕНЬ ПЛОХО - операция может не выполниться!
assert users.append(new_user) is not None

# ✅ Хорошо
users.append(new_user)
assert len(users) > 0
```

### ❌ Ошибка 3: Валидация API

```python
# ❌ Плохо - можно отключить
def pay(amount):
    assert amount > 0

# ✅ Хорошо
def pay(amount):
    if amount <= 0:
        raise ValueError("Сумма должна быть > 0")
```

### ❌ Ошибка 4: Обработка AssertionError

```python
# ❌ Плохо - assert не для обработки!
try:
    assert something()
except AssertionError:
    fix()

# ✅ Хорошо - используйте обычные исключения
if not something():
    raise CustomError()
```

## Основные правила

1. ✅ Assert — для отладки, не для обработки ошибок
2. ✅ Assert можно отключить — код должен работать без него
3. ✅ Используйте для внутренних проверок
4. ❌ Не используйте для валидации пользовательских данных
5. ❌ Не используйте для критичной логики
6. ❌ Не обрабатывайте AssertionError через try-except

## Сравнительная таблица

|Критерий                  |assert           |raise Exception  |
|--------------------------|-----------------|-----------------|
|**Можно отключить**       |✅ Да (флаг -O)   |❌ Нет            |
|**Для отладки**           |✅ Да             |❌ Нет            |
|**Для продакшена**        |❌ Нет            |✅ Да             |
|**Валидация пользователя**|❌ Нет            |✅ Да             |
|**Внутренние проверки**   |✅ Да             |⚠️ Можно          |
|**Производительность**    |✅ Можно отключить|⚠️ Всегда работает|
