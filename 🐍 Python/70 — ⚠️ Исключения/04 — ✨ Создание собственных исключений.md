# ✨ Создание собственных исключений

## 📌 Коротко

> Собственные исключения позволяют создавать специфичные для вашего приложения ошибки. Они создаются путём наследования от встроенного класса `Exception` или его подклассов.

---

## 💡 Полезные советы

- **Наследуйте от Exception, не от BaseException** — исключения приложения должны быть обычными
- **Используйте описательные имена** — заканчивайте на `Error` или `Exception`
- **Документируйте исключения** — добавляйте docstring с объяснением
- **Создавайте иерархию** — группируйте похожие ошибки в базовый класс
- **Вызывайте `super().__init__()`** — при необходимости передачи сообщения

---

## 🚀 Быстрая справка

| Тип | Синтаксис | Для чего |
|---|---|---|
| **Простое исключение** | `class CustomError(Exception): pass` | Специфичная ошибка приложения |
| **С сообщением** | `raise CustomError("сообщение")` | Возбуждение исключения |
| **Иерархия** | `class ValidationError(ValueError): pass` | Группировка ошибок |
| **С доп. данными** | `self.code = code` в `__init__` | Ошибка с метаданными |

## 📖 Теория

## Как происходит исключение?

Более технически точно:

### 1. **Обнаружение ошибки**

Когда интерпретатор Python доходит до строки `result = num ** 2`, он пытается выполнить операцию возведения в степень.

### 2. **Создание объекта исключения**

Python понимает, что типы несовместимы (строка и число), и **немедленно создает экземпляр класса `TypeError`**. Этот объект уже содержит всю информацию об ошибке - сообщение, тип, место в коде и т.д.

### 3. **"Бросок" исключения**

Python "бросает" (raises) этот созданный объект исключения в систему обработки ошибок.

### 4. **Поиск обработчика**

Система Python начинает искать подходящий блок `except`, который может поймать это исключение.

---

## Визуализация процесса

```python
num = '10'
try:
    # Момент выполнения следующей строки:
    result = num ** 2
    # ↑
    # В ЭТОТ МОМЕНТ создается объект TypeError!
    # Дальнейший код в блоке try НЕ выполняется
    
    print(result)  # ← эта строка никогда не выполнится
    print("Этот текст не увидим")  # ← и эта тоже
    
except TypeError as err:  # ← объект исключения "ловится" здесь
    print(err)  # Выводим сообщение об ошибке
```

**Доказательство:** Обратите внимание, что после строки с ошибкой в блоке `try` есть еще команды `print`, но они никогда не выполняются. Выполнение блока `try` прерывается мгновенно в момент создания исключения.
