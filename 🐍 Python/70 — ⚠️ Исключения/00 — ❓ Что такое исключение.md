# ❓ Что такое исключение?

## 📌 Коротко

> **Исключение** — это объект, который интерпретатор Python создаёт во время выполнения программы, когда происходит ошибка. Это экземпляр класса, который наследует от `BaseException`.

---

## 💡 Ключевые моменты

- **Это объект** — конкретный экземпляр класса
- **Создаёт интерпретатор** — автоматически при ошибке
- **Имеет цепочку наследования** — каждое исключение наследует от родительского класса
- **Можно перехватить** — используя `try-except`

---

## 📖 Теория

## Что такое исключение?

**Исключение** — это объект, который интерпретатор Python создаёт во время выполнения программы, когда происходит ошибка «негативное событие».

- Это именно **объект**
- Его создаёт **интерпретатор**, а не программист напрямую

---

## Какой это объект?

Это объект класса, который является потомком `BaseException`.

**Любое исключение — это:**

```
объект → экземпляр конкретного класса → наследует от Exception → наследует от BaseException
```

---

## Пример: KeyError и цепочка наследования

### Код:

```python
d = {}
x = d["нет_такого_ключа"]
```

### Что происходит:

1. Интерпретатор видит, что ключа нет
2. Определяет, какое исключение подходит: `KeyError`
3. Создаёт экземпляр класса `KeyError`
4. «Выбрасывает» его

### Цепочка наследования:

```
BaseException
    ↑
Exception
    ↑
LookupError
    ↑
KeyError
```

**Что это значит:**

- `KeyError` наследует от `LookupError`
- `LookupError` наследует от `Exception`
- `Exception` наследует от `BaseException`

**Объект ошибки является экземпляром всех этих классов благодаря наследованию.**

---

## Итоговая формулировка

> **Исключение** — это объект, который интерпретатор Python создаёт при возникновении ошибки.
> 
> Этот объект является экземпляром класса, наследующегося от `BaseException`.
> 
> Например, при отсутствии ключа в словаре создаётся объект класса `KeyError`, который наследует от `LookupError` → `Exception` → `BaseException`.
