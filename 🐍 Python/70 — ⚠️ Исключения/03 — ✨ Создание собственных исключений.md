# ✨ Создание собственных исключений

## 📌 Коротко

> Python позволяет создавать собственные классы исключений, наследуя их от встроенного класса `Exception` или его подклассов. Это даёт возможность обрабатывать специфичные для вашего приложения ошибки.

---

## 💡 Полезные советы

- **Наследуйте от Exception, не от BaseException** — исключения приложения должны быть обычными
- **Используйте описательные имена** — заканчивайте на `Error` или `Exception`
- **Документируйте исключения** — добавляйте docstring с объяснением
- **Создавайте иерархию** — группируйте похожие ошибки в базовый класс
- **Вызывайте `super().__init__()`** — при необходимости передачи сообщения

---

## 🚀 Быстрая справка

| Тип | Пример | Для чего |
|---|---|---|
| **Простое исключение** | `class CustomError(Exception): pass` | Специфичная ошибка приложения |
| **С сообщением** | `class CustomError(Exception): def __init__(self, msg): super().__init__(msg)` | Ошибка с деталями |
| **Иерархия** | `class ValidationError(ValueError): pass` | Группировка ошибок |
| **С доп. данными** | `class CustomError(Exception): def __init__(self, code, msg): self.code = code` | Ошибка с метаданными |
