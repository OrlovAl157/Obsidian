# 06 — 🔗 Памятка: Цепочки и группы исключений

## 🎯 Суть в 30 секунд

**Цепочки исключений** — когда одна ошибка вызывает другую, Python показывает обе.
**Группы исключений** (Python 3.11+) — несколько ошибок, собранные вместе для одновременной обработки.

---

## 📌 Часть 1: Цепочки исключений

### 🚀 Шпаргалка

```python
# ❌ Плохо - потеряется исходная ошибка
try:
    x = 1 / 0
except ZeroDivisionError:
    raise RuntimeError("Что-то сломалось")

# ✅ Хорошо - явная причина
try:
    x = 1 / 0
except ZeroDivisionError as err:
    raise RuntimeError("Что-то сломалось") from err

# ✅ Пробросить дальше (обработал, но отправляю выше)
try:
    x = 1 / 0
except ZeroDivisionError:
    print("Залоггировали")
    raise  # Пробросили ту же ошибку

# ✅ Скрыть контекст (редко)
try:
    x = 1 / 0
except ZeroDivisionError as err:
    raise RuntimeError("Ошибка") from None  # Контекст скрывается
```

---

### 📊 Три атрибута исключения

| Атрибут | Что хранит | Когда устанавливается | Пример |
|---------|-----------|----------------------|--------|
| **`__cause__`** | Явная причина (через `from`) | Когда напишешь `raise ... from err` | `RuntimeError.__cause__` → ZeroDivisionError |
| **`__context__`** | Контекст (Python заметил сам) | Когда ошибка возникла во время обработки другой | `RuntimeError.__context__` → ZeroDivisionError |
| **`__suppress_context__`** | Флаг подавления контекста | Когда напишешь `from None` | `True` = скрыть контекст в трассировке |

---

### 💡 Понимание разницы

```python
# Вариант 1: БЕЗ from
try:
    x = 1 / 0
except ZeroDivisionError:
    raise RuntimeError("Ошибка")

# Результат:
# __cause__ = None          (мы не указали явно)
# __context__ = ZeroDivisionError (Python заметил сам)
# Сообщение: "During handling of the above exception..."

# Вариант 2: С from err
try:
    x = 1 / 0
except ZeroDivisionError as e:
    raise RuntimeError("Ошибка") from e

# Результат:
# __cause__ = ZeroDivisionError (мы указали явно через from)
# __context__ = ZeroDivisionError (Python всё равно видит)
# Сообщение: "The above exception was the direct cause..."

# Вариант 3: С from None
try:
    x = 1 / 0
except ZeroDivisionError as e:
    raise RuntimeError("Ошибка") from None

# Результат:
# __cause__ = None (мы подавили через from None)
# __context__ = ZeroDivisionError (всё равно есть)
# __suppress_context__ = True (контекст скрыт в трассировке)
```

---

### 🔥 Три типа raise

```python
# 1️⃣ Пробросить текущее исключение (то же самое)
try:
    x = 1 / 0
except ZeroDivisionError:
    print("Обработали")
    raise  # Пробросили ZeroDivisionError дальше
    # __context__ = None (как будто мы его не ловили)

# 2️⃣ Возбудить новое (неявная цепочка)
try:
    x = 1 / 0
except ZeroDivisionError:
    raise RuntimeError("Новая ошибка")
    # Создаёт цепочку автоматически
    # __context__ = ZeroDivisionError

# 3️⃣ Возбудить новое с явной причиной
try:
    x = 1 / 0
except ZeroDivisionError as err:
    raise RuntimeError("Новая ошибка") from err
    # __cause__ = err (явная причина)
    # __context__ = ZeroDivisionError (контекст)
```

---

### 📈 Цепочка с несколькими уровнями

```python
# Каждое звено помнит своё ПРЕДЫДУЩЕЕ
# ZeroDivisionError → RuntimeError → ValueError

# RuntimeError.__context__ = ZeroDivisionError
# RuntimeError.__cause__ = ZeroDivisionError (если from)

# ValueError.__context__ = RuntimeError
# ValueError.__cause__ = RuntimeError (если from)

# В трассировке видны ВСЕ три
```

## 📌 Часть 2: Группы исключений (Python 3.11+)

### 🏗️ Иерархия классов

В Python 3.11 добавлены два класса:

```
BaseExceptionGroup (базовый класс)
    ↓
    └─ ExceptionGroup (наследник)
```

- **BaseExceptionGroup** — базовый класс для всех групп исключений
- **ExceptionGroup** — конкретная реализация, наследник BaseExceptionGroup

**На практике ты используешь ExceptionGroup:**

```python
from exceptions import ExceptionGroup

exc_list = [ValueError('1'), TypeError('2')]
group = ExceptionGroup('Ошибки', exc_list)

print(type(group))  # <class 'exceptions.ExceptionGroup'>
print(isinstance(group, ExceptionGroup))  # True
```

---

### 🚀 Шпаргалка

```python
from exceptions import ExceptionGroup

# 1️⃣ Собрать ошибки в список
exc_list = []
for item in items:
    try:
        process(item)
    except Exception as err:
        err.add_note(f'При обработке {item}')
        exc_list.append(err)

# 2️⃣ Возбудить группу
if exc_list:
    raise ExceptionGroup('Произошли ошибки', exc_list)

# 3️⃣ Обработать по типам
try:
    ...
except* ValueError as err:
    print(f"ValueError-ы: {err.exceptions}")
except* TypeError as err:
    print(f"TypeError-ы: {err.exceptions}")
except* Exception as err:  # Ловит остальное
    print(f"Остальное: {err.exceptions}")
```

---

### 📊 except vs except*

| | **except ExceptionGroup** | **except* ValueError** |
|---|---|---|
| **Что ловит** | Всю группу целиком | Только ValueError из группы |
| **Результат** | Один объект группы | Новая группа только с ValueError |
| **Когда используется** | Обработка группы как одного целого | Обработка по типам ошибок |
| **Пример** | `except ExceptionGroup as grp:` | `except* ValueError as err:` |

---

### 💡 Атрибуты ExceptionGroup

```python
exc_list = [ValueError('1'), TypeError('2'), RuntimeError('3')]
grp = ExceptionGroup('Ошибки', exc_list)

# .message — сообщение группы
print(grp.message)  # 'Ошибки'

# .exceptions — кортеж всех ошибок
print(grp.exceptions)  # (ValueError('1'), TypeError('2'), RuntimeError('3'))

# .__notes__ — заметки, добавленные через add_note()
print(grp.__notes__)  # ['заметка1', 'заметка2', ...]
```

---

### 🎯 Метод add_note() — добавить комментарий

```python
exc_list = []

for n, person in enumerate(people):
    try:
        greet(person)
    except Exception as err:
        # Добавляем контекст к ошибке
        err.add_note(f'Ошибка на итерации {n}')
        err.add_note(f'Обрабатывали: {person}')
        exc_list.append(err)

if exc_list:
    raise ExceptionGroup('Произошли ошибки при обработке', exc_list)
```

### 🔥 Обработка групп: полный пример

```python
from exceptions import ExceptionGroup

exc_list = [
    ValueError('ошибка 1'),
    TypeError('ошибка 2'),
    RuntimeError('ошибка 3'),
    ValueError('ошибка 4')  # Второй ValueError
]

try:
    raise ExceptionGroup('Произошло несколько ошибок', exc_list)

# Вариант 1: Обработка по типам
except* ValueError as err:
    print(f"ValueError: {err.exceptions}")  # Обе ValueError
    for exc in err.exceptions:
        print(f"  - {exc}")

except* TypeError as err:
    print(f"TypeError: {err.exceptions}")

except* RuntimeError as err:
    print(f"RuntimeError: {err.exceptions}")
    
except* Exception as err:
    print(f"Остальное: {err.exceptions}")

# Вариант 2: Обработка всей группы целиком
except ExceptionGroup as grp:
    print(f"Группа: {grp.message}")
    print(f"Всего ошибок: {len(grp.exceptions)}")
    for exc in grp.exceptions:
        print(f"  - {type(exc).__name__}: {exc}")
```

## 📌 Дополнительно: Метод split() и множественная обработка

### 🔀 Метод split() — разделение группы по типу

```python
exc_list = [
    ValueError('Ошибка 1'),
    TypeError('Ошибка 2'),
    RuntimeError('Ошибка 3'),
    ValueError('Ошибка 4')
]

exc_group = ExceptionGroup('Произошло несколько ошибок', exc_list)

# split() возвращает кортеж из двух групп
valueerror_group, other_errors = exc_group.split(ValueError)

print(repr(valueerror_group))
# ExceptionGroup('Произошло несколько ошибок', [ValueError('Ошибка 1'), ValueError('Ошибка 4')])

print(repr(other_errors))
# ExceptionGroup('Произошло несколько ошибок', [TypeError('Ошибка 2'), RuntimeError('Ошибка 3')])
```

**Что возвращает `split()`:**
- **Первая группа** — все исключения указанного типа
- **Вторая группа** — все остальные исключения

**Использование:**
```python
try:
    raise ExceptionGroup('Ошибки', exc_list)
except ExceptionGroup as grp:
    # Разделяем ValueError от остальных
    values, others = grp.split(ValueError)
    
    if values:
        print(f"ValueError: {values.exceptions}")
    
    if others:
        print(f"Остальное: {others.exceptions}")
```

### 🔗 Обработка нескольких типов в одном except*

Можешь обрабатывать несколько типов в одном блоке `except*`:

```python
exc_list = [
    ValueError('Ошибка 1'),
    TypeError('Ошибка 2'),
    RuntimeError('Ошибка 3'),
    ValueError('Ошибка 4')
]

try:
    raise ExceptionGroup('Произошло несколько ошибок', exc_list)

# Один блок для нескольких типов
except* (ValueError, TypeError) as e:
    print('ValueError и TypeError:')
    for exc in e.exceptions:
        print(f"  - {type(exc).__name__}: {exc}")

except* Exception as e:
    print('Остальные ошибки:')
    for exc in e.exceptions:
        print(f"  - {type(exc).__name__}: {exc}")
```

**Вывод:**
```
ValueError и TypeError:
  - ValueError: Ошибка 1
  - TypeError: Ошибка 2
  - ValueError: Ошибка 4
Остальные ошибки:
  - RuntimeError: Ошибка 3
```

**Синтаксис:**
```python
except* (Type1, Type2, Type3) as e:
    # Обработка всех указанных типов вместе
    for exc in e.exceptions:
        handle(exc)
```

## 📌 Частые ошибки

### ❌ Ошибка 1: Использовать except* с типом, не в группе

```python
# ❌ НЕПРАВИЛЬНО - except* не работает с обычными исключениями
try:
    x = 1 / 0
except* ZeroDivisionError:
    print("Поймали")  # ОШИБКА! except* работает только с группами

# ✅ ПРАВИЛЬНО
try:
    x = 1 / 0
except ZeroDivisionError:
    print("Поймали")
```

### ❌ Ошибка 2: except* ExceptionGroup

```python
# ❌ НЕПРАВИЛЬНО
try:
    raise ExceptionGroup('Ошибки', exc_list)
except* ExceptionGroup:
    pass  # TypeError: catching ExceptionGroup with except* is not allowed

# ✅ ПРАВИЛЬНО
except ExceptionGroup as grp:
    pass  # Используй обычный except
```

### ❌ Ошибка 3: Забыть обработать все типы

```python
# ❌ ПЛОХО - TypeError не обработан, вызовет новую группу
try:
    raise ExceptionGroup('Ошибки', [ValueError(...), TypeError(...)])
except* ValueError as err:
    pass  # TypeError всё равно вызовет ошибку

# ✅ ХОРОШО - используй except* Exception в конце
try:
    raise ExceptionGroup('Ошибки', [ValueError(...), TypeError(...)])
except* ValueError as err:
    pass
except* Exception as err:
    pass  # Ловит всё остальное
```

## 📈 Когда использовать что?

### 🔗 Цепочки исключений используй когда:
- Одна ошибка вызывает другую (следствие)
- Нужно показать **причину** проблемы
- Обрабатываешь ошибку и возбуждаешь новую

### 👥 Группы исключений используй когда:
- Обрабатываешь список объектов
- Каждый объект может вызвать ошибку
- Хочешь обработать **все** ошибки, не теряя информацию
- Нужно показать несколько проблем одновременно

## 💎 Типичные схемы

### Схема 1: Цепочка с обработкой и пробросом

```python
def process_data(data):
    try:
        return int(data)
    except ValueError as err:
        # Залоггировали, очистили ресурсы
        print("Логирование ошибки")
        # Пробросили дальше
        raise
```

### Схема 2: Цепочка с преобразованием ошибки

```python
def safe_divide(a, b):
    try:
        return a / b
    except ZeroDivisionError as err:
        # Преобразовали в более понятную ошибку
        raise ValueError("Делитель не может быть нулём") from err
```

### Схема 3: Группа ошибок из цикла

```python
def validate_users(users):
    errors = []
    for user in users:
        try:
            validate_user(user)
        except ValueError as err:
            err.add_note(f'Ошибка при проверке {user["id"]}')
            errors.append(err)
    
    if errors:
        raise ExceptionGroup('Ошибки валидации', errors)
```
