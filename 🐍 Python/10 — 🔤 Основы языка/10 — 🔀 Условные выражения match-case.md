## 📌 Основы

**match-case** — оператор сопоставления с шаблонами (Python 3.10+), альтернатива `if-elif-else`.

**✅ Преимущества:**

- ✅ Читаемый код для множественных проверок
- ✅ Деструктуризация данных
- ✅ Проверка типов и структур

**⚠️ Ограничения:**

- ⚠️ Только проверка на равенство (нет `>`, `<`)
- ⚠️ Python 3.10+

### Синтаксис

```python
match значение:
    case шаблон_1:
        действие_1
    case шаблон_2:
        действие_2
    case _:
        по_умолчанию
```

Проверка **сверху вниз** до первого совпадения. Автоматический выход (как `break`).

---

## 🎯 Базовые примеры

### Простое сопоставление

```python
match lang:
    case "JavaScript":
        print("Фронтенд разработчик")
    case "Python":
        print("Data Scientist")
    case _:
        print("Язык не важен!")
```

### OR паттерн (`|`)

```python
match digit:
    case 0 | 3 | 6 | 9:
        print("Делятся на 3")
    case 1 | 4 | 7:
        print("Остаток 1")
    case _:
        print("Не цифра")
```

### Методы в match

```python
match direction.lower():
    case "north" | "south" | "east" | "west":
        print("Идём!")
    case _:
        print("Неизвестно")
```

## 🎭 Типы паттернов

### 1. Literal — Литералы

```python
case 42:        # Точное совпадение
case "text":
```

### 2. Capture — Захват переменной

```python
case ["go", direction]:  # direction получит значение
    print(f"Идём: {direction}")
```

### 3. Wildcard — `_` заглушка

```python
case (x, y, 0):  # Игнорируем z
    print(f"На плоскости: ({x}, {y})")
```

### 4. Sequence — Последовательности

```python
case []:                    # Пустой список
case [x]:                   # Один элемент
case [x, y, *rest]:         # Первые два + остальные
case [first, *middle, last]: # Начало + середина + конец
```

`*` — захватывает произвольное количество элементов.

### 5. Mapping — Словари

```python
case {"name": n, "role": "admin"}:
    print(f"Админ: {n}")
case {"name": n, **rest}:  # Остальные ключи в rest
    print(f"{n}, доп: {rest}")
```

Словарь может содержать **дополнительные ключи** — они игнорируются.

### 6. Class — Классы

```python
class Point:
    __match_args__ = ('x', 'y')  # Для позиционных аргументов
    
    def __init__(self, x, y):
        self.x = x
        self.y = y

match point:
    case Point(0, 0):              # Позиционно
        print("Начало координат")
    case Point(x=x, y=y):          # По имени
        print(f"({x}, {y})")
```

### 7. OR — Альтернативы

```python
case ["quit"] | ["exit"] | ["q"]:
    print("Выход")
```

### 8. AS — Именование паттерна

```python
case (x, y) as coords:
    print(f"Координаты: {coords}, x={x}")
```

### 9. Guard — Условие `if`

```python
case x if x < 0:
    print("Отрицательное")
case (x, y) if x == y:
    print("На диагонали")
```

## 🔍 Проверка типов

```python
match value:
    case int() | float():
        print("Число")
    case str():
        print("Строка")
    case list():
        print("Список")
```

### С захватом

```python
case int(x):
    print(f"Целое: {x}")
case [int(), int(), int()]:
    print("Три целых числа")
```

## ⚠️ Типичные ошибки

### 1. Переменная вместо литерала

```python
target = 42
match value:
    case target:  # ❌ Захват, не сравнение!

# ✅ Правильно:
case x if x == target:
```

### 2. `case _` не в конце

```python
match value:
    case _:   # ❌ Сработает всегда!
        ...
    case 42:  # Никогда не выполнится
```

### 3. Неверная длина

```python
data = [1, 2, 3, 4]
case [x, y]:  # ❌ Не совпадёт

# ✅ Правильно:
case [x, y, *rest]:
```

### 4. Условия в case

```python
case x > 10:  # ❌ Ошибка!

# ✅ Правильно:
case x if x > 10:
```

## 🆚 match vs if-elif-else

|Используйте|Когда|
|---|---|
|**match-case**|Множество проверок на равенство, деструктуризация, проверка структуры|
|**if-elif-else**|Условия `>`, `<`, `and`, `or`, диапазоны, Python < 3.10|

## 📝 Шпаргалка

```python
# Базовое
match value:
    case 1:                # Литерал
    case 1 | 2 | 3:        # OR
    case x:                # Захват
    case _:                # По умолчанию

# Последовательности
case [x, y]:               # Фиксированная
case [x, *rest]:           # С остатком
case [x, _, z]:            # Игнорирование

# Словари
case {"name": n}:          # Ключ-значение
case {"id": i, **rest}:    # Остальные ключи

# Классы
case Point(x, y):          # Позиционно
case Point(x=a, y=b):      # По имени

# Guard и AS
case x if x > 0:           # Условие
case [x, y] as pt:         # Именование

# Типы
case int():                # Тип
case int(x):               # Тип + захват
```

## 💡 Ключевые моменты

1. **Порядок важен** — проверка сверху вниз
2. **Автоматический выход** — не нужен `break`
3. **`case x:` захватывает**, не сравнивает
4. **Guards** для условий: `case x if x > 0:`
5. **`*` для остатка**: `case [x, *rest]:`
6. **`_` игнорирует** значение
7. **Только равенство** — нет `>`, `<`
8. **Словари гибкие** — могут содержать доп. ключи
9. **Python 3.10+** обязателен

**Мнемоника:**

- `|` = "или"
- `*` = "остальные"
- `_` = "любое"
- `as` = "как" (именование)
- `if` = "если" (условие)
