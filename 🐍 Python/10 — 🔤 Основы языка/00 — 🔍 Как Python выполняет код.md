# 🐍 Как Python выполняет код

## Полная цепочка

```
твой .py файл
      ↓  (компилятор Python)
байткод (.pyc)
      ↓  (PVM читает байткод)
вызов C-функций CPython
      ↓  (уже скомпилированный C)
машинный код (01)
      ↓
   процессор
```

---

## Что такое байткод

Байткод — это **не машинный код**. Это список высокоуровневых инструкций для PVM:

```python
import dis

def add(a, b):
    return a + b

dis.dis(add)
# LOAD_FAST   0 (a)
# LOAD_FAST   1 (b)
# BINARY_OP   0 (+)
# RETURN_VALUE
```

Байткод **никогда не превращается в 01** — он просто говорит PVM что делать.

---

## Что такое PVM

PVM (Python Virtual Machine) — это большой цикл на C внутри CPython (`ceval.c`):

```
PVM видит BINARY_OP
      ↓
вызывает PyNumber_Add()  ← готовая C-функция
      ↓
процессор выполняет её
```

> PVM не переводит байткод в машинный код.  
> PVM **сама является** машинным кодом — она скомпилирована из C заранее.

Каждый объект Python — это структура в C:

```c
typedef struct {
    int ob_refcnt;          // счётчик ссылок (сборщик мусора)
    PyTypeObject *ob_type;  // указатель на тип объекта
    long ob_ival;           // само значение (для int)
} PyLongObject;
```

Поэтому даже простое `1 + 2` — это не одна машинная инструкция, а целая цепочка C-вызовов.

---

## Где происходит преобразование в 01

Один раз — при сборке CPython:

```
разработчики пишут CPython на C
      ↓  (gcc/clang компилирует)
python.exe — это и есть 01
```

Ты скачиваешь уже готовый `01`. Твой Python-код до этого уровня не доходит.

---

## Главная идея

**Python — язык поверх C.**  
Ты пишешь инструкции для PVM → PVM вызывает C-функции → C уже в `01`.

Именно поэтому Python медленнее C — лишний слой интерпретации.  
Именно поэтому `numpy` быстрый — критичные части написаны на C напрямую.

---

## Исключение — когда Python всё же компилирует в 01

**JIT-компиляторы** — PyPy, Numba.  
Они анализируют горячий код и компилируют его в настоящий машинный код прямо во время выполнения (в рантайме).

---

## Почему Python медленнее C

| | C | Python |
|---|---|---|
| Компиляция | один раз → 01 | байткод → PVM → C-функции |
| Простое сложение | 1 инструкция | цепочка C-вызовов |
| Тип переменной | известен заранее | проверяется в рантайме |
| Объект в памяти | просто число | структура PyObject |
