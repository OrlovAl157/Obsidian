match data:

### 💡 Ключевые особенности

- **Список и кортеж** — в шаблонах равнозначны: оба относятся к базовому типу `Sequence`. `(1,2,3)` совпадёт с шаблоном `[1,2,3]` и наоборот.
- **Множество** — нельзя использовать как шаблон (`SyntaxError`).
- **Словарь** — проверяет наличие указанных ключей. Лишние ключи **игнорируются** (в отличие от списков).
- **`*rest`** в списке/кортеже — упаковывает оставшиеся элементы в список. Только один `*` в шаблоне.
- **`**kwargs`** в словаре — упаковывает оставшиеся ключи. Только в конце шаблона.
- **Псевдоним `as`** — захватывает совпавшее значение в переменную для любого шаблона.

### 💡 Полезные советы

- Если нужно проверить именно `list`, а не `tuple` — используй `list([...])` явно.
- `*_` в конце списка — "есть ещё элементы, но они не нужны".
- `**_` в словаре — синтаксически недопустимо, используй `**rest` с реальным именем.
- Псевдоним `as` для OR-шаблона (`'bee' | 'geek' as text`) — удобно когда нужно значение после совпадения.

### 🚀 Быстрая справка

| **Шаблон** | **Синтаксис** | **Что делает** |
|---|---|---|
| Список/кортеж | `case [x, y]:` | совпадение с последовательностью из 2 элементов |
| С остатком | `case [x, *rest]:` | первый + оставшиеся в `rest` |
| Упаковка середины | `case [_, *mid, _]:` | крайние + середина в `mid` |
| Словарь | `case {'key': v}:` | ключ есть → значение в `v` |
| Словарь с остатком | `case {'a': _, **rest}:` | известный ключ + остальные в `rest` |
| Псевдоним | `case [x, y] as pt:` | совпадение + весь шаблон в `pt` |
| Псевдоним OR | `case 'a' \| 'b' as s:` | одно из значений + захват в `s` |

## 📋 Списки и кортежи

**Назначение:** проверить структуру последовательности — количество элементов и их значения.

| Особенность | Описание |
|---|---|
| `[1, 2, 3]` и `(1, 2, 3)` | равнозначны в шаблоне (оба — `Sequence`) |
| `(1, 2, 3)` | кортеж можно писать со скобками и без |
| `{1, 2, 3}` | множество — `SyntaxError`! |
| Количество элементов | должно совпадать точно (без `*`) |

**Примеры:**

```python
numbers = [1, 2, 3]

match numbers:
    case [x]:
        print(f'Один элемент: {x}')
    case [x, y]:
        print(f'Два элемента: {x} и {y}')
    case [x, y, z]:
        print(f'Три элемента: {x}, {y} и {z}')
# → Три элемента: 1, 2 и 3
```

```python
# ❗ Sequence-ловушка: список и кортеж равнозначны в шаблоне
numbers = (1, 2, 3)   # это кортеж!

match numbers:
    case [1, 2, 3]:    # шаблон список — всё равно совпадёт
        print('Список чисел')
# → Список чисел

# ✅ Если нужна строгая проверка типа
match numbers:
    case list([1, 2, 3]):    # только list
        print('Именно список')
    case tuple((1, 2, 3)):   # только tuple
        print('Именно кортеж')
```

```python
# Шаблоны захвата в элементах — захватывают любые значения
numbers = ['1', 'два', {3: 'три'}]
match numbers:
    case [x, y, z]:
        print(f'Три элемента: {x}, {y} и {z}')
# → Три элемента: 1, два и {3: 'три'}
```

**Важное:** в шаблоне списка/кортежа без `*` количество элементов должно совпадать **точно**.

## 📦 Упаковка элементов (`*`)

**Назначение:** захватить произвольное количество элементов в список.

**Примеры:**

```python
numbers = [1, 2, 3, 4, 5]

match numbers:
    case [first, *rest]:
        print(first, rest)           # 1 [2, 3, 4, 5]

    case [x, y, z, *tail]:
        print(x, y, z, tail)         # 1 2 3 [4, 5]

    case [first, *middle, last]:
        print(first, middle, last)   # 1 [2, 3, 4] 5

    case [*head, last]:
        print(head, last)            # [1, 2, 3, 4] 5
```

```python
# Если элементов ровно столько сколько в шаблоне — *rest будет пустым
numbers = [1, 2, 3]
match numbers:
    case [_, _, _, *tail]:
        print(tail)   # []
```

```python
# _ для игнорирования позиции, захват через имя
match numbers:
    case [1, 2, _]:
        print('Первые два — 1 и 2')
    case [1, 2, x]:
        print(f'Третий: {x}')
```

**Важное:** в одном шаблоне только **один** `*`. `[*a, *b]` — `SyntaxError`.

## 🗂 Словари

**Назначение:** проверить наличие определённых ключей в словаре.

| Особенность | Описание |
|---|---|
| Лишние ключи | игнорируются — не нужно перечислять все |
| Порядок ключей | не важен |
| Захват ключей | недоступен (`{key: _}` — `SyntaxError`) |
| `**rest` | только в конце шаблона |

**Примеры:**

```python
d = {'name': 'Tim', 'role': 'admin', 'age': 30}

match d:
    case {'name': n, 'role': 'admin'}:
        print(f'Администратор: {n}')
# → Администратор: Tim (лишний ключ 'age' игнорируется)

# Проверка наличия ключа, значение любое
match d:
    case {'name': name}:
        print(f'Имя: {name}')
```

```python
# **rest захватывает оставшиеся ключи
d = {'bee': None, 'geek': None, 'py': None}
match d:
    case {'bee': _, 'geek': _, **rest}:
        print(rest)   # {'py': None}

# Если остальных ключей нет — пустой словарь
d = {'bee': None, 'geek': None}
match d:
    case {'bee': _, 'geek': _, **rest}:
        print(rest)   # {}
```

```python
# ❗ **rest только в конце
case {**kwargs, 'bee': _}:         # ❌ SyntaxError
case {'bee': _, **_, 'geek': _}:   # ❌ SyntaxError

# ❗ Захват ключей недоступен
case {key: _}:   # ❌ SyntaxError
```

**Важное:** словарный шаблон проверяет только наличие указанных ключей. Остальные **молча игнорируются** — это отличает словари от списков.

## 🏷 Псевдонимы (`as`)

**Назначение:** захватить совпавшее значение в переменную для любого шаблона.

**Примеры:**

```python
# Псевдоним для простого шаблона
match s:
    case 'bee' as bee:
        print(f'Строка {bee}')

# Псевдоним для OR-шаблона
match s:
    case 'bee' | 'geek' as text:
        print(f'Строка {text}')   # bee или geek — в любом случае захватится
```

```python
# Псевдоним для всего контейнера
numbers = [1, 2, 3]
match numbers:
    case [int() | float(),
          int() | float(),
          int() | float()] as nums:
        print(nums)   # [1, 2, 3]

# Псевдоним для каждого элемента
match numbers:
    case [int() | float() as x,
          int() | float() as y,
          int() | float() as z]:
        print(x, y, z)   # 1 2 3
```

```python
# Псевдоним из одного case недоступен в другом case
match s:
    case 'bee' as bee:
        print(bee)
    case 'geek' as geek:
        print(bee)    # ❌ NameError если s == 'geek'

# Одно имя в разных case — ок
match s:
    case 'bee' as text:
        print(text)
    case 'geek' as text:
        print(text)
```

**Важное:** `_` нельзя использовать как псевдоним (`case [1,2,3] as _:` — `SyntaxError`). Псевдоним доступен в охраннике `if`.

## ⚠️ Важные особенности

```python
# ❗ Sequence-ловушка: list и tuple равнозначны в шаблоне
(1, 2, 3)  # совпадёт с шаблоном [1, 2, 3]!

# ❗ Множество как шаблон — SyntaxError
case {1, 2, 3}:   # ❌

# ❗ Кортежи без скобок в OR
case 1, 2, 'bee' | 1, 2, 'geek':     # ❌ не то что ожидается
case (1, 2, 'bee') | (1, 2, 'geek'): # ✅

# ❗ **rest только в конце словарного шаблона
case {**kwargs, 'key': v}:   # ❌ SyntaxError

# ❗ Захват ключей словаря недоступен
case {key: value}:   # ❌ SyntaxError

# ❗ _ нельзя использовать как псевдоним
case [1, 2, 3] as _:   # ❌ SyntaxError
```
