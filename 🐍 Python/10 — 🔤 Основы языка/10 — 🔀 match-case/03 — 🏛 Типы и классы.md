```python
# 🏛 MATCH-CASE — ТИПЫ И КЛАССЫ
match value:
    case int():            # проверка типа без значения
    case int(x):           # проверка типа + захват
    case float(69.0):      # тип + конкретное значение
    case int() | float():  # один из типов

# Пользовательский класс
class Cat:
    __match_args__ = ('name', 'breed')
match cat:
    case Cat('Кемаль', 'Британский'):   # позиционно
    case Cat(breed='Британский'):        # по имени
```

# 🏛 Памятка: match-case — типы и классы

### 💡 Ключевые особенности

- **`case int():`** — проверяет только принадлежность типу. **`case int:`** без скобок — шаблон захвата (ловушка!).
- **`case int(69):`** — проверяет тип И конкретное значение.
- **Числа** в шаблонах сравниваются через базовый тип `Number` — поэтому `69` совпадёт с шаблоном `69.0`.
- **`__match_args__`** — кортеж имён атрибутов, задаёт порядок для позиционных шаблонов. Только кортеж, не список.
- **Датаклассы** автоматически создают `__match_args__` на основе всех полей.
- **В шаблоне** экземпляр класса **не создаётся** — это образец, не конструктор.
- **`Enum`** работает в шаблонах через `Weekday.FRIDAY`, `__match_args__` у него нет.

### 💡 Полезные советы

- Если класс без `__match_args__` — можно использовать **именованные аргументы** в шаблоне: `Cat(name='Кемаль')`.
- `@dataclass(match_args=False)` — отключает автоматическое создание `__match_args__`.
- Для проверки "любое число" используй `Number()` из `numbers`.
- Для проверки "любой контейнер" используй `Container()` из `collections.abc`.

### 🚀 Быстрая справка

| **Шаблон** | **Что проверяет** |
|---|---|
| `case int():` | принадлежность типу `int` |
| `case int(x):` | тип `int` + захват значения в `x` |
| `case int(69):` | тип `int` И значение равно `69` |
| `case int:` | ⚠️ шаблон захвата! (не проверка типа) |
| `case int() \| float():` | `int` или `float` |
| `case Cat('Кемаль'):` | тип `Cat` + `name == 'Кемаль'` (через `__match_args__`) |
| `case Cat(name='Кемаль'):` | тип `Cat` + атрибут `name == 'Кемаль'` |
| `case Cat(name):` | тип `Cat` + захват `name` в переменную |

## 🔍 Проверка типов

**Назначение:** проверить принадлежность объекта типу данных.

| Синтаксис | Что делает |
|---|---|
| `case int():` | тип `int` — любое значение |
| `case int(69):` | тип `int` И значение `== 69` |
| `case int(x):` | тип `int` + захват значения в `x` |
| `case int:` | ⚠️ шаблон захвата, НЕ проверка типа |

**Примеры:**

```python
# Проверка типа без значения
match value:
    case float():
        print('Вещественное число')
    case int():
        print('Целое число')
    case str():
        print('Строка')

# Проверка типа + значение
match num:
    case float(69.0):
        print('Именно 69.0 типа float')
    case int(69):
        print('Именно 69 типа int')

# num = 69.2: float() совпадёт, но float(69.0) — нет (значение не равно)
```

```python
# Захват значения при проверке типа
match value:
    case int(x):
        print(f'Целое: {x}')
    case str(x):
        print(f'Строка: {x}')
```

```python
# ❗ Ловушка: без скобок — шаблон захвата, не проверка типа
num = 69

match num:
    case float():   # проверка типа ✅
        print('float')
    case float:     # шаблон захвата — num захватится в float! ❌
        print(f'Значение {float}')   # → Значение 69
```

**Важное:** всегда пиши скобки: `int()`, `float()`, `str()`. Без скобок имя класса становится шаблоном захвата.

## 🔢 Числовая иерархия в шаблонах

**Назначение:** понять почему `69` совпадает с шаблоном `69.0`.

**Примеры:**

```python
# Числа сравниваются через базовый тип Number
# Поэтому int 69 совпадёт с шаблоном float 69.0!
num = 69

match num:
    case 69.0:    # ← совпадёт первым!
        print('Вещественное 69.0')
    case 69:
        print('Целое 69')
# → Вещественное 69.0  ❗ неожиданно!

# ✅ Чтобы различать — проверяй тип явно
match num:
    case float(69.0):
        print('Именно float 69.0')
    case int(69):
        print('Именно int 69')
# → Именно int 69  ✅
```

```python
# Для проверки "любое число" — Number из numbers
from numbers import Number

match num:
    case float():
        print('float')
    case int():
        print('int')
    case Number():
        print('Другое число (complex, Decimal...)')
```

## 🏛 Пользовательские классы

**Назначение:** сопоставить с экземпляром класса по типу и/или атрибутам.

### `__match_args__`

Кортеж имён атрибутов — задаёт порядок для позиционных шаблонов.

```python
class Cat:
    __match_args__ = ('name', 'breed')   # ← только кортеж, не список!

    def __init__(self, name, breed):
        self.name = name
        self.breed = breed
```

**Примеры:**

```python
cat = Cat('Кемаль', 'Британский')

# Только тип
match cat:
    case Cat():
        print('Это кот')

# Позиционно — через __match_args__
match cat:
    case Cat('Кемаль', 'Британский'):
        print('Кот по имени Кемаль британской породы')

# Только первый атрибут из __match_args__
match cat:
    case Cat('Кемаль'):
        print('Кот по имени Кемаль')

# По имени — __match_args__ не обязателен
match cat:
    case Cat(breed='Британский'):
        print('Кот британской породы')
```

```python
# Захват атрибутов
match cat:
    case Cat(name, 'Британский'):       # name захватится
        print(f'Кот {name} — британец')

match cat:
    case Cat(name=cat_name, breed='Британский'):
        print(f'Кот {cat_name} — британец')
```

```python
# ❗ В шаблоне класс НЕ создаётся — это образец
# __init__ вызывается только при создании cat, не при match
class Cat:
    __match_args__ = ('name',)
    def __init__(self, name, breed):
        print('Создание!')   # вызовется только один раз
        self.name = name
        self.breed = breed

cat = Cat('Кемаль', 'Британский')   # → Создание!

match cat:
    case Cat('Кемаль'):   # __init__ НЕ вызывается повторно
        print('Найден')
```

```python
# ❗ __match_args__ должен быть кортежем
class Cat:
    __match_args__ = ['name']   # ❌ список

match cat:
    case Cat('Кемаль'):
# TypeError: Cat.__match_args__ must be a tuple (got list)

# ❗ Нельзя передать больше аргументов, чем в __match_args__
class Cat:
    __match_args__ = ('name',)   # только name
match cat:
    case Cat('Кемаль', 'Британский'):   # ❌ даёт 2, а в __match_args__ 1
# TypeError: Cat() accepts 1 positional sub-pattern (2 given)
```

## 🗃 Датаклассы

**Назначение:** датаклассы автоматически создают `__match_args__` из всех полей.

**Примеры:**

```python
from dataclasses import dataclass

@dataclass
class Cat:
    name: str
    breed: str

print(Cat.__match_args__)   # ('name', 'breed') — создано автоматически

cat = Cat('Кемаль', 'Британский')

match cat:
    case Cat('Кемаль', 'Британский'):
        print('Кот по имени Кемаль британской породы')
```

```python
# Переопределить __match_args__ в датаклассе
@dataclass
class Cat:
    __match_args__ = ('name',)   # только name
    name: str
    breed: str

match cat:
    case Cat('Кемаль'):   # ✅
        print('Кот по имени Кемаль')

match cat:
    case Cat('Кемаль', 'Британский'):   # ❌ TypeError
        ...
```

```python
# Отключить __match_args__ полностью
@dataclass(match_args=False)
class Cat:
    name: str
    breed: str

Cat.__match_args__   # ❌ AttributeError: no attribute '__match_args__'

# Но именованные аргументы всё ещё работают!
match cat:
    case Cat(name='Кемаль', breed='Британский'):   # ✅
        print('Найден')
```

## 🎌 Enum в шаблонах

**Назначение:** сопоставить с конкретным значением перечисления.

**Примеры:**

```python
from enum import Enum

class Weekday(Enum):
    MONDAY = 1
    FRIDAY = 5
    SUNDAY = 7

# Enum не имеет __match_args__, но работает через Enum.VALUE
weekday = Weekday.FRIDAY

match weekday:
    case Weekday.FRIDAY:
        print('Пятница')   # ✅

print(hasattr(Weekday, '__match_args__'))   # False
```

**Важное:** `Enum` использует синтаксис `Weekday.FRIDAY` (через точку), а не позиционные аргументы. `__match_args__` у `Enum` нет.
