#  Моржовый оператор `:=` — памятка

> Оператор `:=` (появился в Python 3.8, PEP 572) позволяет одновременно присвоить значение переменной и вернуть его как часть выражения.

## 📝 Синтаксис

```python
variable := expression
```

1. вычисляется `expression`
2. результат присваивается `variable`
3. это же значение **возвращается** как результат выражения

**Приоритет:** `:=` имеет **самый низкий приоритет**, как и `=`.
```python
n := len(word) > 5
# равнозначно
n := (len(word) > 5)
```

---

## 🆚 Отличие от `=`

`=` — оператор присваивания, **ничего не возвращает**, не может использоваться внутри выражений.

```python
num = 7
print(num)
```

`:=` — возвращает присвоенное значение, можно использовать прямо как аргумент.

```python
print(num := 7)   # 7 -- присвоили И вывели за раз
```

```python
print(num = 7)
# ❌ TypeError: 'num' is an invalid keyword argument for print()
# num = 7 воспринимается как именованный аргумент, не как присваивание
```

## ✅ Полезные сценарии

### 1. Избежать повторного вычисления в условии

```python
from keyword import kwlist

# ❌ len(word) вычисляется дважды
for word in kwlist:
    if len(word) > 5:
        print(f'{word}: {len(word)} символов')

# ✅ с :=  -- вычисляется один раз
for word in kwlist:
    if (n := len(word)) > 5:
        print(f'{word}: {n} символов')
```

> [!warning] Скобки обязательны
> ```python
> if n := len(word) > 5:   # n = (len(word) > 5) -- n становится bool!
> if (n := len(word)) > 5: # n = len(word) -- правильно
> ```
> Без скобок `:=` свяжется с результатом сравнения (`True`/`False`), а не с `len(word)`.

### 2. Цикл while с вводом без дублирования

```python
# ❌ input() вызывается в двух местах
words = []
word = input()
while word != 'stop':
    words.append(word)
    word = input()

# ✅ с :=
words = []
while (word := input()) != 'stop':
    words.append(word)
```

```python
# Аналогично при чтении файла построчно
with open('input.txt') as file:
    while line := file.readline().rstrip():
        print(line)
```

### 3. Списковые выражения — избежать повторного вызова функции

```python
from math import factorial

data = [1, 7, 10, 3, 23, 5, 12, 8, 6]

# ❌ factorial(x) вызывается дважды
print([factorial(x) for x in data if factorial(x) <= 1000])

# ✅ с :=  -- один вызов
print([result for x in data if (result := factorial(x)) <= 1000])
```

### 4. Проверка на None с присваиванием

```python
users = [{'name': 'Tim', 'occupation': 'guru'}, {'name': None, 'occupation': 'driver'}]

# ✅
for user in users:
    if (name := user.get('name')) is not None:
        print(f'{name} is a {user.get("occupation")}')
```

### 5. Сохранить результат проверки (if/elif с regex)

```python
import re

text = 'Курс Python от BEEGEEK'

if match := re.search(r'beegeek', text):
    print(f'Найдено: {match.group()}')
elif match := re.search(r'Python', text):
    print(f'Найдено: {match.group()}')
else:
    print('Нет совпадений')
```

### 6. any() / all() — сохранить значение, на котором остановились

```python
numbers = [1, 4, 6, 2, 12, 4, 15]

print(any((value := number) > 10 for number in numbers))  # True
print(value)  # 12 -- число, на котором any() остановился
```

## ⚠️ Подводные камни

### Переменная "утекает" из списочного выражения

Переменная, созданная через `:=` внутри списка/генератора, **остаётся доступна** в объемлющей области видимости — может перезаписать существующую переменную.

```python
from math import factorial

result = 0
print(result)  # 0

data = [1, 7, 10, 3]
factorial_data = [result for x in data if (result := factorial(x)) <= 1000]
print(result)  # 6 -- ❗ result перезаписан последним значением из списка
```

### NameError при пустой последовательности

Если генератор/итерируемый объект пуст — `:=` не выполнится ни разу, переменная **не будет создана**.

```python
numbers = []
print(any((value := number) > 10 for number in numbers))  # False
print(value)
# ❌ NameError: name 'value' is not defined
```

### NameError из-за короткого замыкания в условии

Если левая часть `and`/`or` ложна — правая часть с `:=` **не выполняется**, переменная не создаётся.

```python
for i in range(1, 101):
    if (two := i % 2 == 0) and (three := i % 3 == 0):
        print(f'{i} делится на 6')
    elif two:
        print(f'{i} делится на 2')
    elif three:        # ❌ NameError, если i=1 (two=False, three не создана)
        print(f'{i} делится на 3')
```

> [!danger] Главное правило
> `:=` создаёт переменную **только если выражение с ним было вычислено**. Короткое замыкание (`and`/`or`) и пустые итерируемые объекты могут "пропустить" создание переменной.

## 💡 Практические замечания

- `:=` появился в Python **3.8** (PEP 572).
- Используй `:=`, когда это **реально убирает дублирование** (повторный вызов функции, повторный `input()`).
- Не злоупотребляй — оператор противоречит принципам Zen of Python: *"Explicit is better than implicit"* и *"Simple is better than complex"*.
- Каскадное присваивание через `:=` — технически возможно, но нечитаемо:
  ```python
  (x := (y := (z := 0)))   # ❌ громоздко
  x = y = z = 0            # ✅ обычное присваивание лучше
  ```
- Если выражение с `:=` сложное — лучше разбить на две строки для читаемости:
  ```python
  # вместо
  print((x := 18 / (7 - 14 * 8)) > 0)
  # лучше
  x = 18 / (7 - 14 * 8)
  print(x > 0)
  ```
