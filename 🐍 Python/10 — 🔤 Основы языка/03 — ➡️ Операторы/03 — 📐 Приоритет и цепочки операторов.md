## 📊 Таблица приоритета

От наибольшего приоритета к наименьшему. Операторы в одной строке имеют равный приоритет и выполняются **слева направо** (кроме `**`).

| Приоритет | Операторы | Описание |
| --- | --- | --- |
| высший | `**` | возведение в степень |
| | `+x`, `-x`, `~x` | унарные плюс, минус, побитовое НЕ |
| | `*`, `/`, `//`, `%` | умножение и деление |
| | `+`, `-` | сложение и вычитание |
| | `<<`, `>>` | побитовые сдвиги |
| | `&` | побитовое И |
| | `^` | побитовое исключающее ИЛИ |
| | `\|` | побитовое ИЛИ |
| | `==`, `!=`, `>`, `<`, `>=`, `<=`, `in`, `not in`, `is`, `is not` | сравнение, принадлежность, тождественность |
| | `not` | логическое отрицание |
| | `and` | логическое И |
| | `or` | логическое ИЛИ |
| низший | `=` | присваивание |

## ➗ Примеры с приоритетом

```python
print(5 + 3 % 2 / 2 == 5.5)
# 1. % и / — наивысший приоритет здесь, выполняются слева направо: 3 % 2 = 1, 1 / 2 = 0.5
# 2. + выше == : 5 + 0.5 = 5.5
# 3. == : 5.5 == 5.5
# → True
```

**Важное:**
- операторы одинакового приоритета выполняются **слева направо**
- скобки `()` всегда выполняются первыми — используй их для явности

---

## 🔁 Правоассоциативность `**`

`**` — единственный оператор, который выполняется **справа налево**.

```python
print(2 ** 2 ** 3)
# справа налево: 2 ** 3 = 8, затем 2 ** 8 = 256
# → 256

print((2 ** 2) ** 3)
# скобки меняют порядок: 2 ** 2 = 4, затем 4 ** 3 = 64
# → 64
```

**Важное:** если нужен предсказуемый результат с несколькими `**` — используй скобки явно.

## 🔗 Цепочки сравнений

Несколько сравнений подряд — это **сокращённая запись**, где `and` используется неявно.

```python
a, b, c = 5, 10, 15

print(a < b <= c)          # True
print(a < b and b <= c)    # True — эквивалентная расширенная запись
```

`a < b <= c` ≡ `a < b and b <= c`

**Важное:** сокращённая запись — не просто синтаксический сахар, у неё есть отличие в **вычислении операндов** (см. ниже).

---

## ⚡ Короткое замыкание в цепочках

`and` останавливает вычисление цепочки на первом `False`. Операнды правее **не вычисляются**.

```python
def func():
    print('bee')
    return 3

if 5 < 1 < func():
    print('geek')
else:
    print('beegeek')

# 5 < 1 → False
# func() НЕ вызывается!
# → beegeek (без 'bee')
```

**Важное:** функции и выражения справа от первого ложного сравнения вообще не выполняются.

---

## 🔄 Сокращённая vs расширенная запись

**Ключевое отличие:** в сокращённой записи каждый операнд вычисляется **один раз**. В расширенной — может вычисляться **несколько раз**.

```python
def func():
    print('bee')
    return 3

# Сокращённая — func() вызывается ОДИН раз
if 1 < func() < 5:
    print('geek')
# вывод: bee \n geek

# Расширенная — func() вызывается ДВА раза
if 1 < func() and func() < 5:
    print('geek')
# вывод: bee \n bee \n geek
```

```python
from random import randint

def func():
    x = randint(1, 7)
    print(x)
    return x

print(1 < func() < 5)              # func() вызван 1 раз, оба сравнения с одним значением
print(1 < func() and func() < 5)   # func() вызван 2 раза — РАЗНЫЕ значения!
```

> [!warning] Опасность с непостоянными значениями
> Если `func()` возвращает **разные значения** при каждом вызове (случайное число, `input()`, изменяющийся атрибут), сокращённая и расширенная записи дают **разные результаты**. Используй сокращённую запись, когда нужна гарантия одного вызова.

## ➕ Транзитивность операторов

`==` — транзитивен: если `a == b` и `b == c`, то `a == c`. Цепочка работает корректно.

```python
a = b = c = 1
print(a == b == c)   # True
```

`!=` — **не транзитивен**. Цепочка `a != b != c` проверяет только `a != b` и `b != c`, но **не проверяет** `a != c`.

```python
a, b, c = 1, 2, 1

print(a != b != c)
# a != b → True (1 != 2)
# b != c → True (2 != 1)
# a != c НЕ проверяется!
# → True, хотя a == c (оба равны 1)
```

**✅ ВЕРНО — для проверки что все три значения различны:**

```python
print(a != b and a != c and b != c)
# → False (потому что a == c)
```

> [!warning] Не используй цепочки с `!=`
> Цепочка `a != b != c` **не гарантирует**, что все три значения попарно различны. Для этого нужно явное объединение через `and`.

## ⚠️ Смешивание операторов в цепочках

Объединение операторов сравнения с другими операторами может давать **неожиданный результат**.

```python
num = 3
lst = [3, 5]

print(num in lst == True)
# Кажется: (num in lst) == True → True == True → True
# На самом деле: num in lst AND lst == True
#                True        and  [3,5] == True
#                True        and  False
# → False  ❗
```

**Важное:** в цепочке `num in lst == True` неявно используется `and`, как и в обычных цепочках сравнений. `lst == True` — это сравнение списка с булевым значением, что всегда `False`.

> [!danger] Избегай смешанных цепочек
> Смешивание `in`, `is`, операторов сравнения и `==`/`!=` в одной цепочке снижает читаемость и может давать результат, противоречащий интуиции. Разбивай такие выражения на отдельные сравнения с явным `and`/`or`.

---

## 💡 Практические замечания

- Используй скобки `()` для явного порядка — особенно при смешивании `**`, `%`, `/`, сравнений и логики.
- Сокращённая запись (`a < b < c`) предпочтительна когда нужна гарантия одного вычисления каждого операнда.
- Никогда не строй цепочки с `!=` для проверки "все значения разные" — используй `and` явно или `len(set(...)) == len((...))`.
- Не смешивай `in`/`is` с `==`/`!=` в одной цепочке без скобок.
- Полная таблица приоритета — в [официальной документации Python](https://docs.python.org/3/reference/expressions.html#operator-precedence).
