# 07 — 📦 Памятка: Система импорта и пакеты Python

## 🎯 Суть в 30 секунд

**Модули** — файлы .py с кодом. **Пакеты** — папки с модулями и файлом `__init__.py`. Система импорта позволяет переиспользовать код из других модулей.

---

## 📌 Часть 1: Модули и импорт

### 🚀 Три типа импорта

```python
# 1️⃣ Импорт всего модуля
import math
result = math.sqrt(9)  # Обращаемся через имя модуля

# 2️⃣ Импорт конкретных объектов (РЕКОМЕНДУЕТСЯ)
from math import sqrt, pi
result = sqrt(9)  # Обращаемся напрямую
print(pi)

# 3️⃣ Импорт всех объектов (НЕ РЕКОМЕНДУЕТСЯ)
from math import *  # Непонятно, что импортировано
sqrt(9)  # Может быть конфликт имён!
```

**Вывод:** Предпочитай варианты 1️⃣ и 2️⃣. Вариант 3️⃣ затемняет код.

---

### 📊 Способы импорта — таблица

| Способ | Синтаксис | Использование | Когда |
|--------|-----------|---------------|-------|
| **Весь модуль** | `import math` | `math.sqrt(9)` | Когда нужно несколько объектов |
| **Конкретные объекты** | `from math import sqrt` | `sqrt(9)` | Когда используешь 1-3 объекта |
| **С псевдонимом** | `import math as m` | `m.sqrt(9)` | Когда имя модуля длинное |
| **Все объекты** | `from math import *` | `sqrt(9)` | ❌ НЕ ИСПОЛЬЗУЙ! |

---

### 💡 Примеры импорта встроенных модулей

```python
# ✅ ХОРОШО
import operator
from math import sqrt, cos, pi
from datetime import datetime as dt
from random import randint

# ❌ ПЛОХО
from math import *  # Что импортировано?
import numpy as n   # Нестандартный псевдоним
import pandas as p  # Непонятно что это
```

**Стандартные псевдонимы:**
```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
```

### 📈 Приоритет поиска модулей

Python ищет модули в этом порядке:

1. **Текущий каталог проекта** ← Сначала здесь!
2. **Стандартная библиотека Python**
3. **Другие каталоги** (виртуальное окружение, etc)

```python

import my_math
```

## 📌 Часть 3: Точка входа (`__name__`)

### 🎯 Что это?

`__name__` — встроенная переменная, которая показывает:
- `'__main__'` — если модуль запущен напрямую
- `'имя_модуля'` — если модуль импортирован

### 💻 Пример

```python
# beegeek.py
def greet():
    print('Привет!')

print(f"Я импортирован как: {__name__}")

if __name__ == '__main__':
    greet()  # Выполнится только при прямом запуске
```

**При запуске напрямую:**
```
$ python beegeek.py
Я импортирован как: __main__
Привет!
```

**При импорте:**
```python
import beegeek
# Вывод: Я импортирован как: beegeek
# (функция greet() НЕ вызывается)
```

### 🔥 Типичная схема

```python
# main.py
def main():
    print("Запущена основная программа")

if __name__ == '__main__':
    main()
```

**Преимущества:**
- ✅ Код выполняется только при прямом запуске
- ✅ Можно безопасно импортировать модуль в другой файл
- ✅ Избегаются случайные вызовы функций

## 📌 Часть 4: Пакеты Python

### 🏗️ Структура пакета

```
project/
├── greetings/           ← Это пакет
│   ├── __init__.py      ← ОБЯЗАТЕЛЕН для пакета!
│   ├── hello.py         ← Модули пакета
│   └── goodbye.py
│
└── main.py
```

**Пакет = папка с файлом `__init__.py`**

---

### 💻 Импорт из пакета

```python
# ❌ Это НЕ работает — Python не видит модули
import greetings
greetings.hello.greet()  # AttributeError!

# ✅ Вариант 1: Импорт модуля из пакета
import greetings.hello
greetings.hello.greet()

# ✅ Вариант 2: from import
from greetings import hello
hello.greet()

# ✅ Вариант 3: Прямой импорт функции
from greetings.hello import greet
greet()
```

---

### 🔧 Файл `__init__.py` — настройка пакета

#### ❌ Пустой `__init__.py`
```python
# greetings/__init__.py
# Пусто!
```

```python
# main.py
import greetings
greetings.hello.greet()  # AttributeError!
```

#### ✅ Настроенный `__init__.py`

**Вариант 1: Абсолютный импорт**
```python
# greetings/__init__.py
import greetings.hello
import greetings.goodbye
```

**Вариант 2: Относительный импорт** (рекомендуется)
```python
# greetings/__init__.py
from . import hello      # . = текущий пакет
from . import goodbye
```

**Вариант 3: Прямой импорт функций**
```python
# greetings/__init__.py
from .hello import greet
from .goodbye import bye
```

```python
# main.py
from greetings import greet, bye
greet()
bye()
```

---

### 📊 Абсолютный vs Относительный импорт

| | **Абсолютный** | **Относительный** |
|---|---|---|
| **Синтаксис** | `from greetings import hello` | `from . import hello` |
| **Путь** | Полное имя пакета | Относительно текущего |
| **`.`** | Не используется | Текущий пакет |
| **`..`** | Не используется | Родительский пакет |
| **Использование** | Импорт из других пакетов | Импорт в `__init__.py` |

### 🌳 Вложенные пакеты

```
project/
└── app/
    ├── __init__.py

    ├── utils/
```

## 📌 Часть 5: Переменная `__all__`

### 🎯 Что это?

`__all__` — список имён объектов, которые импортируются при `from ... import *`

### 💻 Пример

```python
# greetings/__init__.py
from .hello import greet
from .goodbye import bye

__all__ = ['greet']  # Только greet доступен через import *
```

```python
# main.py
from greetings import *

greet()     # ✅ Работает
bye()       # ❌ NameError! bye не в __all__
```

## 🔥 Частые ошибки

### ❌ Ошибка 1: Циклический импорт

```python
# main.py
import beegeek
beegeek.greet()

# beegeek.py
import main  # ← Циклический импорт!

def greet():
    print('Привет!')
```

**Результат:** `AttributeError: partially initialized module`

**Решение:**
```python
# beegeek.py - используй if __name__
import main

def greet():
    print('Привет!')

if __name__ == '__main__':
    main.something()
```

### ❌ Ошибка 2: Импорт из пакета без `__init__.py`

```python
# greetings/ (без __init__.py)
# hello.py, goodbye.py

# main.py
import greetings
greetings.hello.greet()  # AttributeError!
```

**Решение:** Добавь `greetings/__init__.py`

### ❌ Ошибка 3: Неправильный относительный импорт

```python
# greetings/__init__.py
import hello  # ❌ ModuleNotFoundError: No module named 'hello'

from . import hello  # ✅ Правильно
from .hello import greet  # ✅ Правильно тоже
```

### ❌ Ошибка 4: `from ... import *` и конфликты имён

```python
def format():
    return "my format"

from datetime import *  # ← import * с функцией format()!
format()  # ❌ Вызовет встроенную format(), не твою!
```

**Решение:** Используй `import datetime` или конкретный импорт

## 💎 Лучшие практики

### ✅ DO

```python
# main.py

# 1. Импорты в начале файла
import os
import sys
from datetime import datetime
from my_module import my_function

# 2. Встроенные модули → сторонние → собственные
import json
import requests
from app import utils

# 3. Используй if __name__ == '__main__'
def main():
    print("Программа запущена")

if __name__ == '__main__':
    main()

# 4. Явные импорты в __init__.py
# __init__.py
from .hello import greet
from .goodbye import bye
```

### ❌ DON'T

```python
# 1. Импорты в середине кода
def func():
    import math  # Плохо

# 2. Импорт *
from module import *  # Непонятно что импортировано

# 3. Циклические импорты
# file1.py
import file2
# file2.py
import file1  # ← Циклический!

# 4. Странные псевдонимы
import numpy as n  # n? Что это?
import pandas as p  # p? Неясно
```

## ✅ Шпаргалка по синтаксису

```python
# Встроенные модули
import math
from datetime import datetime

# Собственные модули
import my_module
from my_module import my_function

# Пакеты
import app.utils
from app import utils
from app.utils import helper
from app.utils.helper import format_text

# С псевдонимами
import numpy as np
from datetime import datetime as dt

# Относительный импорт (только в пакетах)
from . import sibling_module      # Текущий уровень
from .subpackage import module    # Подпакет
from .. import parent_module      # Родительский пакет

# Точка входа
if __name__ == '__main__':
    main()

# Управление импортом *
__all__ = ['public_function', 'PublicClass']
```
