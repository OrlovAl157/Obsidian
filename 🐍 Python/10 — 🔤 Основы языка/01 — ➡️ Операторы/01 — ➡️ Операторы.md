## Справка по операторам

| Группа | Операторы | Что делает |
| --- | --- | --- |
| Арифметика | `+`, `-`, `*`, `/`, `//`, `%`, `**` | Математические операции с числами. |
| Сравнение | `==`, `!=`, `>`, `<`, `>=`, `<=` | Сравнивает значения объектов. |
| Логика | `and`, `or`, `not` | Объединяет условия, возвращает операнды. |
| Присваивание | `=`, `+=`, `-=`, `*=` и др. | Присваивает и изменяет значения переменных. |
| Принадлежность | `in`, `not in` | Проверяет наличие элемента в наборе. |
| Тождественность | `is`, `is not` | Сравнивает идентификаторы объектов (адреса в памяти). |
| Битовые | `&`, `\|`, `^`, `~`, `<<`, `>>` | Побитовые операции, также операции над множествами. |

## ➕ Арифметические операторы

**Назначение:** выполнить математическую операцию с числами.

| Оператор | Значение | Пример | Результат |
| --- | --- | --- | --- |
| `+` | сложение | `12 + 2` | `14` |
| `-` | вычитание | `12 - 2` | `10` |
| `*` | умножение | `12 * 2` | `24` |
| `/` | деление | `12 / 2` | `6.0` |
| `//` | целочисленное деление | `12 // 2` | `6` |
| `%` | остаток от деления | `12 % 2` | `0` |
| `**` | возведение в степень | `12 ** 2` | `144` |

**Примеры:**

```python
print(12 / 2)    # 6.0  -- результат всегда float
print(12 // 2)   # 6    -- результат int (если оба операнда int)

# Унарные операторы
print(+7)        # 7   -- без изменений
print(-7)        # -7  -- меняет знак
print(-(-7))     # 7
```

**Важное:** `/` всегда возвращает `float`, даже если делится без остатка. Для целочисленного результата используй `//`.

## ⚖️ Операторы сравнения

**Назначение:** сравнить значения объектов, результат — всегда `True` или `False`.

| Оператор | Значение | Пример | Результат |
| --- | --- | --- | --- |
| `==` | равно | `67 == 53` | `False` |
| `!=` | не равно | `67 != 53` | `True` |
| `>` | больше | `67 > 53` | `True` |
| `<` | меньше | `67 < 53` | `False` |
| `>=` | больше или равно | `67 >= 53` | `True` |
| `<=` | меньше или равно | `67 <= 53` | `False` |

**Примеры:**

```python
print(67 == '53')   # False -- разные типы, сравнение допустимо
print(67 > '53')    # TypeError! -- сравнение > между int и str невозможно

# Цепочки сравнений
age = 25
print(18 <= age < 65)   # True
```

**Важное:** сравнение объектов разных типов через `>`, `<`, `>=`, `<=` обычно вызывает `TypeError`. Через `==` и `!=` — работает, но просто возвращает `False`/`True`.

## 🔗 Логические операторы

**Назначение:** объединять условия и работать с истинностью объектов. Операнды — любые объекты Python, не только `bool`.

| Оператор | Что делает | Что возвращает |
| --- | --- | --- |
| `not` | инвертирует истинность операнда | `True` / `False` |
| `and` | первый falsy-операнд, либо последний | сам объект |
| `or` | первый truthy-операнд, либо последний | сам объект |

**Примеры:**

```python
print(not 0)          # True
print(not 'beegeek')  # False

# and возвращает первый falsy-объект или последний
print([] and () and 0)              # []
print([1, 2, 3] and (1, 2, 3) and 1)  # 1

# or возвращает первый truthy-объект или последний
print([1, 2, 3] or (1, 2, 3) or 1)  # [1, 2, 3]
print([] or () or 0)                # 0
```

**Важное:** `and`/`or` — это не только `True`/`False`. Они работают по принципу **короткого замыкания** и возвращают сам объект, на котором остановились. Это основа паттерна `x = a or default`.

## 🟰 Операторы присваивания

**Назначение:** присвоить или изменить значение переменной.

| Оператор | Эквивалент | Пример |
| --- | --- | --- |
| `=` | простое присваивание | `x = 5` |
| `+=` | `x = x + 3` | `x += 3` |
| `-=` | `x = x - 3` | `x -= 3` |
| `*=` | `x = x * 3` | `x *= 3` |
| `/=` | `x = x / 3` | `x /= 3` |
| `//=` | `x = x // 3` | `x //= 3` |
| `%=` | `x = x % 3` | `x %= 3` |
| `**=` | `x = x ** 3` | `x **= 3` |
| `&=`, `\|=`, `^=`, `<<=`, `>>=` | составное присваивание для битовых операторов | `x &= 3` |

**Виды присваивания:**

```python
# Простое
num = 3.14

# Позиционное -- количество значений и переменных должно совпадать
num1, num2 = 3.14, 2.72

# Каскадное -- все переменные получают одно значение
num1 = num2 = 2.72
```

**Примеры:**

```python
num = 7
num += 3   # 10, эквивалент num = num + 3
num &= 3   # 2,  эквивалент num = num & 3
```

```python
# ❗ Каскадное присваивание с изменяемыми объектами -- общая ссылка!
lst1 = lst2 = []
lst1.append('beegeek')

print(lst1)  # ['beegeek']
print(lst2)  # ['beegeek']  -- изменился тоже, это один и тот же объект
```

**Важное:** при каскадном присваивании изменяемого объекта (`list`, `dict`, `set`) все переменные ссылаются на **один и тот же объект**. Изменение через одну переменную видно через все остальные.

## 📦 Операторы принадлежности

**Назначение:** проверить, входит ли значение в набор значений.

| Оператор | Что делает | Пример | Результат |
| --- | --- | --- | --- |
| `in` | значение есть в наборе | `2 in [1, 2, 3]` | `True` |
| `not in` | значения нет в наборе | `2 not in [1, 2, 3]` | `False` |

**Примеры:**

```python
print(2 in [1, 2, 3])         # True
print('bee' in 'beegeek')     # True
print('bee' not in 'beegeek') # False
```

**Важное:** скорость `in`/`not in` зависит от типа данных. Поиск в `set` происходит намного быстрее, чем в `list`, за счёт внутренней структуры хеш-таблицы.

---
## 🪪 Операторы тождественности

**Назначение:** проверить, являются ли два объекта **одним и тем же объектом** в памяти (а не равными по значению).

| Оператор | Что делает | Пример | Результат |
| --- | --- | --- | --- |
| `is` | объекты идентичны (один адрес) | `[1,2] is [1,2]` | `False` |
| `is not` | объекты не идентичны | `[1,2] is not [1,2]` | `True` |

**Примеры:**

```python
print([1, 2, 3] == [1, 2, 3])  # True  -- одинаковые значения
print([1, 2, 3] is [1, 2, 3])  # False -- разные объекты в памяти

# Малые целые числа кэшируются Python
x = 5
y = 5
print(x is y)  # True

x = 1000
y = 1000
print(x is y)  # False -- не гарантировано, зависит от реализации
```

**Важное:** `==` сравнивает **значения**, `is` сравнивает **идентификаторы (адреса в памяти)**. Для проверки на `None` используй `is None`, а не `== None`. Операторы `is`/`is not` не определяются магическими методами — сравнение адресов происходит напрямую.

## 🔢 Битовые операторы

**Назначение:** выполнить побитовые вычисления с целыми числами. Все, кроме `~`, бинарные.

| Оператор | Значение | Пример | Результат |
| --- | --- | --- | --- |
| `&` | побитовое И | `5 & 17` | `1` |
| `\|` | побитовое ИЛИ | `5 \| 17` | `21` |
| `^` | побитовое исключающее ИЛИ | `5 ^ 17` | `20` |
| `~` | побитовое НЕ (унарный) | `~5` | `-6` |
| `<<` | сдвиг влево | `5 << 17` | `655360` |
| `>>` | сдвиг вправо | `5 >> 17` | `0` |

**Примеры:**

```python
print(5 & 17)   # 1
print(5 | 17)   # 21
print(5 ^ 17)   # 20
print(~5, ~17)  # -6 -18
print(5 << 1)   # 10  -- умножение на 2
print(20 >> 2)  # 5   -- деление на 4
```

**Важное:** `<<` эквивалентно умножению на `2^n`, `>>` — целочисленному делению на `2^n`. `~x` равно `-x - 1`.

## 🎭 Операторы с коллекциями

**Назначение:** некоторые операторы меняют смысл при работе с `list`, `tuple`, `str`, `set`.

| Оператор | Тип данных | Что делает | Пример | Результат |
| --- | --- | --- | --- | --- |
| `+` | `str`, `list`, `tuple` | конкатенация | `[1,2] + [3,4]` | `[1,2,3,4]` |
| `*` | `str`, `list`, `tuple` | повторение | `[1,2,3] * 2` | `[1,2,3,1,2,3]` |
| `-` | `set` | разность множеств | `{1,2,3} - {3,4,5}` | `{1, 2}` |
| `&` | `set` | пересечение множеств | `{1,2,3} & {3,4,5}` | `{3}` |
| `\|` | `set` | объединение множеств | `{1,2,3} \| {3,4,5}` | `{1,2,3,4,5}` |
| `^` | `set` | симметрическая разность | `{1,2,3} ^ {3,4,5}` | `{1,2,4,5}` |

**Примеры:**

```python
print('bee' + 'geek')        # beegeek
print([1, 2] + [3, 4])       # [1, 2, 3, 4]
print((1, 2) + (3, 4))       # (1, 2, 3, 4)
print('beegeek' * 2)         # beegeekbeegeek
print({1, 2, 3} - {3, 4, 5}) # {1, 2}

set1 = {1, 2, 3}
set2 = {3, 4, 5}
print(set1 & set2)  # {3}            -- пересечение
print(set1 | set2)  # {1,2,3,4,5}    -- объединение
print(set1 ^ set2)  # {1,2,4,5}      -- симметрическая разность
```

**Важное:** битовые операторы `&`, `|`, `^` для `set` означают теоретико-множественные операции (пересечение, объединение, симметрическая разность), а не побитовую арифметику.

## 🪄 Магические методы операторов

**Назначение:** чтобы пользовательский класс поддерживал оператор, нужно реализовать соответствующий магический метод.

### Арифметические

| Оператор | Метод | Отражённый метод |
| --- | --- | --- |
| `+` | `__add__` | `__radd__` |
| `-` | `__sub__` | `__rsub__` |
| `*` | `__mul__` | `__rmul__` |
| `/` | `__truediv__` | `__rtruediv__` |
| `//` | `__floordiv__` | `__rfloordiv__` |
| `%` | `__mod__` | `__rmod__` |
| `**` | `__pow__` | `__rpow__` |
| унарный `+` | `__pos__` | — |
| унарный `-` | `__neg__` | — |

### Битовые

| Оператор | Метод | Отражённый метод |
| --- | --- | --- |
| `&` | `__and__` | `__rand__` |
| `\|` | `__or__` | `__ror__` |
| `^` | `__xor__` | `__rxor__` |
| `<<` | `__lshift__` | `__rlshift__` |
| `>>` | `__rshift__` | `__rrshift__` |
| `~` | `__invert__` | — |

### Сравнение

| Оператор | Метод |
| --- | --- |
| `==` | `__eq__` |
| `!=` | `__ne__` |
| `>` | `__gt__` |
| `<` | `__lt__` |
| `>=` | `__ge__` |
| `<=` | `__le__` |

### Составное присваивание (i-методы)

| Оператор | Метод |
| --- | --- |
| `+=` | `__iadd__` |
| `-=` | `__isub__` |
| `*=` | `__imul__` |
| `/=` | `__itruediv__` |
| `//=` | `__ifloordiv__` |
| `%=` | `__imod__` |
| `**=` | `__ipow__` |
| `&=`, `\|=`, `^=`, `<<=`, `>>=` | `__iand__`, `__ior__`, `__ixor__`, `__ilshift__`, `__irshift__` |

### Принадлежность и логика

| Оператор | Метод | Что важно |
| --- | --- | --- |
| `in` / `not in` | `__contains__` | определяет проверку принадлежности |
| `not` | `__bool__`, иначе `__len__` | если ни один не определён — объект всегда truthy |
| `and`, `or`, `is`, `is not` | — | не имеют магических методов |

**Важное:**
- `and`/`or` не имеют магических методов — короткое замыкание невозможно выразить через функции.
- `is`/`is not` сравнивают адреса напрямую, без вызова методов класса.
- Отражённые методы (`__radd__` и т.д.) вызываются, когда левый операнд не поддерживает операцию с правым.

> [!tip] @total_ordering
> Декоратор `@total_ordering` из `functools` достраивает все операторы сравнения, если реализованы `__eq__` и один из `__lt__`/`__le__`/`__gt__`/`__ge__`. Несмотря на это, рекомендуется определять все методы сравнения явно.

## 📊 Приоритет операторов

От высшего приоритета к низшему:

| № | Операторы |
| --- | --- |
| 1 | `()` — скобки |
| 2 | `**` — степень |
| 3 | `+x`, `-x`, `~x` — унарные |
| 4 | `*`, `/`, `//`, `%` |
| 5 | `+`, `-` |
| 6 | `<<`, `>>` |
| 7 | `&` |
| 8 | `^` |
| 9 | `\|` |
| 10 | `==`, `!=`, `>`, `<`, `>=`, `<=`, `is`, `in` |
| 11 | `not` |
| 12 | `and` |
| 13 | `or` |
| 14 | `if / else` (тернарный) |

**Примеры:**

```python
print(2 + 3 * 4)    # 14  -- умножение первое
print((2 + 3) * 4)  # 20  -- скобки первые
print(2 ** 3 ** 2)  # 512 -- степень правоассоциативна: 2**(3**2)
```

**Важное:** `**` вычисляется **справа налево** — это единственный право-ассоциативный оператор в таблице.
