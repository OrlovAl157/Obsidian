# Памятка по рекурсии в Python

## Пример для быстрой справки

```python
def factorial(n):
    # Базовый случай (условие остановки)
    if n == 0 or n == 1:
        return 1
    # Рекурсивный случай
    return n * factorial(n - 1)

print(factorial(5))  # 120
# Шаги: 5 * 4 * 3 * 2 * 1 = 120
```

## Быстрая справка

|Элемент                |Описание                  |Пример                     |
|-----------------------|--------------------------|---------------------------|
|**Базовый случай**     |Условие остановки         |`if n == 0: return 1`      |
|**Рекурсивный случай** |Вызов самой себя          |`return n * factorial(n-1)`|
|**Уменьшение задачи**  |Движение к базовому случаю|`n - 1`, `n // 2`          |
|**Глубина рекурсии**   |Максимум вложенных вызовов|По умолчанию ~1000         |
|**Ограничение глубины**|`sys.setrecursionlimit(N)`|Изменить лимит             |

## Что такое рекурсия?

**Рекурсия** — это когда функция вызывает саму себя для решения задачи, разбивая её на более простые подзадачи.

### Структура рекурсивной функции

```python
def recursive_function(параметры):
    # 1. Базовый случай (условие остановки)
    if базовое_условие:
        return базовое_значение
    
    # 2. Рекурсивный случай
    return recursive_function(измененные_параметры)
```

**Обязательные элементы:**

1. ✅ **Базовый случай** — условие остановки рекурсии
2. ✅ **Рекурсивный случай** — вызов функции с упрощенной задачей
3. ✅ **Движение к базовому случаю** — параметры должны приближаться к базовому случаю

## Базовые примеры

### Факториал

```python
def factorial(n):
    """
    n! = n × (n-1) × (n-2) × ... × 1
    5! = 5 × 4 × 3 × 2 × 1 = 120
    """
    # Базовый случай
    if n == 0 or n == 1:
        return 1
    
    # Рекурсивный случай
    return n * factorial(n - 1)

print(factorial(5))  # 120
print(factorial(0))  # 1
```

**Как это работает:**

```
factorial(5)
= 5 * factorial(4)
= 5 * (4 * factorial(3))
= 5 * (4 * (3 * factorial(2)))
= 5 * (4 * (3 * (2 * factorial(1))))
= 5 * (4 * (3 * (2 * 1)))
= 5 * (4 * (3 * 2))
= 5 * (4 * 6)
= 5 * 24
= 120
```

```python
def find_max(lst):
    """Находит максимальный элемент"""
    # Базовый случай
    if len(lst) == 1:
        return lst[0]
    
    # Рекурсивный случай
    max_rest = find_max(lst[1:])
    return lst[0] if lst[0] > max_rest else max_rest
```

```python
    """Алгоритм Евклида"""
    # Базовый случай
```

### Быстрое возведение в степень

```python
def fast_power(base, exp):
    """Быстрое возведение в степень O(log n)"""
    # Базовые случаи
    if exp == 0:
        return 1
    if exp == 1:
        return base
    
    # Рекурсивный случай
    if exp % 2 == 0:
        half = fast_power(base, exp // 2)
        return half * half
    else:
        return base * fast_power(base, exp - 1)

print(fast_power(2, 10))  # 1024
```

### Бинарный поиск

```python
def binary_search(arr, target, left=0, right=None):
    """Бинарный поиск в отсортированном массиве"""
    if right is None:
        right = len(arr) - 1
    
    # Базовый случай: элемент не найден
    if left > right:
        return -1
    
    mid = (left + right) // 2
    
    # Базовый случай: элемент найден
    if arr[mid] == target:
        return mid
    
    # Рекурсивные случаи
    if arr[mid] > target:
        return binary_search(arr, target, left, mid - 1)
    else:
        return binary_search(arr, target, mid + 1, right)

arr = [1, 3, 5, 7, 9, 11, 13]
print(binary_search(arr, 7))   # 3 (индекс элемента 7)
print(binary_search(arr, 10))  # -1 (не найден)
```

```python
def sum_nested(data):
    """Сумма всех чисел во вложенной структуре"""
    # Базовый случай
    if isinstance(data, int) or isinstance(data, float):
        return data
    
    # Рекурсивный случай
    total = 0
```

## Рекурсия vs Итерация

### Рекурсия

```python
def factorial_recursive(n):
    if n <= 1:
        return 1
    return n * factorial_recursive(n - 1)
```

**Плюсы:**

- ✅ Элегантный код
- ✅ Естественное решение для древовидных структур
- ✅ Проще понять логику

**Минусы:**

- ❌ Расход памяти (стек вызовов)
- ❌ Медленнее
- ❌ Ограничение глубины

### Итерация

```python
def factorial_iterative(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result
```

**Плюсы:**

- ✅ Быстрее
- ✅ Меньше памяти
- ✅ Нет ограничения глубины

**Минусы:**

- ❌ Может быть сложнее для понимания
- ❌ Больше кода для сложных структур

### Сравнительная таблица

|Критерий       |Рекурсия                       |Итерация            |
|---------------|-------------------------------|--------------------|
|**Читаемость** |✅ Высокая для древовидных задач|⚠️ Средняя           |
|**Память**     |❌ Больше (стек)                |✅ Меньше            |
|**Скорость**   |❌ Медленнее                    |✅ Быстрее           |
|**Ограничения**|❌ Глубина ~1000                |✅ Нет               |
|**Код**        |✅ Компактный                   |⚠️ Может быть длиннее|

## Оптимизация рекурсии

### Мемоизация (кэширование)

**Мемоизация** — это техника оптимизации, при которой результаты вызовов функции сохраняются, чтобы избежать повторных вычислений.

#### Проблема без мемоизации

```python
# ❌ Без мемоизации (очень медленно)
def fibonacci_slow(n):
    if n <= 1:
        return n
    return fibonacci_slow(n - 1) + fibonacci_slow(n - 2)

# fibonacci_slow(35) вызовет функцию ~18 миллионов раз!
# Время выполнения: ~5 секунд
```

**Почему медленно?** Множество повторных вычислений:

```
fibonacci(5)
├── fibonacci(4)
│   ├── fibonacci(3)
│   │   ├── fibonacci(2)  ← Вычисляется много раз
│   │   └── fibonacci(1)
│   └── fibonacci(2)      ← Повторное вычисление!
└── fibonacci(3)          ← Еще раз повторное!
    ├── fibonacci(2)
    └── fibonacci(1)
```

#### Вариант 1: Мемоизация через словарь вручную

```python
# ✅ С кэшем (быстро)
def fibonacci_cached(n, cache=None):
    # Инициализация кэша при первом вызове
    if cache is None:
        cache = {}
    
    # Проверяем, есть ли результат в кэше
    if n in cache:
        return cache[n]
    
    # Базовые случаи
    if n <= 1:
        return n
    
    # Вычисляем и сохраняем в кэш
    result = fibonacci_cached(n - 1, cache) + fibonacci_cached(n - 2, cache)
    cache[n] = result
    
    return result

print(fibonacci_cached(35))  # Мгновенно!
print(fibonacci_cached(100))  # Тоже мгновенно!
```

#### Вариант 2: Использование метода get() словаря

```python
# ✅ С использованием dict.get()
def fibonacci_get(n, cache=None):
    if cache is None:
        cache = {}
    
    # Используем get() для проверки наличия в кэше
    cached_value = cache.get(n)
    if cached_value is not None:
        return cached_value
    
    # Базовые случаи
    if n <= 1:
        cache[n] = n
        return n
    
    # Вычисляем и сохраняем
    result = fibonacci_get(n - 1, cache) + fibonacci_get(n - 2, cache)
    cache[n] = result
    
    return result

print(fibonacci_get(35))  # Мгновенно!
```

#### Вариант 4: Декоратор для мемоизации (продвинутый)

```python
def memoize(func):
    """Декоратор для мемоизации любой функции"""
    cache = {}
    
    def wrapper(*args):
        # args должны быть хешируемыми (tuple подходит)
        if args in cache:
            return cache[args]
        
        result = func(*args)
        cache[args] = result
        return result
    
    return wrapper

@memoize
def fibonacci_decorated(n):
    if n <= 1:
        return n
    return fibonacci_decorated(n - 1) + fibonacci_decorated(n - 2)

print(fibonacci_decorated(35))  # Мгновенно!
```

```python
def coin_change(amount, coins, cache=None):
    """Минимальное количество монет для размена"""
    if cache is None:
        cache = {}
    
    if amount in cache:
        return cache[amount]
    
    if amount == 0:
        return 0
    if amount < 0:
        return float('inf')
    
    min_coins = float('inf')
    for coin in coins:
        result = coin_change(amount - coin, coins, cache)
        if result != float('inf'):
            min_coins = min(min_coins, result + 1)
    
    cache[amount] = min_coins
    return min_coins
```

### Хвостовая рекурсия

**Хвостовая рекурсия** — это особый вид рекурсии, когда рекурсивный вызов является **последней операцией** в функции, и результат этого вызова сразу возвращается без дополнительных вычислений.

#### Теория

**Обычная рекурсия:**

```python
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)  # ← Операция умножения ПОСЛЕ рекурсии
```

Стек вызовов:

```
factorial(4)
├── 4 * factorial(3)      # Ждем результата
│   ├── 3 * factorial(2)  # Ждем результата
│   │   ├── 2 * factorial(1)  # Ждем результата
│   │   │   └── 1         # Возвращаем
│   │   └── 2 * 1 = 2     # Вычисляем
│   └── 3 * 2 = 6         # Вычисляем
└── 4 * 6 = 24            # Вычисляем
```

**Хвостовая рекурсия:**

```python
def factorial_tail(n, accumulator=1):
    if n <= 1:
        return accumulator
    return factorial_tail(n - 1, n * accumulator)  # ← Рекурсия в конце, без операций
```

Стек вызовов (теоретически может быть оптимизирован):

```
factorial_tail(4, 1)
└── factorial_tail(3, 4)   # 4 * 1 = 4
    └── factorial_tail(2, 12)  # 3 * 4 = 12
        └── factorial_tail(1, 24)  # 2 * 12 = 24
            └── return 24          # n <= 1
```

**Ключевое отличие:** В хвостовой рекурсии все вычисления происходят **до** рекурсивного вызова через аккумулятор.

**⚠️ Важно:** Python **НЕ оптимизирует** хвостовую рекурсию автоматически! Стек все равно растет. Но это полезная техника для понимания и в других языках (Scheme, Haskell).

```python
print(fibonacci_tail(10))  # 55

# Пошаговое выполнение для fibonacci_tail(5):
# fibonacci_tail(5, 0, 1)
# fibonacci_tail(4, 1, 1)  # a=1, b=0+1=1
# fibonacci_tail(3, 1, 2)  # a=1, b=1+1=2
# fibonacci_tail(2, 2, 3)  # a=2, b=1+2=3
# fibonacci_tail(1, 3, 5)  # a=3, b=2+3=5
# return 5
```

**Переворот списка:**

```python
# Обычная рекурсия
def reverse_regular(lst):
    if len(lst) <= 1:
        return lst
    return [lst[-1]] + reverse_regular(lst[:-1])

# Хвостовая рекурсия
def reverse_tail(lst, accumulator=None):
    if accumulator is None:
        accumulator = []
    
    if len(lst) == 0:
        return accumulator
    
    return reverse_tail(lst[1:], [lst[0]] + accumulator)

print(reverse_tail([1, 2, 3, 4, 5]))  # [5, 4, 3, 2, 1]
```

**Длина списка:**

```python
# Обычная рекурсия
def length_regular(lst):
    if len(lst) == 0:
        return 0
    return 1 + length_regular(lst[1:])  # Сложение после вызова

# Хвостовая рекурсия
def length_tail(lst, count=0):
    if len(lst) == 0:
        return count
    return length_tail(lst[1:], count + 1)  # Вызов в конце

print(length_tail([1, 2, 3, 4, 5]))  # 5
```

#### Признаки хвостовой рекурсии

✅ **Это хвостовая рекурсия:**

```python
return функция(новые_параметры)
```

❌ **Это НЕ хвостовая рекурсия:**

```python
return выражение + функция(параметры)
return выражение * функция(параметры)
x = функция(параметры)
return обработка(x)
```

#### Преобразование в хвостовую рекурсию

**Шаги:**

1. Добавить параметр-аккумулятор
2. Переместить все вычисления в аргументы рекурсивного вызова
3. Возвращать аккумулятор в базовом случае
4. Возвращать результат рекурсивного вызова напрямую

**Пример:**

```python
# ДО: Обычная рекурсия
def sum_list(lst):
    if not lst:
        return 0
    return lst[0] + sum_list(lst[1:])  # Сложение после

# ПОСЛЕ: Хвостовая рекурсия
def sum_list_tail(lst, accumulator=0):
    if not lst:
        return accumulator
    return sum_list_tail(lst[1:], accumulator + lst[0])  # Сложение до
```

#### Сравнение: обычная vs хвостовая

|Критерий                       |Обычная       |Хвостовая              |
|-------------------------------|--------------|-----------------------|
|**Вычисления**                 |После возврата|До рекурсивного вызова |
|**Аккумулятор**                |Не нужен      |Нужен                  |
|**Читаемость**                 |Часто проще   |Может быть сложнее     |
|**Оптимизация в Python**       |Нет           |Нет (не оптимизируется)|
|**Оптимизация в других языках**|Нет           |✅ Да (Scheme, Haskell) |

**Примечание:** В Python хвостовая рекурсия используется в основном для **понимания концепции** и **практики написания кода**. Реальной оптимизации стека она не дает.

## Когда использовать рекурсию?

### ✅ Используйте когда:

1. **Древовидные структуры**
   
   ```python
   # Обход дерева файлов, DOM, AST
   ```
2. **Задачи “разделяй и властвуй”**
   
   ```python
   # Быстрая сортировка, бинарный поиск
   ```
3. **Математические определения**
   
   ```python
   # Факториал, Фибоначчи, НОД
   ```
4. **Рекурсивные структуры данных**
   
   ```python
   # Вложенные списки/словари
   ```

### ❌ Не используйте когда:

1. **Простые итерации**
   
   ```python
   # ❌ Плохо
   def sum_n(n):
       if n == 0:
           return 0
       return n + sum_n(n - 1)
   
   # ✅ Хорошо
   def sum_n(n):
       return sum(range(n + 1))
   ```
2. **Глубокая рекурсия**
   
   ```python
   # Если n > 1000, будет RecursionError
   ```
3. **Производительность критична**
   
   ```python
   # Итерация быстрее
   ```

## Распространенные ошибки

### ❌ Ошибка 1: Отсутствие базового случая

```python
# ❌ Бесконечная рекурсия
def bad_factorial(n):
    return n * bad_factorial(n - 1)  # RecursionError!

# ✅ Правильно
def good_factorial(n):
    if n <= 1:  # Базовый случай!
        return 1
    return n * good_factorial(n - 1)
```

### ❌ Ошибка 2: Не приближается к базовому случаю

```python
# ❌ Всегда вызывает с тем же n
def bad_countdown(n):
    if n == 0:
        return
    print(n)
    bad_countdown(n)  # n не уменьшается!

# ✅ Правильно
def good_countdown(n):
    if n == 0:
        return
    print(n)
    good_countdown(n - 1)  # n уменьшается
```

### ❌ Ошибка 3: Неэффективная рекурсия

```python
# ❌ Медленный Фибоначчи (экспоненциальная сложность)
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

# ✅ С мемоизацией
from functools import lru_cache

@lru_cache(maxsize=None)
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

```python
def factorial(n, depth=0):
    indent = "  " * depth
    print(f"{indent}factorial({n})")
    
    if n <= 1:
        print(f"{indent}return 1")
        return 1
    
    result = n * factorial(n - 1, depth + 1)
    print(f"{indent}return {result}")
    return result

factorial(4)
```
