# 🔍 Области видимости в Python - Полная памятка

## 💡 Главное правило

**В Python область видимости создаётся ТОЛЬКО:**

1. **Функциями** (в явном или скрытом виде)
2. **Модулями**

-----

## ✅ Где создаётся область видимости

### 1. def — Обычная функция

**ДА, создаёт**

```python
def f():
    x = 1  # Локальная переменная
    print(x)

f()  # 1
# print(x)  # NameError: x не существует вне функции
```

**Ключевые моменты:**

- Новая область создаётся при **каждом вызове** функции
- Локальные переменные недоступны снаружи
- Подчиняется правилу LEGB

-----

### 2. lambda — Лямбда-функция

**ДА, создаёт**

```python
f = lambda x: x + 1

# У лямбды своя локальная область
g = lambda x: (y := x + 1, y * 2)[1]  # y локальна

print(f(5))  # 6
# print(x)  # NameError
```

**Ключевые моменты:**

- Лямбда — это обычная функция
- Имеет собственную локальную область
- Подчиняется тем же правилам LEGB
- Переменные внутри lambda не видны снаружи

-----

### 3. Вложенная функция (def внутри def)

**ДА, создаёт**

```python
def outer():
    x = 'outer'
    
    def inner():
        x = 'inner'  # Своя локальная x
        print(x)
    
    inner()  # inner
    print(x)  # outer

outer()
```

**Ключевые моменты:**

- `inner` создаёт свою локальную область
- `outer` — enclosing-область для `inner`
- Каждая функция независима

**Пример с замыканием:**

```python
def outer():
    x = 10
    
    def inner():
        print(x)  # Доступ к enclosing
    
    return inner

f = outer()
f()  # 10
```

-----

### 4. Генераторная функция (yield)

**ДА, создаёт**

```python
def gen():
    x = 1
    yield x
    x = 2
    yield x

g = gen()
print(next(g))  # 1
print(next(g))  # 2
# print(x)  # NameError
```

**Ключевые моменты:**

- Это функция → значит, отдельная область
- Область живёт, пока жив генератор
- Локальные переменные сохраняются между вызовами `next()`

-----

### 5. async def — Асинхронная функция

**ДА, создаёт**

```python
async def f():
    x = 1
    return x

# x недоступна снаружи
```

**Ключевые моменты:**

- Абсолютно та же область видимости, что у обычной функции
- Работает по тем же правилам LEGB

-----

### 6. Модуль (.py файл)

**ДА, создаёт**

```python
# file.py
x = 10  # Глобальная переменная модуля

def func():
    print(x)  # Доступна внутри функций модуля

func()  # 10
```

**Ключевые моменты:**

- Каждый .py файл — отдельное глобальное пространство имён
- Переменные на уровне модуля — глобальные (G в LEGB)

-----

### 7. Встроенная область (builtins)

**ДА, существует всегда**

```python
len([1, 2, 3])  # Встроенная функция
print(int)      # Встроенный тип

# Все встроенные функции/типы в builtins
import builtins
print(dir(builtins))
```

**Ключевые моменты:**

- Самая внешняя область в LEGB (B = Built-in)
- Доступна везде автоматически
- Содержит len, print, int, str, Exception и т.д.

-----

### 8. List / Set / Dict comprehension

**ДА, создаёт (с Python 3)**

```python
x = 10

# Переменная цикла локальна для comprehension
lst = [x for x in range(5)]

print(x)  # 10 (внешняя x не изменилась!)
print(lst)  # [0, 1, 2, 3, 4]
```

**Почему создаёт:**

```python
# Comprehension реализован как скрытая функция
# Примерный эквивалент:
def _comprehension(iterable):
    result = []
    for x in iterable:
        result.append(x)
    return result

lst = _comprehension(range(5))
```

**Вложенные переменные тоже локальны:**

```python
i = 100
j = 200

matrix = [[i*j for j in range(3)] for i in range(3)]

print(i)  # 100 (не изменилась!)
print(j)  # 200 (не изменилась!)
```

-----

### 9. Generator expression

**ДА, создаёт**

```python
x = 10

gen = (x for x in range(5))

print(x)  # 10 (внешняя x не затронута)
print(next(gen))  # 0
print(next(gen))  # 1
```

**Ключевые моменты:**

- Всегда отдельная область (как генератор)
- Реализовано как генераторная функция

-----

## ❌ Где область НЕ создаётся

### ❌ if / elif / else

```python
if True:
    x = 10

print(x)  # 10 ✅ Доступна!
```

**Переменные в if доступны снаружи!**

-----

### ❌ for / while

```python
for i in range(3):
    y = i * 2

print(i)  # 2 ✅ Доступна!
print(y)  # 4 ✅ Доступна!
```

**⚠️ Важно:**

```python
# Переменная цикла остаётся после цикла!
for x in range(5):
    pass

print(x)  # 4
```

**Исключение — comprehension:**

```python
# В comprehension переменная цикла НЕ вытекает
[x for x in range(5)]
# print(x)  # NameError (если x не была определена раньше)
```

-----

### ❌ try / except / finally

```python
try:
    z = 42
except:
    pass

print(z)  # 42 ✅ Доступна!
```

**Переменные в блоках try/except доступны снаружи!**

**Исключение — переменная исключения:**

```python
try:
    1 / 0
except ZeroDivisionError as e:
    print(e)

# print(e)  # NameError! Переменная e удаляется после блока except
```

-----

### ❌ with

```python
with open('file.txt') as f:
    data = f.read()

# f доступна, но файл уже закрыт
print(f)  # <_io.TextIOWrapper name='file.txt' mode='r' encoding='UTF-8'>
print(f.closed)  # True

# data тоже доступна
print(data)  # ✅
```

**Переменные в with доступны снаружи!**

-----

### ❌ match / case (Python 3.10+)

```python
value = 42

match value:
    case 42:
        result = "forty-two"
    case _:
        result = "other"

print(result)  # forty-two ✅ Доступна!
```

**Переменные в match/case доступны снаружи!**

-----

### ❌ class — НЕ область видимости в LEGB

**⚠️ ВАЖНО:** Класс создаёт namespace, но **НЕ участвует** в LEGB!

```python
class A:
    x = 10
    
    def f(self):
        print(x)  # ❌ NameError: name 'x' is not defined

# Класс НЕ является enclosing-областью для методов!
```

**Правильно:**

```python
class A:
    x = 10
    
    def f(self):
        print(self.x)  # ✅ Через self
        print(A.x)     # ✅ Через имя класса

a = A()
a.f()  # 10
```

**Почему так:**

```python
x = 'global'

class A:
    x = 'class'
    
    def f(self):
        # Поиск по LEGB: Local → Enclosing → Global → Built-in
        # Класс НЕ входит в Enclosing!
        print(x)  # 'global' (находит в Global!)

A().f()  # global
```

## 🎓 Правило LEGB

**L**ocal → **E**nclosing → **G**lobal → **B**uilt-in

```python
x = 'global'  # G - Global

def outer():
    x = 'enclosing'  # E - Enclosing
    
    def inner():
        x = 'local'  # L - Local
        print(x)
    
    inner()

outer()  # local
```

**Поиск переменной:**

1. **L**ocal — локальная область функции
2. **E**nclosing — объемлющие функции
3. **G**lobal — глобальная область модуля
4. **B**uilt-in — встроенная область

## 🔥 Хитрые примеры

### Пример 1: Lambda в цикле (классическая ловушка)

```python
# ❌ ОШИБКА
functions = []
for i in range(3):
    functions.append(lambda: i)

print([f() for f in functions])
# [2, 2, 2] ← Все вернули 2!
```

**Почему:** Lambda захватывает ссылку на `i`, а не значение. После цикла `i = 2`.

**Решение:**

```python
# ✅ ПРАВИЛЬНО
functions = []
for i in range(3):
    functions.append(lambda x=i: x)  # Захватываем значение

print([f() for f in functions])
# [0, 1, 2] ✅
```

-----

### Пример 2: Comprehension vs for

```python
x = 'global'

# С обычным for
for x in range(3):
    pass
print(x)  # 2 (перезаписали!)

# С comprehension
x = 'global'
[x for x in range(3)]
print(x)  # 'global' (не перезаписали!)
```

-----

### Пример 3: Класс НЕ в LEGB

```python
x = 'global'

class A:
    x = 'class'
    
    # ❌ Не находит class.x
    y = x + '_var'  # 'global_var'
    
    def method(self):
        # ❌ Не находит class.x
        return x  # 'global'

print(A.y)  # global_var
print(A().method())  # global
```

-----

### Пример 4: Переменная исключения удаляется

```python
e = 'outer'

try:
    1 / 0
except ZeroDivisionError as e:
    print(f'В except: {e}')

print(f'После except: {e}')  # NameError!
```

**Почему:** Python автоматически удаляет переменную исключения после блока except.

**Эквивалент:**

```python
try:
    1 / 0
except ZeroDivisionError as e:
    print(e)
    del e  # ← Автоматически!
```

## 💡 Практические советы

### 1. Используй comprehension вместо циклов

```python
# ❌ ПЛОХО - переменная цикла вытекает
results = []
for x in data:
    results.append(process(x))

# ✅ ХОРОШО - x изолирована
results = [process(x) for x in data]
```

### 2. Будь осторожен с lambda в циклах

```python
# ❌ ПЛОХО
callbacks = [lambda: i for i in range(3)]

# ✅ ХОРОШО
callbacks = [lambda i=i: i for i in range(3)]
```

### 3. Класс ≠ enclosing

```python
# ❌ НЕ РАБОТАЕТ
class A:
    x = 10
    y = x + 1  # NameError в некоторых случаях
```

### 4. Используй nonlocal и global явно

```python
def outer():
    x = 10
    
    def inner():
        nonlocal x  # Явно указываем
        x = 20
    
    inner()
    print(x)  # 20

outer()
```

## 📝 Шпаргалка

### Создают область видимости

```python
def func(): ...           # Функция
lambda: ...               # Лямбда
async def func(): ...     # Async функция
def gen(): yield ...      # Генератор
[... for ... in ...]      # Comprehension
(... for ... in ...)      # Generator expression
# module.py              # Модуль
```

### НЕ создают область видимости

```python
if condition: ...         # if
for i in ...: ...        # for
while ...: ...           # while
try: ... except: ...     # try/except
with ... as ...: ...     # with
match ...: case ...: ... # match/case
class A: ...             # class (не LEGB!)
```

### Правило LEGB

```python
L - Local       # Локальная функция
E - Enclosing   # Объемлющая функция
G - Global      # Модуль
B - Built-in    # Встроенное
```
