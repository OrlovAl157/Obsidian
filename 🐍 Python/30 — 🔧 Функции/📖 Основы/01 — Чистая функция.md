# 🔹 Признаки чистой функции

## 1️⃣ Детерминированность

**Описание:** Функция всегда возвращает один и тот же результат для одинаковых аргументов.

### ✅ Чистая функция

```python
def add(a, b):
    return a + b  # всегда одно и то же для одних a и b

# Примеры вызова:
print(add(2, 3))  # 5
print(add(2, 3))  # 5 (всегда одинаковый результат)
```

### ❌ Нечистая функция

```python
import random

def add_random(a, b):
    return a + b + random.randint(0, 10)  # результат меняется

# Примеры вызова:
print(add_random(2, 3))  # 8
print(add_random(2, 3))  # 12 (разные результаты!)
```

-----

## 2️⃣ Отсутствие побочных эффектов

**Описание:** Функция не изменяет внешние объекты, не пишет в файл, не печатает и не меняет состояние вне своей области видимости.

### ✅ Чистая функция

```python
def pure_add(a, b):
    return a + b  # только возвращает результат

result = pure_add(5, 3)
```

### ❌ Нечистая функция

```python
def impure_add(a, b):
    print(a + b)  # побочный эффект → не чистая
    return a + b

result = impure_add(5, 3)  # выводит "8" в консоль
```

**Другие примеры побочных эффектов:**

```python
# Изменение списка
def add_item(lst, item):
    lst.append(item)  # мутация внешнего объекта
    return lst

# Запись в файл
def save_result(a, b):
    with open('result.txt', 'w') as f:
        f.write(str(a + b))  # побочный эффект
    return a + b
```

-----

## 3️⃣ Не использует и не изменяет глобальные переменные

**Описание:** Все вычисления зависят только от входных параметров.

### ❌ Нечистая функция

```python
x = 10

def impure_function(y):
    return y + x  # зависит от внешней переменной → не чистая

print(impure_function(5))  # 15
x = 20
print(impure_function(5))  # 25 (результат изменился!)
```

### ✅ Чистая функция

```python
def pure_function(x, y):
    return x + y  # зависит только от параметров

print(pure_function(10, 5))  # 15
print(pure_function(10, 5))  # 15 (всегда одинаковый результат)
```

**💻 Пример с изменением глобальной переменной:**

```python
counter = 0

def increment():
    global counter
    counter += 1  # изменяет глобальное состояние
    return counter
```

-----

## 4️⃣ Не полагается на внешние состояния

**Описание:** Результат не зависит от времени, внешнего ввода или других изменяемых объектов.

### ❌ Нечистая функция

```python
from datetime import datetime

def get_greeting():
    hour = datetime.now().hour  # зависит от текущего времени
    if hour < 12:
        return "Доброе утро"
    else:
        return "Добрый день"

# Результат меняется в зависимости от времени суток
```

### ✅ Чистая функция

```python
def get_greeting(hour):
    if hour < 12:
        return "Доброе утро"
    else:
        return "Добрый день"

# Всегда одинаковый результат для одинакового часа
print(get_greeting(10))  # "Доброе утро"
print(get_greeting(10))  # "Доброе утро"
```

**Другие примеры зависимости от внешнего состояния:**

```python
# Зависимость от ввода пользователя
def get_user_age():
    age = input("Введите возраст: ")  # внешний ввод
    return int(age)

# Зависимость от сетевого запроса
def get_weather():
    response = requests.get('https://api.weather.com')  # внешнее состояние
    return response.json()
```

## 🔹 Идея чистой функции

**Вход → Выход, без скрытых эффектов**

### Преимущества:

- ✅ **Легко тестировать** — не нужны моки и заглушки
- ✅ **Легко комбинировать** — функции можно складывать как кубики
- ✅ **Легко параллелить** — нет гонки за ресурсами
- ✅ **Предсказуемость** — понятно, что происходит
- ✅ **Кэшируемость** — можно сохранять результаты (мемоизация)

## 📊 Сравнительная таблица

|Признак              |Чистая функция            |Нечистая функция                           |
|---------------------|--------------------------|-------------------------------------------|
|Детерминированность  |`add(2, 3)` всегда `5`    |`add_random(2, 3)` дает разные результаты  |
|Побочные эффекты     |Только возвращает значение|`print()`, изменение списков, запись в файл|
|Глобальные переменные|Не использует             |Читает или изменяет глобальные переменные  |
|Внешнее состояние    |Не зависит                |Использует `datetime.now()`, `input()`, API|

-----

## 💡 Практический совет

Старайтесь писать как можно больше чистых функций, а побочные эффекты изолируйте на границах приложения (обработчики событий, работа с БД, API, вывод в консоль).

**Паттерн:**

```
Вход (нечистая) → Обработка (чистые функции) → Выход (нечистая)
```
