# 🏷️ Пользовательские атрибуты функций

## 💡 Что это?

У объектов функций есть специальный атрибут `__dict__` — словарь, который позволяет динамически добавлять к функциям дополнительные атрибуты и данные.

```python
def my_func():
    pass

print(type(my_func.__dict__))  # <class 'dict'>
print(my_func.__dict__)         # {}
```

---

## 📝 Два способа установки атрибутов

### 1. Через точечную нотацию (рекомендуется)

```python
def greet():
    pass

greet.counter = 0
greet.name = 'Timur'
greet.values = [1, 2, 3]

print(greet.__dict__)
# {'counter': 0, 'name': 'Timur', 'values': [1, 2, 3]}
```

### 2. В стиле словаря

```python
def greet():
    pass

greet.__dict__['counter'] = 0
greet.__dict__['name'] = 'Timur'
greet.__dict__['values'] = [1, 2, 3]

print(greet.__dict__)
# {'counter': 0, 'name': 'Timur', 'values': [1, 2, 3]}
```

## 🔄 Доступ внутри и снаружи функции

### Установка атрибутов снаружи

```python
def greet():
    pass

# До вызова
print(greet.__dict__)  # {}

# Устанавливаем атрибуты
greet.value = 777
greet.numbers = [1, 2, 3]
greet.name = 'Timur'

print(greet.__dict__)
# {'value': 777, 'numbers': [1, 2, 3], 'name': 'Timur'}
```

### Установка атрибутов изнутри

```python
def greet():
    greet.age = 17
    greet.counter = greet.__dict__.get('counter', 0) + 1
    print(f"Вызов #{greet.counter}")

greet()  # Вызов #1
greet()  # Вызов #2
greet()  # Вызов #3

print(greet.__dict__)
# {'age': 17, 'counter': 3}
```

### Комбинированный доступ

```python
def process():
    # Читаем атрибут, установленный снаружи
    print(f"Config: {process.config}")
    # Устанавливаем новый атрибут
    process.result = "Success"

# Устанавливаем снаружи
process.config = {'debug': True}

# Вызываем
process()  # Config: {'debug': True}

# Читаем атрибут, установленный изнутри
print(process.result)  # Success
```

## 🎯 Практические применения

### 1. Счётчик вызовов функции

```python
def count_calls():
    count_calls.calls = count_calls.__dict__.get('calls', 0) + 1
    print(f"Функция вызвана {count_calls.calls} раз(а)")

count_calls()  # Функция вызвана 1 раз(а)
count_calls()  # Функция вызвана 2 раз(а)
count_calls()  # Функция вызвана 3 раз(а)

print(count_calls.calls)  # 3
```

**✅ Более элегантная версия с декоратором:**

```python
def count_calls(func):
    func.calls = 0
    
    def wrapper(*args, **kwargs):
        func.calls += 1
        print(f"Вызов #{func.calls}")
        return func(*args, **kwargs)
    
    wrapper.__name__ = func.__name__
    wrapper.calls = func.calls
    return wrapper

@count_calls
def greet(name):
    return f"Hello, {name}!"

greet("Alice")  # Вызов #1
greet("Bob")    # Вызов #2
print(greet.calls)  # 2
```

### 2. Кэширование результатов (мемоизация)

**⚠️ Проблема: медленный Fibonacci без кэша**

```python
def fib_slow(num):
    if num < 2:
        return num
    return fib_slow(num - 1) + fib_slow(num - 2)

# fib_slow(35) займёт несколько секунд
```

**Решение: кэширование через **dict****

```python
def fib(num):
    if num < 2:
        return num
    
    # Проверяем кэш
    if num not in fib.__dict__:
        # Вычисляем и сохраняем в кэш
        fib.__dict__[num] = fib(num - 1) + fib(num - 2)
    
    return fib.__dict__[num]

print(fib(100))  # Мгновенно!
print(len(fib.__dict__))  # 99 (закэшировано значений)
```

**Альтернатива с атрибутом cache:**

```python
def fib(num):
    if num < 2:
        return num
    
    # Инициализируем cache при первом вызове
    if not hasattr(fib, 'cache'):
        fib.cache = {}
    
    if num not in fib.cache:
        fib.cache[num] = fib(num - 1) + fib(num - 2)
    
    return fib.cache[num]

print(fib(100))
print(fib.cache)  # {2: 1, 3: 2, 4: 3, ...}
```

print(accumulator(10))  # 10

### 6. Регистрация обработчиков

```python
def handle_command(command):
    handlers = handle_command.handlers
    
    if command in handlers:
        handlers[command]()
    else:
        print(f"Неизвестная команда: {command}")

# Регистрация обработчиков
handle_command.handlers = {}

def register(command):
    def decorator(func):
        handle_command.handlers[command] = func
        return func
    return decorator

@register('start')
def start_handler():
    print("Запуск...")

@register('stop')
def stop_handler():
    print("Остановка...")

handle_command('start')  # Запуск...
handle_command('stop')   # Остановка...
handle_command('pause')  # Неизвестная команда: pause
```

## ⚠️ Важные особенности

### 1. Пустой словарь по умолчанию

```python
def my_func():
    pass

print(my_func.__dict__)  # {}
```

Если атрибуты никогда не устанавливались, `__dict__` будет пустым.

### 2. Независимость от экземпляров

```python
def greet():
    pass

greet.counter = 0

# Все обращения к greet используют один и тот же словарь
print(id(greet.__dict__))  # 140234567890
print(id(greet.__dict__))  # 140234567890 (тот же!)
```

### 3. Конфликты с именами

```python
def func():
    pass

# ❌ Избегайте перезаписи встроенных атрибутов
# func.__name__ = 'other'  # Работает, но не рекомендуется
# func.__doc__ = 'new doc'  # Работает, но не рекомендуется

# ✅ Используйте свои имена
func.custom_name = 'other'
func.custom_doc = 'new doc'
```

### 4. Проверка наличия атрибута

```python
def func():
    pass

# Способ 1: hasattr()
if hasattr(func, 'counter'):
    print(func.counter)

# Способ 2: getattr() с значением по умолчанию
counter = getattr(func, 'counter', 0)

# Способ 3: проверка в __dict__
if 'counter' in func.__dict__:
    print(func.__dict__['counter'])

# Способ 4: get() для __dict__
counter = func.__dict__.get('counter', 0)
```

## 🆚 Сравнение с другими подходами

### Атрибуты функции vs Глобальные переменные

```python
# ❌ Глобальные переменные (плохо)
counter = 0

def increment():
    global counter
    counter += 1
    return counter

# ✅ Атрибуты функции (лучше)
def increment():
    increment.counter = getattr(increment, 'counter', 0) + 1
    return increment.counter

# ✅✅ Замыкание (еще лучше для этой задачи)
def make_counter():
    counter = 0
    
    def increment():
        nonlocal counter
        counter += 1
        return counter
    
    return increment
```

### Атрибуты функции vs functools.lru_cache

```python
# Вручную через __dict__
def fib_manual(n):
    if n < 2:
        return n
    if n not in fib_manual.__dict__:
        fib_manual.__dict__[n] = fib_manual(n-1) + fib_manual(n-2)
    return fib_manual.__dict__[n]

# Проще с lru_cache
from functools import lru_cache

@lru_cache(maxsize=None)
def fib_auto(n):
    if n < 2:
        return n
    return fib_auto(n-1) + fib_auto(n-2)
```

**Вывод:** Для кэширования лучше использовать `@lru_cache`, но знание `__dict__` полезно для других задач.

## 🎨 Продвинутые примеры

### Декоратор с метриками

```python
import time

def track_performance(func):
    func.calls = 0
    func.total_time = 0.0
    func.avg_time = 0.0
    
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        elapsed = time.time() - start
        
        func.calls += 1
        func.total_time += elapsed
        func.avg_time = func.total_time / func.calls
        
        return result
    
    wrapper.__name__ = func.__name__
    wrapper.__doc__ = func.__doc__
    # Копируем атрибуты метрик
    wrapper.calls = func.calls
    wrapper.total_time = func.total_time
    wrapper.avg_time = func.avg_time
    
    return wrapper

@track_performance
def slow_function(n):
    time.sleep(0.1)
    return n * 2

slow_function(5)
slow_function(10)
slow_function(15)

print(f"Вызовов: {slow_function.calls}")           # 3
print(f"Общее время: {slow_function.total_time:.2f}s")  # ~0.30s
print(f"Среднее время: {slow_function.avg_time:.2f}s")  # ~0.10s
```

### Функция с историей вызовов

```python
def with_history(func):
    func.history = []
    
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        func.history.append({
            'args': args,
            'kwargs': kwargs,
            'result': result
        })
        return result
    
    wrapper.__name__ = func.__name__
    wrapper.history = func.history
    return wrapper

@with_history
def add(a, b):
    return a + b

add(2, 3)
add(5, 7)
add(10, 20)

for i, call in enumerate(add.history, 1):
    print(f"Вызов {i}: add{call['args']} = {call['result']}")

# Вызов 1: add(2, 3) = 5
# Вызов 2: add(5, 7) = 12
# Вызов 3: add(10, 20) = 30
```

## 📚 Связь с другими концепциями

### 1. OrderedDict и Counter

Как и `OrderedDict` и `Counter` из модуля `collections`, функции имеют атрибут `__dict__` для хранения дополнительных данных:

```python
from collections import OrderedDict, Counter

# OrderedDict имеет __dict__
od = OrderedDict()
print(hasattr(od, '__dict__'))  # True

# Counter имеет __dict__
c = Counter()
print(hasattr(c, '__dict__'))  # True

# Функции тоже имеют __dict__
def func():
    pass
print(hasattr(func, '__dict__'))  # True
```

### 2. Встроенные атрибуты функций

Помимо `__dict__`, у функций есть множество других атрибутов:

```python
def greet(name: str) -> str:
    """Приветствует пользователя"""
    return f"Hello, {name}!"

print(greet.__name__)        # greet
print(greet.__doc__)         # Приветствует пользователя
print(greet.__module__)      # __main__
print(greet.__annotations__) # {'name': <class 'str'>, 'return': <class 'str'>}
print(greet.__code__)        # <code object greet at 0x...>
```

## ✅ Резюме

|Применение|Рекомендация|
|---|---|
|Счётчик вызовов|✅ Атрибуты или замыкание|
|Кэширование|⚠️ Лучше `@lru_cache`|
|Конфигурация|✅ Атрибуты функции|
|Состояние|✅ Атрибуты или замыкание|
|Метаданные|✅ Атрибуты функции|
|Регистрация|✅ Атрибуты функции|

**✅ Когда использовать атрибуты функций:**

- ✅ Нужно хранить метаданные
- ✅ Нужно состояние между вызовами
- ✅ Создаете фреймворк/библиотеку
- ✅ Нужен простой реестр/регистр

**❌ Когда НЕ использовать:**

- ❌ Для кэширования (используйте `@lru_cache`)
- ❌ Для сложного состояния (используйте классы)
- ❌ Когда нужна изоляция (используйте замыкания)
