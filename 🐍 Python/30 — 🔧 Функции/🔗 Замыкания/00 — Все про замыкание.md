# 🔒 Замыкания (Closures) в Python

## 💡 Что такое замыкание?

**Замыкание (closure)** — это функция, которая “запоминает” переменные из области видимости, в которой она была создана, даже после того, как эта область перестала существовать.

```python
def make_greeter(phrase):
    prefix = phrase + ', '  # Локальная переменная
    
    def wrapper(name):
        return prefix + name + '!'  # Использует prefix из внешней функции
    
    return wrapper  # Возвращаем функцию с "замороженным" prefix

greeter = make_greeter('Hello')
print(greeter('Alice'))  # → "Hello, Alice!"
# make_greeter уже завершилась, но prefix всё ещё доступен!
```

**🎯 Ключевая идея:** Внутренняя функция “запоминает” переменные внешней функции и носит их с собой как “рюкзак”.

-----

## 🧠 Визуализация: Как это работает в памяти

### Шаг 1: Вызов внешней функции

```python
first = make_greeter('Sup men')
```

```
┌─────────────────────────────────────────┐
│ Область видимости make_greeter          │
│                                         │
│ phrase = 'Sup men'                      │
│ prefix = 'Sup men, '  ← создано         │
│                                         │
│ def wrapper(name):                      │
│     return prefix + name + '!'          │
│            ↑                            │
│            └── Замыкание на prefix!     │
└─────────────────────────────────────────┘
                    ↓
        first = wrapper (с prefix='Sup men, ')
```

### Шаг 2: Внешняя функция завершилась

```
make_greeter: 🗑️ УДАЛЕНА из стека вызовов
phrase:       🗑️ Удалена
prefix:       ✅ СОХРАНЁН в замыкании wrapper!

first: ✅ ЖИВЁТ со своим "рюкзаком" (prefix)
```

### Шаг 3: Вызов замыкания

```python
first('Alex')
```

```
┌─────────────────────────────────────────┐
│ Вызов wrapper('Alex')                   │
│                                         │
│ name = 'Alex'          ← новый аргумент │
│ prefix = 'Sup men, '   ← из замыкания!  │
│                                         │
│ return 'Sup men, ' + 'Alex' + '!'       │
└─────────────────────────────────────────┘
           ↓
    'Sup men, Alex!'
```

**❗ Важно:** `make_greeter` НЕ вызывается повторно! Она уже завершилась.

## 📦 Что хранится в замыкании?

Python хранит замкнутые переменные в специальных “ячейках”:

```python
def make_greeter(phrase):
    prefix = phrase + ', '
    
    def wrapper(name):
        return prefix + name + '!'
    
    return wrapper

first = make_greeter('Sup men')

# Посмотрим, что внутри замыкания:
print(first)
# → <function make_greeter.<locals>.wrapper at 0x...>

print(first.__closure__)
# → (<cell at 0x...: str object at 0x...>,)
# Это "ячейка" с переменной prefix!

print(first.__closure__[0].cell_contents)
# → 'Sup men, '
# Вот она, переменная prefix из make_greeter!
```

## 🔍 Детальный пример с выводом

```python
def make_greeter(phrase):
    print(f"✅ make_greeter ВЫЗВАНА с phrase='{phrase}'")
    prefix = phrase + ', '
    
    def wrapper(name):
        print(f"  → wrapper вызван с name='{name}'")
        print(f"  → prefix из замыкания: '{prefix}'")
        return prefix + name + '!'
    
    print(f"✅ make_greeter ЗАВЕРШАЕТСЯ, возвращает wrapper")
    return wrapper

print("=" * 50)
print("1️⃣ Создаём первый greeter:")
print("=" * 50)
first = make_greeter('Sup men')
# Вывод:
# ✅ make_greeter ВЫЗВАНА с phrase='Sup men'
# ✅ make_greeter ЗАВЕРШАЕТСЯ, возвращает wrapper

print("\n" + "=" * 50)
print("2️⃣ Вызываем first:")
print("=" * 50)
result = first('Alex')
print(f"Результат: {result}")
# Вывод:
#   → wrapper вызван с name='Alex'
#   → prefix из замыкания: 'Sup men, '
# Результат: Sup men, Alex!

print("\n" + "=" * 50)
print("3️⃣ Вызываем first ещё раз:")
print("=" * 50)
result = first('Bob')
print(f"Результат: {result}")
# Вывод:
#   → wrapper вызван с name='Bob'
#   → prefix из замыкания: 'Sup men, '
# Результат: Sup men, Bob!

print("\n" + "=" * 50)
print("4️⃣ Создаём ВТОРОЙ greeter:")
print("=" * 50)
second = make_greeter('Hello')
# Вывод:
# ✅ make_greeter ВЫЗВАНА с phrase='Hello'
# ✅ make_greeter ЗАВЕРШАЕТСЯ, возвращает wrapper

print("\n" + "=" * 50)
print("5️⃣ Вызываем second:")
print("=" * 50)
result = second('Alice')
print(f"Результат: {result}")
# Вывод:
#   → wrapper вызван с name='Alice'
#   → prefix из замыкания: 'Hello, '
# Результат: Hello, Alice!
```

**📊 Итого:**

- `make_greeter` вызвана **2 раза** (для first и second)
- `wrapper` вызван **3 раза** (2 раза first, 1 раз second)
- У `first` и `second` **разные замыкания** с разными `prefix`

## 💡 Практические примеры

### 1. Счётчик

```python
def make_counter():
    count = 0  # Переменная в замыкании
    
    def increment():
        nonlocal count  # Изменяем переменную из внешней области
        count += 1
        return count
    
    return increment

counter1 = make_counter()
print(counter1())  # → 1
print(counter1())  # → 2
print(counter1())  # → 3

counter2 = make_counter()  # НОВОЕ замыкание, свой count
print(counter2())  # → 1 (не влияет на counter1!)
print(counter1())  # → 4 (продолжает свой счёт)
```

**Визуализация:**

```
counter1:                counter2:
┌──────────┐            ┌──────────┐
│ count = 3│            │ count = 1│
└──────────┘            └──────────┘
Независимые счётчики!
```

### 2. Приватные переменные

```python
def create_account(initial_balance):
    balance = initial_balance  # "Приватная" переменная
    
    def deposit(amount):
        nonlocal balance
        balance += amount
        return balance
    
    def withdraw(amount):
        nonlocal balance
        if amount > balance:
            return "Недостаточно средств"
        balance -= amount
        return balance
    
    def get_balance():
        return balance
    
    # Возвращаем словарь с методами
    return {
        'deposit': deposit,
        'withdraw': withdraw,
        'balance': get_balance
    }

account = create_account(1000)

print(account['balance']())    # → 1000
print(account['deposit'](500)) # → 1500
print(account['withdraw'](200))# → 1300
print(account['balance']())    # → 1300

# Напрямую к balance доступа нет!
# print(balance)  # NameError!
```

    def apply(a, b):

### 4. Запоминание состояния (toggle)

```python
def make_toggle(initial_state=False):
    state = initial_state
    
    def toggle():
        nonlocal state
        state = not state
        return state
    
    return toggle

light = make_toggle()  # Свет выключен

print(light())  # → True (включили)
print(light())  # → False (выключили)
print(light())  # → True (включили)

door = make_toggle(True)  # Дверь открыта

print(door())   # → False (закрыли)
print(door())   # → True (открыли)
```

### 5. Кэширование с замыканием

```python
def make_cached_function(func):
    cache = {}  # Кэш в замыкании
    
    def wrapper(x):
        if x not in cache:
            print(f"Вычисляем для {x}...")
            cache[x] = func(x)
        else:
            print(f"Из кэша для {x}")
        return cache[x]
    
    return wrapper

@make_cached_function
def expensive_calculation(n):
    return n ** 2

print(expensive_calculation(5))  # Вычисляем для 5... → 25
print(expensive_calculation(5))  # Из кэша для 5 → 25
print(expensive_calculation(3))  # Вычисляем для 3... → 9
print(expensive_calculation(3))  # Из кэша для 3 → 9
```

### 6. Генератор уникальных ID

```python
def make_id_generator(prefix='ID'):

    def generate():
        nonlocal counter
        counter += 1
        return f"{prefix}-{counter:04d}"
    
    return generate

user_id = make_id_generator('USER')
order_id = make_id_generator('ORDER')

print(user_id())   # → USER-0002
print(order_id())  # → ORDER-0001
print(user_id())   # → USER-0003
print(order_id())  # → ORDER-0002
```

def make_validator(min_length, max_length):

## 🔧 nonlocal - изменение переменных в замыкании

### Проблема без nonlocal

```python
def make_counter():
    count = 0
    
    def increment():
        count = count + 1  # ❌ UnboundLocalError!
        return count
    
    return increment

counter = make_counter()
# counter()  # Ошибка!
```

**Почему ошибка?**
Python видит `count = count + 1` и думает, что `count` — локальная переменная функции `increment`. Но справа мы пытаемся прочитать `count` до присваивания!

### ✅ Решение: nonlocal

```python
def make_counter():
    count = 0
    
    def increment():
        nonlocal count  # ✅ Говорим: используй count из внешней области
        count += 1
        return count
    
    return increment

counter = make_counter()
print(counter())  # → 1
print(counter())  # → 2
print(counter())  # → 3
```

**Правило:** Используй `nonlocal`, когда нужно **изменить** переменную из внешней области.

## 📊 Замыкания vs Глобальные переменные

### ❌ Глобальные переменные (плохо)

```python
count = 0  # Глобальная переменная

def increment():
    global count
    count += 1
    return count

def reset():
    global count
    count = 0

print(increment())  # → 1
print(increment())  # → 2
reset()
print(increment())  # → 1

# Проблемы:
# - Любая функция может изменить count
# - Нельзя создать несколько счётчиков
# - Сложно тестировать
```

### ✅ Замыкания (хорошо)

```python
def make_counter():
    count = 0  # Приватная переменная
    
    def increment():
        nonlocal count
        count += 1
        return count
    
    def reset():
        nonlocal count
        count = 0
    
    return {'increment': increment, 'reset': reset}

counter1 = make_counter()
counter2 = make_counter()

print(counter1['increment']())  # → 1
print(counter1['increment']())  # → 2
print(counter2['increment']())  # → 1 (независимый!)

counter1['reset']()
print(counter1['increment']())  # → 1

# Преимущества:
# ✅ Инкапсуляция - count защищён
# ✅ Множественные экземпляры
# ✅ Легко тестировать
```

## 🆚 Замыкания vs Классы

### С замыканием

```python
def make_person(name, age):
    def get_info():
        return f"{name}, {age} лет"
    
    def birthday():
        nonlocal age
        age += 1
        return f"С днём рождения! Теперь {age} лет"
    
    return {'info': get_info, 'birthday': birthday}

person = make_person("Alice", 25)
print(person['info']())      # → Alice, 25 лет
print(person['birthday']())  # → С днём рождения! Теперь 26 лет
print(person['info']())      # → Alice, 26 лет
```

### С классом

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def get_info(self):
        return f"{self.name}, {self.age} лет"
    
    def birthday(self):
        self.age += 1
        return f"С днём рождения! Теперь {self.age} лет"

person = Person("Alice", 25)
print(person.get_info())     # → Alice, 25 лет
print(person.birthday())     # → С днём рождения! Теперь 26 лет
print(person.get_info())     # → Alice, 26 лет
```

**Когда использовать что?**

|Замыкания           |Классы                   |
|--------------------|-------------------------|
|Простая логика      |Сложная структура        |
|1-3 метода          |Много методов            |
|Функциональный стиль|ООП стиль                |
|Лёгкий вес          |Наследование, полиморфизм|

## ⚠️ Частые ошибки

### 1. Забыть nonlocal при изменении

```python
# ❌ НЕПРАВИЛЬНО
def make_counter():
    count = 0
    def increment():
        count += 1  # UnboundLocalError!
        return count
    return increment

# ✅ ПРАВИЛЬНО
def make_counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment
```

### 2. Замыкание на изменяемый объект

```python
# ⚠️ ОСТОРОЖНО
def make_functions():
    funcs = []
    for i in range(3):
        def func():
            return i  # Замыкание на i!
        funcs.append(func)
    return funcs

f1, f2, f3 = make_functions()
print(f1())  # → 2 (не 0!)
print(f2())  # → 2 (не 1!)
print(f3())  # → 2 ✅

# Почему? i = 2 в конце цикла, все функции видят одно i!
```

**Решение 1: Значение по умолчанию**

```python
def make_functions():
    funcs = []
    for i in range(3):
        def func(x=i):  # Фиксируем i через default
            return x
        funcs.append(func)
    return funcs

f1, f2, f3 = make_functions()
print(f1())  # → 0 ✅
print(f2())  # → 1 ✅
print(f3())  # → 2 ✅
```

**Решение 2: Фабрика функций**

```python
def make_functions():
    funcs = []
    for i in range(3):
        def make_func(x):
            def func():
                return x
            return func
        funcs.append(make_func(i))
    return funcs

f1, f2, f3 = make_functions()
print(f1())  # → 0 ✅
print(f2())  # → 1 ✅
print(f3())  # → 2 ✅
```

### 3. Изменяемые объекты в замыкании

```python
def make_functions():
    data = []  # Изменяемый список
    
    def add(item):
        data.append(item)  # ✅ Изменяем объект - nonlocal не нужен!
    
    def get():
        return data
    
    return {'add': add, 'get': get}

funcs = make_functions()
funcs['add'](1)
funcs['add'](2)
print(funcs['get']())  # → [1, 2]

# Для изменяемых объектов (list, dict, set) nonlocal не нужен!
# nonlocal нужен только для переприсваивания переменной
```

## 🎓 Продвинутое: Как Python реализует замыкания

```python
def outer(x):
    y = 10
    z = 20
    
    def inner():
        return x + y  # Использует x и y, но не z
    
    return inner

func = outer(5)

# Что внутри?
print(func.__code__.co_freevars)  # → ('x', 'y')
# Только переменные, которые используются!

print(func.__closure__)
# → (<cell: int(5)>, <cell: int(10)>)

print(func.__closure__[0].cell_contents)  # → 5 (x)
print(func.__closure__[1].cell_contents)  # → 10 (y)
```

## 💭 Итоговая шпаргалка

### Структура замыкания

```python
def outer(param):
    variable = "значение"
    
    def inner(arg):
        # Использует param и variable из outer
        return param + variable + arg
    
    return inner  # Возвращаем функцию с замыканием

closure = outer("hello")
result = closure("world")
```

### Ключевые моменты

✅ **Замыкание** = функция + её окружение (переменные)  
✅ **Внешняя функция** создаёт замыкание и завершается  
✅ **Внутренняя функция** живёт со своим “рюкзаком”  
✅ **nonlocal** нужен для изменения переменных  
✅ **Множественные вызовы** = независимые замыкания

### Проверка понимания

```python
# Что выведет этот код?
def mystery():
    x = 0
    def add():
        nonlocal x
        x += 1
        return x
    return add

f1 = mystery()
f2 = mystery()

print(f1())  # ?
print(f1())  # ?
print(f2())  # ?

# Ответ: 1, 2, 1
# f1 и f2 имеют РАЗНЫЕ замыкания!
```
