

### 0) Что было «до всего» — загрузка интерпретатора

Когда запускается CPython, в его C-исходниках происходит инициализация встроенных типов. В этот момент создаются объекты, соответствующие встроенным типам, в том числе `object` и `type`, и связываются между собой (это инициализация «внутренней мета-системы»).

---

### 1) Инициализация на старте: `type` → создаёт `object` (и другие встроенные типы)

- Во время загрузки интерпретатора код на C **инициализирует** базовые структуры: сначала создаётся структура для `object` (PyBaseObject_Type), затем — для `type` (PyType_Type) и другие.
    
- Результат: `object` — **класс**, а `type` — **класс, который описывает классы**.
    
- Факт: `type(object) is type` → `object` имеет метакласс `type`.
    

> Иначе: `type` — реализация механизма создания классов; при старте интерпретатор использует этот механизм (на низком уровне) чтобы создать `object`.

---

### 2) Создание класса в твоём коде (что делает `type`)

Когда ты пишешь:

`class MyClass(Base1, Base2):     attr = 1     def method(self): ...`

под капотом примерно выполняется то же, что:

`MyClass = type('MyClass', (Base1, Base2), {'attr': 1, 'method': function_object})`

- `type(name, bases, dict)` — **создаёт новый объект класса** (структуру класса), записывает ему:
    
    - имя (`__name__`),
        
    - базовые классы (`__mro__`/MRO),
        
    - словарь атрибутов (`__dict__`),
        
    - мета-информацию.
        
- После этого `MyClass` — объект класса (его тип — `type`).
    

---

### 3) Что получает класс от `object` (наследование)

- Если класс явно не наследует ничего, он **по умолчанию** наследуется от `object`.
    
- Наследование означает: класс получает базовый «скелет» (базовые методы и поведение), определённый в `object` (минимальный набор операций и атрибутов).
    
- Затем `type` (при создании класса) добавляет/комбинирует специфические методы/атрибуты (из `dict`) и строит MRO.
    

---

### 4) Когда создаётся экземпляр (что делает `type`)

Когда ты делаешь:

`obj = MyClass(*args, **kwargs)`

выполняется последовательность (упрощённо):

1. Вызывается `MyClass.__call__(*args, **kwargs)`.  
    По умолчанию `MyClass` — экземпляр `type`, значит используется реализация `type.__call__`.
    
2. `type.__call__` делает примерно так:
    
    - вызывает `MyClass.__new__(MyClass, *args, **kwargs)` — **создание (allocation)** нового объекта в памяти; `__new__` возвращает пустой экземпляр (обычно объект с выделенной памятью).
        
    - если `__new__` вернул экземпляр нужного типа, затем вызывается `MyClass.__init__(instance, *args, **kwargs)` — **инициализация** (наполнение полей).
        
3. В результате:
    
    - у нового объекта установлен `__class__ = MyClass`,
        
    - создан и инициализирован `__dict__` (если класс поддерживает словарь атрибутов),
        
    - объект готов к использованию.
        

> Итого: `type` управляет моментом создания экземпляра — через `__call__` → `__new__` → `__init__`.

---

### 5) Где хранятся данные (атрибуты) и как Python их ищет

- **Атрибуты экземпляра** обычно хранятся в `instance.__dict__` (словарь).
    
- **Атрибуты класса** — в `Class.__dict__`.
    
- При обращении `instance.name` порядок поиска (упрощённо):
    
    1. `instance.__dict__` — есть ли `name` у конкретного экземпляра?
        
    2. `Class.__dict__` — есть ли `name` в самом классе?
        
    3. По MRO — искать в базовых классах слева направо.
        
    4. В `object.__dict__` (последний уровень).
        
- Метод разрешения называется MRO (Method Resolution Order).
    

---

### 6) Почему `type` сам наследуется от `object`

- Всё в Python — объект → значит `type` тоже должен иметь поведение объектов → он наследует от `object`.
    
- Но `type` одновременно реализует поведение «создания классов» — поэтому `type` и `object` оказываются связанными и замыкают систему (как мы обсуждали).
    

---

# Наглядный пример кода

`# создаём класс "вручную" через type MyClass = type('MyClass', (object,), {'x': 10, 'show': lambda self: self.x})  inst = MyClass()         # вызывает type.__call__ -> __new__ -> __init__ print(inst.x)            # 10  (атрибут берётся сначала из instance.__dict__, потом из Class.__dict__) print(type(MyClass))     # <class 'type'> print(type(inst))        # <class '__main__.MyClass'> print(issubclass(MyClass, object))  # True`

# Быстрая табличка (чтобы закрыть тему)

|Событие|Кто действует|Результат|
|---|---|---|
|Интерпретатор стартует|Встроенная инициализация (C)|Создаются `object`, `type` и другие встроенные типы|
|Ты пишешь `class ...`|`type` (вызван автоматически)|Создаётся объект-класс с именем, MRO, `__dict__`|
|Ты вызываешь `MyClass()`|`type.__call__` -> `__new__` -> `__init__`|Появляется экземпляр (`__class__` указывает на `MyClass`)|
|Чтение `inst.attr`|механизм поиска атрибутов (instance → class → bases → object)|Возвращает значение или `AttributeError`|
