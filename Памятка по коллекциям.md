## ⚡ ПРОИЗВОДИТЕЛЬНОСТЬ (Big O)

### Временная сложность операций

|Операция                   |list      |tuple  |set |dict|deque     |array     |
|---------------------------|----------|-------|----|----|----------|----------|
|**Доступ по индексу** `[i]`|O(1)      |O(1)   |❌   |❌   |O(n)      |O(1)      |
|**Поиск элемента** `x in`  |O(n)      |O(n)   |O(1)|O(1)|O(n)      |O(n)      |
|**Добавить в конец**       |O(1)*     |❌      |O(1)|O(1)|O(1)      |O(1)*     |
|**Добавить в начало**      |O(n)      |❌      |O(1)|❌   |O(1)      |O(n)      |
|**Вставка в середину**     |O(n)      |❌      |O(1)|❌   |O(n)      |O(n)      |
|**Удаление элемента**      |O(n)      |❌      |O(1)|O(1)|O(n)**    |O(n)      |
|**Удаление с конца**       |O(1)      |❌      |O(1)|O(1)|O(1)      |O(1)      |
|**Удаление с начала**      |O(n)      |❌      |O(1)|❌   |O(1)      |O(n)      |
|**Сортировка**             |O(n log n)|❌      |❌   |❌   |O(n log n)|O(n log n)|
|**Копирование**            |O(n)      |O(1)***|O(n)|O(n)|O(n)      |O(n)      |

*амортизированная сложность  
**O(1) с концов, O(n) в середине  
***копирование tuple создаёт ссылку, а не новый объект

## 💾 ИСПОЛЬЗОВАНИЕ ПАМЯТИ

|Коллекция|Память     |Комментарий                                        |
|---------|-----------|---------------------------------------------------|
|**list** |Средняя    |Резервирует дополнительное место для роста         |
|**tuple**|Минимальная|Нет резервирования, компактнее list                |
|**set**  |Высокая    |Хеш-таблица требует дополнительную память          |
|**dict** |Высокая    |Хеш-таблица + хранение ключей и значений           |
|**str**  |Средняя    |Оптимизирована для текста                          |
|**deque**|Средняя    |Блоки памяти вместо непрерывного массива           |
|**array**|Минимальная|Самая компактная для чисел (в 3-4 раза меньше list)|

### Сравнение памяти (примерно)

```python
import sys

lst = [1, 2, 3, 4, 5]
tpl = (1, 2, 3, 4, 5)
st = {1, 2, 3, 4, 5}
arr = array('i', [1, 2, 3, 4, 5])

sys.getsizeof(lst)  # ~120 байт
sys.getsizeof(tpl)  # ~80 байт
sys.getsizeof(st)   # ~224 байт
sys.getsizeof(arr)  # ~80 байт (для чисел!)
```

## ✅ ПЛЮСЫ И МИНУСЫ

### list (список)

**✅ Плюсы:**

- Универсальная, подходит для большинства задач
- Быстрый доступ по индексу O(1)
- Динамический размер
- Много встроенных методов
- Можно хранить любые типы данных

**❌ Минусы:**

- Медленная вставка/удаление в начале O(n)
- Больше памяти чем tuple
- Не может быть ключом словаря
- Поиск элемента медленный O(n)

**📌 Использовать:**

- Основная коллекция для большинства задач
- Когда нужен порядок элементов
- Когда размер меняется
- Когда нужен доступ по индексу

**🚫 Не использовать:**

- Когда данные не должны меняться → tuple
- Когда нужны только уникальные → set
- Когда много вставок в начало → deque
- Для больших числовых массивов → array

-----

### tuple (кортеж)

**✅ Плюсы:**

- Быстрее list (меньше операций)
- Меньше памяти
- Может быть ключом словаря
- Защита от случайных изменений
- Безопаснее для передачи данных

**❌ Минусы:**

- Нельзя изменить после создания
- Мало методов (только count, index)
- Нельзя добавлять/удалять элементы

**📌 Использовать:**

- Данные не должны меняться
- Ключи словаря
- Возврат нескольких значений из функции
- Координаты (x, y), RGB (r, g, b)
- Когда нужна небольшая экономия памяти

**🚫 Не использовать:**

- Когда данные будут меняться → list

-----

### set (множество)

**✅ Плюсы:**

- Автоматически убирает дубликаты
- Очень быстрая проверка наличия O(1)
- Быстрое добавление/удаление O(1)
- Математические операции множеств

**❌ Минусы:**

- Нет порядка (нельзя обратиться по индексу)
- Только хешируемые элементы
- Нельзя хранить списки, словари
- Занимает больше памяти

**📌 Использовать:**

- Нужны только уникальные элементы
- Быстрая проверка наличия: `x in set`
- Удаление дубликатов: `list(set(lst))`
- Операции: объединение, пересечение, разность

**🚫 Не использовать:**

- Когда важен порядок → list
- Когда нужны дубликаты → list
- Когда нужен доступ по индексу → list
- Для неххешируемых объектов → list

-----

### dict (словарь)

**✅ Плюсы:**

- Очень быстрый доступ по ключу O(1)
- Гибкая структура данных
- Сохраняет порядок (Python 3.7+)
- Ключи и значения любого типа

**❌ Минусы:**

- Больше памяти чем list
- Ключи должны быть хешируемыми
- Медленнее list для простых данных
- Нельзя использовать как ключ

**📌 Использовать:**

- Связь ключ-значение
- Быстрый поиск по ключу
- Подсчет элементов: `{item: count}`
- Кэширование результатов
- Конфигурации, настройки
- Группировка данных

**🚫 Не использовать:**

- Для простого списка → list
- Когда не нужны пары → list/set
- Когда ключи не хешируемы → list кортежей

-----

### str (строка)

**✅ Плюсы:**

- Неизменяемая (безопасная)
- Много встроенных методов
- Оптимизирована для текста
- Может быть ключом словаря

**❌ Минусы:**

- Нельзя изменить (нужно создавать новую)
- Конкатенация может быть медленной
- Только текстовые данные

**📌 Использовать:**

- Работа с текстом
- Когда данные не меняются
- Ключи словаря

**🚫 Не использовать:**

- Для частых изменений → list символов или io.StringIO
- Для бинарных данных → bytes

-----

### deque (двусторонняя очередь)

**✅ Плюсы:**

- Быстрое добавление/удаление с обоих концов O(1)
- Работает как связный список
- Ограничение размера (maxlen)
- Эффективная ротация

**❌ Минусы:**

- Медленный доступ по индексу O(n)
- Больше памяти чем list (для маленьких коллекций)
- Медленная вставка в середину

**📌 Использовать:**

- Очереди (FIFO): `appendleft()`, `pop()`
- Стеки (LIFO): `append()`, `pop()`
- Частые операции в начале списка
- История действий с ограничением
- Скользящее окно

**🚫 Не использовать:**

- Когда нужен доступ по индексу → list
- Для маленьких коллекций → list

-----

### array (массив)

**✅ Плюсы:**

- Очень компактный (в 3-4 раза меньше list)
- Быстрый для числовых операций
- Методы как у list

**❌ Минусы:**

- Только один тип данных
- Менее универсален чем list
- Нужно указывать тип (‘i’, ‘f’, ‘d’)

**📌 Использовать:**

- Большие числовые массивы
- Экономия памяти критична
- Работа с бинарными данными
- Обмен данными с C

**🚫 Не использовать:**

- Для смешанных типов → list
- Для маленьких коллекций → list
- Для сложных объектов → list

## 🎯 КОГДА ЧТО ИСПОЛЬЗОВАТЬ

### По задаче:

|Задача                 |Лучший выбор          |Почему                         |
|-----------------------|----------------------|-------------------------------|
|Универсальная коллекция|**list**              |Подходит для 80% случаев       |
|Неизменяемые данные    |**tuple**             |Безопасность, может быть ключом|
|Только уникальные      |**set**               |Автоудаление дубликатов        |
|Быстрый поиск          |**set** или **dict**  |O(1) вместо O(n)               |
|Пары ключ-значение     |**dict**              |Очевидный выбор                |
|Очередь (FIFO)         |**deque**             |O(1) с обоих концов            |
|Стек (LIFO)            |**list** или **deque**|list проще, deque быстрее      |
|Числовой массив        |**array**             |Экономия памяти                |
|Подсчет элементов      |**Counter**           |Специализирован для этого      |
|Группировка данных     |**defaultdict**       |Избегает KeyError              |
|История с лимитом      |**deque(maxlen=n)**   |Автоудаление старых            |

## ⚠️ ЧАСТЫЕ ОШИБКИ

### 1. Использование list для проверки наличия

```python
# ❌ ПЛОХО - O(n)
items = [1, 2, 3, 4, 5]
if 100 in items:  # Медленно для больших списков
    print("Found")

# ✅ ХОРОШО - O(1)
items = {1, 2, 3, 4, 5}  # set
if 100 in items:  # Быстро!
    print("Found")
```

### 2. Частые вставки в начало list

```python
# ❌ ПЛОХО - O(n) для каждой вставки
items = []
for i in range(1000):
    items.insert(0, i)  # Очень медленно!

# ✅ ХОРОШО - O(1) для каждой вставки
from collections import deque
items = deque()
for i in range(1000):
    items.appendleft(i)  # Быстро!
```

### 3. Использование list для больших числовых данных

```python
# ❌ ПЛОХО - много памяти
numbers = [1, 2, 3, 4, 5] * 1000000  # ~40 MB

# ✅ ХОРОШО - мало памяти
from array import array
numbers = array('i', [1, 2, 3, 4, 5] * 1000000)  # ~20 MB
```

### 4. Изменение list во время итерации

```python
# ❌ ПЛОХО - пропускает элементы
items = [1, 2, 3, 4, 5]
for item in items:
    if item % 2 == 0:
        items.remove(item)  # Опасно!

# ✅ ХОРОШО
items = [item for item in items if item % 2 != 0]
```

### 5. Использование dict вместо list

```python
# ❌ ПЛОХО - зачем словарь?
items = {0: 'a', 1: 'b', 2: 'c'}

# ✅ ХОРОШО
items = ['a', 'b', 'c']
```

## 💡 ПРАКТИЧЕСКИЕ СОВЕТЫ

1. **По умолчанию используй list** - подходит для 80% задач
2. **Нужна скорость поиска?** → set или dict
3. **Данные не меняются?** → tuple (быстрее и безопаснее)
4. **Много операций в начале?** → deque
5. **Большие числовые данные?** → array или numpy
6. **Нужны уникальные элементы?** → set
7. **Связь ключ-значение?** → dict
8. **Подсчет частоты?** → Counter
9. **Не уверен?** → начни с list, оптимизируй потом

## 📚 ДОПОЛНИТЕЛЬНЫЕ КОЛЛЕКЦИИ

### Counter

```python
from collections import Counter
c = Counter(['a', 'b', 'a', 'c', 'b', 'a'])
# Counter({'a': 3, 'b': 2, 'c': 1})
```

**Когда:** подсчет частоты элементов

### defaultdict

```python
from collections import defaultdict
d = defaultdict(list)
d['key'].append(1)  # Не нужна проверка
```

**Когда:** группировка, избежание KeyError

### OrderedDict

```python
from collections import OrderedDict
d = OrderedDict()
d.move_to_end('key')  # Дополнительные методы
```

**Когда:** нужны методы перемещения (обычно dict достаточно)

### namedtuple

```python
from collections import namedtuple
Point = namedtuple('Point', ['x', 'y'])
p = Point(10, 20)
```

**Когда:** простые структуры данных, читаемость

### ChainMap

```python
from collections import ChainMap
combined = ChainMap(dict1, dict2)
```

**Когда:** объединение нескольких словарей без копирования
