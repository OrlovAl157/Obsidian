> [!warning] ⚠️ Поля с `default` должны идти в конце
>
> Dataclass при наследовании собирает поля:
>
> ```text
> родитель → потомок
> ```
>
> а затем генерирует ОДИН общий `__init__`.
>
> Поэтому итоговая сигнатура должна подчиняться обычным правилам Python:
>
> ```text
> обычные аргументы
> ↓
> аргументы со значением по умолчанию
> ```

```python
from dataclasses import dataclass

@dataclass
class Animal:
    name: str
    age: int = 0

@dataclass
class Dog(Animal):
    breed: str      # ❌ Ошибка
```

---

## 🧠 Что происходит под капотом

Dataclass берёт поля `Animal`:

```text
name
age=0
```

затем добавляет поля `Dog`:

```text
breed
```

---

## 📌 Итоговый порядок полей

```text
name
age=0
breed
```

---

## ❌ Какой `__init__` пытается создать dataclass

```python
def __init__(self,
             name,
             age=0,
             breed):
```

---

> [!failure] Почему ошибка?
>
> Python запрещает:
>
> ```python
> def f(x=10, y):
> ```
>
> потому что обязательный аргумент
> не может идти после аргумента
> со значением по умолчанию.

---

## 💥 Поэтому возникает

```python
TypeError:
non-default argument 'breed'
follows default argument
```

---

# ✅ Правильно

```python
from dataclasses import dataclass

@dataclass
class Animal:
    name: str
    age: int = 0

@dataclass
class Dog(Animal):
    breed: str = "Unknown"
```

---

## 📌 Теперь dataclass сможет создать

```python
def __init__(self,
             name,
             age=0,
             breed="Unknown"):
```

✅ Такой порядок уже валиден.

---

> [!tip] Главное правило
>
> После объединения ВСЕХ полей inheritance:
>
> ```text
> non-default
> ↓
> default
> ```
>
> ✅ допустимо
>
> ---
>
> ```text
> default
> ↓
> non-default
> ```
>
> ❌ ошибка
