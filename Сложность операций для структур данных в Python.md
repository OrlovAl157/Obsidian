
## 💡 Полезные советы

- **O(1)** - константное время, не зависит от размера данных (БЫСТРО)
- **O(n)** - линейное время, пропорционально размеру данных (МЕДЛЕННО для больших данных)
- **O(log n)** - логарифмическое время (БЫСТРО даже для больших данных)
- **Хеширование**: set и dict используют хеш-таблицы для O(1) операций
- **Амортизированное O(1)**: в среднем O(1), но иногда может быть O(n)
- **Индексация**: list и tuple быстрые, set и dict не поддерживают
- **Память**: tuple компактнее list, linked list требует больше памяти
- **Выбор структуры**: зависит от частоты операций, не размера данных

## 🧾 Легенда

### Обозначения скорости

- 🟢 **O(1)** — константное время, **БЫСТРО** (не зависит от размера)
- 🟡 **O(n), O(log n)** — линейное/логарифмическое, **СРЕДНЕ**
- 🔴 **O(n), O(n²)** — **МЕДЛЕННО** для больших данных, или **НЕВОЗМОЖНО** (—)

### Примечания

- `*` **N/A** — операция не поддерживается (set и dict не имеют индексов)
- `**` **Хеширование** — операции set и dict используют хеш-таблицы:
    - В **среднем** O(1)
    - В **худшем случае** может быть O(n) при коллизиях
- `***` **Амортизированное O(1)** — `list.append()`:
    - Обычно O(1)
    - Редко O(n) при перераспределении памяти
- `****` **Если есть ссылка** — для linked list нужна ссылка на конец
- `*****` **O(k)** — где k = длина среза (b - a)

## 💬 Когда использовать какую структуру

|Цель|Оптимальная структура|Почему|
|---|---|---|
|Быстрый доступ по индексу|`list`, `tuple`|O(1) доступ|
|Частое добавление/удаление в конец|`list`, `deque`|O(1) операции|
|Частое добавление/удаление в начало|`deque`|O(1) vs O(n) для list|
|Проверка наличия элемента|`set`, `dict`|O(1) vs O(n) для list|
|Уникальные элементы|`set`|Автоматически убирает дубликаты|
|Пары ключ–значение|`dict`|O(1) доступ по ключу|
|Неизменяемая последовательность|`tuple`|Защита от изменений, меньше памяти|
|FIFO очередь|`deque`|O(1) для обоих концов|
|LIFO стек|`list`, `deque`|O(1) для добавления/удаления|
|Сортированные данные|`list` + `.sort()`|Встроенная быстрая сортировка|
|Приоритетная очередь|`heapq` + `list`|O(log n) операции|

## 📊 Детальная сложность операций

### list (Список)

```python
# ✅ БЫСТРЫЕ операции O(1)
lst[i]                          # Доступ по индексу
lst.append(x)                   # Добавление в конец (амортизированное)
lst.pop()                       # Удаление из конца
len(lst)                        # Длина

# ⚠️ МЕДЛЕННЫЕ операции O(n)
x in lst                        # Поиск элемента
lst.insert(0, x)                # Вставка в начало
lst.pop(0)                      # Удаление из начала
lst.remove(x)                   # Удаление по значению
lst.index(x)                    # Поиск индекса
min(lst), max(lst)              # Минимум/максимум

# 🟡 СРЕДНИЕ операции
lst.sort()                      # Сортировка O(n log n)
sorted(lst)                     # Новый отсортированный O(n log n)
lst[a:b]                        # Срез O(k), где k = b-a
lst1 + lst2                     # Объединение O(n+m)
```

### tuple (Кортеж)

```python
# ✅ БЫСТРЫЕ операции O(1)
tpl[i]                          # Доступ по индексу
len(tpl)                        # Длина

# ⚠️ МЕДЛЕННЫЕ операции O(n)
x in tpl                        # Поиск элемента
tpl.index(x)                    # Поиск индекса
min(tpl), max(tpl)              # Минимум/максимум

# 🟡 СРЕДНИЕ операции
tpl[a:b]                        # Срез O(k)
tpl1 + tpl2                     # Объединение O(n+m)

# 🔴 НЕВОЗМОЖНЫЕ операции
# tpl.append(x)                 # Неизменяемый!
# tpl.remove(x)                 # Неизменяемый!
```

### set (Множество)

```python
# ✅ БЫСТРЫЕ операции O(1) в среднем
x in s                          # Проверка наличия
s.add(x)                        # Добавление элемента
s.remove(x)                     # Удаление (KeyError если нет)
s.discard(x)                    # Удаление (без ошибки)
len(s)                          # Длина

# 🟡 СРЕДНИЕ операции O(n)
s1 | s2                         # Объединение
s1 & s2                         # Пересечение
s1 - s2                         # Разность
min(s), max(s)                  # Минимум/максимум

# 🔴 НЕВОЗМОЖНЫЕ операции
# s[0]                          # Нет индексов!
```

### dict (Словарь)

```python
# ✅ БЫСТРЫЕ операции O(1) в среднем
d[key]                          # Доступ по ключу
d.get(key)                      # Безопасный доступ
d[key] = value                  # Установка значения
del d[key]                      # Удаление по ключу
key in d                        # Проверка наличия ключа
len(d)                          # Длина

# 🟡 СРЕДНИЕ операции O(n)
d.keys()                        # Все ключи (view)
d.values()                      # Все значения (view)
d.items()                       # Все пары (view)
d1 | d2                         # Объединение (Python 3.9+)

# 🔴 НЕВОЗМОЖНЫЕ операции
# d[0]                          # Нет числовых индексов!
```

### deque (Двусторонняя очередь)

```python
from collections import deque

# ✅ БЫСТРЫЕ операции O(1)
dq.append(x)                    # Добавление в конец
dq.appendleft(x)                # Добавление в начало
dq.pop()                        # Удаление из конца
dq.popleft()                    # Удаление из начала
len(dq)                         # Длина

# ⚠️ МЕДЛЕННЫЕ операции O(n)
dq[i]                           # Доступ по индексу (медленнее list!)
x in dq                         # Поиск элемента
dq.remove(x)                    # Удаление по значению
```

## 🎯 Практические рекомендации

### Выбор структуры по задаче

```python
# ========== ЗАДАЧА: Быстрый поиск элемента ==========
# ❌ ПЛОХО: список
numbers = [1, 2, 3, ..., 10000]
if 9999 in numbers:             # O(n) - медленно!
    pass

# ✅ ХОРОШО: множество
numbers = {1, 2, 3, ..., 10000}
if 9999 in numbers:             # O(1) - быстро!
    pass

# ========== ЗАДАЧА: Частое добавление в начало ==========
# ❌ ПЛОХО: список
queue = []
queue.insert(0, x)              # O(n) - медленно!

# ✅ ХОРОШО: deque
from collections import deque
queue = deque()
queue.appendleft(x)             # O(1) - быстро!

# ========== ЗАДАЧА: Удаление дубликатов ==========
# ❌ ПЛОХО: вручную через список
items = [1, 2, 2, 3, 3, 3]
unique = []
for item in items:
    if item not in unique:      # O(n) для каждого элемента!
        unique.append(item)

# ✅ ХОРОШО: через множество
unique = list(set(items))       # O(n) для всего!

# ========== ЗАДАЧА: Счётчик элементов ==========
# ❌ ПЛОХО: список + count
items = ['a', 'b', 'a', 'c', 'b', 'a']
count_a = items.count('a')      # O(n) для каждого подсчёта!

# ✅ ХОРОШО: Counter
from collections import Counter
counter = Counter(items)
count_a = counter['a']          # O(1)

# ========== ЗАДАЧА: Кэш с ограниченным размером ==========
# ❌ ПЛОХО: обычный dict
cache = {}
# Нужно вручную отслеживать размер

# ✅ ХОРОШО: OrderedDict или lru_cache
from collections import OrderedDict
cache = OrderedDict()
# или
from functools import lru_cache
@lru_cache(maxsize=100)
def expensive_func(x):
    pass
```

### Оптимизация производительности

```python
# ========== ПРОВЕРКА НАЛИЧИЯ ==========
# Если нужно часто проверять наличие - используйте set/dict

# Много проверок в списке - МЕДЛЕННО
items = [1, 2, 3, ..., 10000]
for i in range(10000):
    if i in items:              # O(n) каждый раз = O(n²) всего!
        pass

# Преобразовать в set один раз - БЫСТРО
items_set = set(items)          # O(n) один раз
for i in range(10000):
    if i in items_set:          # O(1) каждый раз = O(n) всего!
        pass

# ========== НАКОПЛЕНИЕ ДАННЫХ ==========
# Много добавлений в начало - используйте deque

# Список - МЕДЛЕННО
result = []
for item in data:
    result.insert(0, item)      # O(n) каждый раз!

# Deque - БЫСТРО
from collections import deque
result = deque()
for item in data:
    result.appendleft(item)     # O(1) каждый раз!

# ========== УДАЛЕНИЕ ИЗ СПИСКА ==========
# Много удалений - создайте новый список

# ❌ ПЛОХО: удалять в цикле
items = [1, 2, 3, 4, 5]
for item in items[:]:           # Копия нужна!
    if item % 2 == 0:
        items.remove(item)      # O(n) для каждого!

# ✅ ХОРОШО: list comprehension
items = [item for item in items if item % 2 != 0]  # O(n) один раз!
```

## ⚠️ Частые ошибки

```python
# ❌ Использование списка для частых проверок наличия
seen = []                       # ПЛОХО для проверок!
for item in data:
    if item not in seen:        # O(n) каждый раз!
        seen.append(item)

# ✅ Используйте set
seen = set()                    # ХОРОШО!
for item in data:
    if item not in seen:        # O(1) в среднем
        seen.add(item)

# ❌ Удаление из начала списка в цикле
queue = [1, 2, 3, 4, 5]
while queue:
    item = queue.pop(0)         # O(n) каждый раз!
    process(item)

# ✅ Используйте deque
from collections import deque
queue = deque([1, 2, 3, 4, 5])
while queue:
    item = queue.popleft()      # O(1)
    process(item)

# ❌ Изменение списка во время итерации
items = [1, 2, 3, 4, 5]
for item in items:
    if item % 2 == 0:
        items.remove(item)      # ОШИБКА! Пропускает элементы

# ✅ Итерируйте по копии или создайте новый список
items = [item for item in items if item % 2 != 0]
```

## 🎓 Резюме

### Памятка по выбору структуры

|Если вам нужно...|Используйте|
|---|---|
|Быстрый доступ по индексу|`list` или `tuple`|
|Быстрая проверка "есть ли элемент"|`set` или `dict`|
|Хранить уникальные значения|`set`|
|Связать ключи со значениями|`dict`|
|Часто добавлять/удалять с обоих концов|`deque`|
|FIFO очередь|`deque`|
|LIFO стек|`list` (append/pop)|
|Неизменяемая коллекция|`tuple` или `frozenset`|
|Подсчёт элементов|`Counter`|
|Сохранить порядок добавления в dict|`dict` (Python 3.7+) или `OrderedDict`|

### Золотое правило

**Выбирайте структуру данных исходя из того, какие операции будете выполнять чаще всего, а не из размера данных!**
