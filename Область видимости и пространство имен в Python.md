

## Введение

Область видимости и пространство имен — две фундаментальные концепции Python, которые часто путают начинающие разработчики. Хотя они связаны с управлением идентификаторами в коде, их назначение и принципы работы существенно различаются.

-----

## Область видимости (Scope) в Python

### Определение

**Область видимости** — это часть программы, в которой переменная, функция или другой идентификатор доступны для использования. Она определяет время жизни и доступность элементов кода.

### Правило LEGB в Python

Python использует правило **LEGB** для поиска переменных:

- **L (Local)** — локальная область (внутри функции)
- **E (Enclosing)** — область внешней функции (для вложенных функций)
- **G (Global)** — глобальная область (уровень модуля)
- **B (Built-in)** — встроенная область (встроенные функции Python)

### Примеры областей видимости

#### Локальная область видимости

```python
def my_function():
    x = 10  # локальная переменная
    print(x)  # 10

my_function()
print(x)  # NameError: name 'x' is not defined
```

#### Глобальная область видимости

```python
x = 10  # глобальная переменная

def my_function():
    print(x)  # доступна внутри функции

my_function()  # 10
print(x)  # 10
```

#### Ключевое слово global

```python
counter = 0  # глобальная переменная

def increment():
    global counter  # указываем, что используем глобальную
    counter += 1
    print(f"Счётчик: {counter}")

increment()  # Счётчик: 1
increment()  # Счётчик: 2
print(counter)  # 2
```

#### Вложенные функции (Enclosing scope)

```python
def outer():
    x = "внешняя"
    
    def inner():
        print(x)  # доступна из внешней функции
    
    inner()

outer()  # внешняя
```

#### Ключевое слово nonlocal

```python
def outer():
    count = 0
    
    def inner():
        nonlocal count  # изменяем переменную из внешней функции
        count += 1
        print(f"Count: {count}")
    
    inner()  # Count: 1
    inner()  # Count: 2
    print(f"Final: {count}")  # Final: 2

outer()
```

### Встроенная область видимости

```python
# len, print, str и др. — встроенные функции Python
print(len("Hello"))  # 5

# Можно перекрыть встроенную функцию (но не рекомендуется!)
len = 10
print(len)  # 10
# print(len("Hello"))  # TypeError: 'int' object is not callable
```

### Перекрытие переменных (Shadowing)

```python
x = "глобальная"

def test():
    x = "локальная"  # перекрывает глобальную
    print(x)

test()  # локальная
print(x)  # глобальная
```

## Пространство имен (Namespace) в Python

### Определение

**Пространство имен** — это словарь, который связывает имена с объектами. В Python всё является объектом, и пространства имен — это способ организации этих объектов.

### Типы пространств имен в Python

**Встроенное пространство имен** содержит встроенные функции и исключения (len, print, ValueError и т.д.). Создается при запуске интерпретатора Python.

**Глобальное пространство имен** содержит имена на уровне модуля. Создается при импорте модуля или запуске скрипта.

**Локальное пространство имен** содержит имена внутри функции. Создается при вызове функции, удаляется после завершения.

### Просмотр пространств имен

```python
# Встроенное пространство имен
import builtins
print(dir(builtins))  # список всех встроенных имен

# Глобальное пространство имен
x = 10
y = 20

def func():
    pass

print(globals())  # словарь глобальных переменных
# {'x': 10, 'y': 20, 'func': <function func>, ...}

# Локальное пространство имен
def my_function():
    a = 1
    b = 2
    print(locals())  # {'a': 1, 'b': 2}

my_function()
```

### Модули как пространства имен

```python
# Файл: math_utils.py
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

# Файл: string_utils.py
def add(a, b):
    return a + b  # конкатенация строк

# Основной файл
import math_utils
import string_utils

print(math_utils.add(2, 3))        # 5
print(string_utils.add("Hi", "!"))  # Hi!
```

### Импорт и пространства имен

```python
# Импорт модуля целиком
import math
print(math.sqrt(16))  # 4.0

# Импорт конкретных имен
from math import sqrt, pi
print(sqrt(16))  # 4.0
print(pi)  # 3.141592653589793

# Импорт с алиасом
import numpy as np
array = np.array([1, 2, 3])

# Импорт всего (не рекомендуется!)
from math import *
print(sqrt(16))  # работает, но засоряет пространство имен
```

### Классы и пространства имен

```python
class Person:
    species = "Homo sapiens"  # атрибут класса
    
    def __init__(self, name):
        self.name = name  # атрибут экземпляра
    
    def greet(self):
        return f"Привет, я {self.name}"

# Доступ к пространству имен класса
print(Person.__dict__)  # все атрибуты класса

person = Person("Иван")
print(person.__dict__)  # {'name': 'Иван'}
print(person.greet())   # Привет, я Иван
```

### **name** и **main**

```python
# module.py
def function():
    print("Функция вызвана")

print(f"Имя модуля: {__name__}")

if __name__ == "__main__":
    print("Запущен как главный скрипт")
    function()
else:
    print("Импортирован как модуль")
```

## Сравнение

|Аспект             |Область видимости           |Пространство имен                      |
|-------------------|----------------------------|---------------------------------------|
|**Основной вопрос**|Где доступна переменная?    |Как организованы имена?                |
|**Что это**        |Правила доступа к переменным|Словарь имен и объектов                |
|**Цель**           |Управление доступностью     |Организация и предотвращение конфликтов|
|**Функции Python** |globals(), locals()         |dir(), **dict**, **name**              |
|**Правило**        |LEGB                        |Иерархия модулей и классов             |
|**Время жизни**    |Динамическое (runtime)      |От создания до удаления                |

## Взаимодействие концепций

```python
# Пример взаимодействия областей видимости и пространств имен

import math  # math — это пространство имен (модуль)

x = 100  # глобальная область, глобальное пространство имен

def calculate(number):
    # локальная область видимости
    # локальное пространство имен функции
    
    result = math.sqrt(number)  # обращение к пространству имен math
    multiplier = 2  # локальная переменная
    
    def inner():
        # вложенная локальная область
        # enclosing для inner — это calculate
        return result * multiplier * x
    
    return inner()

print(calculate(16))  # 800.0

# Проверка пространств имен
print("Глобальные имена:", list(globals().keys())[-5:])
```

## Практические примеры

### Пример 1: Счётчик с замыканием

```python
def create_counter():
    count = 0  # enclosing scope
    
    def increment():
        nonlocal count
        count += 1
        return count
    
    return increment

counter1 = create_counter()
counter2 = create_counter()

print(counter1())  # 1
print(counter1())  # 2
print(counter2())  # 1 (независимый счётчик)
```

### Пример 2: Избежание конфликтов имен

```python
# utils/math_ops.py
def calculate(x, y):
    return x + y

# utils/string_ops.py
def calculate(x, y):
    return f"{x}{y}"

# main.py
from utils import math_ops
from utils import string_ops

print(math_ops.calculate(5, 3))      # 8
print(string_ops.calculate("5", "3"))  # 53
```

### Пример 3: Декораторы и области видимости

```python
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"Вызов функции: {func.__name__}")
        result = func(*args, **kwargs)
        print(f"Результат: {result}")
        return result
    return wrapper

@logger
def add(a, b):
    return a + b

add(3, 5)
# Вызов функции: add
# Результат: 8
```

## Распространённые ошибки

### Ошибка 1: Изменение глобальной без global

```python
counter = 0

def increment():
    counter += 1  # UnboundLocalError!

# Правильно:
def increment():
    global counter
    counter += 1
```

### Ошибка 2: Перекрытие встроенных имен

```python
# Плохо
list = [1, 2, 3]
print(list)  # работает
# list([1, 2, 3])  # TypeError! list теперь не функция

# Хорошо
my_list = [1, 2, 3]
```

### Ошибка 3: Импорт *

```python
# Плохо
from math import *
from numpy import *  # может перекрыть имена из math

# Хорошо
import math
import numpy as np
```

## Практические рекомендации

### Для областей видимости

Избегайте глобальных переменных где возможно. Используйте аргументы функций для передачи данных. Применяйте nonlocal осторожно. Помните о правиле LEGB. Используйте явные имена переменных.

### Для пространств имен

Организуйте код в модули логически. Используйте явные импорты вместо *. Применяйте алиасы для длинных имен модулей. Избегайте перекрытия встроенных имен. Используйте **name** == “**main**” для исполняемых скриптов.

-----

## Заключение

В Python **область видимости** определяет, где переменная доступна (правило LEGB), а **пространство имен** — это словарь, связывающий имена с объектами.

Понимание этих концепций критически важно для:

- Написания чистого и понятного кода
- Избежания конфликтов имен
- Правильной организации проектов
- Эффективного использования модулей и пакетов
- Понимания замыканий и декораторов

Эти концепции работают вместе, обеспечивая гибкую и мощную систему управления именами в Python.
