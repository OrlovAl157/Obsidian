# 🏷️ Основы типизации в Python

## 📌 Коротко

> **Type hints (аннотации типов)** — это синтаксис для явного указания ожидаемых типов переменных, параметров функций и возвращаемых значений. Это не обязанность, а возможность, которая помогает писать более понятный и надёжный код.

Python остаётся динамически типизированным языком — аннотации никак не влияют на выполнение программы. Но они помогают IDE, статическим анализаторам (mypy, pyright) и другим разработчикам понять, какие типы данных ожидаются в коде.

-----

## 💡 Полезные советы

- **Type hints не обязательны** — это подсказки, не требования
- **Начните с сигнатур функций** — это даст максимальную пользу
- **Используйте type checker** — mypy или pyright поймут ошибки до запуска
- **IDE подсвечивает ошибки типов** — в VSCode, PyCharm это встроено
- **Аннотации помогают при рефакторинге** — безопаснее менять код
- **Документируйте сложные типы** — комментарии плюс типы = идеально

-----

## 🚀 Быстрая справка

|Что                                  |Синтаксис            |Пример                               |
|-------------------------------------|---------------------|-------------------------------------|
|**Переменная**                       |`var: Type = value`  |`age: int = 25`                      |
|**Параметр функции**                 |`def func(x: Type)`  |`def add(a: int, b: int)`            |
|**Возвращаемое значение**            |`def func() -> Type:`|`def add(a: int) -> int:`            |
|**Параметр с значением по умолчанию**|`x: Type = default`  |`name: str = 'Alice'`                |
|**Никакое значение (None)**          |`-> None`            |`def print_msg() -> None:`           |
|**Без аннотации**                    |просто напиши код    |`x = 5` (тип выводится автоматически)|

## 📖 Теория

### История и зачем нужны аннотации

В Python 3.5 (PEP 484) появились первые аннотации типов. В 3.6 добавили возможность аннотировать переменные везде (PEP 526). С каждой версией это становится проще и удобнее.

**Зачем они нужны?**

1. **Самодокументирование кода** — видно сразу, какие типы ожидаются
1. **IDE помогает лучше** — автодополнение понимает типы объектов
1. **Меньше ошибок** — статические анализаторы ловят проблемы до запуска
1. **Рефакторинг безопаснее** — когда меняешь код, видно что сломается
1. **Командная разработка** — другим разработчикам проще понять твой код

**Важно:** аннотации — это только подсказки! Python их не проверяет при выполнении.

```python
def greet(name: str) -> str:
    return f'Hello, {name}!'

greet(123)  # ❌ Type checker скажет "ошибка!"
            # ✅ Но Python выполнит это нормально
```

### Простые типы в функциях

Самый очевидный способ использовать аннотации — добавить типы к параметрам и возвращаемому значению функции:

```python
def multiply(x: int, y: int) -> int:
    return x * y

def greet(name: str) -> str:
    return f'Hello, {name}!'

def check_even(num: int) -> bool:
    return num % 2 == 0
```

**Синтаксис:**

- Тип параметра указывается после имени через `:` → `name: str`
- Тип возвращаемого значения указывается после `->` → `-> int`

### Параметры с значениями по умолчанию

Когда у параметра есть значение по умолчанию, аннотация указывается **между именем и значением**:

```python
def greet(name: str = 'world') -> str:
    return f'Hello, {name}!'

def power(x: int, exponent: int = 2) -> int:
    return x ** exponent

def configure(debug: bool = False, timeout: int = 30) -> None:
    print(f'Debug: {debug}, Timeout: {timeout}')
```

### Аннотирование переменных (Python 3.6+)

Можно аннотировать переменные в любом месте программы:

```python
# С присваиванием значения
name: str = 'Alice'
age: int = 25
height: float = 1.75
is_active: bool = True
nothing: None = None

# Без значения (только аннотация)
username: str
password: str
```

Обычно переменные не аннотируют, если значение явное (понятно из контекста). Но полезно для:

- Пустых списков: `users: list[str] = []`
- Результатов функций без аннотаций: `data: dict = external_func()`
- Явного указания намерения

### Встроенные базовые типы

```python
# Числа
count: int = 42
price: float = 19.99
result: complex = 1 + 2j

# Строки и байты
message: str = 'hello'
data: bytes = b'data'

# Логические значения
is_valid: bool = True

# Ничего (отсутствие значения)
def cleanup() -> None:
    print('Cleaning up')
    # Возвращает None неявно
```

## 💻 Примеры

### Пример 1: Простые функции

```python
def add(a: int, b: int) -> int:
    """Сумма двух чисел"""
    return a + b

def divide(dividend: float, divisor: float) -> float:
    """Деление с результатом типа float"""
    if divisor == 0:
        raise ValueError('Divisor cannot be zero')
    return dividend / divisor

def is_palindrome(text: str) -> bool:
    """Проверка, является ли строка палиндромом"""
    clean = text.lower().replace(' ', '')
    return clean == clean[::-1]

# Использование
result = add(10, 20)           # 30
quotient = divide(10.5, 2.5)   # 4.2
check = is_palindrome('racecar')  # True
```

### Пример 2: Функции с несколькими типами возвращаемых значений

Если функция может вернуть разные типы, будем использовать `Union` (см. следующий файл). Пока простой пример:

```python
def get_value() -> str | None:  # Python 3.10+
    """Может вернуть строку или None"""
    if some_condition:
        return 'value'
    return None

def find_user(user_id: int) -> dict:
    """Возвращает словарь с данными пользователя"""
    return {'id': user_id, 'name': 'Alice', 'email': 'alice@example.com'}
```

### Пример 3: Аннотирование переменных в коде

```python
# Пустые контейнеры — аннотируем, чтобы было ясно
numbers: list[int] = []
user_ages: dict[str, int] = {}
active_tags: set[str] = set()

# Результаты от функций без типов (из старого кода или библиотек)
config: dict = load_config()  # Функция не имеет аннотаций
data: list = external_api.fetch_data()

# Явное намерение
MAX_USERS: int = 100
API_KEY: str = 'secret123'
DEBUG_MODE: bool = False
```

### Пример 4: Функция с None в параметре

```python
def greet_optional(name: str | None = None) -> str:
    """Приветствие, где имя опционально"""
    if name is None:
        return 'Hello, stranger!'
    return f'Hello, {name}!'

# Использование
print(greet_optional())          # 'Hello, stranger!'
print(greet_optional('Alice'))   # 'Hello, Alice!'
```

## ⚠️ Частые ошибки

```python
# ❌ Забыли стрелку для возвращаемого значения
def add(a: int, b: int) int:  # SyntaxError!
    return a + b

# ✅ Правильно
def add(a: int, b: int) -> int:
    return a + b

# ❌ Неправильный порядок (аннотация после значения по умолчанию)
def greet(name = 'world': str) -> str:  # SyntaxError!
    return f'Hello, {name}!'

# ✅ Правильно
def greet(name: str = 'world') -> str:
    return f'Hello, {name}!'

# ❌ Забыли None для функции без возврата
def print_msg(msg: str):  # Неявный возврат
    print(msg)

# ✅ Правильно
def print_msg(msg: str) -> None:
    print(msg)

# ❌ Думаешь, что аннотация проверяется при выполнении
def add(a: int, b: int) -> int:
    return a + b

add('hello', 'world')  # ❌ Type checker скажет ошибка
                       # ✅ Но Python выполнит: 'helloworld'
```

## 🎯 Главные правила

✅ **Начните с функций** — аннотируйте сигнатуры функций в первую очередь  
✅ **Используйте type checker** — mypy или pyright помогут ловить ошибки  
✅ **Аннотации не влияют на выполнение** — это только для анализа и читаемости  
✅ **Не переусложняйте сразу** — добавляйте типы постепенно  
✅ **None для функций без возврата** — это важно для анализа  
✅ **Комментарии + типы = идеально** — вместе они документируют код лучше  
✅ **IDE благодарна за типы** — автодополнение становится намного лучше
